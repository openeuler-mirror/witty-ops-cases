#标题
Block层blk_mq_dispatch_rq_list逻辑变更导致I/O挂起

#内核版本
5.7 及以上版本（包括 5.11）

#问题现象
系统运行过程中发生 I/O 挂起，内核日志报错显示任务阻塞超过 120 秒。在 Google Compute Engine VM 环境下，执行涉及大量小文件读写的操作（如循环重装 google-cloud-sdk 软件包）30-60 分钟后可复现该故障。

#问题根因
经 Git bisect 定位，问题源于 commit 536167d47ade。该提交修改了 `block/blk-mq.c` 文件中的 `blk_mq_dispatch_rq_list` 函数，将条件判断 `if (q->mq_ops->commit_rqs)` 更改为 `if (q->mq_ops->commit_rqs && queued)`。此修改导致在特定场景下 `commit_rqs` 回调未被正确调用，从而引发 I/O 流程停滞。

#解决方案
暂无官方合并修复。临时规避方案为回退 commit 536167d47ade 的代码修改，恢复原有的判断逻辑。
