#标题
KVM虚拟机关机后AMD GPU直通设备PCI Header损坏

#内核版本
5.4.26, 5.5.11, 5.6-rc7 (5.3.5 版本未出现此问题)

#问题现象
- 宿主机通过 vfio-pci 将 AMD RX Vega 64 显卡直通给 QEMU 虚拟机使用。
- 虚拟机关机或重启后，宿主机上的显卡出现 PCI Header 损坏，执行 `lspci -v` 显示 "Unknown header type 7f"。
- 显卡风扇停转（RPM 0），随后可能飙升至 100%，需对主机断电冷却后才能恢复。
- 内核日志显示设备状态异常，设备无法正常复位。

#问题根因
宿主机内核加载的 `amdgpu` 或 `radeon` 模块在虚拟机关机过程中干扰了已绑定到 `vfio-pci` 的直通设备，导致设备配置空间（Header）损坏或状态异常。

#解决方案
临时规避方案：在宿主机上将 `amdgpu` 和 `radeon` 内核模块加入黑名单，避免宿主机驱动接管直通设备。目前 Bug 状态为 NEW，暂无上游修复补丁。
