#标题
r8152驱动运行时电源管理死锁导致网络子系统挂起

#内核版本
5.3.7（受影响版本包括 5.2.x 至 5.15.x）

#问题现象
- 使用 Dell USB-C 扩展坞（WD15/TB16）内置 Realtek 网卡时，系统网络相关功能完全挂起。
- 内核日志报错 "INFO: task kworker blocked for more than 122 seconds"，进程处于 D 状态（Uninterruptible Sleep）。
- 堆栈跟踪显示阻塞发生在 `rtl_work_func_t [r8152]` -> `usb_autopm_get_interface` -> `rpm_resume` 调用路径中。
- NetworkManager 等进程在 `rtnetlink_rcv_msg` 处因 `__mutex_lock` 阻塞，导致系统无法正常关机，需使用 SysRQ 强制重启。
- 部分情况下出现 "NETDEV WATCHDOG: transmit queue 0 timed out" 告警。

#问题根因
`r8152` 驱动程序在运行时电源管理流程中存在缺陷。当驱动工作队列函数（如 `rtl_work_func_t` 或 `rtl_hw_phy_work_func_t`）调用 `usb_autopm_get_interface` 尝试恢复设备电源时，在 `rpm_resume` 内部发生死锁或长时间阻塞，导致相关内核线程无法继续执行，进而阻塞网络子系统。

#解决方案
该问题已在 Linux 内核 5.13.9 版本中通过 commit `776ac63a98` 修复。建议升级内核至 5.13.9 或更高版本。若无法升级，临时规避方案为将 `r8152` 模块加入黑名单（将导致该网卡不可用）。
