#标题
内核执行 ext4 写操作时触发 soft lockup 导致 CPU 卡死

#内核版本
4.0.5-300.fc22.x86_64+debug

#问题现象
- 系统日志报告 "NMI watchdog: BUG: soft lockup - CPU#1 stuck for 22s!"
- 故障进程为 teamviewerd，触发时 CPU 处于中断上下文切换状态
- Call Trace 指向 copy_user_enhanced_fast_string 函数

#问题根因
未明确。根据堆栈信息，故障发生在 `ext4_file_write_iter` 调用链中，具体位于 `generic_perform_write` -> `iov_iter_copy_from_user_atomic` -> `copy_user_enhanced_fast_string` 路径。评论中曾推测可能与 `hda_intel.c` 中的 `mark_runtime_wc` 函数有关，但未得到验证；亦有观点认为该 Bug 归属的 Watchdog 组件可能不准确。

#解决方案
暂无。Bug 状态为 NEW，尚未修复。
