#标题
VirtualBox未声明USB接口引发内核日志刷屏导致系统资源耗尽

#内核版本
4.7.0-1-amd64

#问题现象
- 连接 Nokia S60 手机至 VirtualBox 虚拟机时，宿主机 CPU 和内存占用率瞬间飙升至 100%，系统出现磁盘空间错误并变得无响应。
- 内核日志（kern.log/messages/syslog）被大量 "usbfs: process 'x' (events) did not claim interface 0 before use" 错误信息刷屏。
- 断开手机连接后系统恢复正常。

#问题根因
VirtualBox 使用了不受支持的内核驱动程序，其进程在操作 USB 设备时未遵循正确的接口声明流程，导致内核持续输出错误日志。日志刷屏操作消耗了大量 I/O 和 CPU 资源，最终导致系统资源耗尽和无响应。

#解决方案
该问题由 VirtualBox 驱动实现不当引起，非 Linux 内核 Bug。需向 VirtualBox 开发团队反馈修复其驱动逻辑。
