#标题
NVMe 并发执行 reset_controller 与 format ioctl 导致 Kernel Panic

#内核版本
5.4 (Maybe earlier)-5.20

#问题现象
在循环并发执行 `reset_controller` 与 `nvme format` 命令时，系统发生 Kernel Panic 或 Mutex 超时。调试信息显示故障发生在 `nvme_cancel_admin_tagset()` 函数调用期间。

#问题根因
存在死锁条件：passthrough 请求（format ioctl）试图在释放自身前刷新 `scan_work`，但 `scan_work` 因管理队列已处于静默状态而无法推进；同时 `reset_work` 正在等待 passthrough 请求转换为空闲状态，导致双方互相等待引发死锁。

#解决方案
暂无（截至报告最后更新，Bug 状态为 NEW，未提及已合并的修复补丁）。
