#标题
USB Audio设备使用中报错cannot submit urb导致系统挂起

#内核版本
v3.6-rc6-25-g4651afb（评论提及 3.2.0-3 和 3.16 版本亦受影响）

#问题现象
- 使用 USB 音频/视频设备（如 Logitech H530、C910、C-Media 6501、TI PCM2900）一段时间后，内核日志报错 "cannot submit urb 0, error -22: internal error"。
- 日志显示 "timeout: still X active urbs on EP #Y" 以及 "usb_set_interface failed (-110)"。
- 出现 "INFO: task ... blocked for more than 120 seconds" 提示，表明发生 hung task。
- 最终导致 USB 设备断开连接，系统响应缓慢或完全挂起（鼠标键盘无响应）。

#问题根因
未明确。报告显示在提交 URB 时返回 -22 (EINVAL) 错误，导致 USB 端点超时并引发系统挂起，但具体代码缺陷位置未在报告中定位。

#解决方案
暂无
