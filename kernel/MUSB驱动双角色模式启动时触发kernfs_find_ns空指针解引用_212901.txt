#标题
MUSB驱动双角色模式启动时触发kernfs_find_ns空指针解引用

#内核版本
5.4.115

#问题现象
- 内核启动阶段发生 Oops，日志显示 "Unable to handle kernel NULL pointer dereference at virtual address 00000050"
- 触发条件：将 USB Support 下的 "Inventra Highspeed Dual Role Controller" 配置为 "Dual Role Mode" 时必现；配置为 "Host only mode" 时正常
- Call Trace: kernfs_find_ns -> kernfs_find_and_get_ns -> sysfs_unmerge_group -> dpm_sysfs_remove -> device_del -> device_unregister -> musb_gadget_setup -> musb_probe

#问题根因
MUSB 驱动在 Dual Role 模式下初始化过程中，`musb_gadget_setup` 函数调用了 `device_unregister`，该调用链经过 `sysfs_unmerge_group` 最终进入 `kernfs_find_ns` 函数。在此函数中，代码试图解引用一个空指针（地址 0x50），导致内核崩溃。

#解决方案
暂无
