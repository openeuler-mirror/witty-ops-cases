#标题
ThinkPad T14s在s2idle挂起时因pcie_aspm策略配置导致功耗过高

#内核版本
6.1.11-arch1-1

#问题现象
- 系统在 s2idle 挂起期间电池耗电异常，过夜耗电量在 10% 到 50% 之间波动。
- 问题与挂起前的 `/sys/module/pcie_aspm/parameters/policy` 设置相关：设置为 "performance" 或 "powersave" 时，系统无法进入 PC8 状态，导致高耗电；设置为 "default" 或 "powersupersave" 时耗电较低。
- 使用 S0ixSelftestTool 检测发现，PCIe 桥接设备 `0000:00:06.0`（连接 Micron NVMe SSD）链路状态停留在 L1，未能进入预期的 L1.1 或 L1.2 低功耗子状态。
- 寄存器 L1SubCtl1 显示 PCI-PM_L1.2、PCI-PM_L1.1、ASPM_L1.2、ASPM_L1.1 均处于禁用状态。

#问题根因
pcie_aspm 的电源管理策略在某些配置下未能正确启用 PCIe 链路的 L1 子状态。具体表现为当策略为 "performance" 或 "powersave" 时，PCIe 桥接设备 `0000:00:06.0` 的 L1SubCtl1 控制位未激活 L1.1/L1.2 子状态，导致链路无法进入深层节能模式，进而阻止 CPU 进入 PC8 状态，引发挂起期间的高功耗。

#解决方案
暂无
