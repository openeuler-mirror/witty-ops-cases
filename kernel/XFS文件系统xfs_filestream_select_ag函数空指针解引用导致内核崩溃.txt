#标题
XFS文件系统xfs_filestream_select_ag函数空指针解引用导致内核崩溃

#内核版本
v6.3-rc3

#问题现象
- 内核触发 NULL pointer dereference，导致 Oops，故障地址为 0000000000000010。
- 崩溃发生在函数 xfs_filestream_select_ag+0x5d5 处。
- 调用栈显示进程为 kworker (wb_workfn)，路径包含 xfs_vm_writepages -> xfs_bmap_btalloc -> xfs_filestream_select_ag。
- 故障发生前，日志中检测到 XFS Metadata CRC error (xfs_agf_read_verify)。

#问题根因
经二分定位，问题由提交 8ac5b996bf5199f15b7687ceae989f8b2a410dda ("xfs: fix off-by-one-block in xfs_discard_folio()") 引入。该提交可能导致了 xfs_filestream_select_ag 函数在特定错误处理路径（如 Metadata CRC error 后）下访问了非法内存地址。

#解决方案
暂无官方修复补丁。临时规避方案为回退问题提交 8ac5b996bf5199f15b7687ceae989f8b2a410dda。
