#标题
xHCI驱动在handle_cmd_completion中因VIA控制器不合规触发WARN_ON导致USB设备不可用

#内核版本
3.15.0-rc7 (cd79bde29f00a346eec3fe17c1c5073c37ed95e7)

#问题现象
- 系统启动期间 dmesg 报告 WARNING：`drivers/usb/host/xhci-ring.c:1615 handle_cmd_completion+0xdc7/0x1000`
- VIA USB 3.0 控制器无法工作，USB 端口不可用
- 连接的鼠标、键盘及大容量存储设备报错、不断重置，执行 `lsusb -v` 会卡死
- 日志中出现 "Reset device command completion for disabled slot 0" 及 "Timeout while waiting for reset device command" 等错误信息

#问题根因
Git 二分查找定位到引入问题的提交为 `20e7acb13ff48` ("xhci: use completion event's slot id rather than dig it out of command")。该提交添加了 `WARN_ON` 检查，用于验证命令完成事件中的 slot ID 是否与命令 TRB 中的一致。涉事的 VIA USB 3.0 控制器可能不符合 xHCI 规范 rev1.0，导致两者 slot ID 不匹配，从而触发警告并导致设备功能失效。
