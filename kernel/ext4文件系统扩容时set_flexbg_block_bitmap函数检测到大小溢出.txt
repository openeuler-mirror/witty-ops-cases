#标题
ext4文件系统扩容时set_flexbg_block_bitmap函数检测到大小溢出

#内核版本
3.12.8 with grsecurity

#问题现象
- 使用 resize2fs 对 ext4 文件系统进行扩容时，内核日志报错："PAX: size overflow detected in function set_flexbg_block_bitmap fs/ext4/resize.c:387"
- Call Trace 显示调用路径为：`set_flexbg_block_bitmap` -> `setup_new_flex_group_blocks` -> `ext4_flex_group_add` -> `ext4_resize_fs`
- 随后触发 hung task 警告，jbd2/dm-4-8 进程阻塞超过 120 秒

#问题根因
在 ext4 文件系统扩容过程中，函数 `set_flexbg_block_bitmap`（位于 fs/ext4/resize.c:387）触发了 grsecurity/PAX 的大小溢出检测机制。该溢出导致扩容操作异常，进而引发 jbd2 日志提交事务长时间阻塞。

#解决方案
暂无
