#标题
Radeon HD 7970M 混合显卡运行时电源管理导致挂起、报错及卸载模块崩溃

#内核版本
3.13-rc1

#问题现象
- 启用 DPM 和 runtime PM 后，系统启动期间出现警告 "Memory manager not clean during takedown"。
- 内核日志报错 "radeon: fence wait failed (-35)" 和 "ib ring test failed (-35)"。
- 系统频繁挂起数秒，X server 无响应，GPU 温度升高。
- 执行 `rmmod radeon` 时触发崩溃，日志显示 "Fixing recursive fault but reboot is needed!" 或导致 kernel panic。
- 独立显卡在 X 运行时无法进入 DynOff 状态，功耗异常。

#问题根因
问题由多个因素共同导致：
1. PCI hotplug 驱动冲突：`acpiphp` 驱动与 `shpchp` 驱动冲突，导致 runtime PM 行为异常。
2. 内核回归：Commit `bbd34fcdd1b201e996235731a7c98fd5197d9e51` 引入了 vgaswitcheroo/runpm 回归问题。
3. Mesa 驱动缺陷：Mesa radeonsi 驱动在没有绘图调用时执行了不必要的 flush 操作，导致 GPU 无法正常进入低功耗状态。

#解决方案
- 临时规避：启动参数添加 `radeon.runpm=0` 关闭 runtime PM（会增加功耗）；或添加 `acpiphp.disable=1` 禁用 acpiphp。
- 正式修复：升级至内核 3.14-rc3 及以上版本，并应用 Mesa 补丁（如 commit `cf0172d46ab940a691da6516057c81f28961482f`）以修复不必要的 flush 操作。
