#标题
mt76驱动使用iw设置txpower不生效导致WiFi信号弱

#内核版本
4.14.162

#问题现象
- 在 TP-Link Archer C50 v4/v5 设备（MediaTek MT7628A 芯片）上，2.4GHz WiFi 信号强度极弱，近距离信号质量差。
- 执行 `iw dev wlan0 set txpower fixed 2000` 或 `iw phy phy0 set txpower fixed 2000` 命令时返回成功，但查询接口信息显示 txpower 仍维持在 1.0 dBm，未发生任何变化。
- 重启网络接口后 dmesg 日志无异常报错。

#问题根因
mt76 驱动程序存在缺陷，未正确处理用户空间下发的 txpower 配置请求，导致发射功率设置无法生效。报告中提及同系列驱动 mt7615 曾存在类似的 txpower 设置问题。

#解决方案
社区已提交修复 Pull Request：https://github.com/openwrt/mt76/pull/442。临时规避措施可尝试通过 `iw phy phy0 set antenna_gain 20` 调整天线增益参数，但这并非根本解决方案。
