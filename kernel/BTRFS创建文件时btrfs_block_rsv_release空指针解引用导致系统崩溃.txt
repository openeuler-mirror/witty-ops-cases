#标题
BTRFS创建文件时btrfs_block_rsv_release空指针解引用导致系统崩溃

#内核版本
6.1.169-ideapad

#问题现象
- 内核日志报告 "BUG: kernel NULL pointer dereference, address: 0000000000000010"
- 触发 Oops，RIP 寄存器指向 btrfs_block_rsv_release+0x2c/0x220
- 系统完全死锁，需强制重启
- 故障发生时进程为 git，调用栈显示正在进行文件创建操作（path_openat -> btrfs_create_common）

#问题根因
在 BTRFS 文件系统执行创建文件相关操作时，函数 btrfs_block_rsv_release 接收到空指针参数（RSI 寄存器值为 0），导致在执行指令 `mov rax, QWORD PTR [rsi+0x10]` 时访问非法地址 0x10 触发缺页异常。故障调用链为：btrfs_create_common -> btrfs_add_link -> btrfs_insert_delayed_dir_index -> btrfs_release_dir_index_item_space -> btrfs_block_rsv_release。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
