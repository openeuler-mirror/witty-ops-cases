#标题
加载Broadcom wl驱动时触发“Unpatched return thunk”警告

#内核版本
6.11.2

#问题现象
- 系统加载 `wl` 无线驱动模块时，内核日志输出 "Unpatched return thunk in use. This should not happen!" 警告。
- 触发 WARNING，位置在 `arch/x86/kernel/cpu/bugs.c:3029 __warn_thunk+0x2a/0x40`。
- Call Trace 显示调用链为 `wl_module_init` -> `getvar` -> `warn_thunk_thunk`。
- 内核被污染标记包含：PROPRIETARY_MODULE, OOT_MODULE, UNSIGNED_MODULE。

#问题根因
专有驱动模块 `wl` 在初始化过程中使用了未修补的返回指令，该行为与内核当前的 CPU 漏洞缓解机制（如 Retbleed 或 SRSO 防护相关的 return thunk 机制）不兼容。由于 `wl` 为闭源驱动，其内部汇编代码未适配新版内核的安全要求。

#解决方案
暂无官方修复。建议联系 Broadcom 或驱动供应商获取更新版本的驱动，或尝试切换至开源驱动替代方案。
