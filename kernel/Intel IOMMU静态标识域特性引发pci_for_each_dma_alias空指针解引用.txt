#标题
Intel IOMMU静态标识域特性引发pci_for_each_dma_alias空指针解引用

#内核版本
6.12.0-rc1 (8c245fe7dde3)

#问题现象
- 系统启动或加载 intel_lpss_pci 模块时触发内核崩溃
- 日志显示：BUG: kernel NULL pointer dereference, address: 00000000000000d8
- RIP 寄存器指向：pci_for_each_dma_alias (drivers/pci/search.c:41)
- 触发 Oops: 0000 [#1] PREEMPT SMP DEBUG_PAGEALLOC

#问题根因
该问题由 commit 2031c469f8161abe74189cb74f50da224f340b71 ("iommu/vt-d: Add support for static identity domain") 引入。该提交引入了全局静态标识域，导致设备的 info->domain 指针为 NULL。在后续的 device_block_translation -> domain_context_clear_one -> pci_for_each_dma_alias 调用链中，代码未正确处理 NULL 指针情况，导致在访问结构体偏移 0xd8 时触发空指针解引用。

#解决方案
回退引入问题的 commit 2031c469f8161abe74189cb74f50da224f340b71 可恢复系统正常运行；用户反馈问题后续已修复（具体修复补丁未在报告中详述）。
