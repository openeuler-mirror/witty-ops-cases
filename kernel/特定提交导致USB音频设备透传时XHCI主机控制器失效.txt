#标题
特定提交导致USB音频设备透传时XHCI主机控制器失效

#内核版本
4.13.6

#问题现象
- 将C-Media USB音频设备透传给虚拟机时，XHCI主机控制器停止响应。
- 内核日志显示警告信息：`uurb->buffer_length is too short 6144 vs 0`。
- 触发内核 WARNING：`WARNING: CPU: 6 PID: 708 at drivers/usb/core/devio.c:1583 proc_do_submiturb+0x993/0xd60 [usbcore]`。
- 随后出现 DMA 错误：`DMAR: [DMA Read] Request device [00:14.0] fault addr 0 [fault reason 06] PTE Read access is not set`。
- 最终 XHCI 控制器报错：`xHCI host controller not responding, assume dead` 并停止工作。

#问题根因
问题由 Commit d328df9085b05f4f3ccdbe2ca1f980d72844641f (USB: devio: Don't corrupt user memory) 引入。该提交增加了对 `uurb->buffer_length` 的校验。在 USB 音频设备透传场景下，该缓冲区长度为 0，触发了校验失败路径。这导致 URB 处理流程异常，进而引发 XHCI 控制器固件/硬件状态异常，最终导致控制器失效。报告者指出，虽然该提交触发了故障，但缓冲区长度为 0 本身可能暗示了更深层的问题。

#解决方案
暂无官方修复。用户可回退至内核版本 4.13.5 以规避此问题。
