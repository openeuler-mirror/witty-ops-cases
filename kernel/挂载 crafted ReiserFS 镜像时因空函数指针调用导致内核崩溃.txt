#标题
挂载 crafted ReiserFS 镜像时因空函数指针调用导致内核崩溃

#内核版本
4.18.0-rc1+

#问题现象
- 挂载特制的 ReiserFS 镜像文件时触发内核崩溃（Oops）。
- 内核日志报错："init_special_inode: bogus i_mode (0) for inode loop0:3"。
- 随后触发 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000000"。
- Call Trace 显示调用路径为：reiserfs_xattr_init -> lookup_one_len -> __lookup_slow。

#问题根因
挂载过程中，ReiserFS 模块在执行 xattr 初始化（reiserfs_xattr_init）时调用 lookup_one_len，最终在 __lookup_slow 函数中执行 `inode->i_op->lookup`。由于特制镜像中包含 i_mode 为 0 的异常 inode，导致 i_op->lookup 函数指针未被正确初始化（为 NULL），调用该空指针引发崩溃。

#解决方案
暂无
