#标题
Goodix 触摸屏启动时读取配置失败导致间歇性不可用

#内核版本
4.8-rc5 (报告中亦涉及 4.12 至 4.14-rc3)

#问题现象
- Chuwi Hi12 平板电脑触摸屏在冷启动后间歇性失效，dmesg 显示 "Invalid config, using defaults"，读取到的配置参数 x_max/y_max 为 0。
- 在内核 4.13 及以上版本，出现 I2C 通信错误 "I2C communication failure: -121"，导致触摸屏完全不可用。
- 部分情况下触摸坐标存在偏移。
- 在旧版本内核中，通过 `rmmod goodix` 和 `modprobe goodix` 重新加载驱动可恢复正常。

#问题根因
1. 驱动在上电阶段未执行正确的 reset 序列（因 ACPI 表中 GPIO 配置缺失或驱动未正确处理），导致设备初始化时序错误，无法读取正确配置。
2. I2C 总线控制器（Designware）在部分内核版本中初始化失败（报错 "i2c_designware: probe of 808622C1:06 failed with error -110"），导致触摸屏驱动 probe 失败（error -121）。
3. 驱动代码中缺少对特定设备 ID (9111) 的支持，导致配置长度计算错误。

#解决方案
暂无上游合并修复。可通过以下方式规避或解决：
- 手动生成并部署 `goodix_9111_cfg.bin` 固件文件以修正设备配置。
- 应用开发者提供的测试补丁以启用 reset 序列（需配合 GPIO 配置）。
- 使用第三方修改版驱动（如 `kernins/linux-chwhi12` 仓库中的版本）。
- 在旧版本内核中，通过重新加载驱动模块作为临时规避方案。
