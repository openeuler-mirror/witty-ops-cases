#标题
KVM虚拟机网络通信触发__cgroup_bpf_run_filter_skb页错误导致内核Panic

#内核版本
5.6.0-2-amd64, 5.7.0-1-amd64

#问题现象
- 宿主机在运行多个虚拟机（Windows 10, Mac, Arch Linux）并使用网络一段时间后，随机触发 Kernel Panic。
- 系统日志显示 "BUG: unable to handle page fault for address: 0000000100000002"。
- 触发错误的进程为 vhost 内核线程（Comm: vhost-45322），错误类型为 supervisor read access 且页面不存在（not-present page）。
- RIP 寄存器指向 `__cgroup_bpf_run_filter_skb+0x11e/0x3d0`。

#问题根因
内核在处理虚拟机发出的网络数据包时，经由 `vhost_net` -> `tun` -> `tcp_v4_rcv` -> `sk_filter_trim_cap` 调用链，最终在 `__cgroup_bpf_run_filter_skb` 函数中试图读取非法内存地址 `0000000100000002`，触发 Page Fault 导致系统崩溃。该地址看起来像是一个被破坏的指针或未初始化的值。

#解决方案
暂无（Bug 状态为 NEW，尚未有修复补丁或临时规避方案）。
