#标题
XFS挂载损坏镜像触发断言失败导致Kernel Panic

#内核版本
5.1.3

#问题现象
- 挂载特制的 XFS 镜像文件时触发 kernel BUG，导致系统崩溃。
- 内核日志显示 "XFS: Assertion failed: 0, file: .../xfs_log_recover.c, line: 551"。
- 触发 invalid opcode (0000 [#1])，Call Trace 指向 `xlog_find_verify_log_record` -> `xlog_find_head` -> `xlog_recover`。

#问题根因
镜像文件中的 XFS 日志区域数据损坏，导致日志恢复逻辑检测到不一致状态。在 `xlog_find_verify_log_record` 函数中触发了 `ASSERT(0)` 断言。该断言设计用于调试内核以立即捕获异常状态，而在生产内核中该断言不生效，代码会返回错误码（如 -EIO）进行优雅处理。

#解决方案
维护者判定此行为非 Bug (NOTABUG)。该断言仅存在于调试构建中，用于辅助开发者定位代码或数据错误；生产环境内核不包含此断言，会通过错误处理路径返回 I/O 错误而不会触发 panic。建议避免在生产环境使用调试内核挂载不可信或损坏的文件系统镜像。
