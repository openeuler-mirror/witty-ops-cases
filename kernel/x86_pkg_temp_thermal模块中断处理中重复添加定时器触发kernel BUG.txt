#标题
x86_pkg_temp_thermal模块中断处理中重复添加定时器触发kernel BUG

#内核版本
4.4.0-140-generic

#问题现象
- 系统在处理 CPU 温度中断时触发 Kernel Panic，日志显示 "Fatal exception in interrupt"。
- 内核日志报错："kernel BUG at .../kernel/time/timer.c:992!"，伴随 "invalid opcode" 错误。
- 故障现场堆栈显示调用路径为：`thermal_interrupt` -> `pkg_temp_thermal_platform_thermal_notify` -> `queue_delayed_work_on` -> `add_timer_on`。

#问题根因
在 `x86_pkg_temp_thermal` 驱动的中断处理流程中，`pkg_temp_thermal_platform_thermal_notify` 函数调用 `queue_delayed_work_on` 尝试添加一个延迟工作项。由于该工作项关联的定时器已经处于 pending 状态（可能因中断频繁触发），导致 `add_timer_on` 函数检测到定时器重复添加，触发 `BUG()` 导致系统崩溃。

#解决方案
暂无
