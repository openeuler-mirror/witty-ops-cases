#标题
从挂起恢复后USB枚举卡死导致用户空间进程挂起

#内核版本
5.19.16, 6.0.10

#问题现象
- 从挂起状态恢复后，读取 USB 设备 "descriptors" sysfs 文件时进程挂起，导致 `lsusb` 命令无响应。
- `gnome-shell`/`gdm` 因 `fprintd` 服务卡死而挂起。
- 内核日志显示 xHCI 控制器 (0000:06:00.3) 下的 USB 设备在恢复过程中断开连接后未能重新枚举。
- 日志中出现 AMD GPU 显示相关错误：`DM_MST: Failed to start MST`。

#问题根因
问题可能与 USB Type-C 端口在恢复时的配置状态有关。当使用支持 DP Alt-mode 的扩展坞时，恢复过程中 `amdgpu` 驱动报错导致 MST 启动失败，随后 USB 设备断开连接，且 Type-C 端口似乎被错误地从 xHCI 控制器切换走，导致设备无法重新枚举。同时，`usbfs` 读取操作未能正确处理设备断开的情况（可能未返回 URB 或无限重试），导致用户空间进程永久阻塞。

#解决方案
暂无正式修复。开发者建议开启 `xhci_hcd` 和 `usbcore` 的动态调试日志以进一步定位问题。
