#标题
IPv6 Mobility未开启转发导致Home Agent触发Kernel Panic

#内核版本
Debian 4.2.6-1

#问题现象
在使用 IPv6 Mobility 功能时，当 Mobile Node (MN) 尝试连接 Home Agent (HA) 时，HA 端触发 kernel panic。

#问题根因
在 Home Agent 未开启 IPv6 forwarding（即 `net.ipv6.conf.all.forwarding` 为 0）的情况下，路由表及其锁可能已被释放，但 dst 条目未持有对路由表的正确引用，导致访问时触发 panic。

#解决方案
规避方法：在 Home Agent 上开启 IPv6 转发功能（执行 `echo 1 > /proc/sys/net/ipv6/conf/all/forwarding`）。
潜在修复：提交 `cdf3464e6c6` 可能已修复该问题，建议升级内核版本验证。
