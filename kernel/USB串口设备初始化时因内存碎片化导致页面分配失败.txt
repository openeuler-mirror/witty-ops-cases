#标题
USB串口设备初始化时因内存碎片化导致页面分配失败

#内核版本
3.12.14

#问题现象
- 插入 USB 设备（如 CP210x UART Bridge）时，内核日志报错 "udevd: page allocation failure: order:4, mode:0x2000d0"。
- SLAB 分配器报告无法在 node 0 分配内存："SLAB: Unable to allocate memory on node 0 (gfp=0xd0)"，涉及 cache kmalloc-65536。
- Call Trace 显示调用链为 `usb_serial_init` -> `__tty_alloc_driver` -> `__kmalloc` -> `__alloc_pages_nodemask`。

#问题根因
系统内存碎片化严重，无法满足 order:4（即 64KB 连续物理内存）的分配请求。尽管系统总体空闲内存充足（Normal 区域约 370MB），但内存碎片化导致缺乏高阶连续页面，`__tty_alloc_driver` 在尝试分配大块内存时失败。

#解决方案
暂无（Bug 处于 NEW 状态，未提供修复补丁或规避措施）。
