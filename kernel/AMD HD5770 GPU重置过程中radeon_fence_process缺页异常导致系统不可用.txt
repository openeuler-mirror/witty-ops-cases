#标题
AMD HD5770 GPU重置过程中radeon_fence_process缺页异常导致系统不可用

#内核版本
3.10 RC5

#问题现象
- 运行图形应用（如 Ryzom 游戏）时触发 GPU 锁定
- 内核尝试重置 GPU 时发生 OOPS，日志显示 "BUG: unable to handle kernel paging request at ffffc90012a9c418"
- 触发 Oops 的指令指针位于 `radeon_fence_process+0x8e/0x160 [radeon]`
- 系统失去图形输出，处于不可用状态，仅能响应 alt-sysrq 指令

#问题根因
GPU 发生锁定后，内核在执行 GPU 重置流程时，`radeon_fence_process` 函数访问了无效的内存地址（PTE 为 0），触发缺页异常。故障调用链为：`radeon_gpu_reset` -> `radeon_ring_backup` -> `radeon_fence_count_emitted` -> `radeon_fence_process`。

#解决方案
暂无内核修复补丁。升级 MESA 至 9.2 版本可规避导致 GPU 锁定的诱因，从而避免触发该故障。
