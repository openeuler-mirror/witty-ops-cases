#标题
b43驱动卸载时LED控制逻辑触发空指针解引用导致Oops

#内核版本
3.12.2

#问题现象
- 执行 `modprobe -r b43` 卸载模块或加载模块时触发内核崩溃。
- 终端输出 "BUG: unable to handle kernel NULL pointer dereference at (null)"。
- 日志显示 Oops 发生在 `__queue_work` 函数，寄存器 RBX 为 0。
- Call Trace 显示调用链为：`b43_exit` -> `b43_ssb_remove` -> `ieee80211_unregister_hw` -> `ieee80211_led_exit` -> `led_trigger_unregister` -> `b43_led_brightness_set` -> `ieee80211_queue_work`。

#问题根因
在驱动卸载过程中，`ieee80211_led_exit` 调用 `led_trigger_unregister` 注销 LED 触发器。该操作触发了 `b43_led_brightness_set` 回调函数，该函数试图通过 `ieee80211_queue_work` 将工作项加入队列。此时驱动上下文已处于无效状态（如硬件结构体指针可能已失效或为空），导致 `__queue_work` 在访问空指针时触发 Oops。

#解决方案
暂无
