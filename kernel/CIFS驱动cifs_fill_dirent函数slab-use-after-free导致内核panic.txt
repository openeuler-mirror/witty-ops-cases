#标题
CIFS驱动cifs_fill_dirent函数slab-use-after-free导致内核panic

#内核版本
6.15.0-rc6+

#问题现象
- 系统触发 Kernel panic，提示 "KASAN: panic_on_warn set ..."。
- KASAN 检测到 slab-use-after-free 错误，发生在 `cifs_fill_dirent+0xb03/0xb60`。
- 调用栈显示：`cifs_fill_dirent` <- `cifs_readdir` <- `iterate_dir` <- `__x64_sys_getdents`。
- KASAN 日志显示内存被 task 342979 释放，但被 task 342975 读取。

#问题根因
CIFS 模块在处理 readdir 操作时存在并发竞态条件。KASAN 日志表明，`cifs_buf_release` 函数（由 task 342979 执行）释放了 `cifs_request` 缓冲区内存，而 `cifs_fill_dirent` 函数（由 task 342975 执行）随后试图访问该已释放内存。这导致 `cifs_request` 对象在生命周期管理上出现 Use-After-Free 漏洞。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
