#标题
AMD Ryzen处理器因BIOS MADT表错误报告导致非物理核心显示为offline

#内核版本
6.6

#问题现象
- 系统中 `/sys/devices/system/cpu/offline` 显示本不存在的 CPU 核心（例如 24-31），而实际物理核心数较少（如 5900X 为 24 线程）。
- 内核启动日志显示 RCU 和 pcpu-alloc 识别到的 `nr_cpu_ids` 为 32，而非实际的 24。
- 使用 `cpupower` 等工具操作 CPU 频率时，提示部分 CPU 处于 offline 状态无法操作。
- 内核可能基于错误的 `cpu_possible_mask` 进行资源分配，造成潜在的资源浪费。

#问题根因
BIOS 在 MADT（Multiple APIC Description Table）表中报告了禁用的 CPU 条目（Enabled: 0），但未设置 "Runtime Online Capable" 位。Linux 内核自 6.3 版本引入 commit `fed8d8773b8ea68ad99d9eee8c8343bef9da2c2c` 后，修改了 `acpi_is_processor_usable()` 的判断逻辑，导致内核将这些本应忽略的禁用 CPU 条目标记为 possible 状态，进而显示为 offline。

#解决方案
暂无内核补丁修复。可通过内核启动参数 `nr_cpus=<实际物理核心数>`（如 nr_cpus=24）规避此问题。根本解决需 BIOS 厂商修正 MADT 表中关于禁用 CPU 的报告方式。
