#标题
ath11k驱动QCN9074网卡在monitor模式下触发CPU soft lockup

#内核版本
6.1.x, 6.2.x, 6.4.3, 6.5.0（5.15.0-78 版本未复现）

#问题现象
- QCN9074 网卡进入 monitor 模式并开始抓包数秒后，系统触发 CPU soft lockup，导致机器完全冻结。
- dmesg 日志显示：`watchdog: BUG: soft lockup - CPU#0 stuck for 26s!`。
- Call Trace 指向中断上下文中的 `_raw_spin_lock_bh`，调用链为：`ath11k_dp_rx_mon_mpdu_pop.constprop.0` -> `ath11k_dp_rx_mon_dest_process.constprop.0` -> `ath11k_dp_rx_process_mon_status`。
- 开启驱动调试后，日志中出现大量 `msdu_pop: invalid buf_id`、`Monitor Link Desc Ring 0 Full` 以及 `dp_rx_monitor_link_desc_return failed` 错误，计数器激增后系统死机。

#问题根因
ath11k 驱动在 monitor 模式下处理接收数据时存在回归问题。函数 `ath11k_dp_rx_mon_mpdu_pop` 在处理过程中遇到无效的 buffer ID，导致 Monitor Link Desc Ring 无法正确回收，进而引发 ring full 并导致 CPU 在自旋锁上死锁。该问题是在内核 5.15 至 6.1/6.2 版本之间引入的。

#解决方案
暂无上游修复补丁。临时规避方案为降级内核至 5.15.0-78 版本。
