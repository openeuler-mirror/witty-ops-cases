#标题
io_uring导致内核空指针解引用引发虚拟机启动失败

#内核版本
6.1.5

#问题现象
- 虚拟机在启动过程中发生锁定，无法正常进入系统。
- dmesg 日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000005"。
- 问题出现在使用 libvirt 配置磁盘驱动为 io_uring 模式时。
- 回退至内核版本 6.1.4 问题消失。

#问题根因
内核 6.1.5 版本在回溯移植时遗漏了 commit 613b14884b8595e20b9fac4126bf627313827fbe。该补丁用于处理 `bio_split_to_limits()` 函数返回 NULL 的情况，缺失该补丁导致在 io_uring 处理 I/O 请求时触发了空指针解引用。

#解决方案
手动 cherry-pick 补丁 commit 613b14884b8595e20b9fac4126bf627313827fbe ("block: handle bio_split_to_limits() NULL return")。根据开发者反馈，该修复已包含在 6.1.7 及之后的版本中。
