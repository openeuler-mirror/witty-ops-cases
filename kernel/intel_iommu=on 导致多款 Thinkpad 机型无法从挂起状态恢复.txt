#标题
intel_iommu=on 导致多款 Thinkpad 机型无法从挂起状态恢复

#内核版本
5.8.7

#问题现象
- 启用 intel_iommu=on 内核参数后，多款 Thinkpad 机型（如 T560, T450s, X1 Yoga, T460）无法从挂起状态恢复，系统卡死，只能长按电源键强制关机。
- 部分情况下，重启或关机操作也会在最后阶段卡死。
- 系统日志中出现 DMAR 错误：DMAR: DRHD: handling fault status reg 3 以及 DMAR: [DMA Read] Request device [00:12.4] fault addr b7fff000 [fault reason 02] Present bit in context entry is clear。

#问题根因
问题根因尚未完全定论，目前存在以下分析：
- Intel 开发者分析认为，问题源于一个隐藏的 ME 设备未被操作系统感知，导致 IOMMU 挂起/恢复流程异常。
- 曾怀疑是 commit 422eac3f7deae34dbaffd08e03e27f37a5394a56 (tpm_crb: fix mapping of the buffers) 引入的回归，但用户测试回退该补丁后问题依旧。
- 另一分析指出，挂起后部分寄存器上下文丢失，驱动未能重新初始化 uarch 状态（如 root table pointer）。

#解决方案
- 规避方案 1：在 GRUB 启动参数中添加 intel_iommu=off。
- 规避方案 2：修改 BIOS 设置，将 Security chip 从 PTT 改为 TPM，或在 "PCH-FW Configuration" 中开启 "PTP aware OS"。
- 截至报告时间，内核主线尚未合并修复补丁。
