#标题
内核版本3.7.0-rc8出现CPU Soft Lockup现象

#内核版本
3.7.0-rc8-g18a2f371+

#问题现象
- 系统日志频繁报错 "BUG: soft lockup - CPU#0 stuck for 22s!"，该错误每隔约 28 秒触发一次。
- 故障发生时系统整体仍保持响应，但在运行特定应用程序（如 digikam）进行负载操作时容易触发。
- dmesg 日志显示 CPU 处于用户态上下文时触发 soft lockup 检测（RIP 指向用户空间地址）。

#问题根因
未明确。用户报告称在启用 CONFIG_SCHED_AUTOGROUP 等额外功能后出现问题，且 3.6.0-rc7 版本未出现此现象，推测可能与调度器配置相关，但缺乏确切的代码定位分析。

#解决方案
暂无。用户尝试关闭 irqbalance 或移除 iTCO_wdt 模块，现象偶有消失，但未确认有效修复方案。
