#标题
挂载受损HFS+镜像执行setxattr触发内核WARNING与越界访问

#内核版本
4.18

#问题现象
- 挂载特制的 HFS+ 镜像并执行 setxattr 系统调用时，内核触发 WARNING
- 内核日志显示 "hfsplus: new node 0 already hashed?"
- 堆栈信息显示崩溃路径为：hfsplus_bnode_create -> hfsplus_bmap_alloc -> hfs_bnode_split -> hfsplus_brec_insert -> hfsplus_create_attr -> __hfsplus_setxattr

#问题根因
HFS+ 文件系统驱动在处理受损镜像的 setxattr 操作时，未能正确处理 B-tree 节点分配逻辑，导致在 hfsplus_bnode_copy 函数中发生越界访问，并在 hfsplus_bnode_create 中触发节点已哈希的异常状态。

#解决方案
暂无
