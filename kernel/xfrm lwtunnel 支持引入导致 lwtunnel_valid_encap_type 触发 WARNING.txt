#标题
xfrm lwtunnel 支持引入导致 lwtunnel_valid_encap_type 触发 WARNING

#内核版本
v6.1-rc3

#问题现象
- 内核日志输出 WARNING：`WARNING: CPU: 0 PID: 579 at net/core/lwtunnel.c:57 lwtunnel_valid_encap_type+0x167/0x1e0`
- 调用栈显示路径为 `rtm_to_fib6_config` -> `inet6_rtm_delroute` -> `rtnetlink_rcv_msg`。
- 问题在 syzkaller 测试中被发现。

#问题根因
经二分定位，问题由 commit `2c2493b9da9166478fe072e3054f8a5741dadb02` ("xfrm: lwtunnel: add lwtunnel support for xfrm interfaces in collect_md mode") 引入。该提交为 xfrm 接口的 collect_md 模式添加 lwtunnel 支持，导致 `lwtunnel_valid_encap_type` 函数触发 WARNING。

#解决方案
修复补丁已提交至 IPsec tree，详见：https://lore.kernel.org/netdev/Y0UwqMZQ6n+G%2F%2FaD@shredder/。
临时规避方案：回退上述引入问题的 commit。
