#标题
GPU驱动_DSM参数类型不匹配导致ACPI警告及系统锁死

#内核版本
3.12.3

#问题现象
- 系统日志中出现大量 ACPI Warning 和 Firmware Bug 信息，例如：`[Firmware Bug]: ACPI(IGPU) defines _DOD but not _DOS` 以及 `ACPI Warning: \\_SB_.PCI0.XVR0.Z01I._DSM: Argument #4 type mismatch - Found [Integer], ACPI requires [Package]`。
- 运行专业音频软件（如 JACK、Ardour）数秒或数分钟后系统发生严重锁死，需强制重启。
- 部分内核版本（如 3.13-rc）在尝试加载 nouveau 驱动时直接 panic。

#问题根因
GPU 驱动（nouveau 或 nvidia）在调用 ACPI 的 `_DSM` 方法时，传递了错误的参数类型（Integer 或 Buffer），而 ACPI 规范要求参数类型必须为 Package，导致内核打印警告信息。该错误可能与系统锁死相关。

#解决方案
