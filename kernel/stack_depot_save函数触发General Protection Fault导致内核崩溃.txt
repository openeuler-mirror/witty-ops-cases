#标题
stack_depot_save函数触发General Protection Fault导致内核崩溃

#内核版本
v5.2-rc6

#问题现象
- 系统触发 general protection fault (GPF)，错误码 0000。
- KASAN 日志提示 GPF 可能由 NULL-ptr deref 或 user memory access 引起。
- 故障 RIP 指针位于 `stack_depot_save+0x101/0x440` (lib/stackdepot.c:241)，内联函数为 `find_stack`。
- 调用栈显示路径为 `sock_ioctl` -> `dev_ioctl` -> `dev_load` -> `__request_module` -> `call_usermodehelper_exec` -> `kfree` -> `__kasan_slab_free` -> `save_stack` -> `stack_depot_save`。

#问题根因
在开启 CONFIG_KASAN_INLINE 的情况下，内核在执行 `stack_depot_save` 函数（具体位于 `find_stack` 内联逻辑）时访问了无效内存地址，触发 General Protection Fault。该故障发生在 `kfree` 调用链中 KASAN 试图保存栈追踪信息（save_stack）的阶段。

#解决方案
暂无（Bug状态为NEW，尚未修复）。
