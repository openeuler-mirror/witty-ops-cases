#标题
加载xhci-hcd驱动时xhci_mem_cleanup空指针解引用导致Oops

#内核版本
3.4.14

#问题现象
- 执行 `modprobe xhci-hcd` 加载驱动时触发内核 Oops，导致 Segmentation fault。
- 内核日志显示："Unable to handle kernel NULL pointer dereference at virtual address 00000000"。
- 堆栈跟踪显示崩溃发生在 `xhci_mem_cleanup+0x390/0x530 [xhci_hcd]`。
- 初始化阶段出现异常告警："WARN: no supported page size"、"Couldn't initialize memory"、"Host not halted"。

#问题根因
xhci 控制器初始化过程中检测到硬件状态异常（Host not halted）及内存初始化失败，驱动逻辑转入 `xhci_mem_cleanup` 函数执行错误处理与资源清理。在清理过程中，代码访问了未经校验的空指针（虚拟地址 0x0），导致内核崩溃。

#解决方案
暂无（Bug 处于 NEEDINFO 状态，开发者仅询问是否在新版本复现，未提供修复补丁）。
