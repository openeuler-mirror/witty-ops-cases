#标题
path_openat函数访问非法内存地址触发内核Oops

#内核版本
5.17.5-1

#问题现象
- 系统运行过程中突然崩溃，触发内核 Oops。
- 日志显示 "BUG: unable to handle page fault for address: fffffffffff9509e"。
- 异常类型为 supervisor read access in kernel mode，错误码 error_code(0x0000) - not-present page。
- 故障发生在进程 `wget` 执行文件打开操作时，Call Trace 显示路径为 `path_openat` -> `do_filp_open` -> `do_sys_openat2` -> `__x64_sys_openat`。

#问题根因
内核在执行 VFS 层的 `path_openat` 函数（偏移 +0x94a）时，尝试解引用指针访问内存地址 `ffffff9509e`（寄存器 RDX），该地址对应的页表项 PMD 为 0，导致缺页异常。具体的指针错误来源（如数据结构损坏或逻辑错误）需进一步分析代码确定。

#解决方案
暂无
