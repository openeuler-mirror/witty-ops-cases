#标题
KVM模块vmx_vcpu_run函数访问非法地址导致kernel panic

#内核版本
4.1.0+

#问题现象
- 系统触发 kernel panic (Oops: 0002)，显示无法处理内核分页请求。
- 日志报错：BUG: unable to handle kernel paging request at 000000005901feb0。
- 崩溃发生在 KVM 模块的 vmx_vcpu_run+0x276/0x700 [kvm_intel] 函数中。

#问题根因
根据提交者分析，崩溃由栈数据异常引起。在 vmx_vcpu_run 函数执行过程中，RSI 寄存器通过 mov -0x68(%rbp),%rsi 指令获取的值（000000005901fc68）错误，导致后续 movl 指令访问非法内存地址（RSI + 0x248 = 000000005901feb0）触发缺页异常。推测原因为栈损坏导致 RSI 寄存器值异常。

#解决方案
暂无修复方案，维护者建议在最新内核版本上尝试复现。
