#标题
VMA调整过程中anon_vma空指针解引用导致Kernel Panic

#内核版本
v5.2-rc6

#问题现象
- Syzkaller 触发 KASAN 报告：null-ptr-deref Write
- 内核触发 Kernel Panic（panic_on_warn set）
- 日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000008"
- Call Trace 1 (CPU 1): down_write -> __vma_adjust -> vma_merge -> mmap_region -> do_mmap
- Call Trace 2 (CPU 0): anon_vma_clone -> anon_vma_fork -> dup_mmap -> copy_process -> _do_fork

#问题根因
在 `__vma_adjust` 函数执行路径中，调用 `down_write` 尝试对 `anon_vma` 进行写锁操作时，访问了非法地址 0000000000000008。该地址偏移表明 VMA 结构体中的 `anon_vma` 指针为 NULL，导致在内存映射或进程 Fork 操作期间触发空指针解引用。

#解决方案
暂无（Bug 状态为 NEW，尚未提供修复补丁）
