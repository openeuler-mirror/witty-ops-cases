#标题
内核页面链表损坏导致list_add校验失败触发kernel BUG

#内核版本
6.13.2

#问题现象
- 系统在使用 Yandex Browser 访问 F2FS 分区时触发 kernel BUG，系统日志记录到 "list_add corruption" 错误。
- 触发 Oops：invalid opcode，RIP 寄存器指向 `__list_add_valid_or_report.cold+0x4f/0x60`。
- 调用栈显示崩溃路径为：`fuse_open` -> `invalidate_inode_pages2_range` -> `free_unref_folios` -> `free_unref_page_commit`。

#问题根因
内核在释放页面过程中检测到链表结构损坏，具体表现为 `next->prev` 指针未指向预期的 `prev` 节点。f2fs 维护者分析认为，虽然崩溃发生在文件系统操作路径上，但核心问题可能位于内存管理或页面分配器层面，而非 f2fs 模块本身。

#解决方案
暂无修复补丁。f2fs 维护者建议将 Bug 报告转至内存管理组件进行进一步分析。用户计划回退至 6.12 内核版本进行验证测试。
