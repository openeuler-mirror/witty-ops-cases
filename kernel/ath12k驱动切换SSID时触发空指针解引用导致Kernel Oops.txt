#标题
ath12k驱动切换SSID时触发空指针解引用导致Kernel Oops

#内核版本
6.14.4-arch1-1

#问题现象
- 使用 Qualcomm WCN785x (Wi-Fi 7) 网卡切换 Wi-Fi 网络（SSID）时触发 Kernel Oops
- 系统部分无响应，UI 挂起，需强制重启
- 日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000000"
- RIP 寄存器指向函数 ath12k_mac_remove_link_interface.isra.0
- Call Trace: cfg80211_wiphy_work -> ath12k_scan_vdev_clean_work -> ath12k_mac_remove_link_interface

#问题根因
在 ath12k 驱动的 ath12k_mac_remove_link_interface 函数中发生了空指针解引用，该问题在 cfg80211_wiphy_work 工作队列处理 SSID 切换相关清理操作时触发。

#解决方案
开发者建议尝试应用补丁：https://patchwork.kernel.org/project/linux-wireless/patch/20250418064008.7172-1-quic_lingbok@quicinc.com/
