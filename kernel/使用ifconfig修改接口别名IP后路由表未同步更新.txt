#标题
使用ifconfig修改接口别名IP后路由表未同步更新

#内核版本
3.13.0

#问题现象
使用 ifconfig 配置接口别名（如 eth0:1）的 IP 地址并添加默认路由后，若修改该别名的 IP 地址，路由表中指向原 IP 的默认路由条目未被自动删除，导致路由表状态与实际接口配置不一致。

#问题根因
Linux 内核中 eth0:1 等网络别名仅为兼容 BSD 模式的旧机制，并非真实网络接口。内核在处理此类兼容性接口的地址变更时，其路由处理逻辑存在限制，无法像处理常规接口那样自动维护路由表的一致性。

#解决方案
暂无内核修复计划。建议使用现代网络配置工具（如 `ip addr add`）替代 ifconfig 和接口别名机制，以避免此类路由不一致问题。
