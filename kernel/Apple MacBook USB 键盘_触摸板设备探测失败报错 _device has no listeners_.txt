#标题
Apple MacBook USB 键盘/触摸板设备探测失败报错 "device has no listeners"

#内核版本
3.17.7-gentoo

#问题现象
- 系统启动过程中内核日志报错：`apple 0003:05AC:0259.0002: device has no listeners, quitting`
- 随后报错：`apple 0003:05AC:0259.0002: hw start failed`
- 故障硬件为 MacBook Pro late 2013 retina 13'' 的内置键盘/触摸板。

#问题根因
`drivers/hid/hid-core.c` 中的设备探测逻辑未正确忽略 Apple 特定的触摸板接口（Product ID 为 `USB_DEVICE_ID_APPLE_WELLSPRING7A_ANSI`）。当该设备的总线类型为 `BUS_USB` 且 HID 类型为 `HID_TYPE_USBNONE` 时，驱动尝试启动硬件但失败，导致报错。

#解决方案
用户提供了针对 `drivers/hid/hid-core.c` 的修复补丁：在忽略设备列表逻辑中增加判断，若厂商为 `USB_VENDOR_ID_APPLE` 且产品 ID 为 `USB_DEVICE_ID_APPLE_WELLSPRING7A_ANSI`（同时满足 bus 为 `BUS_USB`、type 为 `HID_TYPE_USBNONE`），则忽略该设备，从而避免探测失败。上游 Bug 状态为 NEW，暂无官方合并信息。
