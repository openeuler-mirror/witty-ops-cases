#标题
Radeon驱动GPU锁定后恢复失败导致X进程卡死

#内核版本
3.3-rc1

#问题现象
- 内核日志显示 "GPU lockup CP stall for more than 10000msec"，随后 GPU softreset 成功
- GPU 重置后 X server 无法恢复，尝试唤醒屏幕时进程卡死
- SysRq+W 显示 X 进程处于 D 状态（不可中断睡眠），Call Trace 指向 radeon_fence_wait 和 radeon_ib_get
- 此为 3.2 到 3.3-rc1 之间引入的回归问题

#问题根因
该问题涉及多个层面的缺陷：
1. GPU 锁定的触发原因：内核对 DESKTOP_HEIGHT 设置处理错误（已由 commit 1b6192506166 修复）。
2. Mutex 死锁：GPU 锁定处理路径中的 mutex 死锁（已由 commit 9fc04b503df9 修复）。
3. 恢复机制失效（本 Bug 核心问题）：即便修复了上述两个问题，GPU reset 成功后 X 进程仍卡死在内核态，无法完成恢复流程。开发者通过代码审查未定位到具体的代码根因。

#解决方案
针对 GPU 锁定的触发因素和 mutex 死锁已有相应补丁修复，但针对 GPU 锁定后 X 进程卡死无法恢复这一核心回归问题，在报告截止时状态为 NEW，暂无明确修复方案。
