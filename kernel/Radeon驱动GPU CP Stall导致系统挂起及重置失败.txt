#标题
Radeon驱动GPU CP Stall导致系统挂起及重置失败

#内核版本
3.11.4（报告中亦涉及 3.12.6、3.13.0、3.16.4）

#问题现象
- 系统运行过程中随机出现图形界面卡死，无法进行 VT 切换，仅能通过 SSH 访问。
- 内核日志显示 `radeon 0000:01:00.0: GPU lockup CP stall for more than 10000msec`。
- 驱动尝试执行 GPU reset，但经常失败，报错 `ring test failed (scratch(0x15E8)=0xCAFEDEAD)` 或 `Failed to wait GUI idle while programming pipes`。
- 部分案例中，在 `radeon_ring_backup` 函数处触发 `BUG: unable to handle kernel paging request` 导致内核 Oops。

#问题根因
Radeon GPU 的 Command Processor (CP) 发生停滞，触发 GPU lockup 检测。在执行 GPU reset 恢复流程时，ring buffer 测试失败或内存访问异常，导致驱动无法恢复正常工作状态。具体的代码逻辑缺陷未明确，可能与特定 GPU 型号（如 RV570, HD3650, HD3450）的电源管理或硬件状态处理有关。

#解决方案
临时规避方案：在 GRUB 启动参数中添加 `radeon.hard_reset=1`。
官方修复：暂无（Bug 状态为 NEW）。
