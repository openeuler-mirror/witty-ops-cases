#标题
Nouveau 驱动 2D 加速导致 GPU 锁死及系统挂起

#内核版本
4.8.6

#问题现象
- 在未添加 `nouveau.noaccel=1` 内核参数时，系统启动阶段 GPU 锁死，dmesg 显示 lock up 信息，X Server 只能使用 fbdev 驱动。
- 添加 `nouveau.noaccel=1` 参数后，系统启动正常，但 X Server 仍只能使用 fbdev 驱动；若尝试使用 nouveau 驱动，系统会循环打印 "nouveau 0000:0a:00.0: DRM: 0x13DC: Parsing digital output script table"，导致系统不可访问，且 `systemctl stop lightdm` 和 `reboot` 命令挂起。

#问题根因
未明确（Bug 报告状态为 NEW，尚未有开发者确认具体代码层面的故障原因）。

#解决方案
暂无官方修复。临时规避方法为在内核启动参数中添加 `nouveau.noaccel=1` 禁用加速，但会导致 X Server 无法正常使用 nouveau 驱动。
