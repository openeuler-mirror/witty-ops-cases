#标题
KASAN quarantine 释放过程中触发内核缺页异常

#内核版本
v5.0.0

#问题现象
- 内核日志报告 "BUG: unable to handle kernel paging request"，错误地址为 ffffebe644000008
- 触发 Oops (#PF error: normal kernel read fault)，RIP 指向 qlist_free_all+0x7b/0xc0
- 调用栈显示路径为：quarantine_reduce -> __kasan_kmalloc -> kmem_cache_alloc -> getname_flags -> do_sys_open
- 系统处于 SMP KASAN PTI 模式，由 Syzkaller 测试触发

#问题根因
在 KASAN quarantine 释放流程中，qlist_free_all 函数调用 qlink_to_cache -> virt_to_head_page -> compound_head 时，访问了无效的内存地址，导致内核页面请求错误。

#解决方案
暂无
