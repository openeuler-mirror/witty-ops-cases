#标题
amdgpu驱动pqm_debugfs_mqds函数空指针解引用导致内核Oops

#内核版本
5.10.1

#问题现象
- 运行 rocminfo 工具时触发内核 Oops，系统日志显示 "BUG: unable to handle page fault for address: 0000000000001000"
- RIP 寄存器指向 amdgpu 模块中的 pqm_debugfs_mqds+0x169 函数位置
- 触发 Oops 前日志中出现多次 "amdgpu: Failure to set tba address. error -1." 错误

#问题根因
在函数 pqm_debugfs_mqds 执行过程中，寄存器 RAX 的值为 0（NULL），代码试图向 RAX+0x1000 的地址（即非法地址 0x1000）写入数据，触发空指针解引用导致的页面错误。此前 "Failure to set tba address" 错误表明相关硬件状态设置失败，可能导致后续流程中指针未正确初始化。

#解决方案
暂无
