#标题
Baytrail-T平台ASUS T100TAF因ACPI ID及DSP配置错误导致无声音

#内核版本
3.17（初始报告），问题延续至 3.18-rc1、4.8 及 4.9 版本

#问题现象
- 系统无声音输出，声卡无法正常工作。
- 内核日志中出现错误："No matching ASoC machine driver found"、"ASoC: CPU DAI baytrail-pcm-audio not registered"、"ipc: error DSP boot timeout"。
- 驱动 snd-soc-sst-byt-rt5640-mach 加载失败或请求延迟探测。
- 错误的混音器设置可能导致扬声器输入直流电压，存在烧毁硬件的风险。

#问题根因
- ACPI ID 不匹配：设备使用 Realtek RT5642 编解码器（ACPI ID 为 "10EC5642"），但驱动程序仅支持 "10EC5640"。
- DSP 中断索引错误：DSDT 表中 DSP-host 中断定义的索引位置与标准 Baytrail 机器不同，导致 DSP 初始化失败。
- SSP 端口配置错误：硬件连接使用 SSP0 接口，而默认固件/驱动硬编码为 SSP2。
- 时钟源缺失：驱动无法获取 pmc_plt_clk_3 提供的 MCLK 时钟。

#解决方案
- 应用补丁以支持 RT5642 ACPI ID 及 T100TAF 的特定 quirk 配置。
- 使用支持 SSP0 的固件文件 fw_sst_0f28_ssp0.bin。
- 合并 clk-x86-Add-Atom-PMC-platform-clocks 补丁（Linux 4.9 已合入相关修复）。
- 临时规避方案：将 snd_hdmi_lpe_audio 模块加入黑名单。
