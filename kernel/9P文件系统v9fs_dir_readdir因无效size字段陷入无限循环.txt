#标题
9P文件系统v9fs_dir_readdir因无效size字段陷入无限循环

#内核版本
3.13.0

#问题现象
- 当服务器发送目录或stat条目时，若size字段被错误设置为(u16)-2或大于实际记录大小，9p驱动程序进入无限循环。
- 系统后续操作发生挂起。

#问题根因
vfs_dir.c 文件第148行代码盲目信任服务器提供的 size 字段来计算已消耗记录长度（st.size+2）。若 size 字段为 (u16)-2，计算出的长度为0，导致 rdir->head 指针未更新，while 循环条件 rdir->head < rdir->tail 永远无法满足；若 size 字段大于实际记录，会导致 rdir->head 超过 rdir->tail，外层循环重试时无法读取新数据，最终均导致无限循环。

#解决方案
修改代码使用解析后的记录长度替代服务器提供的 size 字段来更新指针。相关补丁已提交上游，计划合入 4.20 版本。
