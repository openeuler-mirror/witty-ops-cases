#标题
BBSwitch驱动_DSM参数类型不匹配导致PCIe总线错误及系统冻结

#内核版本
4.9.0-1-amd64 (Debian 4.9.2-2)

#问题现象
- 笔记本电脑（MSI GP72 6QE）在使用 Optimus 技术时，系统无法正常启动或在使用电池供电、运行 3D 应用时发生冻结/崩溃。
- dmesg 日志持续报错 PCIe 总线错误：`pcieport 0000:00:1c.0: PCIe Bus Error: severity=Corrected, type=Physical Layer, id=00e0(Receiver ID)`。
- 系统日志显示 ACPI 错误：`ACPI Error: Method parse/execution failed [\_SB.PCI0.LPCB.EC._REG]` 以及 `ACPI Error: Region EmbeddedControl (ID=3) has no handler`。
- 使用 `acpi=off` 引导参数虽可启动系统，但会导致触控板、电池信息等功能失效。

#问题根因
- 开发者分析确认，第三方模块 `bbswitch` 在调用 `_DSM` 方法时违反 ACPI 规范：传递了 Buffer 类型的参数，而规范要求第四个参数（Arg3）必须是 Package 类型。
- 日志中明确记录了该类型不匹配警告：`ACPI Warning: \_SB.PCI0.GFX0._DSM: Argument #4 type mismatch - Found [Buffer], ACPI requires [Package]`。
- 同时，EC（EmbeddedControl）区域在 `_REG` 方法执行期间无可用处理器，导致 ACPI 解析执行失败。

#解决方案
- 临时规避方案：在 GRUB 启动参数中添加 `acpi_osi=! acpi_osi="Windows 2009"`，可解决电池模式下的系统冻结问题。
- 由于 `bbswitch` 非内核上游模块，需联系 `bbswitch` 开发者修复 `_DSM` 参数类型问题。
- 用户反馈安装 `irqbalance` 服务后系统稳定性有所改善。
