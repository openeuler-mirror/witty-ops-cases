#标题
F2FS挂载损坏镜像触发非法块地址访问及内核告警

#内核版本
4.18.0-rc3+

#问题现象
- 挂载特制的 F2FS 镜像并执行文件操作（如 listxattr 或 ls）时，触发内核 WARNING。
- 内核日志显示 "F2FS-fs (loop0): access invalid blkaddr:1024" 及 "invalid blkaddr: 1024, type: 5, run fsck to fix"。
- Call Trace 显示在 `f2fs_is_valid_blkaddr` 和 `f2fs_iget` 中触发告警。
- Bug 摘要指出可能导致 `fscrypt_do_page_crypto` 处发生空指针解引用。

#问题根因
- 用户提供的 F2FS 镜像文件已损坏（包含无效块地址）。
- 内核在处理该损坏镜像的元数据（如查找条目或读取数据页）时，未正确处理无效的块地址，触发了 `f2fs_is_valid_blkaddr` 中的校验失败。
- 开发者分析指出，镜像未开启加密特性，但代码路径进入了加密处理流程，可能导致状态不一致。

#解决方案
- 该问题已被分配 CVE-2018-14616。
- 修复补丁已合入主线：https://git.kernel.org/linus/91291e9998d208370eb8156c760691b873bd7522。
