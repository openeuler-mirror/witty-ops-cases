#标题
e1000e驱动在网线断开状态下加载导致后续无法接收流量

#内核版本
> 3.11 (受影响版本包括 3.12)

#问题现象
- 在网线断开的情况下加载 e1000e 驱动模块，重新插上网线后，系统无法接收入站流量。
- DHCP 客户端无法获取 IP 地址，手动配置 IP 地址后仍无法通信。
- 客户端抓包显示发出 DHCP 请求，路由端抓包确认收到请求并回复，但客户端内核未收到回复包。
- 若加载驱动时网线已连接，则网络功能正常。

#问题根因
经 git bisect 定位，问题由 commit da1e2046e5f5ab268e55d30d6b74099ade0aeb6f ("e1000e: iAMT connections drop on driver unload when jumbo frames enabled") 引入。
具体原因为该补丁在 `e1000e_down` 函数中新增了对 `e1000_lv_jumbo_workaround_ich8lan` 函数的调用。该函数会禁用 RX 路径，执行相关操作后再重新启用。推测在特定触发条件下，RX 路径未能正确重新启用，导致驱动无法接收数据。

#解决方案
暂无（Bug 状态为 NEW，尚未合并修复补丁）
