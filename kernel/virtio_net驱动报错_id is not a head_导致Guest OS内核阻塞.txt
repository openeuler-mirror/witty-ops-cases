#标题
virtio_net驱动报错"id is not a head"导致Guest OS内核阻塞

#内核版本
3.18~4.0

#问题现象
- Guest OS 内核打印错误日志："virtio_net virtio0: input.0:id 0 is not a head!"
- 内核发生阻塞。
- 触发场景：启动虚拟机（2 vcpus）后执行 attach interface，随后执行 detach interface。
- 堆栈信息显示异常发生在 `virtnet_poll` 函数，调用链为 `net_rx_action` -> `__do_softirq` -> `do_IRQ`。

#问题根因
由提交 `78fd1d0ab072d4d9b5f0b7c14a1516665170b565` 引入的回归问题，导致 `virtnet_poll` 处理过程中出现缓冲区描述符状态异常（报错 "id is not a head"）。

#解决方案
回退提交 `78fd1d0ab072d4d9b5f0b7c14a1516665170b565` 可规避此问题；上游暂无正式修复补丁。
