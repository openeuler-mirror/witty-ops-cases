#标题
vesafb占用帧缓冲区导致OpenChrome DDX启动失败

#内核版本
4.5

#问题现象
- 使用 Linux 4.5 及以上内核版本时，OpenChrome DDX 无法正常启动 X Server
- OpenChrome 驱动加载时无法分配帧缓冲区
- 该问题在 Arch Linux 和 Fedora 23 上被观察到

#问题根因
vesafb 驱动程序声明了 VIA IGP 的帧缓冲区区域，导致 X Server 加载 OpenChrome 驱动时，因帧缓冲区资源已被 vesafb 占用而分配失败。

#解决方案
暂无内核层面的正式修复。临时规避方法为禁止加载 vesafb 驱动。OpenChrome 驱动侧已在 Version 0.4.186 及以后版本修复了帧缓冲区分配失败时的错误处理问题。
