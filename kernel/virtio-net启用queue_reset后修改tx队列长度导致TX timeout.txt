#标题
virtio-net启用queue_reset后修改tx队列长度导致TX timeout

#内核版本
6.14.0-rc7

#问题现象
- 在Guest中执行 `ethtool -G eth0 tx 8` 修改发送队列长度后，ping外部主机失败，提示“Destination Host Unreachable”。
- 系统日志反复打印 NETDEV WATCHDOG 告警：`transmit queue 0 timed out 5004 ms`。
- 出现 TX timeout 报错：`virtio_net virtio1 eth0: TX timeout on queue: 0, sq: output.0, vq: 0x1, name: output.0`。

#问题根因
未明确
