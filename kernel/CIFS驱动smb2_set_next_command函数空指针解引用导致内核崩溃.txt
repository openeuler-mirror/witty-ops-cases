#标题
CIFS驱动smb2_set_next_command函数空指针解引用导致内核崩溃

#内核版本
v5.2.0-rc6+, v5.1

#问题现象
在CIFS文件系统上运行lustre-racer测试套件时，系统触发kernel panic/Oops，复现概率约20%。
日志关键信息：
- BUG: kernel NULL pointer dereference, address: 0000000000000014
- RIP: 0010:smb2_set_next_command+0x34/0x170 [cifs]
- Oops: 0002 [#1] SMP PTI
- 前置告警包含 "CIFS VFS: Cancelling wait for mid ..." 和 "CIFS VFS: Close unmatched open"

#问题根因
在执行文件时间属性修改操作（utimensat系统调用）时，CIFS模块进入smb2_compound_op处理流程。在调用smb2_set_next_command函数时，由于RAX寄存器值为0，导致访问偏移地址0x14时触发空指针解引用（supervisor write access），进而引发内核崩溃。

#解决方案
暂无（Bug状态为NEW，尚未合并修复补丁）。
