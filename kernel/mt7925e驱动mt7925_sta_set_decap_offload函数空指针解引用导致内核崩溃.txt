#标题
mt7925e驱动mt7925_sta_set_decap_offload函数空指针解引用导致内核崩溃

#内核版本
6.11.8, 6.12, 6.12.3, 6.14.0-rc1

#问题现象
- 使用 mt7925e 网卡在 AP 模式下运行 hostapd 时，系统触发 kernel panic/oops。
- 内核日志显示 "BUG: unable to handle page fault for address: ffffffffffffffa0" 或 "Unable to handle kernel paging request at virtual address ffffffffffffffa0"。
- RIP/PC 指针位于 `mt7925_sta_set_decap_offload` 函数。
- 调用栈包含：`ieee80211_check_fast_rx` -> `_sta_info_move_state` -> `sta_apply_auth_flags` -> `ieee80211_add_station` -> `nl80211_new_station`。

#问题根因
在函数 `mt7925_sta_set_decap_offload` 中发生了空指针解引用。根据 x86_64 架构下的日志，寄存器 RAX 值为 0，CR2 错误地址为 `ffffffffffffffa0`（即 `0x0 - 0x60`），表明代码试图访问空指针偏移 -0x60 处的内存成员。该问题在添加无线站点触发 `nl80211_new_station` 流程时触发。

#解决方案
暂无
