#标题
amdgpu驱动从睡眠唤醒时PSP恢复失败导致系统死锁

#内核版本
5.2.9（回归引入于 v5.1 与 v5.2 之间）

#问题现象
- 系统从睡眠状态唤醒时黑屏，无法响应键盘/鼠标输入，系统挂死。
- 内核日志中出现大量报错：`[drm:amdgpu_job_run [amdgpu]] *ERROR* Error scheduling IBs (-22)`。
- 日志显示无法调度 IB：`amdgpu 0000:05:00.0: couldn't schedule ib on ring <gfx>`。
- 开启调试选项后可见 PSP 恢复失败链：`psp command failed` -> `PSP load tmr failed!` -> `PSP resume failed` -> `resume of IP block <psp> failed -22`。
- 受影响硬件包括 AMD Ryzen 5 2500U、3500U、Ryzen 7 2700U 及 4700u 等 APU 平台。

#问题根因
经 Bisect 定位，问题引入于合并提交 `4f8b49092c37cf0c87c43bb2698d43c71cf0e4e5`。根本原因为 amdgpu 驱动在恢复过程中，PSP（Platform Security Processor）硬件或固件初始化命令执行失败（response status 为 0），导致 TMR（Trusted Memory Region）加载错误，进而返回 -EINVAL (-22)，致使设备恢复流程中断。

#解决方案
暂无明确的修复补丁信息。根据最新反馈，该问题在较新的内核版本中已不再复现，推测已被后续版本迭代修复。临时规避方法为：在挂起前切换至文本控制台（如 Ctrl+Alt+F4），唤醒后再切回图形界面或重启显示管理器。
