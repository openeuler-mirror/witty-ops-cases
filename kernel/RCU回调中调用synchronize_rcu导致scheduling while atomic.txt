#标题
RCU回调中调用synchronize_rcu导致scheduling while atomic

#内核版本
4.11.7

#问题现象
- 系统关机时无法正常断电。
- 内核日志报错：BUG: scheduling while atomic: rcuc/5/41/0x00000201。
- Call Trace 显示调用链为：synchronize_rcu <- blk_queue_bypass_start <- blkcg_deactivate_policy <- blk_throtl_exit <- blkcg_exit_queue <- blk_release_queue <- __blkg_release_rcu <- rcu_do_batch。

#问题根因
在块层释放队列的过程中，RCU 回调函数 __blkg_release_rcu 触发了 blk_release_queue 调用链，最终在 blk_queue_bypass_start 函数中调用了 synchronize_rcu。由于 RCU 回调运行在原子上下文中，不允许睡眠，而 synchronize_rcu 会睡眠等待 RCU 宽限期结束，从而触发 scheduling while atomic 错误。

#解决方案
暂无
