#标题
asus-nb-wmi驱动因散热策略返回值检查错误导致加载失败及功能缺失

#内核版本
6.12-rc4, 6.12-rc7, 6.12

#问题现象
- 升级至 6.12 内核后，ASUS 笔记本（如 Vivobook S14）的 `/sys/class/power_supply/BAT0/charge_control_end_threshold` 节点缺失，无法设置充电阈值。
- 系统日志显示 `asus-nb-wmi` 驱动加载失败，报错 "probe with driver asus-nb-wmi failed with error -5"。
- 日志中出现 "Failed to set throttle thermal policy (retval): 0x0" 和 "Failed to set default thermal profile" 错误信息。
- 部分机型伴随键盘背光控制、触摸板切换按键失效等问题。

#问题根因
内核 6.12 版本中新引入的 platform profile 支持功能存在逻辑缺陷。驱动在初始化 `throttle_thermal_policy` 时，调用的 ACPI 方法返回了传入的策略参数值而非预期的 0。驱动代码严格检查返回值是否为 0，导致误判 ACPI 调用失败，进而导致整个 `asus-nb-wmi` 驱动 probe 失败，相关功能（如充电控制、热键等）全部不可用。

#解决方案
该问题已在内核版本 6.13-rc2 及 6.12.5 中修复。升级内核至上述版本或应用相关补丁可解决该问题。
