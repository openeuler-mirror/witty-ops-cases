#标题
AMDGPU驱动fence_default_wait阻塞导致Xorg进程卡死及屏幕冻结

#内核版本
4.7.0

#问题现象
- 屏幕冻结无响应，但系统仍可远程 SSH 登录。
- 内核日志报错：INFO: task Xorg:539 blocked for more than 120 seconds。
- Xorg 进程处于 D 状态（不可中断睡眠），调用栈显示阻塞在 `fence_default_wait` -> `schedule_timeout`。
- 尝试通过 `/sys/kernel/debug/dri/0/amdgpu_gpu_reset` 重置 GPU 时触发 Kernel Oops（NULL pointer dereference）。

#问题根因
Xorg 进程在调用 `amdgpu_cs_ioctl` 接口时，因等待 fence 信号超时，长期阻塞在 `fence_default_wait` 函数中，导致图形界面冻结。该问题在 CPU 负载较高时更容易触发。此外，GPU 重置逻辑中存在空指针解引用缺陷。

#解决方案
暂无官方修复补丁。尝试更换为 `_k_` 版本的 GPU 固件可降低故障发生频率。
