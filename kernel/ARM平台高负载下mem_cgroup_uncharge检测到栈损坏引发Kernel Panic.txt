#标题
ARM平台高负载下mem_cgroup_uncharge检测到栈损坏引发Kernel Panic

#内核版本
5.12.0

#问题现象
- 系统触发 Kernel panic，提示 "stack-protector: Kernel stack is corrupted in: mem_cgroup_uncharge+0x7c/0x88"。
- 故障发生在内核编译高负载场景下（执行 "make -j6 Image"）。
- 平台为 ARM 架构。
- 调用栈显示路径为：`mem_cgroup_uncharge` <- `__put_page` <- `put_page` <- `free_page_and_swap_cache` <- `tlb_remove_table_rcu` <- `rcu_core`。

#问题根因
内核栈保护器在 `mem_cgroup_uncharge` 函数返回时检测到栈帧被破坏从而触发 panic。具体的内存破坏深层原因（如缓冲区溢出或野指针写入）在现有报告中未明确。

#解决方案
暂无
