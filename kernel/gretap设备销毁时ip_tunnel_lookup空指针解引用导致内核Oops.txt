#标题
gretap设备销毁时ip_tunnel_lookup空指针解引用导致内核Oops

#内核版本
3.11.10.4

#问题现象
- 在销毁LXC容器及其关联的gretap接口时触发内核Oops。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 000000fc"。
- 故障调用栈显示：`ip_tunnel_lookup` -> `ipgre_rcv` -> `gre_cisco_rcv`。

#问题根因
在 `ip_tunnel_lookup` 函数中，访问 `ip_tunnel` 结构体的 `dev` 成员时，该指针为 NULL，导致解引用失败触发 Oops。

#解决方案
提交者提供了补丁（附件129391），建议在解引用前增加 `t->dev` 的非空检查，可规避该崩溃问题。但社区建议进一步排查 `t->dev` 被置为 NULL 的根本原因。
