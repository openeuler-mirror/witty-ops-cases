#标题
gsmi 模块加载失败引发内核空指针解引用导致系统启动异常

#内核版本
5.3.8

#问题现象
- 系统启动过程中内核打印大量日志信息，干扰 TTY 登录及 OpenRC 启动显示。
- 系统无法正常挂起，udevd 拒绝冻结。
- dmesg 显示错误日志：`gsmi: failed to allocate name buffer` 与 `gsmi: failed to load: -12`。
- 触发 Kernel Oops：`BUG: kernel NULL pointer dereference, address: 0000000000000030`，RIP 寄存器指向 `__device_attach_driver+0x1a/0xd0`。

#问题根因
发行版在内核配置中启用了 Google SMI (gsmi) 和 coreboot memconsole 选项。在 Coreboot 环境下，gsmi 驱动尝试分配 name buffer 失败（返回错误码 -12），随后在驱动加载或设备附加流程中触发了空指针解引用。

#解决方案
- 临时规避：将 gsmi 模块加入黑名单。
- 发行版处理：Arch Linux 在后续版本（5.3.9）中已禁用 Google SMI 配置项。
- 上游修复：暂无明确的内核补丁信息，相关代码问题仍存在。
