#标题
get_wchan调用__unwind_start触发缺页异常导致内核崩溃

#内核版本
5.15.3（受影响版本包括 5.14.19、5.15.2）

#问题现象
- 内核日志报错 "BUG: unable to handle page fault for address: ..."，错误码显示为 supervisor read access in kernel mode。
- 触发 Oops，RIP 寄存器指向 `__unwind_start+0x10b/0x1e0`。
- 调用链显示为 `__get_wchan` -> `get_wchan` -> `do_task_stat` -> `proc_single_show` -> `seq_read` -> `vfs_read`。
- 系统发生崩溃。
- 复现条件：在 Chrome 浏览器中发起 Google Meet 通话，或系统启动时执行 NFS 初始化脚本中的 `pidof` 命令。

#问题根因
该问题由提交 `5d1ceb3969b6b2e47e2df6d17790a7c5a20fcbb4` 引入。在 `get_wchan` 函数调用 `unwind_start` 进行栈回溯初始化时，访问了无效的内存地址，导致触发 page fault。用户验证移除该提交后问题消失。

#解决方案
暂无官方修复补丁。临时规避方法是回退提交 `5d1ceb3969b6b2e47e2df6d17790a7c5a20fcbb4`。
