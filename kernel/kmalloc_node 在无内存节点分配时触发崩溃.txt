#标题
kmalloc_node 在无内存节点分配时触发崩溃

#内核版本
3.1.0-7

#问题现象
调用 kmalloc_node 函数尝试在没有内存的节点（例如在仅有 Node 0 内存的系统上指定 Node 1）上分配内存时，内核发生崩溃。

#问题根因
kmalloc_node 底层调用 ____cache_alloc_node 时，未对传入的节点 ID 进行有效性校验。当节点 ID 指向不存在或无内存的节点时，函数未返回 NULL 指针，而是触发了内核崩溃。

#解决方案
该问题已在 Linux v3.11-rc1 版本中修复。相关补丁增加了对 nodeid 有效性的检查，在调试模式下若 nodeid > num_online_nodes() 则触发断言，防止崩溃。
补丁链接：http://git.kernel.org/cgit/linux/kernel/git/torvalds/linux.git/commit/?id=14e50c6a9bc2b283bb4021026226268312ceefdd
