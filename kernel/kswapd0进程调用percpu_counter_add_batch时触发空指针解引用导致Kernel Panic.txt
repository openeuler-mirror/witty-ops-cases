#标题
kswapd0进程调用percpu_counter_add_batch时触发空指针解引用导致Kernel Panic

#内核版本
5.15.34.3

#问题现象
- 系统触发 Kernel Panic，日志显示 "Unable to handle kernel NULL pointer dereference at virtual address 00000050"
- 触发 Oops 的进程为 kswapd0 (PID: 73)，架构为 ARM SMP
- 系统在 10 秒后自动重启

#问题根因
内核在执行内存回收路径时，函数 `clear_page_dirty_for_io` 调用 `percpu_counter_add_batch` 过程中，访问了非法地址 0x50（寄存器 r0 为 0x38，推测访问了偏移量 0x18），导致空指针解引用。调用链为：kswapd -> shrink_node -> shrink_lruvec -> shrink_inactive_list -> shrink_page_list -> clear_page_dirty_for_io -> percpu_counter_add_batch。

#解决方案
暂无
