#标题
r8152驱动RTL8153网卡连接中断并报Tx status -71错误

#内核版本
4.16-rc2（报告版本），受影响版本包括 4.15 至 5.11

#问题现象
- 使用 RTL8153 芯片的 USB 网卡（常见于 Dell WD15 Dock、Lenovo USB-C Dock 等）在网络负载较高或长时间运行后连接中断。
- 内核日志频繁打印错误信息：`r8152 ... Tx status -71` 或 `Tx status -2`、`Rx status -71`。
- 触发 `NETDEV WATCHDOG` 警告：`transmit queue 0 timed out`，调用栈显示在 `dev_watchdog` 函数。
- 必须重新插拔设备或执行 `rmmod r8152 && modprobe r8152` 才能恢复网络。

#问题根因
- 根据社区分析，问题可能源于 RTL8153 芯片固件或硬件设计缺陷。
- 在高吞吐量场景下，RTL8153 芯片无法跟上驱动下发的传输队列（tx_queue）速度，停止触发状态中断，导致 `write_bulk_callback` 无法执行，进而触发 netdev watchdog 超时。
- USB 链路电源管理（LPM）功能可能加剧了该问题的发生频率。

#解决方案
目前尚无彻底的内核代码修复，建议尝试以下规避方案：
1. 禁用该设备的 USB LPM 功能：在内核启动参数中添加 `usbcore.quirks=0bda:8153:k`。
2. 禁用 USB 自动挂起：在内核启动参数中添加 `usbcore.autosuspend=-1`。
3. 尝试组合参数：`usbcore.old_scheme_first=1 usbcore.use_both_schemes=0 usbcore.autosuspend=-1 pci_aspm=off`。
4. 调整 Network Manager 设置，将链路协商（Link Negotiation）从 Ignore 改为 Auto 或手动指定速率。
