#标题
FUSE 模块触发 list_del corruption 导致内核崩溃

#内核版本
5.15.3

#问题现象
- 系统运行中频繁触发 kernel BUG，导致系统不可用。
- 日志显示 "list_del corruption. next->prev should be ffffdfd4c8b732c8, but was ffff98adb59f5830"。
- 触发 invalid opcode (kernel BUG) 位于 `lib/list_debug.c:54`。
- Call Trace 关键路径：`release_pages` -> `__pagevec_release` -> `truncate_inode_pages_range` -> `fuse_evict_inode` -> `fuse_reverse_inval_entry` -> `fuse_dev_do_write`。
- 另有报告在 arm64 设备上出现 "Bad page state" 及 "virt_to_cache: Object is not a Slab page!" 警告，提示存在内存损坏。

#问题根因
问题触发于 FUSE 文件系统处理 inode 释放流程中，`release_pages` 函数检测到链表节点指针异常。具体根因未明确，可能与内存管理子系统在特定条件下的并发或内存损坏有关。

#解决方案
暂无修复补丁。报告者反馈该问题在 5.16 内核版本中无法复现，建议尝试升级内核版本。
