#标题
Intel THC驱动缺失设备ID 8086:7749导致触摸屏不可用及运行时错误

#内核版本
6.17（测试涉及 6.16.5、6.16.8、6.6.18）

#问题现象
- Lenovo Yoga Pro 9i 触摸屏无法使用，设备 8086:7749 未被内核识别。
- 手动绑定 `intel_quicki2c` 驱动失败，报错 `Get ACPI resources failed, ret = -95`。
- 初步修复后出现新问题：冷启动后触摸屏无响应，需手动恢复；出现间歇性超时 `Wait Get Report Response timeout`、`DMA write failed, ret = -110`。
- 日志显示 Runtime PM 计数下溢：`Runtime PM usage count underflow`。
- 日志显示 ACPI 错误：`\_SB.PC00.THC0._RST: Excess arguments - Caller passed 1, ACPI requires 0`。

#问题根因
1. Intel THC 驱动（`intel_quickspi`/`intel_quicki2c`）的设备表中缺少设备 ID 8086:7749，导致硬件无法匹配驱动。
2. ACPI 调用 `_RST` 方法时参数传递错误，固件要求 0 个参数，但调用者传递了 1 个。
3. `intel_quickspi` 驱动在 probe/resume 流程中存在 Runtime PM 引用计数不平衡问题。

#解决方案
1. 应用补丁（Bug 附件 308697）将设备 ID 8086:7749 添加到驱动支持列表。
2. 应用更新补丁（Bug 附件 308709）修复 ACPI `_RST` 参数传递错误，解决大部分启动和功能问题。
3. 针对残留的 Runtime PM usage count underflow 警告，可通过 udev 规则设置 `power/control` 为 `on` 规避冷启动失效问题，等待上游进一步修复。
