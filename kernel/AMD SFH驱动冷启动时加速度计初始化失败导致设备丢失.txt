#标题
AMD SFH驱动冷启动时加速度计初始化失败导致设备丢失

#内核版本
6.16.6-3, 6.17.0-rc5, 6.18.0-rc1

#问题现象
- Lenovo Yoga 7 笔记本冷启动进入 Linux 时，内置加速度计无法被检测到。
- 系统中 `/sys/bus/iio/devices` 下无加速度计设备节点，`monitor-sensor` 无数据。
- 若先启动 Windows 再进入 Linux，或 Linux 关机后立即开机，设备可正常识别；重启 Linux 或长时间关机后再开机则复现故障。
- dmesg 日志显示 "Failed to enable sensor 0: -1" 以及 "sid 0x0 (accelerometer) status 0x5"。

#问题根因
问题出在 `drivers/hid/amd-sfh-hid/sfh1_1/amd_sfh_init.c` 文件的 `amd_sfh1_1_hid_client_init` 函数中。在冷启动时，传感器硬件可能处于未知的初始状态，驱动直接调用 `mp2_ops->start()` 启动传感器会失败（等待响应超时或返回错误状态）。对比测试发现，在卸载模块（触发 stop）后再加载模块（触发 start）能成功初始化，表明需要在启动前先执行停止操作以重置硬件状态。

#解决方案
修改 `drivers/hid/amd-sfh-hid/sfh1_1/amd_sfh_init.c`，在 `amd_sfh1_1_hid_client_init` 函数中调用 `mp2_ops->start()` 之前，显式调用 `mp2_ops->stop()` 和 `amd_sfh_wait_for_response(..., DISABLE_SENSOR)` 来重置传感器状态。用户验证该补丁后冷启动加速度计工作正常。
