#标题
ARM平台调用Socket CAN API时因copy_from_user参数异常导致内核崩溃

#内核版本
4.1 and 4.1 after

#问题现象
使用 Socket CAN API 时系统发生内核崩溃；故障发生时传入参数 arg=0，且 .fixup 异常处理机制未生效。

#问题根因
`copy_from_user` 函数未对传入指针参数进行有效性检查（如 arg=0 的情况），直接调用 `__copy_from_user`；同时汇编文件中 `.fixup` 段的定义位置不正确（使用了 `.fixup` 而非 `.text.fixup`），导致异常处理代码无法被正确链接或执行，无法捕获该错误进而引发崩溃。

#解决方案
用户建议通过以下修改规避问题：
1. 在 `arch/arm/include/asm/uaccess.h` 的 `copy_from_user` 函数中增加对 `from` 指针的非空检查。
2. 在 `arch/arm/lib/copy_from_user.S` 中将 `.pushsection .fixup,"ax"` 修改为 `.pushsection .text.fixup,"ax"`。
