#标题
SD卡读写时IOMMU分配IOVA失败导致数据损坏及内核告警

#内核版本
4.0.0

#问题现象
- SD卡数据损坏。
- 内核日志报错：`Allocating 16-page iova for 0000:0c:00.0 failed`。
- 触发内核 WARNING：`WARNING: CPU: 0 PID: 0 at drivers/mmc/host/sdhci.c:857 sdhci_send_command+0xa8b/0xb90()`。
- DMAR 报错：`DMAR:[DMA Write] Request device [0c:00.0] fault addr 0 ... PTE Write access is not set`。

#问题根因
系统为设备 0000:0c:00.0 分配 16 页 iova 失败，导致 DMA 映射异常，进而触发 DMAR 故障和 sdhci_send_command 中的告警。

#解决方案
暂无（Bug处于NEW状态，未提供修复方案）
