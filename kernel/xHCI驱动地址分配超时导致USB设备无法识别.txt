#标题
xHCI驱动地址分配超时导致USB设备无法识别

#内核版本
3.5.0-23-generic (Ubuntu Precise)

#问题现象
- Android设备在刷写或重启后，主机无法检测到USB设备，需重启主机或切换USB端口才能恢复。
- dmesg日志显示xHCI控制器超时："xhci_hcd 0000:00:14.0: Timeout while waiting for address device command"。
- 内核触发WARNING，调用栈指向 `xhci_address_device+0x2c8/0x2f0`。
- 后续报错："device not accepting address ..., error -22" 以及 "couldn't allocate port ... usb_device"。

#问题根因
xHCI主机控制器驱动在执行地址设备命令时发生超时，导致设备无法接受分配的地址。日志中伴随出现 "Virt dev invalid for slot_id"，表明驱动内部维护的虚拟设备状态异常。由于内核版本较旧，上游未提供具体的代码级根因分析。

#解决方案
上游维护者指出内核版本 3.5 已过时且不再受社区支持，建议升级至最新内核版本（如 3.18+）验证，或向 Ubuntu 发行版官方提交 Bug。临时规避措施为更换 USB 端口或重启系统。
