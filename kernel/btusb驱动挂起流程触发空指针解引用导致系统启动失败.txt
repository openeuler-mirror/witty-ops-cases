#标题
btusb驱动挂起流程触发空指针解引用导致系统启动失败

#内核版本
6.11, 6.12.3

#问题现象
- 系统启动过程中偶发 Kernel Oops，导致启动卡死或图形界面登录后无法进入会话。
- 日志显示 `BUG: kernel NULL pointer dereference, address: 0000000000000000`。
- 崩溃发生在 `btusb_suspend` 函数，寄存器 RBX 为 0，导致解引用错误。
- 调用栈显示流程为 `pm_runtime_work` -> `usb_runtime_suspend` -> `usb_suspend_both` -> `btusb_suspend`。

#问题根因
btusb 驱动在执行运行时电源管理挂起操作时，`btusb_suspend` 函数内部访问了空指针。这是一个回归问题，可能与 USB 自动挂起机制的并发或状态检查缺失有关。

#解决方案
- 临时规避：在内核启动参数中添加 `btusb.enable_autosuspend=0` 禁用蓝牙 USB 自动挂起。
- 上游修复：开发者指出提交 `6f3f7e9414834fc4210a2d11ff6172031e98d9ff` 已修复该问题，建议升级至包含该补丁的内核版本。
