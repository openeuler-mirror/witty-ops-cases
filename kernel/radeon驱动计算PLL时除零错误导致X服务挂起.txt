#标题
radeon驱动计算PLL时除零错误导致X服务挂起

#内核版本
3.15-rc3

#问题现象
- X启动过程中内核触发除零错误，导致X服务挂起，需重启恢复。
- 日志显示 "divide error: 0000 [#1]"，RIP 位于 `radeon_compute_pll_avivo+0xb6`。
- 该问题在 xserver 与 xf86-video-ati 驱动版本不匹配时触发。

#问题根因
用户态传递了无效的时钟参数（`target_clock` 为 0）给内核。内核态函数 `radeon_compute_pll_avivo` 在执行 `post_div_min = vco_min / target_clock` 运算前，未对除数进行有效性检查，直接触发了除零异常。

#解决方案
开发者提供了修复补丁，增加了参数校验逻辑。修复后，当传入无效参数时内核不再崩溃，而是返回错误（如 -EINVAL），并在日志中输出 "failed to set mode" 错误信息。
