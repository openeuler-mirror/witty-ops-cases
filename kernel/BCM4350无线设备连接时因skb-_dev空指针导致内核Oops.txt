#标题
BCM4350无线设备连接时因skb->dev空指针导致内核Oops

#内核版本
v4.7-rc1

#问题现象
- BCM4350 无线设备连接网络时立即触发内核 Oops。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000048"。
- 崩溃点位于 enqueue_to_backlog+0x56/0x230，调用栈显示路径为 brcmf_pcie_isr_thread -> brcmf_msgbuf_process_rx -> brcmf_netif_rx -> netif_rx_ni -> enqueue_to_backlog。

#问题根因
在 enqueue_to_backlog 函数中访问 skb->dev 时，该指针为 NULL。该问题为 v4.6-rc7 之后的回归错误。

#解决方案
该问题已通过 Git 提交 31143e2933 修复。
