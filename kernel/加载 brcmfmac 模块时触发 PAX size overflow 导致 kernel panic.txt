#标题
加载 brcmfmac 模块时触发 PAX size overflow 导致 kernel panic

#内核版本
4.4.4 (grsec patched)

#问题现象
- 在应用了 grsec 补丁的内核启动过程中或手动加载模块时发生 kernel panic。
- 系统日志显示错误：PAX: size overflow detected in function pkt_align drivers/net/wireless/brcm80211/brcmfmac/sdio.c:733。

#问题根因
grsecurity 补丁中的 PAX 安全特性检测到 brcmfmac 驱动中 pkt_align 函数存在整数溢出风险，具体位于 drivers/net/wireless/brcm80211/brcmfmac/sdio.c 文件第 733 行，从而触发 panic 机制。

#解决方案
暂无上游修复补丁。可通过禁用 PAX 的 "Prevent various integer overflows in function size parameter"（防止函数大小参数中的各种整数溢出）特性来规避此问题。
