#标题
stress-ng测试触发ReiserFS断言失败导致内核panic

#内核版本
Linux 6.12.0-rc7

#问题现象
- 运行 stress-ng fsize 压力测试时触发 kernel panic
- 内核日志显示：REISERFS panic: assertion failure check_internal_node: (!(!bh)) at fs/reiserfs/do_balan.c:1616
- 日志包含：kernel BUG at fs/reiserfs/prints.c:390! 与 Oops: invalid opcode
- 调用栈包含：__reiserfs_panic -> check_internal_node -> do_balance -> reiserfs_paste_into_item

#问题根因
ReiserFS 在执行 do_balance 树平衡操作过程中，check_internal_node 函数内部断言检查失败。日志显示 "bh == 0"，即缓冲头指针为空，触发了 fs/reiserfs/do_balan.c:1616 处的断言失败，进而调用 __reiserfs_panic 导致系统崩溃。

#解决方案
暂无
