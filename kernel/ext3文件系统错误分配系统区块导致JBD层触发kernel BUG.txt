#标题
ext3文件系统错误分配系统区块导致JBD层触发kernel BUG

#内核版本
3.0.1-gentoo

#问题现象
- 系统日志大量报错：EXT3-fs error (device dm-0): ext3_new_block: Allocating block in system zone
- 系统日志报错：EXT3-fs error (device dm-0): ext3_valid_block_bitmap: Invalid block bitmap
- 触发 kernel BUG：kernel BUG at fs/jbd/transaction.c:1155!
- 进程 mv 触发 invalid opcode (0000 [#1])，Call Trace 指向 journal_dirty_metadata -> __ext3_journal_dirty_metadata -> ext3_try_to_allocate_with_rsv

#问题根因
在使用 4k 扇区 GPT 分区及 LUKS 加密的 3TB USB 硬盘上进行文件操作时，ext3 块分配器出现异常，试图从系统保留区域分配数据块。错误的块分配导致元数据状态不一致，进而触发 JBD 层 journal_dirty_metadata 函数中的 BUG()（通常为断言失败）。

#解决方案
暂无官方修复方案。用户反馈 fsck 可能导致文件系统进一步损坏。建议排查大容量存储设备（特别是 4k 扇区盘）的分区对齐情况或文件系统元数据完整性。
