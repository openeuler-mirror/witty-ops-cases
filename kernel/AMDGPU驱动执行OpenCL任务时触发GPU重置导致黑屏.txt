#标题
AMDGPU驱动执行OpenCL任务时触发GPU重置导致黑屏

#内核版本
5.10.5-gentoo

#问题现象
- 运行 OpenCL 任务（如 Tesseract OCR）时屏幕黑屏，连接的显示器无信号，但系统网络仍可达。
- 内核日志显示 `amdgpu_dm_atomic_commit_tail` 等待 fences 超时。
- 触发 `ring gfx timeout`，导致 GPU reset 开始并成功执行。
- GPU 重置过程中，函数 `dc_commit_state` 触发 WARNING。
- 重置后 `amdgpu_ib_ring_tests` 报告 gfx ring 测试失败（错误码 -110）。

#问题根因
GPU ring gfx 发生超时，触发 `amdgpu_job_timedout` 并启动 GPU reset 流程；在重置恢复阶段，`dc_commit_state` 函数触发了 WARNING，且 gfx ring 的 IB test 失败。具体导致 ring gfx 超时的根本原因未明确。

#解决方案
暂无。评论中建议排查内核是否使用 clang 编译（可能导致兼容性问题），但报告者确认使用 gcc 编译，问题未解决。
