#标题
VMX MSR不一致导致kvm_intel模块启动加载失败

#内核版本
5.15.29, 5.15.97, 6.1

#问题现象
- 系统启动时内核日志报错 "kvm: CPU 0 feature inconsistency!"
- 手动执行 `modprobe kvm_intel` 失败，提示 "modprobe: ERROR: could not insert 'kvm_intel': Input/output error"
- 系统经过挂起并唤醒后，`kvm_intel` 模块可正常加载，KVM 功能恢复正常。

#问题根因
CPU 微码版本过旧（当前为 0x39，最新为 0x49），导致不同 CPU 核心间的 VMX MSR (0x48b) 值不一致。启动后检查发现 MSR 0x48b 存在两种不同的值（`2007fff00000000` 和 `7fff00000000`），KVM 检测到 CPU 特性不一致从而拒绝加载 kvm_intel 模块。挂起恢复操作可能重置或同步了微码状态，使 MSR 值暂时统一。

#解决方案
更新 CPU 微码至最新版本（建议 0x49 或更高）以解决 CPU 间特性不一致问题。
