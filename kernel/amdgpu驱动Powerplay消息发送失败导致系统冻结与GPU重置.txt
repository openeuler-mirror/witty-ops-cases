#标题
amdgpu驱动Powerplay消息发送失败导致系统冻结与GPU重置

#内核版本
5.3.0-rc3-drm-next（问题亦见于 5.4-rc7, 5.5.14, 5.7.0）

#问题现象
- 系统启动或运行过程中内核日志持续报错："amdgpu: [powerplay] failed send message ... response 0xffffffc2"。
- 启动阶段出现长时间延迟（约2分钟），GPU传感器（fan1_input, temp1_input等）读取失败并返回 I/O error。
- 在多显示器环境（特别是多 DisplayPort 连接）下易触发故障，可能导致 X Server 加载延迟。
- 高负载下 GPU 过热，触发 GPU reset，日志中出现 "suspend of IP block <powerplay> failed -22" 及大量 "AMD-Vi: Completion-Wait loop timed out" 错误。
- 并发读取传感器信息或切换屏幕模式时可能引发系统死锁或冻结。

#问题根因
根据日志分析，问题源于 amdgpu 驱动的 powerplay 模块与 SMU（System Management Unit）通信失败。用户分析指出，内核态与用户态在访问 powerplay 接口（如传感器读取）时可能存在缺少互斥锁（mutex）保护的竞态条件，导致 SMU 状态异常或响应超时，进而引发电源管理命令发送失败和系统冻结。

#解决方案
暂无官方正式修复。
临时规避方法：
1. 强制 DPM 性能级别：执行 `echo "low" > /sys/class/drm/card0/device/power_dpm_force_performance_level`。
2. 启动时仅连接单个显示器，进入系统后再连接第二个显示器。
3. 有用户尝试在内核 5.4 中为 powerplay 相关操作添加 mutex 补丁以缓解问题。
