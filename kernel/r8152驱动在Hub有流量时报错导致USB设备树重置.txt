#标题
r8152驱动在Hub有流量时报错导致USB设备树重置

#内核版本
4.12.8-041208-generic

#问题现象
- 系统日志中出现 "r8152 ... Rx status -71" 错误，随后设备断开连接："USB disconnect, device number ..."。
- 当 USB Hub 上有其他设备流量时，Realtek 8153 设备会断开连接。
- 设备断开时会触发 Hub 重置，导致连接在同一 Hub 上的所有同级和子设备也被断开重置。
- 可能周期性发生 USB 总线重置（约每两周一次），影响所有连接的 USB 设备。
- 硬件支持 USB 3.0 (5 Gb/s)，但在 Linux 下仅工作在 USB 2.0 (480 Mb/s) 模式。

#问题根因
r8152 驱动在处理 Realtek 8153 芯片时存在稳定性问题。当 USB Hub 存在其他设备流量或特定条件下，驱动接收到异常状态（Rx status -71，即 EPROTO 协议错误），导致设备断开。该断开行为触发了 xhci_hcd 或 Hub 的重置逻辑，进而导致整个 USB 设备树重置。

#解决方案
暂无官方修复。临时规避方案：
1. 将 Realtek USB 设备连接到独立的 USB 端口，避免与其他设备共享 Hub。
2. 更换使用 ax88179_178a 驱动的其他品牌 USB 网卡设备。
