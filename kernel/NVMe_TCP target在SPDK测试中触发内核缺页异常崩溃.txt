#标题
NVMe/TCP target在SPDK测试中触发内核缺页异常崩溃

#内核版本
5.4.52

#问题现象
使用 SPDK perf (v20.04.X) 作为 host 连接 Linux 内核 target 进行 NVMe/TCP 性能测试时，触发内核崩溃。dmesg 日志显示 "BUG: unable to handle page fault for address: 0000000001eb1ca8"，错误类型为 supervisor read access in kernel mode，not-present page。Call Trace 显示崩溃发生在 `nvmet_tcp_try_recv_pdu` 函数中，调用链为 `nvmet_tcp_io_work` -> `process_one_work`。

#问题根因
未明确（故障发生在 `nvmet_tcp_try_recv_pdu` 函数偏移 0x432 处，尝试访问非法内存地址，具体原因待定）。

#解决方案
暂无
