#标题
ALX驱动触发致命中断导致高CPU占用及系统死锁

#内核版本
4.1.3, 4.2-rc4, 4.9.0, 5.3.11

#问题现象
- kworker 线程出现高 CPU 占用
- dmesg 日志被 "alx 0000:08:00.0 enp8s0: fatal interrupt 0x4019607, resetting" 等错误信息填满
- 系统触发 soft lockup，例如 "NMI watchdog: BUG: soft lockup - CPU#0 stuck for 22s!"
- 系统无响应，无法正常关机，需强制断电

#问题根因
用户通过 git bisect 定位到 commit 8954672d86d036643e3ce7ce3b2422c336db66d0，但回退该提交后问题依旧存在。开发者分析发现 4.0 至 4.4 版本间 alx 驱动代码无实质改动，推测问题由内核核心代码变更导致驱动适配器进入异常状态（寄存器读取失败），具体触发机制未明确。

#解决方案
暂无官方修复方案。尝试加载模块参数 `msix=1` 无效。个别用户反馈重置 BIOS/UEFI 设置后问题消失，但不确定是否为通用解决方案。
