#标题
关机时执行ACPI _PTS导致系统挂起并打印ATA错误

#内核版本
4.8.rc3 - 4.8.2

#问题现象
- 系统在关机过程中发生挂起
- 等待约90秒超时后打印ATA错误信息：ata exception Emask、ata Failed command FLUSH CACHE EXT
- 回退至 4.7.x 内核版本关机功能正常

#问题根因
内核引入的回归问题，由提交 "ACPI: Execute _PTS before system reboot" 导致。该提交在系统重启或关机前执行 ACPI _PTS（Prepare To Sleep）方法，引发了 ATA 设备异常。

#解决方案
升级内核至 4.8.12 版本；该问题通过回退提交 "ACPI: Execute _PTS before system reboot"（补丁ID: 9713adc2a1a5488f4889c657a0c0ce0c16056d3c）修复。
