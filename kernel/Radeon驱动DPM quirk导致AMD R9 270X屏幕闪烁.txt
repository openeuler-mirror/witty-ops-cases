#标题
Radeon驱动DPM quirk导致AMD R9 270X屏幕闪烁

#内核版本
4.1.6

#问题现象
- 系统从内核 4.0 升级至 4.1 后，屏幕出现严重闪烁，导致系统不可用
- dmesg 日志显示错误：`[drm:si_dpm_set_power_state [radeon]] *ERROR* si_restrict_performance_levels_before_switch failed`
- 使用 DisplayPort 连接时报错：`[drm:radeon_dp_link_train [radeon]] *ERROR* clock recovery failed`
- 添加内核启动参数 `radeon.dpm=0` 禁用 DPM 后问题消失

#问题根因
经 git bisect 定位，问题由 commit `5dfc71bc44d91d1620505c064fa22b0b3db58a9d` 引入。该提交标题为 "drm/radeon: add a dpm quirk for Sapphire Radeon R9 270X 2GB GDDR5"，旨在为特定型号的 Sapphire R9 270X 显卡添加 DPM quirk 以修复另一问题，但该改动意外导致用户的 AMD R9 270X 显卡出现兼容性问题。

#解决方案
临时规避方案：在 GRUB 内核启动参数中添加 `radeon.dpm=0`。
临时修复：回退或反向应用 commit `5dfc71bc44d91d1620505c064fa22b0b3db58a9d`。
官方修复：暂无（Bug 状态为 NEW）。
