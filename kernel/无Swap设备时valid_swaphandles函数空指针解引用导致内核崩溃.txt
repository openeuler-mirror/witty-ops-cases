#标题
无Swap设备时valid_swaphandles函数空指针解引用导致内核崩溃

#内核版本
3.0.101-0.47.52

#问题现象
- 系统触发内核 Oops，日志显示 "BUG: unable to handle kernel NULL pointer dereference at 000000000000000c"
- 故障发生在进程 nginx 上下文中
- Call Trace 显示调用路径为：valid_swaphandles -> swapin_readahead -> do_swap_page -> handle_pte_fault -> do_page_fault
- 系统环境存在大量空闲内存，且未配置任何 Swap 分区（nr_swapfiles=0, total_swap_pages=0）

#问题根因
在 valid_swaphandles 函数中，代码通过 `si = swap_info[swp_type(entry)]` 获取 swap_info_struct 指针。由于系统未配置 Swap 设备，swap_info 数组为空，导致 `si` 指针为 NULL。函数未对 `si` 进行有效性检查便访问其成员（如 `si->max`），从而触发空指针解引用。至于为何在无 Swap 设备的情况下仍进入 swapin 流程，原因未明确。

#解决方案
暂无
