#标题
VMD驱动域重置操作触发VT-d错误导致系统挂起

#内核版本
6.1-rc2

#问题现象
- 在 BIOS 中启用 VMD 后，主机操作系统无法启动，系统挂起。
- 串口日志显示 VMD 绑定到 PCI domain 10000 后，出现大量 DMAR 错误，包括 "VT-d detected Invalidation Queue Error"、"VT-d detected Invalidation Time-out Error" 以及 "VT-d detected Invalidation Completion Error"。
- 故障硬件平台为 Skylake-D Purley，VMD 控制器型号为 8086:201d。

#问题根因
Git bisect 定位到引入问题的提交为 6aab5622296b ("PCI: vmd: Clean up domain before enumeration")。该补丁在 `vmd_domain_reset()` 函数中引入了 `memset_io()` 操作以清理域配置，该操作在特定硬件环境下触发了 VT-d 错误，导致系统无法继续启动。

#解决方案
回退提交 6aab5622296b 可修复该问题；或注释掉 `vmd_domain_reset` 函数中的 `memset_io` 调用。截至报告时间，上游尚未合并正式修复补丁，开发者建议排查 BIOS 设置（如中断重映射）或确认 BIOS 版本兼容性。
