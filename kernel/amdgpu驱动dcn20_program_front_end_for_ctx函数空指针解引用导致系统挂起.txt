#标题
amdgpu驱动dcn20_program_front_end_for_ctx函数空指针解引用导致系统挂起

#内核版本
5.14.13-100.fc33.x86_64

#问题现象
- 系统发生 AMD GPU 冻结及内核挂起。
- 内核日志报告空指针解引用错误：`BUG: kernel NULL pointer dereference, address: 0000000000000560`。
- 触发 Oops 的 RIP 位于 `dcn20_program_front_end_for_ctx+0x384/0xcf0 [amdgpu]`。
- 调用栈显示崩溃路径涉及 `dc_commit_updates_for_stream` 和 `amdgpu_dm_atomic_commit_tail`。
- 另外在 `dcn20_hwseq.c:1484` 处触发了内核 WARNING，进程上下文为 Xorg。

#问题根因
在 `amdgpu` 驱动的 Display Core Next (DCN) 2.0 硬件序列代码中，函数 `dcn20_program_front_end_for_ctx` 尝试访问偏移地址 `0x560` 时，基址指针为 NULL，导致页面错误异常。

#解决方案
暂无
