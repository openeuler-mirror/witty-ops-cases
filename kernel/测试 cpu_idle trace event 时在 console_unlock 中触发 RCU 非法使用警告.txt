#标题
测试 cpu_idle trace event 时在 console_unlock 中触发 RCU 非法使用警告

#内核版本
3.3.0-rc2+

#问题现象
- 系统启动过程中测试 trace event 'cpu_idle' 时触发内核 WARNING。
- 日志中出现 "suspicious RCU usage" 和 "RCU used illegally from extended quiescent state" 错误信息。
- 调用栈显示警告路径为 `console_unlock` -> `vprintk` -> `__atomic_notifier_call_chain` -> `cpuidle_idle_call`。

#问题根因
系统处于 extended quiescent state（扩展静默状态，如 idle loop）时，`cpuidle_idle_call` 流程中触发了 `console_unlock`，进而调用 `atomic_notifier_call_chain`。该操作尝试在 RCU 保护的 notifier chain 上执行 `rcu_dereference_check`，导致在非法上下文中使用 RCU。

#解决方案
暂无
