#标题
radeon驱动GPU重置期间访问无效地址触发内核缺页错误

#内核版本
5.4.80-gentoo-r1-x86_64

#问题现象
- 系统日志显示 radeon 显卡 ring 0 停滞超过 23 秒，触发 GPU lockup
- 内核报错 "BUG: unable to handle page fault for address: ffffac09404bcffc"
- 触发 Oops (0000 [#1])，RIP 指向 `radeon_ring_backup+0xc0/0x140 [radeon]`
- 故障发生在通过 sysfs 热插拔移除另一张 PCIe 设备时

#问题根因
热插拔移除非显卡的 PCIe 设备导致 radeon 显卡发生 GPU lockup；在执行 GPU 重置流程（`radeon_gpu_reset`）调用 `radeon_ring_backup` 函数备份 ring 数据时，访问了无效的内存地址（CR2: ffffac09404bcffc），触发 supervisor read access page fault。

#解决方案
暂无
