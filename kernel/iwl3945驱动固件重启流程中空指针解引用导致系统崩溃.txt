#标题
iwl3945驱动固件重启流程中空指针解引用导致系统崩溃

#内核版本
3.2.5-, 3.1.0

#问题现象
- 使用 iwl3945 无线网卡进行高负载操作（如 Azureus 下载）时，系统日志频繁报错 "Microcode SW error detected. Restarting 0x82000008"
- 固件报错 SYSASSERT (0x5)，Error Reply type 0x00000005 cmd C_TX
- 触发硬件重启请求："ieee80211 phy0: Hardware restart was requested"
- 严重情况下导致系统死机，无法正常恢复

#问题根因
驱动程序在处理微码错误触发的固件重启流程时，延迟工作队列（delayed works）中访问 `il->txq` 指针前未进行有效性检查，导致 NULL pointer dereference，进而引发系统崩溃。

#解决方案
该崩溃问题已通过 commit 210787e82a0ac1ffb5d7be1c796f0c51050849ad ("iwl3945: fix possible il->txq NULL pointer dereference in delayed works") 修复，并合入 Linux v3.3 版本。对于微码错误本身，开发者建议尝试 `disable_hw_scan=0` 模块参数或使用提供的 tracing 补丁进一步定位。
