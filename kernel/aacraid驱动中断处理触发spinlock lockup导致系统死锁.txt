#标题
aacraid驱动中断处理触发spinlock lockup导致系统死锁

#内核版本
3.4, 3.8, 3.9

#问题现象
- 系统在运行 mongodb 和 sqlite 等高负载任务时，出现 "BUG: spinlock lockup suspected" 错误。
- 日志显示 CPU 在尝试获取 `async_umap_flush_lock` 时发生锁死。
- 部分案例中出现 "rcu_sched detected stalls" 和 "aacraid: Host adapter abort request" 告警。
- 系统最终完全死锁，无响应 alt+sysrq 操作。

#问题根因
aacraid 驱动程序的中断处理路径存在问题。在中断处理函数 `aac_rx_intr_message` 和 `aac_intr_normal` 执行过程中，调用 `io_callback` -> `scsi_dma_unmap` -> `intel_unmap_sg` -> `add_unmap` 时，尝试获取 `async_umap_flush_lock` 自旋锁失败，导致 spinlock lockup。

#解决方案
暂无
