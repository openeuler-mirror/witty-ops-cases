#标题
卸载AOE模块时blk_cleanup_queue函数触发空指针解引用

#内核版本
3.4.36

#问题现象
- 执行 `rmmod aoe` 卸载模块时触发 Kernel Oops
- 日志报错："BUG: unable to handle kernel NULL pointer dereference at 00000000000003e0"
- 故障现场堆栈显示：`blk_cleanup_queue` <- `aoedev_freedev` <- `aoedev_exit` <- `aoe_exit`

#问题根因
在 AOE 设备清理流程中，`aoedev_freedev` 函数向 `blk_cleanup_queue` 传递了 NULL 指针（寄存器 RDI 值为 0）。`blk_cleanup_queue` 尝试执行 `mov 0x3e0(%rdi),%r12` 指令访问该指针偏移处的成员时触发空指针解引用。

#解决方案
暂无
