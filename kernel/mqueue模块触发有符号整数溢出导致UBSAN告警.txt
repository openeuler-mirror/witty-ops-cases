#标题
mqueue模块触发有符号整数溢出导致UBSAN告警

#内核版本
v4.18-rc2

#问题现象
系统运行时触发 UBSAN（Undefined Behavior Sanitizer）告警，内核日志显示如下信息：
- UBSAN: Undefined behaviour in ipc/mqueue.c:414:49
- signed integer overflow: 65537 * 2251799813685248 cannot be represented in type 'long int'
- Call Trace: mqueue_evict_inode+0x9cf/0xce0 <- evict <- iput <- mqueue_get_inode <- mqueue_create_attr <- do_mq_open

#问题根因
在 ipc/mqueue.c 文件第 414 行的 mqueue_evict_inode 函数中，执行乘法运算时发生了有符号整数溢出。计算结果 65537 * 2251799813685248 超出了 long int 类型的最大表示范围。

#解决方案
暂无
