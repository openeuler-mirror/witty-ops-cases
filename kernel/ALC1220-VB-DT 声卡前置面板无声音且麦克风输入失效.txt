#标题
ALC1220-VB-DT 声卡前置面板无声音且麦克风输入失效

#内核版本
5.6.13

#问题现象
- Gigabyte Designare TRX40 主板集成声卡（Realtek ALC1220-VB-DT）在 Linux 下前置面板耳机孔无声音输出
- 麦克风输入完全无法工作
- 后置面板音频输出正常
- 插孔检测功能正常，PulseAudio 能正确识别设备插拔动作
- 非硬件故障，同一硬件在 Windows 系统下工作正常

#问题根因
该 USB 音频设备为不同的 I/O 接口提供了多个独立的 PCM 流，默认的音频路由配置未正确映射到前置面板对应的 PCM 设备。前置面板输出需要访问特定的设备节点（如 card #3, device 1），而非默认的设备节点。

#解决方案
开发者建议通过以下方式调试或规避：
1. 安装 alsa-ucm-conf 仓库中的 UCM profile（需配合 git 版本的 alsa-lib 及 PulseAudio）。
2. 绕过 PulseAudio 直接测试特定 PCM 设备，例如使用命令 `pasuspender -- aplay -Dplughw:3,1 -vv foo.wav` 测试前置耳机输出。
3. 使用 `alsamixer -c3` 检查并解除相关通道的静音状态。
