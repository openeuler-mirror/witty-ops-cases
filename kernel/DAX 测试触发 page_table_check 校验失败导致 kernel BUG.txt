#标题
DAX 测试触发 page_table_check 校验失败导致 kernel BUG

#内核版本
5.19.0-0.rc0

#问题现象
- 运行 fstests 测试集 generic/623 用例时触发 kernel BUG。
- 系统日志显示 "kernel BUG at mm/page_table_check.c:51!"，伴随 "invalid opcode: 0000 [#1]" 错误。
- 调用栈显示崩溃点位于 `page_table_check_set.part.0`，触发路径为 `dax_fault_iter` -> `__vm_insert_mixed` -> `insert_pfn`。
- 该问题在 XFS 和 Ext4 文件系统开启 DAX 选项时均可复现。

#问题根因
在处理 DAX 页面故障并调用 `insert_pfn` 插入页表项时，触发了 `mm/page_table_check.c` 中的 BUG 校验。由于该问题在 XFS 和 Ext4 上均能复现，定位为内存管理子系统（Memory Management）在处理 DAX 映射时的通用缺陷，而非特定文件系统问题。

#解决方案
暂无
