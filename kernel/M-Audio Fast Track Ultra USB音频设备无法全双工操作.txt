#标题
M-Audio Fast Track Ultra USB音频设备无法全双工操作

#内核版本
4.1.5-1-ARCH

#问题现象
- M-Audio Fast Track Ultra USB音频设备无法同时进行录音和播放（全双工模式）
- 单独播放或录音功能正常，但当其中一个流处于活动状态时，启动另一个流会失败
- dmesg 显示错误信息："usb 4-1.1: Unable to change format on ep #81: already in use"
- 应用层（如 arecord、speaker-test）报错："Unable to set hw params for playback: Device or resource busy" 或 "Unable to install hw params"

#问题根因
这是所有使用 implicit feedback（隐式反馈）进行同步的 USB 音频设备的通病。该设备在 sound/usb/pcm.c 的 set_sync_ep_implicit_fb_quirk() 函数中被匹配处理，导致在全双工模式下 endpoint 格式变更失败。

#解决方案
暂无正式修复。临时规避方法是从 sound/usb/pcm.c 文件的 set_sync_ep_implicit_fb_quirk() 函数中移除该设备，但这会导致播放过程中出现爆音和咔嗒声。
