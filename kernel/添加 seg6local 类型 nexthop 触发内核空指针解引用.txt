#标题
添加 seg6local 类型 nexthop 触发内核空指针解引用

#内核版本
5.14.0-rc3+

#问题现象
- 执行命令 `ip -6 nexthop add id 1 encap seg6local action End.DT4 vrftable 1 dev eth0` 时触发 Kernel Oops。
- 系统日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000ba8"。
- RIP 寄存器指向函数 `vrf_ifindex_lookup_by_table_id`，系统发生崩溃。

#问题根因
在添加封装类型为 seg6local 的 nexthop 时，内核调用链经过 `seg6_end_dt4_build` -> `__seg6_end_dt_vrf_build` -> `l3mdev_ifindex_lookup_by_table_id`，最终在 VRF 模块的 `vrf_ifindex_lookup_by_table_id` 函数中访问了非法内存地址（空指针偏移），导致 NULL pointer dereference。

#解决方案
暂无
