#标题
CONFIG_DEBUG_TEST_DRIVER_REMOVE 导致不可移除驱动重复探测触发 Kernel BUG

#内核版本
v4.8-2283-ga3443cd (4.9.0-0.rc0.git2.1.fc26.aarch64)

#问题现象
- 启用 CONFIG_DEBUG_TEST_DRIVER_REMOVE 后系统启动过程中触发 Kernel BUG，导致系统崩溃。
- 日志显示 "kernel BUG at lib/ioremap.c:64!" 及 "Internal error: Oops - BUG: 0 [#1] SMP"。
- 调用栈显示崩溃发生在 ioremap_page_range+0x168/0x2f8。
- 控制台日志显示 pci-host-generic 驱动执行了两次探测。

#问题根因
commit bea5b158ff0da9c7246ff391f754f5f38e34577a 引入的 CONFIG_DEBUG_TEST_DRIVER_REMOVE 选项旨在通过 "probe -> remove -> probe" 序列测试驱动移除路径。当驱动未实现 remove() 回调（即设计为不可移除）时，测试逻辑跳过了 remove() 回调的调用，但仍然错误地执行了资源释放操作（如 devres_release_all、driver_sysfs_remove、dev->driver = NULL 等）。这导致设备在第一次探测后资源被错误释放，随后的重新探测尝试重新映射已存在的资源时触发 BUG。

#解决方案
暂无（问题已通过邮件上报至 LKML 社区讨论）。
