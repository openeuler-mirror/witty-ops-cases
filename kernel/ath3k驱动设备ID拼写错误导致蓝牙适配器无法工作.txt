#标题
ath3k驱动设备ID拼写错误导致蓝牙适配器无法工作

#内核版本
3.15

#问题现象
升级至 3.15 内核后，Trust Bluetooth 蓝牙适配器无法正常工作。系统日志报错 "Can't get version to change to load ram patch err"。设备被错误地绑定到 ath3k 驱动模块，而在先前版本（如 3.14）中该设备使用 btusb 驱动可正常工作。

#问题根因
引入的 commit ca58e594da2486c1d28e7ad547d82266604ec4ce 中存在拼写错误，将 AR3012 蓝牙适配器的 Product ID 由 e005 错写为 3005（即代码中 USB_DEVICE(0x0cf3, 0x3005) 应为 0x0cf3, 0xe005）。这导致内核错误地将 VID:PID 为 0cf3:3005 的设备匹配到 ath3k 驱动，而非正确的 btusb 驱动，从而引发固件加载失败。

#解决方案
上游已通过补丁 "Revert "Bluetooth: Add a new PID/VID 0cf3/e005 for AR3012."" 修复了此问题，该修复已合入主线内核并回溯至 3.15.x stable 版本。临时规避措施为将 ath3k 模块加入黑名单。
