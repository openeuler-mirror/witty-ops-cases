#标题
mlx5_core 驱动在 AMD-Vi 开启时触发 IO_PAGE_FAULT 及硬件校验失败

#内核版本
4.19.157（日志显示 4.19.85）

#问题现象
- 系统日志大量输出 "AMD-Vi: Event logged [IO_PAGE_FAULT...]" 错误信息，设备指向 Mellanox Connect-X4 网卡 (mlx5_core)。
- 出现 "ethstb: hw csum failure" 硬件校验和错误。
- Call Trace 显示错误发生在网络接收软中断处理流程中，涉及 `mlx5e_napi_poll`、`tcp4_gro_receive` 及 `nf_conntrack` 模块。

#问题根因
未明确。根据用户反馈，问题与 AMD IOMMU (AMD-Vi) 功能开启有关，疑似 mlx5_core 驱动与 AMD IOMMU 硬件虚拟化交互异常导致 DMA 页错误。

#解决方案
暂无官方补丁修复。可通过在 BIOS 中禁用 IOMMU 功能规避该问题。
