#标题
Amazon ENA 驱动在 Xen dom0 模式下引发死锁及 RCU Stall

#内核版本
未明确

#问题现象
- 内核日志频繁报告 "NETDEV WATCHDOG: CPU: 1: transmit queue 0 timed out"，表明网络传输队列 0 发送超时。
- 触发 RCU Stall，日志显示 "rcu_preempt detected stalls on CPUs/tasks"，CPU 0 处于停滞状态。
- 系统尝试从 CPU 1 向 CPU 0 发送 NMI 中断以响应停滞。
- 故障发生在 EC2 系统作为嵌套 Xen 管理程序运行时，使用 Amazon VPC 功能（如 AWS CNI）会加剧该问题。

#问题根因
根据报告者推测，ENA 驱动内部可能存在互斥锁争用进而引发死锁，但确切的技术根因尚未确认。

#解决方案
暂无。报告者尝试过开启或关闭 PV spinlocks、强制驱动轮询模式、禁用内核抢占支持等方法，均未解决问题。
