#标题
启用CONFIG_HARDENED_USERCOPY_PAGESPAN导致启动时触发usercopy_abort

#内核版本
5.1

#问题现象
- 系统在启动早期触发 kernel BUG，导致崩溃。
- 日志显示 "usercopy: Kernel memory overwrite attempt detected to spans multiple pages"。
- Call Trace: usercopy_abort -> __check_object_size.cold -> build_test_sglist -> cryptomgr_test。
- 触发条件：开启了 CONFIG_CRYPTO_FIPS 和 CONFIG_HARDENED_USERCOPY_PAGESPAN。

#问题根因
在启用 CONFIG_HARDENED_USERCOPY_PAGESPAN 选项后，内核加强了 usercopy 的安全检查。加密管理测试模块（cryptomgr_test）在执行 build_test_sglist 函数时，其内存操作触发了跨越多页面的安全检查机制，导致 mm/usercopy.c 中的检查失败并调用 usercopy_abort。

#解决方案
临时规避方法是在内核配置中设置 CONFIG_CRYPTO_MANAGER_DISABLE_TESTS=y 以禁用加密算法自检测试。暂无上游补丁修复信息。
