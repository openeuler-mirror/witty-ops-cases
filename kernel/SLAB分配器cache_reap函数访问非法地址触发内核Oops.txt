#标题
SLAB分配器cache_reap函数访问非法地址触发内核Oops

#内核版本
5.10.93（报告中亦提及 5.10.90）

#问题现象
- 系统日志报错："BUG: unable to handle page fault for address: ffffebde00000008"
- 触发 Oops：#PF: supervisor read access in kernel mode, error_code(0x0000) - not-present page
- Call Trace：`free_block.constprop.0` -> `drain_array_locked.constprop.0` -> `drain_array.constprop.0` -> `cache_reap`
- 故障发生后，CPU 逐一出现 soft lockup，系统完全挂起，无法响应 ping 或控制台登录，需硬重启。
- 触发场景：在执行 VM 关闭、LVM 卷重命名或 ext4 文件系统卸载等 I/O 密集型操作后数分钟内发生。

#问题根因
内核在处理 SLAB 分配器的 `cache_reap` 工作队列时，`free_block.constprop.0` 函数内部访问了无效的内存地址（CR2: ffffebde00000008），导致缺页异常。具体的内存破坏或指针错误原因未明确。

#解决方案
用户反馈升级至内核 5.15 并将 SLAB 分配器切换为 SLUB 后，问题不再复现。暂无官方修复补丁。
