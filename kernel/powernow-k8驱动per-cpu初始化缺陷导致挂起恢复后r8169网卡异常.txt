#标题
powernow-k8驱动per-cpu初始化缺陷导致挂起恢复后r8169网卡异常

#内核版本
3.12 及更高版本

#问题现象
- 系统从挂起状态恢复后，r8169 网卡工作异常。
- 严重情况下，网卡设备会从 PCI 总线上消失，即使重启也无法恢复。
- 内核日志中出现 "cpufreq: Warning: CPU frequency out of sync... is 0 kHz" 警告信息。

#问题根因
内核 commit 23d328994b 调整了 CPU hotplug 回调顺序，导致 `cpufreq_update_policy()` 在 CPU 上线时被调用。此时 `powernow-k8` 驱动的 per-cpu 数据尚未正确初始化，`cpufreq_driver->get()` 返回 0。系统错误地将当前频率设置为 0 kHz，进而导致系统行为异常，影响网卡正常工作。

#解决方案
应用修复 powernow-k8 驱动 per-cpu 初始化问题的补丁（Bugzilla 附件 ID 126391）。该补丁已标记为稳定分支合并。
