#标题
uinput设备移除时在led_classdev_unregister触发空指针解引用导致内核崩溃

#内核版本
4.13

#问题现象
- 快速创建并销毁 uinput 设备会导致内核崩溃。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 000000000000009e"。
- 触发 Oops 的 Call Trace 为：device_del -> device_unregister -> led_classdev_unregister -> input_leds_disconnect -> uinput_destroy_device。

#问题根因
这是一个竞态条件问题。在移除 uinput 设备的过程中，`input_leds_disconnect` 调用 `led_classdev_unregister` 时，相关数据结构可能已被破坏或未正确初始化，导致在 `device_del` 中访问了无效地址（NULL 指针偏移 0x9e），从而触发内核 Oops。

#解决方案
暂无
