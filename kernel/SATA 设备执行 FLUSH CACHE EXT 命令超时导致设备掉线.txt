#标题
SATA 设备执行 FLUSH CACHE EXT 命令超时导致设备掉线

#内核版本
3.6.9（报告中涉及版本包括 3.0.0-16, 3.2.12）

#问题现象
- 系统运行一段时间后变得无响应，平均每隔几天出现一次。
- 内核日志报错：`ata2.00: exception Emask 0x0 SAct 0x0 SErr 0x0 action 0x6 frozen`
- 随后出现命令失败：`failed command: FLUSH CACHE EXT`，状态显示超时 `Emask 0x4 (timeout)`。
- 尝试硬重置链路失败：`ata2: COMRESET failed (errno=-16)`。
- 最终设备被禁用：`ata2.00: disabled`，系统报告 `Unhandled error code` 和 `I/O error`，文件系统（EXT4）报错。

#问题根因
硬盘在执行 FLUSH CACHE EXT 命令时停止响应，导致 libata 错误处理机制（EH）触发链路重置，但 COMRESET 失败，设备无法恢复。开发者分析认为可能是硬盘本身故障或供电不足导致设备掉线，但用户反馈 Windows 下未出现此问题，具体是否为内核驱动兼容性问题未明确。

#解决方案
暂无内核修复补丁。建议检查硬盘 SMART 日志确认硬件健康状况，或检查电源供电稳定性。
