#标题
GeForce 6150SE 显卡启动 X Server 时因 nouveau 驱动重构导致系统挂起

#内核版本
3.9.9（回归版本为 3.7-rc，3.6.6 正常）

#问题现象
- 系统启动 X Server 时出现花屏并挂起，无响应。
- 内核日志显示 nouveau 驱动加载失败：`nouveau: probe of 0000:00:0d.0 failed with error -12`。
- 日志中出现 `Failed to idle channel 0` 及 `Memory manager not clean` 错误。
- 触发调用栈警告：`WARNING: at drivers/gpu/drm/nouveau/nouveau_gpuobj.c:241 nouveau_gpuobj_takedown`。

#问题根因
经二分定位，问题引入于 commit `ebb945a94bba2ce8dff7b0942ff2b3f2a52a0a69` ("drm/nouveau: port all engines to new engine module format")。该提交对 nouveau 驱动引擎模块进行了大规模重构，导致 NV40 系列（如 GeForce 6150SE / NV4C）显卡在初始化阶段无法正确 idle 通道或清理内存管理器，进而引发 probe 失败。

#解决方案
暂无官方修复。临时规避措施为降级内核至 3.6.x 版本。
