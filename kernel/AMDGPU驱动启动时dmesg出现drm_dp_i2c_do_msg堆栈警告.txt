#标题
AMDGPU驱动启动时dmesg出现drm_dp_i2c_do_msg堆栈警告

#内核版本
4.17.10, 4.18-rc6

#问题现象
- 系统启动过程中，dmesg 日志输出 stacktrace，起始函数为 `drm_dp_i2c_do_msg`。
- 该问题在内核版本 4.17.9 中不存在，升级至 4.17.10 后出现，并在 4.18-rc6 中复现。
- 使用硬件为 AMD RX 580 显卡。

#问题根因
问题由 commit `9168c089a2122606e2c01f6beda85ff950ac189` 引入。该提交回退了之前的 DDC 修复代码（"drm/amd/display: Don't return ddc result and read_bytes in same return value"）以解决 DDC 故障，但在某些场景下触发了内核堆栈警告。开发者确认该警告本身是无害的。

#解决方案
升级至内核 4.19 或更高版本。4.19 版本通过一系列补丁（如 `drm/amd/display: Serialize is_dp_sink_present`、`drm/amd/display: Return aux replies directly to DRM` 等）彻底解决了 DDC 问题并消除了此警告。
