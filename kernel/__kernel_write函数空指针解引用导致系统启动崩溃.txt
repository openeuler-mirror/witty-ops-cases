#标题
__kernel_write函数空指针解引用导致系统启动崩溃

#内核版本
5.9-next-20201015

#问题现象
- 系统在启动过程中发生 Kernel Oops 并崩溃。
- 日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000000"。
- RIP 寄存器指向 `__kernel_write+0x11b/0x2c0`。
- 调用栈显示崩溃路径为 `autofs_notify_daemon` -> `autofs_wait` -> `__kernel_write`。

#问题根因
Autofs 模块与 `__kernel_write` 的代码变更（提交 4d03e3cc59828 "fs: don't allow kernel reads and writes without iter ops"）交互异常。当 `autofs_notify_daemon` 调用 `__kernel_write` 时，因相关文件操作缺失 iter ops 导致触发空指针解引用。

#解决方案
该问题已在 Al Viro 的 vfs for-next 分支中修复，补丁提交 ID 为 4c207ef48269377236cd38979197c5e1631c8c16。
