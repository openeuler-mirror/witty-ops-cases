#标题
连接USB硬盘时内核重复发送ATA pass-through命令导致日志刷屏

#内核版本
4.4.14

#问题现象
- 连接USB硬盘后，内核日志每隔10分钟重复打印错误信息
- 日志显示 `sd ... tag#0 FAILED Result: hostbyte=DID_ERROR driverbyte=DRIVER_SENSE`，Sense Key 为 Hardware Error
- CDB 内容显示为 `ATA command pass through(16)` 或 `ATA command pass through(12)`
- 错误日志容易误导用户认为硬盘存在硬件故障

#问题根因
引入问题的 commit 为 `0dec8c0d67c64401d97122e4eba347ccc5850622` (scsi_lib: correctly retry failed zero length REQ_TYPE_FS commands)。该补丁修复了零长度 REQ_TYPE_FS 命令在遇到 UNIT ATTENTION 等可重试错误时未被重试的问题。此变更导致 udisks2 发起的 SMART 检测命令（通过 SG_IO ioctl 发送 ATA pass-through）在不支持该功能的 USB-to-SATA 桥接器上失败时，错误处理逻辑发生变化，使得失败日志被持续打印，而之前这些命令可能被静默处理。

#解决方案
暂无（Bug状态为NEW，尚未合并修复补丁）
