#标题
SLUB分配器在kmem_cache_free过程中触发kernel BUG

#内核版本
4.14.0-rc5

#问题现象
- 系统启动过程中触发 kernel BUG，导致 Oops。
- 日志显示 "kernel BUG at mm/slub.c:294!" 及 "invalid opcode: 0000 [#1]"。
- EIP 指向 `__slab_free+0x1f0/0x310` 或 `kmem_cache_free+0x1d0/0x1e0`。
- 调用栈显示故障路径涉及 `sd_ioctl` -> `scsi_cmd_ioctl` -> `sg_io` -> `blk_rq_unmap_user` -> `bio_free` -> `kmem_cache_free`。

#问题根因
SLUB 分配器在 `mm/slub.c:294` 处执行一致性检查失败，触发 `BUG()`。故障发生在 SCSI 命令处理完成后的 bio 结构释放路径中（`blk_rq_unmap_user` -> `bio_free` -> `kmem_cache_free`），表明内存对象可能已损坏或存在非法释放操作。具体根因未明确。

#解决方案
暂无。报告中提到尝试添加内核启动参数 `slub_debug=FZPU` 进行调试，但未能复现故障。
