#标题
DSA驱动卸载时通知未清理导致释放后使用引发内核崩溃

#内核版本
6.12.60

#问题现象
- 在循环执行 `rmmod rtl8365mb` 和 `insmod` 操作几次后，系统触发 Kernel panic。
- 日志显示 "Unable to handle kernel paging request at virtual address 702e7660"。
- 触发 Oops，寄存器 ra 指向 `work_notifysig+0x10/0x18`，表明崩溃发生在信号通知处理路径中。

#问题根因
根据日志中 "DSA: tree 0 torn down" 后紧接着发生崩溃的现象，以及返回地址指向 `work_notifysig`，推测 DSA 子系统在驱动移除并销毁资源后，未能正确注销或排空待处理的通知队列。这导致内核在后续处理通知时访问了已释放的内存区域，触发释放后使用错误。

#解决方案
暂无
