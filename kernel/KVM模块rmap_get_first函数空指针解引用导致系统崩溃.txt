#标题
KVM模块rmap_get_first函数空指针解引用导致系统崩溃

#内核版本
4.6.3

#问题现象
- 系统偶发崩溃，内核日志显示 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000010"。
- 触发 Oops 的指令指针位于 `rmap_get_first+0x21/0x60 [kvm]`。
- 故障发生在 `kswapd0` 进程上下文中，调用栈显示路径为：`kswapd` -> `shrink_active_list` -> `page_referenced` -> `kvm_mmu_notifier_clear_flush_young` -> `kvm_zap_rmapp` -> `rmap_get_first`。

#问题根因
在内存回收过程中，kswapd 进程调用 KVM 的 MMU notifier 回调函数处理反向映射时，`rmap_get_first` 函数试图访问非法地址（空指针偏移 0x10），导致内核空指针解引用。具体的代码逻辑触发条件在报告中尚未明确分析。

#解决方案
暂无正式修复补丁。用户尝试降级内核至 4.4.15 版本以规避该问题。
