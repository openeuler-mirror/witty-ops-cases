#标题
ARM平台内核启动时crypto_ahash_init访问无效地址触发Oops

#内核版本
6.15.0-rc1

#问题现象
- 系统启动过程中触发 Kernel Oops，日志显示 "Unable to handle kernel paging request at virtual address fefeff3e when write"
- 故障发生在 ARM 平台（Rockchip RK3288），进程为 cryptomgr_test
- Internal error: Oops: 805 [#1] SMP ARM
- Call trace: crypto_ahash_init -> test_ahash_vec_cfg -> __alg_test_hash -> alg_test_hash -> alg_test -> cryptomgr_test

#问题根因
在执行 `crypto_ahash_init` 函数时，寄存器 `r4` 包含无效指针值 `fefefefe`，导致计算出的写入地址 `fefeff3e`（r4 + 0x40）触发分页错误。具体导致该指针无效的深层原因未明确。

#解决方案
暂无
