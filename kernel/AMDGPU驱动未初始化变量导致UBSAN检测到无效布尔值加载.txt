#标题
AMDGPU驱动未初始化变量导致UBSAN检测到无效布尔值加载

#内核版本
5.15.12

#问题现象
系统启动或图形服务初始化时，dmesg 日志输出 UBSAN 告警信息：
- UBSAN: invalid-load in drivers/gpu/drm/amd/amdgpu/../display/amdgpu_dm/amdgpu_dm.c:5882:84
- load of value 32 is not a valid value for type '_Bool'（部分情况值为 4）
- Call Trace 指向 create_stream_for_sink -> create_validate_stream_for_sink

#问题根因
AMDGPU 驱动模块 `amdgpu_dm.c` 中存在未初始化的 `_Bool` 类型变量，代码在未对该变量赋值的情况下进行了读取操作，导致加载了非法值（非 0 或 1），从而触发 UBSAN（Undefined Behavior Sanitizer）的 invalid-load 检查告警。

#解决方案
应用补丁 https://patchwork.freedesktop.org/patch/468484 修复未初始化变量问题。
