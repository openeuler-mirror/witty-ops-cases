#标题
ARM平台TCMU模块处理命令环填充时访问非法地址导致内核崩溃

#内核版本
4.19.118, 5.4.44, 5.6.16

#问题现象
- ARM架构服务器作为iSCSI Target运行时，Initiator登录过程中触发内核Oops，日志显示 "Unable to handle kernel paging request"，调用栈为 `flush_dcache_page` <- `is_ring_space_avail [target_core_user]`。
- 修复登录问题后，在执行文件系统格式化（ext4）及数据写入时再次触发内核Oops，日志显示 "Unable to handle kernel access to user memory outside uaccess routines"，调用栈为 `flush_dcache_page` <- `tcmu_handle_completions [target_core_user]`。

#问题根因
根因位于内核模块 `target_core_user` (TCMU) 中：
1. 在 `is_ring_space_avail` 函数中，检查命令环可用空间时逻辑错误，导致向 `flush_dcache_page` 传递了无效的虚拟地址。
2. 在 `tcmu_handle_completions` 函数中，当命令环末尾存在小于 `sizeof(struct tcmu_cmd_entry)` 的填充时，处理逻辑未正确处理该边界情况，导致访问非法内存地址。

#解决方案
应用开发者提供的系列补丁（Patch 1-3 修复登录崩溃，Patch 4 修复IO完成崩溃）。用户测试确认在 4.19.118 版本应用补丁后，登录及IO读写均恢复正常。补丁已提交至 linux-scsi 和 target-devel 社区。
