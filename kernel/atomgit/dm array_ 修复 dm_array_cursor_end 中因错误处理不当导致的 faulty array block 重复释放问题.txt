#标题
dm array: 修复 dm_array_cursor_end 中因错误处理不当导致的 faulty array block 重复释放问题

#内核版本
OLK-6.6

#问题现象
当 dm_bm_read_lock() 因校验和或锁定错误失败时，dm_array_cursor 错误地缓存了无效的 dm_block 指针。随后在调用 dm_array_cursor_end() 时，会对该无效指针进行重复释放，触发 dm-bufio 中的 BUG_ON，导致内核 panic。通过构造损坏的 cache metadata 设备并尝试重新打开即可复现。

#主要错误日志
device-mapper: array: array_block_check failed: blocknr 0 != wanted 10
device-mapper: block manager: array validator check failed for block 10
device-mapper: array: get_ablock failed
device-mapper: cache metadata: dm_array_cursor_next for mapping failed
------------[ cut here ]------------
kernel BUG at drivers/md/dm-bufio.c:638!

#问题根因
当 dm_bm_read_lock() 遇到校验和或锁定错误时，它会在内部隐式释放故障块，但留给调用者的是一个无效的输出指针。dm_array_cursor 的 load_ablock 函数在 get_ablock 失败时，没有将缓存的块指针 c->block 置为 NULL，导致后续 dm_array_cursor_end() 误以为该指针有效并再次尝试释放，引发 double free 并触发 dm-bufio 缓存逻辑中的 BUG_ON。

#解决方案
修改 drivers/md/persistent-data/dm-array.c 中的 load_ablock 函数。在 get_ablock 失败或 dm_btree_cursor_get_value 失败的错误处理路径中，显式将 c->block 和 c->ab 指针置为 NULL，防止后续 cursor end 操作对无效指针进行释放。
