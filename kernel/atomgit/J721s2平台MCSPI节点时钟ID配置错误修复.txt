#标题
J721s2平台MCSPI节点时钟ID配置错误修复

#内核版本
OLK-6.6

#问题现象
TI J721s2平台上的多个MCSPI实例（包括main_spi0~7及mcu_spi0~2）因时钟ID配置与硬件文档不一致，可能导致SPI控制器无法正常工作、时钟源选择错误或通信异常。

#主要错误日志
未知

#问题根因
设备树文件 `k3-j721s2-main.dtsi` 和 `k3-j721s2-mcu-wakeup.dtsi` 中，MCSPI节点的时钟ID（clock IDs）设置与硬件技术参考手册不一致。原代码中 `main_spi*` 实例使用了错误的时钟ID 1，`mcu_spi*` 实例使用了错误的时钟ID 0，导致驱动无法正确匹配硬件时钟资源。

#解决方案
根据TI官方文档（https://software-dl.ti.com/tisci/esd/latest/5_soc_doc/j721s2/clocks.html），修正设备树中所有受影响的MCSPI节点的时钟ID：
1. 将 `main_spi0` 至 `main_spi7` 的时钟引用从 `<&k3_clks ID 1>` 修改为 `<&k3_clks ID 2>`。
2. 将 `mcu_spi0` 至 `mcu_spi2` 的时钟引用从 `<&k3_clks ID 0>` 修改为 `<&k3_clks ID 2>`。
