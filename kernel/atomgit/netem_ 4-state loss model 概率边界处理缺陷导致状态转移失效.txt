#标题
netem: 4-state loss model 概率边界处理缺陷导致状态转移失效

#内核版本
OLK-6.6

#问题现象
在使用 netem qdisc 的 4-state Markov loss model 进行丢包模拟时，存在概率边界漏洞。当随机数 `rnd` 恰好等于状态转移概率的边界值（如 `a4` 或 `a1 + a4`）时，代码中的条件判断分支均无法匹配。这导致状态机在该次处理中不发生任何状态转移且未正确返回丢包判断结果，造成丢包行为不符合预期，无法匹配 GI (General and Intuitive) loss model 规范。

#主要错误日志
未知

#问题根因
`loss_4state()` 函数中的条件判断逻辑存在缺陷。代码使用了诸如 `if (a4 < rnd && rnd < a1 + a4)` 的判断语句，这导致当 `rnd` 等于边界值时，既不满足前面的 `if` 分支，也不满足当前的 `else if` 分支，形成了逻辑漏洞。同时，`else if` 中对下边界的冗余检查（如 `a4 < rnd`）在逻辑上已被前面的分支排除，属于冗余代码且导致了边界无法覆盖。

#解决方案
移除条件判断中冗余的下边界检查（如 `clg->a4 < rnd`），并将最后的 `else if` 改为 `else`。这使得概率范围连续且无缺口，确保任意随机值都能触发确定的状态转移，修复了边界值不转移的问题。
