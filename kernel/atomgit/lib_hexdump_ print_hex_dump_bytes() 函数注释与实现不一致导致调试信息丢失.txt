#标题
lib/hexdump: print_hex_dump_bytes() 函数注释与实现不一致导致调试信息丢失

#内核版本
OLK-6.6

#问题现象
调用 `print_hex_dump_bytes()` 函数时，若未启用（动态）DEBUG 配置选项，系统没有任何打印输出。开发者根据函数注释认为其是 `print_hex_dump()` 的封装，预期在默认日志级别下应能正常输出数据流，但实际上该函数仅在开启 DEBUG 时生效，导致调试困难或关键信息缺失。

#主要错误日志
未知

#问题根因
`print_hex_dump_bytes()` 函数的文档注释声称它是 `print_hex_dump()` 的简单封装，但代码实现上它实际上调用的是 `print_hex_dump_debug()`。由于 `print_hex_dump_debug()` 依赖于 DEBUG 配置，当内核配置中禁用 DEBUG 时，该函数不会产生任何输出。文档与实现的不一致误导了开发者对该函数行为的预期。

#解决方案
修改 `include/linux/printk.h` 中 `print_hex_dump_bytes()` 的函数注释文档，将其描述从“`print_hex_dump()` 的简写形式”更正为“`print_hex_dump_debug()` 的简写形式”，并明确指出其调用的是 `print_hex_dump_debug()`，以确保文档与实际代码实现保持一致，消除误解。
