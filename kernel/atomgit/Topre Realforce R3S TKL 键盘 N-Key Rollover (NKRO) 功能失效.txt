#标题
Topre Realforce R3S TKL 键盘 N-Key Rollover (NKRO) 功能失效

#内核版本
OLK-6.6

#问题现象
Topre Realforce R3S 87键（TKL）键盘连接系统后，N-Key Rollover（全键无冲）功能无法正常工作，仅支持 6-Key Rollover（6键无冲），导致在同时按下多个按键时出现按键冲突或丢失，影响高频输入场景下的用户体验。

#主要错误日志
fixing up Topre REALFORCE keyboard report descriptor
（注：此为驱动修复生效时的提示日志，故障发生时通常无明显的错误报错，主要表现为功能受限）

#问题根因
新款 Topre Realforce R3S 系列键盘的硬件 ID 未包含在 `hid-topre` 驱动表中，导致该设备未加载专用修复驱动。该系列键盘与旧款 R2 系列存在相同的报表描述符缺陷，需要针对特定字节进行修正才能开启 N-Key Rollover 功能。由于驱动未匹配，内核使用了通用协议处理，导致键盘回退到标准的 6-Key Rollover 模式。

#解决方案
1. 在 `hid-ids.h` 中新增 Topre Realforce R3S 87键键盘的设备 ID 定义（`USB_DEVICE_ID_TOPRE_REALFORCE_R3S_87`，VID: 0x0853, PID: 0x0313）。
2. 在 `hid-topre.c` 驱动的设备 ID 表中添加对该新设备的匹配支持。
3. 扩展 `topre_report_fixup` 函数逻辑，针对 R3S 键盘的报表描述符特征（`rsize >= 106` 且偏移量 28-30 处为特定值）进行检测，并将描述符偏移量 31 处的字节修正为 0x02，从而激活 NKRO 功能。
