#标题
ath12k: 修复 monitor ring 描述符中 end offset 位定义错误导致的数据解析异常

#内核版本
OLK-6.6

#问题现象
在处理 monitor destination ring 描述符时，由于 end offset 的位定义与固件规范不一致，导致从描述符中获取的 end offset 信息无效或错误，可能引发数据解析异常或功能不符合预期。

#主要错误日志
未知

#问题根因
End offset 在驱动中被定义为 16 位（bit 0 到 bit 15），而固件定义规定其仅为 12 位（bit 0 到 bit 11），bit 12 到 bit 15 为保留位。驱动使用错误的掩码读取时包含了保留位，这些保留位可能包含垃圾值，导致提取的 end offset 信息无效。

#解决方案
修改 `drivers/net/wireless/ath/ath12k/hal_desc.h` 中的宏定义，将 `HAL_MON_DEST_INFO0_END_OFFSET` 的掩码从 `GENMASK(15, 0)` 更正为 `GENMASK(11, 0)`，以确保只读取有效的 12 位数据，屏蔽掉包含垃圾值的保留位。
