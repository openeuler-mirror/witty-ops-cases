#标题
BME680 驱动测量等待时间计算错误导致频繁返回 EBUSY

#内核版本
OLK-6.6

#问题现象
在读取 BME680 传感器数据时，用户空间偶尔会收到 EBUSY 错误。内核内部检查“设备测量周期未完成”偶尔会判定失败。

#主要错误日志
未知

#问题根因
驱动代码中计算测量等待时间的公式存在错误，其中一个常数项由 Bosch 官方 API 定义的 1963 误写为 1936（看起来像是两位数字的笔误）。这导致计算出的等待时间过短，在传感器尚未完成测量时就开始检查状态，从而导致测量周期未完成的检查失败，并向用户空间返回 EBUSY 错误。

#解决方案
修正驱动代码 `bme680_core.c` 中 `bme680_wait_for_eoc` 函数内的计算常数，将 1936 修改为 1963，使其与 Bosch BME680 API 实现保持一致，从而保证等待时间足够长以完成测量。
