#标题
mmc: bcm2835 修复 dma_unmap_sg() 参数传入错误导致 DMA 取消映射异常

#内核版本
OLK-6.6

#问题现象
在 bcm2835 mmc 驱动执行 DMA 传输过程中，若 `dmaengine_prep_slave_sg` 函数调用失败（返回 NULL desc），驱动尝试释放 DMA 映射时传入的 nents 参数不正确。该问题会导致 DMA 取消映射操作无法正确执行，可能引发内存泄漏、DMA 映射表不一致或内核内存管理异常。

#主要错误日志
未知

#问题根因
在 `bcm2835_prepare_dma` 函数的错误处理路径中，调用 `dma_unmap_sg()` 时错误地使用了局部变量 `sg_len`（即 `dma_map_sg()` 的返回值）。根据内核 DMA API 规范，`dma_unmap_sg()` 函数的 nents 参数必须与 `dma_map_sg()` 调用时传入的 nents 值（即原始 scatter-gather 列表长度）保持一致，而不是 map 函数执行成功后返回的映射条目数。

#解决方案
修改 `drivers/mmc/host/bcm2835.c` 文件中的 `bcm2835_prepare_dma` 函数，将 `dma_unmap_sg()` 的第二个参数由 `sg_len` 更正为 `data->sg_len`，确保取消映射时使用的是原始散列表长度，符合 DMA API 的调用契约。
