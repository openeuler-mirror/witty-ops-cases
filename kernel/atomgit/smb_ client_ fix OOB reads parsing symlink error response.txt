#标题
smb: client: fix OOB reads parsing symlink error response

#内核版本
OLK-6.6

#问题现象
在处理 SMB2 符号链接错误响应时，内核存在越界读取（OOB reads）风险。当服务器返回的数据包经过构造或畸形时，解析逻辑可能访问缓冲区之外的内存区域，导致内核内存非法访问，可能引发内核崩溃或信息泄露。

#主要错误日志
未知

#问题根因
1. **循环遍历逻辑缺陷**：在 `smb2_error_context_rsp` 循环中，原代码使用 `do...while(p < end)` 结构，且 `end` 指针计算位置不当（在循环体内定义），导致循环边界检查失效，可能在指针未正确校验的情况下进行访问。
2. **边界检查缺失**：在定位到 `smb2_symlink_err_rsp` 结构体后，未校验该结构体自身是否完整位于响应缓冲区内（`(u8 *)sym + sizeof(*sym) > end`）。
3. **字段偏移校验不当**：在 `smb2_parse_symlink_response` 中，校验 `PathBuffer` 中的偏移量和长度时，未基于结构体实际位置进行越界检查，而是简单地累加固定结构大小，导致校验逻辑无法准确拦截越界访问。

#解决方案
1. **修正循环边界检查**：将缓冲区结束指针 `end` 的计算移至函数开头，并将循环改为 `while ((u8 *)p + sizeof(*p) <= end)`，确保在每次访问指针对象前，其基本结构大小都在缓冲区范围内。
2. **增加结构体越界校验**：在返回 `sym` 指针前，增加对 `smb2_symlink_err_rsp` 结构体整体大小的越界检查，防止读取到结构体末尾时发生越界。
3. **修正字段边界检查逻辑**：在 `smb2_parse_symlink_response` 中，将偏移和长度的校验逻辑修改为基于 `sym->PathBuffer` 实际地址与缓冲区结束地址的比较，确保计算出的访问范围严格位于 `iov` 缓冲区之内。
