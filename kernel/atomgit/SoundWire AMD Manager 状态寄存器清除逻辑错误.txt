#标题
SoundWire AMD Manager 状态寄存器清除逻辑错误

#内核版本
OLK-6.6

#问题现象
当 AMD SoundWire Manager 处于异常状态（bad state）时，系统尝试清除命令状态寄存器失败。具体表现为驱动检测到无效结果状态位（AMD_SDW_IMM_RES_VALID）置位，打印错误信息 "SDW%x manager is in bad state" 后，原本意图清除该状态的写操作未能正确生效，可能导致后续命令处理异常或状态残留。

#主要错误日志
[   ] SDW%x manager is in bad state

#问题根因
根据硬件规范，要清除 ACP_SDW_IMM_CMD_STS 寄存器中的有效结果状态，应当向该寄存器写入 1。原代码在处理异常状态分支时，错误地执行了 `writel(0x00, ...)` 操作（写入全 0），这不符合硬件清除状态的要求，导致状态位无法被正确清除。

#解决方案
修改 `drivers/soundwire/amd_manager.c` 中的寄存器写入逻辑。将异常处理分支中的 `writel(0x00, amd_manager->mmio + ACP_SW_IMM_CMD_STS)` 修改为 `writel(AMD_SDW_IMM_RES_VALID, amd_manager->mmio + ACP_SW_IMM_CMD_STS)`，确保写入数值 1 以正确清除状态寄存器。
