#标题
of/irq: of_irq_parse_one() 函数引发 device node refcount 泄漏

#内核版本
OLK-6.6

#问题现象
在设备树中使用 `interrupts-extended` 属性时，调用 `of_irq_parse_one()` API 会导致对应的 device node 引用计数（refcount）发生泄漏。随着该 API 被反复调用（例如设备频繁初始化或重载），系统内核对象的引用计数持续增长，无法正常释放，可能导致内存泄漏或设备资源无法正常回收。

#主要错误日志
未知

#问题根因
在函数 `of_irq_parse_one()` 处理 `interrupts-extended` 属性的代码路径中，`of_parse_phandle_with_args()` 成功解析后会增加 `@i_th_phandle` 节点的引用计数。然而，该函数在返回前未调用 `of_node_put()` 来减少该节点的引用计数，从而导致了引用计数的泄漏。此外，旧代码的错误处理路径也存在潜在的资源释放问题。

#解决方案
重构引用计数管理逻辑，引入 `__free(device_node)` 自动清理机制。
具体修改包括：
1. 将局部变量 `p` 声明为 `struct device_node __free(device_node) *p`，利用编译器的自动清理特性确保在变量离开作用域时自动调用 `of_node_put()`，防止遗漏释放。
2. 简化代码流程，移除手动的 `goto out` 跳转和显式的 `of_node_put(p)` 调用，统一通过自动清理机制处理。
3. 明确注释说明：函数成功返回时，`@out_irq->np` 的引用计数会加 1。
