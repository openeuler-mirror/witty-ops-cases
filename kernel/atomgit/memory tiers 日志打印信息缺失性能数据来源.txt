#标题
memory tiers 日志打印信息缺失性能数据来源

#内核版本
OLK-6.6

#问题现象
内核在检测到 DRAM 节点之间的性能不匹配时，会打印性能差异的日志信息。但在日志输出中，仅显示了 DRAM 节点的 ID，未指明这些性能数据的来源，导致用户在排查日志时无法直观判断性能属性的具体出处。

#主要错误日志
memory-tiers: the performance of DRAM node %d mismatches that of the reference
DRAM node %d.
  performance of reference DRAM node %d:
  performance of DRAM node %d:
  disable default DRAM node performance based abstract distance algorithm.

#问题根因
在 `mt_set_default_dram_perf` 函数中，打印 DRAM 节点性能信息的代码逻辑不完整。输出日志时只打印了节点 ID，遗漏了记录性能数据来源的变量（`default_dram_perf_ref_source` 和 `source`），导致日志信息不够明确，缺乏关键上下文。

#解决方案
修改日志打印逻辑，在显示 DRAM 节点性能信息时，增加打印性能数据的来源字段。具体修改为将 `pr_info("  performance of reference DRAM node %d:\n", ...)` 修改为 `pr_info("  performance of reference DRAM node %d from %s:\n", ...)`，从而在日志中明确显示性能数据的出处。
