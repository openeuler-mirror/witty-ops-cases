#标题
libbpf: 修复 set_kcfg_value_str 函数中的越界读取漏洞

#内核版本
OLK-6.6

#问题现象
在 libbpf 处理字符串类型的内核配置时，若输入数据为仅包含一个起始双引号的非法字符串，程序会在校验过程中发生越界读取。

#主要错误日志
extern (kcfg) '%s': invalid string config '%s'

#问题根因
在 `set_kcfg_value_str` 函数中，代码假定字符串至少包含两个字符（起始和结束双引号），并直接使用 `value[len - 1]` 来检查结束引号。然而，当输入字符串仅包含一个起始双引号时（长度为 1），`len - 1` 指向该起始引号本身，导致逻辑检查失效，同时也构成了对非信任输入的越界访问风险。

#解决方案
在检查结束引号前，增加对字符串长度的显式校验，确保字符串长度至少为 2，防止对单字符输入进行越界访问。修改后的判断逻辑为：`if (len < 2 || value[len - 1] != '"')`。
