#标题
misc: tps6594-pfsm: Add NULL pointer check in tps6594_pfsm_probe()

#内核版本
OLK-6.6

#问题现象
在驱动探测函数 `tps6594_pfsm_probe` 中，调用 `devm_kasprintf` 生成设备名称时，若内存分配失败返回 NULL，后续代码直接使用该指针将导致内核发生空指针解引用（NULL pointer dereference）。

#主要错误日志
未知

#问题根因
函数 `devm_kasprintf()` 在内存不足等情况下可能返回 NULL。原代码未对返回值 `pfsm->miscdev.name` 进行有效性检查，直接在后续流程中使用了该指针，从而引发了潜在的空指针解引用风险。该问题通过静态分析工具发现。

#解决方案
在 `tps6594_pfsm_probe` 函数中，对 `devm_kasprintf` 的返回值进行 NULL 检查。如果返回值为 NULL，则函数直接返回 `-ENOMEM` 以终止探测过程，避免后续解引用空指针。
