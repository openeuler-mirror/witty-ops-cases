#标题
Intel PMT Discovery 驱动初始化失败时 kobject 内存泄漏

#内核版本
OLK-6.6

#问题现象
在 Intel PMT（Platform Monitoring Technology）Discovery 驱动的初始化过程中，如果 `kobject_init_and_add()` 函数调用失败，系统将发生内存泄漏。该问题违反了 kobject API 的契约，导致相关内核对象分配的资源无法被正确释放。

#主要错误日志
未知

#问题根因
在函数 `pmt_features_discovery()` 中，当 `kobject_init_and_add()` 执行失败时，代码直接返回错误码，未调用 `kobject_put()`。根据 kobject API 规范，即使在初始化失败的情况下，也必须调用 `kobject_put()` 来清理已分配的资源（如 kobject 结构体本身的内存），否则将导致内存泄漏。

#解决方案
修改 `pmt_features_discovery()` 函数的错误处理路径。在 `kobject_init_and_add()` 失败的分支中，显式增加 `kobject_put(&feature->kobj)` 调用，确保在初始化错误时能正确释放 kobject 资源，修复内存泄漏问题。
