#标题
HiSilicon PA PMU 事件名称定义歧义且 tx_req 事件 ID 错误

#内核版本
OLK-6.6

#问题现象
使用 PA PMU 组件监控性能时，`rx_req` 和 `tx_req` 事件仅统计 link0 的 flit 包请求数量，但原始命名容易让用户误解为统计所有链路（link0-3）的总和。同时，`tx_req` 事件使用了错误的事件 ID（0x5c），导致该事件无法正确统计或统计数据错误。

#主要错误日志
未知

#问题根因
1. 事件命名不规范：原始事件名 `rx_req` 和 `tx_req` 缺少作用域标识（如 link0），导致语义模糊，误导用户认为其统计范围覆盖所有链路。
2. 事件 ID 定义错误：`tx_req` 的事件 ID 被错误定义为 0x5c，正确的硬件事件标识符应为 0x60。

#解决方案
1. 修正事件名称：将 `rx_req` 重命名为 `pa_rx_req_link0`，将 `tx_req` 重命名为 `pa_tx_req_link0`，通过添加 "pa" 前缀和 "link0" 后缀明确界定事件归属和统计范围。
2. 修正事件 ID：将 `tx_req` 对应的事件 ID 从 0x5c 更正为 0x60，确保硬件事件统计准确。
