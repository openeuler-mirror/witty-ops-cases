#标题
atmel_soc_device_init 中 device_node 资源释放缺失导致内存泄漏

#内核版本
OLK-6.6

#问题现象
系统启动或模块加载过程中，`atmel_soc_device_init` 函数通过 `of_find_node_by_path` 获取设备节点后，在部分执行路径（如匹配失败提前返回）未调用 `of_node_put` 释放资源，导致内核设备节点引用计数未减少，产生内存泄漏资源。

#主要错误日志
未知

#问题根因
函数 `atmel_soc_device_init` 使用 `of_find_node_by_path("/")` 获取了 `device_node` 结构体指针，该函数会增加节点的引用计数。根据内核编程规范，通过 `of_find_node_by_path()` 获取的设备节点在不再需要时必须显式调用 `of_node_put()` 以释放引用。原代码在 `of_match_node` 匹配失败返回 0 时，直接退出函数而未释放 `np`，导致资源泄漏。

#解决方案
使用 `__free(device_node)` 清理属性修饰 `device_node` 指针变量 `np`。该宏利用 GCC 的自动清理属性，当变量超出作用域（即函数返回）时，编译器会自动插入对 `of_node_put()` 的调用，从而确保所有退出路径均能正确释放设备节点资源，避免手动添加 `of_node_put` 可能引入的遗漏。
