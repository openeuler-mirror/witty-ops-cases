#标题
media: venus: Fix pm_runtime_set_suspended() with runtime pm enabled

#内核版本
OLK-6.6

#问题现象
在设备运行时电源管理（Runtime PM）已启用的情况下调用 `pm_runtime_set_suspended()` 函数失效，该函数返回 `-EAGAIN` 错误，导致设备状态设置不正确。

#主要错误日志
未知

#问题根因
根据 Runtime PM 机制，对于已启用 Runtime PM 的设备调用 `pm_runtime_set_suspended()` 是无效操作，此时该函数会直接返回 `-EAGAIN`，因为设备可能已经处于工作状态。代码逻辑中未在调用该函数前禁止 Runtime PM，导致了此错误。

#解决方案
调整 `venus_probe` 错误处理路径中的函数调用顺序：先调用 `pm_runtime_disable()` 禁用 Runtime PM，随后再调用 `pm_runtime_set_suspended()` 设置设备状态，从而确保状态设置操作合法有效。
