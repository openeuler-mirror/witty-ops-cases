#标题
amdgpu: JPEG v2.5/v2.6 ring 不支持 64-bit user fence 导致 CS 提交异常

#内核版本
OLK-6.6

#问题现象
在涉及 JPEG v2.5 或 v2.6 ring 的场景下，若用户态提交的命令流包含 64-bit user fence 写操作，由于硬件不支持该特性，可能导致 CS 提交失败或引发未定义的硬件行为。

#主要错误日志
未知

#问题根因
JPEG rings 硬件本身不支持 64-bit user fence writes，但驱动层未对该功能进行屏蔽或标记。由于缺少 `no_user_fence` 属性设置，驱动未能正确拒绝带有 user fence 的 CS 提交。

#解决方案
在 JPEG v2.5 和 v2.6 的 ring 函数结构体 `jpeg_v2_5_dec_ring_vm_funcs` 和 `jpeg_v2_6_dec_ring_vm_funcs` 中，添加 `.no_user_fence = true` 属性。此举明确告知驱动层该 ring 不支持 user fence，从而在 CS 提交阶段拒绝包含 user fence 的请求，避免后续硬件异常。
