#标题
scsi: target: spc: RSOC 响应参数数据头部长度计算错误

#内核版本
OLK-6.6

#问题现象
SCSI target 子系统在处理 REPORT SUPPORTED OPERATION CODES (RSOC) 命令时，返回的参数数据头部的 COMMAND DATA LENGTH 字段数值计算错误。根据 SPC 规范，该字段应表示命令描述符列表的长度，但当前实现将实际长度减去了 3 字节，导致返回给发起端的长度指示比实际内容少 1 字节，可能引发发起端解析异常或合规性测试失败。

#主要错误日志
未知

#问题根因
在 `spc_emulate_report_supp_op_codes` 函数中，计算 RSOC 响应数据长度时逻辑有误。SPC 文档规定 "The COMMAND DATA LENGTH field indicates the length in bytes of the command descriptor list"，该长度应通过总长度减去 4 字节（头部大小）来获得，而代码错误地减去了 3 字节。

#解决方案
修正 `drivers/target/target_core_spc.c` 中的长度计算逻辑，将 `put_unaligned_be32(response_length - 3, buf)` 修改为 `put_unaligned_be32(response_length - 4, buf)`，确保返回的 COMMAND DATA LENGTH 字段准确反映命令描述符列表的实际字节长度。
