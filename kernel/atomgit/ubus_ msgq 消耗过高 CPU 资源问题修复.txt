#标题
ubus: msgq 消耗过高 CPU 资源问题修复

#内核版本
OLK-6.6

#问题现象
系统运行过程中，ubus 模块在进行端口复位消息发送（port reset msg）处理时，导致 msgq 相关任务长时间占用 CPU 资源，造成 CPU 使用率异常升高，可能影响系统其他任务的实时性。

#主要错误日志
ub_info(uent, "Sync request port reset msg\n");
ub_err(uent, "msg sync request ret=%d\n", ret);

#问题根因
在 `hi_send_port_reset_msg` 函数中，调用 `hi_message_sync_request` 发送同步请求时，该函数在等待响应期间未主动让出 CPU，导致 CPU 资源被持续占用且无法调度给其他任务，从而引发 CPU 资源消耗过高的问题。

#解决方案
将同步请求函数调用由 `hi_message_sync_request` 替换为 `hi_message_sync_request_sched`。后者在等待同步请求响应的过程中支持调度切换，避免了忙等或长时间占用 CPU 的情况，从而解决了 CPU 资源消耗过高的问题。
