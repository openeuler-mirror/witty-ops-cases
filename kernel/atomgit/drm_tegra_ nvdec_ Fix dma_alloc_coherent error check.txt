#标题
drm/tegra: nvdec: Fix dma_alloc_coherent error check

#内核版本
OLK-6.6

#问题现象
在 Tegra NVDEC 驱动加载固件过程中，当 `dma_alloc_coherent` 分配内存失败返回 NULL 时，内核错误地调用了 `dma_mapping_error` 进行检查。这导致驱动未能正确处理内存分配失败的情况，可能导致后续代码访问非法指针引发内核异常或加载流程处理不当。

#主要错误日志
未知

#问题根因
驱动程序在调用 `dma_alloc_coherent` 后，错误地使用 `dma_mapping_error` 来检查返回的 DMA 地址 `iova`，而没有检查虚拟地址 `virt` 是否为 NULL。根据 DMA API 规范，`dma_alloc_coherent` 分配失败时返回 NULL，此时调用 `dma_mapping_error` 检查 `iova` 属于 DMA API 误用，导致无法正确捕获内存分配失败的错误。

#解决方案
修改 `nvdec_load_falcon_firmware` 函数中的错误检查逻辑，移除对 `dma_mapping_error` 的调用，改为直接检查 `dma_alloc_coherent` 返回的虚拟地址指针 `virt` 是否为 NULL。若为 NULL，则直接返回 `-ENOMEM` 错误码，从而正确处理内存分配失败的场景。
