#标题
ALSA: ump: 修复 MIDI 1.0 Function Block 信息解析错误导致设备无法识别

#内核版本
OLK-6.6

#问题现象
在使用支持 UMP (USB MIDI 2.0 Packet Protocol) 的内核时，当 UMP Function Block 的 MIDI 1.0 属性字段值为 2（表示 MIDI 1.0 且低速）时，系统无法正确检测到该 Function Block 为 MIDI 1.0 设备，导致 MIDI 1.0 功能块识别缺失，相关音频/MIDI 功能异常。

#主要错误日志
未知

#问题根因
内核在解析 UMP Function Block info 的 `m1.0` 字段时存在逻辑错误。开发者最初将该字段误解为位标志，但实际上该字段是一个枚举值（取值范围 0-2），其中数值 2 代表“MIDI 1.0 且低速”。原代码直接将枚举值赋给标志位，导致当值为 2 时，未能设置正确的 MIDI 1.0 标志位，从而丢失了设备类型信息。

#解决方案
修正 `m1.0` 字段的解析逻辑，不再将其直接作为标志位赋值，而是进行枚举值判断：
- 若 `m1.0` 值小于 2，则直接使用该值；
- 若 `m1.0` 值等于 2，则显式设置 `SNDRV_UMP_BLOCK_IS_MIDI1` 和 `SNDRV_UMP_BLOCK_IS_LOWSPEED` 标志位，以确保正确识别 MIDI 1.0 低速设备。
