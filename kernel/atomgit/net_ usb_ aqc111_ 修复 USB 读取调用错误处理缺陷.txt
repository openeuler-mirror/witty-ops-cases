#标题
net: usb: aqc111: 修复 USB 读取调用错误处理缺陷

#内核版本
OLK-6.6

#问题现象
AQC111 USB 网卡驱动在进行 USB 读取操作时，若 `usbnet_read_cmd` 或 `usbnet_read_cmd_nopm` 返回非负值但小于预期的数据长度（即“短读”），系统未能正确识别为错误。这可能导致驱动读取到不完整或无效的寄存器数据，进而引发后续逻辑异常或功能故障。

#主要错误日志
- `Failed to read(0x%x) reg index 0x%04x: %d\n`

#问题根因
驱动原有的错误处理逻辑仅检查了 `ret < 0` 的情况，忽略了 `ret >= 0` 但 `ret < size` 的“短读”场景。根据 USB 子系统的返回值约定，当读取的数据长度小于请求的大小时，同样属于异常情况，原有代码未对此进行处理，导致错误被忽略。

#解决方案
修改 `aqc111_read_cmd` 和 `aqc111_read_cmd_nopm` 函数中的错误判断逻辑：
1. 将错误检查条件从 `ret < 0` 修改为 `ret < size`，确保短读情况也能被捕获。
2. 在触发告警日志后，增加返回值修正逻辑：若 `ret` 小于 0 则保持原错误码，若 `ret` 大于等于 0 但小于 `size` 则返回 `-ENODATA` 错误码。
