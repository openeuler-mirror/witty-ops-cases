#标题
soc_cache: Fix incorrect use of cpuhp_remove_multi_state parameter

#内核版本
OLK-6.6

#问题现象
HiSilicon SoC cache 驱动在初始化失败或模块卸载时，调用 `cpuhp_remove_multi_state` 函数使用了错误的参数（`CPUHP_AP_ONLINE_DYN` 而非动态分配的状态变量），导致 CPU 热插拔状态移除操作不当。

#主要错误日志
未知

#问题根因
`cpuhp_remove_multi_state` 被错误地调用，传入了静态常量 `CPUHP_AP_ONLINE_DYN` 而非动态分配的 `hisi_l3c_cpuhp_state`。这导致驱动无法正确移除先前注册的 CPU 热插拔状态。

#解决方案
修正 `hisi_soc_l3c_init` 退出路径和 `hisi_soc_l3c_exit` 卸载函数中的参数传递，将 `cpuhp_remove_multi_state` 的参数从 `CPUHP_AP_ONLINE_DYN` 修改为 `hisi_l3c_cpuhp_state`，确保移除的是实际分配的状态。
