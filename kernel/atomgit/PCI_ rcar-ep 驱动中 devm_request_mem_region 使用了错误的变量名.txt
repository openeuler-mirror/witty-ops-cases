#标题
PCI: rcar-ep 驱动中 devm_request_mem_region 使用了错误的变量名

#内核版本
OLK-6.6

#问题现象
Renesas R-Car PCIe Endpoint 驱动在解析出站地址区间时，内存区域请求操作失败，导致驱动加载或功能初始化异常，内核日志打印无法请求内存区域的错误信息。

#主要错误日志
dev_err(pcie->dev, "Cannot request memory region %s.\n", outbound_name);

#问题根因
在调用 devm_request_mem_region 函数时，传入的名称参数错误地使用了局部变量 outbound_name，而该变量可能未正确初始化或与实际资源不匹配。正确的逻辑应当传入资源结构体中的名称成员 res->name。

#解决方案
修改 drivers/pci/controller/pcie-rcar-ep.c 文件中 rcar_pcie_parse_outbound_ranges 函数，将 devm_request_mem_region 调用的第四个参数从 outbound_name 修正为 res->name，确保请求内存区域时使用正确的资源名称。
