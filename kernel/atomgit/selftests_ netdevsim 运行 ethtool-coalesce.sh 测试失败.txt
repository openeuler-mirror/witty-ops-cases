#标题
selftests: netdevsim 运行 ethtool-coalesce.sh 测试失败

#内核版本
OLK-6.6

#问题现象
在执行 drivers/net/netdevsim 目录下的 ethtool-coalesce.sh 自测试脚本时，因找不到依赖文件导致测试失败，具体表现为：
1. 脚本提示 "No such file or directory"；
2. 多个内部函数（如 make_netdev, check）提示 "command not found"；
3. 测试结果为 FAILED，退出码为 1。

#主要错误日志
# ./ethtool-coalesce.sh: line 4: ethtool-common.sh: No such file or directory
# ./ethtool-coalesce.sh: line 25: make_netdev: command not found
# ethtool: bad command line argument(s)
# ./ethtool-coalesce.sh: line 124: check: command not found
# ./ethtool-coalesce.sh: line 126: [: -eq: unary operator expected
# FAILED /0 checks
not ok 1 selftests: drivers/net/netdevsim: ethtool-coalesce.sh # exit=1

#问题根因
脚本 "ethtool-common.sh" 未被添加到 Makefile 的 TEST_FILES 变量中，导致执行 make install 时该依赖文件未被安装到 INSTALL_PATH 目录下，从而在运行测试时因找不到文件而报错。

#解决方案
在 tools/testing/selftests/drivers/net/netdevsim/Makefile 中，将 "ethtool-common.sh" 添加到 TEST_FILES 变量中，确保该脚本随测试套件一同安装，修复后测试可正常通过（PASSED all 22 checks）。
