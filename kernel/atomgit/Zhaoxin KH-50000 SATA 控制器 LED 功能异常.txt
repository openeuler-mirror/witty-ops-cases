#标题
Zhaoxin KH-50000 SATA 控制器 LED 功能异常

#内核版本
OLK-6.6

#问题现象
在 Zhaoxin KH-50000 平台上，SATA 控制器（Device ID: 0x9083, Revision: 0x40）的 Enclosure Management (EM) LED 功能无法正常工作。具体表现为硬盘活动指示灯状态不正确或无法响应 LED 控制指令。若在运行时移除相关设备（如 P1 设备），内核可能会输出警告信息。

#主要错误日志
- `ahci: zx led quirk init`
- `P1 removed, LED mode unavailable`（推断：在 P1 设备移除且尝试操作 LED 时由 `dev_warn_ratelimited` 输出）

#问题根因
Zhaoxin 处理器平台的 SATA 控制器存在硬件设计特性（quirk）。其 EM（Enclosure Management）LED 功能依赖特定的内存映射 I/O（MMIO）地址管理机制。具体而言，该控制器分为 P0 和 P1 两部分，LED 控制逻辑需要正确识别并访问这两部分的 MMIO 地址。由于内核驱动默认未针对该型号（Device ID 0x9083, Revision 0x40）实现特定的 MMIO 地址映射与跨设备关联逻辑，导致 LED 控制指令无法发送到正确的硬件寄存器地址，从而引发功能失效。

#解决方案
针对 Zhaoxin SATA 控制器（Device ID 0x9083, Revision 0x40）实施特定的驱动层修复：
1. **增加硬件识别**：在 `ahci_init_one` 初始化阶段，通过读取特定 MSR 寄存器（`ZX_GET_BUS_NUMBER_QUIRK`）识别控制器的 P0/P1 总线号与索引。
2. **扩展数据结构**：在 `ahci_host_priv` 结构体中新增 `p1_mmio`、`sx_index`、`px_index`、`has_p0_p1` 字段，用于存储跨设备的 MMIO 地址和拓扑信息。
3. **实现初始化 Quirk**：新增 `ahci_zx_led_init_quirk` 函数，遍历 PCI 设备列表，建立 P0 与 P1 设备之间的关联，共享 MMIO 地址，确保 LED 控制逻辑能找到正确的寄存器基址。
4. **修正 LED 读写路径**：在 `ahci_transmit_led_message` 和 `ahci_store_em_buffer` 函数中引入 `led_get_mmio_quirk` 辅助函数，根据当前设备索引（P0 或 P1）动态选择正确的 MMIO 地址进行寄存器读写。
5. **实现卸载清理**：新增 `ahci_zx_led_remove_quirk` 函数，在设备移除时清理相关的 Quirk 数据，防止访问已释放的资源。
