#标题
rtl8723bs: rtw_restruct_wmm_ie 函数潜在越界读修复

#内核版本
OLK-6.6

#问题现象
在处理 rtl8723bs 无线驱动 WMM IE 信息时，若输入数据长度 `in_len` 接近临界值，函数 `rtw_restruct_wmm_ie` 可能在校验数组边界前访问越界索引，导致内核发生潜在越界读取。此问题可能引发内核崩溃或内存破坏。

#主要错误日志
未知

#问题根因
代码逻辑存在顺序错误。原代码在 if 语句末尾才检查 `i + 5 < in_len`，但在检查之前已经访问了 `in_ie[i + 5]` 以及 `in_ie[i + 2]` 到 `in_ie[i + 4]` 等元素。如果 `i + 5` 超出 `in_len` 范围，访问 `in_ie[i + 5]` 时将发生越界读取。

#解决方案
将长度校验条件 `i + 5 < in_len` 移至 if 语句的开头。利用逻辑运算符的短路求值特性，确保在访问数组元素 `in_ie[i + 2]` 到 `in_ie[i + 5]` 之前，先验证索引处于有效范围内，从而避免越界访问。
