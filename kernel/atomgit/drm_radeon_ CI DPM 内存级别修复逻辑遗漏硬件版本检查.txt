#标题
drm/radeon: CI DPM 内存级别修复逻辑遗漏硬件版本检查

#内核版本
OLK-6.6

#问题现象
特定 Hawaii 版本的 AMD Radeon 显卡（Device ID 为 0x67B0 或 0x67B1）在非 Revision 0 的硬件上，内核错误地应用了仅适用于 Revision 0 SKU 的内存级别变通处理。这会导致电源管理（DPM）配置错误，可能引发显存频率设置异常、电压配置不当或系统不稳定等问题。

#主要错误日志
未知

#问题根因
驱动程序在应用内存级别变通方案时，仅检查了设备 ID（0x67B0 或 0x67B1），遗漏了对硬件版本号的检查。该变通方案仅适用于 Revision 0 的 SKU，但在修复前被错误地应用到了所有修订版本的硬件上。

#解决方案
在 `ci_dpm.c` 文件的三个相关位置（`ci_register_patching_mc_arb`、`ci_populate_all_memory_levels` 和 `ci_register_patching_mc_seq` 函数中）增加条件判断：`rdev->pdev->revision == 0`。确保内存时钟和电压的相关调整逻辑仅在设备 ID 匹配且硬件版本为 0 时才执行。
