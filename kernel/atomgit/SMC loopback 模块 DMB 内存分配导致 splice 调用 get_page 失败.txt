#标题
SMC loopback 模块 DMB 内存分配导致 splice 调用 get_page 失败

#内核版本
OLK-6.6

#问题现象
在使用 SMC (Shared Memory Communications) loopback 功能时，如果应用程序对 SMC socket 进行 splice 操作（例如调用 `smc_rx_splice()`），内核会触发警告信息。这是因为底层内存分配方式不兼容 `get_page()` 接口要求，导致内存引用计数处理异常。

#主要错误日志
未知

#问题根因
SMC loopback 模块在注册 DMB（Data Memory Buffer）时，使用了 `kzalloc`（或 `kvfree` 对应的分配方式）分配内存。由于 `kzalloc` 分配的内存不属于独立的 struct page 结构管理，在进行 splice 操作调用 `get_page()` 时，无法正确处理该内存页的引用计数，从而触发内核警告或异常。

#解决方案
修改 DMB 内存分配策略，将 `kzalloc` 替换为 `folio_alloc` 接口进行分配。
1. 使用 `folio_alloc` 结合 `__GFP_ZERO` 标志分配内存，确保分配的内存由 struct page（folio）管理。
2. 通过 `folio_address` 获取分配到的虚拟地址。
3. 在 DMB 注销释放内存时，将 `kvfree` 替换为 `folio_put(virt_to_folio(...))`，确保内存页引用计数正确释放。此举保证了 splice 操作中 `get_page()` 调用的正确性。
