#标题
HID pidff 方向盘力反馈方向错误导致失效或反转

#内核版本
OLK-6.6

#问题现象
部分赛车游戏方向盘设备（如 Moza Racing 设备）在游戏中出现力反馈完全失效、部分失效或力反馈方向反转（unexpected force reversal）的现象。该问题通常发生在游戏或中间转换层（如 dinput8 -> wine -> SDL）未正确设置力反馈效果的 DIRECTION 字段时，且设备严格遵循 PID 标准进行力计算的情况下。

#主要错误日志
未知

#问题根因
大多数方向盘设备会忽略 PID 协议中的 DIRECTION 字段，但部分严格遵循 PID 标准的设备会将其用于力计算。由于游戏软件或 API 转换层（dinput8 -> wine -> SDL -> Linux API）经常未能正确设置该字段，导致传递给设备的 DIRECTION 值不正确。错误的 DIRECTION 值被用于力计算后，引发了力反馈丢失或方向反转。

#解决方案
在 HID PIDFF 驱动中引入 `HID_PIDFF_QUIRK_FIX_WHEEL_DIRECTION` quirk 标志。对于存在此问题的设备，在 `pidff_set_effect_report` 函数中强制将效果方向重置为固定的 90 度（0x4000，即北极方向），忽略上层应用传入的错误方向值。这修复了受影响设备上的力反馈功能。
