#标题
ACPI: processor: idle: Fix NULL pointer dereference in hotplug path

#内核版本
OLK-6.6

#问题现象
系统启动过程中 cpuidle_device 可能注册失败，但系统继续运行。后续在进行 CPU 热插拔操作时，触发了空指针引用错误，导致内核异常。

#主要错误日志
未知

#问题根因
在 acpi_processor_hotplug 函数中，代码未对 per-cpu 变量 acpi_cpuidle_device 进行有效性检查。当 cpuidle_device 在启动阶段注册失败时，该指针为 NULL，热插拔路径直接访问该指针引发了空指针引用。

#解决方案
在 acpi_processor_hotplug 函数中增加对 per-cpu 变量 acpi_cpuidle_device 的 NULL 指针检查。如果指针为空，函数直接返回 -ENODEV，避免后续操作解引用空指针。
