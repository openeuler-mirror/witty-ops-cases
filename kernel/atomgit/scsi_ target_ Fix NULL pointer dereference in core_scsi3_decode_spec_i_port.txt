#标题
scsi: target: Fix NULL pointer dereference in core_scsi3_decode_spec_i_port

#内核版本
OLK-6.6

#问题现象
在 SCSI target 子系统中处理 Persistent Reservations 相关操作时，调用 `core_scsi3_decode_spec_i_port` 函数可能触发内核空指针解引用，导致内核崩溃或 Oops。

#主要错误日志
未知

#问题根因
在函数 `core_scsi3_decode_spec_i_port` 的错误处理路径中，变量 `dest_se_deve` 可能未初始化或为 NULL。代码在调用 `kmem_cache_free` 释放内存后，无条件地调用 `core_scsi3_lunacl_undepend_item(dest_se_deve)`，未对 `dest_se_deve` 进行有效性检查，从而引发了空指针解引用问题。

#解决方案
在调用 `core_scsi3_lunacl_undepend_item` 函数前增加判断条件，仅当 `dest_se_deve` 不为 NULL 时才执行该函数调用，避免对空指针进行解引用操作。
