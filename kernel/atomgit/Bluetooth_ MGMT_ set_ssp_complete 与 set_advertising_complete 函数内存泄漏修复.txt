#标题
Bluetooth: MGMT: set_ssp_complete 与 set_advertising_complete 函数内存泄漏修复

#内核版本
OLK-6.6

#问题现象
内核蓝牙子系统存在内存泄漏问题。在执行 SSP（Secure Simple Pairing）设置命令或广播设置命令完成时，`mgmt_pending_cmd` 结构体及其关联的参数数据未被释放。随着命令完成次数增加，系统内存会逐渐减少，导致内存资源耗尽。

#主要错误日志
未知

#问题根因
Commit 302a1f674c00 ("Bluetooth: MGMT: Fix possible UAFs") 为了修复 UAF 漏洞，将 `mgmt_pending_foreach()` 调用替换为了单独的命令处理逻辑，但在 `set_ssp_complete()` 函数的成功和错误返回路径中，以及 `set_advertising_complete()` 函数中，遗漏了调用 `mgmt_pending_free(cmd)` 来释放从 pending list 中移除的命令结构体，从而导致内存泄漏。

#解决方案
在 `set_ssp_complete()` 函数的错误处理路径（err 分支）末尾、正常执行路径末尾，以及 `set_advertising_complete()` 函数末尾，补充调用 `mgmt_pending_free(cmd)`，确保命令处理完成后相关内存资源被正确释放。
