#标题
KVM x86 虚拟机 TSC 为 0 时导致无限循环故障

#内核版本
OLK-6.6

#问题现象
在 KVM 虚拟化场景下，当客户机的 TSC（时间戳计数器）值为 0 时，`kvm_guest_time_update` 函数会陷入无限循环。这会导致相关的内核线程卡死，无法正常推进时间更新流程，进而影响虚拟机的正常运行。

#主要错误日志
未知

#问题根因
在 `__scale_tsc` 函数中，计算 TSC 缩放值的逻辑 `mul_u64_u64_shr` 在输入 TSC 为 0 时返回 0。上层函数 `kvm_guest_time_update` 依赖该返回值进行循环判断或计算，由于返回值为 0 导致循环条件始终无法满足（或计算异常），从而引发无限循环。

#解决方案
修改 `__scale_tsc` 函数，增加对计算结果为 0 的判断。如果 TSC 缩放后的结果为 0，强制返回 1，从而打破上层逻辑的无限循环，确保时间更新流程正常推进。
