#标题
idxd: fix memory leak in error handling path of idxd_setup_engines

#内核版本
OLK-6.6

#问题现象
在 idxd_setup_engines() 函数的执行过程中，若 dev_set_name 调用失败，函数将进入错误处理路径。此时，此前已经为 engine 结构体分配的内存未被释放，导致内核内存泄漏（memory leak）。

#主要错误日志
未知

#问题根因
在 idxd_setup_engines() 函数中，如果为设备设置名称（dev_set_name）失败，代码会跳转到错误处理标签（err）。原错误处理逻辑仅释放了设备对象（put_device），但未释放为 engine 结构体分配的内存，从而导致内存泄漏。

#解决方案
在错误处理路径中，增加对已分配的 engine 内存（engine pointer）的释放操作（kfree(engine)），确保在函数退出前回收资源。同时，完善 err 标签后的清理逻辑，确保所有已分配的资源按申请的逆序被正确释放。
