#标题
drm/amdgpu: Runtime resume failure due to incorrect power state flag setting

#内核版本
OLK-6.6

#问题现象
系统在进行 runtime suspend（运行时挂起）流程后，触发 runtime resume（运行时恢复）失败，导致显卡无法正常从低功耗状态唤醒。

#主要错误日志
未知

#问题根因
在 `amdgpu_choose_low_power_state` 函数中，未对运行时电源管理进行判断。当系统进入 runtime suspend 时，代码错误地设置了电源状态标志（如 `in_s0ix` 或 `in_s3`），这导致了后续 runtime resume 流程失败。

#解决方案
在 `amdgpu_choose_low_power_state` 函数入口处增加判断，如果设备处于运行时电源管理挂起流程中（`adev->in_runpm` 为真），则直接返回，不再设置电源状态标志，从而避免 runtime resume 失败。
