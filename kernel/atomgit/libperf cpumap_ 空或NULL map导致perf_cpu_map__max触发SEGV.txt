#标题
libperf cpumap: 空或NULL map导致perf_cpu_map__max触发SEGV

#内核版本
OLK-6.6

#问题现象
当向 `perf_cpu_map__max` 函数传入一个空（empty）或 NULL 的 map 时，触发段错误（SEGV），导致程序崩溃。

#主要错误日志
未知

#问题根因
函数 `perf_cpu_map__max` 在处理传入参数时，未显式检查 map 是否为 NULL 或空。
根据代码逻辑，0 长度的 map 被编码为 NULL，若直接访问 `__perf_cpu_map__nr(map)` 或 `__perf_cpu_map__cpu` 相关内存，会引发空指针解引用，导致 SEGV。

#解决方案
在 `perf_cpu_map__max` 函数入口处增加对 map 指针的非空校验。
若 map 为 NULL，直接返回预设的无效结果（cpu = -1）；否则继续执行原有逻辑获取最大 CPU 编号。这确保了函数在处理空 map 时的健壮性。
