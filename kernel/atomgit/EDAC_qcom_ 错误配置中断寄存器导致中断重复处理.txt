#标题
EDAC/qcom: 错误配置中断寄存器导致中断重复处理

#内核版本
OLK-6.6

#问题现象
QCOM SoC 的 LLCC EDAC 驱动在配置 Tag 和 Data RAM ECC 中断时，错误地使用了 `cmn_interrupt_2_enable` 寄存器，导致中断处理异常。具体表现为：中断被重复处理，即 EL1（内核驱动）和 EL3（固件/安全环境）同时响应并处理了同一个中断。

#主要错误日志
未知

#问题根因
代码逻辑错误，在 `qcom_llcc_core_setup` 函数中配置 Tag 和 Data RAM ECC 中断时，错误地使用了 `cmn_interrupt_2_enable` 寄存器。根据硬件设计，`cmn_interrupt_2_enable` 是专门用于配置需由 EL3 处理的中断，而 EL1 LLCC EDAC 驱动应当使用 `cmn_interrupt_0_enable` 寄存器来配置这些中断。

#解决方案
修改 `drivers/edac/qcom_edac.c` 文件中的 `qcom_llcc_core_setup` 函数，将配置 Tag RAM 和 Data RAM ECC 中断的寄存器从 `cmn_interrupt_2_enable` 修正为 `cmn_interrupt_0_enable`，确保中断仅由 EL1 驱动处理，避免双重处理问题。
