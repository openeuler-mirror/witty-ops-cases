#标题
soc cache 驱动用户态输入校验整数溢出漏洞

#内核版本
OLK-6.6

#问题现象
HiSilicon SoC cache 驱动在进行缓存维护操作时，对用户态传入的 size 参数校验存在缺陷。恶意用户通过传入超大的 size 值，可导致整数溢出，从而绕过缓冲区大小的合法性校验，可能引发非法内存访问或内核安全漏洞。

#主要错误日志
未知

#问题根因
驱动在校验用户控制的 size 参数时，通过将其加到 vm_start 上并与 end 比较的方式进行检查。当恶意用户传入一个极大的 size 值时，`vaddr + size` 发生整数溢出回绕，导致计算结果变小，从而绕过了原有的 `vaddr + size > vma->vm_end` 大小检查，使得非法参数通过验证。

#解决方案
修复代码移除了依赖加法溢出的检查逻辑，改为直接检查 size 是否为 0 以及 `vaddr + size` 是否发生回绕（`vaddr + size < vaddr`），并利用 `range_in_vma(vma, vaddr, vaddr + size)` 函数严格校验地址范围是否完全包含在 VMA 中，从而防止整数溢出导致的检查绕过。
