#标题
drm/amd/display 传递空指针导致 dcn20_validate_apply_pipe_split_flags 访问异常

#内核版本
OLK-6.6

#问题现象
在 AMD GPU 显示驱动进行带宽快速校验（fast_validate_bw）流程中，调用 `dcn20_validate_apply_pipe_split_flags` 函数时，最后一个参数传入为 NULL，导致内核可能发生空指针引用错误或不可预测的显示行为。

#主要错误日志
未知

#问题根因
在 `dcn20_fast_validate_bw` 和 `dcn21_fast_validate_bw` 函数中，调用 `dcn20_validate_apply_pipe_split_flags` 时直接传递了 NULL 作为 `merge` 参数。而被调用函数内部可能会对该参数指针进行解引用操作，从而引发空指针访问故障。

#解决方案
在 `dcn20_fast_validate_bw` 和 `dcn21_fast_validate_bw` 函数内部定义有效的 `merge` 数组变量（`bool merge[MAX_PIPES] = { false };`），并将该数组的地址作为参数传递给 `dcn20_validate_apply_pipe_split_flags` 函数，替代原有的 NULL 指针传入。
