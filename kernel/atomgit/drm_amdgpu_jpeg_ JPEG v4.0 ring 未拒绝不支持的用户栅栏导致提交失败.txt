#标题
drm/amdgpu/jpeg: JPEG v4.0 ring 未拒绝不支持的用户栅栏导致提交失败

#内核版本
OLK-6.6

#问题现象
在使用 JPEG v4.0 环进行任务提交（CS）时，如果用户指定了 64-bit user fence，由于硬件不支持该功能但驱动未拦截，会导致功能异常或提交处理错误。

#主要错误日志
未知

#问题根因
JPEG rings 硬件本身不支持 64-bit user fence writes（用户栅栏写操作），但驱动程序在初始化 jpeg_v4_0_dec_ring_vm_funcs 结构体时未设置 no_user_fence 标志位，导致驱动层未能正确拦截包含 user fence 的 CS（Command Submission）请求。

#解决方案
在 drivers/gpu/drm/amd/amdgpu/jpeg_v4_0.c 文件的 `jpeg_v4_0_dec_ring_vm_funcs` 结构体定义中，添加 `.no_user_fence = true` 属性。该修改明确告知驱动层该 ring 不支持用户栅栏，从而在提交阶段直接拒绝包含 user fence 的非法请求。
