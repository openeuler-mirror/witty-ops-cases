#标题
[net: ipa: fix event ring index not programmed for IPA v5.0+]

#内核版本
OLK-6.6

#问题现象
在 IPA v5.0+ 版本硬件上，事件环索引未能正确编程写入硬件寄存器，导致网络数据通道配置异常，可能引发数据传输功能故障或硬件状态错误。

#主要错误日志
未知

#问题根因
在文件 `drivers/net/ipa/reg/gsi_reg-v5.0.c` 中，定义 `reg_ch_c_cntxt_1_fmask` 数组时，索引 `ERINDEX` 对应的枚举成员名称错误。代码中使用了 `[ERINDEX]`，但根据硬件寄存器定义或驱动枚举定义，正确的成员名称应为 `[CH_ERINDEX]`。该名称不匹配导致相应的掩码 `GENMASK(31, 24)` 未能正确赋值给预期的字段，致使事件环索引配置失效。

#解决方案
修正寄存器字段定义中的枚举成员名称，将 `reg_ch_c_cntxt_1_fmask` 数组中的 `[ERINDEX]` 修改为 `[CH_ERINDEX]`，确保事件环索引（ERINDEX）字段能够被正确编程。
