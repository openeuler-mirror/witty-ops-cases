#标题
网桥组播查询间隔设置过大导致 soft lockup

#内核版本
OLK-6.6

#问题现象
当用户将网桥的组播查询间隔或启动查询间隔设置为极大的数值时，内核在 `br_multicast_query_expired` 函数中触发 soft lockup，导致 CPU 长时间被困在循环中无法调度，系统响应迟缓或卡死。

#主要错误日志
- trying to set multicast query interval above maximum, setting to %lu (%ums)
- trying to set multicast startup query interval above maximum, setting to %lu (%ums)

#问题根因
系统对 `multicast_query_interval` 和 `multicast_startup_query_interval` 这两个参数缺少最大值校验。当用户配置的时间间隔过大时，会导致定时器溢出或计算错误，进而引发 soft lockup。

#解决方案
在 `br_multicast_set_query_intvl` 和 `br_multicast_set_startup_query_intvl` 函数中增加最大值限制检查。
1. 定义最大值宏 `BR_MULTICAST_QUERY_INTVL_MAX` 为 24 小时（86400000 ms）。
2. 在设置参数时，如果传入的间隔超过最大值，强制将其限制为最大值，并打印提示日志告知用户。
