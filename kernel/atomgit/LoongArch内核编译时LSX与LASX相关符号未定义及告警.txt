#标题
LoongArch内核编译时LSX与LASX相关符号未定义及告警

#内核版本
OLK-6.6

#问题现象
在编译LoongArch架构内核时，若配置项 `CONFIG_CPU_HAS_LBT` 已设置，但 `CONFIG_CPU_HAS_LSX` 和 `CONFIG_CPU_HAS_LASX` 未设置，编译过程中会触发 objtool 意外重定位告警，且 modpost 阶段会因缺少符号定义而报错，导致构建失败。

#主要错误日志
arch/loongarch/kernel/fpu.o: warning: objtool: unexpected relocation symbol type in .rela.discard.func_stack_frame_non_standard: 0
arch/loongarch/kvm/switch.o: warning: objtool: unexpected relocation symbol type in .rela.discard.func_stack_frame_non_standard: 0
ERROR: modpost: "kvm_restore_lsx" [arch/loongarch/kvm/kvm.ko] undefined!
ERROR: modpost: "kvm_restore_lasx" [arch/loongarch/kvm/kvm.ko] undefined!

#问题根因
在启用 `CONFIG_CPU_HAS_LBT` 但未启用 `CONFIG_CPU_HAS_LSX` 和 `CONFIG_CPU_HAS_LASX` 的配置场景下，汇编文件 `fpu.S` 和 `switch.S` 中的 `_restore_lsx`、`_restore_lasx` 以及 `kvm_restore_lsx`、`kvm_restore_lasx` 函数定义不存在。然而，`STACK_FRAME_NON_STANDARD` 宏引用了这些不存在的符号，导致编译器生成无效的重定位条目，进而触发 objtool 告警和 modpost 的未定义符号错误。

#解决方案
在汇编文件中针对 LSX 和 LASX 相关的 `STACK_FRAME_NON_STANDARD` 宏调用添加 `#ifdef CONFIG_CPU_HAS_LSX` 和 `#ifdef CONFIG_CPU_HAS_LASX` 条件编译判断，确保只有在相应功能开启时才引用相关符号，从而消除编译警告与错误。
