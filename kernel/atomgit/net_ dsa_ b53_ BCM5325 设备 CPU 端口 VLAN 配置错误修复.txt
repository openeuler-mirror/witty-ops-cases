#标题
net: dsa: b53: BCM5325 设备 CPU 端口 VLAN 配置错误修复

#内核版本
OLK-6.6

#问题现象
在 BCM5325 交换机设备上，内核设置 CPU 端口 VLAN 掩码时使用了错误的端口号（使用 B53_CPU_PORT_25 而非 B53_CPU_PORT_8），导致 CPU 端口无法正确加入 Port-based VLAN，进而造成网络数据包无法正确转发到 CPU，网络通信功能异常。

#主要错误日志
未知

#问题根因
b53_imp_vlan_setup 函数在处理 BCM5325（及 BCM5365）设备时，直接使用了传入的 cpu_port 参数（通常为 B53_CPU_PORT_25，即端口 5）来设置 B53_PVLAN_PORT_MASK 寄存器。然而，BCM5325 设备的 CPU 端口实际上是端口 8（B53_CPU_PORT），这导致设置的 Port-based VLAN 掩码指向了错误的物理端口，破坏了 VLAN 隔离与转发逻辑。

#解决方案
在 b53_imp_vlan_setup 函数中增加针对 BCM5325 和 BCM5365 设备的判断逻辑。当检测到设备为这两类型号且传入的 cpu_port 为 B53_CPU_PORT_25 时，将其修正为 B53_CPU_PORT（端口 8），确保 B53_PVLAN_PORT_MASK 寄存器写入正确的 CPU 端口号。
