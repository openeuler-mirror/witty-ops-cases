#标题
EDAC/i10nm: Granite Rapids CPU Flat2LM 配置下远端内存错误源指示器错误

#内核版本
OLK-6.6

#问题现象
在 Granite Rapids CPU 且配置为 Flat2LM 内存模式下，系统可能会将近内存错误误报为远内存错误。该错误分类导致 ADXL（Address Translation）解码组件选择错误，进而导致解码结果无效，系统日志中打印错误的 IMC（内存控制器）ID。

#主要错误日志
EDAC skx: Bad imc -1

#问题根因
Granite Rapids CPU 在 Flat2LM 内存配置下，硬件可能会错误地将近内存错误标记为远内存错误。驱动程序在解析错误源时未对这种情况进行校验，直接使用了错误的错误源标识，导致后续解码流程选择了错误的 ADXL 组件，最终解析出无效的内存控制器 ID（-1）。

#解决方案
在 `skx_adxl_decode` 函数中增加针对 Granite Rapids 处理器的特殊处理逻辑。预取解码后的远内存控制器 ID，若检测到该 ID 为无效值（-1），则将错误源指示器修正为近内存，从而确保后续内存解析流程选择正确的组件。同时在驱动初始化时通过 `skx_set_res_cfg` 保存资源配置指针以用于类型判断。
