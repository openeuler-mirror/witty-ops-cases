#标题
PCI/AER: 修复 aer_info 访问空指针导致的 NULL pointer access

#内核版本
OLK-6.6

#问题现象
在内核初始化 PCI AER（Advanced Error Reporting）功能时，若内存分配失败，后续代码未做防护直接访问 `dev->aer_stats` 指针，导致发生 NULL pointer access 异常。

#主要错误日志
未知

#问题根因
在函数 `pci_aer_init` 中，调用 `kzalloc` 分配 `aer_stats` 内存后，缺少对分配结果的校验。当内存分配失败返回 NULL 时，后续代码（如 `aer_stats` 相关的访问逻辑）未做判断直接使用了该空指针，从而触发空指针访问错误。

#解决方案
在 `kzalloc` 分配内存后增加返回值检查逻辑。若分配失败（返回 NULL），则将 `dev->aer_cap` 置 0 以禁用 AER 功能，并直接返回，防止后续代码访问空指针。
