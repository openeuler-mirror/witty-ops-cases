#标题
phy: starfive: JH7110 USB 2.0 host 偶发性检测失败

#内核版本
OLK-6.6

#问题现象
JH7110 SoC 的 USB 2.0 主机控制器在连接 USB 2.0 设备时，偶发性出现无法检测到设备的现象，导致设备无法正常识别和使用。

#主要错误日志
未知

#问题根因
经过长时间的调试与测试定位，发现该问题是由于 USB PHY 的 Rx clock gating（接收时钟门控）控制信号配置不当引起。在初始化过程中未将 Rx clock gating 设置为正常功耗模式，导致在特定条件下接收链路工作异常，从而引发设备检测失败。

#解决方案
在 USB PHY 初始化函数 `jh7110_usb2_phy_init` 中增加寄存器配置逻辑：
读取 `USB_CLK_MODE_OFF` (0x0) 偏移处的寄存器值，通过置位 `USB_CLK_MODE_RX_NORMAL_PWR` (BIT(1)) 将 Rx clock gating 控制信号设置为正常功耗模式，然后回写寄存器，从而修复偶发的设备检测失败问题。
