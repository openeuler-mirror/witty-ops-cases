#标题
net/sched: cbs_set_port_rate 函数整数溢出故障

#内核版本
OLK-6.6

#问题现象
在调用 cbs_set_port_rate 函数设置端口速率时，当网卡速度较高（如 speed = SPEED_20000）时，计算 `speed * 1000 * BYTES_PER_KBIT` 的过程中会发生整数溢出，导致计算出的 port_rate 数值错误，进而影响 CBS（Credit-Based Shaper）调度器的配置与运行。

#主要错误日志
未知

#问题根因
变量 `port_rate` 被定义为 `int` 类型。在进行速率计算 `port_rate = speed * 1000 * BYTES_PER_KBIT` 时，虽然 `BYTES_PER_KBIT` 是 long long 类型，但参与计算的 `speed` 和 `1000` 均为 int 类型。当 speed 值较大（例如 SPEED_20000）时，乘法运算的中间结果会超过 INT_MAX，从而在赋值给 port_rate 之前就已经发生了整数溢出。

#解决方案
将变量 `port_rate` 的数据类型从 `int` 修改为 `s64`（signed 64-bit），扩大变量存储范围，确保高速场景下的计算结果不会超出整型范围，从而修复溢出问题。
