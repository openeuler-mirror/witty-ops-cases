#标题
HID: pidff: 修复 pidff_find_fields 中的空指针解引用问题

#内核版本
OLK-6.6

#问题现象
在调用 `pidff_find_fields` 或 `pidff_find_special_field` 函数搜索设备上未实现的报告（report）时，触发空指针解引用，导致内核崩溃或异常。

#主要错误日志
pidff_find_fields, null report
pidff_find_special_field, null report

#问题根因
函数 `pidff_find_fields` 和 `pidff_find_special_field` 在处理搜索请求时，未对传入的 `report` 指针进行非空检查。当设备不支持特定报告时，传入的 `report` 指针可能为 NULL，后续代码直接对该指针进行解引用操作，导致空指针解引用故障。

#解决方案
在 `pidff_find_fields` 和 `pidff_find_special_field` 函数入口处增加对 `report` 指针的非空判断。如果指针为空，打印调试日志并直接返回错误码（-1 或 NULL），从而避免后续的解引用操作。
