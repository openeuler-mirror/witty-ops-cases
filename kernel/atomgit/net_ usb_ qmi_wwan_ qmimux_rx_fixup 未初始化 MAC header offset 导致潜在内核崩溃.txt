#标题
net: usb: qmi_wwan: qmimux_rx_fixup 未初始化 MAC header offset 导致潜在内核崩溃

#内核版本
OLK-6.6

#问题现象
在使用 QMI WWAN 驱动的 qmimux 功能接收 Raw IP 数据包时，若系统启用了其他子系统（如 xfrm）访问 MAC header offset，会因 offset 未初始化而导致内核访问非法内存，可能引发 kernel panic 或 oops。

#主要错误日志
未知

#问题根因
在 `qmimux_rx_fixup` 函数处理 Raw IP 数据包时，新分配的 skbn（socket buffer）没有设置 MAC header offset。Raw IP 数据包本身虽然没有 MAC 头，但内核的其他子系统（如 xfrm）可能会尝试访问 skbn 的 MAC header offset 字段。由于该 offset 未被初始化，导致访问非法内存地址。

#解决方案
在 `qmimux_rx_fixup` 函数中，分配新的 skbn 后，调用 `skb_reset_mac_header(skbn)` 显式初始化 MAC header offset，确保其他子系统访问时不会越界。
