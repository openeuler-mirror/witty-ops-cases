#标题
coda: 修复 SMMU 域配置未初始化导致的空指针引用故障

#内核版本
OLK-6.6

#问题现象
在系统启动或设备初始化过程中，当设备的 SMMU 域（smmu_domain）关联的 SMMU 设备指针未正确设置时，内核在访问该指针时触发空指针引用错误，可能导致内核 panic 或 Oops，相关设备功能异常。

#主要错误日志
CoDA: Security smmu has not been correctly set

#问题根因
在 `virtcca_create_ste_entry` 函数中，代码逻辑直接解引用 `smmu_domain->smmu` 指针，但未预先检查该指针是否为空。当 SMMU 域的 SMMU 设备未正确配置或初始化时，该指针为 NULL，导致空指针引用错误。

#解决方案
在访问 `smmu_domain->smmu` 前增加非空校验。若检测到该指针为空，则打印错误日志 "CoDA: Security smmu has not been correctly set"，释放已分配的内存（params_ptr），并返回 `-EINVAL` 错误码，从而避免非法内存访问。
