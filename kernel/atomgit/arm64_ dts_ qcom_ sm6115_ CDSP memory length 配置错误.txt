#标题
arm64: dts: qcom: sm6115: CDSP memory length 配置错误

#内核版本
OLK-6.6

#问题现象
设备树中 CDSP (Compute DSP) remoteproc 节点的内存长度配置错误。虽然 Commit 描述指出该问题对 Linux 用户没有功能性影响（因为 PAS loader 不使用该地址空间），但这属于硬件描述不匹配的静态配置缺陷。

#主要错误日志
未知

#问题根因
设备树节点 `remoteproc@b300000` 的 `reg` 属性中，内存长度被错误地配置为 `0x100000`（1MB）。根据硬件设计，该地址空间应指向 QDSP PUB 地址空间，正确的长度应为 `0x4040`。原有的 `0x100000` 覆盖了整个 CDSP 内存块，远大于实际所需的寄存器空间。

#解决方案
修改设备树源码 `arch/arm64/boot/dts/qcom/sm6115.dtsi`，将 `remoteproc_cdsp` 节点的 `reg` 属性第三个字段（长度）从 `0x100000` 修正为 `0x4040`，以匹配实际的 QDSP PUB 地址空间大小。
