#标题
block: Fix blk_validate_atomic_write_limits() build for arm32

#内核版本
OLK-6.6

#问题现象
在 arm32 架构下编译内核时，`blk_validate_atomic_write_limits` 函数触发编译警告，提示比较不同指针类型时缺少类型转换。

#主要错误日志
```text
/tmp/next/build/block/blk-settings.c: In function 'blk_validate_atomic_write_limits':
/tmp/next/build/include/asm-generic/div64.h:222:35: warning: comparison of distinct pointer types lacks a cast
  222 |         (void)(((typeof((n)) *)0) == ((uint64_t *)0));  \
      |                                   ^~
```

#问题根因
在 `blk_validate_atomic_write_limits` 函数中调用 `do_div` 宏进行除法运算。`do_div` 宏要求被除数必须是 64 位类型，但代码中传入的 `chunk_sectors` 变量为 `unsigned int` 类型（32 位），导致 `asm-generic/div64.h` 中的类型检查逻辑报错。

#解决方案
移除 `chunk_sectors` 临时变量，直接使用取模运算符 `%` 替代 `do_div` 宏进行整除检查。由于涉及的两个变量 `lim->chunk_sectors` 和 `boundary_sectors` 均为 `unsigned int` 类型，可以直接使用 `%` 运算符，从而避免 `do_div` 的 64 位类型约束问题。
