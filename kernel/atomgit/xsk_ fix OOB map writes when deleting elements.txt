#标题
xsk: fix OOB map writes when deleting elements

#内核版本
OLK-6.6

#问题现象
在删除 XSK（AF_XDP socket）map 元素时，若传入的 key 值为负数（即大于等于 0x80000000 的无符号整数），内核可能发生越界（OOB）写入，导致内存破坏或内核崩溃。

#主要错误日志
未知

#问题根因
在 `xsk_map_delete_elem` 函数中，用于存储 map 索引的变量 `k` 被错误地定义为有符号整型 `int`。当用户空间传入一个大于等于 0x80000000 的 key 值时，`k` 将被解释为负数。随后的边界检查 `if (k >= map->max_entries)` 会因为负数小于 `max_entries` 而判断通过，导致后续操作访问非法内存地址，引发越界写入。

#解决方案
将变量 `k` 的数据类型从 `int` 修改为 `u32`，确保其作为无符号数进行边界比较，从而正确拦截超出 `max_entries` 范围的索引值，防止越界访问。
