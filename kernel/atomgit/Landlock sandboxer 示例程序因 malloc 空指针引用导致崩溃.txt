#标题
Landlock sandboxer 示例程序因 malloc 空指针引用导致崩溃

#内核版本
OLK-6.6

#问题现象
`samples/landlock/sandboxer.c` 示例程序在运行过程中，若系统内存不足，`parse_path` 函数调用 `malloc` 失败返回 NULL，随后程序访问该空指针导致 NULL dereference（空指针引用），进而引发程序崩溃。

#主要错误日志
Failed to allocate memory

#问题根因
`parse_path` 函数内调用 `malloc()` 分配内存时，未对返回值进行 NULL 检查。当内存分配失败返回 NULL 时，后续代码直接对该指针进行解引用操作，导致 NULL dereference。

#解决方案
1. 在 `parse_path` 函数中增加 `malloc` 返回值的 NULL 检查，若分配失败则返回 -1。
2. 在调用者 `populate_ruleset` 函数中增加对 `parse_path` 返回值的错误检查，若返回值小于 0，打印错误信息 "Failed to allocate memory" 并跳转至资源释放流程（`goto out_free_name`），确保程序安全退出而非崩溃。
