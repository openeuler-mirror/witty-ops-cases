#标题
x86/cpu/hygon: Hygon 0x4-0x8 型号处理器 die_id 与 logical_die_id 解析错误

#内核版本
OLK-6.6

#问题现象
在 Hygon 0x4 至 0x8 型号的处理器上，内核未能正确从 CPUID leaf 0x8000001e ecx 的 NodeId 字段获取 die_id，导致 CPU 拓扑信息（die_id 和 logical_die_id）解析错误。这可能导致后续依赖拓扑结构的内核子系统（如调度、NUMA 策略）行为异常或功能失效。

#主要错误日志
未知

#问题根因
Hygon 0x4-0x8 型号处理器的 die ID 应当从 CPUID leaf 0x8000001e ecx 的 NodeId 字段获取，但现有代码逻辑未针对这些特定型号进行正确处理，沿用了错误的拓扑获取方式。

#解决方案
在内核初始化 Hygon CPU 时，增加针对 model 0x4 至 0x8 的拓扑修复函数 `cpu_topology_fixup_hygon`。在该函数中，通过 `cpuid_ecx(0x8000001e) & 0xff` 正确获取 die_id，并据此重新计算 logical_die_id，覆盖默认的错误值。
