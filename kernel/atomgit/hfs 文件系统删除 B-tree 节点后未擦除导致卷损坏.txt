#标题
hfs 文件系统删除 B-tree 节点后未擦除导致卷损坏

#内核版本
OLK-6.6

#问题现象
运行 xfstests 套件中的 generic/001 测试用例失败，导致 HFS 卷损坏。使用 fsck.hfs 工具检查文件系统时报错：“Unused node is not erased”，并提示“The volume untitled was found corrupt and needs to be repaired”。

#主要错误日志
```
_check_generic_filesystem: filesystem on /dev/loop50 is inconsistent
** Checking catalog file.
   Unused node is not erased (node = 2)
   Unused node is not erased (node = 4)
<skipped>
   Unused node is not erased (node = 253)
   Unused node is not erased (node = 254)
   Unused node is not erased (node = 255)
   Unused node is not erased (node = 256)
** The volume untitled was found corrupt and needs to be repaired.
```

#问题根因
内核在处理 HFS 文件系统 B-tree 节点删除时逻辑缺失。当 B-tree 节点被标记为删除（HFS_BNODE_DELETED）并从哈希表中移除后，代码直接调用 hfs_bmap_free 释放空间，但遗漏了对该节点数据区域的擦除操作。这导致磁盘上保留了未被清零的“脏”数据，文件系统一致性检查工具（fsck）检测到这些未被擦除的未使用节点，从而判定文件系统损坏。

#解决方案
在 fs/hfs/bnode.c 文件的 hfs_bnode_put 函数中，针对已标记为删除的节点，在解除哈希锁定后、释放位图空间前，增加调用 hfs_bnode_clear(node, 0, tree->node_size)。该操作将已删除节点的数据内容清零，确保磁盘上对应区域被正确擦除，从而修复文件系统不一致问题。
