#标题
net: stmmac: fix TSO DMA API usage causing oops

#内核版本
OLK-6.6

#问题现象
在使用 stmmac 驱动进行 TSO（TCP Segmentation Offload）传输时，因 DMA 地址计算错误引发 Oops，导致内核崩溃。

#主要错误日志
未知

#问题根因
在 `stmmac_tso_xmit` 函数中，变量 `des` 被错误地用作后续 DMA 映射操作的基地址。代码逻辑中对 `des` 进行了自增操作（`des += proto_hdr_len`），导致该变量保存的不再是原始的 DMA 映射地址。随后将该修改后的地址传递给 `stmmac_tso_allocator`，违反了 DMA API 的使用规范，从而触发了 Oops。

#解决方案
引入新的局部变量 `tso_des` 用于存储中间计算值。在需要调整 DMA 地址偏移量的场景下，使用 `tso_des` 进行计算（如 `tso_des = des + proto_hdr_len`），确保原始 DMA 映射地址 `des` 不被破坏，并将正确的地址传递给 `stmmac_tso_allocator` 函数。
