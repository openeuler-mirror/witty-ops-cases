#标题
amphion VPU 驱动probe失败路径中pm_runtime_set_suspended调用返回错误

#内核版本
OLK-6.6

#问题现象
在 amphion VPU 驱动初始化（probe）过程中，若发生错误并进入错误处理分支，调用 `pm_runtime_set_suspended` 函数时会返回 `-EAGAIN` 错误，导致设备的运行时电源管理状态设置无效。

#主要错误日志
未知

#问题根因
根据内核运行时电源管理（Runtime PM）机制，对于已经使能了 Runtime PM 的设备，调用 `pm_runtime_set_suspended()` 是无效的，该函数会直接返回 `-EAGAIN`。
驱动在 `vpu_probe` 的错误处理路径中，先调用了 `pm_runtime_set_suspended(dev)`，随后才调用 `pm_runtime_disable(dev)`。由于此时设备的 Runtime PM 处于使能状态，导致 `pm_runtime_set_suspended` 调用失败。

#解决方案
调整错误处理路径中函数调用的顺序，先调用 `pm_runtime_disable(dev)` 禁用 Runtime PM，随后再调用 `pm_runtime_set_suspended(dev)` 设置设备状态，从而确保状态设置操作成功执行。
