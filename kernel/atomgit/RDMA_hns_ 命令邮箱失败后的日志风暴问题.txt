#标题
RDMA/hns: 命令邮箱失败后的日志风暴问题

#内核版本
OLK-6.6

#问题现象
在 hns RDMA 驱动中，当硬件命令邮箱（cmd_mbox）操作失败时，驱动会打印错误日志。如果在短时间内大量触发此类失败（如 CQ 创建、SRQ 修改/查询、MPT 操作等），由于错误打印未受速率限制，会导致内核日志（dmesg）急剧刷屏，造成日志洪泛，影响系统性能及关键日志的查看。

#主要错误日志
- failed to send create cmd for CQ(0x%lx), ret = %d.
- failed to handle cmd of modifying SRQ, ret = %d.
- failed to process cmd of querying SRQ, ret = %d.
- failed to process cmd when querying MPT, ret = %d.
- failed to destroy MPT, ret = %d.
- failed to create MPT, ret = %d.
- failed to config SRQC, ret = %d.

#问题根因
驱动在处理硬件命令邮箱失败的场景时，使用了 `ibdev_err` 或 `ibdev_warn` 进行日志打印。这些函数不带速率限制机制。当底层硬件或固件出现故障导致命令持续失败时，内核日志会被同一错误信息迅速填满，导致日志风暴。

#解决方案
将涉及命令邮箱失败处理路径中的 `ibdev_err` 和 `ibdev_warn` 替换为对应的速率限制版本 `ibdev_err_ratelimited` 和 `ibdev_warn_ratelimited`。这样既保留了故障定位所需的错误信息，又防止了高频错误导致的日志洪泛。
