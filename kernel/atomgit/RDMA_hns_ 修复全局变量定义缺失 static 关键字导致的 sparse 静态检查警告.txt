#标题
RDMA/hns: 修复全局变量定义缺失 static 关键字导致的 sparse 静态检查警告

#内核版本
OLK-6.6

#问题现象
在使用 sparse 工具对内核代码进行静态代码检查时，报告 "Should it be static?" 警告。该警告表明某些变量具有文件作用域但未声明为 static，可能污染全局命名空间。

#主要错误日志
warning: symbol 'hns_roce_port_attr_dcqcn_ai' was not declared. Should it be static?
warning: symbol 'hns_roce_port_attr_dcqcn_rpg_time_reset' was not declared. Should it be static?
warning: symbol 'hns_roce_port_attr_ldcp_cwd0' was not declared. Should it be static?
warning: symbol 'hns_roce_port_attr_hc3_ai' was not declared. Should it be static?
warning: symbol 'hns_roce_port_attr_dip_ai' was not declared. Should it be static?
（注：以上为根据 Commit Message 描述的 sparse 警告类型还原的典型输出）

#问题根因
在宏定义 `HNS_PORT_DCQCN_CC_ATTR_RW`、`HNS_PORT_LDCP_CC_ATTR_RW`、`HNS_PORT_HC3_CC_ATTR_RW` 和 `HNS_PORT_DIP_CC_ATTR_RW` 中，通过宏展开定义的结构体变量（如 `hns_roce_port_attr_dcqcn_ai` 等）仅使用了 `struct hns_port_cc_attr` 声明，遗漏了 `static` 关键字。这导致这些本应仅在本文件内可见的变量变成了全局变量，触发了 sparse 的静态检查警告。

#解决方案
在相关宏定义中，为结构体变量声明添加 `static` 关键字，将变量作用域限制在当前文件内，消除 sparse 警告并避免命名空间污染。
