#标题
irqchip/gic-v3: 修复 xint 操作中在无效上下文调用睡眠函数的问题

#内核版本
OLK-6.6

#问题现象
在执行 xint 相关操作（通过 debugfs/proc 写入触发）时，内核检测到在原子上下文（中断禁用）中调用了可能睡眠的函数，导致内核抛出 "scheduling while atomic" 或 "BUG: sleeping function called from invalid context" 等错误信息，甚至引发系统恐慌。

#主要错误日志
BUG: sleeping function called from invalid context

#问题根因
在 `xint_proc_write` 函数中，代码逻辑先通过 `local_irq_disable()` 禁用了本地中断，随后调用 `xint_transform`。而在 `xint_transform` 内部（根据补丁前代码逻辑），会根据操作类型调用 `xint_add_debugfs_entry` 或 `xint_remove_debugfs_entry`。这些 debugfs 相关函数可能包含内存分配等可能睡眠的操作。在中断禁用的原子上下文中执行睡眠操作违反了内核规则，从而导致故障。

#解决方案
将可能睡眠的 `xint_add_debugfs_entry` 和 `xint_remove_debugfs_entry` 操作移到 `local_irq_disable()` 之前执行。具体修改如下：
1. 在写入操作判断 xint 状态后、禁用中断前，预先根据状态调用 debugfs 的 add 或 remove 接口。
2. 调整 `disable_irq` 和 `enable_irq` 的调用位置，确保原子上下文仅包含严格必要的硬件操作（如 `xint_transform` 和位操作），避免在临界区内调用可能睡眠的函数。
