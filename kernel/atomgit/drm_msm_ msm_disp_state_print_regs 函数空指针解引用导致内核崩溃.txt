#标题
drm/msm: msm_disp_state_print_regs 函数空指针解引用导致内核崩溃

#内核版本
OLK-6.6

#问题现象
当 msm_disp_state_dump_regs() 中的内存分配失败时，`block->state` 为 NULL。随后在调用 msm_disp_state_print_regs() 时，由于原有逻辑无法正确处理该 NULL 指针，导致对 `dump_addr` 进行解引用操作，引发空指针解引用，造成内核崩溃或 Oops。

#主要错误日志
推断：BUG: kernel NULL pointer dereference

#问题根因
在 msm_disp_state_print_regs() 函数中，变量 `dump_addr` 被初始化为 NULL。虽然代码中包含 `if (*reg) dump_addr = *reg;` 的判断逻辑，但在 `*reg`（即 `block->state`）为 NULL 时，该判断条件不成立，导致 `dump_addr` 依然保持 NULL。随后代码直接对 `dump_addr` 进行解引用，未对 NULL 情况做有效防护，从而导致空指针解引用。

#解决方案
修改 msm_disp_state_print_regs() 函数逻辑：
1. 在函数入口处直接判断传入的指针是否为 NULL。
2. 若指针为 NULL，则打印 "Registers not stored" 提示信息并直接返回，避免后续解引用操作。
3. 同时优化函数参数，不再传递双指针（u32 **reg），改为传递 const 单指针（const u32 *dump_addr），使代码逻辑更清晰。
