#标题
vhost: vhost_add_used_n() 未处理 __vhost_add_used() 失败导致 used ring 索引更新异常

#内核版本
OLK-6.6

#问题现象
在 vhost 子系统中，当 __vhost_add_used() 函数执行失败时，vhost_add_used_n() 函数未及时终止流程。这导致驱动继续执行后续逻辑，使用陈旧的（stale）used ring 信息更新 used idx，可能引发虚拟设备通信异常或数据不一致问题。

#主要错误日志
未知

#问题根因
在函数 vhost_add_used_n() 中，调用 __vhost_add_used_n() 的返回值未被检查。当底层 __vhost_add_used() 失败返回错误时，上层函数忽略了该错误，继续执行更新索引的操作，导致 used idx 被错误地更新。

#解决方案
在 vhost_add_used_n() 函数中增加对 __vhost_add_used_n() 返回值的判断。如果返回值小于 0（即调用失败），则立即返回错误，防止在写入失败的情况下继续更新 used ring 索引。
