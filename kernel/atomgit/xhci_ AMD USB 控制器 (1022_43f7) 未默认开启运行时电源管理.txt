#标题
xhci: AMD USB 控制器 (1022:43f7) 未默认开启运行时电源管理

#内核版本
OLK-6.6

#问题现象
AMD 1022:43f7 型号的 xHCI 1.1 主机控制器未默认启用 Runtime PM（运行时电源管理），导致该设备无法像更新的 xHCI 1.2 主机那样自动进入低功耗状态，影响系统整体功耗表现。

#主要错误日志
未知

#问题根因
驱动程序仅在较新的 AMD xHCI 1.2 主机上默认启用了 Runtime PM，而忽略了较旧的 AMD 1022:43f7 xHCI 1.1 主机。实际上该型号硬件已证明支持 Runtime PM 功能，但驱动未将其加入默认启用列表。

#解决方案
在 `xhci_pci_quirks` 函数中增加对 AMD 设备 ID 0x43f7 的判断，为其设置 `XHCI_DEFAULT_PM_RUNTIME_ALLOW` quirks 标志，从而允许该设备默认开启运行时电源管理。
