#标题
ubcore: query route list by EID pair fails due to incomplete implementation

#内核版本
OLK-6.6

#问题现象
通过 EID 对查询路由列表功能失败，系统无法正确获取路由信息。该故障表现为 `UBCORE_CMD_GET_ROUTE_LIST` 命令处理缺失，导致相关业务流程无法正常执行。

#主要错误日志
- "Failed to parse ubcore cmd tlv.\n"
- "Failed to get_route_list, ret: %d.\n"
- "Failed to get topo port eid, src_v_eid: " EID_FMT ", dst_v_eid: " EID_FMT ".\n"
- "Invalid parameter.\n"
- "Invalid tp tpye: %u.\n"

#问题根因
内核中缺少对 `UBCORE_CMD_GET_ROUTE_LIST` 命令的处理逻辑：
1. 全局命令处理数组 `g_ubcore_uvs_global_cmd_funcs` 中未注册 `UBCORE_CMD_GET_ROUTE_LIST` 的处理函数。
2. TLV 处理器数组 `g_global_tlv_handler` 中缺少对应的 spec 填充回调函数。
3. 原有的 `ubcore_get_topo_eid` 接口逻辑不完善，仅支持单个 EID 对的查询，无法满足基于 EID 对查询路由列表的需求，且未覆盖多 IO Die 场景的遍历查找。

#解决方案
1. 新增 `ubcore_cmd_get_route_list` 函数，实现对 `UBCORE_CMD_GET_ROUTE_LIST` 命令的解析与路由列表获取逻辑。
2. 在 `g_ubcore_uvs_global_cmd_funcs` 数组中注册 `UBCORE_CMD_GET_ROUTE_LIST` 命令及其处理函数。
3. 在 `g_global_tlv_handler` 中补充 `UBCORE_CMD_GET_ROUTE_LIST` 的 TLV 处理配置，包括输入输出的 spec 填充。
4. 重构路由查询逻辑，新增 `ubcore_get_route_list` 接口替代原有的 `ubcore_get_topo_eid`。新接口支持遍历所有 IO Die (`IODIE_NUM`) 和端口，能够查找并返回完整的路由列表（`ubcore_route_list`），同时支持 RTP、UTP、CTP 等多种路由类型。
