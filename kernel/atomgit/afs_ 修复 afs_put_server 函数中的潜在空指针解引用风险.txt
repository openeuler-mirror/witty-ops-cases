#标题
afs: 修复 afs_put_server 函数中的潜在空指针解引用风险

#内核版本
OLK-6.6

#问题现象
在调用 afs_put_server 函数时，若传入的 server 指针为 NULL，内核在执行 NULL 检查之前会先访问 server->debug_id 成员，导致发生空指针解引用，进而引发内核 panic 或 Oops。

#主要错误日志
未知

#问题根因
afs_put_server() 函数在定义局部变量时，直接访问 server->debug_id 进行赋值（`unsigned int a, debug_id = server->debug_id;`），该操作发生在 `if (!server) return;` 的 NULL 指针检查之前。当 server 指针为 NULL 时，解引用操作先于防御性检查执行，从而导致空指针解引用故障。

#解决方案
将 debug_id 变量的声明与赋值分离，把 `debug_id = server->debug_id;` 赋值语句移动到 NULL 指针检查之后执行，确保只有在 server 指针非空时才访问其成员变量。
