#标题
ASoC: qcom: audioreach 修复潜在的空指针解引用漏洞

#内核版本
OLK-6.6

#问题现象
在加载音频拓扑时，若 `audioreach_parse_common_tokens` 函数返回 NULL 指针，内核后续代码可能会对该指针进行解引用操作，从而导致内核空指针解引用故障，可能引发内核 panic 或 Oops。

#主要错误日志
未知

#问题根因
拓扑解析函数 `audioreach_widget_load_module_common()` 调用 `audioreach_parse_common_tokens()` 时，该函数可能返回 NULL 或错误指针。原代码仅使用 `IS_ERR()` 检查了错误指针情况，遗漏了对 NULL 指针的检查。如果返回 NULL，代码继续执行会导致对 NULL 指针的非法解引用。

#解决方案
修改 `sound/soc/qcom/qdsp6/topology.c` 中的检查逻辑，将 `IS_ERR(mod)` 修改为 `IS_ERR_OR_NULL(mod)`。当检测到返回值为 NULL 时，直接返回 `-ENODEV` 错误码；若为错误指针则返回 `PTR_ERR(mod)`，从而避免对 NULL 指针的解引用。
