#标题
sched: Fix sched_numa_find_nth_cpu() if mask offline

#内核版本
OLK-6.6

#问题现象
当 CPU 处于离线状态时，`sched_numa_find_nth_cpu()` 函数在处理 mask 时可能发生内核空指针引用或访问非法内存，导致内核 panic 或 Oops。

#主要错误日志
未知

#问题根因
在 `sched_numa_find_nth_cpu()` 函数中，调用 `bsearch()` 查找 hop_masks 时，若 mask 对应的 CPU 处于离线状态，`bsearch()` 可能返回 NULL。代码未对返回值进行非空检查直接使用，导致后续指针解引用引发异常。

#解决方案
在 `kernel/sched/topology.c` 的 `sched_numa_find_nth_cpu()` 函数中增加对 `bsearch()` 返回值的判断。若返回值为 NULL，直接跳转至 unlock 流程返回，避免访问空指针。
