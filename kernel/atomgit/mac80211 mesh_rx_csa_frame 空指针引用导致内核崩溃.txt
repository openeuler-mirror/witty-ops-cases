#标题
mac80211 mesh_rx_csa_frame 空指针引用导致内核崩溃

#内核版本
OLK-6.6

#问题现象
在 Mesh 网络环境中，当已建立对等链接的远程 Mesh 节点发送一个特制的 CSA（Channel Switch Announcement）动作帧时，若该帧包含匹配的 Mesh ID 和 Mesh Configuration IE，但省略了 Mesh Channel Switch Parameters IE（元素 ID 118），接收端内核在处理该帧时会触发空指针引用，导致系统崩溃。

#主要错误日志
BUG: kernel NULL pointer dereference, address: 0000000000000000
Oops: Oops: 0000 [#1] SMP NOPTI
RIP: 0010:ieee80211_mesh_rx_queued_mgmt+0x143/0x2a0 [mac80211]
CR2: 0000000000000000

#问题根因
在函数 `mesh_rx_csa_frame()` 中，代码逻辑直接解引用了 `elems->mesh_chansw_params_ie` 指针以获取 `mesh_ttl` 和 `mesh_pre_value`，但在解引用前未检查该指针是否为空。
函数 `mesh_matches_local()` 仅校验 Mesh ID、Mesh Configuration 和 Supported Rates IE，并不验证 Mesh Channel Switch Parameters IE 是否存在。当接收到的 CSA 帧省略该 IE 时，`ieee802_11_parse_elems()` 解析后该指针保持为 NULL，随后的无条件解引用触发了内核空指针引用错误。

#解决方案
在 `mesh_rx_csa_frame()` 函数中，于 `mesh_matches_local()` 校验通过后、使用 `elems->mesh_chansw_params_ie` 成员前，增加对该指针的非空检查。若指针为 NULL，则直接跳转至释放资源流程，避免非法访问。
