#标题
kconfig 加载默认配置时警告信息显示错误的文件名

#内核版本
OLK-6.6

#问题现象
在执行 `make *config` 命令且当前目录下不存在 `.config` 文件时，Kconfig 会尝试加载 `KCONFIG_DEFCONFIG_LIST` 列表中的默认配置文件（如 `/boot/config-xxx`）。若该默认配置文件中包含无效的符号值，内核发出的警告信息会错误地显示文件名为 `.config`，而不是实际加载的默认配置文件路径（如 `/boot/config-6.12.9-amd64`），导致用户定位配置问题来源时受到误导。

#主要错误日志
.config:6804:warning: symbol value 'm' invalid for FB_BACKLIGHT
.config:9895:warning: symbol value 'm' invalid for ANDROID_BINDER_IPC

#问题根因
自提交 b75b0a819af9 ("kconfig: change defconfig_list option to environment variable") 引入变更后，代码逻辑在处理 `KCONFIG_DEFCONFIG_LIST` 环境变量指定的文件时，打开了正确的文件流（`in = zconf_fopen(env)`），但未将全局变量 `name` 更新为实际的文件名（`env`）。因此，后续调用 `conf_message` 打印警告时，`name` 指针仍指向初始的 `.config` 字符串，导致日志输出了错误的文件名。

#解决方案
修改 `scripts/kconfig/confdata.c` 文件，在成功打开 `KCONFIG_DEFCONFIG_LIST` 指定的文件前，将全局变量 `name` 赋值为实际打开的文件路径（`env`），确保后续 `conf_message` 调用能够打印正确的文件名。
