#标题
usb: gadget: midi2: Fix interpretation of is_midi1 bits

#内核版本
OLK-6.6

#问题现象
当 USB MIDI2 gadget 驱动的 UMP Function Block info 的 m1.0 字段（is_midi1 sysfs 条目）值为 2 时，驱动错误地将其作为位标志直接拷贝至 snd_ump_block_info.flags，导致设置了错误的比特位，致使UMP块信息标志位状态异常。

#主要错误日志
未知

#问题根因
UMP Function Block info 的 m1.0 字段（is_midi1）是一个取值范围为 0 到 2 的枚举值，而驱动代码在 f_midi2_create_card 函数中错误地将其直接作为比特位通过按位或操作（|=）赋值给 flags 字段。当 is_midi1 值为 2 时，错误的逻辑导致了不正确的位标志被设置。

#解决方案
修改 drivers/usb/gadget/function/f_midi2.c 中的逻辑，根据 is_midi1 的枚举值正确设置 flags：
- 如果 is_midi1 小于 2，则直接将其值赋给 flags；
- 如果 is_midi1 等于 2，则设置 SNDRV_UMP_BLOCK_IS_MIDI1 和 SNDRV_UMP_BLOCK_IS_LOWSPEED 标志位。
