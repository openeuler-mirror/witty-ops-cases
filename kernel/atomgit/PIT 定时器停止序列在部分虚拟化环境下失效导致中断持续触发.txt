#标题
PIT 定时器停止序列在部分虚拟化环境下失效导致中断持续触发

#内核版本
OLK-6.6

#问题现象
在特定有缺陷的虚拟机（如 KVM、QEMU、Hyper-V）环境中，尝试关闭 PIT（Programmable Interval Timer）定时器 0 时，定时器未能正确停止。
具体表现为：
1. 写入 MODE 寄存器后，计数器未能立即停止，中断持续触发。
2. 在 Hyper-V 环境下，即使处于 MODE 0，中断仍然反复触发（本应仅触发一次），导致系统出现额外的中断开销或 steal time 增加。

#主要错误日志
未知

#问题根因
根据硬件数据手册，写入 MODE 寄存器应当停止计数器（从而停止中断），这在物理硬件上工作正常。
但在部分有缺陷的虚拟机实现中，MODE 寄存器的变更直到计数器被随后加载（如 KVM 内核 PIT）或下一次中断触发（如 QEMU）时才生效。此外，Hyper-V 即使在 MODE 0 下如果计数器仍在运行也会反复触发 IRQ。
原有代码逻辑仅依赖写入 MODE 寄存器或在特定条件下清零计数器，未覆盖这些虚拟化场景下的时序缺陷。

#解决方案
修改 `clockevent_i8253_disable` 函数中的停止序列，确保兼容有缺陷的虚拟机实现：
1. 首先写入 MODE 0（0x30）到 PIT_MODE 寄存器。
2. 随后立即向 PIT_CH0 通道写入计数器值（0），强制那些需要计数器加载才能生效的虚拟机实现停止中断。
3. 最后再次写入 MODE 0（0x30），确保符合标准规范的实现也能正确停止计数器。
此序列消除了对 `i8253_clear_counter_on_shutdown` 全局变量的依赖，统一了处理逻辑。
