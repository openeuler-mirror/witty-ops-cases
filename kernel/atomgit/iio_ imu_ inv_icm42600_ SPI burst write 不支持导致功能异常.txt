#标题
iio: imu: inv_icm42600: SPI burst write 不支持导致功能异常

#内核版本
OLK-6.6

#问题现象
ICM42600 系列 IMU 芯片在 SPI 接口通信时，批量写操作无法正常工作。该问题具体表现为通过 regmap_bulk_write 设置用户偏移量时功能失效或数据写入错误。

#主要错误日志
未知

#问题根因
ICM42600 芯片的 SPI 接口不支持突发写操作，但驱动程序此前使用了通用的 regmap 配置，该配置未禁用 burst write 功能，导致在 SPI 总线上执行批量写入时硬件无法正确处理。

#解决方案
为 SPI 接口定义专用的 regmap 配置结构体 `inv_icm42600_spi_regmap_config`，在其中强制设置 `use_single_write = true`，确保所有写操作均采用单次写模式，从而规避硬件对 burst write 的限制。同时在 SPI 驱动探测函数中将 `devm_regmap_init_spi` 的配置参数由通用配置替换为该 SPI 专用配置。
