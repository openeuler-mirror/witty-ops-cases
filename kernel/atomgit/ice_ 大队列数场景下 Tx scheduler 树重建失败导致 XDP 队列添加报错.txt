#标题
ice: 大队列数场景下 Tx scheduler 树重建失败导致 XDP 队列添加报错

#内核版本
OLK-6.6

#问题现象
在 CPU 数量较多（例如至少 321 个 CPU，且已存在 128 个 Tx/Rx 队列对）的机器上添加 XDP Tx 队列时，驱动虽然遵循固件返回的队列限制，但 Tx 调度器无法将队列添加到其树结构中，导致配置失败。

#主要错误日志
- Failed VSI LAN queue config for XDP, error: -5
- Failed to set LAN Tx queue context, error: -22

#问题根因
当前的 Tx 调度器实现在用户向 VSI 添加更多 Tx 队列时允许重建树，通过增加子节点来支持新的队列数。然而，该算法未考虑 VSI 支持节点的限制可能被超过的情况。在队列数量巨大时，单个 VSI 节点无法处理所有请求的队列，需要额外的 VSI 层节点来分担，但原逻辑未创建这些额外节点，导致树重建失败并返回错误。

#解决方案
扩展树重建算法以检查当前 VSI 节点是否能支持请求的队列数量。如果不能，则创建必要数量的额外 VSI 支持节点来处理所有所需的 Tx 队列。同时，调整 VSI 节点移除算法，以移除与给定 VSI 相关的所有节点；使对下一个空闲 VSI 节点的搜索更加严格，仅将队列组节点添加到具有匹配 VSI 句柄的 VSI 支持节点中。
