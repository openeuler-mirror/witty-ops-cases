#标题
usb: xhci: Prevent interrupt storm on host controller error (HCE)

#内核版本
OLK-6.6

#问题现象
当 XHCI 主机控制器发生硬件错误时，系统产生中断风暴，导致 CPU 负载过高，严重影响系统性能和响应能力。

#主要错误日志
xhci_warn(xhci, "WARNING: Host Controller Error\n");

#问题根因
在 `xhci_irq` 处理函数中，当检测到主机控制器错误（`STS_HCE`）状态位时，代码仅打印了警告日志 `xhci_warn`，但并未停止控制器。这导致控制器持续处于错误状态并不断产生中断，由于中断处理逻辑未清除中断源，从而引发中断风暴。

#解决方案
在检测到 `STS_HCE` 错误时，立即调用 `xhci_halt(xhci)` 函数停止主机控制器。通过 halt 控制器，可以阻断中断的持续产生，从而防止中断风暴的发生。
