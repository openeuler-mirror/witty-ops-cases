#标题
gpio: grgpio: Add NULL check in grgpio_probe

#内核版本
OLK-6.6

#问题现象
在 grgpio 驱动探测过程中，若内存分配失败，`devm_kasprintf` 返回 NULL，导致后续代码访问空指针，引发内核 panic 或 oops。

#主要错误日志
未知

#问题根因
在 `grgpio_probe` 函数中，调用 `devm_kasprintf` 给 `gc->label` 赋值时，未对返回值进行 NULL 检查。当系统内存不足导致 `devm_kasprintf` 分配失败返回 NULL 时，后续代码未做防护直接使用了该指针，从而触发空指针访问异常。

#解决方案
在 `grgpio_probe` 函数中，对 `devm_kasprintf` 的返回值进行判断，如果返回 NULL，则直接返回 `-ENOMEM` 错误码，终止探测流程，避免空指针引用。
