#标题
cpupower set 子命令 -t 选项未生效

#内核版本
OLK-6.6

#问题现象
cpupower 工具的 set 子命令中，用于 boost 配置的 -t 选项虽然在文档中有说明，但实际无法正常工作。用户执行带有 -t 参数的命令时，选项未被正确解析，导致相关配置无法生效。

#主要错误日志
未知

#问题根因
在 `tools/power/cpupower/utils/cpupower-set.c` 文件的 `cmd_set` 函数中，调用 `getopt_long` 进行命令行参数解析时，传入的短选项字符串参数为 "b:e:m:"，遗漏了 "t:"。这导致程序无法识别 -t 选项，从而使得该功能失效。

#解决方案
修复 `getopt_long` 函数调用的参数，在短选项字符串 "b:e:m:" 中补充缺失的 "t:"，修改为 "b:e:m:t:"，使程序能够正确解析 -t 选项。
