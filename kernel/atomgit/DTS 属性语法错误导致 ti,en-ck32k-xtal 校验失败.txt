#标题
DTS 属性语法错误导致 ti,en-ck32k-xtal 校验失败

#内核版本
OLK-6.6

#问题现象
在使用 dtbs_check 对设备树文件进行校验时出现验证错误，提示 "size (4) error for type flag"。

#主要错误日志
size (4) error for type flag

#问题根因
在设备树源文件中，`ti,en-ck32k-xtal` 属性在 Schema 中定义为布尔类型，但代码中错误地为其赋值为 `<1>`。由于驱动程序通过 `of_property_read_bool()` 读取该属性期望其为纯布尔存在形式，赋值操作导致类型不匹配从而引发校验失败。

#解决方案
修改 `arch/arm/boot/dts/ti/omap/am335x-baltos.dtsi` 文件，将 `ti,en-ck32k-xtal = <1>;` 修改为 `ti,en-ck32k-xtal;`，移除错误的赋值，使其符合布尔属性的标准语法。
