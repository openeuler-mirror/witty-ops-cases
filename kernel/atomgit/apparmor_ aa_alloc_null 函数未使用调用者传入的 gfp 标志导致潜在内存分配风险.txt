#标题
apparmor: aa_alloc_null 函数未使用调用者传入的 gfp 标志导致潜在内存分配风险

#内核版本
OLK-6.6

#问题现象
在调用 `aa_alloc_null` 函数进行内存分配时，函数内部硬编码使用了 `GFP_KERNEL` 标志，而忽略了调用者传入的 `gfp` 标志。虽然当前所有调用者均传入 `GFP_KERNEL` 尚未引发运行时故障，但这违反了内核内存分配 API 的设计规范，可能在特定上下文（如持有锁或原子上下文）调用该函数时引发睡眠或死锁风险。

#主要错误日志
未知

#问题根因
`aa_alloc_null` 函数在分配 `rules->file.perms` 和 `rules->policy.perms` 时，直接硬编码使用了 `GFP_KERNEL` 参数，未使用函数参数列表中传入的 `gfp` 标志变量，导致内存分配行为无法根据调用上下文进行调整。

#解决方案
修改 `security/apparmor/policy.c` 中的 `aa_alloc_null` 函数，将两处 `kcalloc` 调用的标志参数从 `GFP_KERNEL` 修改为传入的 `gfp` 变量，确保内存分配行为与调用者的意图一致。
