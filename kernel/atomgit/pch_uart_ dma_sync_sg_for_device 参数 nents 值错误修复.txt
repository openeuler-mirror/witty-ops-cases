#标题
pch_uart: dma_sync_sg_for_device 参数 nents 值错误修复

#内核版本
OLK-6.6

#问题现象
pch_uart 驱动在进行 DMA 发送操作时，调用 `dma_sync_sg_for_device` 函数同步数据时传入了错误的 nents 参数值（使用了 `dma_map_sg` 的返回值而非原始传入值）。这违反了 DMA API 的使用规范，可能导致 DMA 内存同步不一致，进而引发数据传输错误或内存损坏。

#主要错误日志
未知

#问题根因
根据 Linux 内核文档 `Documentation/core-api/dma-api.rst` 要求，`dma_sync_sg_for_device()` 函数的参数必须与 `dma_map_sg()` 传入的参数完全一致。代码中错误地将 `dma_map_sg()` 的返回值 `nent`（表示映射后的条目数）传递给了 `dma_sync_sg_for_device()`，而实际上应该传递原始的 scatter-gather 条目数量 `num`。

#解决方案
修改 `drivers/tty/serial/pch_uart.c` 文件中 `dma_handle_tx` 函数的实现，将 `dma_sync_sg_for_device()` 调用的第二个参数从 `nent` 修正为 `num`，以确保符合 DMA API 规范。
