#标题
Intel QAT 驱动功能缺陷与稳定性问题修复合集

#内核版本
OLK-6.6

#问题现象
Intel QAT 驱动在特定场景下存在多个功能与稳定性问题，具体包括：
1. 系统关机或重启时，QAT 设备未正确执行清理流程，可能导致设备状态异常或关机流程卡顿。
2. SR-IOV 虚拟化场景下，VF（虚拟功能）恢复与重启流程存在竞态条件，导致故障恢复失败。
3. 驱动加载或编译时，对禁用 PCI_IOV 配置的情况处理不当，导致链接错误。
4. 驱动内部对设备服务类型的判断逻辑与固件配置不一致，可能导致设备初始化失败或功能不可用。
5. vfio 相关接口写入时存在整数溢出检查缺失风险。
6. 电源管理状态下 AER（高级错误处理）初始化失败时未正确销毁工作队列，导致资源泄漏。
7. 访问 GEN4 设备不存在的奇偶校验寄存器，可能触发硬件错误或日志异常。

#主要错误日志
- 推断：驱动加载或卸载时的调试日志显示服务类型判断错误：`is_esp: yes/no`（原实现逻辑不清晰）。
- 推断：AER 初始化失败时的内核日志：`Failed to allocate workqueue` 或在后续操作中访问已释放资源。
- 推断：VF 恢复过程中的错误状态日志。

#问题根因
该补丁集合了上游 v6.10 至 v6.16 的 43 个修复，主要根因如下：
1. **shutdown 回调缺失**：QAT 驱动（c3xxx, c62x, dh895xcc, 4xxx, 420xx 等）未注册 `shutdown` 回调，导致系统关机时驱动未执行 `adf_dev_down` 进行资源释放。
2. **SR-IOV 流程缺陷**：VF 重启处理程序中顺序错误及恢复流程逻辑错误，未在 `adf_dev_stop` 中正确禁用 IOV，导致 VF 状态不一致。
3. **PCI ID 与服务枚举不标准**：使用了非标准的 PCI ID 宏定义，且服务类型枚举（如 `SVC_CY`、`SVC_DC_ASYM`）与固件实际配置不匹配，导致硬件能力识别错误。
4. **资源管理泄漏**：`adf_init_aer` 函数在错误路径下未销毁已创建的工作队列；`adf_ctl_class` 未设置为常量，存在潜在安全隐患。
5. **硬件访问越界**：尝试访问 GEN4 设备不支持的 parity register，以及部分数组访问存在 off-by-one 错误。
6. **构建与代码缺陷**：禁用 CONFIG_PCI_IOV 时编译报错；使用已废弃的 PCI 函数；日志字符串格式不规范。

#解决方案
通过反向移植上游修复补丁，主要实施以下修复措施：
1. **添加 shutdown 处理**：为各 QAT 设备驱动注册 `shutdown` 回调函数，确保在系统关机时调用 `adf_dev_down` 正确关闭设备。
2. **修正 SR-IOV 逻辑**：修复 VF 重启处理程序的执行顺序，确保在设备停止阶段禁用 IOV，并修正 VF 的故障恢复流程。
3. **规范化接口与数据结构**：
   - 统一使用标准内核 PCI ID 宏（如 `PCI_DEVICE_ID_INTEL_QAT_4XXX`）。
   - 重构服务解析逻辑，使用标准的 `SVC_SYM_ASYM` 等枚举替代旧定义，确保软硬件配置一致。
   - 引入 `fuse array` 机制替代单一 `fuses` 变量，以支持更复杂的硬件融合配置。
4. **修复资源管理**：
   - 在 `adf_init_aer` 中增加 `destroy_workqueue` 的错误处理路径。
   - 移除不必要的全局初始化（如 `instances = 0`），并将 `adf_ctl_class` 修改为常量。
5. **修复代码缺陷**：
   - 移除对 GEN4 设备 parity register 的访问。
   - 修复 `qat_vf_resume_write` 中的整数溢出检查。
   - 修复 Makefile 目标定义（使用 `-y` 替代 `-objs`）以解决构建问题。
