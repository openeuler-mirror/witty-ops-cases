#标题
idxd: Fix possible invalid memory access after FLR

#内核版本
OLK-6.6

#问题现象
在执行 PCI FLR（Function Level Reset）后，驱动程序后续操作可能访问无效内存，导致内核崩溃或内存越界访问。

#主要错误日志
未知

#问题根因
在 `idxd_reset_done` 函数中，执行 `kfree(idxd->idxd_saved)` 释放内存后，指针 `idxd->idxd_saved` 未被置空。这导致该指针成为悬空指针，后续若再次访问该指针指向的地址，将引发无效内存访问。

#解决方案
在 `kfree(idxd->idxd_saved)` 释放内存后，立即将 `idxd->idxd_saved` 指针置为 NULL，防止悬空指针引发的非法内存访问。
