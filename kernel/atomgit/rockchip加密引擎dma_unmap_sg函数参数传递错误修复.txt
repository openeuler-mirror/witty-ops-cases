#标题
rockchip加密引擎dma_unmap_sg函数参数传递错误修复

#内核版本
OLK-6.6

#问题现象
在使用 Rockchip 加密引擎进行哈希运算时，驱动程序在数据处理完毕后清理 DMA 映射时，向 `dma_unmap_sg()` 函数传递了错误的 nents 参数（传递了映射后返回的条目数而非原始散列表的条目数）。这违反了 DMA API 的使用规范，可能导致 DMA 映射清理不完整或内存管理异常。

#主要错误日志
未知

#问题根因
根据 Linux DMA API 规范，`dma_unmap_sg()` 函数的调用参数必须与 `dma_map_sg()` 传入的参数保持一致，即应传入原始散列表的 nents（条目数），而不是 `dma_map_sg()` 执行后返回的值。代码中 `rk_hash_unprepare` 函数错误地使用了 `rctx->nrsg`（通常存储的是映射后的返回值），导致参数不匹配。

#解决方案
修改 `drivers/crypto/rockchip/rk3288_crypto_ahash.c` 文件中的 `rk_hash_unprepare` 函数，将 `dma_unmap_sg()` 的第二个参数由 `rctx->nrsg` 修正为 `sg_nents(areq->src)`，确保传入的是原始散列表的条目数，符合 DMA API 的规范要求。
