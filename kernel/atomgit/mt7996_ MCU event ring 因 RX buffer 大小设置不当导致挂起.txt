#标题
mt7996: MCU event ring 因 RX buffer 大小设置不当导致挂起

#内核版本
OLK-6.6

#问题现象
当固件通过 MCU event ring 向驱动传递管理帧时，如果数据包大小在 1728 到 2048 字节之间，会导致 event ring 挂起。

#主要错误日志
未知

#问题根因
在 CONNAC3 架构中，MCU event ring 不支持分散收集机制且有 2048 字节的大小限制。驱动原先使用 `MT_RX_BUF_SIZE` 分配 RX buffer，未预留 `struct skb_shared_info` 的空间。当收到 1728 到 2048 字节的包时，驱动尝试使用 scatter-gather 处理，导致 ring 挂起。

#解决方案
为 MT_RXQ_MCU 和 MT_RXQ_MCU_WA 队列定义专用的 buffer 大小宏 `MT7996_RX_MCU_BUF_SIZE`，将其值设置为 2048 字节加上 `skb_shared_info` 结构体经对齐后的大小，以此禁用 scatter-gather 处理并预留足够空间。
