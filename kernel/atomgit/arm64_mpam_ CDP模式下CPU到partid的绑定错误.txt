#标题
arm64/mpam: CDP模式下CPU到partid的绑定错误

#内核版本
OLK-6.6

#问题现象
在开启MPAM CDP（Code and Data Prioritization）模式时，系统内部对代码和数据分区ID（partid）的映射配置发生倒置。这导致CPU在切换任务或配置资源组时，无法正确应用预设的缓存划分策略，进而导致资源隔离失效或系统行为与预期不符。

#主要错误日志
未知

#问题根因
在函数 `resctrl_arch_set_cdp_enabled` 中，当启用 CDP 模式时，代码错误地将 `partid_d`（Data partid）赋值为 `CDP_CODE` 的索引，将 `partid_i`（Instruction partid）赋值为 `CDP_DATA` 的索引。这导致了 Data 和 Code 的 partid 绑定关系颠倒。此外，在 `mpam_thread_switch` 函数中，原代码使用 `regval == READ_ONCE(mpam_resctrl_default_group)` 进行判断，逻辑不够准确，可能导致默认值处理不当。

#解决方案
修正 `resctrl_arch_set_cdp_enabled` 函数中的赋值逻辑，将 `partid_d` 正确映射到 `CDP_DATA`，将 `partid_i` 正确映射到 `CDP_CODE`。同时，优化 `mpam_thread_switch` 中的判断逻辑，将条件改为 `if (!regval)` 以正确处理默认配置情况。
