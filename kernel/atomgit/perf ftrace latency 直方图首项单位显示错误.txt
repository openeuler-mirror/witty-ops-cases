#标题
perf ftrace latency 直方图首项单位显示错误

#内核版本
OLK-6.6

#问题现象
在使用 `perf ftrace latency` 命令并指定 `--use-nsec` 参数时，输出结果中的直方图第一项（first entry）的时间单位显示错误。具体表现为：虽然用户指定了纳秒单位，但第一行记录仍然显示为微秒，而后续行均正确显示为纳秒。

#主要错误日志
无相关内核日志，问题体现为工具输出显示异常。错误输出示例如下（第一行单位错误显示为 us）：
```
#   DURATION     |      COUNT | GRAPH                                          |
     0 - 1    us |          0 |                                                |
     1 - 2    ns |          0 |                                                |
```

#问题根因
在 `tools/perf/builtin-ftrace.c` 文件的 `display_histogram` 函数中，打印直方图第一项的代码逻辑硬编码了时间单位字符串为 "us"，未根据传入的 `use_nsec` 布尔参数进行判断和切换。这导致无论是否使用纳秒选项，首项单位始终显示为微秒。

#解决方案
修改 `display_histogram` 函数中打印直方图首项的 `printf` 调用，将原本硬编码的 "us" 字符串替换为三元表达式 `use_nsec ? "ns" : "us"`。根据用户是否指定 `--use-nsec` 参数动态选择显示 "ns" 或 "us"，确保首项单位与后续行保持一致。
