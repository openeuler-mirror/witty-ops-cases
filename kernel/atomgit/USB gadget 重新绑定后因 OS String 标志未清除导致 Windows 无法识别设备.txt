#标题
USB gadget 重新绑定后因 OS String 标志未清除导致 Windows 无法识别设备

#内核版本
OLK-6.6

#问题现象
在 ConfigFS gadget 功能切换场景下，先使用支持 OS String 的功能（如 adb/mtp），绑定成功后解绑，切换为不支持 OS String 的功能（如 hid）并重新绑定时，Windows 系统无法识别该 USB 设备，导致连接失败。

#主要错误日志
未知

#问题根因
`use_os_string` 标志位在首次绑定支持 OS String 的功能（如 adb/mtp）时被设置为 true。在重新绑定（re-bind）使用其他不支持 OS String 的功能（如 hid）时，该标志位未被正确重置为 false，导致 USB 设备错误地响应 OS String 描述符，从而引发 Windows 系统识别失败。

#解决方案
在 `configfs_composite_bind` 函数中增加逻辑判断，若当前配置不需要 OS String 支持（else 分支），显式将 `cdev->use_os_string` 设置为 false，确保每次绑定时该标志位状态正确。
