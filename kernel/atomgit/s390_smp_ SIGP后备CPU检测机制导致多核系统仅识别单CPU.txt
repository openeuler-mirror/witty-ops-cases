#标题
s390/smp: SIGP后备CPU检测机制导致多核系统仅识别单CPU

#内核版本
OLK-6.6

#问题现象
在 s390 架构下，当 SCLP CPU 检测机制失效而触发 SIGP 后备检测机制时，系统无法正确识别所有 CPU。具体表现为：本应识别出多个 CPU 的系统仅能看到单个 CPU（启动 CPU），导致系统无法利用多核资源。

#主要错误日志
未知

#问题根因
在进行代码清理（commit d08d94306e90）后，SIGP 后备检测机制的逻辑出现缺陷。原逻辑中，CPU 类型的判断信息是由后备机制生成的结构体提供；重写后改为使用全局 SCLP 状态。当 SCLP 状态指示应考虑某种 CPU 类型，而后备机制添加的 CPU 类型与启动 CPU 类型不一致时，由于新 CPU 类型匹配逻辑错误，导致这些 CPU 未被加入系统。

#解决方案
在 SIGP 后备检测机制生成 CPU 信息时，将启动 CPU 的类型（boot_core_type）直接复制到生成的核心信息结构体中，确保后续添加 CPU 时类型匹配正确，从而修复多核识别失败的问题。
