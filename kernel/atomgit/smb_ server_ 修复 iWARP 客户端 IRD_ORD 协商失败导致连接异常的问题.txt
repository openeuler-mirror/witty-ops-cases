#标题
smb: server: 修复 iWARP 客户端 IRD/ORD 协商失败导致连接异常的问题

#内核版本
OLK-6.6

#问题现象
使用 iWARP 协议的 SMB 直连客户端（特别是旧版 cifs.ko 或仅支持 MPA v1 的客户端）在连接 openEuler SMB 服务器时，因 IRD（Initiator Depth）和 ORD（Responder Resources）协商处理不当，可能导致连接建立失败或功能异常。

#主要错误日志
未知

#问题根因
服务端在 `smb_direct_accept_client` 中未正确处理 iWARP 协议的 IRD/ORD 私有数据协商。
1. 未将协商动作提前到连接请求处理阶段（`smb_direct_handle_connect_request`），导致无法正确识别客户端请求的深度与资源数。
2. 在向遗留 iWARP 客户端发送响应时，IRD/ORD 值未按照大端序（big endian）发送。
3. 对于发送了错误小端序（little endian）请求的旧版客户端（如旧 cifs.ko），服务端缺乏兼容性处理，导致协商值解析错误。

#解决方案
1. 将 IRD/ORD 的协商逻辑移至 `smb_direct_handle_connect_request` 函数中提前处理，并在连接接受阶段直接使用协商好的值。
2. 在结构体中新增 `legacy_iwarp`、`initiator_depth` 和 `responder_resources` 字段以保存协商状态与参数。
3. 在检测到遗留 iWARP 客户端时，将 IRD/ORD 值使用 `cpu_to_be32` 转换为大端序发送。
4. 增加对旧版客户端发送的小端序 IRD/ORD 数据的兼容检测与处理，确保协商过程正确完成。
