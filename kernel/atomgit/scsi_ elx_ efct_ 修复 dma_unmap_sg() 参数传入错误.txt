#标题
scsi: elx: efct: 修复 dma_unmap_sg() 参数传入错误

#内核版本
OLK-6.6

#问题现象
在使用 elx: efct 驱动进行 SCSI 数据传输时，由于 DMA 映射解除函数 `dma_unmap_sg()` 的参数传入不正确，可能导致内核 DMA 子系统状态异常。该问题属于底层驱动缺陷，可能引发内存损坏、I/O 路径异常或难以复现的系统不稳定现象。

#主要错误日志
未知

#问题根因
根据 Linux 内核 DMA API 规范，`dma_unmap_sg()` 函数的 `nents` 参数必须传入调用 `dma_map_sg()` 时使用的原始 scatterlist 条目数量，而不是 `dma_map_sg()` 返回的映射后条目数量。
代码中原实现错误地将 `ocp->seg_map_cnt`（即 `dma_map_sg()` 的返回值）传给了 `dma_unmap_sg()`，违反了 API 约定，导致 DMA 解映射操作未能正确匹配映射时的状态。

#解决方案
修正 `drivers/scsi/elx/efct/efct_lio.c` 文件中 `efct_lio_sg_unmap` 函数的调用参数。将 `dma_unmap_sg()` 的 `nents` 参数从 `ocp->seg_map_cnt` 更改为 `cmd->t_data_nents`，确保传入的是原始 scatterlist 的条目数量，符合内核 DMA API 的配对使用要求。
