#标题
crypto: img-hash - dma_unmap_sg 参数传入错误修复

#内核版本
OLK-6.6

#问题现象
在使用 Imagination Technologies 硬件哈希加速器进行 DMA 操作时，`dma_unmap_sg` 函数被传入了错误的 nents 值。虽然此问题主要由代码审查发现，但错误的 unmap 参数可能导致 DMA 映射表管理混乱，进而在高负载或特定内存压力下引发系统异常、内存泄露或 DMA 相关的稳定性问题。

#主要错误日志
未知

#问题根因
根据 Commit 描述，代码在调用 `dma_unmap_sg()` 时错误地使用了 `ctx->dma_ct`（即 `dma_map_sg()` 的返回值）作为 nents 参数。根本原因在于：`dma_unmap_sg()` 函数要求传入的 nents 必须与 `dma_map_sg()` 调用时传入的原始 nents 值一致，而不是 map 函数的返回值。

#解决方案
修正 `drivers/crypto/img-hash.c` 文件中 `img_hash_write_via_dma_stop` 函数的 `dma_unmap_sg` 调用参数。将第三个参数从 `ctx->dma_ct` 修改为 `1`，以确保传入的是原始映射时的 nents 值，符合 DMA API 的使用规范。
