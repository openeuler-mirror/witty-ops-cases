#标题
BPF freplace 扩展程序修改数据包行为校验测试用例

#内核版本
OLK-6.6

#问题现象
在 BPF 程序扩展（freplace）场景下，验证器对被替换函数与替换函数关于数据包修改属性的一致性校验逻辑需要被覆盖验证。具体涉及以下场景的组合测试：
- 替换修改数据包的函数为修改数据包的函数（应当成功）。
- 替换修改数据包的函数为不修改数据包的函数（应当成功）。
- 替换不修改数据包的函数为修改数据包的函数（应当失败）。
- 替换不修改数据包的函数为不修改数据包的函数（应当成功）。

#主要错误日志
Extension program changes packet data

#问题根因
本次提交为测试用例补丁，用于验证 BPF 验证器对函数替换逻辑的正确性。其背景是验证器需要确保：如果被替换的目标函数不会修改数据包数据，那么用于替换的扩展程序也不应修改数据包数据；如果目标函数本身修改数据包，则扩展程序可以修改也可以不修改。

#解决方案
新增一组 BPF 自测试用例（selftests/bpf），通过定义具有不同数据包修改属性的函数组合，并使用 `freplace` 机制进行替换加载，验证验证器在不同组合下的行为是否符合预期（特别是对 "Extension program changes packet data" 错误的触发条件进行确认）。
