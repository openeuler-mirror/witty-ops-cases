#标题
bpftool调用qsort传入NULL指针导致未定义行为

#内核版本
OLK-6.6

#问题现象
当系统中没有 netfilter 条目需要显示时，bpftool 工具在运行时触发未定义行为。运行时检测工具（如 UBSan）会捕获此问题并报错，指出向声明为非空的参数传递了空指针。

#主要错误日志
net.c:827:2: runtime error: null pointer passed as argument 1, which is declared to never be null

#问题根因
当 netfilter 没有条目需要显示时，`nf_link_info` 指针为 NULL，数量为 0。代码逻辑直接调用 `qsort(NULL, 0, ...)`。根据 C 标准第 7.1.4 节关于库函数使用的描述，向函数参数传递无效值（如空指针）属于未定义行为。

#解决方案
在调用 `qsort` 之前增加判断，如果 `nf_link_info` 为 NULL，则直接返回，避免使用空指针调用库函数。
