#标题
修复 struct scsi_device 中 no_start_on_resume 标志位移除导致的 KABI 破损问题

#内核版本
OLK-6.6

#问题现象
内核二进制接口（KABI）一致性检查失败。由于先前提交移除了 `struct scsi_device` 结构体中的 `no_start_on_resume` 标志位，导致结构体布局发生变化，破坏了内核模块与内核核心之间的二进制兼容性，可能引发加载兼容模块时的异常或系统不稳定。

#主要错误日志
未知

#问题根因
先前提交 9e6075e112ee ("scsi: Remove scsi device no_start_on_resume flag") 从 `struct scsi_device` 中直接删除了 `no_start_on_resume` 标志位。该操作改变了结构体的内存布局，破坏了内核定义的 KABI（Kernel Application Binary Interface）稳定性规则。

#解决方案
使用 `KABI_DEPRECATE` 宏将已删除的 `no_start_on_resume` 标志位重新添加回 `struct scsi_device` 结构体中。该宏允许保留字段以维护结构体大小和布局的兼容性，同时将该字段标记为废弃状态，从而在不恢复功能逻辑的情况下修复 KABI 破损问题。
