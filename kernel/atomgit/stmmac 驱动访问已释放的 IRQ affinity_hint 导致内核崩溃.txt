#标题
stmmac 驱动访问已释放的 IRQ affinity_hint 导致内核崩溃

#内核版本
OLK-6.6

#问题现象
在卸载 stmmac 网卡驱动或释放中断资源时，内核可能发生崩溃或触发内存访问错误，原因是访问了已经释放的 CPU 掩码内存指针。

#主要错误日志
未知

#问题根因
在 `stmmac_request_irq_multi_msi` 函数中，变量 `cpu_mask` 被定义为局部变量（栈上分配）。代码逻辑中将其地址传递给 `irq_set_affinity_hint` 函数作为亲和性提示的存储位置。当函数返回后，该局部变量的栈内存被释放并回收，导致存储的 IRQ affinity_hint 指针指向了无效的内存区域。后续在移除设备驱动或清理中断时，内核若尝试访问该指针以获取或清除亲和性配置，就会触发内存访问违规。

#解决方案
移除局部变量 `cpu_mask` 的定义，改用内核 API `cpumask_of(cpu)` 动态获取指定 CPU 的掩码指针。该 API 返回一个有效的全局指针，避免了使用局部栈内存存储亲和性提示，从而防止了内存被释放后访问的问题。
