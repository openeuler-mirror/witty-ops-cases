#标题
MIPS Lantiq Danube 设备树缺失属性导致模式检查告警

#内核版本
OLK-6.6

#问题现象
在编译或校验 Lantiq Danube 平台设备树文件时，触发 YAML 模式检查警告，提示 cpus 节点和 cpu 子节点缺少必要的属性字段。

#主要错误日志
arch/mips/boot/dts/lantiq/danube_easy50712.dtb: cpus: '#address-cells' is a required property
	from schema $id: http://devicetree.org/schemas/cpus.yaml#
arch/mips/boot/dts/lantiq/danube_easy50712.dtb: cpus: '#size-cells' is a required property
	from schema $id: http://devicetree.org/schemas/cpus.yaml#
arch/mips/boot/dts/lantiq/danube_easy50712.dtb: cpu@0 (mips,mips24Kc): 'reg' is a required property
	from schema $id: http://devicetree.org/schemas/mips/cpus.yaml#

#问题根因
设备树源文件 `danube.dtsi` 中的 `cpus` 节点未定义 `#address-cells` 和 `#size-cells` 属性，且 `cpu@0` 节点缺少 `reg` 属性，不符合 `cpus.yaml` 和 `mips/cpus.yaml` 规范要求。

#解决方案
在设备树 `cpus` 节点中添加 `#address-cells = <1>` 和 `#size-cells = <0>` 属性，并在 `cpu@0` 节点中添加 `reg = <0>` 属性，以符合设备树规范，消除模式检查告警。
