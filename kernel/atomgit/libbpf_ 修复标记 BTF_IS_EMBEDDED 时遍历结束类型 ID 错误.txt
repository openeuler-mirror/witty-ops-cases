#标题
libbpf: 修复标记 BTF_IS_EMBEDDED 时遍历结束类型 ID 错误

#内核版本
OLK-6.6

#问题现象
在进行 BTF（BPF Type Format）重定位时，如果将 split BTF 重定向到 vmlinux 基础 BTF，内核或 libbpf 工具在遍历 split BTF 类型以标记 `BTF_IS_EMBEDDED` 时，因遍历结束条件错误导致部分类型未被处理或处理逻辑异常。这可能导致后续的类型匹配校验（要求 name 和 size 均匹配）出现错误，影响 BPF 程序的正确加载或运行。

#主要错误日志
未知

#问题根因
在函数 `btf_relocate_map_distilled_base` 中，遍历 split BTF 类型 ID 的循环结束条件设置错误。原代码使用 `id < r->nr_split_types` 作为结束条件，而正确的遍历范围应当是从 `nr_dist_base_types` 开始，直到 `nr_dist_base_types + nr_split_types` 结束。这导致遍历并未覆盖完整的 split BTF 类型区间。

#解决方案
修正循环遍历的结束类型 ID 计算方式。将循环条件从 `id < r->nr_split_types` 修改为 `id < r->nr_dist_base_types + r->nr_split_types`，确保正确遍历整个 split BTF 类型范围，从而正确标记所有相关的结构体/联合体成员。
