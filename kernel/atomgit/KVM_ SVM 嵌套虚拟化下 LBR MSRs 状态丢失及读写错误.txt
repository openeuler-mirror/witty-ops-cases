#标题
KVM: SVM 嵌套虚拟化下 LBR MSRs 状态丢失及读写错误

#内核版本
OLK-6.6

#问题现象
在支持 LBRV (Last Branch Record Virtualization) 扩展的 AMD 平台上运行嵌套虚拟化时，L1 Hypervisor 无法正确读写 L2 虚拟机的 Last Branch Record (LBR) 相关 MSR 寄存器（DEBUGCTL、LASTBRANCHFROMIP 等）。当 L1 尝试访问这些寄存器时，可能读取到错误数据（应返回 0 但返回了无效值），或者在尝试写入时导致操作失败（返回 Unsupported），导致 L1 层面的性能分析工具（如 perf）无法正常工作或 L2 虚拟机迁移后状态不一致。

#主要错误日志
未知

#问题根因
KVM SVM 模块在处理嵌套虚拟化状态切换时存在逻辑缺失：
1. `svm_copy_vmrun_state` 函数在从 L1 切换到 L2 时，遗漏了对 LBR 相关字段（dbgctl, br_from, br_to, last_excp_from, last_excp_to）的拷贝，导致 L2 运行时 LBR 状态丢失。
2. `svm_get_msr` 函数在读取 LBR MSRs 时，未检查 `lbrv` 是否开启。当 LBRV 未启用时，应返回 0，但代码错误地尝试从 vmcb 中读取，可能返回无效数据。
3. `svm_set_msr` 函数缺少对 `MSR_IA32_LASTBRANCHFROMIP` 等四个寄存器的写处理逻辑，且未正确校验 `lbrv` 标志位，导致写入操作不被支持或被错误处理。
4. `msrs_to_save_base` 数组未包含 LBR 相关 MSRs，导致迁移或保存虚拟机状态时遗漏这些寄存器。

#解决方案
1. 在 `svm_copy_vmrun_state` 函数中增加对 LBR 相关字段的拷贝逻辑，并增加 `kvm_cpu_cap_has(X86_FEATURE_LBRV)` 判断及保留位检查。
2. 修改 `svm_get_msr` 中 LBR 寄存器的读取逻辑，增加 `lbrv` 判断：若 LBRV 未启用，直接返回 0。
3. 在 `svm_set_msr` 中补全对 `MSR_IA32_LASTBRANCHFROMIP`、`MSR_IA32_LASTBRANCHTOIP`、`MSR_IA32_LASTINTFROMIP`、`MSR_IA32_LASTINTTOIP` 的写处理 case，增加 `lbrv` 校验，若不支持则返回 `KVM_MSR_RET_UNSUPPORTED`，否则更新 vmcb 并标记 dirty。
4. 将 LBR 相关 MSRs 添加到 `msrs_to_save_base` 数组中，确保状态保存与迁移的完整性。
