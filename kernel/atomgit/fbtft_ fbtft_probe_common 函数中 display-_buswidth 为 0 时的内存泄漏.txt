#标题
fbtft: fbtft_probe_common 函数中 display->buswidth 为 0 时的内存泄漏

#内核版本
OLK-6.6

#问题现象
在 fbtft_probe_common() 函数执行过程中，若 display->buswidth 为 0，函数会直接返回错误码 -EINVAL，但未释放此前通过 fbtft_framebuffer_alloc() 分配的 "info" 内存空间，导致内存泄漏。

#主要错误日志
buswidth is not set

#问题根因
fbtft_probe_common() 函数在检测到 "display->buswidth == 0" 时，直接进行了错误返回（return -EINVAL），跳过了后续代码，也未能执行到函数末尾的内存释放逻辑，从而导致已分配的 "info" 对象内存泄漏。

#解决方案
修改错误处理路径，当 "display->buswidth == 0" 条件成立时，不再直接返回，而是将错误码赋值给 ret 变量，并通过 goto out_release 跳转至统一的释放处理入口，调用 fbtft_framebuffer_release() 释放内存，确保资源被正确回收。
