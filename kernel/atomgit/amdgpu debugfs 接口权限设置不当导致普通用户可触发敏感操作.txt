#标题
amdgpu debugfs 接口权限设置不当导致普通用户可触发敏感操作

#内核版本
OLK-6.6

#问题现象
在 AMD GPU 驱动的 debugfs 接口中，`amdgpu_evict_vram`、`amdgpu_evict_gtt` 和 `amdgpu_test_ib` 三个文件的访问权限被错误地设置为 `0444`（所有用户可读）。这导致非特权用户能够读取并触发这些接口，进而执行 VRAM 显存驱逐、GTT 驱逐或 IB（Indirect Buffer）测试等高危操作，可能导致系统不稳定或图形渲染异常。

#主要错误日志
未知

#问题根因
驱动代码在初始化 debugfs 节点时，将 `amdgpu_evict_vram`、`amdgpu_evict_gtt` 和 `amdgpu_test_ib` 的文件权限错误地配置为 `0444`。根据 Commit 描述 "Users should not be able to run these"，这些调试接口旨在供管理员或驱动层内部使用，不应暴露给普通用户执行。

#解决方案
将上述三个 debugfs 文件的权限位从 `0444` 修改为 `0400`（仅所有者可读），限制非特权用户对这些调试接口的访问，从而防止普通用户触发驱逐或测试动作。
