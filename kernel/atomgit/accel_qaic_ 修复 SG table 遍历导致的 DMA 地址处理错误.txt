#标题
accel/qaic: 修复 SG table 遍历导致的 DMA 地址处理错误

#内核版本
OLK-6.6

#问题现象
在对 QAIC 设备进行 DMA 数据传输或控制路径操作时，由于使用了错误的接口遍历 SG table（散列表），导致获取的 DMA 总线地址和长度不正确。这可能引发硬件访问错误的内存地址、数据传输损坏、IO 操作失败或设备功能异常。

#主要错误日志
未知

#问题根因
代码在调用 DMA MAP API 更新 SG table 之后，错误地使用了 `for_each_sgtable_sg()` 宏来遍历 SG table。该宏遍历的是原始的 SG table 条目，而非 DMA 映射后的条目。正确的做法是使用 `for_each_sgtable_dma_sg()` 宏，因为它能正确遍历经过 DMA MAP API 更新后的 SG table，从而获取正确的总线地址（bus address）和长度对。

#解决方案
将 `drivers/accel/qaic/qaic_control.c` 和 `drivers/accel/qaic/qaic_data.c` 文件中涉及 SG table 遍历的代码统一修改为使用 `for_each_sgtable_dma_sg()` 宏，确保在 DMA 映射后能够正确获取总线地址和长度信息。具体修改点包括 `encode_addr_size_pairs`、`clone_range_of_sgt_for_slice` 以及 `encode_reqs` 函数中的循环逻辑。
