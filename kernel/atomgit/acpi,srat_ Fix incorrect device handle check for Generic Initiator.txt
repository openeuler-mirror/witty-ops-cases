#标题
acpi,srat: Fix incorrect device handle check for Generic Initiator

#内核版本
OLK-6.6

#问题现象
在处理 SRAT（System Resource Affinity Table）表中的 Generic Initiator Affinity Structure 时，内核对设备类型的判断逻辑错误，导致无法正确识别 PCI 设备类型，影响设备 proximity domain 的分配与打印逻辑。

#主要错误日志
未知

#问题根因
根据 ACPI 规范，SRAT 表中 `device_handle_type` 字段值为 1 时代表 PCI 设备。现有代码错误地判断 `device_handle_type == 0` 为 PCI 设备，导致判断条件与规范不符，无法正确处理 PCI 设备的亲和性结构。

#解决方案
修正 `drivers/acpi/numa/srat.c` 中的判断逻辑，将 `device_handle_type` 的判断值从 0 修改为 1，使其符合 ACPI 规范要求，正确识别 PCI 设备。
