#标题
perf sched timehist 命令解析时间参数失败导致会话对象内存泄漏

#内核版本
OLK-6.6

#问题现象
执行 `perf sched timehist` 命令并指定时间窗口参数时，若时间字符串解析失败，工具并未正确释放此前已创建的 session 对象，导致内存泄漏。

#主要错误日志
Invalid time string

#问题根因
在 `perf_sched__timehist()` 函数中，当 `perf_time__parse_str()` 解析失败时，代码逻辑直接返回了 `-EINVAL` 错误码，跳过了后续的错误处理流程，导致函数开始时通过 `perf_session__new()` 创建的 session 对象未被释放。

#解决方案
修改 `tools/perf/builtin-sched.c` 文件中的错误处理逻辑。当 `perf_time__parse_str()` 失败时，不再直接 return 返回，而是将错误码赋值给 err 变量，并通过 `goto out` 跳转至统一的清理退出路径，确保 session 资源被正确释放。
