#标题
[qmi_wwan: Telit Cinterion FN990B 设备无法识别或网络功能异常]

#内核版本
OLK-6.6

#问题现象
Telit Cinterion FN990B (PID: 0x10d0) 4G模组在接入系统时，qmi_wwan 驱动无法正确匹配加载，导致设备无法建立网络连接，lsusb 可见设备但无法通过 QMI 协议进行通信。

#主要错误日志
未知

#问题根因
内核驱动 `qmi_wwan.c` 的设备支持列表中缺少 Telit Cinterion FN990B (VID: 0x1bc7, PID: 0x10d0) 的设备 ID 定义。该设备需要设置 DTR 信号才能正常工作，但由于 ID 缺失，驱动未能匹配并应用 `QMI_QUIRK_SET_DTR` 初始化逻辑。

#解决方案
在 `drivers/net/usb/qmi_wwan.c` 文件的 `products` 数组中新增该设备的 USB ID 条目：
`{QMI_QUIRK_SET_DTR(0x1bc7, 0x10d0, 0)}`，使驱动能正确识别设备并应用所需的 DTR 配置。
