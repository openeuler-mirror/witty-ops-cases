#标题
ntb: ntb_hw_switchtec 修复 MW LUT 数为 0 时的移位越界问题

#内核版本
OLK-6.6

#问题现象
当 NTB 配置中 MW LUT（Look-Up Table）数量被设置为 0 时，驱动初始化过程中调用 `rounddown_pow_of_two` 函数会导致未定义行为。这通常表现为内核运行时错误或 UBSAN（Undefined Behavior Sanitizer）捕获到的移位越界错误，可能影响 NTB 设备的正常初始化。

#主要错误日志
未知

#问题根因
MW LUT 的数量取决于 NTB 配置，可以通过配置设置为 0。原代码逻辑在获取 `nr_lut_mw` 后，无条件调用 `rounddown_pow_of_two(sndev->nr_lut_mw)` 进行向下取整运算。当输入值为 0 时，该宏内部实现的位运算会导致未定义行为。

#解决方案
在调用 `rounddown_pow_of_two` 宏之前增加非零判断。仅在 `nr_lut_mw` 或 `peer_nr_lut_mw` 的值大于 0 时才执行向下取整操作，避免了传入 0 值引发的未定义行为。
