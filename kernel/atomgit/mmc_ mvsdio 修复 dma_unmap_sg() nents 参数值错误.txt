#标题
mmc: mvsdio 修复 dma_unmap_sg() nents 参数值错误

#内核版本
OLK-6.6

#问题现象
Marvell SoCs 的 SDIO 驱动在进行 DMA 数据传输完成后，调用 `dma_unmap_sg()` 解除映射时传入的参数（nents）不正确。该错误可能导致 DMA 映射表管理异常，进而引发内存泄漏、内核崩溃或数据损坏等问题。

#主要错误日志
未知

#问题根因
驱动代码在 `mvsd_finish_data` 函数中调用 `dma_unmap_sg()` 时，错误地使用了 `host->sg_frags`（即 `dma_map_sg()` 的返回值）作为 nents 参数。根据内核 DMA API 规范，`dma_unmap_sg()` 函数应该使用与 `dma_map_sg()` 相同的 nents 值（即传入的原始 scatterlist 条目数），而不是 map 函数返回的映射后的条目数。

#解决方案
修改 `drivers/mmc/host/mvsdio.c` 文件中的 `dma_unmap_sg()` 调用，将第二个参数由 `host->sg_frags` 修正为 `data->sg_len`，确保 unmap 操作使用正确的 scatterlist 条目数量，符合 DMA API 的调用契约。
