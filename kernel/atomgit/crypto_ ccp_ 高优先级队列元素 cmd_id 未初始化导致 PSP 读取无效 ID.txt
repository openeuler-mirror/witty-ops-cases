#标题
crypto: ccp: 高优先级队列元素 cmd_id 未初始化导致 PSP 读取无效 ID

#内核版本
OLK-6.6

#问题现象
在某些 PSP 固件版本中，当高优先级队列为空时，固件仍尝试读取队列头部的元素进行处理。若此时头部元素的 `cmd_id` 为无效值（未初始化），PSP 会返回 `PSP_RBHEAD_QPAUSE_INT_STAT` 错误，导致命令处理流程异常。此外，原代码在发送命令前修改队列头部 `cmd_id` 时缺乏锁保护，存在与正在入队元素发生竞态条件导致数据不一致的风险。

#主要错误日志
未知

#问题根因
1. **无效内存读取**：高优先级队列的 `cmd_id` 在初始化阶段未赋有效值，内存默认为 0 或随机值。当 PSP 固件尝试处理空队列头部元素时，读取到非法的 `cmd_id`。
2. **竞态条件风险**：原修复逻辑在每次下发 ringbuffer 命令前，若发现高优先级队列为空，会动态修改头部元素的 `cmd_id`。此操作未持有锁，可能与并发的元素入队操作产生竞态，破坏了队列中已有元素的 `cmd_id`。

#解决方案
将修复逻辑从“运行时动态修改”调整为“初始化阶段全量预设”：
1. 在高优先级队列初始化函数 `__psp_ringbuffer_queue_init` 中，增加遍历逻辑，将队列中所有元素的 `cmd_id` 预先设置为有效值 `TKM_PSP_CMDID`。
2. 移除 `__psp_do_ringbuffer_cmds_locked` 函数中针对空队列头部 `cmd_id` 的动态修改代码，彻底消除了竞态条件风险。
3. 在头文件中新增定义 `CSV_RING_BUFFER_ELEMENT_NUM` 宏以支持队列元素的遍历初始化。
