#标题
rtc: rv3028 修复时钟频率超出范围时的错误处理

#内核版本
OLK-6.6

#问题现象
当调用 `rv3028_clkout_round_rate` 函数请求的时钟频率高于设备支持的最高频率时，函数错误地返回 0，导致时钟输出被意外禁用，而非返回设备支持的最近频率。

#主要错误日志
未知

#问题根因
函数 `rv3028_clkout_round_rate` 在处理超出范围的请求频率时逻辑错误。当请求频率高于支持的频率列表中的最大值时，循环结束未找到匹配项，原代码直接返回 0。根据 clk API 规范，`round_rate()` 应返回最接近的支持频率（此处即最大支持频率），返回 0 导致时钟被禁用。

#解决方案
修改 `drivers/rtc/rtc-rv3028.c` 文件中的 `rv3028_clkout_round_rate` 函数。当请求频率超出最大支持范围时，将返回值从 `0` 修改为 `clkout_rates[0]`（即数组中定义的最大支持频率），确保符合 clk API 规范并保持时钟正常工作。
