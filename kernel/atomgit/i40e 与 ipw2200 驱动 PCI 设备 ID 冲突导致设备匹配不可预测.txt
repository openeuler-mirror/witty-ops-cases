#标题
i40e 与 ipw2200 驱动 PCI 设备 ID 冲突导致设备匹配不可预测

#内核版本
OLK-6.6

#问题现象
PCI 设备 ID 8086:104f 同时被 i40e（有线网卡）和 ipw2200（无线网卡）两个驱动匹配。由于同一个设备 ID 存在于多个驱动中，导致内核加载驱动时行为不可预测，可能会出现错误的驱动绑定到设备上，导致相应的网络设备（i40e 网卡或 ipw2200 无线网卡）无法正常工作或初始化失败。

#主要错误日志
未知

#问题根因
ID 8086:104f 被错误地同时添加到了 i40e 和 ipw2200 驱动的 PCI 设备 ID 表中。根据设计，同一设备 ID 不应出现在多个驱动中，这导致了驱动匹配的冲突和不可预测性。

#解决方案
通过区分设备类来消除冲突。修改 i40e 和 ipw2200 驱动的 PCI 设备 ID 表，将单纯的厂商/设备 ID 匹配（PCI_VDEVICE）修改为带有 PCI Class 码过滤的匹配（PCI_DEVICE）：
1. i40e 驱动：在匹配 ID 8086:104f 时，显式指定匹配 PCI_CLASS_NETWORK_ETHERNET 类。
2. ipw2200 驱动：在匹配 ID 8086:104f 时，显式指定匹配 PCI_CLASS_NETWORK_OTHER 类。
以此利用两类设备硬件 Class 不同的特性，确保内核能准确区分并绑定正确的驱动。
