#标题
icmp: fix NULL pointer dereference in icmp_tag_validation

#内核版本
OLK-6.6

#问题现象
在调用 `icmp_tag_validation` 函数时，若 `inet_protos[proto]` 对应的协议指针为 NULL，内核在解引用该指针获取 `icmp_strict_tag_validation` 成员时会触发空指针解引用错误，可能导致内核崩溃或异常行为。

#主要错误日志
未知

#问题根因
代码在 RCU 读锁保护下直接对 `rcu_dereference(inet_protos[proto])` 的返回值进行解引用操作，未检查该指针是否为空。当传入的 `proto` 参数对应的协议未注册或不存在时，数组查找返回 NULL，导致访问非法内存地址。

#解决方案
在解引用指针前增加空指针检查。引入局部变量 `ipprot` 存储 `rcu_dereference` 的结果，仅当 `ipprot` 非 NULL 时才访问其成员变量，否则返回 `false`，从而避免空指针解引用。
