#标题
ASoC: cs35l56 未根据扬声器型号加载正确的固件调优文件

#内核版本
OLK-6.6

#问题现象
在装配了多种不同来源扬声器（需要不同调优参数）的硬件型号上，声卡驱动 cs35l56 无法根据当前硬件上实际安装的扬声器类型自动匹配并加载正确的 bin 固件文件。这导致扬声器无法获得最佳性能表现，可能出现音质异常或性能未达标的情况。

#主要错误日志
- 推断：Failed to get spk-id-gpios
- 推断：Failed to read spk-id[%d]
- 推断：Speaker ID = %d

#问题根因
部分硬件制造商使用多种来源的扬声器，这些扬声器需要不同的固件调优才能达到最佳性能。硬件上通过一个或多个 GPIO 的电平组合来指示当前安装的扬声器类型。此前的驱动代码仅使用子系统 ID（PCI SSID 或固件 UID）作为固件标识，忽略了 GPIO 指示的扬声器变体信息，导致无法区分同一设备型号下的不同扬声器配置，从而加载了错误的或不匹配的调优文件。

#解决方案
1. 新增 `cs35l56_get_speaker_id` 函数，用于读取设备树中 "spk-id-gpios" 属性对应的 GPIO 状态，并将其组合为一个整数值作为扬声器 ID。
2. 在驱动初始化流程中调用该函数获取扬声器 ID。
3. 修改固件系统名称的生成逻辑：若获取到有效的扬声器 ID，则将其附加到子系统 ID 后面（格式如 "%04x%04x-spkid%d" 或 "%s-spkid%d"），确保生成的固件文件名能与特定扬声器型号的调优文件精确匹配。
4. 在 `cs35l56_private` 结构体中新增 `speaker_id` 成员用于存储该 ID，默认初始化为 `-ENOENT`。
