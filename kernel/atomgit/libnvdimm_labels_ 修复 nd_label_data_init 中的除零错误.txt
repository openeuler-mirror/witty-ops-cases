#标题
libnvdimm/labels: 修复 nd_label_data_init 中的除零错误

#内核版本
OLK-6.6

#问题现象
在初始化 nvdimm 设备配置数据区域时，若硬件或驱动层返回 `config_size` 为 0，内核在后续处理逻辑中会触发除零错误，导致系统崩溃或异常。

#主要错误日志
dev_dbg(ndd->dev, "failed to init config data area: (%u:%u)\n", ndd->nsarea.max_xfer, ndd->nsarea.config_size);

#问题根因
函数 `nd_label_data_init` 在检查输入参数时，未对 `ndd->nsarea.config_size` 是否为 0 进行校验。当 `config_size` 为 0 时，代码继续执行后续涉及该变量的除法运算，导致发生除零错误。

#解决方案
在 `nd_label_data_init` 函数的入口参数检查逻辑中，增加对 `ndd->nsarea.config_size == 0` 的判断条件。若该值为 0，函数直接返回 `-ENXIO` 错误，避免后续发生除零错误。
