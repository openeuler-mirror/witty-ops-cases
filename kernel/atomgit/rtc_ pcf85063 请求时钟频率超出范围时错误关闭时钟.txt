#标题
rtc: pcf85063 请求时钟频率超出范围时错误关闭时钟

#内核版本
OLK-6.6

#问题现象
当调用 `pcf85063_clkout_round_rate()` 请求的时钟频率高于设备支持的最高频率时，函数错误地返回 0，导致时钟被禁用。根据 Common Clock Framework API 规范，此时应返回设备支持的最大频率，而非禁用时钟。

#主要错误日志
未知

#问题根因
函数 `pcf85063_clkout_round_rate` 在遍历支持的频率列表时，如果请求的频率 `rate` 大于所有支持的频率，循环结束后函数直接返回 0。这违反了 clk API 的设计约定，`round_rate()` 钩子应返回最接近的支持频率（即最大支持频率），返回 0 会导致时钟框架错误地判断需要关闭时钟。

#解决方案
修改 `drivers/rtc/rtc-pcf85063.c` 文件中的 `pcf85063_clkout_round_rate` 函数。当请求频率超过最大支持频率时，将返回值从 `0` 修改为 `clkout_rates[0]`（即数组中存储的最大支持频率），确保函数返回合法的频率值。
