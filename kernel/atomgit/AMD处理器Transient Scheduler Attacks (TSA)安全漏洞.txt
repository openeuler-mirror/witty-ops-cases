#标题
AMD处理器Transient Scheduler Attacks (TSA)安全漏洞

#内核版本
OLK-6.6

#问题现象
受影响的 AMD CPU（主要是 Zen3 和 Zen4 架构）存在硬件安全漏洞。该漏洞可能导致无效信息被转发到后续指令，从而影响指令执行时机并造成数据泄露。在 SMT（超线程）环境下，TSA-SQ 可能导致线程间的信息泄露。

#主要错误日志
Transient Scheduler Attacks: Vulnerable
Transient Scheduler Attacks: Vulnerable: No microcode
Transient Scheduler Attacks: Mitigation: Clear CPU buffers
Transient Scheduler Attacks: Mitigation: Clear CPU buffers: user/kernel boundary
Transient Scheduler Attacks: Mitigation: Clear CPU buffers: VM
tsa_init: current revision: 0x%x

#问题根因
AMD 处理器存在 Transient Scheduler Attacks (TSA) 硬件漏洞。该漏洞源于 CPU 调度行为，可能导致无效信息转发至后续指令，进而影响指令执行时序并引发数据泄露。

#解决方案
1. 新增 `CONFIG_MITIGATION_TSA` 配置项，默认开启针对 AMD CPU 的 TSA 缓解机制。
2. 引入 `tsa=` 内核启动参数控制缓解策略（支持 off, on, user, vm）。
3. 通过检测微码版本确认 CPU 是否支持 `VERW` 指令清除缓冲区功能（X86_FEATURE_VERW_CLEAR）。
4. 在内核态与用户态切换、VM 宿主机与客户机切换（VMRUN）以及空闲（Idle）路径中，根据策略插入 `VERW` 指令以清除 CPU 缓冲区。
5. 新增 sysfs 接口 `/sys/devices/system/cpu/vulnerabilities/tsa` 用于展示当前系统漏洞状态与缓解措施。
