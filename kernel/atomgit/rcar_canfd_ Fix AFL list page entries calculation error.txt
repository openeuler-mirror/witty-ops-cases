#标题
rcar_canfd: Fix AFL list page entries calculation error

#内核版本
OLK-6.6

#问题现象
在 RZ/G3E 等平台上测试时，CAN 控制器的 Channel 1 和 Channel 4 获取到的 AFL（Acceptance Filter List）起始规则索引（start value）错误地计算为 0。这导致 AFL 寄存器内容被错误覆盖，破坏了 Channel 0 的规则配置，进而引发 CAN 通讯功能异常或接收逻辑错误。

#主要错误日志
未知

#问题根因
驱动程序中 `RCANFD_GAFL*` 宏在计算页面条目时，错误地使用了一个局部变量 `start` 来查找 AFL 条目。对于非 Channel 0 的通道，`start` 变量的计算逻辑存在问题，导致某些通道（如 ch1 和 ch4）获取到的 `start` 值为 0，进而导致寄存器写入位置错误，覆盖了之前的配置。

#解决方案
修改 `rcar_canfd_configure_afl_rules` 函数接口，增加 `rule_entry` 参数。在调用处维护一个累加的 `rule_entry` 变量，并将其传递给配置函数，使用该 `rule_entry` 来正确计算 AFL 列表中的页面条目，而不是依赖之前错误的 `start` 局部变量计算逻辑，从而确保不同通道的规则写入正确的 AFL 条目位置。
