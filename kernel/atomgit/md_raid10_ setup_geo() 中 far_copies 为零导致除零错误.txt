#标题
md/raid10: setup_geo() 中 far_copies 为零导致除零错误

#内核版本
OLK-6.6

#问题现象
在配置 RAID10 阵列时，若用户提供的 layout 参数中提取出的 near_copies (nc) 或 far_copies (fc) 为 0，且选择了 "improved" far set 布局，内核在计算 `geo->far_set_size` 时会触发除零错误，导致内核崩溃或异常。

#主要错误日志
未知

#问题根因
`setup_geo()` 函数直接从用户提供的 layout 参数中提取 `near_copies` (nc) 和 `far_copies` (fc) 而未进行有效性检查。当 fc 为 0 时，代码执行 `geo->far_set_size = disks / fc` 逻辑，触发了除零错误。

#解决方案
在 `setup_geo()` 函数中提取 nc 和 fc 变量后立即增加有效性校验，若 nc 或 fc 为 0，则直接返回 -1 以拒绝无效配置，防止后续计算发生除零错误。
