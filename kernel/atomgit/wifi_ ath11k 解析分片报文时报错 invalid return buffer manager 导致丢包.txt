#标题
wifi: ath11k 解析分片报文时报错 invalid return buffer manager 导致丢包

#内核版本
OLK-6.6

#问题现象
在 WCN6750 或 WCN6855 设备上解析分片 RX 报文时，系统打印错误日志 "invalid return buffer manager 4"，导致分片报文被错误丢弃。

#主要错误日志
ath11k 17a10040.wifi: invalid return buffer manager 4

#问题根因
先前的修复补丁（commit 71c748b5e01e）为了解决 QCA6390 的问题，在代码中引入了对 HAL_RX_BUF_RBM_SW3_BM 的硬编码检查。然而，WCN6750 和 WCN6855 设备的正确返回缓冲区管理器 ID 应为 HAL_RX_BUF_RBM_SW1_BM。由于代码未包含对该 ID 的检查，导致逻辑误判为无效 ID，从而触发错误打印并丢弃报文。

#解决方案
修改 `ath11k_dp_process_rx_err` 和 `ath11k_hal_wbm_desc_parse_err` 函数中的判断逻辑，在检查 HAL_RX_BUF_RBM_SW3_BM 的基础上，增加对 HAL_RX_BUF_RBM_SW1_BM 的合法性检查，从而兼容 WCN6750/WCN6855 设备，防止正常的分片报文被丢弃。
