#标题
drm/omap: 修复 omapdss_device_disconnect 函数潜在的 NULL 指针解引用风险

#内核版本
OLK-6.6

#问题现象
内核静态代码分析工具（smatch）报错，指出在 `drivers/gpu/drm/omapdrm/dss/base.c:176` 的 `omapdss_device_disconnect()` 函数中存在逻辑缺陷：代码在第 169 行假设 `src` 可能为 NULL，但在后续第 176 行未做检查直接解引用，存在空指针解引用导致内核崩溃的风险。

#主要错误日志
未知

#问题根因
代码中 `omapdss_device_disconnect()` 函数在处理 `src` 参数时逻辑不一致。先前代码假定 `src` 可能为 NULL，但在函数内部获取 `dss` 指针时使用了 `src ? src->dss : dst->dss` 的逻辑，而在后续流程或特定分支（如 `!dst` 检查块内的 `WARN_ON(!src->bridge)`）中，若 `src` 为 NULL 则会触发空指针解引用。实际上，调用者总是传入 NULL 作为 `src` 参数，这导致函数内部的复杂防御性逻辑不仅冗余，还引入了上述隐患。

#解决方案
简化并修正 `omapdss_device_connect` 和 `omapdss_device_disconnect` 函数的参数与逻辑：
1. 移除两个函数中从未实际使用或总是为 NULL 的 `src` 参数。
2. 修改 `omapdss_device_disconnect` 函数签名，显式传入 `struct dss_device *dss` 指针，避免通过可能为 NULL 的 `src` 或 `dst` 指针获取 `dss`。
3. 优化错误检查逻辑，使用 `WARN_ON(!dst)` 替代原有的复杂判断，确保在 `dst` 为 NULL 时安全返回。
