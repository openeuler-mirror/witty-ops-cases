#标题
LoongArch 架构 VM_DATA_DEFAULT_FLAGS 宏定义未与主流架构保持一致

#内核版本
OLK-6.6

#问题现象
LoongArch 架构中 `VM_DATA_DEFAULT_FLAGS` 宏定义采用硬编码方式实现，未使用内核统一抽象的 `VM_DATA_FLAGS_TSK_EXEC` 宏，导致该架构下进程数据段的标志位设置逻辑与其他主流架构不一致，增加了代码维护难度并可能引入潜在的内存管理行为差异。

#主要错误日志
未知

#问题根因
内核提交 c62da0c35d58518d ("mm/vma: define a default value for VM_DATA_DEFAULT_FLAGS") 已在各平台统一了 `VM_DATA_DEFAULT_FLAGS` 的默认值定义，但 LoongArch 架构未能及时跟进同步该修改，仍保留了旧的显式硬编码定义方式，导致代码逻辑不一致。

#解决方案
修改 `arch/loongarch/include/asm/page.h` 文件，将 `VM_DATA_DEFAULT_FLAGS` 的定义从显式的位运算表达式替换为内核统一宏 `VM_DATA_FLAGS_TSK_EXEC`，以消除架构间的代码差异并保持内存管理行为的一致性。
