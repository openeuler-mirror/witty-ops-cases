#标题
THP迁移期间deferred split scan导致无效PMD解引用引发内核缺页

#内核版本
OLK-6.6

#问题现象
在迁移透明大页（THP）期间，如果此时并发进行 deferred split scan 操作，内核会访问无效的 PMD migration entry，导致发生 kernel panic。具体表现为无法处理页面错误，系统崩溃。

#问题根因
在 `split_huge_pmd_locked` 函数中，代码原本对 `pmd_trans_huge`、`pmd_devmap` 或 `is_pmd_migration_entry` 三种情况统一处理。当 PMD 指向一个 migration entry 时，代码错误地调用了 `pmd_folio(*pmd)` 进行比较。由于 migration entry 不是一个正常的 folio 指针，对该地址进行解引用会导致无效地址访问，从而触发 panic。此外，PMD migration entry 已经被锁定，不可能作为目标 folio 进行替换，因此无需进行 folio 比较。

#解决方案
在 `split_huge_pmd_locked` 函数中增加对 PMD migration entry 的特定检查。如果检测到是 PMD migration entry，则直接跳过 folio 一致性检查（`pmd_folio` 调用）并提前返回，避免对无效地址进行解引用。修改后的逻辑确保不会对 migration entry 调用 `pmd_folio`，从而修复了并发访问导致的 crash 问题。
