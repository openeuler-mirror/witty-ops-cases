#标题
sign-file与extract-cert因使用OpenSSL 3.0已废弃接口导致编译或运行时告警

#内核版本
OLK-6.6

#问题现象
内核构建工具 `sign-file` 和 `extract-cert` 在编译或运行过程中，因调用了自 OpenSSL 3.0 起被废弃的 `ERR_get_error_line()` 函数，产生废弃接口告警，可能影响构建洁净度或在后续 OpenSSL 版本中引发兼容性问题。

#主要错误日志
未知

#问题根因
OpenSSL 3.0 版本将 `ERR_get_error_line()` 函数标记为废弃（deprecated）。`scripts/sign-file.c` 和 `certs/extract-cert.c` 中的辅助函数 `drain_openssl_errors` 和 `display_openssl_errors` 直接调用了该废弃接口。

#解决方案
将废弃接口 `ERR_get_error_line()` 替换为推荐的 `ERR_peek_error_line()`。同时，重构错误处理逻辑，将原有的 `display_openssl_errors`（显示错误）与 `drain_openssl_errors`（静默清除错误）合并为一个统一的 `drain_openssl_errors` 函数，通过传入 `silent` 参数控制是否打印错误信息。
