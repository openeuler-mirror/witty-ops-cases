#标题
drm/vc4: Fix memory leak of BO array in hang state

#内核版本
OLK-6.6

#问题现象
在 vc4 DRM 驱动中，当触发 GPU hang 状态处理流程时，`vc4_save_hang_state()` 函数中分配的 BO（Buffer Object）数组内存未被释放。每次发生 GPU hang 都会导致该数组对应的内存泄漏，随着 hang 次数增加，系统内存消耗持续增长。

#主要错误日志
未知

#问题根因
在 `vc4_save_hang_state()` 函数中，hang state 的 BO 数组是通过 `kzalloc()` 单独分配的，但在 `vc4_free_hang_state()` 函数的清理路径中，代码仅释放了 hang state 结构体本身以及释放了 BO 的引用计数，却遗漏了调用 `kfree()` 来释放 `state->bo` 指针指向的数组内存，导致内存泄漏。

#解决方案
在 `vc4_free_hang_state()` 函数中，在释放 hang state 结构体之前，增加对 `state->bo` 指针的 `kfree()` 调用，确保 BO 数组的内存被正确回收。
