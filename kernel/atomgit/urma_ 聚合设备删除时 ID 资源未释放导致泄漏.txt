#标题
urma: 聚合设备删除时 ID 资源未释放导致泄漏

#内核版本
OLK-6.6

#问题现象
在删除 URMA 聚合设备时，对应的 bond device ID 资源未被释放。随着设备反复创建与删除，会导致 ID 资源耗尽，进而无法创建新的 bond 设备（受限于 `UBAGG_MAX_BONDING_DEV_NUM` 定义的上限）。

#主要错误日志
未知

#问题根因
在 `ubagg_delete_dev` 函数中，删除设备的流程逻辑缺失。虽然执行了从列表移除、注销设备、资源反初始化等操作，但遗漏了调用 `release_bond_device_id_with_name` 释放 bond device ID 的步骤，导致 ID 资源在设备删除后仍被占用，引发资源泄漏。

#解决方案
在 `ubagg_delete_dev` 函数中，于 `uninit_ubagg_res` 之后、引用计数减少之前，增加调用 `release_bond_device_id_with_name(dev->master_dev_name)`，确保设备删除时正确释放对应的 ID 资源。同时，补丁将 `UBAGG_MAX_BONDING_DEV_NUM` 的上限从 256 调整为 1024，以支持更多的设备实例。
