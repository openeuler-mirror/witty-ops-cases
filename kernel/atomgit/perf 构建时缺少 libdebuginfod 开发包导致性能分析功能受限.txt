#标题
perf 构建时缺少 libdebuginfod 开发包导致性能分析功能受限

#内核版本
OLK-6.6

#问题现象
在编译 perf 工具时，若系统缺少 libdebuginfod 开发文件（如头文件），构建过程默认会静默禁用 debuginfod 支持。用户在执行 `perf version --build-options` 时会发现 `debuginfod: [ OFF ]`，导致部分 perf 工具无法自动从 debuginfod 服务器获取所需的调试信息，进而影响性能分析与探测功能。

#主要错误日志
Makefile.config:563: No elfutils/debuginfod.h found, no debuginfo server support, please install elfutils-debuginfod-client-devel or equivalent

#问题根因
perf 的构建系统在 `tools/perf/Makefile.config` 中检测 libdebuginfod 依赖时，如果未找到 `elfutils/debuginfod.h` 头文件（`feature-libdebuginfod` 检测不为 1），仅是不编译相关功能，并未输出明确的警告信息告知用户缺失了推荐依赖，导致用户无法及时感知功能缺失。

#解决方案
在 `tools/perf/Makefile.config` 配置文件中增加构建警告逻辑：当 libdebuginfod 特性检测失败时，输出警告信息提示用户安装对应的开发包（如 libdebuginfod-dev 或 elfutils-debuginfod-client-devel），从而显式告知用户功能受限的原因及解决方法。
