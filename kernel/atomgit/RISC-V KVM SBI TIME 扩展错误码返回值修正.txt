#标题
RISC-V KVM SBI TIME 扩展错误码返回值修正

#内核版本
OLK-6.6

#问题现象
在 RISC-V KVM 虚拟化环境中，当 Guest 通过 SBI（Supervisor Binary Interface）调用 TIME 扩展功能时，如果使用了无效的 function ID，系统错误地返回了 `SBI_ERR_INVALID_PARAM`（无效参数），而非规范要求的 `SBI_ERR_NOT_SUPPORTED`（不支持）。这可能导致 Guest 对 SBI 扩展功能的探测或错误处理逻辑出现异常行为。

#主要错误日志
未知

#问题根因
根据 SBI 规范，当调用无效的 function ID 时，应返回 `SBI_ERR_NOT_SUPPORTED`。但在 `kvm_sbi_ext_time_handler` 函数的实现中，对 `cp->a6 != SBI_EXT_TIME_SET_TIMER` 的判断分支错误地将 `retdata->err_val` 设置为了 `SBI_ERR_INVALID_PARAM`，违反了接口规范。

#解决方案
修改 `arch/riscv/kvm/vcpu_sbi_replace.c` 文件中的 `kvm_sbi_ext_time_handler` 函数，将错误码从 `SBI_ERR_INVALID_PARAM` 修正为 `SBI_ERR_NOT_SUPPORTED`，以符合 SBI v0.2 及更高版本的规范要求。
