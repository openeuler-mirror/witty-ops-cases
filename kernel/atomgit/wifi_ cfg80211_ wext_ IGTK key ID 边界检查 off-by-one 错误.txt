#标题
wifi: cfg80211: wext: IGTK key ID 边界检查 off-by-one 错误

#内核版本
OLK-6.6

#问题现象
在使用 WEXT 接口配置 IGTK（Integrity Group Temporal Key）密钥时，密钥 ID 的有效性校验逻辑存在错误，导致合法的密钥 ID 被错误判定为非法，可能导致密钥配置失败或管理帧加密功能异常。

#主要错误日志
未知

#问题根因
IGTK 的密钥 ID 必须为 4 或 5，但代码在检查密钥 ID 时使用了 off-by-one 错误的逻辑。代码检查的是 `idx` 变量，而该变量此时已经存放了 `key ID + 1` 的值，因此正确的比较对象应该是 5 和 6，而不是 4 和 5。原代码错误地检查了 `idx < 4 || idx > 5`，导致合法的 key ID 4（此时 idx 为 5）被拒或逻辑判断错误。

#解决方案
修正 `net/wireless/wext-compat.c` 中的边界检查条件，将判断语句 `if (idx < 4 || idx > 5)` 修改为 `if (idx < 5 || idx > 6)`，以正确匹配 IGTK key ID 4 和 5 对应的索引值。
