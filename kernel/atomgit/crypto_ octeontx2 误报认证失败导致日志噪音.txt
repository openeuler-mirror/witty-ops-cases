#标题
crypto: octeontx2 误报认证失败导致日志噪音

#内核版本
OLK-6.6

#问题现象
在运行 "ccm(aes)" 自测进行消极测试时，驱动程序会将预期的失败结果误判为软件错误，并疯狂打印错误级别的日志，导致产生大量不必要的内核日志噪音。

#主要错误日志
Request failed with software error code 0x%x

#问题根因
驱动程序在处理 "ccm(aes)" 算法的消极测试用例时，未区分“预期内的认证失败”与“真正的软件错误”。对于消极测试而言，认证失败是正常结果，但代码逻辑中统一使用了 `dev_err` 进行报错打印，导致日志误报。

#解决方案
将相关错误打印逻辑从 `dev_err` 修改为 `pr_debug`，以此抑制消极测试产生的正常失败日志，避免日志噪音。同时在调试信息中增加了算法名称和驱动名称，以便于后续调试识别。
