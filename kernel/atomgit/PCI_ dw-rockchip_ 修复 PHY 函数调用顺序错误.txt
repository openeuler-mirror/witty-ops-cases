#标题
PCI: dw-rockchip: 修复 PHY 函数调用顺序错误

#内核版本
OLK-6.6

#问题现象
在 Rockchip RK356X PCIe 控制器驱动卸载或去初始化过程中，调用 `rockchip_pcie_phy_deinit` 时违反了 PHY 子系统的 API 使用规范。由于 `phy_exit()` 在 `phy_power_off()` 之前被调用，可能导致 PHY 状态管理混乱，影响后续设备的正常使用或导致内核子系统状态异常。

#主要错误日志
未知

#问题根因
在 `rockchip_pcie_phy_deinit` 函数中，代码原本先调用 `phy_exit()` 再调用 `phy_power_off()`。根据内核文档明确说明，`phy_power_off()` 必须在 `phy_exit()` 之前调用。当前调用顺序颠倒了，违反了 PHY 子系统的生命周期管理规则。

#解决方案
调整 `rockchip_pcie_phy_deinit` 函数中的调用顺序，先执行 `phy_power_off(rockchip->phy)` 关闭 PHY 电源，再执行 `phy_exit(rockchip->phy)` 退出 PHY 初始化，使其符合内核 API 规范。
