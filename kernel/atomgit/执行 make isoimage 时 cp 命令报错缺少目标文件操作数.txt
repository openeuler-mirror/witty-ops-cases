#标题
执行 make isoimage 时 cp 命令报错缺少目标文件操作数

#内核版本
OLK-6.6

#问题现象
在执行 "make isoimage" 命令构建 ISO 镜像时，脚本执行中断并报错，提示 cp 命令参数错误。

#主要错误日志
cp: missing destination file operand after 'arch/x86/boot/isoimage/'

#问题根因
genimage.sh 脚本在执行 geniso() 函数时，未判断 FDINITRDS 数组是否为空就直接执行 cp 命令。当 FDINITRDS 数组为空时，cp 命令解析参数失败，导致报错 "missing destination file operand"。

#解决方案
在 arch/x86/boot/genimage.sh 脚本中，执行 cp "${FDINITRDS[@]}" 操作前增加判断逻辑：检查数组长度是否大于 0（if [ ${#FDINITRDS[@]} -gt 0 ]），仅当数组非空时才执行复制操作。该修复方式与同脚本中 hdimage 的处理逻辑保持一致。
