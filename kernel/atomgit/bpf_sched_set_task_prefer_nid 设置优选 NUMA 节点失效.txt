#标题
bpf_sched_set_task_prefer_nid 设置优选 NUMA 节点失效

#内核版本
OLK-6.6

#问题现象
调用 bpf_sched_set_task_prefer_nid 设置任务的优选 NUMA 节点后，设置失效，调度器完全忽略了设置的优选 CPU，导致优选节点功能不生效。

#主要错误日志
未知

#问题根因
bpf_sched_set_task_prefer_nid() 函数直接将完整的节点掩码传递给 set_prefer_cpus_ptr()，未与任务允许的 CPU 掩码取交集。导致优选 CPU 可能位于 task->cpus_ptr 之外，在此类情况下，调度器会直接完全忽略这些优选 CPU，导致优选节点设置无效。

#解决方案
在设置优选 CPU 前，先计算任务允许 CPU 掩码与指定节点 CPU 掩码的交集，确保最终的优选掩码是任务允许 CPU 的子集。
