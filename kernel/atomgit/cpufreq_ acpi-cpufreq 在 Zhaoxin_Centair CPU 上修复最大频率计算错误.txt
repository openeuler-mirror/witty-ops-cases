#标题
cpufreq: acpi-cpufreq 在 Zhaoxin/Centair CPU 上修复最大频率计算错误

#内核版本
OLK-6.6

#问题现象
在 Zhaoxin（兆芯）或 Centaur CPU 平台上，acpi-cpufreq 驱动在进行最大频率（max-freq）计算时结果不正确，导致系统显示或设置的 CPU 最大频率值与硬件实际支持的最高频率不符，影响 CPU 频率策略的正确运行。

#主要错误日志
未知

#问题根因
acpi-cpufreq 驱动中的 `get_max_boost_ratio` 函数在进行频率计算逻辑时，未针对 Zhaoxin 和 Centaur 这两类 CPU 进行特殊处理。这两类处理器的硬件特性或 BIOS 报告的 CPPC 数据特性导致通用的计算逻辑得出了错误的最大频率值。

#解决方案
在 `get_max_boost_ratio` 函数中增加对 CPU 厂商的检查逻辑。当检测到 CPU 厂商为 `X86_VENDOR_ZHAOXIN` 或 `X86_VENDOR_CENTAUR` 时，直接返回 0，跳过后续错误的计算流程，从而规避不正确的最大频率计算问题。
