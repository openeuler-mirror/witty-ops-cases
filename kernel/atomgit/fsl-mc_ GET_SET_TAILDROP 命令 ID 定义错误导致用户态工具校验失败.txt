#标题
fsl-mc: GET/SET_TAILDROP 命令 ID 定义错误导致用户态工具校验失败

#内核版本
OLK-6.6

#问题现象
用户在使用 `restool` 用户空间工具调用 GET_TAILDROP 或 SET_TAILDROP 命令时，命令无法通过内核层的校验，导致相关尾丢弃配置获取或设置功能异常。

#主要错误日志
未知

#问题根因
内核 `fsl-mc` 总线驱动的 `fsl_mc_accepted_cmds` 数组中，`DPSW_GET_TAILDROP` 和 `DPSW_SET_TAILDROP` 的命令 ID（cmdid_value）定义颠倒了，不符合硬件用户手册规范，导致命令 ID 校验失败。

#解决方案
修正 `drivers/bus/fsl-mc/fsl-mc-uapi.c` 中两个命令的 ID 值，将 `DPSW_GET_TAILDROP` 的 `cmdid_value` 改为 `0x0A90`，将 `DPSW_SET_TAILDROP` 的 `cmdid_value` 改为 `0x0A80`，使其与硬件手册一致。
