#标题
drm/rockchip: vop2: Fix cluster windows alpha ctrl registers offset

#内核版本
OLK-6.6

#问题现象
Rockchip VOP2 驱动中 Cluster 窗口的 Alpha 控制寄存器偏移量计算错误。由于物理 ID（phys_id）并不总是连续递增（即并不总是按 0, 1, 2, 3 顺序排列），原代码使用 `phys_id * 0x10` 计算偏移会导致读写错误的硬件寄存器地址。这会导致显示混色（Alpha Blending）功能异常，可能表现为窗口透明度设置无效、画面显示错误或图层混合紊乱。

#主要错误日志
未知

#问题根因
原代码逻辑假设 Cluster 窗口的物理 ID（phys_id）是连续递增的（0, 1, 2...），因此直接使用公式 `phys_id * 0x10` 来计算寄存器偏移。但实际上硬件设计中 Cluster 窗口的物理 ID 不是每个窗口增加 1，导致计算出的寄存器偏移地址与实际硬件不匹配。

#解决方案
移除原有的线性计算公式 `offset = (main_win->data->phys_id * 0x10)`，改为使用 switch-case 语句根据具体的物理 ID（ROCKCHIP_VOP2_CLUSTER0/1/2/3）显式匹配对应的寄存器偏移值（分别为 0x0, 0x10, 0x20, 0x30），确保写入正确的硬件寄存器地址。
