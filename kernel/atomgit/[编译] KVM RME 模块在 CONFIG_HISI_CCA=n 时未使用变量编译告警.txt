#标题
[编译] KVM RME 模块在 CONFIG_HISI_CCA=n 时未使用变量编译告警

#内核版本
OLK-6.6

#问题现象
在禁用 CONFIG_HISI_CCA 配置项编译内核时，`arch/arm64/kvm/rme.c` 文件出现多个“未使用变量”的编译告警，具体现象如下：
1. `rme.c:66:23: warning: unused variable ‘folio’ [-Wunused-variable]`
2. `rme.c:1764:23: warning: unused variable ‘realm’ [-Wunused-variable]`
3. `rme.c:1765:23: warning: unused variable ‘folio’ [-Wunused-variable]`
4. `rme.c:1763:13: warning: unused variable ‘ret’ [-Wunused-variable]`

#主要错误日志
未知

#问题根因
在 `rmi_granule_delegate` 和 `rmi_granule_delegate_get` 函数中，变量定义（如 `folio`、`realm`、`ret`）位于 `#ifdef CONFIG_HISI_CCA` 宏判断之外，而实际使用这些变量的代码块被包裹在 `#ifdef CONFIG_HISI_CCA` 之内。当 `CONFIG_HISI_CCA` 未配置时，变量被定义但未被使用，导致编译器产生 unused variable 告警。

#解决方案
调整 `#ifdef CONFIG_HISI_CCA` 宏的位置，将相关的变量定义语句移入宏内部。确保当 `CONFIG_HISI_CCA` 未定义时，这些变量不会被编译，从而消除编译告警。
