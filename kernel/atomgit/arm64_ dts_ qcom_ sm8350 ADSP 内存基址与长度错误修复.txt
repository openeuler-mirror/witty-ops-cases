#标题
arm64: dts: qcom: sm8350 ADSP 内存基址与长度错误修复

#内核版本
OLK-6.6

#问题现象
Qualcomm SM8350 平台设备树中 ADSP（Audio DSP）remoteproc 节点的寄存器地址配置错误。原配置错误地指向了未使用的内存空间中间区域，导致设备树描述与硬件实际 QDSP PUB 地址空间不符。虽然 Commit 信息指出这对 Linux 用户可能没有功能性影响（因为 PAS loader 不使用该地址空间），但设备树节点单元地址与 reg 属性不一致，且未正确描述 ADSP PAS 区域。

#主要错误日志
未知

#问题根因
设备树中 ADSP PAS（Peripheral Authentication Service）remoteproc 节点的地址空间配置错误：
1. 原配置 `reg = <0 0x17300000 0 0x100>` 错误地使用了从下游 DTS 复制的地址 `0x17300000`，该地址位于未使用的空间中间，且描述的是 PIL 加载器接口而非 PAS 接口。
2. 正确的 QDSP PUB 地址空间应为 `0x0300_0000`，长度为 `0x10000`。

#解决方案
修正 ADSP remoteproc 节点的 `reg` 属性及单元地址：
1. 将节点名称从 `remoteproc@17300000` 修改为 `remoteproc@3000000`。
2. 将 `reg` 属性值从 `<0 0x17300000 0 0x100>` 修改为 `<0x0 0x03000000 0x0 0x10000>`。
3. 在设备树文件中移动节点位置以保持按单元地址排序。
