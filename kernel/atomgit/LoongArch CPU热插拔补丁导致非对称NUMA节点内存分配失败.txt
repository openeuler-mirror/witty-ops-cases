#标题
LoongArch CPU热插拔补丁导致非对称NUMA节点内存分配失败

#内核版本
OLK-6.6

#问题现象
在物理机上，当 Node0 的 CPU 正常启动而 Node1 的 CPU 未能启动时（非对称启动场景），Node1 上的设备无法分配到内存空间，导致设备初始化或运行失败。

#主要错误日志
未知

#问题根因
先前引入的 "LoongArch: Fix cpu hotplug issue" 补丁（此处被回退）破坏了正常的 CPU 拓扑初始化逻辑。该补丁在处理非对称的 NUMA 节点启动时存在缺陷，导致在部分 CPU 未启动的情况下，对应节点上的设备内存分配逻辑出现异常。此外，该补丁本身也引入了其他问题，且上游社区已提供更合理的解决方案。

#解决方案
回退有问题的补丁（Commit 01ee8cea2373），恢复原有的 CPU 拓扑初始化逻辑，后续采用上游内核社区的方案来解决 CPU 热插拔问题。具体代码回退包括：
1. 移除自定义的 `topo_add_cpu` 和 `topo_get_cpu` 接口函数。
2. 恢复 `set_processor_mask` 和 `fdt_smp_setup` 中基于 `cpu_present_mask` 的逻辑 CPU 编号分配方式。
3. 恢复 `acpi_boot_table_init` 中 `boot_cpu_id` 的读取时机，确保初始化顺序正确。
