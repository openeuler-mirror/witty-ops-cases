#标题
media: ccs: CCS 静态数据解析在大块尺寸下的长度计算错误

#内核版本
OLK-6.6

#问题现象
解析 CCS（Camera Control Set）静态数据时，对于大小为 16 KiB 的大数据块，其长度头信息解析出错。由于解析出的长度值被错误计算，导致对数据块长度的理解和校验出现偏差。该问题极难复现，因为此类大数据块在实际使用中非常罕见。

#主要错误日志
未知

#问题根因
CCS 静态数据块的长度字段解析逻辑存在低级错误。在处理 3 字节长度的长度说明符时，代码错误地重复使用了 `length[0]` 来计算中间的 8 位数据，正确的逻辑应使用 `length[1]` 和 `length[2]`。原代码 `(__len3->length[0] << 8)` 导致计算出的块长度值错误。

#解决方案
修正 `ccs_data_parse_length_specifier` 函数中 3 字节长度说明符的计算逻辑。将中间字节的获取来源从 `length[0]` 更正为 `length[1]`，即：
- 修正前：`... + (__len3->length[0] << 8) + __len3->length[1];`
- 修正后：`... + (__len3->length[1] << 8) + __len3->length[2];`
从而确保大尺寸数据块（如 16 KiB）的长度头被正确解析。
