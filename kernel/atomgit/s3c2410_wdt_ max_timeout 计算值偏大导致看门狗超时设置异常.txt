#标题
s3c2410_wdt: max_timeout 计算值偏大导致看门狗超时设置异常

#内核版本
OLK-6.6

#问题现象
看门狗驱动的 max_timeout 计算值比实际硬件支持的最大值更大。由于计算过程中整除运算丢弃了余数，导致软件层允许设置的超时时间上限超过了硬件实际能计时的最大值，可能导致配置的超时时间无效或看门狗行为不符合预期。

#主要错误日志
未知

#问题根因
在计算 max_timeout 时，原逻辑使用表达式 `S3C2410_WTCNT_MAXCNT / (freq / (S3C2410_WTCON_PRESCALE_MAX + 1) / S3C2410_WTCON_MAXDIV)`。该计算过程包含多次整除，中间步骤 `freq / (S3C2410_WTCON_PRESCALE_MAX + 1) / S3C2410_WTCON_MAXDIV` 会丢弃余数，导致分母变小，最终计算出的 max_timeout 结果比实际值偏大。

#解决方案
重构 max_timeout 的计算公式，通过调整计算顺序来消除整除带来的误差。
1. 引入 `<linux/math64.h>` 头文件以支持 64 位高精度除法。
2. 将原公式调整为先计算分子部分（总计数最大值）：`n_max = (S3C2410_WTCON_PRESCALE_MAX + 1) * S3C2410_WTCON_MAXDIV * S3C2410_WTCNT_MAXCNT`。
3. 使用 `div64_ul` 进行 64 位除法计算总时长：`t_max = n_max / freq`，从而避免了中间步骤的精度丢失。
4. 增加边界检查，若计算结果超过 `UINT_MAX` 则将其限制为 `UINT_MAX`，最后返回该值。
