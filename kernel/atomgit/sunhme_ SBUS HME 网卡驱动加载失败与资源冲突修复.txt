#标题
sunhme: SBUS HME 网卡驱动加载失败与资源冲突修复

#内核版本
OLK-6.6

#问题现象
在 Sun Ultra 2 机器上使用 SBUS HME 网卡时，驱动探测失败，系统日志显示无法请求资源区域（error -EBUSY）以及无法映射 TCVR 寄存器，导致网卡无法正常工作。

#主要错误日志
- hme f0072f38: error -EBUSY: can't request region for resource [mem 0x1ffe8c07000-0x1ffe8c0701f]
- hme f0072f38: Cannot map TCVR registers.
- hme f0072f38: probe with driver hme failed with error -16
- hme f007ab44: error -EBUSY: can't request region for resource [mem 0x1ff28c07000-0x1ff28c0701f]
- hme f007ab44: Cannot map TCVR registers.
- hme f007ab44: probe with driver hme failed with error -16

#问题根因
提交 cc216e4b44ce ("net: sunhme: Switch SBUS to devres") 将显式指定大小的 `of_ioremap`（使用 BMAC_REG_SIZE）更改为 `devm_platform_ioremap_resource`，后者会映射整个资源区域。在某些机器（如 Sun Ultra 2）和 PROM 版本上，Open Firmware 提供的资源存在重叠，导致驱动尝试映射 BIGMAC 资源时越界占用了 TCVR 寄存器的资源区域（resource[3] 与 resource[4] 重叠），从而引发 -EBUSY 冲突。

#解决方案
在映射 BIGMAC 寄存器资源前，增加资源大小检查与修正逻辑：如果 `resource[3]` 的大小大于或等于 `BMAC_REG_SIZE`，则强制将其 end 地址截断为 `start + BMAC_REG_SIZE - 1`，避免资源重叠导致的映射冲突。
