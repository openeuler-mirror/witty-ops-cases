#标题
can: peak_usb: USB FD设备因端点判断逻辑错误导致潜在功能异常

#内核版本
OLK-6.6

#问题现象
使用最新固件版本的 PEAK-System USB CAN FD 接口设备时，驱动程序可能无法正确识别设备导出的端点（EP）编号，导致设备无法正常通信或功能异常（潜在故障）。

#主要错误日志
未知

#问题根因
驱动程序在解析设备厂商请求响应结构中的 `type` 字段时存在逻辑错误：
1. **比较运算符使用不当**：原代码使用 `==` 或 `!=` 进行严格相等判断，而最新固件版本规定当 `type` 字段值大于或等于 2 时，响应结构中才包含有效的端点编号。
2. **字节序处理错误**：原代码错误地将 CPU 字节序的常量（`PCAN_USBFD_TYPE_EXT`）与设备返回的小端序数据（`fw_info->type`）直接比较，未进行 `le16_to_cpu` 转换，导致比较结果不可靠。

#解决方案
修正驱动程序中对 `fw_info->type` 字段的判断逻辑：
1. 将判断条件从 `fw_info->type == cpu_to_le16(PCAN_USBFD_TYPE_EXT)` 修正为 `le16_to_cpu(fw_info->type) >= PCAN_USBFD_TYPE_EXT`，确保正确识别固件版本并提取端点信息。
2. 对数据结构体注释进行了同步更新，明确说明扩展数据在 `type >= 2` 时有效。
