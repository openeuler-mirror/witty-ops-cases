#标题
UV系统初始化时对已卸载CPU插槽分配内存导致潜在异常

#内核版本
OLK-6.6

#问题现象
在 SGI UV 系统初始化阶段，当系统中存在已被反配置的 socket 时，内核在为其分配 per hub 信息内存时可能使用了无效的节点 ID。这会导致内存分配在错误的 NUMA 节点上进行，可能引发内存分配失败、内核警告或系统初始化流程异常。

#主要错误日志
未知

#问题根因
在 `uv_system_init_hub` 函数中，代码通过 `uv_blade_to_node(bid)` 获取 NUMA 节点 ID 以进行内存分配。对于已被反配置的 sockets，该函数返回 `SOCK_EMPTY`（即无效的 socket 标识）。直接将此无效值作为 `kzalloc_node` 的参数传递，导致内存分配行为不可预测或非法。

#解决方案
在调用 `kzalloc_node` 分配内存前，增加对节点 ID 的有效性检查。如果 `uv_blade_to_node(bid)` 返回值为 `SOCK_EMPTY`，则将节点 ID 重置为 `NUMA_NO_NODE`，确保内存分配在一个合法的有效节点上进行。
