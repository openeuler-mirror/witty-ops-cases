#标题
sch_hfsc: Fix qlen accounting bug when using peek in hfsc_enqueue()

#内核版本
OLK-6.6

#问题现象
在 HFSC 类中入队第一个数据包时，如果子 qdisc 使用 qdisc_peek_dequeued()，会触发即时出队或丢包，导致 HFSC qdisc 的 qlen 和 backlog 计数不一致。该缺陷会导致空的 HFSC 类留在活动列表中，进而引发 use-after-free 等严重后果。

#主要错误日志
未知

#问题根因
当向 HFSC 类入队第一个数据包时，hfsc_enqueue() 在调用子 qdisc 的 peek() 操作之前，未先增加 sch->q.qlen 和 sch->qstats.backlog。如果子 qdisc 使用 qdisc_peek_dequeued()，这可能触发即时出队和潜在丢包，导致调用 qdisc_tree_reduce_backlog() 时，HFSC qdisc 的 qlen 和 backlog 尚未更新，从而导致队列统计信息不一致。

#解决方案
将 sch->q.qlen 和 sch->qstats.backlog 的增加操作移至调用子 qdisc 的 peek() 操作之前。这确保了在 peek 期间触发丢包或出队时，队列长度和积压统计数据始终是准确的。
