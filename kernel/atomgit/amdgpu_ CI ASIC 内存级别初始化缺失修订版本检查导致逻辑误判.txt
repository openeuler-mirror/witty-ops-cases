#标题
amdgpu: CI ASIC 内存级别初始化缺失修订版本检查导致逻辑误判

#内核版本
OLK-6.6

#问题现象
在特定 CI ASIC（设备 ID 0x67B0 或 0x67B1）的非 Revision 0 显卡上，驱动程序错误地应用了仅适用于 Revision 0 SKU 的内存电源管理变通方案。这可能导致显存频率或电压设置异常，进而影响显卡在特定电源状态下的稳定性和性能。

#主要错误日志
未知

#问题根因
在 `ci_populate_all_memory_levels` 函数中，代码仅检查了设备 ID（0x67B0 或 0x67B1）和 DPM 表计数，用于应用针对特定硬件缺陷的内存级别填充变通方案。然而，该变通方案实际上仅适用于 Revision 0 的 SKU。由于缺少对 `adev->pdev->revision` 的检查，导致该变通方案被错误地应用到了非 Revision 0 的硬件版本上。

#解决方案
在条件判断语句中增加对硬件修订版本（Revision）的检查。具体修改为：在原有的设备 ID 和计数检查基础上，增加 `&& (adev->pdev->revision == 0)` 条件，确保该内存级别调整逻辑仅在 Revision 0 的硬件上执行。
