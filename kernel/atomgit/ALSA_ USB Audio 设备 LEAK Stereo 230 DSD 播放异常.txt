#标题
ALSA: USB Audio 设备 LEAK Stereo 230 DSD 播放异常

#内核版本
OLK-6.6

#问题现象
LEAK Stereo 230 集成放大器（USB ID: 0x2622:0x0061）在 Linux 系统下播放 DSD 音频文件时工作异常。设备无法按预期使用 Native DSD 或 DSD-over-PCM (DoP) 模式，而是错误地尝试进行 DSD-to-PCM 转换，导致播放功能无法正常使用。

#主要错误日志
未知

#问题根因
该设备内置由 XMOS 控制器提供的 ESS9038Q2M DAC，虽然硬件支持 DSD Native 和 DoP 模式，但设备固件未正确向内核声明其 DSD 能力，或者内核默认的探测逻辑与该设备不兼容。由于内核缺少针对该特定 USB ID 的 DSD 格式怪癖，导致驱动程序无法识别并启用正确的 DSD 传输模式。

#解决方案
在 `sound/usb/quirks.c` 文件中添加针对 LEAK Stereo 230 设备的特定怪癖：
1. 在 `snd_usb_interface_dsd_format_quirks` 函数的 switch case 列表中添加设备 ID `USB_ID(0x2622, 0x0061)`，使其匹配 DSD 格式怪癖。
2. 在 `quirk_flags_table` 数组中为 Vendor ID `0x2622` (IAG Limited) 添加 `QUIRK_FLAG_DSD_RAW` 标志，允许该系列设备使用原生 DSD 原始数据传输模式。
