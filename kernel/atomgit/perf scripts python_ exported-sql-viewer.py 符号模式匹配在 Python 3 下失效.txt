#标题
perf scripts python: exported-sql-viewer.py 符号模式匹配在 Python 3 下失效

#内核版本
OLK-6.6

#问题现象
在使用 Python 3 环境运行 `exported-sql-viewer.py` 脚本时，用户输入通配符模式（如 `*` 或 `?`）查找符号功能失效，脚本执行过程中因调用不兼容的 API 抛出异常，导致查询操作无法正常完成。

#主要错误日志
未知

#问题根因
脚本在将用户输入的通配符（`*` 和 `?`）转换为 SQL LIKE 语句（`%` 和 `_`）时，使用了 Python 2 的 `string.maketrans()` 方法。在 Python 3 中，`string.maketrans()` 已被移除或变更，应当使用 `str.maketrans()` 静态方法。由于代码未对 Python 版本进行区分处理，导致在 Python 3 环境下调用失败。

#解决方案
修改 `tools/perf/scripts/python/exported-sql-viewer.py` 文件，在进行字符转换前增加 Python 版本判断：若版本为 Python 3，调用 `str.maketrans()`；否则保持使用 `string.maketrans()`，从而兼容不同版本的 Python 解释器。
