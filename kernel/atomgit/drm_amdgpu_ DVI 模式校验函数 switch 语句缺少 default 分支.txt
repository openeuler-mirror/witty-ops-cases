#标题
drm/amdgpu: DVI 模式校验函数 switch 语句缺少 default 分支

#内核版本
OLK-6.6

#问题现象
静态分析工具（smatch）报错，指出 `amdgpu_connector_dvi_mode_valid` 函数中的 switch 语句未处理所有可能的枚举值（non-exhaustive）。虽然该代码路径在正常操作中不应被触发，但缺少默认分支可能导致潜在的逻辑缺陷风险。

#主要错误日志
未知

#问题根因
函数 `amdgpu_connector_dvi_mode_valid()` 根据连接器类型（connector_object_id）赋值 max_digital_pixel_clock_khz 时，switch 语句缺少 default 分支。虽然现有分支已覆盖该函数所涉及的所有数字连接器类型，但为了代码完备性及消除静态分析工具的告警，需要增加默认处理分支。

#解决方案
在 `drivers/gpu/drm/amd/amdgpu/amdgpu_connectors.c` 文件的 `amdgpu_connector_dvi_mode_valid` 函数 switch 语句中添加 default 分支。该分支直接返回 MODE_BAD，作为防御性编程措施，确保 switch 语句覆盖所有情况并消除 smatch 静态分析错误。
