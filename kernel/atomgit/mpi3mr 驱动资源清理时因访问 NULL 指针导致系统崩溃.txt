#标题
mpi3mr 驱动资源清理时因访问 NULL 指针导致系统崩溃

#内核版本
OLK-6.6

#问题现象
在 mpi3mr 驱动执行资源清理过程中，若 reply 或 request 队列创建失败，驱动会先释放内存，随后尝试对已释放的内存执行 memset 操作，导致访问 NULL 指针从而引发系统崩溃。

#主要错误日志
未知

#问题根因
当 reply 或 request 队列创建失败时，驱动程序在资源清理阶段先释放了队列内存，但随后仍尝试访问该已释放内存的内容进行清零操作。由于指针已被置空，导致访问空指针引发系统崩溃。

#解决方案
在 `mpi3mr_memset_buffers` 函数中，在访问 `op_reply_qinfo` 和 `req_qinfo` 数组成员进行清零操作前，增加对这两个指针的 NULL 检查。若指针为 NULL 则跳过对该队列的处理，避免解引用空指针。
