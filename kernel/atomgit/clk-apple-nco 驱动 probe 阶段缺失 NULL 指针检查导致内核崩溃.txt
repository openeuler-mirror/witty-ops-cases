#标题
clk-apple-nco 驱动 probe 阶段缺失 NULL 指针检查导致内核崩溃

#内核版本
OLK-6.6

#问题现象
在加载 Apple NCO 时钟驱动过程中，若内存分配失败，将引发内核 NULL 指针引用错误，导致驱动加载失败或内核 panic。

#主要错误日志
Unable to handle kernel NULL pointer dereference

#问题根因
在 `applnco_probe` 函数中，调用 `devm_kasprintf` 为 `init.name` 赋值后，未对该返回值进行 NULL 指针检查。当系统内存不足导致 `devm_kasprintf` 分配失败返回 NULL 时，后续代码直接引用了该 NULL 指针。

#解决方案
在 `applnco_probe` 函数中增加对 `init.name` 有效性的判断，若 `devm_kasprintf` 返回 NULL，则函数直接返回 `-ENOMEM`，避免后续访问空指针。
