#标题
virtio_vdpa: 修复 void 函数中存在误导性的 return 语句

#内核版本
OLK-6.6

#问题现象
在代码审计或静态检查中，发现 `virtio_vdpa_set_status()` 函数定义的返回值类型为 `void`，但函数实现中却使用了 `return vdpa_set_status(...)` 语句。这在语法上虽然合法（C 标准允许在 void 函数中使用 `return` 后跟 void 类型的表达式），但在语义上不仅多余，而且容易误导代码阅读者认为该函数可能返回一个值，属于代码逻辑不规范问题。

#主要错误日志
未知

#问题根因
`virtio_vdpa_set_status()` 函数被声明为返回 `void` 类型，但其实现中调用了同样返回 `void` 的 `vdpa_set_status()` 函数，并错误地添加了 `return` 关键字。该 `return` 语句是不必要的，且对开发者具有误导性。

#解决方案
删除 `virtio_vdpa_set_status()` 函数中调用 `vdpa_set_status()` 前的 `return` 关键字，使其符合常规的 void 函数编码风格。修改后的代码直接调用函数而不进行返回。
