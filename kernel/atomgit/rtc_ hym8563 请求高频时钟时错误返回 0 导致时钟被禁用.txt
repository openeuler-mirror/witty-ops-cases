#标题
rtc: hym8563 请求高频时钟时错误返回 0 导致时钟被禁用

#内核版本
OLK-6.6

#问题现象
当调用 `hym8563_clkout_round_rate()` 函数请求一个高于设备支持的最高频率的时钟速率时，函数错误地返回 0，导致时钟被意外禁用，违反了 clk API 应返回最接近支持频率的预期行为。

#主要错误日志
未知

#问题根因
函数 `hym8563_clkout_round_rate()` 在遍历支持频率列表时，若请求频率高于所有支持的频率（即未找到小于等于请求频率的项），原有逻辑直接返回 0。这导致了时钟被禁用，而非返回硬件支持的最高频率。

#解决方案
修改 `hym8563_clkout_round_rate()` 函数的返回逻辑，当请求频率超过最大支持频率时，返回 `clkout_rates[0]`（即支持的最高频率），而不是返回 0，以此符合 clk API 规范。
