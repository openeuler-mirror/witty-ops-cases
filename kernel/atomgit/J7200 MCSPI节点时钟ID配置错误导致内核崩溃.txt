#标题
J7200 MCSPI节点时钟ID配置错误导致内核崩溃

#内核版本
OLK-6.6

#问题现象
在 TI J7200 平台上启用 MCSPI 实例时触发内核崩溃（kernel crash）。

#主要错误日志
未知

#问题根因
设备树文件 `k3-j7200-main.dtsi` 和 `k3-j7200-mcu-wakeup.dtsi` 中多个 MCSPI 实例的时钟 ID 配置不正确，与硬件文档定义不符。错误的时钟 ID 导致驱动在使能这些 SPI 设备时发生崩溃。

#解决方案
根据 TI 官方文档修正设备树中 MCSPI 节点的时钟 ID：
1. 将 main_spi0 ~ main_spi3、main_spi5 ~ main_spi7 以及 mcu_spi0、mcu_spi1 的时钟 ID 从错误的值（0 或 1）修正为 4。
2. 将 main_spi4 和 mcu_spi2 的时钟 ID 修正为 2。
