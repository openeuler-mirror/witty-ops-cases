#标题
bpf_sched_set_task_prefer_nid 设置任务亲和性未与进程允许 CPU 掩码取交集导致异常

#内核版本
OLK-6.6

#问题现象
调用 `bpf_sched_set_task_prefer_nid` 接口设置任务偏好 NUMA 节点时，如果传入的 nid 对应的 CPU 掩码不完全包含在当前进程允许运行的 CPU 掩码（`cpus_ptr`）中，可能导致设置结果不符合预期或后续调度行为异常。

#主要错误日志
未知

#问题根因
函数 `bpf_sched_set_task_prefer_nid` 在设置任务偏好 CPU 时，直接使用了传入参数 `nid` 对应的节点 CPU 掩码 `cpumask_of_node(nid)`，未将其与任务自身允许的 CPU 掩码 `task->cpus_ptr` 进行交集运算。这导致设置的偏好 CPU 可能超出了任务实际允许运行的范围。

#解决方案
在调用 `set_prefer_cpus_ptr` 之前，引入本地变量 `mask`，通过 `cpumask_and` 函数将目标节点的 CPU 掩码与任务自身的 `cpus_ptr` 进行交集运算，确保最终设置的偏好 CPU 既符合指定的 NUMA 节点要求，又不超出任务允许的 CPU 范围。
