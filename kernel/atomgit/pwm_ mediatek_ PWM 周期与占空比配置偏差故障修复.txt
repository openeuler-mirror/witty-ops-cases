#标题
pwm: mediatek: PWM 周期与占空比配置偏差故障修复

#内核版本
OLK-6.6

#问题现象
MediaTek PWM 控制器输出的实际波形与预期配置不符。使用逻辑分析仪或示波器测量会发现，硬件生成的周期和占空比均略高于软件设定的目标值。具体表现为：当驱动请求特定周期时，硬件实际输出的周期比请求值稍长；占空比设置也存在同样的偏差。

#主要错误日志
未知

#问题根因
根据硬件文档与实测分析，MediaTek PWM 硬件生成周期的计算公式为：`((PWMDWIDTH + 1) << CLKDIV) / freq`。
原有驱动代码在计算寄存器值 `cnt_period` 和 `cnt_duty` 时，未考虑硬件逻辑中隐含的 `+1` 操作。驱动直接将计算出的计数值写入寄存器，导致硬件实际产生的周期和占空比均比预期值偏大。

#解决方案
修正 PWM 周期与占空比的寄存器写入逻辑：
1. **修正寄存器值**：在写入 `reg_width`（周期）和 `reg_thres`（占空比）寄存器时，将计算值减 1（即写入 `cnt_period - 1` 和 `cnt_duty - 1`），以抵消硬件内部的 `+1` 逻辑。
2. **边界处理**：增加对 `cnt_period` 为 0 的校验，若为 0 则直接返回 `-EINVAL`，避免产生非预期的极短周期波形。
3. **占空比为 0 处理**：当占空比 `cnt_duty` 为 0 时，直接禁用 PWM 输出，确保输出恒定低电平。
