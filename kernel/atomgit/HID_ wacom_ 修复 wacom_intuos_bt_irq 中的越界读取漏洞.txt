#标题
HID: wacom: 修复 wacom_intuos_bt_irq 中的越界读取漏洞

#内核版本
OLK-6.6

#问题现象
Wacom 数位板在处理蓝牙 HID 报告时，若收到恶意构造的超短数据包（Report ID 0x03 或 0x04），会触发内核越界读取。该缺陷可能导致内核内存信息泄露或内核崩溃。

#主要错误日志
- Report 0x04 too short: %zu bytes\n
- Report 0x03 too short: %zu bytes\n

#问题根因
`wacom_intuos_bt_irq()` 函数在处理蓝牙 HID 报告时缺乏充分的长度校验。具体而言，报告 ID 0x03 至少需要 22 字节才能安全读取处理后数据和电池状态，而报告 ID 0x04（向下穿透至 0x03 处理逻辑）需要 32 字节。当收到长度不足的恶意报文时，直接进行数据拷贝操作导致了越界读取。

#解决方案
在 `wacom_intuos_bt_irq` 函数中增加对报告长度的显式检查。对于 Report 0x04，检查长度是否小于 32 字节；对于 Report 0x03，检查长度是否小于 22 字节。如果检测到短报告，则打印警告日志并终止处理流程，从而避免越界访问。
