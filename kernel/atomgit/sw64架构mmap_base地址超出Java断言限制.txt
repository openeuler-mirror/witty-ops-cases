#标题
sw64架构mmap_base地址超出Java断言限制

#内核版本
OLK-6.6

#问题现象
在sw64架构下，Java动态库映射的mmap_base地址超过了Java设定的断言地址（0x7fff7ff8000），导致Java程序运行过程中触发地址断言失败，影响Java应用的正常加载与执行。

#主要错误日志
未知

#问题根因
根据修复补丁及关联提交描述，先前的修复提交 e67b665d0354 ("sw64: fix random mmap base range") 调整了mmap基址范围，导致 `DEFAULT_MAP_WINDOW` 设置过大，使得Java动态库映射的地址超出了Java虚拟机预期的合法断言地址范围。

#解决方案
修改 `arch/sw_64/include/asm/processor.h` 文件中的 `DEFAULT_MAP_WINDOW` 定义，在原值 `DEFAULT_MAP_WINDOW_64` 的基础上减去 8GB（SZ_8G），缩小映射窗口范围，确保mmap_base地址落在Java断言地址之内。
