#标题
imx8mq 设备树中 GPU AHB 时钟频率配置错误

#内核版本
OLK-6.6

#问题现象
i.MX 8M Quad 平台 GPU 驱动初始化或运行过程中，GPU_AHB_CLK_ROOT 时钟频率被错误地设置为 800MHz，超过了硬件手册规定的最大值 400MHz。这可能导致 GPU 子系统运行不稳定、硬件功能异常或在特定负载下触发硬件故障。

#主要错误日志
未知

#问题根因
设备树源文件 `imx8mq.dtsi` 中 `assigned-clock-rates` 属性将 GPU AHB 时钟频率错误配置为 800MHz。根据 i.MX 8M Quad Reference Manual，GPU_AHB_CLK_ROOT 的最大支持频率仅为 400MHz。

#解决方案
修改 `arch/arm64/boot/dts/freescale/imx8mq.dtsi` 文件中 GPU 节点的 `assigned-clock-rates` 属性，将第五个时钟频率参数（对应 GPU_AHB 时钟）从 800000000 (800MHz) 修正为 400000000 (400MHz)，使其符合硬件规格。
