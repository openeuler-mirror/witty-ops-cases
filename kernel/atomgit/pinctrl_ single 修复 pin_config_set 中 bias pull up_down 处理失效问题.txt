#标题
pinctrl: single 修复 pin_config_set 中 bias pull up/down 处理失效问题

#内核版本
OLK-6.6

#问题现象
在配置引脚的上下拉（PIN_CONFIG_BIAS_PULL_DOWN 或 PIN_CONFIG_BIAS_PULL_UP）时，驱动虽然执行了清除原有 bias 配置的操作，但随后写入寄存器的数据仍为旧值，导致清除操作被覆盖，引脚的 bias 配置未能正确生效。

#主要错误日志
未知

#问题根因
在 `pin_config_set` 函数处理 `PIN_CONFIG_BIAS_PULL_DOWN` 或 `PIN_CONFIG_BIAS_PULL_UP` 时，代码调用了 `pcs_pinconf_clear_bias()` 来写寄存器清除旧配置。然而，后续操作继续使用的是调用 `pcs_pinconf_clear_bias()` 之前缓存的旧 `data` 值进行计算和写入，这导致刚清除的 bias 配置被旧数据覆盖，使得清除操作实际上没有生效。

#解决方案
在调用 `pcs_pinconf_clear_bias()` 清除 bias 后，立即重新从寄存器读取最新的 `data` 值，确保后续的配置操作基于最新的寄存器状态进行，从而避免覆盖清除结果。
