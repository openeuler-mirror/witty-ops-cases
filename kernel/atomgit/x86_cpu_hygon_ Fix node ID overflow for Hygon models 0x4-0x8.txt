#标题
x86/cpu/hygon: Fix node ID overflow for Hygon models 0x4-0x8

#内核版本
OLK-6.6

#问题现象
Hygon 处理器型号（model）在 0x4 到 0x8 范围内的 CPU，在初始化过程中可能出现 node ID 溢出问题，导致 NUMA 节点分配错误或拓扑信息异常。

#主要错误日志
未知

#问题根因
在 `srat_detect_node` 函数中，当 `numa_cpu_node` 返回 `NUMA_NO_NODE` 时，原代码逻辑直接使用 `c->topo.llc_id` 作为节点 ID。对于 Hygon 型号 0x4-0x8 的处理器，该逻辑错误导致了 node ID 溢出。

#解决方案
针对 Hygon 型号 0x4 到 0x8 的处理器，修改 `srat_detect_node` 逻辑：当 `numa_cpu_node` 返回 `NUMA_NO_NODE` 时，不再使用 `llc_id`，而是通过 `cpu_to_node(cpu)` 获取正确的节点 ID，并调用 `numa_set_node` 进行设置；其他型号处理器保持原有逻辑不变。
