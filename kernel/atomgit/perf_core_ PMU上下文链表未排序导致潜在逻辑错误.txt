#标题
perf/core: PMU上下文链表未排序导致潜在逻辑错误

#内核版本
OLK-6.6

#问题现象
内核代码中 `ctx->pmu_ctx_list` 链表未能保证按 PMU type 排序，可能触发内核警告或导致后续遍历逻辑异常。

#主要错误日志
未知

#问题根因
在 `find_get_pmu_context` 函数中，当向 `pmu_ctx_list` 添加新的 `perf_event_pmu_context` 节点时，原代码直接调用 `list_add` 将新节点添加到链表头部，未根据 PMU 的 type 值进行排序插入。这破坏了链表预期的有序性，可能导致内核发出关于 unordered pmu_ctx_list 的警告。

#解决方案
修改 `find_get_pmu_context` 函数逻辑，在遍历链表查找现有节点的同时，根据 PMU type 找到合适的插入位置（变量 `pos`）。若找到合适位置，使用 `list_add` 将新节点插入到该位置之前；若遍历结束未找到（即新节点 type 最大），则使用 `list_add_tail` 将其添加到链表尾部，从而确保 `pmu_ctx_list` 始终按 PMU type 值有序排列。
