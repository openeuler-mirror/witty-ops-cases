#标题
net/oenetcls: RPS CPU索引选择存在分布偏差

#内核版本
OLK-6.6

#问题现象
在使用 NUMA 或 cluster 策略进行 RPS（Receive Packet Steering）CPU 选择时，最后一个候选 CPU 无法被选中，导致 CPU 负载分布不均，出现持续的流量分配偏差。

#主要错误日志
未知

#问题根因
函数 `get_rps_cpu()` 在调用 `reciprocal_scale()` 计算 CPU 索引时，传入的参数为 `cpus_num - 1`。这导致哈希计算的范围排除了最后一个 bucket，使得候选列表中的最后一个 CPU 永远不会被选中，从而引入了持续的分布偏差。

#解决方案
修正 `reciprocal_scale()` 的调用参数，直接使用完整的候选 CPU 计数值（如 `rps_cpus_nums` 和 `cluster_rps_cpus_nums`），确保所有候选 CPU 都能被均匀选中。
