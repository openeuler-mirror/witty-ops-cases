#标题
spi: zynqmp-gqspi 驱动退出时未正确撤销 runtime PM 配置

#内核版本
OLK-6.6

#问题现象
zynqmp-gqspi 驱动在移除（remove）或初始化失败（probe error path）时，未撤销 `pm_runtime_use_autosuspend()` 的设置，可能导致电源管理状态机异常或资源释放不完全，影响设备后续重新加载或系统电源管理行为。

#主要错误日志
未知

#问题根因
驱动在初始化（probe）时调用了 `pm_runtime_use_autosuspend()` 启用了自动挂起功能，但在驱动退出（remove）及初始化失败的错误处理路径中，遗漏了与之配对的 `pm_runtime_dont_use_autosuspend()` 调用，导致运行时 PM 配置未被正确还原。

#解决方案
在驱动的移除函数 `zynqmp_qspi_remove` 以及初始化失败的错误处理标签 `clk_dis_all` 处，增加调用 `pm_runtime_dont_use_autosuspend(&pdev->dev)`，确保在驱动退出时正确撤销 runtime PM 的自动挂起设置。
