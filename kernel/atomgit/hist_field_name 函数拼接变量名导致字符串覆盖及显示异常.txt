#标题
hist_field_name 函数拼接变量名导致字符串覆盖及显示异常

#内核版本
OLK-6.6

#问题现象
在内核追踪系统中使用 hist 触发器功能时，若涉及跨系统的变量引用（VAR_REF 且带有 system 字段），会导致变量名拼接逻辑错误：
1. 使用 `strcat` 向静态缓冲区循环拼接字符串，导致缓冲区内容无法清空，后续生成的变量名会追加在上一次内容的末尾，造成名称错误。
2. 变量名显示不符合预期，对于跨系统的变量引用，错误地添加了 `$` 前缀，应显示为 `system.event.$name` 或 `system.event.name` 格式，实际显示混乱。

#主要错误日志
未知

#问题根因
`hist_field_name` 函数处理带 `system` 字段的变量引用时存在两个缺陷：
1. 使用 `strcat` 向静态缓冲区 `full_name` 拼接字符串，但未在拼接前清空缓冲区，导致多次调用时字符串不断追加，产生错误的拼接结果。
2. 格式化逻辑与显示逻辑不一致，对于带有 `system` 的 `VAR_REF` 类型字段，未正确处理 `$` 前缀的打印格式。

#解决方案
1. 将 `strcat` 多次拼接改为 `snprintf` 一次性格式化输出，确保每次调用时静态缓冲区都被正确覆写，并根据字段类型选择正确的格式字符串（`%s.%s.$%s` 或 `%s.%s.%s`）。
2. 增加 `snprintf` 返回值检查，防止缓冲区溢出。
3. 修正 `expr_field_str` 和 `hist_field_print` 函数的逻辑，仅当字段不包含 `system` 字段时才添加 `$` 前缀，保证跨系统变量引用的显示一致性。
