#标题
DTS 属性语法错误导致 OMAP3 设备树校验失败

#内核版本
OLK-6.6

#问题现象
在使用 `dtbs_check` 对设备树文件进行校验时，出现验证错误提示 "size (4) error for type flag"，导致设备树编译或校验流程无法通过。

#主要错误日志
size (4) error for type flag

#问题根因
`ti,keep-vref-on` 属性在设备树模式中定义为布尔类型，但在 DTS 源文件中错误地赋值为 `<1>`。布尔类型属性不应携带具体数值，这种语法不匹配导致了校验工具报错。

#解决方案
修改 DTS 文件中的属性定义，移除 `ti,keep-vref-on` 后面的赋值部分 `<1>`，将其修改为无值的布尔标记形式（即 `ti,keep-vref-on;`），以符合设备树模式定义并配合驱动程序中的 `device_property_read_bool()` 接口正确解析。
