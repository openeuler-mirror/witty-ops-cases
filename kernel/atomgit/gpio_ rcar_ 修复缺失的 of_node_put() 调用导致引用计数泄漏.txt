#标题
gpio: rcar: 修复缺失的 of_node_put() 调用导致引用计数泄漏

#内核版本
OLK-6.6

#问题现象
Renesas R-Car GPIO 驱动在解析设备树节点时，调用 `of_parse_phandle_with_fixed_args()` 后未释放设备节点引用。这将导致内核设备节点引用计数泄漏，可能引发内存泄漏或设备卸载异常等问题。

#主要错误日志
未知

#问题根因
根据 Linux 内核 API 规范，函数 `of_parse_phandle_with_fixed_args()` 要求调用者在处理完成后，必须对输出结构中的节点指针调用 `of_node_put()` 以释放引用。原代码逻辑中仅判断了返回值并提取参数，遗漏了 `of_node_put()` 调用，导致节点引用计数无法正确归零。

#解决方案
在驱动代码 `drivers/gpio/gpio-rcar.c` 的 `gpio_rcar_parse_dt` 函数中，针对 `of_parse_phandle_with_fixed_args()` 调用成功的分支，补充调用 `of_node_put(args.np)`。通过调整代码逻辑结构，确保在解析成功并获取 pin 数量后，正确释放设备节点引用。
