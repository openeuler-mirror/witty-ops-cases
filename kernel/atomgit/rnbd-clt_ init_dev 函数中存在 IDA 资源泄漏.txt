#标题
rnbd-clt: init_dev 函数中存在 IDA 资源泄漏

#内核版本
OLK-6.6

#问题现象
在 RNBD 客户端初始化设备（init_dev）过程中，若 kstrdup() 调用失败（例如内存不足），此前分配的 IDA ID 将不会被释放，导致 IDA 资源泄漏。长期运行或频繁触发该错误路径可能导致 ID 资源耗尽，进而导致后续设备初始化失败。

#主要错误日志
pr_err("Failed to initialize device '%s' from session %s, allocating idr failed, err: %d\n", pathname, sess->sessname, ret);

#问题根因
在 `init_dev` 函数中，代码先通过 `ida_alloc_max` 分配了一个 ID 并保存在局部变量 `ret` 中。随后调用 `kstrdup` 复制路径名，如果 `kstrdup` 失败，代码跳转到 `out_queues` 标签进行错误处理并返回。然而，`out_queues` 处理路径中没有调用 `ida_free` 来释放刚才成功分配的 ID，导致该 ID 泄漏。

#解决方案
修改错误处理路径。在 `ida_alloc_max` 成功后直接将 ID 赋值给 `dev->clt_device_id`。当后续 `kstrdup` 失败时，跳转到新增的 `out_ida` 标签，显式调用 `ida_free(&index_ida, dev->clt_device_id)` 释放已分配的 ID，然后再跳转到 `out_queues` 继续其他清理工作。
