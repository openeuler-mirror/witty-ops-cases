#标题
mtd: rawnand: fsmc: DMA映射失败未检查导致潜在内存访问错误

#内核版本
OLK-6.6

#问题现象
在使用 FSMC NAND 控制器进行 DMA 数据传输时，若 `dma_map_single` 映射失败（例如设备 DMA 映射错误），系统未对映射结果进行校验，直接使用无效的 DMA 地址进行数据传输。这可能导致内核发生不可预测的行为、内存损坏或数据传输错误，严重时可能触发 kernel panic 或 Oops。

#主要错误日志
未知

#问题根因
驱动程序 `drivers/mtd/nand/raw/fsmc_nand.c` 中的 `dma_xfer` 函数在进行 DMA 映射时，调用了 `dma_map_single` 函数，但未检查其返回值是否为错误状态（`dma_mapping_error`）。根据内核 API 规范，DMA 映射函数可能失败，缺乏错误检查会导致驱动程序在映射失败时继续执行后续的 DMA 传输操作，使用了无效的 DMA 地址。

#解决方案
在 `dma_map_single` 调用之后，立即添加 `dma_mapping_error` 检查逻辑。如果检测到映射错误，函数应立即返回 `-EINVAL`，从而中止当前的传输操作并向上层报告错误，避免使用无效地址进行 DMA 操作。
