#标题
urma: 移除ubcore_get_route_list中的冗余回环路由处理逻辑

#内核版本
OLK-6.6

#问题现象
在 loopback 场景下调用 `ubcore_get_route_list` 获取路由列表时，内核可能输出错误日志 "Failed to query primary id in loopback mode"，导致路由查询流程异常或返回非预期的结果。该现象在特定配置下的回环通信初始化阶段可观测到。

#主要错误日志
- "Failed to query primary id in loopback mode, ret: %d.\n" （推断：由被移除的错误分支打印）

#问题根因
在 `ubcore_get_route_list` 函数中，存在一段针对 loopback 场景（源 EID 等于目的 EID）的独立处理逻辑。该逻辑调用 `ubcore_get_route_loopback_primary_eid` 并直接返回。根据补丁行为（删除该逻辑并统一走后续的标准流程），该 loopback 判断分支属于冗余或错误实现，导致了回环场景下的逻辑错误或功能异常。

#解决方案
删除 `ubcore_get_route_list` 函数中对源 EID 和目的 EID 是否相等的判断逻辑（`ubcore_eid_equals`），移除针对 loopback 模式的特殊处理代码块。修改后，loopback 场景与其他场景统一通过 `ubcore_get_route_primary_eid` 获取路由，确保逻辑一致性。
