#标题
t7xx: t7xx_dpmaif_rx_buf_alloc函数存在差一错误导致内存泄漏

#内核版本
OLK-6.6

#问题现象
在 t7xx_dpmaif_rx_buf_alloc 函数分配缓冲区的错误处理路径中，存在内存泄漏问题。当缓冲区分配过程中发生错误需要回滚时，循环释放逻辑未能正确释放索引为 0 的第一个 skb 结构，导致该内存块未被释放和取消映射。

#主要错误日志
未知

#问题根因
错误处理路径中的循环条件 `while (--i > 0)` 存在差一错误。该条件在 i 自减后判断是否大于 0，导致当 i 为 1 时，先自减为 0，判断 0 > 0 为假，循环直接退出。这使得索引为 0 的首个已分配 skb 未被释放，从而造成内存泄漏。

#解决方案
修改循环条件，将 `while (--i > 0)` 改为 `while (i--)`。修改后，变量 i 在判断前先进行自减操作，确保索引 0 的情况能进入循环体执行释放操作，从而修正了内存释放遗漏的问题。
