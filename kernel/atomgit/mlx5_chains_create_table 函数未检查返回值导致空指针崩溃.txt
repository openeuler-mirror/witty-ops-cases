#标题
mlx5_chains_create_table 函数未检查返回值导致空指针崩溃

#内核版本
OLK-6.6

#问题现象
在调用 mlx5_chains_create_table() 函数时，若 mlx5_get_fdb_sub_ns() 或 mlx5_get_flow_namespace() 获取失败（返回 NULL），程序继续执行后续逻辑会导致 NULL 指针引用，从而引发内核崩溃。

#主要错误日志
mlx5_core_warn(chains->dev, "Failed to get flow namespace\n");

#问题根因
函数 mlx5_chains_create_table() 中未对 mlx5_get_fdb_sub_ns() 和 mlx5_get_flow_namespace() 的返回值进行有效性检查。当这些函数返回 NULL 时，代码未做处理直接使用该指针，导致了空指针引用风险。

#解决方案
在获取 flow namespace 后增加判断逻辑：检查指针 `ns` 是否为空。如果为空，则调用 mlx5_core_warn() 打印警告日志，并返回 ERR_PTR(-EOPNOTSUPP) 错误指针，阻止后续代码使用无效指针。
