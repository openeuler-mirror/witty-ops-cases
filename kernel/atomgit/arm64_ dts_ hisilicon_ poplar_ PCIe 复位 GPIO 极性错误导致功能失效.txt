#标题
arm64: dts: hisilicon: poplar: PCIe 复位 GPIO 极性错误导致功能失效

#内核版本
OLK-6.6

#问题现象
Poplar 开发板上的 PCIe 功能失效（Commit 描述为 "PCIe on Poplar got broken"）。该问题在内核驱动切换使用 gpiod API 后开始显现，在此之前因驱动代码未严格遵循设备树（DT）配置，错误的 GPIO 极性配置未导致功能异常。

#主要错误日志
未知

#问题根因
设备树中 PCIe 复位 GPIO 的极性定义错误，被错误地定义为 `GPIO_ACTIVE_HIGH`，而硬件实际为低电平有效（active low）。此前内核驱动未遵守设备树中的极性设置，掩盖了该错误；在引入 commit 1d26a55fbeb9 ("PCI: histb: Switch to using gpiod API") 后，驱动开始严格遵循设备树配置，导致复位信号时序错误，PCIe 无法正常工作。

#解决方案
修正设备树文件 `arch/arm64/boot/dts/hisilicon/hi3798cv200-poplar.dts` 中的 PCIe 复位 GPIO 属性，将 `reset-gpios` 的极性标志由 `GPIO_ACTIVE_HIGH` 修改为 `GPIO_ACTIVE_LOW`，以匹配硬件的实际电气特性。
