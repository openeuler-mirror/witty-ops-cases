#标题
bpf: Fix NULL deref in map_kptr_match_type for scalar regs

#内核版本
OLK-6.6

#问题现象
在 BPF 程序中将 scalar register（标量寄存器）存储到 kptr slot 时，内核发生 NULL 指针解引用，导致系统崩溃或 verifier 异常。

#主要错误日志
未知

#问题根因
Commit ab6c637ad027 重构 map_kptr_match_type() 时，先执行 btf_is_kernel() 分支判断，再检查 base_type()。当向 kptr slot 存入 scalar register 时，该寄存器没有 btf 信息（reg->btf 为 NULL），导致 btf_is_kernel(reg->btf) 调用时引发 NULL 指针解引用。

#解决方案
调整代码逻辑顺序，在访问 reg->btf 之前，先通过 base_type(reg->type) != PTR_TO_BTF_ID 检查来过滤非 BTF ID 指针类型，防止对 scalar register 进行 btf 相关操作。
