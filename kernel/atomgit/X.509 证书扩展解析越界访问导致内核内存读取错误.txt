#标题
X.509 证书扩展解析越界访问导致内核内存读取错误

#内核版本
OLK-6.6

#问题现象
非特权用户通过 keyrings API 提交包含空 Basic Constraints 或 Key Usage 扩展字段的特制 X.509 证书时，触发内核越界访问。该漏洞可能导致内核崩溃或内存信息泄露，且可被普通用户利用。

#主要错误日志
未知

#问题根因
在解析 X.509 证书的 Basic Constraints 或 Key Usage 扩展时，代码逻辑先读取了扩展数据的第一个字节 `v[0]`，之后才检查数据长度 `vlen` 是否合规。若提交的证书扩展内容为空（长度不足），该读取操作将导致越界访问。

#解决方案
调整代码执行顺序，在进行任何数据读取操作前，优先检查扩展数据的长度 `vlen`。若长度小于最小要求（如 Key Usage 需大于等于 4，Basic Constraints 需大于等于 2），直接返回 `-EBADMSG` 错误，从而避免越界读取。
