#标题
powercap: dtpm_cpu: Fix NULL pointer dereference in get_pd_power_uw()

#内核版本
OLK-6.6

#问题现象
系统在运行过程中，当某个 CPU 在运行时变为 impossible 状态（如被热拔出或失效），此时若调用 `get_pd_power_uw()` 函数，会触发空指针解引用导致内核崩溃。

#主要错误日志
未知

#问题根因
在 `get_pd_power_uw()` 函数中，调用 `em_cpu_get()` 获取性能域指针时，因 CPU 状态变为 impossible 导致 `get_cpu_device()` 返回 NULL，进而导致 `em_cpu_get()` 返回 NULL。后续代码在未检查返回值有效性的情况下，调用 `em_span_cpus(pd)` 解引用了该 NULL 指针，引发崩溃。

#解决方案
在 `get_pd_power_uw()` 函数中增加对 `em_cpu_get()` 返回值的非空检查。若返回值为 NULL，则直接返回 0，该处理方式与已有的 `__dtpm_cpu_setup()` 函数中的降级行为保持一致。
