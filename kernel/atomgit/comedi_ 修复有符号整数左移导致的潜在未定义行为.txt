#标题
comedi: 修复有符号整数左移导致的潜在未定义行为

#内核版本
OLK-6.6

#问题现象
在 Comedi 子系统处理数字 I/O 配置或状态更新时，当通道号（channel number）达到 31 或特定条件下，对有符号常量 1 进行左移操作可能导致符号位被置位。这属于 C 语言未定义行为，可能引发内核逻辑错误、配置掩码计算异常或相关功能失效。

#主要错误日志
未知

#问题根因
代码中使用了有符号整数常量 `1` 进行左移操作（如 `1 << CR_CHAN(insn->chanspec)` 或 `1 << s->n_chan`）。当移位数值满足 `shift >= 31` 时，结果会将 1 移入符号位。根据 C 语言标准，对负数进行移位或导致符号位变化的有符号左移属于未定义行为。根因在于缺少类型限定，导致编译器将字面量 `1` 视为有符号整型。

#解决方案
将相关代码中的整型常量 `1` 修改为无符号整型常量 `1U`。具体修改点包括：
1. `comedi_dio_insn_config()` 函数中计算掩码时，将 `1 << chan` 改为 `1U << chan`。
2. `comedi_dio_update_state()` 函数中计算掩码时，将 `(1 << s->n_chan)` 改为 `(1U << s->n_chan)`。
3. `insn_rw_emulate_bits()` 函数中构造掩码和位数据时，将 `1 << ...` 改为 `1U << ...`。
4. `__comedi_device_postconfig()` 函数中初始化 `io_bits` 时，将 `(1 << s->n_chan)` 改为 `(1U << s->n_chan)`。
