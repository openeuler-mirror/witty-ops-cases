#标题
af_unix: MSG_PEEK 模式下读取 OOB 数据导致零字节读取循环

#内核版本
OLK-6.6

#问题现象
在 AF_UNIX 套接字通信中，当接收队列最后一个数据包为带外数据，且用户态程序使用 MSG_PEEK 标志尝试读取 0 字节数据时，内核处理逻辑未正确跳出循环，导致系统陷入无限读取循环。

#主要错误日志
未知

#问题根因
在 `unix_stream_read_generic` 函数中，处理 OOB 数据后检查 skb 指针是否为空时，原逻辑仅判断 `copied` 是否非零。当调用参数 `size` 为 0（请求读取 0 字节）时，`copied` 变量始终为 0，导致条件 `!skb && copied` 不成立。此时内核未执行 break 语句跳出读取循环，引发死循环。

#解决方案
修改 `net/unix/af_unix.c` 中的判断逻辑，在判断条件中增加对 `size` 为 0 的检查。将 `if (!skb && copied)` 修改为 `if (!skb && (copied || !size))`，确保在请求读取 0 字节且 OOB 处理导致 skb 为空时，能正确跳出循环结束读取。
