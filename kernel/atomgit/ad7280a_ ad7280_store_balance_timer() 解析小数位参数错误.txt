#标题
ad7280a: ad7280_store_balance_timer() 解析小数位参数错误

#内核版本
OLK-6.6

#问题现象
在 ad7280a 驱动中通过 sysfs 接口设置 balance timer 时，输入的小数值无法被正确解析。由于传给 `iio_str_to_fixpoint()` 函数的精度参数错误，导致用户期望的 3 位小数精度实际上未能正确转换，数值处理结果与预期不符。

#主要错误日志
未知

#问题根因
函数 `iio_str_to_fixpoint()` 的 API 定义较为反直觉：其 `fract_mult` 参数代表第一个小数位作为整数时的乘数。为了解析 3 位小数，参数应当传入 100，但代码中错误地传入了 1000，导致数值解析逻辑错误。

#解决方案
修改 `drivers/iio/adc/ad7280a.c` 文件中 `ad7280_store_balance_timer` 函数的调用参数，将 `iio_str_to_fixpoint(buf, 1000, &val, &val2)` 修正为 `iio_str_to_fixpoint(buf, 100, &val, &val2)`，以确保正确解析 3 位小数。
