#标题
DMI 条目类型枚举索引定义错误导致与 SMBIOS 规范不匹配

#内核版本
OLK-6.6

#问题现象
内核 DMI 子系统中 `enum dmi_entry_type` 的定义与 SMBIOS 规范不一致。缺少 Type 33（64-Bit Memory Error Information）以及 Type 43-46 的定义，导致该枚举列表中 Type 33 之后所有条目的索引值（Index）发生错位，与标准规范不符。这可能影响依赖这些特定 DMI 条目类型进行硬件信息解析、故障诊断或驱动匹配的功能，导致解析数据错误或功能异常。

#主要错误日志
未知

#问题根因
在 `include/linux/dmi.h` 文件的 `enum dmi_entry_type` 定义中，遗漏了 SMBIOS 规范中定义的 Type 33 条目（`64-Bit Memory Error Information`）。由于枚举类型自动递增的特性，缺失该条目导致后续所有条目的数值索引比规范定义的值少 1。同时，代码中也缺失了 Type 43 至 Type 46 的条目定义。

#解决方案
修改 `include/linux/dmi.h` 文件，在 `enum dmi_entry_type` 中补充缺失的条目定义：
1. 在 `DMI_ENTRY_SYSTEM_BOOT` 后添加 `DMI_ENTRY_64_MEM_ERROR`（对应 Type 33），修正后续条目的索引值。
2. 在列表末尾添加 `DMI_ENTRY_TPM_DEVICE`、`DMI_ENTRY_PROCESSOR_ADDITIONAL`、`DMI_ENTRY_FIRMWARE_INVENTORY` 和 `DMI_ENTRY_STRING_PROPERTY`（分别对应 Type 43-46），确保内核定义与 SMBIOS 规范完全对齐。
