#标题
perf/x86/intel: ARCH_PERFMON_NUM_COUNTER_LEAF 子叶检测位错误修复

#内核版本
OLK-6.6

#问题现象
在物理机上不可见。但在虚拟化环境中，当 VMM 仅枚举 CPUID Leaf 023H 的子叶 0（sub-leaf 0）时，由于内核错误地检查了子叶位图（EAX）的 bit 0 而非 bit 1，可能导致内核错误地认为子叶 1（Counter Mask枚举）可用，进而访问不可用的 CPUID 子叶，可能引发功能异常或性能监控能力探测错误。

#主要错误日志
未知

#问题根因
CPUID Leaf 023H（EAX）用于枚举有效子叶的掩码。内核在检测子叶 1（ARCH_PERFMON_NUM_COUNTER_LEAF，用于枚举计数器掩码）是否可用时，原代码错误地检查了 EAX 的 bit 0 (0x1)。根据架构定义，子叶 1 的有效性应由 bit 1 (0x2) 指示。代码使用了错误的宏 `ARCH_PERFMON_NUM_COUNTER_LEAF_BIT` (定义为 0x1) 进行位测试，导致判断逻辑错误。

#解决方案
修正子叶可用性检测逻辑，将检查位从 bit 0 修正为 bit 1。同时，引入 `union cpuid35_eax` 和 `union cpuid35_ebx` 联合体替代原有的宏定义，通过位域结构体（如 `eax.split.cntr_subleaf`）直接访问特定位，既修复了检测位错误，也提升了代码可读性与架构一致性。
