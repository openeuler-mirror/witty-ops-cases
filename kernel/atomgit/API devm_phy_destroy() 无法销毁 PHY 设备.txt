#标题
API devm_phy_destroy() 无法销毁 PHY 设备

#内核版本
OLK-6.6

#问题现象
调用托管式 API `devm_phy_destroy()` 销毁 PHY 设备时，函数执行未生效，PHY 设备无法被销毁。若调用该 API 的模块后续尝试重新申请或操作该 PHY 资源，可能会导致资源泄露或潜在的冲突。

#主要错误日志
未知

#问题根因
在 `devm_phy_destroy()` 函数实现中，错误地使用了 `devres_destroy()` 函数。根据内核托管资源机制，`devres_destroy()` 仅释放资源节点结构，并不会调用 `devm_phy_consume()` 回调函数来执行实际的 `phy_destroy()` 操作。这导致尽管托管资源被释放，但底层的 PHY 设备并未被真正销毁。

#解决方案
将 `devm_phy_destroy()` 函数中的 `devres_destroy()` 调用替换为 `devres_release()`。`devres_release()` 会正确调用传入的 `devm_phy_consume()` 回调函数，从而确保 `phy_destroy()` 被执行，实现 PHY 设备的有效销毁。
