#标题
modpost 中 do_input 函数遗漏 max bit 迭代导致输入设备模态别名生成不完整

#内核版本
OLK-6.6

#问题现象
在使用 modpost 工具生成内核模块依赖信息时，输入设备input设备的模态别名处理逻辑存在缺陷。具体表现为：当遍历输入设备的 bitmap 以生成模态别名时，代码遗漏了对最后一个有效位max bit的检查与处理。这导致生成的模态别名缺少了最大值对应的按键或事件代码，进而可能导致内核模块自动加载机制udev/mdev无法正确匹配并加载依赖特定 max bit 事件的硬件驱动，造成部分输入设备功能缺失或无法识别。

#主要错误日志
未知

#问题根因
scripts/mod/file2alias.c 文件中的 do_input 函数在遍历输入位图时，循环条件写成了 `i < max`，而正确的逻辑应当包含 max 值本身，即 `i <= max`。该逻辑错误源自 commit 1d8f430c15b3 ("[PATCH] Input: add modalias support")，导致在生成模态别名时，始终跳过了范围定义中的最大值那一位的检查。

#解决方案
修改 scripts/mod/file2alias.c 文件中 do_input 函数的循环条件，将 `for (i = min; i < max; i++)` 修改为 `for (i = min; i <= max; i++)`，确保循环能够正确遍历并处理从 min 到 max 包含边界值在内的所有位。
