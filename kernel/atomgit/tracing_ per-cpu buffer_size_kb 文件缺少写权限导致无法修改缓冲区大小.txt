#标题
tracing: per-cpu buffer_size_kb 文件缺少写权限导致无法修改缓冲区大小

#内核版本
OLK-6.6

#问题现象
在 tracefs 文件系统中，针对每个 CPU 的 `buffer_size_kb` 文件虽然支持写入操作以动态调整每 CPU 的环形缓冲区大小，但该文件被错误地设置为只读权限。这导致用户无法通过写入该文件来修改对应 CPU 的缓冲区大小，功能受限。

#主要错误日志
未知

#问题根因
在之前的修复提交（commit 21ccc9cd7211）中，为了安全考虑禁用了 tracefs 文件的“其他”权限位，但在处理 `buffer_size_kb` 文件时，错误地使用了 `TRACE_MODE_READ` 标志创建文件。由于该文件对应的功能逻辑（`tracing_entries_fops`）本身支持写入操作以改变缓冲区大小，因此文件权限设置与功能实现不匹配。

#解决方案
将 `tracing_init_tracefs_percpu` 函数中创建 `buffer_size_kb` 文件的权限标志从 `TRACE_MODE_READ` 修改为 `TRACE_MODE_WRITE`，确保文件具备写权限，从而允许用户正常调整每 CPU 的环形缓冲区大小。
