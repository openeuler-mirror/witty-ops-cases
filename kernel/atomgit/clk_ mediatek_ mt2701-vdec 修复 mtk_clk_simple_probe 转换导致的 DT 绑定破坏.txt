#标题
clk: mediatek: mt2701-vdec 修复 mtk_clk_simple_probe 转换导致的 DT 绑定破坏

#内核版本
OLK-6.6

#问题现象
MediaTek MT2701 视频解码器的时钟驱动在转换为 `mtk_clk_simple_probe` 后，破坏了设备树（DT）绑定。由于时钟 ID 计数从 1 开始而非从 0 开始，导致驱动初始化失败或设备功能异常，具体表现为时钟索引偏移错误。

#主要错误日志
未知

#问题根因
在执行 commit 973d1607d936 ("clk: mediatek: mt2701: use mtk_clk_simple_probe to simplify driver") 进行驱动简化转换时，忽略了该控制器时钟 ID 编号是从 1 开始的，而非从 0 开始。这导致最高索引（highest index）被意外减少了 1，从而破坏了原有的设备树绑定关系和时钟映射。

#解决方案
参照其他存在相同问题的驱动程序，在时钟数组索引 0 处添加一个虚拟时钟，以修正 ID 计数偏移问题。具体代码变更为在 `vdec_clks` 数组开头增加 `GATE_DUMMY(CLK_DUMMY, "vdec_dummy")`。
