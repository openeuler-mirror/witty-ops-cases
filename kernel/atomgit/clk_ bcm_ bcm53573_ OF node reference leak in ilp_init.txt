#标题
clk: bcm: bcm53573: OF node reference leak in ilp_init

#内核版本
OLK-6.6

#问题现象
驱动代码在 `bcm53573_ilp_init` 函数中存在 OF node 引用计数泄漏问题。该问题由 `of_get_parent()` 调用引起，导致内核对象引用计数未能正确释放，可能造成内存泄漏。

#主要错误日志
未知

#问题根因
Driver code is leaking OF node reference from of_get_parent() in bcm53573_ilp_init(). 使用 `of_get_parent()` 获取父节点会增加父节点的引用计数，但在后续代码中未进行相应的 `of_node_put()` 操作来释放引用。

#解决方案
修复补丁将 `syscon_node_to_regmap(of_get_parent(np))` 修改为 `syscon_node_to_regmap(np->parent)`。
直接通过 `np->parent` 访问父节点，而不是调用 `of_get_parent()`。因为在 `CLK_OF_DECLARE()` 触发的初始化过程中，父节点不会被释放，无需增加引用计数，从而避免了引用泄漏问题。
