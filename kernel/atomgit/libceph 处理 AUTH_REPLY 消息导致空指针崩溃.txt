#标题
libceph 处理 AUTH_REPLY 消息导致空指针崩溃

#内核版本
OLK-6.6

#问题现象
当收到 `CEPH_MSG_AUTH_REPLY` 类型的消息且该消息中 protocol 和 result 字段均为 0 时，若客户端处于协商状态（`ac->negotiating == true`）且当前协议号大于 0，内核将发生空指针解引用，导致系统崩溃。

#主要错误日志
未知

#问题根因
在函数 `ceph_handle_auth_reply` 中，处理逻辑存在漏洞：
1. 当收到的消息中 protocol 为 0 时，代码将 `ac->protocol` 设为 0，并将 `ac->ops` 设为 NULL。
2. 原有的条件判断 `if (ac->protocol != protocol)` 在此时结果为 false（因为两者均为 0），导致 `init_protocol()` 函数未被调用，`ac->ops` 依然保持 NULL。
3. 后续代码直接调用 `ac->ops->handle_reply()` 时，因 `ac->ops` 为 NULL 触发空指针解引用。

#解决方案
修改 `net/ceph/auth.c` 中的判断逻辑，将 `if (ac->protocol != protocol)` 改为 `if (!ac->protocol)`。这使得当消息中的协议号为 0 时，代码能正确识别为无效的认证协议并转入错误处理流程，从而避免后续对空指针的调用。
