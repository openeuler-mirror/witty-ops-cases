#标题
devm_usb_put_phy() 无法正确释放 USB PHY 资源

#内核版本
OLK-6.6

#问题现象
调用内核 API `devm_usb_put_phy()` 无法释放对应的 USB PHY 设备资源，导致资源泄露。该 API 无法完全撤销 `devm_usb_get_phy()` 的操作，破坏了托管资源获取与释放的配对逻辑。

#主要错误日志
未知

#问题根因
函数 `devm_usb_put_phy()` 内部错误地使用了 `devres_destroy()`。根据 API 设计注释，该函数应当调用 `usb_put_phy()` 来释放 PHY，但 `devres_destroy()` 仅销毁设备资源记录而不执行释放回调（release callback），导致实际上并未执行释放 PHY 的操作。

#解决方案
将 `devm_usb_put_phy()` 函数中的 `devres_destroy()` 调用替换为 `devres_release()`。`devres_release()` 会正确调用释放回调函数 `devm_usb_phy_release`，从而确保 `usb_put_phy()` 被执行，正确释放 PHY 资源。
