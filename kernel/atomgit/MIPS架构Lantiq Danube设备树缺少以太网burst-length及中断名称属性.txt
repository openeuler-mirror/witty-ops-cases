#标题
MIPS架构Lantiq Danube设备树缺少以太网burst-length及中断名称属性

#内核版本
OLK-6.6

#问题现象
内核设备树校验（dtbs_check）失败，以太网节点 `etop@e180000` 报告缺少必需属性。可能导致以太网驱动无法正确配置 DMA 突发传输参数或中断处理逻辑异常。

#主要错误日志
arch/mips/boot/dts/lantiq/danube_easy50712.dtb: etop@e180000 (lantiq,etop-xway): 'interrupt-names' is a required property
	from schema $id: http://devicetree.org/schemas/net/lantiq,etop-xway.yaml#
arch/mips/boot/dts/lantiq/danube_easy50712.dtb: etop@e180000 (lantiq,etop-xway): 'lantiq,tx-burst-length' is a required property
	from schema $id: http://devicetree.org/schemas/net/lantiq,etop-xway.yaml#
arch/mips/boot/dts/lantiq/danube_easy50712.dtb: etop@e180000 (lantiq,etop-xway): 'lantiq,rx-burst-length' is a required property
	from schema $id: http://devicetree.org/schemas/net/lantiq,etop-xway.yaml#

#问题根因
Lantiq Danube 设备树文件中，以太网节点缺失了 `interrupt-names` 属性以及 `lantiq,rx-burst-length`、`lantiq,tx-burst-length` 属性。根据 YAML schema 定义，这些属性是必需的，且上游提交 14d4e308e0aa 引入了 burst length 配置需求但当时未同步更新 DTS 文件。

#解决方案
修改 `arch/mips/boot/dts/lantiq/danube_easy50712.dts` 文件中的 `etop@e180000` 节点，补充缺失属性：
1. 新增 `interrupt-names = "tx", "rx";` 以匹配中断号。
2. 新增 `lantiq,rx-burst-length = <4>;` 配置接收突发长度。
3. 新增 `lantiq,tx-burst-length = <4>;` 配置发送突发长度。
