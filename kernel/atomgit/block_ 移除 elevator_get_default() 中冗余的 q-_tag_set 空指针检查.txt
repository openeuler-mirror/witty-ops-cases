#标题
block: 移除 elevator_get_default() 中冗余的 q->tag_set 空指针检查

#内核版本
OLK-6.6

#问题现象
内核代码中存在冗余的空指针检查逻辑，导致静态代码分析工具（static checkers）报错或产生混淆；该问题为代码逻辑缺陷，不影响运行时功能，但会导致代码检查工具误报。

#主要错误日志
未知

#问题根因
在 `elevator_get_default()` 和 `elv_support_iosched()` 函数中，代码检查了 `q->tag_set` 是否为非空（non-NULL），但实际上该指针在此上下文中不可能为空。这种冗余且不一致的空指针检查导致静态检查工具无法正确理解代码逻辑并报错。

#解决方案
移除 `block/elevator.c` 中 `elv_support_iosched()` 和 `elevator_get_default()` 函数里对 `q->tag_set` 的冗余 NULL 检查，直接访问 `q->tag_set->flags`，从而简化逻辑并消除静态检查工具的错误告警。
