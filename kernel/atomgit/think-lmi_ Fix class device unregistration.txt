#标题
think-lmi: Fix class device unregistration

#内核版本
OLK-6.6

#问题现象
在 think-lmi 驱动初始化失败或模块卸载时，可能出现设备注销异常。由于 `firmware_attributes_class` 下的设备没有唯一的 `dev_t`，错误的注销接口可能导致匹配到错误的设备进行销毁，引发潜在的内核状态不一致或崩溃。

#主要错误日志
未知

#问题根因
`firmware_attributes_class` 下的设备没有唯一的设备号 (`dev_t`)。代码中错误地使用了 `device_destroy()` 接口，该接口根据 `dev_t` 匹配设备，这可能导致匹配到系统中其他具有相同 `dev_t` (MKDEV(0, 0)) 的设备并错误地将其销毁，而不是销毁预期的 class 设备。

#解决方案
将 `device_destroy(&firmware_attributes_class, MKDEV(0, 0))` 替换为 `device_unregister(tlmi_priv.class_dev)`。直接使用指向具体设备的指针进行注销，避免了基于设备号匹配带来的不确定性，确保注销正确的设备对象。
