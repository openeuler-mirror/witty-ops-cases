#标题
regulator: rk808: OF node reference imbalance

#内核版本
OLK-6.6

#问题现象
在卸载（unbind）rk808 多功能设备（MFD）驱动并注销子设备时，由于 OF node 引用计数不平衡，可能导致内核崩溃或内存释放异常。

#主要错误日志
未知

#问题根因
驱动在 probe 阶段直接将父设备的 OF node 赋值给子设备，并设置了 `of_node_reused` 标志，但未增加 node 的引用计数。当 platform bus 代码在注销子设备时释放（drop）了该 node 的引用，导致引用计数失衡，进而引发错误。

#解决方案
使用辅助函数 `device_set_of_node_from_dev()` 来设置设备的 OF node。该函数在复制父设备 node 指针的同时会正确增加引用计数，从而平衡后续注销流程中释放的引用。
