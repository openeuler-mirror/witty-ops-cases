#标题
rtl8723bs 驱动解析 Beacon 帧时 ESR IE 越界读取导致内核崩溃

#内核版本
OLK-6.6

#问题现象
在处理含有 Extended Supported Rates (ESR) IE 的恶意或畸形 Beacon 帧时，若该 IE 位于帧缓冲区末尾，驱动程序在解析过程中会发生越界读取。这可能导致内核内存泄露，严重情况下直接触发内核 Panic/Oops，造成系统不可用。

#主要错误日志
未知

#问题根因
在 `OnBeacon` 函数处理 ESR IE 时，代码直接访问 `*(p + 1 + ielen)` 和 `*(p + 2 + ielen)` 指向的内存，未校验这些偏移地址是否超出了接收帧缓冲区的有效范围。当恶意构造的 Beacon 帧将 ESR IE 放置在帧末尾时，上述指针计算结果位于缓冲区之外，从而触发越界读取。

#解决方案
在访问 ESR IE 后续字节之前增加边界检查条件 `if (p + 2 + ielen < pframe + len)`，确保目标访问地址位于帧缓冲区范围内。若越界则跳过修正逻辑，从而防止 OOB（Out-of-Bounds）读取。
