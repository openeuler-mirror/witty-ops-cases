#标题
hisi_cpa_pmu event sysfs 接口位掩码限制错误

#内核版本
OLK-6.6

#问题现象
CPA PMU 驱动的 sysfs 接口中，event 事件的位掩码（bitmask）限制设置错误。原代码将 event 配置定义为 "config:0-15"，但实际硬件支持的事件位宽仅为 8 位，导致用户态工具可能配置超出硬件范围的事件值，引发功能异常或配置失效。

#主要错误日志
未知

#问题根因
驱动代码中 `hisi_cpa_pmu_format_attr` 数组对 event 属性的定义有误。原代码使用 `HISI_PMU_FORMAT_ATTR(event, "config:0-15")` 声明了 16 位的事件空间，这与硬件实际支持的 8 位事件宽度不符，导致掩码限制不正确。

#解决方案
修正 event 属性的位掩码定义，将 `hisi_cpa_pmu_format_attr` 中的 format 字符串从 "config:0-15" 修改为 "config:0-7"，使其与硬件实际规格一致。
