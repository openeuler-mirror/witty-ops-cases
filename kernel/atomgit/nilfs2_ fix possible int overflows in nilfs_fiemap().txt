#标题
nilfs2: fix possible int overflows in nilfs_fiemap()

#内核版本
OLK-6.6

#问题现象
在 nilfs2 文件系统上执行 fiemap 相关操作（如查询大文件 extent 信息）时，由于计算过程中的整数溢出，可能导致返回的 extent 长度或位置信息错误，进而引发文件系统信息查询结果不准确或用户态程序处理异常。

#主要错误日志
未知

#问题根因
在 `nilfs_fiemap` 函数中，变量 `n` 和 `blkbits` 均为整型。在进行移位操作 `n << blkbits` 时，若 `n` 值较大，运算结果会超出 int 类型的表示范围发生整数溢出，随后才被赋值给 `size` 变量（即使 `size` 可能是 64 位类型），导致计算出的 extent 大小不正确。

#解决方案
修改 `fs/nilfs2/inode.c` 文件，在移位操作前将变量 `n` 强制转换为 `u64` 类型（即 `(u64)n << blkbits`），确保整个表达式在 64 位空间进行运算，从而避免整数溢出，保证 extent 长度计算的正确性。
