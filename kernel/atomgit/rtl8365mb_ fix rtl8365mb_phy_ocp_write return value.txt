#标题
rtl8365mb: fix rtl8365mb_phy_ocp_write return value

#内核版本
OLK-6.6

#问题现象
函数 rtl8365mb_phy_ocp_write() 在寄存器访问过程中发生错误时，错误码未能正确传递给调用者，导致调用方无法感知底层硬件操作失败，可能导致后续操作异常或配置丢失。

#主要错误日志
未知

#问题根因
函数 rtl8365mb_phy_ocp_write() 的代码逻辑存在缺陷，在函数出口处直接硬编码返回 0，而忽略了 regmap 操作过程中变量 ret 中存储的实际错误码。

#解决方案
修改 drivers/net/dsa/realtek/rtl8365mb.c 文件中 rtl8365mb_phy_ocp_write 函数的返回值逻辑，将 `return 0;` 修改为 `return ret;`，确保函数能够正确向上层返回 regmap 操作的实际错误码。
