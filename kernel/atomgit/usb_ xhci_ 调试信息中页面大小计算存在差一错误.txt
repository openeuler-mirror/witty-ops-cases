#标题
usb: xhci: 调试信息中页面大小计算存在差一错误

#内核版本
OLK-6.6

#问题现象
系统启动或 XHCI 控制器初始化阶段，内核日志中打印的 USB XHCI 支持的页面大小数值不正确，比实际值大一倍。例如，若实际页面大小为 4K，日志错误地显示为 8K。此问题仅影响调试信息的准确性，不影响控制器实际功能的正常运行。

#主要错误日志
Supported page size register = 0x[寄存器值]
Supported page size of [错误的大小]K

#问题根因
在计算页面大小时使用了 `ffs()` 函数，该函数返回第一个置位 bit 的索引（从 1 开始计数）。而页面大小的计算公式 `Page size: (2^(n+12))` 中的 `n` 是从 0 开始计数的。代码直接使用了 `ffs()` 的返回值进行移位计算，导致计算结果出现“差一”错误，即多移了一位，使得显示的页面大小是实际大小的 2 倍。

#解决方案
修正计算逻辑，在使用 `ffs()` 的返回值前减去 1，使其与页面大小计算公式要求的 0 基数对齐。将 `i = ffs(page_size)` 修改为 `val = ffs(page_size) - 1`，并在后续的移位计算中使用正确的变量 `val`，从而保证打印出的页面大小数值与硬件实际配置一致。
