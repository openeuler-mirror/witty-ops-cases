#标题
net: enetc: 修复 CONFIG_FSL_ENETC_MDIO 未启用时的编译警告

#内核版本
OLK-6.6

#问题现象
当编译内核且配置项 CONFIG_FSL_ENETC_MDIO 未启用（disabled）时，编译过程中会产生 "no previous prototype for function" 的构建警告，提示函数 `enetc_hw_alloc` 缺少原型声明。

#主要错误日志
include/linux/fsl/enetc_mdio.h:62:18: warning: no previous prototype for function 'enetc_hw_alloc' [-Wmissing-prototypes]
      62 | struct enetc_hw *enetc_hw_alloc(struct device *dev, void __iomem *port_regs)
         |                  ^
include/linux/fsl/enetc_mdio.h:62:1: note: declare 'static' if the function is not intended to be used outside of this translation unit
      62 | struct enetc_hw *enetc_hw_alloc(struct device *dev, void __iomem *port_regs)
         | ^
         | static

#问题根因
在头文件 `include/linux/fsl/enetc_mdio.h` 中，当 CONFIG_FSL_ENETC_MDIO 配置项未定义时，`enetc_hw_alloc` 函数的 stub（占位）实现缺少 `static` 描述符和 `inline` 关键字。这导致该函数被认为是具有外部链接的全局函数，但头文件中未提供对应的原型声明，从而触发编译器警告。

#解决方案
在 `include/linux/fsl/enetc_mdio.h` 文件中，为 `enetc_hw_alloc` 函数的 stub 实现添加 `static` 和 `inline` 关键字修饰，明确该函数仅在当前编译单元内使用，消除编译警告。
