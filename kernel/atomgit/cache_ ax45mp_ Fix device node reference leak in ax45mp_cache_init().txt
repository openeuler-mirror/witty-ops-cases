#标题
cache: ax45mp: Fix device node reference leak in ax45mp_cache_init()

#内核版本
OLK-6.6

#问题现象
内核在调用 `ax45mp_cache_init()` 初始化 Andes AX45MP RISC-V 核心 L2 缓存时，因未正确释放设备节点引用计数，导致 device node reference leak（设备节点引用计数泄漏）。

#主要错误日志
未知

#问题根因
在 `ax45mp_cache_init()` 函数中，`of_find_matching_node()` 返回的设备节点 `np` 具有增加的引用计数，该引用计数必须通过调用 `of_node_put()` 来释放。当前代码在退出前未调用 `of_node_put()`，导致引用计数泄漏。

#解决方案
使用 `__free(device_node)` 属性声明设备节点指针变量 `np`。当变量超出作用域时，编译器会自动执行清理操作释放引用，从而确保引用计数被正确释放，修复内存泄漏问题。
