#标题
emulex/benet: 修复 be_cmd_get_stats 命令版本选择错误

#内核版本
OLK-6.6

#问题现象
在 Emulex BE2 网卡设备上调用 `be_cmd_get_stats` 获取统计信息时，命令版本被错误地设置为 2 或 1，而非预期的版本 0。这导致驱动程序向不支持该版本命令的硬件发送了错误版本的指令，可能导致获取统计信息失败、驱动功能异常或硬件响应错误。

#主要错误日志
未知

#问题根因
在 `be_cmd_get_stats` 函数中，代码逻辑使用了独立的 `if` 语句进行判断。当设备为 BE2 时，代码首先将版本设置为 0；随后代码继续执行后续的 `if` 判断，若 BE3_chip 或 lancer_chip 的检查条件（在特定逻辑下）也被满足或逻辑存在重叠，会导致版本被再次修改。原本的逻辑缺陷导致 BE2 芯片可能被错误地赋予了 version 1 或 version 2，无法保证 version 0 对 BE2 芯片的独占性。

#解决方案
将判断 BE3_chip/lancer_chip 的 `if` 语句修改为 `else if`，使其与前面的 BE2_chip 判断互斥。这确保了逻辑分支的正确性：BE2 芯片使用 version 0，BE3 和 Lancer 芯片使用 version 1，其他芯片使用 version 2，防止了对 BE2 芯片版本的错误覆盖。
