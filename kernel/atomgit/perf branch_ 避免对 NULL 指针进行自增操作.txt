#标题
perf branch: 避免对 NULL 指针进行自增操作

#内核版本
OLK-6.6

#问题现象
在执行 "perf record LBR tests" 测试用例时，`has_stitched_lbr` 函数调用 `perf_sample__branch_entries` 过程中，传入的 `branch_stack` 可能为 NULL，导致代码逻辑尝试对 NULL 指针进行自增操作（increment of NULL）。

#主要错误日志
未知

#问题根因
在 `perf_sample__branch_entries` 函数中，变量 `entry` 直接由 `sample->branch_stack` 转换而来，未进行非空校验。当传入的 `branch_stack` 为 NULL 时，代码继续执行 `entry++` 操作，导致对 NULL 指针进行算术运算。

#解决方案
在 `perf_sample__branch_entries` 函数中增加对 `entry` 指针的判断逻辑：如果 `entry` 为 NULL，则直接返回 NULL，避免后续对 NULL 指针进行自增操作。
