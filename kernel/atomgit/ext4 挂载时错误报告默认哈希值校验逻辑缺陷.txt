#标题
ext4 挂载时错误报告默认哈希值校验逻辑缺陷

#内核版本
OLK-6.6

#问题现象
当 ext4 文件系统的超级块中配置了无效的默认哈希版本（特别是 DX_HASH_SIPHASH）时，系统在挂载过程中报错并拒绝挂载。原有的报错信息逻辑关联了 casefold 特性，提示信息存在误导性，且校验位置不正确，无法准确反映“默认哈希值不应为 SIPHASH”这一约束条件。

#主要错误日志
- 原错误日志（误导性）："Filesystem without casefold feature cannot be mounted with siphash"
- 修复后的正确日志："SIPHASH is not a valid default hash value"
- 修复后的通用校验日志："Invalid default hash set in the superblock"

#问题根因
Commit 985b67cd8639 引入的检查逻辑存在缺陷。该逻辑在 `ext4_feature_set_ok` 函数中检查默认哈希版本是否为 `DX_HASH_SIPHASH`，并将其与 casefold 特性标志强关联。但实际上，`DX_HASH_SIPHASH` 绝不应该作为默认哈希版本，此限制与是否开启 casefold 特性无关。此外，该检查位置导致错误信息未能准确描述拒绝挂载的根本原因。

#解决方案
1. 删除 `ext4_feature_set_ok` 中关于 SIPHASH 与 casefold 关联的检查代码。
2. 将默认哈希版本的校验逻辑移至 `ext4_hash_info_init` 函数中。
3. 引入 `DX_HASH_LAST` 宏定义边界检查，若默认哈希版本超出有效范围或等于 `DX_HASH_SIPHASH`，则返回 `-EINVAL` 并打印准确的错误日志（"SIPHASH is not a valid default hash value"），从而修正挂载校验逻辑。
