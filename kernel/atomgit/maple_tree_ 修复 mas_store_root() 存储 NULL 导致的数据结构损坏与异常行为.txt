#标题
maple_tree: 修复 mas_store_root() 存储 NULL 导致的数据结构损坏与异常行为

#内核版本
OLK-6.6

#问题现象
在 maple tree 中进行连续 NULL 值存储操作时，可能导致内核数据结构损坏或返回错误的数值。具体表现为：
1. 当树被扩展存储数值后，若被全部覆写为 NULL，后续在空白区域继续存储 NULL 可能导致树节点损坏。
2. 存储操作可能产生非预期的行为，例如：在任意范围存储 NULL 导致生成新节点；在单一入口树的范围 [m, n] (m > 0) 存储 NULL 导致生成包含 NULL 的新节点；在空树的范围 [m, n] (m > 0) 存储 NULL 导致连续的 NULL 槽位；允许通过扩展 root 在空树中存储多个 NULL 条目。

#主要错误日志
未知

#问题根因
在 `mas_store_root()` 函数中，处理存储 NULL 值的逻辑存在缺陷。原逻辑未能正确处理将整个树范围（0 - ULONG_MAX）覆写为 NULL 的情况，导致没有将树重置为空树，而是保留了不必要的节点。这不仅造成内存浪费，还导致在后续操作中产生连续的 NULL 槽位，进而引发数据结构损坏和错误的数值返回。

#解决方案
修改 `lib/maple_tree.c` 中的 `mas_store_root()` 函数逻辑。当存储的 entry 为 NULL 且起始索引 `mas->index` 为 0 时，直接通过 `rcu_assign_pointer` 将树的 root 指针设置为 NULL，从而将树重置为空树状态，避免创建不必要的节点并防止后续的数据损坏。
