#标题
drm/amd/display: 修复 dcn20_validate_apply_pipe_split_flags 空指针解引用问题

#内核版本
OLK-6.6

#问题现象
在 AMD 显示驱动模块（dcn20/dcn21）的带宽快速验证流程中，调用 `dcn20_validate_apply_pipe_split_flags` 函数时传入了 NULL 指针作为 `merge` 参数。由于该函数内部会对 `merge` 参数进行解引用操作，导致发生空指针解引用错误，这可能引发内核空指针崩溃或地址访问错误（BUG/Oops）。静态代码分析工具 Coverity 报告了 2 个 FORWARD_NULL 问题。

#主要错误日志
未知

#问题根因
函数 `dcn20_validate_apply_pipe_split_flags` 的实现逻辑中明确对传入的 `merge` 参数进行了解引用操作，但在调用该函数的位置（`dcn20_fast_validate_bw` 和 `dcn21_fast_validate_bw`），代码错误地传递了 NULL 指针作为实参，导致了空指针解引用风险。

#解决方案
在调用 `dcn20_validate_apply_pipe_split_flags` 函数前，先在栈上定义并初始化一个有效的 `merge` 数组（`bool merge[MAX_PIPES] = { false };`），然后将该数组的地址作为参数传递给函数，替代原有的 NULL 指针，从而避免函数内部的空指针解引用。
