#标题
PRUSS 驱动头文件编译错误及警告修复

#内核版本
OLK-6.6

#问题现象
当内核配置项 CONFIG_TI_PRUSS 未设置时，编译 PRUSS 相关代码会产生大量编译错误和警告，导致编译失败。具体表现为函数返回类型不匹配以及语法解析错误。

#主要错误日志
warning: returning 'void *' from a function with return type 'int' makes integer from pointer without a cast [-Wint-conversion]
error: expected identifier or '(' before '{' token

#问题根因
在 `include/linux/pruss_driver.h` 文件中，当 `CONFIG_TI_PRUSS` 未配置时，对应的 stub 内联函数实现存在两个缺陷：
1. 多个函数（如 `pruss_cfg_get_gpmux` 等）错误地使用 `return ERR_PTR(-EOPNOTSUPP);` 返回值，但函数定义的返回类型为 `int`，导致类型不匹配。
2. 函数 `pruss_cfg_xfr_enable` 的定义末尾错误地添加了分号（`;`），导致函数体定义前出现语法错误。

#解决方案
修复 `pruss_driver.h` 中的代码缺陷：
1. 将相关 stub 函数中的 `return ERR_PTR(-EOPNOTSUPP);` 修改为 `return -EOPNOTSUPP;`，使其与 `int` 返回类型匹配。
2. 删除 `pruss_cfg_xfr_enable` 函数定义末尾多余的处分号，修复语法错误。
