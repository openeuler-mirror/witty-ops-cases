#标题
PCI: Fix reset_method_store() memory leak

#内核版本
OLK-6.6

#问题现象
在通过 sysfs 接口向 `reset_method` 写入数据时，存在内存泄漏问题。

#主要错误日志
未知

#问题根因
在 `reset_method_store` 函数中，`strsep` 函数会修改传入的第一个参数指针（`options`）的值。代码逻辑在解析完字符串后可能直接返回或跳出循环，导致原始通过 `kstrdup` 分配的内存指针（`options` 的初始值）丢失，无法进行 `kfree` 释放，从而引发内存泄漏。

#解决方案
引入临时指针变量 `tmp_options` 用于 `strsep` 的迭代解析操作，保持原始指针 `options` 指向已分配内存的起始地址不变，确保在函数退出前能够正确释放该内存，修复内存泄漏问题。
