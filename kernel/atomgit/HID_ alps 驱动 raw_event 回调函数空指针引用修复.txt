#标题
HID: alps 驱动 raw_event 回调函数空指针引用修复

#内核版本
OLK-6.6

#问题现象
在处理 Alps 触摸板设备的 HID 原始事件时，若驱动未正确声明 HID_CLAIMED_INPUT 或 input 设备指针未初始化，内核将触发空指针引用，导致系统崩溃或异常重启。

#主要错误日志
未知

#问题根因
在之前的修复提交（"HID: appleir: Fix potential NULL dereference at raw event handle"）中，已经发现 HID 驱动在 `raw_event` 回调中需要检查设备是否被正确声明，以避免空指针引用。然而，`hid-alps` 驱动遗漏了这一检查。当调用 `alps_raw_event` 函数时，代码直接访问 `hdata->input` 成员，而未验证 `hdev->claimed` 是否包含 `HID_CLAIMED_INPUT` 或 `hdata->input` 是否为空，从而导致 NULL pointer dereference。

#解决方案
在 `drivers/hid/hid-alps.c` 文件的 `alps_raw_event` 函数入口处增加防护性检查。如果设备未被声明为 `HID_CLAIMED_INPUT` 或者 `hdata->input` 指针为空，函数直接返回 0，不再继续执行后续的数据处理逻辑。
