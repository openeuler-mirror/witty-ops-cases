#标题
HID: pidff: 修复 pidff_find_fields 中的空指针解引用问题

#内核版本
OLK-6.6

#问题现象
在 HID PIDFF（Force Feedback）子系统处理设备报告时，若传入的报告对象为空，内核在调用 `pidff_find_fields` 或 `pidff_find_special_field` 函数尝试访问报告字段时会触发空指针解引用，导致内核 panic 或 oops。

#主要错误日志
pidff_find_fields, null report
pidff_find_special_field, null report

#问题根因
函数 `pidff_find_fields` 和 `pidff_find_special_field` 在处理输入参数时，未对 `struct hid_report *report` 参数进行有效性检查。当该指针为 NULL 时，代码逻辑直接尝试访问 `report` 结构体成员（如 `report->maxfield`），导致空指针解引用故障。

#解决方案
在 `pidff_find_fields` 和 `pidff_find_special_field` 函数入口处增加对 `report` 指针的非空判断。若指针为空，打印调试信息并返回错误码（-1 或 NULL），从而避免后续代码访问空指针。
