#标题
regulator: max20086: 修复 MAX20086 芯片 ID 定义错误导致驱动探测失败

#内核版本
OLK-6.6

#问题现象
当系统使用 MAX20086 硬件设备时，对应的 regulator 驱动在加载过程中探测失败，导致设备无法正常初始化和使用。

#主要错误日志
未知

#问题根因
根据 MAX20086-MAX20089 数据手册，MAX20086 的芯片 ID 应为 0x30，但内核代码中错误地将其宏定义写为了 0x40。驱动在 probe 阶段通过 I2C 总线读取芯片 ID 进行比对时，因定义值与硬件实际返回值不符，导致驱动探测失败。

#解决方案
修正 drivers/regulator/max20086-regulator.c 文件中的 MAX20086_DEVICE_ID_MAX20086 宏定义，将其值从错误的 0x40 修改为正确的 0x30，以确保驱动能正确识别芯片。
