#标题
LoongArch: FDT 系统下 early_numa_add_cpu 传参错误导致 NUMA 节点分配异常

#内核版本
OLK-6.6

#问题现象
在基于 FDT（Flattened Device Tree）的 LoongArch 系统上，SMP 初始化过程中 NUMA 节点 0 的 CPU 添加逻辑存在错误。由于 `early_numa_add_cpu()` 函数错误地接收了逻辑 CPU ID 而非物理 CPU ID，导致 NUMA 节点的 CPU 位图映射错误，可能引发后续内存分配策略异常或 NUMA 调度错误。

#主要错误日志
未知

#问题根因
函数 `early_numa_add_cpu()` 的设计意图是作用于物理 CPU ID，但在 `fdt_smp_setup()` 调用该函数时，错误地传递了逻辑 CPU ID（变量 `cpu`）。正确的做法是传递物理 CPU ID（变量 `cpuid`），该错误源于提交 ae9e39a2fbf5。

#解决方案
修改 `arch/loongarch/kernel/smp.c` 文件中的 `fdt_smp_setup()` 函数，将调用 `early_numa_add_cpu()` 时的第一个参数从 `cpu` 修改为 `cpuid`，确保传入的是物理 CPU ID。
