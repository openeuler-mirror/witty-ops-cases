#标题
urma 模块 UM+CTP 模式下 import 函数未校验 order_type 导致非预期行为

#内核版本
OLK-6.6

#问题现象
在调用 `ubcore_import_jetty` 或 `ubcore_import_jfr` 接口时，若配置参数中 `trans_mode` 为 UM 且 `tp_type` 为 CTP，当 `order_type` 取值不为 0 (DEF) 或 3 (OL) 时，系统未拦截非法参数，导致产生非预期行为（unexpected behavior）。

#主要错误日志
Invalid order_type %u for UM+CTP, only 0(DEF) or 3(OL) allowed.

#问题根因
`ubcore_import_jetty` 和 `ubcore_import_jfr` 函数内部缺少对 `order_type` 参数的合法性校验。在 UM+CTP 组合场景下，`order_type` 仅支持 0 (DEF) 或 3 (OL)，代码未对其他非法值进行过滤和拦截，导致后续处理流程出现非预期行为。

#解决方案
在 `ubcore_import_jetty`、`ubcore_import_jfr` 及其扩展版本（`_ex` 后缀）函数入口处增加参数校验逻辑。新增 `ubcore_validate_order_type_for_um_ctp` 校验函数，当检测到 `trans_mode` 为 UM 且 `tp_type` 为 CTP 时，强制检查 `order_type` 是否为合法值（0 或 3），若校验失败则打印错误日志并返回 `-EINVAL`，阻止后续流程执行。
