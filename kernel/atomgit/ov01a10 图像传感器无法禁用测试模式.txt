#标题
ov01a10 图像传感器无法禁用测试模式

#内核版本
OLK-6.6

#问题现象
当用户将 ov01a10 摄像头传感器的测试模式控制值设置为 0（即 "Disabled"）时，驱动程序未向硬件寄存器写入关闭指令，导致测试模式无法正常关闭，传感器可能持续输出测试图案而非正常图像流。

#主要错误日志
未知

#问题根因
驱动程序 `ov01a10_test_pattern` 函数逻辑错误：当传入参数 `pattern` 为 0 时，代码直接 `return 0` 返回，未执行寄存器写操作。这导致硬件无法接收到关闭测试模式的指令，测试模式一直保持开启状态。

#解决方案
修改 `drivers/media/i2c/ov01a10.c` 中的 `ov01a10_test_pattern` 函数逻辑：
1. 移除 `if (!pattern) return 0;` 的提前返回判断。
2. 调整逻辑，仅在 `pattern` 非 0 时设置使能位 `OV01A10_TEST_PATTERN_ENABLE`。
3. 确保当 `pattern` 为 0 时，函数继续执行并将值 0 写入 `OV01A10_REG_TEST_PATTERN` 寄存器，从而正确关闭测试模式。
