#标题
perf/x86/zhaoxin/uncore: PCI驱动冲突导致pcieport功能失效

#内核版本
OLK-6.6

#问题现象
系统内部 PMU uncore 驱动与系统自带的 pcieport 驱动发生 PCI 设备 ID 冲突，导致 pcieport 驱动加载失败，相关的 PCIe 端口服务（如 AER、热插拔等）无法正常工作。

#主要错误日志
未知

#问题根因
Zhaoxin CPU 的 PMU uncore 驱动代码中，错误地将 PCIe 端口设备（Device ID 0x0717-0x073B）注册到了 `uncore_pci_ids` 表中。这导致 uncore 驱动与内核标准的 pcieport 驱动竞争同一批 PCI 设备，引发驱动绑定冲突，致使 pcieport 驱动失效。

#解决方案
从 `zhaoxin/uncore.c` 文件中移除 `kh40000_uncore_pci_ids` 和 `kx7000_uncore_pci_ids` 数组内所有 PCIe 端口相关的设备 ID 条目（包括 D2F、D3F、D4F、D5F 等系列），确保这些设备仅由 pcieport 驱动管理，从而消除冲突。
