#标题
binderfs: ida_alloc_max() 上界错误导致次设备号越界

#内核版本
OLK-6.6

#问题现象
在 binderfs 挂载且设备数量较多时，若系统分配到了 ID 为 `BINDERFS_MAX_MINOR` (1 << 20) 的次设备号，会导致 `dev->minor` 超出 20-bit 的限制，引发越界错误或设备创建异常。

#主要错误日志
未知

#问题根因
`ida_alloc_max()` 函数的 `max` 参数应接受最大有效的 ID 值，而非数量（count）。原代码直接传入 `BINDERFS_MAX_MINOR` 作为上界，导致分配到的 ID 可能等于 `BINDERFS_MAX_MINOR`。由于次设备号定义为 20 位，ID 值为 `BINDERFS_MAX_MINOR` (1 << 20) 时会超出有效位宽限制，这是一个典型的差一错误。

#解决方案
修正 `ida_alloc_max()` 的调用参数，将上界减 1（即 `BINDERFS_MAX_MINOR - 1` 和 `BINDERFS_MAX_MINOR_CAPPED - 1`），确保分配的最大 ID 值不超过 20-bit 限制，修复该差一错误。
