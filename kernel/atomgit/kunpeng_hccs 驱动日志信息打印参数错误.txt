#标题
kunpeng_hccs 驱动日志信息打印参数错误

#内核版本
OLK-6.6

#问题现象
在查询 HCCS（High-Speed Congestion Control System）端口空闲状态时，若获取状态失败或端口处于忙状态，内核日志打印的芯片 ID、Die ID 以及端口 ID 信息显示不正确，导致运维人员无法定位具体的故障物理位置。

#主要错误日志
- `dev_err`: "hccs%u on chip%u/die%u get idle status failed, ret = %d.\n"
- `dev_info`: "hccs%u on chip%u/die%u is busy.\n"

#问题根因
在 `hccs_get_all_spec_port_idle_sta` 函数中，打印日志时使用了循环变量 `k`、`i`、`j` 作为端口 ID、芯片 ID 和 Die ID，而非结构体中实际存储的 `port->port_id`、`chip->chip_id` 和 `die->die_id`。循环变量 `k` 仅表示遍历过程中的索引，不对应真实的硬件端口编号，导致输出的日志信息错误。

#解决方案
修改 `drivers/soc/hisilicon/kunpeng_hccs.c` 文件中的日志打印参数，将循环变量替换为正确的结构体成员变量：
1. 将端口 ID 从循环变量 `k` 修改为 `port->port_id`。
2. 将芯片 ID 从循环变量 `i` 修改为 `chip->chip_id`。
3. 将 Die ID 从循环变量 `j` 修改为 `die->die_id`。
