#标题
cpufreq: acpi-cpufreq 修复 Zhaoxin/Centaur CPU 最大频率计算错误问题

#内核版本
OLK-6.6

#问题现象
在 Zhaoxin（兆芯）或 Centaur CPU 平台上，acpi-cpufreq 驱动计算的最大频率数值不正确，可能导致 CPU 频率策略配置异常，影响系统性能表现或功耗管理。

#主要错误日志
未知

#问题根因
acpi-cpufreq 驱动中的 `get_max_boost_ratio` 函数在计算最大频率比例时，未针对 Zhaoxin 和 Centaur CPU 的特性进行处理，导致计算逻辑在这些平台上返回了错误的频率比值。根据补丁描述，这是该类 CPU 上存在的频率计算缺陷。

#解决方案
在 `get_max_boost_ratio` 函数中增加对 CPU 厂商的判断逻辑。当检测到 CPU 厂商为 `X86_VENDOR_ZHAOXIN` 或 `X86_VENDOR_CENTAUR` 时，直接返回 0，以此绕过错误的计算路径，作为修复最大频率计算错误的手段。
