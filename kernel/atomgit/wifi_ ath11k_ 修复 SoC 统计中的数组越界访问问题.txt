#标题
wifi: ath11k: 修复 SoC 统计中的数组越界访问问题

#内核版本
OLK-6.6

#问题现象
在 ath11k 驱动处理接收数据时，访问 `hal_reo_error` 数组发生越界。这是因为在索引数组时错误地使用了 SRNG (Shared Ring) ring ID，而该 ID 与数组定义的正常 ring ID 不一致，可能导致内核内存数据被破坏或引发内核崩溃。

#主要错误日志
未知

#问题根因
数组 `ath11k_soc_dp_stats::hal_reo_error` 定义的最大大小为 `DP_REO_DST_RING_MAX`。但在 `ath11k_dp_process_rx()` 函数中，代码 `ab->soc_stats.hal_reo_error[dp->reo_dst_ring[ring_id].ring_id]++` 错误地使用了 SRNG ring ID 来访问数组。SRNG ring ID 与正常的 ring ID 不同，这种错误的映射关系导致了数组越界访问。

#解决方案
修改 `ath11k_dp_process_rx()` 函数，直接使用传入的 `ring_id`（即 normal ring ID）作为数组索引，而不是通过 `dp->reo_dst_ring[ring_id].ring_id` 转换获取的 SRNG ring ID，从而确保索引值在数组边界范围内。
