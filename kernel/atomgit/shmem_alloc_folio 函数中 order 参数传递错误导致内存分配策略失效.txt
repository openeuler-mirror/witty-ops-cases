#标题
shmem_alloc_folio 函数中 order 参数传递错误导致内存分配策略失效

#内核版本
OLK-6.6

#问题现象
在调用 shmem_alloc_folio 进行大页内存分配时，函数内部忽略了传入的 order 参数，固定传递 0 给内存策略相关接口。这导致在配置了 NUMA 策略的场景下，无法按预期分配指定阶数的内存页，内存分配行为与策略配置不符。

#主要错误日志
未知

#问题根因
在合并 shmem_alloc_hugefolio() 与 shmem_alloc_folio() 的重构提交（c7be1041758）之后，shmem_alloc_folio() 变为一个通用的分配辅助函数，但在调用 shmem_get_pgoff_policy() 和 alloc_pages_mpol() 时，代码错误地将参数 order 硬编码为 0，未使用函数传入的 order 变量，导致内存分配策略判断和实际分配请求不匹配。

#解决方案
修改 mm/shmem.c 文件中的 shmem_alloc_folio 函数，将调用 shmem_get_pgoff_policy() 和 alloc_pages_mpol() 时的第三个参数从常量 0 修正为变量 order，确保内存分配策略和页面分配接口能正确获取到请求的阶数信息。
