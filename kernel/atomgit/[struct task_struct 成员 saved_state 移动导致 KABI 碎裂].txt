#标题
[struct task_struct 成员 saved_state 移动导致 KABI 碎裂]

#内核版本
[OLK-6.6]

#问题现象
[内核编译或运行时检查发现 struct task_struct 的结构体布局发生变化，导致 KABI（内核二进制接口）一致性检查失败，影响内核模块兼容性。]

#主要错误日志
[未知]

#问题根因
[在修复 Commit 31981b6fdbd5 ("sched/core: Remove ifdeffery for saved_state") 的过程中，直接将 unsigned int saved_state 成员添加到了 struct task_struct 的核心区域，而没有使用 KABI 预留空间。这破坏了 struct task_struct 的二进制布局，导致了 KABI breakage。]

#解决方案
[将 saved_state 成员从 struct task_struct 的原有位置移除，利用 KABI_RESERVE(3) 预留空间来定义该成员。具体通过 KABI_USE2 宏将 saved_state 定义在 kabi_reserved_int 之后，从而在保持结构体整体布局不变的前提下容纳新成员，修复了 KABI 碎裂问题。]
