#标题
mtd: hyperbus: rpc-if 驱动模块缺少 MODULE_DEVICE_TABLE 导致无法自动加载

#内核版本
OLK-6.6

#问题现象
当 rpc-if-hyperflash 驱动被编译为内核模块时，系统无法根据硬件设备 ID 自动加载该模块，导致设备功能不可用，需手动加载或内核内置。

#主要错误日志
未知

#问题根因
rpc-if-hyperflash 驱动代码中缺少 MODULE_DEVICE_TABLE() 宏定义。这导致模块编译时无法生成正确的模块别名信息，用户空间的 udev 或 kmod 工具无法依据硬件 ID 匹配并自动触发 modprobe 加载模块。

#解决方案
在驱动源文件 `drivers/mtd/hyperbus/rpc-if.c` 中添加设备 ID 表 `rpc_if_hyperflash_id_table`，并通过 `MODULE_DEVICE_TABLE(platform, rpc_if_hyperflash_id_table)` 宏将其导出，同时将此表注册到 `platform_driver` 结构体的 `id_table` 成员中，从而支持模块的自动加载功能。
