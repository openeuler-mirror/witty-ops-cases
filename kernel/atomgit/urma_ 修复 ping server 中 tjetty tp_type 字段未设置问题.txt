#标题
urma: 修复 ping server 中 tjetty tp_type 字段未设置问题

#内核版本
OLK-6.6

#问题现象
在 URMA（User Space Remote Memory Access）模块的 ping server 功能中，创建 tjetty（target jetty）对象时，由于 `ubcore_tp_cfg` 结构体中的 `tp_type` 字段未初始化（默认为 0 或错误值），可能导致底层硬件或驱动无法正确识别传输层类型，进而引发 ping 功能异常或相关配置失败。

#主要错误日志
未知

#问题根因
在 `ubmgr_ping.c` 文件的 `__ping_tjetty_new_entry` 函数中，构造 `ubcore_tp_cfg` 结构体时遗漏了 `tp_type` 字段的赋值。该结构体用于配置传输路径（TP），缺少 `tp_type` 的设置会导致结构体配置不完整。

#解决方案
在 `ubcore_tp_cfg` 结构体的初始化列表中添加 `.tp_type = UBCORE_CTP`，显式设置传输路径类型为 UBCORE_CTP，确保配置信息的完整性。
