#标题
regulator: mt6357 OF node reference imbalance

#内核版本
OLK-6.6

#问题现象
在卸载（unbind）mt6357 多功能设备（MFD）并注销其子设备时，由于 OF node 引用计数不平衡，导致内核对设备树节点的管理出现异常，可能引发内核引用计数泄漏或相关的内存管理问题。

#主要错误日志
未知

#问题根因
驱动程序在 probe 阶段直接将父设备的 OF node 赋值给子设备（`pdev->dev.of_node = pdev->dev.parent->of_node`），复用了父设备的设备树节点，但未增加引用计数。当 MFD 设备解绑并注销子设备时，Platform 总线代码会释放该 OF node 的引用，导致引用计数失衡（dropped one reference without having taken it）。

#解决方案
使用专用的辅助函数 `device_set_of_node_from_dev` 替代直接赋值操作。该函数在复用父设备 OF node 的同时会正确增加引用计数，从而在设备注销时平衡引用计数，修复泄漏问题。
