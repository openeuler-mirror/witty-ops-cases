#标题
ixgbevf: E610 适配器 VF 创建后无法设置链路 up

#内核版本
OLK-6.6

#问题现象
在 E610 系列适配器上创建虚拟功能（VF）并尝试设置链路 up 时，链路无法正常启动。该问题发生在 VF 驱动支持 mailbox API 1.6 或更高版本，但物理功能（PF）驱动不支持该 API 版本（版本低于 1.6）的不匹配场景下。

#主要错误日志
未知

#问题根因
当 VF 驱动识别到适配器类型为 E610 时，代码强制使用新的 `ixgbevf_get_pf_link_state` 方法获取链路状态。如果协商后的 `api_version` 小于 1.6，PF 驱动不支持该新版请求方式，会返回 `-EOPNOTSUPP` 错误。该错误导致驱动过早退出链路配置流程，从而导致链路设置失败。

#解决方案
在检查是否使用新的 PF 链路状态获取方法时，增加对 `api_version` 的判断。只有当 MAC 类型为 `ixgbe_mac_e610_vf` 且 API 版本大于等于 `ixgbe_mbox_api_16` 时，才调用 `ixgbevf_get_pf_link_state`；否则回退使用传统的 VFLINKS 方式处理，确保在 API 版本不匹配时链路能正常配置。
