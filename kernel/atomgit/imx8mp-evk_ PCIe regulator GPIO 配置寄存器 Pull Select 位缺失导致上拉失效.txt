#标题
imx8mp-evk: PCIe regulator GPIO 配置寄存器 Pull Select 位缺失导致上拉失效

#内核版本
OLK-6.6

#问题现象
NXP i.MX8MP EVK 开发板上，PCIe regulator 对应的 GPIO 引脚 (GPIO2_IO06) 虽然配置了弱上拉，但由于 Pull Select 位未使能，导致上拉电阻实际上并未生效。这会导致该引脚的电平状态不稳定，可能引起 PCIe 外设（如 M.2 接口的 W_DISABLE1 信号）误判断或功能异常。

#主要错误日志
未知

#问题根因
在设备树 pinctrl 配置中，寄存器值被设置为 0x40，这仅配置了上拉使能，但未设置 IOMUX 寄存器的 Bit 8 (Pull Select 位)。根据硬件规格，必须同时设置 Pull Select 位和上拉位，弱上拉功能才能真正生效。原配置导致引脚处于悬空或非预期状态。

#解决方案
修改设备树 `imx8mp-evk.dts` 中 `pinctrl_pcie0_reg` 节点下的引脚配置值，将 `MX8MP_IOMUXC_SD1_DATA4__GPIO2_IO06` 的配置值从 `0x40` 修改为 `0x140`。设置 Bit 8 以启用 Pull Select 功能，确保弱上拉配置生效，使引脚电平行为符合预期。
