#标题
sama5d2 SoC Flexcom SPI 节点 FIFO 容量属性配置错误

#内核版本
OLK-6.6

#问题现象
Microchip sama5d2 处理器上 Flexcom SPI 控制器的 FIFO 容量属性被错误配置为 16 字节，导致 SPI 通信因 FIFO 深度设置与硬件实际不符（硬件实际为 32 字节）而出现数据传输异常或吞吐量下降。

#主要错误日志
未知

#问题根因
设备树（DTS）中 `atmel,fifo-size` 属性赋值错误。sama5d2 的 Flexcom SPI 外设与独立的 SPI 外设不同，其 FIFO 实际大小为 32 个数据，但设备树中错误地将其设置为 16，导致驱动程序使用了错误的 FIFO 参数。

#解决方案
修改 `arch/arm/boot/dts/microchip/sama5d2.dtsi` 文件，将 FLX0、FLX1、FLX2、FLX3、FLX4 等 Flexcom SPI 节点的 `atmel,fifo-size` 属性值从 `<16>` 更正为 `<32>`，以匹配硬件实际规格。
