#标题
rtc: ds1307: 请求频率超出范围时错误地禁用时钟

#内核版本
OLK-6.6

#问题现象
当调用 `ds3231_clk_sqw_round_rate()` 函数请求的时钟频率高于 DS3231 支持的最高频率时，该函数错误地返回 0，导致时钟被意外禁用，而非使用支持的最高频率继续运行。

#主要错误日志
未知

#问题根因
函数 `ds3231_clk_sqw_round_rate()` 在请求频率高于支持的最高频率时，直接返回了 0。根据 Linux clk API 的规范，`round_rate()` 回调函数在此类情况下应返回设备支持的最高频率，原实现违反了该接口约定。

#解决方案
修改 `drivers/rtc/rtc-ds1307.c` 文件中的 `ds3231_clk_sqw_round_rate` 函数。当请求频率超出支持范围时，不再返回 0，而是返回数组 `ds3231_clk_sqw_rates` 中的最后一个元素（即支持的最高频率值），以符合 clk API 的预期行为。
