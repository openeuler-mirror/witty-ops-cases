#标题
ext4: 孤儿 inode 检查逻辑不一致导致 e2fsck 报错与测试失败

#内核版本
OLK-6.6

#问题现象
当启用 orphan file 特性时，内核对孤儿 inode 的判断逻辑存在缺陷，导致以下现象：
1. 偶发 xfstests 测试集 ext4/307 测试失败。
2. 系统重启后 e2fsck 检查文件系统时报错并抱怨存在不应有的孤儿 inode。
3. 在 inode 销毁时，若检测到残留的孤儿标记，内核打印错误日志提示 orphan list check failed。

#主要错误日志
- ext4_destroy_inode 中触发的报错（修复前日志）：
  `EXT4-fs error (device ...): Inode %lu (%p): orphan list check failed!`
- 修复后更准确的日志格式（推断）：
  `EXT4-fs error (device ...): Inode %lu (%p): inode tracked as orphan!`

#问题根因
启用了 orphan file 特性后，inode 可以通过两种方式被追踪为孤儿状态：标准的孤儿链表（`i_orphan`）或孤儿文件（通过 `EXT4_STATE_ORPHAN_FILE` 标志位记录）。代码中多处判断 inode 是否为孤儿的逻辑不完整，仅检查了 `i_orphan` 链表是否为空，而遗漏了对 `EXT4_STATE_ORPHAN_FILE` 标志位的检查。
这导致部分本应被识别为孤儿的 inode 被错误判定为非孤儿，或者非孤儿 inode 未被正确清理孤儿状态，进而导致不必要的孤儿追踪残留，引发 e2fsck 报错和测试用例失败。

#解决方案
1. 在 `fs/ext4/ext4.h` 中新增辅助函数 `ext4_inode_orphan_tracked()`，统一封装孤儿 inode 的判断逻辑，同时检查 `EXT4_STATE_ORPHAN_FILE` 标志与 `i_orphan` 链表。
2. 修改 `fs/ext4/file.c`、`fs/ext4/inode.c`、`fs/ext4/orphan.c`、`fs/ext4/super.c` 中分散的孤儿 inode 判断代码，统一替换调用新增的辅助函数，确保逻辑一致性。
