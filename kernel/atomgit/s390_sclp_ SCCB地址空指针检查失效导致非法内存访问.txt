#标题
s390/sclp: SCCB地址空指针检查失效导致非法内存访问

#内核版本
OLK-6.6

#问题现象
在内核身份映射起始地址不为 0 的情况下，SCLP 中断处理程序对 SCCB 地址的 NULL 检查失效。这可能导致内核错误地访问身份映射的第一页内存，引发非法内存访问或系统异常。

#主要错误日志
未知

#问题根因
SCLP 中断处理程序中的 tracing 代码在对 SCCB 地址进行 NULL 检查前，先执行了物理地址到虚拟地址的转换（__va）。
当内核身份映射不是从地址 0 开始时，转换后的虚拟地址永远不会为 0，导致原本的 NULL 检查条件无法成立。这使得后续代码在 SCCB 实际为 NULL 的情况下继续执行，导致对身份映射第一页的错误访问。

#解决方案
引入新函数 `sclpint_to_sccb`，在物理地址转换为虚拟地址之前优先判断地址是否为 0：若地址为 0 则直接返回 NULL，否则再进行 `__va` 转换。修复了 `ok_response` 和 `sclp_interrupt_handler` 中的相关调用，确保 NULL 检查逻辑正确执行。
