#标题
fsl-mc: fsl_mc_get_endpoint函数存在双重设备引用导致引用计数泄漏

#内核版本
OLK-6.6

#问题现象
在调用 fsl_mc_get_endpoint() 函数时，若第一次查找成功找到设备，函数将继续执行 rescan 逻辑并尝试第二次查找。这导致该设备的引用计数被错误地累加两次，造成引用计数泄漏，可能导致设备无法正常释放或注销。

#主要错误日志
未知

#问题根因
fsl_mc_get_endpoint() 函数逻辑存在缺陷，在第一次调用 fsl_mc_device_lookup() 成功找到 endpoint 设备后，未立即返回，而是继续向下执行 rescan 和第二次 lookup 流程。根据提交信息描述：“The fsl_mc_get_endpoint() function may call fsl_mc_device_lookup() twice, which would increment the device's reference count twice if both lookups find a device.”

#解决方案
在第一次调用 fsl_mc_device_lookup() 查找到 endpoint 设备后，增加判断逻辑：若查找成功（endpoint 指针非空），则直接返回该设备指针，避免执行后续不必要的 rescan 操作和第二次查找，从而防止引用计数重复增加。
