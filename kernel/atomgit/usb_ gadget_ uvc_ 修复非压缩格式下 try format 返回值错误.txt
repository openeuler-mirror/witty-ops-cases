#标题
usb: gadget: uvc: 修复非压缩格式下 try format 返回值错误

#内核版本
OLK-6.6

#问题现象
在 USB Gadget UVC 功能中，当应用程序调用 try format 接口设置非压缩格式时，返回的结构体中 `bytesperline` 和 `sizeimage` 字段值可能不准确，未能根据实际的宽高和像素格式正确计算，导致后续视频流配置参数异常或协商失败。

#主要错误日志
未知

#问题根因
`uvc_v4l2_try_format` 函数在处理非压缩格式时，直接使用了通用的计算逻辑，未利用 `v4l2_fill_pixfmt` 辅助函数来根据具体的像素格式正确填充 `bytesperline` 和 `sizeimage`，导致返回给用户空间的格式参数不正确。

#解决方案
修改 `uvc_v4l2_try_format` 函数逻辑，增加对非压缩格式的分支判断。当格式类型为 `UVCG_UNCOMPRESSED` 时，调用 `v4l2_fill_pixfmt` 辅助函数来计算并填充 `bytesperline` 和 `sizeimage`；对于其他格式，保留原有的计算逻辑。
