#标题
urma: ping server 中 tjetty 的 tp type 未设置导致状态错误

#内核版本
OLK-6.6

#问题现象
在 URMA ping server 功能中创建 tjetty 时，由于 `tp_type` 字段未初始化（保持为默认值或垃圾值），可能导致后续传输层逻辑处理异常，影响 ping server 的正常通信或初始化流程。

#主要错误日志
未知

#问题根因
在函数 `__ping_tjetty_new_entry` 中，结构体 `ubcore_tjetty_cfg` 初始化时遗漏了 `tp_type` 字段的赋值。根据代码逻辑，该字段应当被设置为 `UBCORE_CTP`，但原代码中缺失了该赋值操作。

#解决方案
在 `drivers/ub/urma/ubcore/ubmgr/ubmgr_ping.c` 文件的 `__ping_tjetty_new_entry` 函数中，为 `ubcore_tjetty_cfg` 结构体补全 `.tp_type = UBCORE_CTP` 的初始化赋值。
