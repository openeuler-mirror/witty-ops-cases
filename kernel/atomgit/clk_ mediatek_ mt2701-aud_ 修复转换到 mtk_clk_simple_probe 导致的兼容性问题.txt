#标题
clk: mediatek: mt2701-aud: 修复转换到 mtk_clk_simple_probe 导致的兼容性问题

#内核版本
OLK-6.6

#问题现象
MediaTek MT2701 音频子系统的时钟驱动在转换为 `mtk_clk_simple_probe` 接口后，由于驱动中定义的时钟 ID 列表与设备树（DT）绑定中定义的 ID 存在差异，导致时钟注册过程出现异常。`mtk_clk_simple_probe` 函数期望最大的时钟 ID 由 `ARRAY_SIZE` 定义，但部分头文件中定义的时钟 ID 在驱动数组中缺失（存在空洞），破坏了 ID 与数组索引的一一对应关系，导致驱动加载失败或时钟功能不可用。

#主要错误日志
未知

#问题根因
部分音频子系统时钟 ID（如 `CLK_AUD_LRCK_DETECT`、`CLK_AUD_I2S` 等）在 `clk-mt2701.h` 头文件中有定义，但在驱动代码 `audio_clks` 数组中并未实际使用和实现。在转换为 `mtk_clk_simple_probe` 时，该接口逻辑要求时钟 ID 必须是连续的或者数组大小必须覆盖最大 ID 值。由于数组中存在“空洞”（缺少某些 ID 的定义），导致 `mtk_clk_simple_probe` 无法正确处理时钟注册，破坏了与现有设备树绑定的兼容性。

#解决方案
在 `drivers/clk/mediatek/clk-mt2701-aud.c` 文件的 `audio_clks` 数组中，针对缺失的时钟 ID 插入 `GATE_DUMMY` 类型的占位符时钟。这些占位符填补了 ID 空缺，确保数组的 `ARRAY_SIZE` 能够覆盖最大的时钟 ID，从而满足 `mtk_clk_simple_probe` 的逻辑要求，同时保持了与现有 DT 绑定的兼容性。
