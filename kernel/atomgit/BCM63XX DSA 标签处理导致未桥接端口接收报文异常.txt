#标题
BCM63XX DSA 标签处理导致未桥接端口接收报文异常

#内核版本
OLK-6.6

#问题现象
在 BCM63XX SoCs 上，当端口处于独立模式或连接至 VLAN 不感知网桥时，接收到的数据包被错误地标记了 VID 为 0 的 802.1Q VLAN 标签。这导致用户态应用程序（如 STP 守护进程）在处理原始数据包时解析失败，同时也导致网桥功能测试（如 tcpdump 抓包对比）结果不符合预期。

#主要错误日志
未知

#问题根因
BCM63XX SoCs 内部交换机在 802.1Q 模式启用时，会无条件地在发往 CPU 的报文出方向添加 802.1Q VLAN 标签。对于非 VLAN 感知网桥或独立端口，报文被标记为默认 VID（即 0）。虽然内核协议栈将其视为未标记报文处理，但带有 VID 0 的标签会破坏用户态应用对原始未标记流量的预期，且破坏测试用例的输出匹配。代码逻辑中未能正确识别并剥离这种特殊的 VID 0 标签。

#解决方案
修改 `net/dsa/tag_brcm.c` 中的 `brcm_leg_tag_rcv` 函数逻辑。在接收处理时，检查报文是否带有 VID 为 0 的 802.1Q 标签。如果检测到 VLAN 协议为 `ETH_P_8021Q` 且 TCI 字段（VID）为 0，则增加剥离长度以移除该 VLAN 标签，从而恢复为标准的未标记报文格式；若优先级（priority）字段非 0 则保留标签。
