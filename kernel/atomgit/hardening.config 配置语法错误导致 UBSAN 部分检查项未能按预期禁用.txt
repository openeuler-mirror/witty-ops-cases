#标题
hardening.config 配置语法错误导致 UBSAN 部分检查项未能按预期禁用

#内核版本
OLK-6.6

#问题现象
在使用 `hardening.config` 碎片配置内核时，原本预期禁用的 UBSAN 检查项（如 `CONFIG_UBSAN_BOOL` 和 `CONFIG_UBSAN_ENUM`）仍然处于启用状态。通过查看编译后的 `.config` 文件或运行 `make savedefconfig`，可以发现这些本应被关闭的配置项显示为 `=y`，导致内核在硬ening模式下启用了非预期的行为检查。

#主要错误日志
未知

#问题根因
配置文件 `kernel/configs/hardening.config` 中禁用配置项的语法书写错误。
在 Linux 内核配置文件格式中，禁用某个选项必须显式写出 `is not set`（例如 `# CONFIG_UBSAN_BOOL is not set`）。原提交中仅写了 `# CONFIG_UBSAN_BOOL`，缺少了关键的 `is not set` 后缀。这导致 Kconfig 解析器无法正确识别为禁用指令，从而导致 `CONFIG_UBSAN=y` 时这些选项保持了默认启用状态。

#解决方案
修正 `kernel/configs/hardening.config` 文件中相关配置项的语法格式，补全缺失的 `is not set` 字符串。
具体修改如下：
- `# CONFIG_UBSAN_DIV_ZERO` 改为 `# CONFIG_UBSAN_DIV_ZERO is not set`
- `# CONFIG_UBSAN_UNREACHABLE` 改为 `# CONFIG_UBSAN_UNREACHABLE is not set`
- `# CONFIG_UBSAN_BOOL` 改为 `# CONFIG_UBSAN_BOOL is not set`
- `# CONFIG_UBSAN_ENUM` 改为 `# CONFIG_UBSAN_ENUM is not set`
- `# CONFIG_UBSAN_ALIGNMENT` 改为 `# CONFIG_UBSAN_ALIGNMENT is not set`
