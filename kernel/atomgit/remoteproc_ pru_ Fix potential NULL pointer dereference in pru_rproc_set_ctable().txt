#标题
remoteproc: pru: Fix potential NULL pointer dereference in pru_rproc_set_ctable()

#内核版本
OLK-6.6

#问题现象
在调用 `pru_rproc_set_ctable()` 函数时，如果传入的 `rproc` 参数为 NULL，函数在执行参数有效性检查前就尝试访问 `rproc->priv` 成员，导致内核发生空指针解引用，进而引发内核 panic 或 oops。

#主要错误日志
未知

#问题根因
函数 `pru_rproc_set_ctable()` 在进行 `IS_ERR_OR_NULL` 等有效性检查之前，提前执行了 `struct pru_rproc *pru = rproc->priv` 赋值操作。当 `rproc` 指针为 NULL 时，该赋值操作直接触发了空指针解引用。

#解决方案
将 `pru` 指针的赋值操作（`pru = rproc->priv`）移动到 `rproc` 指针的有效性检查（`IS_ERR_OR_NULL` 检查及设备父节点检查）之后。确保只有在 `rproc` 指针被验证为非 NULL 后才对其进行解引用访问。
