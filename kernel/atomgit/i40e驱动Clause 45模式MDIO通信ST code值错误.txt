#标题
i40e驱动Clause 45模式MDIO通信ST code值错误

#内核版本
OLK-6.6

#问题现象
使用Intel i40e网卡驱动进行Clause 45模式的MDIO通信时，由于状态码设置错误，可能导致MDIO读写操作失败或无法正确寻址PHY设备，进而影响链路状态检测或PHY配置功能。

#主要错误日志
未知

#问题根因
在之前的代码重构（commit 8196b5fd6c73）中，宏 `I40E_GLGEN_MSCA_STCODE_MASK` 被修改为固定值 0x1。该值对于 Clause 22 模式是正确的，但对于 Clause 45 模式是错误的。Clause 45 模式要求 ST code 必须为 0，而错误的重构导致其使用了 Clause 22 的值 1。

#解决方案
修改 `I40E_GLGEN_MSCA_STCODE_MASK` 宏定义，使其接受一个参数以指定具体的 ST code 值。将 Clause 22 的调用修改为传入参数 1，将 Clause 45 的调用修改为传入参数 0，从而修正了 Clause 45 模式下的控制字段值。
