#标题
LoongArch IP 校验码计算代码越界访问

#内核版本
OLK-6.6

#问题现象
当调用 IP 校验码计算函数 `do_csum` 时，若传入的长度参数 `len` 为负数，代码未能正确处理，导致发生越界访问。

#主要错误日志
未知

#问题根因
在 `arch/loongarch/lib/csum.c` 的 `do_csum` 函数中，入口处的长度校验逻辑仅判断了 `len == 0` 的情况。当 `len` 为负数时，该检查被绕过，程序继续向下执行，导致后续计算过程中发生越界访问。

#解决方案
修改入口校验条件，将 `if (unlikely(len == 0))` 变更为 `if (unlikely(len <= 0))`，确保当长度为 0 或负数时直接返回 0，从而避免越界访问。
