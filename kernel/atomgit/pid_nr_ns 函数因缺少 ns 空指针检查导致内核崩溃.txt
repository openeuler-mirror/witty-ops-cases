#标题
pid_nr_ns 函数因缺少 ns 空指针检查导致内核崩溃

#内核版本
OLK-6.6

#问题现象
当调用 `pid_nr_ns` 函数时，若传入的命名空间参数 `ns` 为 NULL，内核将触发空指针解引用异常，导致系统 panic 或 Oops。

#主要错误日志
未知

#问题根因
函数 `pid_nr_ns` 在校验条件中直接访问了 `ns->level`，未预先判断 `ns` 指针是否为空。当 `ns` 为 NULL 时，逻辑运算短路机制未生效，导致对 NULL 指针进行成员访问从而触发异常。

#解决方案
在 `pid_nr_ns` 函数的条件判断语句中增加对 `ns` 指针的非空检查，将原判断条件 `if (pid && ns->level <= pid->level)` 修改为 `if (pid && ns && ns->level <= pid->level)`，确保在访问 `ns->level` 前确认 `ns` 指针有效。
