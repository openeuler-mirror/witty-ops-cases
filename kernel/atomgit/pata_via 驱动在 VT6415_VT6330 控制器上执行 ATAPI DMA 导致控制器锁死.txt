#标题
pata_via 驱动在 VT6415/VT6330 控制器上执行 ATAPI DMA 导致控制器锁死

#内核版本
OLK-6.6

#问题现象
在搭载 VIA VT6415 或 VT6330 芯片的系统上，当连接 ATAPI 设备（如光驱）并尝试进行 DMA 传输时，控制器会发生锁死，导致数据传输中断或系统无响应。

#主要错误日志
[ATA device]: controller locks up on ATAPI DMA, forcing PIO

#问题根因
VIA VT6415（以及 VT6330，二者 ID 相同）控制器在处理 ATAPI 设备的 DMA 传输时存在硬件缺陷，会导致控制器锁死。原代码逻辑中未将该特定型号 ID 加入 DMA 传输的禁止检查列表中。

#解决方案
修改 `drivers/ata/pata_via.c` 文件中的 `via_mode_filter` 函数，在判断是否禁用 ATAPI DMA 的条件中增加对 `PCI_DEVICE_ID_VIA_6415` 的检查。如果检测到该控制器型号，强制 ATAPI 设备使用 PIO 模式传输，从而规避控制器锁死风险。
