#标题
hrtimer: Fix softirq base check in update_needs_ipi()

#内核版本
OLK-6.6

#问题现象
hrtimer 子系统在调用 update_needs_ipi() 函数检查是否需要发送 IPI（处理器间中断）时，由于对 softirq clock bases（软中断时钟基）的判断逻辑错误，导致判断条件失效。这可能导致本应触发的 IPI 被错误跳过，或者引发不符合预期的计时器行为。

#主要错误日志
未知

#问题根因
在 update_needs_ipi() 函数中，检查 softirq base 的逻辑存在错误。代码原本使用 base->clockid 字段与 HRTIMER_BASE_MONOTONIC_SOFT 进行比较来判断是否为软中断基，但实际上 clockid 并不是用于区分 base 类型的正确字段。正确的做法应该是检查 base->index 字段。

#解决方案
修改 kernel/time/hrtimer.c 文件中 update_needs_ipi() 函数的判断逻辑，将判断条件从 `base->clockid < HRTIMER_BASE_MONOTONIC_SOFT` 修改为 `base->index < HRTIMER_BASE_MONOTONIC_SOFT`，通过正确的 index 字段来识别 softirq clock bases。
