#标题
sched_ext: Fix kabi for switch_class in struct sched_class

#内核版本
OLK-6.6

#问题现象
内核在开启 CONFIG_SCHED_EXT 配置编译时，或者在使用 struct sched_class 结构体进行 KABI 兼容性检查时，可能触发编译错误或 KABI 校验失败。此外，直接修改核心结构体 sched_class 可能破坏内核二进制接口（KABI）稳定性，导致依赖旧结构体布局的内核模块（如驱动程序）加载失败或运行时异常。

#主要错误日志
未知

#问题根因
在引入 sched_ext 特性的补丁中，直接在 struct sched_class 结构体内新增了 `void (*switch_class)(struct rq *rq, struct task_struct *next)` 函数指针字段，未遵循内核 KABI（Kernel Application Binary Interface）兼容性规范。直接在结构体中间新增字段会改变结构体大小和成员偏移量，破坏了二进制兼容性。

#解决方案
利用 KABI 占位符机制修复该问题。删除直接添加的 `switch_class` 字段，改为使用 `KABI_EXTEND` 宏在结构体末尾的安全区域进行扩展。该宏允许在不破坏既有结构体布局的前提下新增成员，从而保证内核二进制接口的稳定性。
