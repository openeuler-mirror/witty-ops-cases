#标题
scsi: core: sysfs 属性 supported_mode 与 active_mode 权限设置错误

#内核版本
OLK-6.6

#问题现象
SCSI host 设备的 sysfs 接口 `supported_mode` 和 `active_mode` 在实际为只读属性（未实现 store 方法）的情况下，错误地向用户空间暴露了写权限（S_IWUSR）。虽然尝试写入不会实际更改硬件状态，但错误的权限位可能误导用户或管理工具认为这些属性是可配置的，可能导致不必要的写入尝试或权限检查逻辑异常。

#主要错误日志
未知

#问题根因
在定义 `supported_mode` 和 `active_mode` 这两个 sysfs 属性时，`DEVICE_ATTR` 宏错误地包含了 `S_IWUSR` 标志。由于这两个属性仅实现了 `show` 回调函数（`show_shost_supported_mode` 和 `show_shost_active_mode`），而 `store` 回调函数被设为 `NULL`，因此它们本质上是只读属性，不应包含写权限标志。

#解决方案
修正 `drivers/scsi/scsi_sysfs.c` 中的代码，将 `supported_mode` 和 `active_mode` 属性的定义从 `S_IRUGO | S_IWUSR` 修改为 `S_IRUGO`，移除错误的写权限标志，确保 sysfs 接口权限与实际功能（只读）一致。
