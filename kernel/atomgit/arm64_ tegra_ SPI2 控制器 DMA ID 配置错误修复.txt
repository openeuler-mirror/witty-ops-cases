#标题
arm64: tegra: SPI2 控制器 DMA ID 配置错误修复

#内核版本
OLK-6.6

#问题现象
Tegra234 平台 SPI2 控制器在使能 DMA 模式进行数据传输时功能异常，无法正常工作。

#主要错误日志
未知

#问题根因
设备树中 SPI2 节点的 DMA ID 配置错误，代码中误写为 19，而 SPI2 实际对应的 DMA ID 应为 16。该错误自引入 SPI 设备树节点的提交 bb9667d8187b 起便存在。

#解决方案
修改 `arch/arm64/boot/dts/nvidia/tegra234.dtsi` 文件中 SPI2 节点的 `dmas` 属性，将 DMA ID 从错误的 19 更正为 16。
