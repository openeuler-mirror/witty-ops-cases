#标题
wifi: mt76: mt7915: Fix null-ptr-deref in mt7915_mmio_wed_init()

#内核版本
OLK-6.6

#问题现象
在调用 mt7915_mmio_wed_init 函数进行 WED（Wireless Ethernet Dispatcher）初始化时，若 devm_ioremap 映射失败，后续代码直接使用空的 wed->wlan.base 指针，导致内核发生空指针引用，引发系统崩溃或调用栈报错。

#主要错误日志
Unable to handle kernel NULL pointer dereference

#问题根因
在 mt7915_mmio_wed_init 函数中，针对 PCI 和 AXI 两种总线类型分别调用了 devm_ioremap 进行内存映射，但代码未对返回值进行有效性检查。当 ioremap 因资源不足或其他原因返回 NULL 时，程序继续向下执行并访问该无效地址，触发了空指针引用错误。

#解决方案
在 devm_ioremap 调用后立即增加返回值检查逻辑。如果 wed->wlan.base 为 NULL，函数立即返回 -ENOMEM 错误码，防止后续代码访问空指针，从而避免内核崩溃。
