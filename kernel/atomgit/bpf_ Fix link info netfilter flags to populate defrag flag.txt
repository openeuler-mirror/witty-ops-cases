#标题
bpf: Fix link info netfilter flags to populate defrag flag

#内核版本
OLK-6.6

#问题现象
在使用 BPF Netfilter Link 并设置了 `BPF_F_NETFILTER_IP_DEFRAG` 标志后，通过 `bpf_link_info` 查询链接信息时，`netfilter.flags` 字段返回值错误地显示为 0，未能正确反映内核实际启用的 IP 分片重组功能状态。

#主要错误日志
未知

#问题根因
在函数 `bpf_nf_link_fill_link_info` 中，代码直接将 `info->netfilter.flags` 硬编码赋值为 0，未根据链接实际的 `defrag_hook` 状态设置相应的标志位，导致用户空间无法通过 link info 获取正确的 defrag 配置信息。

#解决方案
修改 `bpf_nf_link_fill_link_info` 函数逻辑，不再硬编码赋值 0，而是检查 `nf_link->defrag_hook` 是否存在。若 `defrag_hook` 不为空，则将 `info->netfilter.flags` 设置为 `BPF_F_NETFILTER_IP_DEFRAG`，否则设置为 0，从而确保 link info 正确填充 defrag 标志。
