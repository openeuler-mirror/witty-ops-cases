#标题
oenetcls RPS 流分发与 IPv6 支持功能增强

#内核版本
OLK-6.6

#问题现象
该提交主要涉及功能增强与重构，无明确的内核崩溃或功能失效故障现象。根据代码变更推断，主要解决了以下潜在问题：
1.  loopback（回环）接口的 RPS（Receive Packet Steering）分流功能缺失或不完善，原逻辑未针对回环流量进行专门处理。
2.  IPv6 协议的流量分类规则（ntuple rule）配置不支持，导致 IPv6 流量无法进行硬件分流或软件加速。
3.  网络流量在不同 CPU 或 NUMA 节点间的分发策略不够灵活，缺乏针对性的优化策略。
4.  原有数据结构使用 `DECLARE_BITMAP` 可能存在对齐或空间浪费问题。

#主要错误日志
未知

#问题根因
该提交为功能增强型补丁，主要基于以下几点进行改进：
1.  **回环流量支持缺失**：原代码在 `netif_rx_internal` 等路径未针对 `ARPHRD_LOOPBACK` 类型设备做区分处理，无法利用 `oecls_set_localcpu` 钩子优化本地回环性能。
2.  **IPv6 支持不完整**：原有规则配置逻辑仅处理 IPv4 地址（`dip4`），缺少对 `dip6` 字段的设置和匹配逻辑，导致 IPv6 场景下功能受限。
3.  **分发策略局限性**：原有的 CPU 选择逻辑较简单，缺乏对 NUMA 节点、Cluster 以及特定概率模型的支持，无法满足复杂场景下的性能优化需求。
4.  **代码逻辑优化**：为了支持更复杂的 RPS 策略和 IPv6 地址存储，对相关数据结构和钩子函数进行了必要的重构和扩展。

#解决方案
1.  **增加回环接口 RPS 支持**：新增 `oecls_set_localcpu` 钩子及相关静态 Key `oecls_localrps_needed`，在 `netif_rx_internal` 中增加对 loopback 设备的判断逻辑，调用专用处理函数。
2.  **扩展 IPv6 支持**：
    *   在 `cmd_context` 和 `oecls_sk_rule` 结构体中增加 `dip6` 和 `is_ipv6` 字段。
    *   更新 `cfg_ethtool_rule` 及相关哈希计算逻辑，支持 IPv6 地址的解析、配置与匹配。
3.  **增强 RPS 分发策略**：
    *   新增 `mode=2` 模式及 `rps_policy`、`lo_rps_policy`、`rcpu_probability` 等模块参数，提供更细粒度的分发控制。
    *   引入 `get_rps_cpu` 函数，支持基于 NUMA 节点或 Cluster 的 CPU 选择策略。
    *   优化 `rxq` 分配逻辑，使用引用计数（`status`）替代简单的状态标记，支持队列复用。
4.  **重构数据结构**：
    *   将 `bitmap_rxq` 从位图改为 `unsigned char` 数组，用于存储使用计数。
    *   在 `sk_buff` 中使用 `KABI_USE` 引入 `sym_hash` 字段，优化哈希计算。
    *   更新 `oecls_flow_update` 接口，增加 `struct sk_buff *skb` 参数以获取更精确的流信息。
