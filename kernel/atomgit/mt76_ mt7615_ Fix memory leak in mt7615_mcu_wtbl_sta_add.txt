#标题
mt76: mt7615: Fix memory leak in mt7615_mcu_wtbl_sta_add

#内核版本
OLK-6.6

#问题现象
在调用 mt7615_mcu_wtbl_sta_add 函数添加无线站点时，若内部调用 mt76_connac_mcu_alloc_wtbl_req 失败，会导致内核内存泄漏。随着错误路径的频繁触发，系统可用内存会逐渐减少，可能影响系统稳定性。

#主要错误日志
未知

#问题根因
在函数 mt7615_mcu_wtbl_sta_add 中，代码预先分配了一个 skb 结构体。随后的 mt76_connac_mcu_alloc_wtbl_req 调用如果失败，函数直接返回了错误码，但未释放之前分配的 skb 内存，导致内存泄漏。

#解决方案
在 mt76_connac_mcu_alloc_wtbl_req 调用失败的处理路径中，增加对 sskb 的释放操作（调用 dev_kfree_skb(sskb)），确保在返回错误码前正确释放已分配的内存资源。
