#标题
lib/buildid: Build ID 解析逻辑错误导致获取错误的 Build ID 数据

#内核版本
OLK-6.6

#问题现象
内核在解析 Build ID 时，计算数据指针偏移量未包含 `Elf32_Nhdr` 头部大小，导致读取到错误的 Build ID 数据。该问题仅影响合入了特定修复补丁（c83a80d8b84f）的稳定版内核分支，上游代码因重构已规避此问题。

#主要错误日志
未知

#问题根因
函数 `parse_build_id_buf` 在计算 Build ID 数据的起始地址时，直接使用了 `note_start + note_off + ALIGN(note_name_sz, 4)`，遗漏了 `Elf32_Nhdr` 头部结构体本身的尺寸。正确的计算方式应包含该头部大小，即 `note_start + note_off + sizeof(Elf32_Nhdr) + ALIGN(note_name_sz, 4)`。

#解决方案
修改 `lib/buildid.c` 文件中的 `parse_build_id_buf` 函数，在计算 Build ID 数据指针偏移量时，显式加上 `sizeof(Elf32_Nhdr)`，确保指针跳过 Note Header 结构体，指向正确的数据区域。
