#标题
f2fs: truncate节点路径解析异常导致内存越界访问

#内核版本
OLK-6.6

#问题现象
在进行文件系统 truncate 操作时，若 inode 的 node block 处于损坏状态（level 计算为 0），内核未正确处理该异常情况，继续向下执行导致越界访问，可能触发内核 panic 或内存破坏。

#主要错误日志
f2fs_err(sbi, "%s: inode ino=%lx has corrupted node block, from:%lu addrs:%u", __func__, inode->i_ino, from, ADDRS_PER_INODE(inode));

#问题根因
函数 `f2fs_truncate_inode_blocks` 中调用 `get_node_path` 获取节点路径层级 `level`。原逻辑仅判断了 `level < 0` 的错误情况，忽略了 `level == 0` 的场景。当 node block 损坏导致 `level` 为 0 时，代码未中止流程，导致后续逻辑访问数组越界。

#解决方案
修改 `fs/f2fs/node.c` 中的判断逻辑，将 `if (level < 0)` 修改为 `if (level <= 0)`。当 `level` 为 0 时，将其设为 `-EFSCORRUPTED` 错误码，打印错误日志标记文件系统需要 FSCK 检查，并立即返回错误，防止越界访问。
