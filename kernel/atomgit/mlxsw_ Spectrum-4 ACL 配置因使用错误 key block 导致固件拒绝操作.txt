#标题
mlxsw: Spectrum-4 ACL 配置因使用错误 key block 导致固件拒绝操作

#内核版本
OLK-6.6

#问题现象
在 Spectrum-4 设备上，驱动程序使用了固件不支持的 ACL key block。在旧版固件上，由于驱动仅使用了该 key block 中位于特定偏移量的单个字段，该字段恰好与 Spectrum-4 等效 key block 的偏移量相同，因此问题被掩盖。当固件升级并开始严格执行检查、拒绝使用不支持的 key block 时，会导致 ACL 相关配置下发失败。

#主要错误日志
未知

#问题根因
驱动程序在 Spectrum-4 设备上错误地使用了 ACL key block（block 0x39，对应 `mlxsw_sp_afk_element_info_ipv4_1`），而该 key block 并不被 Spectrum-4 硬件支持。此前因字段偏移巧合而未暴露问题，直到固件更新后拒绝此配置。

#解决方案
修改代码以使用 Spectrum-4 支持的正确 key block。具体变更包括：
1. 将 `mlxsw_sp4_afk_blocks` 数组中原本使用的 block 0x39 替换为 block 0x3F。
2. 关联该 block 的元素实例从 `mlxsw_sp_afk_element_info_ipv4_1` 更改为新增的 `mlxsw_sp_afk_element_info_ipv4_1b`，该实例正确定义了 `SRC_IP_0_31` 字段，以匹配硬件规格。
