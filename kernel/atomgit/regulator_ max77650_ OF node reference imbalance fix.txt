#标题
regulator: max77650: OF node reference imbalance fix

#内核版本
OLK-6.6

#问题现象
在卸载（unbind）MAX77650 多功能设备（MFD）并注销其子设备时，由于 OF node 引用计数管理不当，可能导致内核引用计数失衡，进而引发潜在的内存管理问题或设备重新注册时的异常行为。

#主要错误日志
未知

#问题根因
驱动程序复用了父级多功能设备（MFD）的 OF node，但未增加引用计数。当 MFD 解绑并注销子设备时，平台总线代码会释放（drop）该 OF node 的一个引用。由于驱动直接赋值 `dev->of_node = parent->of_node` 而未获取新的引用，导致引用计数不平衡。

#解决方案
修改 `max77650_regulator_probe` 函数，将直接赋值操作 `dev->of_node = parent->of_node` 替换为辅助函数 `device_set_of_node_from_dev(dev, parent)`。该函数在复用 OF node 时会正确处理引用计数，从而平衡平台总线代码释放的引用。
