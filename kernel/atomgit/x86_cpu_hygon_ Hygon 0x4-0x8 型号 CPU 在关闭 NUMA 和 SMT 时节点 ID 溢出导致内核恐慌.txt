#标题
x86/cpu/hygon: Hygon 0x4-0x8 型号 CPU 在关闭 NUMA 和 SMT 时节点 ID 溢出导致内核恐慌

#内核版本
OLK-6.6

#问题现象
在 Hygon 平台（型号 0x4-0x8）的 BIOS 设置中同时关闭 NUMA 和 SMT 功能时，节点 ID（Node ID）计算发生溢出，导致系统在启动过程中触发内核恐慌。

#主要错误日志
未知

#问题根因
在 Hygon 型号 0x4 到 0x8 的 CPU 上，当 BIOS 中同时禁用 NUMA 和 SMT 时，原有的节点 ID 计算逻辑直接使用了 `c->topo.llc_id` 进行赋值，导致节点 ID 溢出，进而引发系统启动时的 kernel panic。

#解决方案
针对 Hygon 型号 0x4 到 0x8 的 CPU，修改 `srat_detect_node` 函数中的节点 ID 计算逻辑：在 `numa_cpu_node` 返回 `NUMA_NO_NODE` 的情况下，增加对 CPU 型号的判断，优先调用 `cpu_to_node(cpu)` 获取节点 ID 并通过 `numa_set_node` 设置，从而避免溢出；其他型号则保持原有逻辑。
