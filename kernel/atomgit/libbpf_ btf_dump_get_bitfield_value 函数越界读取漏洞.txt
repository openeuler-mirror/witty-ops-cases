#标题
libbpf: btf_dump_get_bitfield_value 函数越界读取漏洞

#内核版本
OLK-6.6

#问题现象
在调用 `btf_dump__dump_type_data()` API 进行 BTF 类型数据转储时，若提供的数据缓冲区（data_sz）小于底层数据类型的大小（t->size），程序在处理 bitfield 数据过程中触发越界读取。该问题可被 AddressSanitizer (ASan) 工具检测确认。

#主要错误日志
未知

#问题根因
函数 `btf_dump_get_bitfield_value()` 在读取 bitfield 数据时，仅依据底层数据类型的大小（t->size）来读取数据，未对提供的数据缓冲区大小进行校验。当传入的缓冲区 `data_sz` 小于类型大小时，程序继续读取超出缓冲区范围的内存，导致越界读取。

#解决方案
在 `btf_dump_get_bitfield_value()` 函数中增加边界检查逻辑。计算覆盖 bitfield 所需的字节数（nr_bytes），并判断读取位置是否超出数据缓冲区的结束地址（data_end）。若超出则直接返回 `-E2BIG` 错误，避免越界访问。
