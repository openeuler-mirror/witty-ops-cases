#标题
mpt3sas: IOCTL超时后Target Reset失败导致恢复流程不完整

#内核版本
OLK-6.6

#问题现象
当 IOCTL 命令发生超时，驱动程序尝试下发 Target Reset 进行复位恢复时，若固件侧的任务管理命令执行失败，驱动程序未能进一步提升恢复级别，导致控制器处于不稳定状态，IOCTL 命令无法恢复。

#主要错误日志
推断日志：
"target reset failed, issue hard reset: handle (0x%04x)\n"

#问题根因
在处理 IOCTL 超时场景时，代码逻辑中虽然调用了 Target Reset 函数，但未检查其返回值。当固件执行 Target Reset 失败时，驱动程序缺乏后续的升级处理机制，导致恢复流程中断，设备无法恢复正常工作。

#解决方案
在 IOCTL 超时处理逻辑中，增加对 Target Reset 函数返回值的检查。如果 Target Reset 返回失败，则提升恢复级别，调用 `mpt3sas_base_hard_reset_handler` 对控制器执行 diag reset（强制硬复位），确保故障能够被彻底恢复。
