#标题
sw64 KVM: kvm_arch_prepare_memory_region 函数 NULL 指针访问导致内核 Oops

#内核版本
OLK-6.6

#问题现象
在 sw64 架构下使用 KVM 虚拟化功能时，若进行内存槽删除（KVM_MR_DELETE）或仅修改标志位（KVM_MR_FLAGS_ONLY）的操作，内核触发 NULL 指针访问异常，导致宿主机内核 Oops。

#主要错误日志
kernel Oops

#问题根因
在函数 `kvm_arch_prepare_memory_region` 中，代码逻辑错误地在判断操作类型（`change == KVM_MR_FLAGS_ONLY || change == KVM_MR_DELETE`）之前访问了 `struct kvm_memory_slot *new` 指针成员。当操作类型为删除或仅修改标志时，`new` 指针为 NULL，导致解引用空指针引发故障。

#解决方案
将操作类型的判断逻辑提前执行。若检测到 `change` 为 `KVM_MR_FLAGS_ONLY` 或 `KVM_MR_DELETE`，函数直接返回 0，避免后续对 `new` 指针成员（如 `flags`、`base_gfn`、`npages` 等）的非法访问。同时删除了冗余的赋值代码。
