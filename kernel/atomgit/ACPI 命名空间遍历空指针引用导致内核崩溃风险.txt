#标题
ACPI 命名空间遍历空指针引用导致内核崩溃风险

#内核版本
OLK-6.6

#问题现象
当调用 `acpi_ns_walk_namespace` 函数时，如果传入的 `start_node` 参数为 `ACPI_ROOT_OBJECT` 且全局根节点 `acpi_gbl_root_node` 尚未初始化（为 NULL），或者直接传入 `start_node` 为 NULL 时，原有逻辑未能正确拦截，导致后续代码试图访问无效内存地址，引发内核 Panic 或 Oops。

#主要错误日志
未知

#问题根因
在函数 `acpi_ns_walk_namespace` 中，对 `start_node` 的 NULL 指针检查逻辑存在缺陷。原代码仅在 `start_node` 被赋值为 `ACPI_ROOT_OBJECT` 的分支内部检查 `acpi_gbl_root_node` 是否为 NULL。如果 `start_node` 在进入函数时即为 NULL 但不等于 `ACPI_ROOT_OBJECT`，或者检查逻辑被绕过，代码将继续执行并试图遍历以 NULL 为起点的命名空间，导致空指针引用。

#解决方案
将 NULL 指针检查逻辑移至条件分支外部，使其成为函数的通用检查步骤。修改后的代码无论 `start_node` 是否为 `ACPI_ROOT_OBJECT`，都会在后续流程中统一检查 `start_node` 是否为 NULL，若为 NULL 则直接返回 `AE_NO_NAMESPACE` 状态，阻止非法内存访问。
