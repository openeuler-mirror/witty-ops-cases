#标题
perf list 参数顺序错误导致 JSON 输出中 Topic 与 Unit 字段信息混淆

#内核版本
OLK-6.6

#问题现象
在 Intel Tigerlake 平台上执行 `perf list -j` 命令查看 JSON 格式输出时，"Topic" 与 "Unit" 字段显示的信息出现混淆。具体表现为 "Topic" 字段错误地显示了本应属于 "Unit" 的 PMU 名称（如 "cpu"），而 "Unit" 字段则错误地包含了 "cache" 等本应属于 "Topic" 的内容。

#主要错误日志
未知

#问题根因
函数定义 `default_print_event` 和 `json_print_event` 中的参数顺序被声明为 `(void *ps, const char *pmu_name, const char *topic, ...)`，但这与头文件中的声明不一致。同时，调用者 `print_libpfm_event` 和 `perf_pmus__print_pmu_events` 在调用这些函数时，传入的 `topic` 和 `pmu_name` 实参顺序颠倒，导致数据传递错位。

#解决方案
调整 `default_print_event` 和 `json_print_event` 函数定义中的参数顺序，将 `const char *topic` 移至 `const char *pmu_name` 之前，以匹配头文件声明及调用者的意图。同时修正 `util/pfm.c` 和 `util/pmus.c` 中调用处的实参顺序，确保 `topic` 和 `pmu_name` 正确传递。
