#标题
btrfs: tree-checker 修复 root drop_level 错误日志信息误导性问题

#内核版本
OLK-6.6

#问题现象
当 BTRFS 文件系统进行树检查（tree-checker）校验时，若 ROOT_ITEM 中的 `drop_level` 字段值非法（大于或等于 BTRFS_MAX_LEVEL），内核日志会打印一条错误信息。该错误信息将 `drop_level` 错误地标识为 `root level`，导致运维人员或开发者在排查问题时对故障变量产生误解。

#主要错误日志
invalid root level, have %u expect [0, %u]
（注：此为修复前的错误日志，修复后改为 "invalid root drop_level..."）

#问题根因
在 `check_root_item` 函数中，当检测到 `btrfs_root_drop_level` 值非法时，错误打印函数 `generic_err` 的格式化字符串中错误地使用了 "invalid root level" 描述，而实际检测与报错的变量是 `drop_level`。这是 commit 259ee7754b67 引入代码时的笔误。

#解决方案
修改 `fs/btrfs/tree-checker.c` 文件中的错误提示字符串，将 "invalid root level" 更正为 "invalid root drop_level"，确保错误日志准确反映实际出错的变量名称。
