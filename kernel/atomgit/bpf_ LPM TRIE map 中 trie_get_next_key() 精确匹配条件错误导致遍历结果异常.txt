#标题
bpf: LPM TRIE map 中 trie_get_next_key() 精确匹配条件错误导致遍历结果异常

#内核版本
OLK-6.6

#问题现象
调用 BPF LPM TRIE map 的 `trie_get_next_key()` 函数时，如果传入的 key 是最近被删除的 key 或者不存在的 key，函数可能会跳过某些 key，导致返回的下一个 key 结果不正确。

#主要错误日志
未知

#问题根因
`trie_get_next_key()` 函数中使用 `node->prefixlen == key->prefixlen` 来判断是否为精确匹配，该逻辑存在缺陷。当目标 key 与找到的节点未完全匹配时（即 `node->prefixlen != matchlen`），这两个节点可能仍然具有相同的 `prefixlen`。这种错误的判断条件导致了遍历逻辑错误，进而返回错误的结果。

#解决方案
修改 `kernel/bpf/lpm_trie.c` 中的判断逻辑，将精确匹配的判断条件从 `node->prefixlen != key->prefixlen` 修正为 `node->prefixlen != matchlen`，并初始化变量 `matchlen` 为 0，确保在搜索完成后能正确识别精确匹配情况。
