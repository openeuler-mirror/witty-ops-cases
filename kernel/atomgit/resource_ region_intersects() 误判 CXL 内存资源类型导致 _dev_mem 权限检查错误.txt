#标题
resource: region_intersects() 误判 CXL 内存资源类型导致 /dev/mem 权限检查错误

#内核版本
OLK-6.6

#问题现象
在配置了 CXL 内存的系统中，通过 /dev/mem 写入 CXL 内存范围时，虽然写入操作最终被阻止，但报错信息不正确。系统返回 "Bad address" 错误，而非预期的 "Operation not permitted"。
更严重的是，devmem_is_allowed() 函数错误地通过了权限检查，允许了对该内存范围的访问尝试。虽然后续因 ioremap() 拒绝映射系统 RAM 而阻止了实际写入，但这构成了潜在的安全风险。

#问题根因
在使用 add_memory_driver_managed() 将 CXL 内存上线时，资源树（/proc/iomem）中的 "System RAM" 会成为 "CXL Window" 的子节点，而非顶层资源。
region_intersects() 函数在检查指定范围内是否存在特定类型资源（如 System RAM）时，原逻辑仅遍历顶层资源。由于 "System RAM" 位于子层级，导致该函数错误地报告 CXL 内存区域无 System RAM，进而导致 devmem_is_allowed() 返回 1（允许访问），引发权限检查失效及后续的 ioremap 错误。

#解决方案
修改 region_intersects() 函数逻辑，使其能够正确处理非顶层的资源节点。在遍历顶层资源时，如果发现有不匹配的资源覆盖了目标范围，继续递归搜索其子节点（descendant resources）。通过这种方式，确保不会遗漏资源树中任意层级的匹配资源，从而正确识别 CXL 内存区域中的 System RAM 类型，修复权限检查和返回值错误。
