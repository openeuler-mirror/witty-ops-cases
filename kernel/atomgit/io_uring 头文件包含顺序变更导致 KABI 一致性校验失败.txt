#标题
io_uring 头文件包含顺序变更导致 KABI 一致性校验失败

#内核版本
OLK-6.6

#问题现象
在引入将 io_uring cmd API 分离到独立头文件的修改后，内核 ABI（KABI）完整性检查出现异常或校验结果发生变化，导致 KABI 扩展计算结果与预期不符，影响内核版本的 ABI 稳定性保障。

#主要错误日志
未知

#问题根因
在提交 b66509b8497f 将 io_uring cmd API 分离到独立头文件后，改变了原有头文件的包含顺序。这种包含顺序的改变影响了 KABI 扩展的计算逻辑，导致计算结果发生变化，从而破坏了 KABI 的一致性。

#解决方案
调整头文件的包含顺序以修复 KABI 问题。具体措施包括：
1. 在 `include/linux/io_uring.h` 中直接包含 `linux/io_uring/cmd.h`。
2. 将 `io_uring_cmd_flags` 枚举和 `IOU_F_TWQ_LAZY_WAKE` 相关定义从 `include/linux/io_uring_types.h` 移动到 `include/linux/io_uring/cmd.h` 中。
通过这些调整，确保头文件包含层级和顺序正确，恢复 KABI 扩展计算的预期结果。
