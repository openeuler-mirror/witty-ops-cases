#标题
most: core: fix resource leak in most_register_interface error paths

#内核版本
OLK-6.6

#问题现象
在 `most_register_interface` 函数执行过程中，若在设备注册之前的早期阶段（如分配设备 ID 失败或分配私有结构体内存失败）发生错误，函数直接返回错误码，导致为接口分配的内存资源泄露，且未正确释放相关资源。

#主要错误日志
- "Failed to allocate device ID\n"
- "Failed to register interface device\n"

#问题根因
函数 `most_register_interface()` 在早期错误路径（如 `ida_alloc` 失败或 `kzalloc` 失败）中直接返回错误码，而没有调用 `put_device()` 来释放已分配的资源。根据设计，调用者期望通过 `put_device()` 来触发资源释放（调用 `release_mdev` 或 `dim2_release`），缺失该调用导致内存泄露。

#解决方案
1. 将 `device_initialize()` 调用提前，以便在早期错误路径中即可通过 `put_device()` 释放资源。
2. 在 `ida_alloc` 失败和 `kzalloc` 失败的错误处理分支中增加 `put_device(iface->dev)` 调用。
3. 将原来的 `device_register()` 调用替换为 `device_add()`，以适配分离初始化（`device_initialize` + `device_add`）的注册模式，确保资源管理逻辑一致。
