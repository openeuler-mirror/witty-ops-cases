#标题
btrfs: 追踪 inode 驱逐时因 root 为 NULL 导致的空指针解引用

#内核版本
OLK-6.6

#问题现象
在 BTRFS 文件系统执行 inode 驱逐操作时，若对应的 root 对象为 NULL，系统在设置追踪点的过程中会触发空指针解引用，导致内核崩溃或 Oops。

#主要错误日志
未知

#问题根因
在 `btrfs_evict_inode()` 函数中，`root` 指针可能为 NULL。但在设置 inode 追踪点时，代码直接解引用了 `BTRFS_I(inode)->root->root_key.objectid` 而未做非空检查，导致了 NULL 指针解引用。

#解决方案
修改 `include/trace/events/btrfs.h` 中的追踪点定义，在获取 `root_objectid` 时增加对 `BTRFS_I(inode)->root` 的非空判断：若 root 指针不为 NULL 则获取其 ID，否则将 `root_objectid` 设置为 0，从而避免空指针访问。
