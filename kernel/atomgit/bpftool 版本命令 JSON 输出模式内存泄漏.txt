#标题
bpftool 版本命令 JSON 输出模式内存泄漏

#内核版本
OLK-6.6

#问题现象
执行 `bpftool --version -j` 或 `bpftool --version -p` 命令时，会发生内存泄漏（memory leak），每次执行该命令都会泄漏一个 JSON writer 对象的内存。

#主要错误日志
未知

#问题根因
在 `main.c` 的主函数中处理 `version_requested` 逻辑时，原代码直接 `return do_version(argc, argv);` 返回，导致跳过了函数末尾的 `jsonw_destroy(&json_wtr)` 清理逻辑。这意味着在使用 JSON 输出格式（`-j` 或 `-p` 参数）执行版本命令时，JSON writer 对象没有被正确销毁，从而引起内存泄漏。

#解决方案
修改 `main.c` 中的控制流逻辑，将 `do_version` 的返回值赋给 `ret` 变量，并移除立即返回的语句。这确保了程序在执行完版本命令后，能够继续执行后续的 `jsonw_destroy(&json_wtr)` 代码，正确释放 JSON writer 资源。
