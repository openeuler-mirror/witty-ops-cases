#标题
build_id_parse 处理 memfd_secret 文件导致内核缺页异常

#内核版本
OLK-6.6

#问题现象
当系统调用 `build_id_parse` 处理通过 `memfd_secret()` 创建的 VMA 时，触发 "unable to handle kernel paging request" 异常，导致内核崩溃或 Oops。

#主要错误日志
BUG: unable to handle kernel paging request in build_id_parse

#问题根因
`build_id_parse` 函数在处理 VMA 时，未对 `memfd_secret()` 创建的特殊文件映射进行判断。该函数直接尝试访问页映射（page mapping）来获取 Build ID，但 `memfd_secret()` 创建的内存区域受特殊保护，不允许常规的内核访问，导致访问非法地址引发缺页异常。

#解决方案
在 `lib/buildid.c` 的 `build_id_parse` 函数中增加对 `secretmem` 映射的检查。如果检测到 VMA 的映射属于 `secretmem`（通过 `secretmem_mapping(vma->vm_file->f_mapping)` 判断），则直接返回 `-EFAULT` 错误，避免后续非法内存访问。同时，将 `mm/gup.c` 中的 `folio_is_secretmem` 与 `folio_fast_pin_allowed` 逻辑合并重构为 `gup_fast_folio_allowed`，以统一处理 GUP 快速路径下的权限检查，确保在 GUP 流程中也正确拒绝 secretmem 页面。
