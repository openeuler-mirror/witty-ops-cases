#标题
rcu/nocb: 离线 CPU 从 softirq 唤醒 rcuog 触发 smp_call_function_single 警告

#内核版本
OLK-6.6

#问题现象
当 CPU 设置自身为 offline 状态后，且在调用 rcutree_report_cpu_dead() 之前，若此时有回调函数入队（例如来自 softirq），会导致内核触发 WARNING 并打印调用栈。该问题发生在 NOCB（No-callback）模式下，尝试通过 IPI 唤醒在线 CPU 上的 rcuog 时。

#问题根因
在 CPU 下线过程中，此时 CPU 已标记为 offline，但尚未执行 `rcutree_report_cpu_dead()`。如果此时发生回调入队（例如由 softirq 触发），原有代码逻辑会尝试直接唤醒 rcuog。由于此时 CPU 处于 offline 状态，不应直接调用调度器相关接口，代码试图通过 `swake_up_one_online` 执行同步 IPI 来唤醒在线 CPU 上的 rcuog。然而，在 softirq 上下文中执行同步 IPI（`smp_call_function_single`）是不合法的，会导致内核触发 WARNING 并打印调用栈。

#解决方案
修改 `kernel/rcu/tree_nocb.h` 中的 `__call_rcu_nocb_wake` 函数逻辑。在判断是否直接唤醒 rcuog 时，增加对 CPU 状态的检查。如果当前 CPU 不在线（`!cpu_online(rdp->cpu)`），则强制通过 NOCB 定时器延迟唤醒。这样避免了在离线 CPU 的 softirq 上下文中调用 `swake_up_one_online`，实际的唤醒操作将延迟至 `rcutree_report_cpu_dead()` 中安全执行。
