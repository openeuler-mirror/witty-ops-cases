#标题
cpufreq: loongson3-acpi 初始化 scaling_cur_freq 值为 0 导致频率显示异常

#内核版本
OLK-6.6

#问题现象
在 LoongArch 平台上使用 loongson3-acpi cpufreq 驱动时，CPU 初始化期间 `policy->cur` 未被正确初始化，导致其值始终报告为 0。这造成系统无法正确获取和显示当前的 CPU 运行频率，用户查询 `scaling_cur_freq` 接口时返回 0，无法反映 CPU 实际的工作频率。

#主要错误日志
未知

#问题根因
驱动在 `loongson3_cpufreq_cpu_init` 函数中初始化 cpufreq_policy 时，遗漏了对 `policy->cur` 字段的赋值。根据 Commit 描述，这导致该字段默认值为 0，使得内核无法正确反馈 CPU 当前的运行频率状态。

#解决方案
在 `loongson3_cpufreq_cpu_init` 函数中添加对 `policy->cur` 的显式初始化代码。具体修改为将 `policy->cur` 设置为 `core->normal_max_freq * 1000`（转换为 KHz），确保初始化阶段当前频率值被正确设置为 CPU 的常规最高频率，从而反映真实的 CPU 运行状态。
