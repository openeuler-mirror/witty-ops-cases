#标题
netdevsim 自测试 rtnetlink.sh 因邻居表项缺失导致失败

#内核版本
OLK-6.6

#问题现象
在运行 netdevsim 的 rtnetlink.sh 自测试脚本时，`kci_test_ipsec_offload` 测试项失败。由于代码逻辑中未创建对端节点（peer），且缺少邻居表项配置，导致 ping 测试无法发出数据包，无法正常执行 Tx 路径验证。

#主要错误日志
未知

#问题根因
测试脚本 `kci_test_ipsec_offload` 在执行 ping 命令验证发送路径前，没有为目标 IP 地址创建邻居表项。因为没有对应的 peer 设备，内核无法解析目标 MAC 地址，导致 ping 无法发包，测试无法达到验证 Tx 路径的目的。

#解决方案
在执行 ping 命令前，显式添加一条静态邻居表项。修改代码在 ping 前增加 `ip neigh add $dstip dev $dev lladdr 00:11:22:33:44:55`，手动指定目标 IP 对应的链路层地址，确保 ping 包能正常发出以测试 Tx 路径。
