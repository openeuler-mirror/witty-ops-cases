#标题
wifi: wlcore: 修复固件启动失败时 Runtime PM usage count 下溢问题

#内核版本
OLK-6.6

#问题现象
在 wlcore 驱动加载过程中，若固件启动失败，系统日志中出现运行时电源管理引用计数下溢的错误告警，具体表现为 `Runtime PM usage count underflow!`。

#主要错误日志
[12092.708099] wlcore: ERROR firmware boot failed despite 3 retries
[12092.708099] wl18xx_driver wl18xx.1.auto: Runtime PM usage count underflow!

#问题根因
函数 `wl1271_op_add_interface` 在执行流程中存在逻辑缺陷：在调用 `pm_runtime_get_sync` 获取引用计数之前，代码路径中有多个错误处理分支直接跳转至 `out` 标签。`out` 标签处执行了 `pm_runtime_put_noidle` 操作。因此，当固件启动失败或早期检查出错时，代码未执行 `get` 操作却执行了 `put` 操作，导致引用计数不平衡，进而引发 usage count 下溢。

#解决方案
修改 `wl1271_op_add_interface` 函数的错误跳转逻辑。将调用 `pm_runtime_get_sync` 之前的所有错误处理分支（包括 `WL1271_FLAG_RECOVERY_IN_PROGRESS` 检查、`wl12xx_init_vif_data` 失败、`WL12XX_INVALID_ROLE_TYPE` 检查、`wlcore_allocate_hw_queue_base` 失败以及 `wl12xx_init_fw` 失败）的目标标签从 `out` 修改为 `out_unlock`。`out_unlock` 标签仅执行解锁操作而不执行 `pm_runtime_put_noidle`，从而避免了未获取引用计数却释放引用计数的错误。
