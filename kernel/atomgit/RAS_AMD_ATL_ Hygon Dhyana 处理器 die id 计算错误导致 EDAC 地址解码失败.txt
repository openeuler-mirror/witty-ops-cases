#标题
RAS/AMD/ATL: Hygon Dhyana 处理器 die id 计算错误导致 EDAC 地址解码失败

#内核版本
OLK-6.6

#问题现象
在 Hygon Dhyana 处理器（非 family 18h model 4h）上，使用 AMD Address Translation Library (ATL) 将 UMC MCA 地址转换为系统地址时失败，导致 EDAC 模块无法正确解码内存错误地址。

#主要错误日志
未知

#问题根因
在 `get_die_id` 函数中获取 die id 时，错误地调用了 `amd_get_nodes_per_socket()` 函数来获取 "nodes_per_socket" 的值。对于 Hygon CPU，该函数返回错误的数值，导致计算出的 die id 不正确，进而导致后续地址转换失败。

#解决方案
在 `arch/x86/kernel/cpu/hygon.c` 中新增并导出专用函数 `hygon_get_nodes_per_socket()`。修改 `drivers/ras/amd/atl/umc.c` 中的 `get_die_id` 逻辑，判断当前 CPU 供应商为 Hygon 时，调用 `hygon_get_nodes_per_socket()` 以获取正确的 nodes_per_socket 值，从而修正 die id 的计算。
