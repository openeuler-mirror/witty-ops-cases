#标题
rapidio: 修复 rio_add_net() 失败时的 API 误用及潜在 UAF 问题

#内核版本
OLK-6.6

#问题现象
在 RapidIO 子系统中，当 `rio_add_net()` 函数执行失败时，系统未正确释放设备资源。这导致了对设备内存的 API 误用（直接调用 kfree 释放已初始化的设备对象），并可能引发 Use-After-Free（释放后使用）问题。

#主要错误日志
rmcd_debug(RDEV, "failed to register net, err=%d", err);

#问题根因
`rio_add_net()` 内部调用了 `device_register()`，当注册失败时，设备的引用计数已经被初始化。原代码逻辑在失败路径中直接调用 `kfree()` 释放内存，而没有使用 `put_device()` 来正确减少引用计数并释放资源，这违反了 Linux 设备模型的 API 使用规范。此外，释放后 `mport->net` 指针未被清零，导致后续可能发生 Use-After-Free。

#解决方案
修改错误处理路径：
1. 将错误的 `kfree(net)` 替换为 `put_device(&net->dev)`，以确保设备引用计数正确处理并安全释放资源。
2. 添加 `mport->net = NULL` 语句，防止指针悬挂引发的 Use-After-Free 风险。
