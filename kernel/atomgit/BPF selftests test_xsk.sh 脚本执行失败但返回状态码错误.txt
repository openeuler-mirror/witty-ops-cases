#标题
BPF selftests test_xsk.sh 脚本执行失败但返回状态码错误

#内核版本
OLK-6.6

#问题现象
执行 BPF 自测脚本 `test_xsk.sh` 时，即使部分子测试用例失败，脚本最终仍打印 "ok 1 selftests: bpf: test_xsk.sh" 并以状态码 0 退出，导致测试框架误判测试通过，无法正确检测到测试失败。

#主要错误日志
未知

#问题根因
脚本 `test_xsk.sh` 的逻辑中，虽然定义了 `failures` 变量来记录失败次数，但在脚本末尾判断时，仅处理了 `failures -eq 0` 的情况，缺少对 `failures` 不为 0 时的处理逻辑，导致脚本默认以状态码 0 退出。

#解决方案
修改 `tools/testing/selftests/bpf/test_xsk.sh` 文件，在脚本末尾的判断逻辑中增加 `else` 分支：当 `failures` 计数不为 0 时，显式调用 `exit 1` 返回失败状态码，确保测试失败能被正确上报。
