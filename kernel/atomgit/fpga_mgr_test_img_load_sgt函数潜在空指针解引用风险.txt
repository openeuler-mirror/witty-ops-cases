#标题
fpga_mgr_test_img_load_sgt函数潜在空指针解引用风险

#内核版本
OLK-6.6

#问题现象
在运行 FPGA Manager 的 KUnit 测试用例 `fpga_mgr_test_img_load_sgt` 时，如果内部的 `kunit_kzalloc` 内存分配失败，内核会触发空指针解引用，导致系统 panic 或 oops。

#主要错误日志
BUG: kernel NULL pointer dereference

#问题根因
函数 `fpga_mgr_test_img_load_sgt()` 使用 `kunit_kzalloc()` 为 `sgt` 指针分配内存，但未检查分配是否成功。随后代码将 `sgt` 传递给 `sg_alloc_table()`，该函数内部调用 `__sg_alloc_table()` 时会对其执行 `memset()` 清零操作。若分配失败导致 `sgt` 为 NULL，`memset()` 操作将引发空指针解引用。

#解决方案
在调用 `sg_alloc_table()` 之前，使用 `KUNIT_ASSERT_NOT_ERR_OR_NULL()` 宏对 `sgt` 指针进行检查，确保内存分配成功，若失败则立即终止测试用例执行，避免后续发生空指针解引用。
