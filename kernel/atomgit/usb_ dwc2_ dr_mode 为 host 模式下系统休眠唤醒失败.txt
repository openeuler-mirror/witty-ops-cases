#标题
usb: dwc2: dr_mode 为 host 模式下系统休眠唤醒失败

#内核版本
OLK-6.6

#问题现象
当 dwc2 控制器的 dr_mode 配置为 host 时，系统从挂起到内存（S2RAM）状态唤醒后，USB 控制器无法作为主机模式恢复工作。进一步调试显示，控制器被错误地恢复为外设模式而非主机模式。

#主要错误日志
未知

#问题根因
commit 13b1f8e25bfd1 ("usb: dwc2: Force mode optimizations") 删除了当 dr_mode 为 host 时 `dwc2_force_dr_mode()` 函数中的 `dwc2_force_mode(hsotg, true)` 调用。这导致在从 S2RAM 唤醒时，由于 GINTSTS 寄存器默认值为 0，`dwc2_resume()` 中的 `dwc2_is_device_mode` 误判控制器处于外设模式，从而导致唤醒失败。

#解决方案
在 `dwc2_force_dr_mode()` 函数中，针对 `USB_DR_MODE_HOST` 分支，恢复添加 `dwc2_force_mode(hsotg, true)` 调用，强制控制器进入主机模式。
