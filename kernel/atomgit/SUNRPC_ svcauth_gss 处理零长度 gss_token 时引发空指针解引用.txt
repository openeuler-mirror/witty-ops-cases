#标题
SUNRPC: svcauth_gss 处理零长度 gss_token 时引发空指针解引用

#内核版本
OLK-6.6

#问题现象
在处理零长度的 gss_token 时，内核会发生空指针解引用。尽管拷贝长度为 0，代码仍会无条件执行 `page_address(in_token->pages[0])` 获取地址参数，此时 `in_token->pages[0]` 为 NULL，从而导致内核崩溃或异常行为。

#主要错误日志
未知

#问题根因
当 gss_token 长度为零时，`pages` 计数为 0，导致 `in_token->pages[0]` 指针为 NULL。代码在执行 `memcpy` 前无条件调用 `page_address(in_token->pages[0])` 来获取目标地址，触发了对 NULL 指针的解引用。

#解决方案
在执行 `memcpy` 操作前增加对 `length` 的判断。只有当 `length` 大于 0 时，才执行 `memcpy` 操作，从而避免了对零长度 token 情况下 NULL 指针的解引用。
