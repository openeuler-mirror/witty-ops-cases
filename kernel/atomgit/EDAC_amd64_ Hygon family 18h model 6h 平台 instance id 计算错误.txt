#标题
EDAC/amd64: Hygon family 18h model 6h 平台 instance id 计算错误

#内核版本
OLK-6.6

#问题现象
在 Hygon family 18h model 6h 平台上进行地址翻译时，由于 instance id 计算错误，导致无法正确访问 DF 寄存器，进而影响内存错误的定位与处理逻辑。

#主要错误日志
未知

#问题根因
在 Hygon family 18h model 6h 平台上，每个 DDR 拥有两个子通道（subchannels），每个子通道对应一个 MCA bank 和一个 CS（Chip Select）。在进行地址翻译时，需要正确的 CS instance id 才能访问 DF 寄存器。原有代码仅使用 `err.channel << 1` 计算 instance id，遗漏了对 IPID[13] 位的判断。实际上，instance id 应由 IPID[23:20]（UMC 控制器编号）和 IPID[13]（子通道编号）共同决定，计算公式缺失了 IPID[13] 这一关键位，导致计算出的 ID 不正确。

#解决方案
修正 instance id 的计算逻辑，在原有基础上增加 IPID[13] 位的提取与计算。将代码从 `umc = err.channel << 1` 修改为 `umc = (err.channel << 1) + ((m->ipid & BIT(13)) >> 13)`，确保正确识别 UMC 控制器所属的子通道。
