#标题
net: mvpp2: cls: 非 IP 带 VLAN 标签流定义错误导致 flow-type 规则失效

#内核版本
OLK-6.6

#问题现象
使用 ethtool 配置基于 VLAN 优先级的 Non IP flow 规则时，规则无法按预期工作。具体命令示例如下：
`ethtool -N eth1 flow-type ether vlan 0x8000 vlan-mask 0x1fff action 0 loc 0`

#主要错误日志
未知

#问题根因
在 `mvpp2_cls.c` 的流定义表中，Non IP 带 VLAN 标签的流定义（`MVPP2_DEF_FLOW`）使用了错误的 Key 提取选项 `MVPP22_CLS_HEK_OPT_VLAN`。该宏定义不匹配预期的数据包特征，导致硬件解析器无法正确识别和处理带 VLAN 标签的非 IP 报文。

#解决方案
修改 `drivers/net/ethernet/marvell/mvpp2/mvpp2_cls.c` 文件中的流定义，将 Non IP 带 VLAN 标签流的参数从 `MVPP22_CLS_HEK_OPT_VLAN` 修正为 `MVPP22_CLS_HEK_TAGGED`，以正确匹配带 VLAN 标签的报文特征。
