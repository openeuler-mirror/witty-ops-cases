#标题
Omoton KB066键盘F6键在Linux下映射为KEY_UNKNOWN导致按键失效

#内核版本
OLK-6.6

#问题现象
Omoton KB066键盘（Apple A1255仿制款）的F6按键在未按住Fn键单独按下时，无法正常触发F6功能。该键盘在Amazon上有大量评测，属于常见硬件。由于键盘硬件发送了非标准的HID usage code，Linux内核将其翻译为`KEY_UNKNOWN`，导致按键功能丢失。

#主要错误日志
未知

#问题根因
Omoton KB066键盘存在硬件设计特殊行为（quirk）。当F6键被按下（未按Fn）时，它发送了保留区的Consumer Page usage code `0xC0301`，而不是标准的Keyboard Page usage code `0x7003F`。内核HID驱动无法识别该非标准码，将其映射为`KEY_UNKNOWN`，导致按键无效。

#解决方案
在内核HID Apple驱动（`drivers/hid/hid-apple.c`）的 `hidinput_apple_event` 函数中增加特殊处理逻辑：检测到 usage code 为 `0xc0301` 时，强制将其映射为 `KEY_F6`，从而修复该键盘的按键功能。
