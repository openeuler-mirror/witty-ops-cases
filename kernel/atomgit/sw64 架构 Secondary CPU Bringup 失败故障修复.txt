#标题
sw64 架构 Secondary CPU Bringup 失败故障修复

#内核版本
OLK-6.6

#问题现象
在 sw64 架构下启动从 CPU（secondary CPU）时，可能出现从 CPU 唤醒失败的问题，导致从 CPU 无法正常上线。

#主要错误日志
未知

#问题根因
在原有代码逻辑中，`reset_cpu` 函数在设置 `smp_rcb->ready` 标志位之前被调用。如果在发送复位信号（reset signal）后、从 CPU 接收到信号前，`smp_rcb->ready` 被置位，会导致从 CPU 唤醒流程出现竞争或状态异常，从而导致唤醒失败。

#解决方案
调整 `reset_cpu` 的调用时机，将其移动到 `set_secondary_ready` 设置就绪标志位之后执行。这确保了从 CPU 在接收到复位信号时，相关的状态标志位已正确设置，从而保证从 CPU 能被正确唤醒并成功上线。
