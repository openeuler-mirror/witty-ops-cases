#标题
x86/sched: 修复 arch_sched_node_distance 函数缺少原型声明的编译警告

#内核版本
OLK-6.6

#问题现象
在编译内核时，编译器报出函数缺少原型声明的警告：
arch/x86/kernel/smpboot.c:548:5: warning: no previous prototype for function 'arch_sched_node_distance' [-Wmissing-prototypes]

#主要错误日志
arch/x86/kernel/smpboot.c:548:5: warning: no previous prototype for function 'arch_sched_node_distance' [-Wmissing-prototypes]

#问题根因
在之前的提交（"sched/topology: Fix sched domain build error for GNR, CWF in SNC-3 mode"）中引入了 `arch_sched_node_distance` 函数，但未在对应的头文件中添加函数原型声明，导致编译器在处理 smpboot.c 时无法找到该函数的原型定义，从而触发 `-Wmissing-prototypes` 编译警告。

#解决方案
在 `arch/x86/include/asm/topology.h` 头文件中补充该函数的外部声明：
`extern int arch_sched_node_distance(int from, int to);`
