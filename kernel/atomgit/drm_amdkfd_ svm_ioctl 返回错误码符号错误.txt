#标题
drm/amdkfd: svm_ioctl 返回错误码符号错误

#内核版本
OLK-6.6

#问题现象
在 AMD KFD 驱动的 SVM（共享虚拟内存）子系统中，当用户程序调用 `svm_ioctl` 并传入一个无效或不支持的 ioctl 操作码（触发 default 分支）时，函数返回了一个正数的错误码 `EINVAL`（22），而不是符合 Linux 内核 API 规范的负数错误码 `-EINVAL`。这违反了内核函数返回值约定，可能导致调用者无法正确判断函数执行状态或引发后续逻辑异常。

#主要错误日志
未知

#问题根因
在 `drivers/gpu/drm/amd/amdkfd/kfd_svm.c` 文件的 `svm_ioctl` 函数中，`switch` 语句的 `default` 分支错误地将返回值 `r` 赋值为正数 `EINVAL`。Linux 内核约定系统调用和 IOCTL 处理函数在发生错误时应返回负的错误码（例如 `-EINVAL`），以便上层调用者通过判断返回值是否小于 0 来识别错误。

#解决方案
修改 `drivers/gpu/drm/amd/amdkfd/kfd_svm.c` 文件中 `svm_ioctl` 函数 `default` 分支的赋值语句，将 `r = EINVAL` 修改为 `r = -EINVAL`，确保返回符合内核规范的负错误码。
