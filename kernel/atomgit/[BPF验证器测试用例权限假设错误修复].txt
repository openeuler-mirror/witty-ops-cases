#标题
[BPF验证器测试用例权限假设错误修复]

#内核版本
OLK-6.6

#问题现象
BPF自测试用例 `ARG_PTR_TO_LONG uninitialized` 和 `ARG_PTR_TO_LONG half-uninitialized` 在非特权模式下运行时，与内核验证器的实际行为不一致。原本测试代码预期在非特权模式下会读取未初始化栈内存并报错，但实际验证器探测的是读权限而非写权限，导致测试预期判定逻辑错误。

#主要错误日志
invalid indirect read from stack R4 off -16+0 size 8
invalid indirect read from stack R4 off -16+4 size 8

#问题根因
测试用例中的注释假设“在特权模式下允许读取未初始化的栈位置”是不正确的。实际上，验证器探测的是读访问权限而不是写访问权限，导致测试用例对于特权和非特权模式下的行为预期与验证器逻辑不符。

#解决方案
移除测试用例 `ARG_PTR_TO_LONG uninitialized` 和 `ARG_PTR_TO_LONG half-uninitialized` 中的 `__failure_unpriv` 标记及相关错误消息断言（`__msg_unpriv`），将它们统一标记为 `__success`，以修正测试用例对验证器行为的预期。
