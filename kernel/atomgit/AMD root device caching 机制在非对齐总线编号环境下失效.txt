#标题
AMD root device caching 机制在非对齐总线编号环境下失效

#内核版本
OLK-6.6

#问题现象
在非 AMD 参考设计的主板上（PCI 总线编号未严格对齐），内核无法正确缓存 AMD 根设备。
由于依赖硬件寄存器值进行设备查找，当总线编号非对齐时（例如 0x00 | 0x07 | 0xE0 | 0x15 = 0x1F），寄存器读取值（Bitwise-OR 结果）不匹配任何实际总线号，导致无法找到对应的 PCI 根设备，进而影响后续依赖于 AMD Node 缓存的内核功能（如 SMN 访问等）。

#主要错误日志
未知

#问题根因
近期 AMD node 重构移除了“搜索并计数”方法，改为依赖 Data Fabric 寄存器中读取的值来缓存 AMD 根设备。
开发人员错误地预期该寄存器保存了连接到该 Fabric 的某个根设备的 PCI 总线号。实际上，当从 PCI 配置空间读取时，该寄存器返回的是所有连接根设备总线号的按位或值。
在 AMD 参考设计板上，总线号是对齐的（如 0x00, 0x40, 0xA0, 0xE0），按位或结果恰好匹配其中一个总线号，问题被掩盖。但在总线号非对齐的板子上，按位或结果（如 0x1F）不匹配任何实际总线号，导致设备查找失败。

#解决方案
恢复使用“搜索并计数”的方法来缓存 AMD 根设备，不再依赖硬件寄存器的值。
具体改动：
1. 移除 `amd_node_get_root` 函数及其依赖的寄存器读取逻辑。
2. 新增 `get_next_root` 辅助函数，通过 PCI 类代码（PCI_CLASS_BRIDGE_HOST）而非固定 PCI ID 来搜索根设备，避免了对特定 PCI ID 的依赖。
3. 重构 `amd_smn_init` 函数，先遍历查找所有 AMD 根设备，再根据 CPU 节点数计算每个节点对应的根设备，正确处理各种总线编号分配情况。
