#标题
blk-cgroup: 修复 class @block_class 子系统引用计数泄漏问题

#内核版本
OLK-6.6

#问题现象
内核块层控制组的子系统引用计数发生泄漏。这通常表现为在执行涉及块设备统计的操作后，相关子系统的引用计数无法正确归零，可能导致内核模块无法正常卸载，或者在长期运行的系统中出现资源耗尽的情况。

#主要错误日志
未知

#问题根因
函数 `blkcg_fill_root_iostats()` 通过调用 `class_dev_iter_init()` 和 `class_dev_iter_next()` 遍历 `@block_class` 的设备列表，但在遍历结束后未调用 `class_dev_iter_exit()` 进行清理，导致 `@block_class` 所属子系统的引用计数泄漏。

#解决方案
在函数 `blkcg_fill_root_iostats()` 的遍历循环结束后，显式调用 `class_dev_iter_exit(&iter)` 来正确结束迭代操作，释放持有的子系统引用计数。
