#标题
usb: core: usb_phy_roothub_set_mode 错误处理逻辑错误导致 PHY 状态不平衡

#内核版本
OLK-6.6

#问题现象
在 USB 主控初始化过程中（`usb_add_hcd` 调用阶段），若 `phy_set_mode()` 设置 PHY 模式失败，内核会执行错误的错误处理逻辑，尝试关闭尚未开启的 PHY 电源。这会导致 PHY 电源使用计数（usage count）不平衡，进而引发 USB 子系统后续电源管理流程异常或设备无法正常工作。

#主要错误日志
未知

#问题根因
`usb_phy_roothub_set_mode()` 函数中的错误处理代码存在逻辑缺陷：当 `phy_set_mode()` 调用失败时，错误处理路径执行了 `phy_power_off()` 操作。这违反了 PHY 操作的逻辑顺序，因为 `phy_power_off()` 必须与 `phy_power_on()` 配对使用以平衡引用计数，而本函数仅用于设置模式，并不包含开启电源的操作。此外，该函数被调用时 PHY 尚未上电，此时执行下电操作毫无意义。该错误代码推测是从 `phy_power_on()` 函数复制粘贴而来，但未正确调整错误处理逻辑。

#解决方案
移除 `usb_phy_roothub_set_mode()` 函数中的错误处理路径代码。修改后，一旦 `phy_set_mode()` 调用失败，函数直接返回错误码，不再执行多余的 `phy_power_off()` 操作，从而避免破坏 PHY 的电源状态平衡。
