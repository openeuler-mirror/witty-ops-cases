#标题
LS7A2000 PCIe 设备电源状态转换导致 CPU 无法访问 PCI header

#内核版本
OLK-6.6

#问题现象
在 LS7A2000 芯片组环境下，当 PCIe 设备处于低功耗模式（如 D3 状态）时，CPU 可能无法访问该设备的 PCI header，导致设备功能异常或系统行为不可控。

#主要错误日志
未知

#问题根因
根据 Commit 描述，CPU 在设备进入低功耗模式时无法访问其 PCI header。结合代码分析，这是由于 LS7A2000 的特定 PCIe Port（Device ID 0x7a39, 0x7a49, 0x7a59, 0x7a69）在电源管理转换过程中存在硬件缺陷，不支持设备正常进入 D3 状态或进行链路速度切换。

#解决方案
在内核 PCI quirks 机制中增加针对 LS7A2000 PCIe Port 的特定修复函数 `loongson_d3_and_link_quirk`。该函数遍历设备总线层级，若发现设备位于受影响的 PCIe Port 下方，则通过设置设备标志位 `PCI_DEV_FLAGS_NO_D3` 和 `PCI_DEV_FLAGS_NO_LINK_SPEED_CHANGE`，并置位 `no_d1d2`，强制禁止该设备进入 D3 状态及进行链路速度变更，从而规避硬件缺陷带来的访问问题。
