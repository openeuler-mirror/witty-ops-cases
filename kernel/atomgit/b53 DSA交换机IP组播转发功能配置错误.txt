#标题
b53 DSA交换机IP组播转发功能配置错误

#内核版本
OLK-6.6

#问题现象
在 b53 系列交换机驱动中，IP 组播转发功能未能正确开启。具体表现为：
1. 对于较旧的交换机芯片（非 BCM5325），IP 组播流量可能无法正常转发，因为控制位默认为关闭状态。
2. 对于较新的交换机芯片，驱动错误地操作了保留位或功能不同的位（Bit 1），可能引发非预期的行为，如丢弃长度在 1500 到 1536 之间且 CRC 正确的帧。

#主要错误日志
未知

#问题根因
驱动代码中使用了错误的宏定义 `B53_IPMC_FWD_EN`（对应 Bit 1）来试图开启 IP 组播功能。根据硬件规范，Bit 1 在新控制寄存器中是保留位或定义为“越界错误丢弃”功能，而非 IP 组播使能。真正的 IP 组播使能位是 Bit 0。原代码仅在 BCM5325 芯片上正确操作了 Bit 0，而在其他芯片上错误地操作了 Bit 1。

#解决方案
修正寄存器位定义与操作逻辑：
1. 删除错误的宏定义 `B53_IPMC_FWD_EN`（Bit 1）和 `B53_IP_MCAST_25`（Bit 0）。
2. 新增统一的宏定义 `B53_IP_MC` 指向正确的 Bit 0。
3. 修改 `b53_set_forwarding` 函数逻辑，统一使用 `B53_IP_MC` 宏来启用 IP 组播功能，确保所有兼容芯片正确配置该位。
