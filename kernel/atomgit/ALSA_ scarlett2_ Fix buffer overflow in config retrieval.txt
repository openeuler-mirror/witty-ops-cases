#标题
ALSA: scarlett2: Fix buffer overflow in config retrieval

#内核版本
OLK-6.6

#问题现象
scarlett2_usb_get_config() 函数在处理配置数据时存在逻辑错误。当 `count > 1` 时，代码在进行字节序转换过程中会发生缓冲区越界访问（buffer overflow），可能导致内核内存损坏或程序崩溃。

#主要错误日志
未知

#问题根因
函数在进行字节序转换时判断条件错误。原代码检查的是总缓冲区大小 `if (size == 2)`，但在循环处理时，却按照每个元素为 u16（2 字节）的大小进行遍历。这导致当 `count` 大于 1 时，循环实际访问了 `count * 2` 字节的内存，而缓冲区仅分配了 `size` 字节，从而引发缓冲区越界。

#解决方案
修改判断逻辑，将检查总缓冲区大小 `size` 改为检查配置项元素大小 `config_item->size`。确保字节序转换逻辑与实际元素类型（16位或32位）相匹配，避免越界访问。具体修复为：将 `if (size == 2)` 修改为 `if (config_item->size == 16)`，并补充了 `else if (config_item->size == 32)` 的处理分支。
