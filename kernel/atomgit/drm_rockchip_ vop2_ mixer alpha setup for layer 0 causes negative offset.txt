#标题
drm/rockchip: vop2: mixer alpha setup for layer 0 causes negative offset

#内核版本
OLK-6.6

#问题现象
Rockchip VOP2 驱动在处理图层混合时，Alpha 设置逻辑错误地从第一层（layer 0）开始计算偏移量。当 zpos 为 0 时，根据公式 `offset = (mixer_id + zpos - 1) * 0x10` 计算将得到负数偏移量。这导致硬件寄存器配置错误，可能引发显示异常、图层混合功能失效或潜在的硬件访问越界问题。

#主要错误日志
未知

#问题根因
Alpha 设置的计算逻辑存在缺陷。代码在遍历图层时未跳过第一层（zpos == 0），导致计算偏移量的公式 `offset = (mixer_id + zpos - 1) * 0x10` 计算出负值，破坏了正确的寄存器地址偏移。根据硬件设计，Alpha 配置应从第二个图层开始。

#解决方案
在 `vop2_setup_alpha` 函数的图层遍历循环中增加判断：若 `zpos == 0`，则直接跳过本次迭代（continue），确保 Alpha 设置仅从第二个图层开始处理，从而避免产生负数偏移量。
