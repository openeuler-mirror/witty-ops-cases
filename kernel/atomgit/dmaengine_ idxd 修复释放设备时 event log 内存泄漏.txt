#标题
dmaengine: idxd 修复释放设备时 event log 内存泄漏

#内核版本
OLK-6.6

#问题现象
在卸载或禁用 idxd 设备驱动时，Event Log (EVL) 相关的内存未能正确释放，导致内存泄漏资源泄露问题。具体表现为设备释放过程中，若硬件配置寄存器中 `evl_en` 位已被硬件或外部清除，释放流程会提前退出，导致内存未释放。

#主要错误日志
未知

#问题根因
在 `idxd_device_evl_free` 函数中，代码逻辑错误地检查了硬件寄存器 `IDXD_GENCFG_OFFSET` 中的 `evl_en` 状态位。如果该位为 0（表示硬件侧未启用或已禁用），函数直接 return 退出，跳过了后续 `mutex_lock` 及内存释放逻辑。这导致虽然硬件状态已变更，但内核侧分配的 Event Log 内存结构体未被释放，从而造成内存泄漏。

#解决方案
移除 `idxd_device_evl_free` 函数中对硬件寄存器 `gencfg.evl_en` 状态的读取与提前返回判断逻辑。确保无论硬件当前状态如何，只要 `evl` 结构体存在，都继续执行后续的互斥锁获取、硬件寄存器禁用写入及内存释放流程，保证内存资源被正确回收。
