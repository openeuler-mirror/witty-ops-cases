#标题
IIO DAC ad5770r 驱动 read_raw 接口错误返回值修复

#内核版本
OLK-6.6

#问题现象
在调用 ad5770r 驱动的 read_raw 接口读取数据时，若底层 I/O 操作（regmap_bulk_read）失败，该接口错误地返回 0（代表成功或有效数据），导致上层应用或框架无法感知硬件通信故障，可能导致读取到未初始化或无效的数据，且错误状态未被正确传播。

#主要错误日志
未知

#问题根因
在函数 ad5770r_read_raw() 中，调用 regmap_bulk_read() 进行硬件寄存器读取后，代码检查了返回值 ret，但在 ret 非零（表示失败）的情况下，代码逻辑错误地执行了 return 0，而非 return ret。这导致 I/O 失败信息被丢弃，未正确传递给调用者。

#解决方案
修改 drivers/iio/dac/ad5770r.c 文件中的 ad5770r_read_raw 函数，在检测到 regmap_bulk_read 返回错误（ret 不为 0）时，直接返回 ret 错误码，确保故障能够正确向上层传递。
