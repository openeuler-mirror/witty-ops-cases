#标题
sched_ext: selftests/dsp_local_on 测试用例偶发性失败

#内核版本
OLK-6.6

#问题现象
sched_ext 子系统的 selftests/dsp_local_on 测试用例在运行过程中出现偶发性失败。失败发生在任务迁移被禁用的场景下，测试程序错误地尝试将任务分发到其他 CPU，导致测试行为不符合预期。

#主要错误日志
未知

#问题根因
测试代码 `dsp_local_on_dispatch()` 存在错误的假设：认为 `p->nr_cpus_allowed` 总是能准确反映 `p->cpus_ptr` 的状态。
实际上，当任务处于 migration disabled（迁移禁用）状态并被调度切换时，内核会临时将 `p->cpus_ptr` 覆盖为之前所在的 CPU，而 `p->nr_cpus_allowed` 的值保持不变。这导致条件判断 `p->nr_cpus_allowed == nr_cpus` 误判任务允许在多个 CPU 上运行，进而尝试将任务非法分发到其他 CPU，引发测试失败。

#解决方案
修改 `tools/testing/selftests/sched_ext/dsp_local_on.bpf.c` 中的调度逻辑，在判断是否随机选择目标 CPU 时，增加对 `p->migration_disabled` 状态的检查。
修改后的逻辑确保：仅当任务允许在多个 CPU 运行且未被禁用迁移时，才进行随机分发；否则，维持任务在原 CPU 运行。
