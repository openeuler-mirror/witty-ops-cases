#标题
of/irq: Fix using uninitialized variable @addr_len in API of_irq_parse_one()

#内核版本
OLK-6.6

#问题现象
在内核 API `of_irq_parse_one()` 执行过程中，当 `of_get_property(device, "reg", &addr_len)` 调用失败时（例如设备节点缺少 "reg" 属性），该函数会使用未初始化的变量 `@addr_len` 进行后续的内存大小判断和 `memcpy` 拷贝操作。这可能导致内核读取非法内存地址、内存越界读取或数据损坏，进而引发未定义的行为或内核崩溃。

#主要错误日志
未知

#问题根因
变量 `@addr_len` 在定义时未进行初始化。函数调用 `of_get_property(device, "reg", &addr_len)` 在查找属性失败时不会修改 `@addr_len` 的值。随后的代码逻辑未检查 `of_get_property` 的返回值就直接使用了可能未被赋值的 `@addr_len`，导致使用了随机值。

#解决方案
在调用 `of_get_property` 之前，显式将变量 `@addr_len` 初始化为 0。这确保了即使 `of_get_property` 失败，`@addr_len` 也是一个确定的值，从而避免了使用未初始化变量的问题。
