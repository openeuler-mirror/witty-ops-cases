#标题
auxdisplay: hd44780 驱动内存释放 API 误用修复

#内核版本
OLK-6.6

#问题现象
hd44780 驱动在模块卸载或探测失败（probe fail）的错误处理路径中，对 `charlcd_alloc()` 分配的变量使用了错误的释放函数，导致内存管理 API 不匹配。

#主要错误日志
未知

#问题根因
在 `hd44780_probe` 的错误处理路径和 `hd44780_remove` 函数中，代码直接使用 `kfree()` 来释放由 `charlcd_alloc()` 函数分配的 `lcd` 变量。根据 API 设计规范，通过 `charlcd_alloc()` 分配的内存应当由 `charlcd_free()` 来释放，直接使用 `kfree()` 属于 API 误用。

#解决方案
修正内存释放方式，将 `hd44780_probe` 函数错误处理路径（fail2 标签处）和 `hd44780_remove` 函数中用于释放 `lcd` 变量的 `kfree(lcd)` 调用统一替换为 `charlcd_free(lcd)`，以确保内存分配与释放接口的匹配。
