#标题
selftest: hid测试框架缺失tests目录导致执行报错

#内核版本
OLK-6.6

#问题现象
在执行 HID 自测试用例时，因缺少必要的 tests 目录导致测试脚本运行失败。具体表现为：执行 `make install` 安装测试后，运行 `./run_kselftest.sh -t hid:hid-core.sh` 时，pytest 解析命令行参数报错，提示无法识别参数，测试无法正常进行。

#主要错误日志
```
/usr/lib/python3.11/site-packages/_pytest/config/__init__.py:331: PluggyTeardownRaisedWarning: A plugin raised an exception during an old-style hookwrapper teardown.
Plugin: helpconfig, Hook: pytest_cmdline_parse
UsageError: usage: __main__.py [options] [file_or_dir] [file_or_dir] [...]
__main__.py: error: unrecognized arguments: --udevd
  inifile: None
  rootdir: /root/linux/kselftest_install/hid
```

#问题根因
在 commit 160c826b4dd0 ("selftest: hid: add missing run-hid-tools-tests.sh") 中，虽然添加了 `run-hid-tools-tests.sh` 脚本以支持安装，但遗漏了 `tests` 目录。该脚本依赖 `tests` 目录下的脚本来实际执行测试，由于 Makefile 中未将 `tests` 目录加入到安装列表中，导致测试环境不完整，运行时出错。

#解决方案
在 `tools/testing/selftests/hid/Makefile` 中，将 `tests` 目录添加到 `TEST_FILES` 变量中，确保在执行测试安装时，`tests` 目录能够被正确拷贝至安装路径，从而修复测试脚本缺失依赖目录的问题。
