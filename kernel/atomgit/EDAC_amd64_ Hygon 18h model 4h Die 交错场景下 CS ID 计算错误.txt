#标题
EDAC/amd64: Hygon 18h model 4h Die 交错场景下 CS ID 计算错误

#内核版本
OLK-6.6

#问题现象
在 Hygon family 18h model 4h 处理器平台上，当内存配置为 Die Interleave（Die 交错）场景时，EDAC 子系统在进行地址转换过程中计算 Chip Select ID (cs_id) 错误。该问题会导致内存错误定位不准确，可能引发 EDAC 报告错误的内存位置信息，或在相关的内存故障处理流程中产生异常。

#主要错误日志
未知

#问题根因
在 `hygon_umc_normaddr_to_sysaddr` 函数中，针对 Hygon family 18h model 4h 的 Die interleave 场景，计算 `cs_id` 的公式存在逻辑错误。原代码直接使用右移后的值进行赋值，未考虑特定的偏移量修正，导致计算出的 `cs_id` 不符合硬件规范。

#解决方案
修正 `drivers/edac/amd64_edac.c` 文件中 `cs_id` 的计算逻辑。在 Die interleave 场景下，对提取出的 Die ID 值减去偏移量 4 后再进行移位操作。
具体代码变更为：
将 `cs_id |= ((cs_fabric_id & die_id_mask) >> die_id_shift) << die_id_bit;`
修改为 `cs_id |= (((cs_fabric_id & die_id_mask) >> die_id_shift) - 4) << die_id_bit;`
