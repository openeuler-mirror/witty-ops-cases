#标题
bpftool cgroup tree 命令在旧版本内核上执行报 Invalid argument 错误

#内核版本
OLK-6.6

#问题现象
在运行内核版本低于 bpftool 编译版本的系统中，执行 `bpftool cgroup tree` 命令时，工具无法正常列出 BPF 程序信息，直接报错退出。具体表现为终端输出 "Error: can't query bpf programs attached to /sys/fs/cgroup: Invalid argument"。

#主要错误日志
```text
$ bpftool cgroup tree
CgroupPath
ID       AttachType      AttachFlags     Name
Error: can't query bpf programs attached to /sys/fs/cgroup: Invalid argument
```

#问题根因
在引入 commit 98b303c9bf05 ("bpftool: Query only cgroup-related attach types") 后，bpftool 在 `cgroup_has_attached_progs` 函数中遍历查询 `cgroup_attach_types` 数组中的所有附加类型。当 bpftool 的编译版本较新，而运行环境的内核版本较旧时，bpftool 尝试查询的某些 cgroup attach type 在旧内核中尚未实现。这导致内核返回 `-EINVAL` 错误，而 bpftool 此时对所有负值错误直接返回失败，破坏了向前兼容性。

#解决方案
修改 `tools/bpf/bpftool/cgroup.c` 中的 `cgroup_has_attached_progs` 函数逻辑，恢复对 `-EINVAL` 错误的忽略处理。在查询 attached progs 时，如果返回错误且 errno 为 `EINVAL`，不直接返回失败，而是继续处理其他 attach types，从而兼容内核不支持特定 attach type 的情况。
