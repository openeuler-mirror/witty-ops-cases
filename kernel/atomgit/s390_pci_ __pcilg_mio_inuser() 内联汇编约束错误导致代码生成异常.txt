#标题
s390/pci: __pcilg_mio_inuser() 内联汇编约束错误导致代码生成异常

#内核版本
OLK-6.6

#问题现象
在 s390 架构下使用 MIO 指令进行 PCI MMIO 读操作时，若编译器选择了寄存器 0（register zero）作为移位操作数，会导致生成的机器码行为异常。具体表现为 `srlg` 指令忽略移位寄存器内容，始终执行 0 bit 移位，导致数据读取逻辑错误。

#主要错误日志
未知

#问题根因
函数 `__pcilg_mio_inuser()` 的内联汇编中，移位操作数 `shift` 使用了 `"d"` 约束。该约束允许编译器使用任意通用寄存器，包括寄存器 0。在 s390 架构的 `srlg`（Shift Right Logical Long）指令中，若指定寄存器 0 作为移位计数寄存器，硬件会忽略该寄存器内容并固定执行 0 bit 移位，从而破坏了原本的计算逻辑。

#解决方案
将 `__pcilg_mio_inuser()` 内联汇编中 `shift` 操作数的约束从 `"d"` 修改为 `"a"`。`"a"` 约束排除了寄存器 0 的使用，确保编译器选择其他通用寄存器来存放移位值，从而保证 `srlg` 指令按预期执行移位操作。
