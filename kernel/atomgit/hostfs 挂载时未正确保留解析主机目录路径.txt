#标题
hostfs 挂载时未正确保留解析主机目录路径

#内核版本
OLK-6.6

#问题现象
在使用 hostfs 挂载时，当主机目录（host directory）参数为空（默认情况）时，系统错误地使用 `fc->source` 作为主机根目录，导致挂载点路径解析错误，无法正确访问预期的主机目录。

#主要错误日志
未知

#问题根因
在将 hostfs 转换为使用新的 mount API 后，`hostfs_fill_super` 函数中的逻辑存在缺陷。当主机目录参数未指定时，代码直接回退使用 `fc->source` 作为主机根目录，这违反了预期的挂载行为。正确逻辑应当区分新旧挂载路径：对于旧版挂载路径应使用 `parse_monolithic` 解析根目录，对于新版挂载路径应使用 `parse_param` 解析主机目录参数。

#解决方案
重构挂载参数解析逻辑，移除 `hostfs_fill_super` 中错误的默认赋值代码。新增 `hostfs_parse_param` 函数处理新版 mount API 的参数解析，新增 `hostfs_parse_monolithic` 函数处理旧版 monolithic 挂载模式的数据解析，正确处理空参数情况，确保 `fsi->host_root_path` 在两种挂载模式下均能被正确构建。
