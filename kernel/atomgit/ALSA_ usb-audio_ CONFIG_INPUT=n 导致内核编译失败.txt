#标题
ALSA: usb-audio: CONFIG_INPUT=n 导致内核编译失败

#内核版本
OLK-6.6

#问题现象
在禁用 CONFIG_INPUT 选项（CONFIG_INPUT=n）的内核配置下编译内核时，因 Sony DualSense PS5 控制器的 mixer quirk 补丁引用了 input 设备相关接口，导致构建过程失败。

#主要错误日志
未知

#问题根因
近期新增的 Sony DualSense mixer quirk 功能依赖 input 设备句柄，但在代码实现中未对 CONFIG_INPUT 配置项进行有效性检查。当 CONFIG_INPUT 未设置时，相关依赖项不可达，从而引发编译错误。

#解决方案
使用 `IS_REACHABLE(CONFIG_INPUT)` 宏对 DualSense 相关代码段进行条件编译包裹。具体修改为在 `sound/usb/mixer_quirks.c` 文件中，将 `snd_dualsense_controls_create` 函数定义及其在 `snd_usb_mixer_apply_create_quirk` 中的调用 case 分支用 `#if IS_REACHABLE(CONFIG_INPUT)` 和 `#endif` 包裹起来，确保在 CONFIG_INPUT 未启用时跳过相关代码编译。
