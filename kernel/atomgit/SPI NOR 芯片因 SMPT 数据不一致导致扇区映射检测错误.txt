#标题
SPI NOR 芯片因 SMPT 数据不一致导致扇区映射检测错误

#内核版本
OLK-6.6

#问题现象
特定 SPI NOR 芯片（如 Infineon S25FS512S）在解析 SFDP 参数时，由于芯片内部 Sector Map Parameter Table (SMPT) 数据存在不一致性，导致驱动识别出错误的 map ID，进而无法正确检测到扇区映射配置，造成设备功能异常。

#主要错误日志
未知

#问题根因
特定芯片（如 Infineon S25FS512S）的 SMPT 中 map ID 定义与实际获取的配置寄存器值之间存在不一致。驱动在解析 SMPT 表并根据配置寄存器确定 map ID 的过程中，缺乏对这种不一致情况进行修正的机制，导致最终确定的 map ID 错误。

#解决方案
在 `struct spi_nor_fixups` 中引入 `smpt_map_id` 回调钩子函数。该函数在 SMPT 解析流程中初步确定 map ID 之后被调用，允许厂商或芯片特定的驱动逻辑对错误的 map ID 进行修正。同时，在 `sfdp.c` 的 `spi_nor_get_map_in_use` 函数中增加对此 fixup 钩子的调用点。
