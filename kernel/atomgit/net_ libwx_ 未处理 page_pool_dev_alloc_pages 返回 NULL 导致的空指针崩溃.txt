#标题
net: libwx: 未处理 page_pool_dev_alloc_pages 返回 NULL 导致的空指针崩溃

#内核版本
OLK-6.6

#问题现象
当 `page_pool_dev_alloc_pages` 内存分配失败返回 NULL 时，内核虽然会触发 WARN_ON 告警，但代码逻辑未做返回处理，继续使用该 NULL 指针，导致系统崩溃。

#主要错误日志
未知

#问题根因
在 `wx_alloc_mapped_page` 函数中调用 `page_pool_dev_alloc_pages` 分配内存页时，未对返回值进行 NULL 检查保护。原有代码仅包含 `WARN_ON(!page)` 断言，但在断言触发后程序仍继续向下执行，尝试对 NULL 指针进行解引用操作（如获取 dma 地址），从而导致空指针引用崩溃。

#解决方案
修改 `drivers/net/ethernet/wangxun/libwx/wx_lib.c` 文件中的逻辑，将 `WARN_ON(!page)` 替换为 `if (unlikely(!page)) return false;`。在内存分配失败时立即返回错误，阻止后续代码使用无效的 NULL 指针，避免系统崩溃。
