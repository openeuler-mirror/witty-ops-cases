#标题
i2c: omap: 修复使用 deprecated API 读取非布尔属性引发的内核警告

#内核版本
OLK-6.6

#问题现象
在 OMAP I2C 驱动加载过程中，内核运行时产生警告信息，提示驱动正在使用已废弃的 `of_property_read_bool()` 函数读取非布尔类型的设备树属性。

#主要错误日志
WARN: of_property_read_bool() used on non-boolean property "mux-states"

#问题根因
驱动代码在检查设备树节点中是否存在 "mux-states" 属性时，错误地使用了 `of_property_read_bool()` 函数。该属性并非布尔类型，而是一个用于描述 mux 状态的属性。自内核提交 c141ecc3cecd ("of: Warn when of_property_read_bool() is used on non-boolean properties") 起，内核会对这种不合规的 API 使用行为发出警告。

#解决方案
将 `of_property_read_bool(node, "mux-states")` 替换为 `of_property_present(node, "mux-states")`。新的 API 专门用于检测属性是否存在，适用于非布尔类型的属性存在性检查，从而消除运行时警告。
