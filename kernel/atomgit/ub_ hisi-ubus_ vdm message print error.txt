#标题
ub: hisi-ubus: vdm message print error

#内核版本
OLK-6.6

#问题现象
内核日志中打印 VDM 消息错误信息时，提示文本存在单词拼写错误，将 "opcode" 错误地打印为 "opvode"，影响日志的可读性和准确性。

#主要错误日志
vdm sub opvode is invalid, sub_opcode = %#x

#问题根因
在提交 86fec00cb73a ("ub:hisi-ubus: Support UBUS vdm entity enable message") 中引入的代码存在笔误，`dev_err` 打印语句中的字符串 "opcode" 被错误地拼写为 "opvode"。

#解决方案
修正 drivers/ub/ubus/vendor/hisilicon/vdm.c 文件中 `ub_vdm_msg_info_handle` 函数内的 `dev_err` 日志打印字符串，将 "opvode" 更正为 "opcode"。
