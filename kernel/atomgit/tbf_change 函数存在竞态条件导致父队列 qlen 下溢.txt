#标题
tbf_change 函数存在竞态条件导致父队列 qlen 下溢

#内核版本
OLK-6.6

#问题现象
在 TBF（Token Bucket Filter）流量控制配置变更过程中，若 SFQ（Stochastic Fairness Queueing）扰动定时器在特定时机触发，会触发竞态条件。该竞态可被利用导致父队列的 qlen（队列长度）计数发生下溢（underflow），造成内核内存管理或流量调度逻辑异常。

#主要错误日志
未知

#问题根因
在 `tbf_change` 函数中，代码执行顺序存在逻辑漏洞：
1. CPU 0 持有锁并调用 `qdisc_tree_flush_backlog()` 刷新积压数据。
2. CPU 0 释放锁。
3. CPU 0 在锁外调用 `qdisc_put()` 释放资源。
4. 在 CPU 0 释放锁后、调用 `qdisc_put()` 前，CPU 1 获取锁并执行 rehash（重哈希）及 `qdisc_tree_reduce_backlog()`。
该时序导致竞态窗口，使得父队列的 qlen 计数被错误减少，从而引发下溢。

#解决方案
将 `tbf_change` 函数中的 `qdisc_tree_flush_backlog()` 调用替换为 `qdisc_purge_queue()`。
此修复确保在释放锁之前，qdisc 中的所有数据包已被清除，从而消除了竞态窗口，防止 qlen 下溢。
