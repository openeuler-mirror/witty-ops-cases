#标题
x86/microcode/AMD: AMD Zen5/Strix Halo 处理器 Entrysign 修订检查错误

#内核版本
OLK-6.6

#问题现象
AMD Zen5 架构下 Family 1Ah、Model 70h-7Fh（Strix Halo）系列的处理器在进行微码加载或修订版本检查时，会被错误地判定为不支持 Entrysign 特性，导致微码更新流程中相关的签名检查逻辑未能正确执行或被跳过，影响微码加载的正确性。

#主要错误日志
未知

#问题根因
在 `cpu_has_entrysign()` 函数中，对 AMD Family 1Ah 处理器的 Model 范围检查存在遗漏。原代码逻辑中，Model 范围 `0x60 <= model && model <= 0x6f` 仅覆盖了部分 Zen5 型号，而遗漏了 Model 70h-7Fh（Strix Halo）的处理器型号，导致这部分合法的处理器无法通过 Entrysign 特性检查。

#解决方案
修正 `arch/x86/kernel/cpu/microcode/amd.c` 文件中 `cpu_has_entrysign()` 函数的 Model 范围判断逻辑，将上限从 `0x6f` 修改为 `0x7f`，以包含 Model 70h-7Fh 的处理器，确保 Zen5/Strix Halo 系列处理器能正确通过 Entrysign 特性检查。
