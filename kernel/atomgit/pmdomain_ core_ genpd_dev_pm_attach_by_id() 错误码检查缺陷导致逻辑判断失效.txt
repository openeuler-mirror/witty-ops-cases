#标题
pmdomain: core: genpd_dev_pm_attach_by_id() 错误码检查缺陷导致逻辑判断失效

#内核版本
OLK-6.6

#问题现象
在 `genpd_dev_pm_attach_by_id()` 函数中，当 `of_count_phandle_with_args()` 返回负数错误码时，错误检查逻辑未能正确识别。由于隐式类型提升，负数错误码被误判为合法的极大数值，导致本应返回 NULL 的错误处理路径被跳过，函数继续向下执行，可能引发不可预期的行为或系统异常。

#主要错误日志
未知

#问题根因
变量 `num_domains` 用于存储 `of_count_phandle_with_args()` 的返回值，该函数可能返回负数错误码。原代码判断条件 `if (index >= num_domains)` 中，`index` 变量类型为 `u32`（无符号整型）。当 `num_domains` 为负数时，在比较操作中会被类型提升为极大的无符号正数，导致 `index >= num_domains` 条件恒为假，从而跳过了错误检查，使得程序错误地认为索引在有效范围内。

#解决方案
修改错误检查逻辑，在判断索引范围前优先检查返回值是否为负数。将判断条件从 `if (index >= num_domains)` 修改为 `if (num_domains < 0 || index >= num_domains)`，确保先捕获负数错误码，再校验索引合法性。
