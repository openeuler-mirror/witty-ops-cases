#标题
qla2xxx: NPIV 场景下 NVMe 连接命令失败

#内核版本
OLK-6.6

#问题现象
在 NPIV（N_Port ID Virtualization）主机环境下，NVMe 控制器无法发送 connect 命令。系统日志显示连接命令失败（errno: -18），创建关联失败，并不断尝试重连。由于无法定位 NVMe 队列 0 的硬件上下文缓冲区，NVMe over FC 功能无法正常工作。

#主要错误日志
kernel: nvme nvme5: Connect command failed, errno: -18
 kernel: nvme nvme5: qid 0: secure concatenation is not supported
 kernel: nvme nvme5: NVME-FC{5}: create_assoc failed, assoc_id 2e9100 ret 401
 kernel: nvme nvme5: NVME-FC{5}: reset: Reconnect attempt failed (401)
 kernel: nvme nvme5: NVME-FC{5}: Reconnect attempt in 2 seconds

#问题根因
NPIV 主机未初始化 `vha->irq_offset` 字段。该字段用于传递给 blk-mq（blk_mq_pci_map_queues）以定位 IO Queues 的起始位置，进而帮助定位 NVMe 队列 0。由于该字段未初始化，导致 NVMe 无法找到队列 0 的硬件上下文缓冲区（blk_mq_hw_ctx），从而引发连接失败。

#解决方案
在 `qla24xx_create_vhost` 函数中创建 NPIV vhost 时，显式初始化 `vha->irq_offset` 字段为 `QLA_BASE_VECTORS`，确保 blk-mq 能正确映射队列，修复 NVMe 连接问题。
