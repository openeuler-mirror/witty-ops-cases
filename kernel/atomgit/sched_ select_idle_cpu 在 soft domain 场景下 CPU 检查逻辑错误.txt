#标题
sched: select_idle_cpu 在 soft domain 场景下 CPU 检查逻辑错误

#内核版本
OLK-6.6

#问题现象
在启用 soft domain 的调度场景下，`select_idle_cpu` 函数在选择空闲 CPU 时逻辑判断有误，导致无法正确筛选符合条件的 CPU，可能导致进程调度效率下降或未按预期策略选核。

#主要错误日志
未知

#问题根因
在 `select_idle_cpu` 函数处理 soft domain 的循环中，判断 CPU 是否符合条件的逻辑存在笔误。代码原本应检查当前 CPU 是否在目标 CPU 集合 `cpus` 中，却错误地检查了该 CPU 是否在 `tmpmask`（即 `tg->sf_ctx->span`）中。由于 `tmpmask` 是循环的遍历对象，`cpumask_test_cpu(cpu, tmpmask)` 始终为真，导致该检查失效，未能正确过滤非目标 CPU 集合中的 CPU。

#解决方案
修正 `kernel/sched/fair.c` 中的判断逻辑，将 `cpumask_test_cpu(cpu, tmpmask)` 修改为 `cpumask_test_cpu(cpu, cpus)`，确保正确检查 CPU 是否属于预期的 `cpus` 集合。
