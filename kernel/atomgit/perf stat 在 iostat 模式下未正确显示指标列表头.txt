#标题
perf stat 在 iostat 模式下未正确显示指标列表头

#内核版本
OLK-6.6

#问题现象
执行 `perf stat --iostat` 命令时，输出信息中缺少具体的指标列表头（如 Inbound Read(MB) 等），导致数据列含义不明，仅显示端口号。

#主要错误日志
无内核日志输出。命令行输出异常示例如下：
```
   Performance counter stats for 'system wide':
      port
  0000:00                    0                    0                    0                    0
```

#问题根因
在聚合模式下，代码逻辑限制仅打印 metric leader 的指标头。由于 `perf iostat` 模式会进行全局聚合且不具备常规 metric events，导致其被视为非 metric leader 而跳过了指标头的打印逻辑。

#解决方案
在打印指标头的判断逻辑中增加对 `config->iostat_run` 的检查，若为 iostat 运行模式则跳过原有的 metric leader 限制，确保 iostat 的表头能够正常显示。
