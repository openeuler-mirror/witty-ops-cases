#标题
imx8mp 设备树中 SDHC 控制器 ipg 时钟定义错误

#内核版本
OLK-6.6

#问题现象
i.MX8MP 平台的 SDHC（USDHC）控制器设备树节点中，ipg 时钟被错误地定义为 DUMMY 时钟。这可能导致驱动无法正确获取和管理 ipg 时钟，进而影响 SDHC 控制器的正常功能或时钟框架的正确性。

#主要错误日志
未知

#问题根因
根据 i.MX 8M Plus Applications Processor Reference Manual 手册中的 Table 5-2. System Clocks，SDHC 控制器的 ipg 时钟应来源于 IPG_CLK_ROOT。但在设备树源文件 imx8mp.dtsi 中，usdhc1、usdhc2 和 usdhc3 节点的 ipg 时钟被错误地指向了 IMX8MP_CLK_DUMMY，而非正确的 IMX8MP_CLK_IPG_ROOT。

#解决方案
修改设备树文件 `arch/arm64/boot/dts/freescale/imx8mp.dtsi`，将 usdhc1、usdhc2 和 usdhc3 节点中的 ipg 时钟源从 `IMX8MP_CLK_DUMMY` 更正为 `IMX8MP_CLK_IPG_ROOT`，以符合硬件参考手册的定义。
