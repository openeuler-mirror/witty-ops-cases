#标题
Zhaoxin PMU uncore 驱动与 pcieport 驱动冲突导致功能失效

#内核版本
OLK-6.6

#问题现象
系统中 Zhaoxin 处理器的 PMU uncore 驱动在注册时，其 PCI 设备 ID 表与内核 inbox pcieport 驱动产生冲突。该冲突导致 pcieport 驱动绑定失败或功能异常，进而引发 PCIe 端口服务（如 AER、热插拔等）无法正常工作。

#主要错误日志
未知

#问题根因
在 `kh40000_uncore_pci_ids` 和 `kx7000_uncore_pci_ids` 这两个 PCI 设备 ID 列表中，错误地添加了用于 PCIE 功能的设备 ID（如 Vendor 0x1D17, Device 0x0717-073B 等）。这些设备 ID 本应由通用的 pcieport 驱动程序接管处理，但被 uncore 驱动声明所有权，导致两个驱动发生资源冲突，造成 pcieport 驱动失效。

#解决方案
从 Zhaoxin uncore 驱动的 PCI 设备 ID 表中移除所有 PCIE 相关的设备条目（包括 `PCI_DEVICE(0x1D17, 0x0717)` 等），确保这些 PCIE 设备由标准的 pcieport 驱动程序管理，从而解决驱动冲突问题。
