#标题
DAMON 位标机制因错误返回值处理不当导致行为异常

#内核版本
OLK-6.6

#问题现象
DAMON（Data Access Monitor）的水位线（Watermarks）机制在处理未知或无效的 metric 类型时，错误地将错误码 `-EINVAL` 作为无符号长整型数值返回。这导致该数值被误认为是合法的度量值，破坏了水位线的高低判断逻辑，可能导致 DAMON 方案无法正确激活或停止，引发内核功能异常。

#主要错误日志
未知

#问题根因
函数 `damos_wmark_metric_value` 的返回值类型定义为 `unsigned long`。当遇到未知的 metric 类型时，代码直接返回了整型错误码 `-EINVAL`。在 C 语言中，负数被转换为无符号长整型时，会变成一个非常大的正数（即 `(unsigned long)-EINVAL`）。调用者 `damos_wmark_wait_us` 将此大数值误判为合法的内存度量值，导致后续的水位线比较逻辑失效。

#解决方案
修改函数接口与返回值处理逻辑：
1. 将原函数 `damos_wmark_metric_value` 重构为 `damos_get_wmark_metric_value`。
2. 将返回值类型从 `unsigned long` 改为 `int`，用于返回错误码（0 表示成功，非 0 表示失败）。
3. 实际的度量值通过函数参数 `unsigned long *metric_value` 指针回传。
4. 在调用处 `damos_wmark_wait_us` 中检查返回值，若返回非 0（即获取失败），则直接返回 0 而不是继续使用错误的数值进行比较。
