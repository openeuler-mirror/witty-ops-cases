#标题
bpf: test_run: bpf_lwt_xmit_push_encap 空指针引用导致内核崩溃

#内核版本
OLK-6.6

#问题现象
在通过 `bpf_prog_test_run_skb` 接口测试 `BPF_PROG_TYPE_LWT_XMIT` 类型的 BPF 程序时，若调用 `bpf_lwt_push_ip_encap` 函数，会触发内核空指针引用，导致内核 Oops 甚至崩溃。

#主要错误日志
[  105.186365] BUG: kernel NULL pointer dereference, address: 0000000000000000
[  105.186382] #PF: supervisor read access in kernel mode
[  105.186388] #PF: error_code(0x0000) - not-present page
[  105.186404] Oops: 0000 [#1] PREEMPT SMP NOPTI
[  105.186443] RIP: 0010:bpf_lwt_push_ip_encap+0x1eb/0x520
[  105.186513]  bpf_lwt_xmit_push_encap+0x2b/0x40
[  105.186579]  bpf_prog_test_run_skb+0x2f5/0x4f0

#问题根因
`bpf_lwt_xmit_push_encap` 辅助函数需要访问 `skb_dst(skb)->dev` 来计算所需的头部空间：
`err = skb_cow_head(skb, len + LL_RESERVED_SPACE(skb_dst(skb)->dev));`
但是，当 `skb` 由 `bpf_prog_test_run_skb` 函数构建时，`skb->_skb_refdst` 可能未被初始化。在此场景下执行 `bpf_lwt_push_ip_encap` 函数会触发空指针引用。

#解决方案
在 `bpf_prog_test_run_skb` 函数中调用 `bpf_test_run` 之前，针对 `BPF_PROG_TYPE_LWT_XMIT` 类型的程序，显式设置 `skb` 的 dst 条目（引用 `net->ipv6.ip6_null_entry->dst`），确保 `skb_dst` 有效，从而修复空指针引用问题。
