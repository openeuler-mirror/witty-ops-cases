#标题
[Bluetooth: MGMT: 修复 MGMT_OP_READ_EXP_FEATURES_INFO 未正确暴露 debug UUID 的问题]

#内核版本
[OLK-6.6]

#问题现象
[当插入蓝牙适配器时，bluetoothd 进程调用 MGMT_OP_READ_EXP_FEATURES_INFO 命令无法获取到 debug UUID 信息，导致依赖该特性的功能异常或无法正常初始化。]

#主要错误日志
[未知]

#问题根因
[在 CONFIG_BT_FEATURE_DEBUG 配置开启的情况下，代码逻辑错误地依赖 hdev 是否为空来决定是否设置 debug UUID。由于 bluetoothd 仅在适配器插入时调用 MGMT_OP_READ_EXP_FEATURES_INFO（此时 hdev 已设置），导致 `if (!hdev)` 判断条件不满足，debug UUID 未被拷贝到返回数据中，从而未能正确暴露给用户态。]

#解决方案
[移除 `if (!hdev)` 的判断限制，无论 hdev 是否存在，只要调试功能开启（bt_dbg_get() 为真）就在返回信息中设置并暴露 debug UUID，确保适配器插入后能正确读取该特性。]
