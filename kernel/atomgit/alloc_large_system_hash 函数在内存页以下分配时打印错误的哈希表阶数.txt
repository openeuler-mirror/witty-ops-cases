#标题
alloc_large_system_hash 函数在内存页以下分配时打印错误的哈希表阶数

#内核版本
OLK-6.6

#问题现象
系统启动或运行过程中，当哈希表分配大小小于一个内存页时，内核日志打印出的哈希表分配阶数为负值，显示不符合逻辑。

#主要错误日志
TCP established hash table entries: 32 (order: -4, 256 bytes, linear)
TCP bind hash table entries: 32 (order: -2, 1024 bytes, linear)

#问题根因
在 `alloc_large_system_hash()` 函数中，打印日志信息时计算分配阶数的逻辑有误。代码无条件地从 `ilog2(size)` 中减去 `PAGE_SHIFT` 来计算阶数。当分配大小 `size` 小于一个页时，`ilog2(size)` 的值小于 `PAGE_SHIFT`，导致计算结果为负数。

#解决方案
修改 `mm/mm_init.c` 文件中的日志打印逻辑，使用 `get_order(size)` 函数替换原有的 `ilog2(size) - PAGE_SHIFT` 计算方式。`get_order()` 能够正确处理分配大小小于一个页的情况，返回正确的阶数 0。
