#标题
efi: unaccepted memory table 预留计算错误导致内核崩溃

#内核版本
OLK-6.6

#问题现象
在启动内存大小超过 64GB 的 Intel TDX 虚拟机时，内核在 accept_memory() 函数中发生 panic（崩溃）。原因是 unaccepted memory table 被覆盖或无法访问。

#主要错误日志
未知

#问题根因
reserve_unaccepted() 函数在计算 unaccepted memory table 的 memblock 预留大小时逻辑错误。原代码仅对表的大小进行页对齐，但未考虑到表的起始物理地址（efi.unaccepted）可能不是页对齐的情况。当表在页面中有起始偏移且结尾跨入下一个页面时，原计算方式无法覆盖表的末尾部分，导致该部分内存未被预留，进而可能被覆盖或导致访问错误引发内核 panic。

#解决方案
修改预留范围的计算方式：先计算包含非对齐起始地址在内的结束地址，再对其进行向上页对齐，最后通过（end - start）计算实际预留大小。这确保了整个表的范围都被 memblock 正确预留。
