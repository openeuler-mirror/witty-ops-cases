#标题
i2c-hid: 获取报告错误日志信息描述不准确

#内核版本
OLK-6.6

#问题现象
在 I2C HID 设备驱动中，当 `i2c_hid_get_report` 函数执行失败时，系统日志输出的错误信息描述不准确。现象表现为日志显示 "failed to set a report to device"（设置报告失败），而实际操作是 "get a report"（获取报告），导致运维或开发人员在排查故障时被误导，难以定位真实的代码执行路径。

#主要错误日志
- 错误（修复前）：`i2c_hid_acpi i2c-FTSC1000:00: failed to set a report to device: -11`
- 正确（修复后）：`i2c_hid_acpi i2c-FTSC1000:00: failed to get a report from device: -11`

#问题根因
在 `drivers/hid/i2c-hid/i2c-hid-core.c` 文件的 `i2c_hid_get_report` 函数中，错误处理分支的打印语句存在复制粘贴错误。开发者直接沿用了设置报告函数中的日志文本 "failed to set a report to device"，未根据当前函数上下文修改为获取报告相关的描述，导致日志内容与实际发生的操作不一致。

#解决方案
修正 `i2c_hid_get_report` 函数中的 `dev_err` 打印内容，将字符串 "failed to set a report to device" 修改为 "failed to get a report from device"，确保日志描述与函数实际执行的“获取报告”动作相符。
