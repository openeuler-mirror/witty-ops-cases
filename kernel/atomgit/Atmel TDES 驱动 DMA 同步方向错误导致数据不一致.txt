#标题
Atmel TDES 驱动 DMA 同步方向错误导致数据不一致

#内核版本
OLK-6.6

#问题现象
在非一致性（non-coherent）平台上，使用 Atmel TDES 加密驱动进行数据加密/解密操作后，CPU 读取的输出数据可能为过时的缓存数据，导致运算结果错误或数据不一致。

#主要错误日志
未知

#问题根因
在 DMA 传输结束且 CPU 消费数据前，驱动程序错误地调用了 `dma_sync_single_for_device()` 而不是 `dma_sync_single_for_cpu()`。根据 DMA API 规范，当 CPU 需要读取 DMA 输出缓冲区时，必须先调用 `dma_sync_single_for_cpu()` 以同步缓存；使用错误的同步方向或函数会导致 CPU 在非一致性平台上读取到错误的陈旧缓存数据。

#解决方案
修正 DMA 同步函数调用，将 `atmel_tdes_crypt_pdc_stop` 和 `atmel_tdes_crypt_dma_stop` 函数中的 `dma_sync_single_for_device` 替换为 `dma_sync_single_for_cpu`，确保在 CPU 访问缓冲区前正确同步 DMA 数据。
