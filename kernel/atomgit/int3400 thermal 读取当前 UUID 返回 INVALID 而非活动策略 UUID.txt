#标题
int3400 thermal 读取当前 UUID 返回 INVALID 而非活动策略 UUID

#内核版本
OLK-6.6

#问题现象
在 Ice Lake 之前的平台上，当用户空间将 `current_uuid` 属性设置为某个活动的散热策略 UUID 后，再次读取该属性时，返回值为 "INVALID" 字符串，而非之前设置的活动策略 UUID。

#主要错误日志
未知

#问题根因
在 commit c7ff29763989 ("thermal: int340x: Update OS policy capability handshake") 中，代码逻辑错误地将 `priv->current_uuid_index` 等于 0 视为无效情况。实际上，`priv->current_uuid_index` 为 0 代表活动策略（即 UUID 列表中的第一个有效项），只有当其为 -1 时才代表无效。该判断逻辑错误导致索引为 0 的合法活动策略无法正确显示。

#解决方案
修正 `drivers/thermal/intel/int340x_thermal/int3400_thermal.c` 中 `current_uuid_show` 函数的判断条件，将 `priv->current_uuid_index > 0` 修改为 `priv->current_uuid_index >= 0`，从而允许索引为 0 的活动策略 UUID 被正确读取。
