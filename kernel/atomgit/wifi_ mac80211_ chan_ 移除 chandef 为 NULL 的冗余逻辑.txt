#标题
wifi: mac80211: chan: 移除 chandef 为 NULL 的冗余逻辑

#内核版本
OLK-6.6

#问题现象
在 mac80211 信道上下文处理代码中，存在一段针对 `compat` 参数为 NULL 的历史兼容逻辑。由于上游代码演进，调用 `ieee80211_chanctx_reserved_chandef` 函数时已不再传入 NULL 参数（最后一个传入 NULL 的调用者已随 non-chanctx 代码移除），因此该段 NULL 检查逻辑已废弃且不再必要。

#主要错误日志
未知

#问题根因
该问题属于代码清理范畴。在旧版内核中，`ieee80211_chanctx_reserved_chandef` 函数需要处理 `compat` 参数为 NULL 的情况。但在当前版本中，所有调用该函数的代码路径均保证传入有效的 `compat` 指针（即 `chandef is non-NULL for reserved`）。原先代码中的 `if (!compat)` 判断分支已成为死代码（dead code），且逻辑上与参数不应为空的契约不符。

#解决方案
移除了 `ieee80211_chanctx_reserved_chandef` 函数内部对 `compat` 是否为 NULL 的检查逻辑，并在循环开始前直接增加 `WARN_ON(!compat)` 断言检查。若此时 `compat` 为 NULL 则直接返回 NULL，确保代码逻辑的正确性与简洁性，移除了遗留的冗余兼容处理。
