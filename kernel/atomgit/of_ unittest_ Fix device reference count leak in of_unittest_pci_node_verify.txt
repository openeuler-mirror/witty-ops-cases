#标题
of: unittest: Fix device reference count leak in of_unittest_pci_node_verify

#内核版本
OLK-6.6

#问题现象
在运行 OpenFlame 单元测试（unittest）时，`of_unittest_pci_node_verify` 函数在处理非添加场景（add 参数为 false）下存在设备引用计数泄漏问题。该问题通过代码审查发现，表现为设备对象的引用计数未能正确释放，可能导致设备无法被正常注销或资源无法释放。

#主要错误日志
未知

#问题根因
在 `of_unittest_pci_node_verify()` 函数中，当 `add` 参数为 false 时，代码调用 `device_find_any_child()` 获取子设备指针。根据内核 API 设计，`device_find_any_child()` 会隐式调用 `get_device()` 增加设备的引用计数。然而，调用者在获取指针后，未能在不需要时调用 `put_device()` 释放该引用，从而导致设备引用计数泄漏。

#解决方案
在 `else` 分支中，即 `child_dev` 使用完毕后，立即调用 `put_device(child_dev)` 来释放由 `device_find_any_child()` 获取的设备引用，确保引用计数平衡。
