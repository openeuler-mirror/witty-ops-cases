#标题
scripts/gdb: x86平台MCP后中断显示信息丢失

#内核版本
OLK-6.6

#问题现象
在使用 GDB 调试 x86 架构内核时，执行 MCP（Machine Check Polls）相关的中断信息显示操作后，中断统计信息的文本输出不完整。具体表现为 Machine check polls（MCP）的信息未能正确追加到显示结果中，导致中断显示功能异常，输出内容缺失。

#主要错误日志
未知

#问题根因
在 `scripts/gdb/linux/interrupts.py` 脚本的 `x86_show_interupts` 函数中，处理 MCP 信息时存在 Python 语法错误。原代码误将字符串追加操作 `+=` 写成了比较操作 `==`（`text == x86_show_mce(...)`）。这导致函数执行的是比较逻辑而非字符串拼接逻辑，MCP 的中断信息因此未被添加到最终输出的 `text` 变量中。

#解决方案
修改 `scripts/gdb/linux/interrupts.py` 文件，将第 145 行的错误操作符 `==` 修正为 `+=`，确保 MCP 的中断信息能正确追加到输出文本中。
