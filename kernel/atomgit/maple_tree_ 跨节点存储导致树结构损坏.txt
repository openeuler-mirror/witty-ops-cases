#标题
maple_tree: 跨节点存储导致树结构损坏

#内核版本
OLK-6.6

#问题现象
内核 maple tree 数据结构发生微妙损坏，导致系统内存管理不稳定。该问题自 maple tree 算法引入之初便存在，但在应用了特定的 VMA 树优化补丁（如 commit f8d112a4e657）后，触发概率显著增加，导致用户报告系统不稳定。损坏发生在特定的“跨越存储”场景下。

#主要错误日志
未知

#问题根因
问题源于 `mas_wr_spanning_store()` 函数在处理跨越两个叶节点的存储操作时存在逻辑缺陷。
当执行一次跨越存储，且右侧叶节点是其共享父节点的最右侧子节点，同时该存储操作完全消耗了该最右侧节点（即存储范围恰好结束于该节点的最大 pivot）时，代码错误地判断需要复制右侧节点的内容。
原代码使用 `r_mas.offset <= r_mas.end` 作为判断条件，导致在特定边界条件下（如存储范围为 [0x4000, 0xffff] 且节点最大 pivot 为 0xffff 时），错误地重复复制了位于最大 pivot 处的条目，从而导致 maple tree 数据结构损坏。

#解决方案
修改 `lib/maple_tree.c` 中的 `mas_wr_spanning_store()` 函数逻辑。将判断是否需要复制右侧节点内容的条件，从基于偏移量的检查 `r_mas.offset <= r_mas.end` 修改为基于数值范围的检查 `r_mas.max > r_mas.last`。
该修复通过检测右侧节点是否存在大于当前存储范围的条目（即是否有需要保留的内容），来决定是否执行复制操作，从而避免了错误的重复复制。同时更新了相关注释并移除了 `mas_wr_walk_index()` 中未使用的返回值。
