#标题
ALSA: usb-audio: convert_chmap_v3 函数 size 校验缺陷修复

#内核版本
OLK-6.6

#问题现象
在处理 USB 音频设备 UAC3 cluster segment descriptors 时，由于 `convert_chmap_v3` 函数内部的尺寸校验逻辑错误，未能正确拦截长度不足的描述符数据，可能导致后续解析越界或内存读取异常。

#主要错误日志
未知

#问题根因
代码中使用了 `void *p` 指针进行 `sizeof(*p)` 计算以校验长度。由于 `p` 是 void 指针，C 语言标准规定 `sizeof(void)` 或 `sizeof(*p)` 在大多数编译器环境下结果为 1（而非目标结构体的大小）。代码本意是校验缓冲区大小是否足以容纳 `cs_desc` 结构体（大小为 3 字节），但错误的 `sizeof` 使用导致校验条件恒为 `len < 1`，使得校验失效，无法正确判断数据长度是否符合结构体要求。

#解决方案
修正 `convert_chmap_v3` 函数中的 sizeof 计算对象，将 `sizeof(*p)` 修改为 `sizeof(*cs_desc)`，确保在解析前正确验证缓冲区长度是否满足 `uac3_cluster_segment_descriptor` 结构体（包含 wLength, wClusterDescrType 等字段）的基本大小要求。
