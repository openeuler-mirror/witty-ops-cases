#标题
crypto: api 中 crypto_alg_tested 算法活性检测不准确导致不必要的低优先级算法替换

#内核版本
OLK-6.6

#问题现象
在内核进行加密算法测试期间，若算法在测试过程中被移除，`crypto_alg_tested` 函数会误判算法状态。虽然算法已从主算法列表移除但仍处于待移除列表中，系统却仍尝试更新其状态并杀死其他低优先级算法的 spawns，导致产生不必要的系统动荡和性能开销。

#主要错误日志
未知

#问题根因
算法测试过程未持有主要的 crypto 锁，导致算法在测试期间可能被并发移除。
原代码仅通过检查 `list_empty(&alg->cra_list)` 来判断算法是否存活。该判断不准确，因为算法可能已从主列表移除，但仍存在于待移除列表中。在此时更新算法状态并清理低优先级算法的 spawns 是一种缺陷行为，会增加不必要的系统开销。

#解决方案
修改判断逻辑，将 `list_empty(&alg->cra_list)` 替换为 `crypto_is_dead(alg)`。通过直接检查算法是否已被标记为 DEAD 状态，准确判断算法是否已失效，从而避免对处于移除过程中的算法进行多余的状态更新和 spawns 清理操作。
