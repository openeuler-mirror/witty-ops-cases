#标题
[ crypto: hisilicon/qm ] switch语句缺少default分支导致编译告警

#内核版本
OLK-6.6

#问题现象
内核编译过程中，驱动代码因 switch 语句缺少 default 分支触发编译器告警，不符合内核代码风格规范要求。

#主要错误日志
未知

#问题根因
在函数 `qm_vft_data_cfg` 中，switch 语句处理 `vft_type` 类型时未包含 default 分支。当传入的 type 不在已定义的 case 范围内时，代码逻辑存在缺省处理缺失，既导致编译器产生告警，也未能满足内核代码风格对 switch 语句完备性的要求。

#解决方案
在 `drivers/crypto/hisilicon/qm.c` 文件的 `qm_vft_data_cfg` 函数 switch 语句中补充 default 分支。该分支当前仅执行 break 操作以消除告警，并在注释中说明当前逻辑仅需处理既定类型，若后续新增类型需同步补充处理逻辑以防未定义行为。
