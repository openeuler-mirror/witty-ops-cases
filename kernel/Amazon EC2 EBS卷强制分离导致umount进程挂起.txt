#标题
Amazon EC2 EBS卷强制分离导致umount进程挂起

#内核版本
ubuntu 3.9.6, 3.8.0.25, debian 7.0 kernel

#问题现象
- 在 Amazon EC2 半虚拟化实例上，当 mdraid 设备（由 2 个 EBS 卷组成）中的其中一个卷被强制分离后，执行 umount 操作会导致系统挂起。
- umount 进程处于不可中断睡眠状态（D state）。
- 内核日志显示调用栈阻塞在 `schedule` -> `md_write_start` -> `make_request` [raid1] -> `submit_bio` -> `blkdev_issue_flush` 路径上。

#问题根因
当底层 EBS 块设备被强制分离后，MD RAID1 驱动在处理写请求或刷新操作时，因无法与丢失的设备通信导致 I/O 请求无法完成，进而引起 `md_write_start` 或 `blkdev_issue_flush` 无限期阻塞，最终导致 umount 进程挂起。

#解决方案
暂无。该 Bug 报告因长期未更新已被关闭。
