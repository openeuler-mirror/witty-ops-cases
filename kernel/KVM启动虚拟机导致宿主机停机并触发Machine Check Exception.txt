#标题
KVM启动虚拟机导致宿主机停机并触发Machine Check Exception

#内核版本
4.2

#问题现象
- 启动特定虚拟机时，宿主机立即死机。
- 宿主机日志显示 Machine Check Exceptions (MCE)，例如：`mce: [Hardware Error]: CPU 3: Machine Check Exception: 5 Bank 17: be2000000003110a`。
- 出现 MTRR 覆盖相关的警告：`pmd_set_huge: Cannot satisfy [mem 0x383fe0000000-0x383fe0200000] with a huge-page mapping due to MTRR override`。

#问题根因
经二分排查，问题由 commit fd717f11015f673487ffc826e59b2bad69d20fe5 引入。该补丁标题为 "KVM: x86: apply guest MTRR virtualization on host reserved pages"，它取消了之前对 `kvm_is_reserved_pfn` 为真时跳过客户机 MTRR 的限制，启用了完整的客户机 MTRR 虚拟化。这导致直通设备（如 VGA 帧缓冲区）的内存类型设置与宿主机预期不符，从而触发硬件异常。

#解决方案
暂无官方修复补丁。临时规避方法：将故障虚拟机的 CPU 核心数设置为 2 或更少。
