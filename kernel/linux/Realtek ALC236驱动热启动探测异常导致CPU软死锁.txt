#标题
Realtek ALC236驱动热启动探测异常导致CPU软死锁

#内核版本
6.5.0+（亦确认存在于 6.5.8, 6.1.57, 5.10.198）

#问题现象
- 系统热启动后触发 CPU soft lockup，日志显示 "watchdog: BUG: soft lockup - CPU#2 stuck for 26s!"。
- Call Trace 指向 `sof_probe_work` -> `hda_codec_probe_bus` -> `snd_hdac_bus_send_cmd`。
- 开启音频调试发现，冷启动时 HDA codec #0 正确返回 ID `10ec0236`，但热启动时错误返回 `fe05`，随后系统卡死。
- 涉及硬件：HP Laptop 15s-fq4xxx (Tiger Lake), HP 250 G9 (Alder Lake)。

#问题根因
`sound/pci/hda/patch_realtek.c` 文件中的代码变更（具体为 commit 5aec989）引入了回归问题，导致 Realtek ALC236 编解码器在热启动过程中探测异常，返回错误的响应码，进而引发内核软死锁。

#解决方案
临时规避：回退（revert）提交 `5aec989`。
上游状态：目前 Bug 处于 NEW 状态，尚未合并正式修复补丁。
