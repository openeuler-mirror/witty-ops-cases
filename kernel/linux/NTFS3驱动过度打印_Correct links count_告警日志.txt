#标题
NTFS3驱动过度打印"Correct links count"告警日志

#内核版本
6.9.12 ~ 6.15.4（受影响），6.16（曾修复），6.19.8 ~ 7.0（回归）

#问题现象
系统运行过程中，dmesg 日志被 NTFS3 文件系统的大量告警信息刷屏，严重影响日志可读性。典型日志内容如下：
- `ntfs3: nvme0n1p3: ino=35bac, Correct links count -> 2.`
- `ntfs3: 130 callbacks suppressed`
受影响的磁盘分区经 Windows chkdsk 工具检查未发现文件系统错误。

#问题根因
NTFS3 驱动程序在 `fs/ntfs3/inode.c` 中调用 `ntfs_inode_warn` 函数，过于频繁地输出 "Correct links count" 信息。这是驱动层面的日志记录策略问题，而非文件系统本身损坏导致。

#解决方案
临时规避方法：注释掉内核源码 `fs/ntfs3/inode.c` 中的 `ntfs_inode_warn(inode, "Correct links count -> %u.", names);` 语句并重新编译内核。该问题在内核 6.16 版本中曾修复（状态变为 RESOLVED），但在后续版本 6.19.8 和 7.0 中再次复现（状态 REOPENED），目前尚无最终彻底修复方案。
