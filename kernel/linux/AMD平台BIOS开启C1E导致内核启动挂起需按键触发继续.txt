#标题
AMD平台BIOS开启C1E导致内核启动挂起需按键触发继续

#内核版本
2.6.33 - 4.7 (问题持续存在)

#问题现象
- 内核启动过程中随机挂起（hang），需按下键盘按键或电源键触发中断后方可继续引导。
- 关机过程中亦可能出现类似挂起现象。
- 主要发生在 Gigabyte AMD 主板（如 SB700/SB800 芯片组）且 BIOS 中开启了 "AMD C1E Support" 的情况下。
- dmesg 日志显示 IOAPIC ID 与 CPU LAPIC ID 存在冲突（均为 0）。

#问题根因
BIOS SMM 处理程序缺陷。BIOS 在实现 SMI Initiated C1E 功能时存在 Bug，导致系统在进入 C1E 状态后，IRQ0（定时器中断）路由失效或被阻塞。内核提交 73472a46b5b28116b145fb5fc05242c1aa8e1461（禁用 ATI SB700/SB800 的 HPET MSI）迫使系统使用传统的 IRQ0 路由，从而暴露了 BIOS 在 C1E 状态下无法正确处理 IRQ0 的问题。此外，BIOS 报告了非唯一的 APIC ID 也是潜在诱因。

#解决方案
临时规避方法（任选其一）：
1. 在 BIOS 设置中禁用 "AMD C1E Support"。
2. 在内核启动参数中添加 `idle=mwait`。
3. 在内核启动参数中添加 `acpi_skip_timer_override`。
4. 在内核启动参数中添加 `processor.max_cstate=1`。
根本解决需等待 BIOS 厂商修复 SMM 处理程序缺陷。
