#标题
__do_page_fault在无效上下文中调用睡眠函数引发内核NULL指针解引用

#内核版本
5.3.10

#问题现象
- 内核日志报告 "BUG: sleeping function called from invalid context in __do_page_fault"，位置在 arch/x86/mm/fault.c。
- 上下文检查显示 irqs_disabled(): 1，表明在禁用中断的上下文中调用了睡眠函数。
- 随后触发 "BUG: kernel NULL pointer dereference"，访问地址为 00000000000000e4，导致内核 Oops (0002)。
- Call Trace 指向：__do_page_fault -> async_page_fault -> do_syscall_64。

#问题根因
在 arch/x86/mm/fault.c 的 __do_page_fault 函数处理路径中，代码在 IRQ 中断被禁用的无效上下文中调用了可能睡眠的函数。这违反了内核原子上下文的编程规则，导致系统状态异常并最终引发 NULL 指针解引用错误。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）。
