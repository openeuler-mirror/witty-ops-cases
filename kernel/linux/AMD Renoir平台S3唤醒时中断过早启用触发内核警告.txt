#标题
AMD Renoir平台S3唤醒时中断过早启用触发内核警告

#内核版本
6.1.21

#问题现象
- 系统从S3挂起状态恢复时触发内核警告："Interrupts enabled before system core resume"
- Call Trace指向 `syscore_resume+0x173/0x180`
- 随后伴随出现 "BUG: using smp_processor_id() in preemptible code" 和 "BUG: using __this_cpu_read() in preemptible code" 错误

#问题根因
在 AMD Renoir APU 平台上，系统恢复过程中中断在 `syscore_resume` 执行前被意外启用。报告者初步分析认为可能是固件问题，因为在底层恢复代码与 `syscore_resume` 之间未观察到显式的中断相关代码执行。该问题复现概率极低（数百次S3循环中出现一两次）。

#解决方案
暂无
