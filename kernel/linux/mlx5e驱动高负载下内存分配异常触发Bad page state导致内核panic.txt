#标题
mlx5e驱动高负载下内存分配异常触发Bad page state导致内核panic

#内核版本
4.13.0+

#问题现象
- 系统发生 kernel panic。
- dmesg 日志中多次出现 "BUG: Bad page state in process ksoftirqd"，显示页面引用计数 count 为 -1。
- Call Trace 指向 `mlx5e_handle_rx_cqe_mpwrq` -> `__napi_alloc_skb` -> `page_frag_alloc` -> `get_page_from_freelist`。
- 伴随出现 `ixgbe` 驱动报 "Detected Tx Unit Hang" 并反复重置适配器。
- 触发场景为 Mellanox Connectx-5 100G 网卡接收 12Mpps UDP 流量，同时 Intel 82599 网卡发送流量。

#问题根因
在高流量网络负载下，Mellanox 驱动模块 `mlx5e` 在处理接收队列 (`mlx5e_handle_rx_cqe_mpwrq`) 并通过 `__napi_alloc_skb` 分配内存时，内存页面状态异常（`page count:-1`）。这表明系统存在内存引用计数管理错误（如双重释放或计数下溢），导致内存管理子系统检测到损坏并触发 panic。

#解决方案
暂无
