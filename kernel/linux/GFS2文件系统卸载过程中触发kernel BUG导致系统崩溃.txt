#标题
GFS2文件系统卸载过程中触发kernel BUG导致系统崩溃

#内核版本
5.2.2

#问题现象
- 系统在执行 umount 操作时触发 kernel BUG，导致内核 panic。
- 日志显示 "kernel BUG at fs/gfs2/glock.c:247!" 及 "invalid opcode: 0000 [#1]"。
- 故障现场堆栈显示调用路径为：`__gfs2_glock_put` -> `gfs2_glock_put` -> `gfs2_glock_put_eventually` -> `gfs2_evict_inode`。

#问题根因
问题发生在 GFS2 文件系统卸载流程中，具体位置在 `fs/gfs2/glock.c` 文件的第 247 行。在执行 `__gfs2_glock_put` 函数时触发了内核 BUG 宏（断言失败）。报告者推测可能与并发或随机因素有关，导致 glock 释放逻辑异常。

#解决方案
暂无
