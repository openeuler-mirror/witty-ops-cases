#标题
NEC uPD720200 USB 3.0 控制器使用 UAS 协议时导致 I/O 错误及进程阻塞

#内核版本
4.4.0-58-generic

#问题现象
- 系统启动后不久，USB 3.0 驱动停止响应。
- 内核日志显示 ATA 异常：`ata14.00: exception Emask 0x1`，提示 `HSM violation`。
- 触发 SCSI 错误处理流程：`uas_eh_abort_handler` 和 `uas_eh_bus_reset_handler`。
- 进程阻塞超过 120 秒：`task rs:main Q:reg:1720 blocked for more then 120 seconds`。
- 出现块设备 I/O 错误：`blk_update_request: I/O error, dev sda`，导致文件系统写入失败。

#问题根因
未明确。日志显示 UAS（USB Attached SCSI）驱动在处理 NEC uPD720200 USB 3.0 控制器时发生 HSM 违规及命令超时，导致块层 I/O 错误。

#解决方案
暂无官方修复。临时规避方法：将设备连接至 USB 2.0 接口，降级使用 USB 2.0 协议可恢复正常。
