#标题
USB音箱在Gnome声音测试中输出静电噪音

#内核版本
kernel-5.11.13-300.fc34.x86_64

#问题现象
- Philips WeCall USB Speaker 在 Gnome 设置的声音测试中输出静电噪音，但在 Chrome 浏览器播放 YouTube 视频时声音正常。
- 内核日志中出现 "cannot set freq 16000 to ep 0x3" 报错。
- Jabra SPEAK 510 USB 设备也出现类似日志 "cannot set freq 48000 to ep 0x3"。

#问题根因
未明确。开发者指出日志中的 "cannot set freq" 消息应是无害的，未确认噪音产生的根本原因。

#解决方案
暂无。开发者建议使用 speaker-test 或 aplay 测试原始 ALSA API 以进一步排查。
