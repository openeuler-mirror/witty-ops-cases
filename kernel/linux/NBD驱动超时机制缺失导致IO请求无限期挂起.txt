#标题
NBD驱动超时机制缺失导致IO请求无限期挂起

#内核版本
5.4.32

#问题现象
- 在NBD设备上执行btrfs balance操作时任务卡住。
- 内核日志持续打印 "block nbd3: Possible stuck request ... Runtime XXX seconds"，显示请求运行时间超过数千秒。
- 系统调用 sync 及其他涉及IO的进程（如 btrfs-cleaner、btrfs-transacti、dmcrypt_write）进入不可中断睡眠状态（D状态），导致系统无响应。
- 进程堆栈显示阻塞在 wbt_wait、wait_on_page_bit 等函数。

#问题根因
NBD驱动在 timeout 设置为 0 时，请求超时处理函数仅打印警告信息并返回 BLK_EH_RESET_TIMER，导致请求被无限重置定时器而不会失败返回。
若内核编译时未开启 CONFIG_DEBUG_FS，用户无法通过调试接口修改 timeout 参数，导致请求在服务端异常或网络中断时无法超时退出，永久阻塞持有资源的进程。

#解决方案
规避措施：在挂载NBD设备时，通过 nbd-client 工具的 -t 选项设置超时时间（调用 NBD_SET_TIMEOUT ioctl），避免请求无限期挂起。
内核修复建议：移除 timeout 设置对 CONFIG_DEBUG_FS 的依赖，允许用户在运行时配置超时参数（Bug状态为NEW，暂无合并补丁）。
