#标题
ksmd进程在unstable_tree插入节点时触发空指针解引用导致内核panic

#内核版本
5.10.0-136.71.0.151.u109.fos23.aarch64

#问题现象
- 系统运行 LTP ksm 测试用例及 CPU 压力测试约 4 天后触发 Kernel Panic 并重启。
- 日志报错："Unable to handle kernel NULL pointer dereference at virtual address 0000000000000008"。
- 故障堆栈显示：`rb_insert_color` <- `unstable_tree_search_insert` <- `cmp_and_merge_page` <- `ksm_do_scan` <- `ksm_scan_thread`。
- 触发进程为 ksmd，平台为 ARM（Kunpeng-920）。

#问题根因
初步崩溃分析表明，KSM 的 unstable_tree 中存在错误的红色根节点，导致在执行红黑树插入操作（`rb_insert_color`）时访问非法地址，触发空指针解引用。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）。
