#标题
btusb驱动挂起过程中触发空指针解引用导致内核Oops

#内核版本
6.11.2

#问题现象
- 系统运行过程中触发内核 Oops，日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000000"
- 故障发生在电源管理工作队列处理流程中，Call Trace 显示调用链为：pm_runtime_work -> rpm_idle -> usb_runtime_idle -> usb_runtime_suspend -> btusb_suspend
- 寄存器 RBX 值为 0，RIP 指向 btusb_suspend+0x1b/0x1d0 [btusb]

#问题根因
在 btusb_suspend 函数执行过程中，由于 RBX 寄存器值为 0（空指针），导致后续指令访问该地址时触发缺页异常。根据社区反馈，该问题与 MediaTek 蓝牙设备的电源管理有关，属于近期内核版本中的回归问题。

#解决方案
建议升级内核至 6.11.5 版本，相关 btusb 修复补丁已在 6.11.3 或 6.11.4 版本中合入。
