#标题
ext4 启用 inline_data 时写入操作触发 kernel BUG

#内核版本
5.15.0

#问题现象
- 系统日志打印 "kernel BUG at fs/ext4/inode.c:1721!" 及 "invalid opcode: 0000 [#1]"
- 故障发生在 ext4 文件系统启用 inline_data 的情况下，执行写入操作（如使用 Google Chrome）时触发
- Call Trace 显示路径为 ext4_da_get_block_prep -> __block_write_begin_int -> ext4_da_write_begin

#问题根因
Linux 5.15 版本合入的提交 948ca5f30e1df0c11eb5b0f410b9ceb97fa77ad9 在 fs/ext4/inode.c:1721 处引入了一个 BUG_ON() 检查。当 ext4 文件系统启用 inline_data 特性并进行写入操作时，该检查条件被触发导致内核 panic。

#解决方案
回退（revert）引发问题的提交 948ca5f30e1df0c11eb5b0f410b9ceb97fa77ad9 可规避此问题。
