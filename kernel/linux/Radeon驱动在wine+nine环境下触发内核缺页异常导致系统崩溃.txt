#标题
Radeon驱动在wine+nine环境下触发内核缺页异常导致系统崩溃

#内核版本
4.2-rc8（评论中提及 4.1、4.4-rc5、4.4-rc7 亦受影响）

#问题现象
- 系统日志报错 "BUG: unable to handle kernel paging request"，指令指针指向 `ttm_bo_del_from_lru` 函数。
- 在 PRIME (Intel + AMD Radeon) 环境下运行 CS:GO (wine+nine) 约 20-60 分钟后触发故障。
- 伴随 CPU soft lockup 和 RCU stall，系统显示冻结，无法通过 SSH 正常重启。
- 故障发生后 GPU 无法通过 runtime PM 关闭电源，需强制重启。

#问题根因
未明确。崩溃发生在 TTM (Translation Table Manager) 子系统的 `ttm_bo_del_from_lru` 函数中，尝试访问非法内存地址（如 ffff8001c4069120）。该问题可能与 Radeon 驱动在高负载下的内存管理或 DMA 处理逻辑有关。

#解决方案
暂无。用户尝试设置 `R600_DEBUG=nodma` 但未能解决问题。
