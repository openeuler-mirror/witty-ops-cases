#标题
amdgpu驱动非法寄存器访问导致GPU超时重置

#内核版本
5.4.16

#问题现象
- 系统每隔数小时随机崩溃，屏幕出现伪影，必须重启才能恢复。
- 内核日志报错：`[drm:gfx_v9_0_priv_reg_irq [amdgpu]] *ERROR* Illegal register access in command stream`。
- 随后触发 GPU reset，日志显示 `ring gfx timeout`。
- 在 GPU 重置流程中，`kthread_park` 触发 WARNING，调用栈为：`drm_sched_entity_fini+0x46/0x1d0 [gpu_sched]` -> `amdgpu_vm_fini` -> `amdgpu_driver_postclose_kms`。

#问题根因
GPU 命令流中发生非法寄存器访问，导致 GPU 挂起并触发 ring gfx 超时；在执行 GPU 重置及后续驱动清理流程（如关闭文件句柄释放 VM 资源）时，调度器实体销毁逻辑触发了内核线程状态异常。

#解决方案
暂无
