#标题
aacraid驱动IRQ亲和性映射缺陷导致控制器挂起及复位

#内核版本
6.4.0, 6.1.53 (受影响版本); 6.3.9 (正常版本)

#问题现象
- 系统升级至 Linux 6.4 内核后，Adaptec 系列 RAID 控制器（如 ASR71605, ASR8805）在 I/O 负载下出现间歇性挂起。
- 挂起持续数秒至数分钟，随后触发控制器复位。
- dmesg 日志显示大量 "aacraid: Host adapter abort request" 和 "aacraid: Host bus reset request. SCSI hang ?" 错误。
- 日志中出现 "aacraid: Outstanding commands on (x,x,x,x)" 以及 "outstanding cmd: firmware-XX" 信息。
- 高负载操作（如 btrfs balance、大文件拷贝）极易触发此问题。

#问题根因
由提交 `9dc704dcc09eae7d21b5da0615eb2ed79278f63e` ("scsi: aacraid: Reply queue mapping to CPUs based on IRQ affinity") 引入回归。该补丁修改了 Reply queue 到 CPU 的映射逻辑，导致驱动在处理命令时出现异常，引发命令超时和固件挂起。

#解决方案
- 临时规避：回退引入问题的提交 `9dc704dcc09eae7d21b5da0615eb2ed79278f63e`。
- 官方修复：上游已合并回退补丁 `c5becf57dd56` ("Revert "scsi: aacraid: Reply queue mapping to CPUs based on IRQ affinity"")。
- 后续进展：开发者正在提交新的补丁集以正确实现该功能。
