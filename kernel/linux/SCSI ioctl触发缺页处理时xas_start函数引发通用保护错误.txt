#标题
SCSI ioctl触发缺页处理时xas_start函数引发通用保护错误

#内核版本
v5.2-rc6

#问题现象
- 内核触发 general protection fault，RIP 位于 `xas_start` 函数 (lib/xarray.c:178)。
- KASAN 报告提示 GPF 可能由空指针解引用或用户内存访问引起。
- Call Trace 显示调用路径为：`sg_scsi_ioctl` -> `copy_to_user` -> `do_page_fault` -> `handle_mm_fault` -> `do_swap_page` -> `lookup_swap_cache` -> `find_get_entry` -> `xas_load` -> `xas_start`。

#问题根因
在执行 SCSI ioctl 命令（`SCSI_IOCTL_SEND_COMMAND`）并触发缺页异常处理（`do_swap_page`）时，内核在查找 swap cache 过程中调用 `xas_start` 访问 xarray 结构，因访问了无效内存地址（KASAN 检测到疑似空指针解引用）导致 GPF。

#解决方案
暂无（Bug 状态为 NEW，尚未提交修复补丁）。
