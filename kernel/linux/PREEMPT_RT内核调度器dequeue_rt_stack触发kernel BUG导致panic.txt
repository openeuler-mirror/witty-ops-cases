#标题
PREEMPT_RT内核调度器dequeue_rt_stack触发kernel BUG导致panic

#内核版本
6.12.13-talos, 6.12.18-mxie (PREEMPT_RT)

#问题现象
- 系统运行 PREEMPT_RT 内核时每天随机崩溃一到两次。
- 日志显示 "kernel BUG at kernel/sched/rt.c:1035!"，伴随 "Oops: invalid opcode"。
- Call Trace 显示崩溃路径为：dequeue_rt_stack -> enqueue_task_rt -> enqueue_task -> try_to_wake_up。
- 最终触发 "Kernel panic - not syncing: Fatal exception in interrupt"。
- 在 6.12.18 版本中，崩溃前出现 "psi: inconsistent task state" 告警。

#问题根因
内核在函数 `dequeue_rt_stack` (kernel/sched/rt.c:1035) 中触发了 `BUG()` 断言失败。日志中出现的 "psi: inconsistent task state" 表明调度器内部任务状态记录不一致，导致实时调度器在入队/出队操作时触发 panic。具体触发条件与代码缺陷尚在分析中。

#解决方案
暂无
