#标题
i2c-designware驱动关机时因msgs指针为空触发内核panic

#内核版本
5.8-5.8.0-43.49

#问题现象
- 系统在关机或重启过程中发生 Kernel panic 并挂起。
- 日志显示 RIP 指针位于 `i2c_dw_isr.cold+0x262/0x5ac`。
- 终端输出 "Kernel panic - not syncing : Fatal Exception in interrupt"。

#问题根因
在 `drivers/i2c/busses/i2c-designware-master.c` 文件的 `i2c_dw_xfer_msg` 函数中（对应 git commit 0daede80f8700），代码执行 `u32 addr = msgs[dev->msg_write_idx].addr;` 时，`msgs` 指针为 NULL，导致空指针解引用从而触发 panic。

#解决方案
用户通过重新编译内核，在相关代码路径增加对 `msgs` 指针的 NULL 检查临时规避了该问题。截至报告时间，上游尚未提供正式修复补丁。
