#标题
RT内核调度器pick_eevdf返回NULL引发空指针崩溃

#内核版本
6.12.18 (Debian 6.12.18-1.stx.109, PREEMPT_RT)

#问题现象
- 系统在进行高负载压力测试（K8s pod，RT线程，isolated CPU）时触发内核告警与崩溃。
- 日志中出现 WARNING: `!se->on_rq` at `kernel/sched/fair.c:704 update_entity_lag+0x7c/0x90`。
- 随后触发 BUG: `kernel NULL pointer dereference, address: 0000000000000051`。
- Call Trace 显示崩溃发生在 `pick_task_fair+0x4e/0x140` -> `pick_next_task_fair`。

#问题根因
函数 `pick_next_entity` 调用 `pick_eevdf(cfs_rq)` 获取调度实体时，`pick_eevdf` 返回了 NULL 指针。代码未对返回值进行校验即访问 `se->sched_delayed`（对应偏移量 0x51），导致空指针解引用。此前触发的 `update_entity_lag` 告警表明调度实体状态可能存在异常（`!se->on_rq`）。

#解决方案
暂无
