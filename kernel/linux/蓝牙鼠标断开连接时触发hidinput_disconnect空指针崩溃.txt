#标题
蓝牙鼠标断开连接时触发hidinput_disconnect空指针崩溃

#内核版本
4.1

#问题现象
- 使用 Intel 7260 Wifi/BT 组合模块连接蓝牙鼠标，当鼠标进入待机或断开连接时，系统间歇性发生 Kernel Panic。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at (null)"。
- 崩溃发生在 `hidinput_disconnect+0x2e/0xd0` 函数中。
- Call Trace: `hid_disconnect` -> `hid_device_remove` -> `hid_destroy_device` -> `hidp_session_remove` -> `l2cap_conn_del` -> `hci_rx_work`。

#问题根因
经 Bisect 定位，问题由提交 `297d716f6260cc9421d971b124ca196b957ee458` ("power_supply: Change ownership from driver to core") 引入。该提交改变了电源供应设备的所有权机制，导致在蓝牙 HID 设备断开连接并调用 `hidinput_disconnect` 时，访问了空指针。

#解决方案
升级至内核版本 4.2 可解决该问题。
