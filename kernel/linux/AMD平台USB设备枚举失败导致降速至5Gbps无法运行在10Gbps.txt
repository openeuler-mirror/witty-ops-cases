#标题
AMD平台USB设备枚举失败导致降速至5Gbps无法运行在10Gbps

#内核版本
6.8+

#问题现象
- AMD AM4/AM5 平台上，USB 3.2 Gen 2 (10Gbps) 设备（如 AVerMedia 采集卡、Hub）在 Linux 下仅能以 5Gbps (SuperSpeed) 运行，无法达到 10Gbps (SuperSpeedPlus)，而在 Windows 下正常。
- dmesg 日志显示设备初始被识别为 "new SuperSpeed Plus Gen 2x1 USB device"，随后读取描述符失败（`unable to get BOS descriptor`, `can't read configurations, error -71`）。
- 设备地址设置失败（`Device not responding to setup address`, `device not accepting address, error -71`），触发端口电源重置（`attempt power cycle`）。
- 最终设备以 "new SuperSpeed USB device" (5Gbps) 重新枚举成功。

#问题根因
主机控制器（疑似 AsMedia）在枚举过程中报告了错误的端口速度 ID (Port Speed ID)。这导致设备在尝试建立 10Gbps 连接时发生通信错误（读取描述符失败），内核驱动触发端口电源重置进行恢复，设备随后降级协商为 5Gbps 模式。

#解决方案
暂无主线内核修复。开发者提供了测试补丁（修改速度获取逻辑，从设备通知中获取速度而非依赖主机控制器报告），可修复部分 Hub 的 10Gbps 识别问题，但针对特定 AVerMedia 设备仍存在兼容性问题。厂商表示不支持 Linux。
