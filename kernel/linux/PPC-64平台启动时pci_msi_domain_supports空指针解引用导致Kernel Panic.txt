#标题
PPC-64平台启动时pci_msi_domain_supports空指针解引用导致Kernel Panic

#内核版本
6.16.8

#问题现象
- 系统启动过程中触发 Kernel Panic，日志显示 "BUG: Kernel NULL pointer dereference on read at 0x00000000"。
- 故障指令地址位于 `pci_msi_domain_supports+0x64/0xa0`。
- Call Trace 显示调用链为 `usb_hcd_pci_probe` -> `pci_alloc_irq_vectors_affinity` -> `__pci_enable_msi_range` -> `pci_msi_domain_supports`。
- 前置日志包含大量 PCI MSI 相关错误，如 "PCI: failed to find MSIs for bridge" 和 "of_irq_parse_pci: failed with rc=-22"。
- 问题在加载 ehci_pci 和 ohci_pci 模块时复现。

#问题根因
在 IBM POWER5+ (PPC-64) 平台上，内核初始化 PCI 子系统时未能为部分 PCI 桥接器找到 MSI domain（日志提示 "failed to find MSIs for bridge"）。当 USB 驱动尝试探测设备并调用 `pci_msi_domain_supports` 检查 MSI 支持情况时，因传入的 domain 指针为空（NULL），触发了空指针解引用异常。用户确认 5.15.* 版本无此问题，疑似内核版本升级引入的回归问题。

#解决方案
暂无修复补丁。开发者建议进行 git bisect 以定位引入问题的具体提交。用户反馈 5.15.* 版本正常，计划进一步测试 6.1、6.6 和 6.12 版本以缩小排查范围。
