#标题
断开蓝牙鼠标连接时触发 evdev 驱动 kernel BUG 导致系统死机

#内核版本
4.1.6

#问题现象
- 系统在蓝牙鼠标休眠或拔出蓝牙适配器时发生死机。
- 内核日志报错："kernel BUG at mm/slub.c:3413!" 和 "invalid opcode: 0000 [#1]"。
- 崩溃调用栈显示：`kfree` -> `evdev_free` -> `device_release` -> `kobject_release`。

#问题根因
在 `drivers/input/evdev.c` 文件的 `evdev_cleanup` 函数中，调用 `input_flush_device(handle, NULL)` 触发了内存管理错误，导致在 `kfree` 过程中触发 slub 分配器的 BUG 检查失败。

#解决方案
开发者建议修改 `drivers/input/evdev.c` 中的 `evdev_cleanup` 函数，移除 `input_flush_device(handle, NULL);` 调用，仅保留 `input_close_device(handle);`。该修复方案尚待验证。
