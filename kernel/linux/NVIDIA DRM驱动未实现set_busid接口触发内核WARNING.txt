#标题
NVIDIA DRM驱动未实现set_busid接口触发内核WARNING

#内核版本
3.19-rc4

#问题现象
系统运行时内核日志产生 WARNING 信息，堆栈指向 `drm_setversion+0x185/0x190 [drm]`。日志明确提示："No drm_driver.set_busid() implementation provided by nv_drm_driver [nvidia]. Use drm_dev_set_unique() to set the unique name explicitly."

#问题根因
NVIDIA 驱动（版本 346.35）中的 `nv_drm_driver` 结构体未实现 `set_busid` 回调函数。内核 DRM 子系统在 `drm_setversion` 流程中检查该实现时发现缺失，从而触发警告。

#解决方案
该问题需由 NVIDIA 驱动厂商修复，在驱动代码中实现 `set_busid` 接口或调用 `drm_dev_set_unique()` 显式设置唯一名称。内核侧暂无修复方案。
