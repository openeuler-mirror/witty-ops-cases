#标题
卸载 xhci-pci 模块触发 debugfs_remove_recursive 空指针解引用导致 Kernel Oops

#内核版本
4.15.3

#问题现象
- 执行 `rmmod xhci-pci` 卸载 USB 3.0 PCI-E 卡驱动模块时触发 Kernel Oops
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 00000000000000a8"
- 故障后 USB 集线器不可用，系统关机过程中发生挂起

#问题根因
在卸载 xhci 驱动过程中，调用链 `xhci_debugfs_remove_slot` -> `debugfs_remove_recursive` 执行清理操作时，传入了无效指针，导致在 `down_write` 函数中尝试访问地址 0xa8 时触发空指针解引用。

#解决方案
暂无（维护者建议将问题提交至 linux-usb@vger.kernel.org 邮件列表，并确认是否在最新内核版本中仍存在）。
