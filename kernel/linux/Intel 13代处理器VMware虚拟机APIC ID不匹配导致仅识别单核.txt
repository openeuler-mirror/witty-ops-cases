#标题
Intel 13代处理器VMware虚拟机APIC ID不匹配导致仅识别单核

#内核版本
未明确

#问题现象
- 系统启动时内核日志报错：`[Firmware Bug]: CPU1: APIC id mismatch. Firmware: 1 APIC: 2`，以及 CPU2、CPU3 类似的不匹配告警。
- 伴随报错：`atkbd serio0: failed to deactivate keyboard on isa0060/serio0`。
- 虚拟机运行缓慢，仅能使用一个处理器核心，导致该核心长期满载并产生过热。

#问题根因
内核日志明确标记为 Firmware Bug。在 Intel 13代 Raptor Lake 处理器的 VMware 虚拟机环境中，固件（BIOS/虚拟化层）报告的 CPU APIC ID 与内核通过 APIC 获取的 ID 不一致，导致多核处理器无法被正确识别和使用。
