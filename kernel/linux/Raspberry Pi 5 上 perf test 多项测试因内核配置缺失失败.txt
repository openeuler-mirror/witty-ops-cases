#标题
Raspberry Pi 5 上 perf test 多项测试因内核配置缺失失败

#内核版本
6.1（Raspberry Pi OS）

#问题现象
在 Raspberry Pi 5 上运行 `perf test` 时，多项测试失败，包括：
- 4.1: Read samples using the mmap interface - FAILED
- 6.1: Test event parsing - FAILED
- 14: syscalls:sys_enter_openat event fields - FAILED
- 80: kernel lock contention analysis test - FAILED
- 88: perf record offcpu profiling tests - FAILED

错误信息显示：
- "failed to parse event 'syscalls:sys_enter_openat', err 1, str 'unknown tracepoint'"
- "File /sys/kernel/tracing//events/syscalls/sys_enter_openat not found"
- "Hint: Perhaps this kernel misses some CONFIG_ set"

#问题根因
部分测试失败（如 kernel lock contention analysis、perf record offcpu profiling）是因为内核缺少 CONFIG_DEBUG_INFO_BTF 配置选项。其他失败（如 tracepoint sys_enter_getsid、sys_enter_openat 不存在）可能与内核 tracepoint 配置未启用有关。

#解决方案
重新编译内核并启用 CONFIG_DEBUG_INFO_BTF 配置选项，可修复 kernel lock contention analysis 和 perf record offcpu profiling 测试失败问题。
