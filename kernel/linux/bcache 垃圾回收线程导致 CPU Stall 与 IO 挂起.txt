#标题
bcache 垃圾回收线程导致 CPU Stall 与 IO 挂起

#内核版本
3.13.5

#问题现象
- 系统负载与 IO wait 急剧升高，机器近乎锁死，仅能通过 SysRq 响应。
- 内核日志报告 "INFO: rcu_sched self-detected stall on CPU { 0}"。
- 故障发生在 bcache 作为 ZFS 底层设备进行 resilvering 数据重建约 10GB 后。
- Call Trace 显示 `bcache_gc` 线程长时间运行，堆栈指向 `bch_btree_gc` -> `btree_gc_recurse` -> `bch_btree_iter_next`。

#问题根因
`bcache_gc` 线程在执行 B-tree 垃圾回收迭代过程中长时间占用 CPU 不释放，触发 RCU stall 检测机制，导致系统调度停滞。堆栈显示进程卡在 `bch_btree_iter_next` 及 `btree_iter_cmp` 函数中。

#解决方案
暂无（Bug 状态为 NEW，维护者询问是否在 Linux v5.5 或 v5.6-rc 中复现）。
