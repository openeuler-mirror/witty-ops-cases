#标题
btusb驱动加载Broadcom固件失败触发内核WARNING

#内核版本
3.16.0

#问题现象
- 系统从挂起状态唤醒时，内核日志打印 WARNING 堆栈信息。
- 报错位于 `drivers/base/firmware_class.c` 的 `_request_firmware` 函数。
- 关键调用栈：`hci_power_on` -> `hci_dev_do_open` -> `btusb_setup_bcm_patchram` -> `request_firmware`。
- 日志提示：`bluetooth hci0: firmware: brcm/BCM20702A0-0a5c-21e6.hcd will not be loaded` 以及 `Bluetooth: hci0: BCM: patch brcm/BCM20702A0-0a5c-21e6.hcd not found`。
- 该问题会导致内核被标记为 tainted，可能掩盖后续真实的内核错误信息。

#问题根因
问题由 commit `10d4c6736ea6e6ff293dd588551270bca00ca45d` ("Bluetooth: btusb: Add Broadcom patch RAM support") 引入。该补丁为 Broadcom 蓝牙适配器增加了 Patch RAM 支持。在系统唤醒过程中，`btusb` 驱动在工作队列中尝试加载 `.hcd` 固件文件。由于固件文件缺失或加载时机不当，触发了 `firmware_class` 中的 WARNING。

#解决方案
暂无内核补丁合并记录。规避方法：手动获取或转换对应的 Broadcom 固件文件（如 `BCM20702A0-0a5c-21e6.hcd`），并将其放置到 `/lib/firmware/brcm/` 目录下，可消除报错。
