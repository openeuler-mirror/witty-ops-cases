#标题
ext4_xattr_block_set 函数触发内核 WARNING

#内核版本
v6.1-rc8

#问题现象
- 系统日志中出现内核 WARNING：`WARNING: CPU: 0 PID: 567 at fs/ext4/xattr.c:2069 ext4_xattr_block_set+0x170d/0x1770`
- 触发路径为 chmod 操作相关流程：`chmod_common` -> `notify_change` -> `ext4_setattr` -> `__mark_inode_dirty` -> `ext4_expand_extra_isize_ea` -> `ext4_xattr_block_set`

#问题根因
经 Bisect 定位，问题由提交 `cebe85d570cf84804e848332d6721bc9e5300e07` ("ext4: switch to the new mount api") 引入。具体的代码逻辑错误细节未在报告中明确。

#解决方案
