#标题
SDHCI驱动加载时因sysfs重复创建文件名触发内核警告

#内核版本
2.6.39-rc7-git1（报告版本），复现版本包括 3.0-rc2、3.5

#问题现象
- 系统启动过程中产生内核警告信息：`WARNING: at fs/sysfs/dir.c:455 sysfs_add_one+0x83/0x95()`。
- 日志显示 sysfs 创建重复文件名错误：`sysfs: cannot create duplicate filename '/devices/pci0000:00/0000:00:1e.0/0000:02:0b.4/microamps_requested_vmmc'`。
- Call Trace 显示调用链为 `sdhci_pci_probe` -> `sdhci_add_host` -> `regulator_get` -> `create_regulator`。
- 系统提示无法找到 vmmc 电源供应，使用了 dummy regulator：`_regulator_get: 0000:02:0b.4 supply vmmc not found, using dummy regulator`。

#问题根因
`sdhci` 驱动在初始化过程中调用 `regulator_get` 获取电源供应（vmmc），因未找到有效供应而切换使用 dummy regulator。在此过程中，驱动程序或 regulator 框架尝试创建 sysfs 文件 `microamps_requested_vmmc` 时发生命名冲突。分析认为这是驱动程序错误地请求了两次电源供应，或者是在创建 sysfs 接口时未正确处理已存在文件的情况。

#解决方案
