#标题
notify_change 函数触发 WARNING 导致 Kernel Panic

#内核版本
v5.0.0

#问题现象
- 系统触发 Kernel Panic，提示 "panic_on_warn set ..."。
- 日志显示内核在 fs/attr.c:234 处触发 WARNING：`WARNING: CPU: 1 PID: 5384 at fs/attr.c:234 notify_change+0x807/0xc80`。
- 故障调用栈显示路径为：`blkdev_write_iter` -> `__generic_file_write_iter` -> `file_remove_privs` -> `__remove_privs` -> `notify_change`。

#问题根因
在执行文件写入操作并尝试移除特权（`file_remove_privs`）时，`notify_change` 函数在 `fs/attr.c:234` 处触发了内核 WARNING 检查。由于系统配置了 `panic_on_warn`，该 WARNING 进一步引发了 Kernel Panic。具体的 WARNING 触发逻辑条件未在报告中详细说明。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）。
