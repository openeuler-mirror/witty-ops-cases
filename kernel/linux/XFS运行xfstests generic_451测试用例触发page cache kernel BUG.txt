#标题
XFS运行xfstests generic/451测试用例触发page cache kernel BUG

#内核版本
v6.1-rc0

#问题现象
- 在 XFS 文件系统（配置 64k 目录块大小）上运行 xfstests generic/451 测试用例时触发 kernel panic。
- 故障现象一：触发 `kernel BUG at mm/truncate.c:669!`，日志显示 `page dumped because: VM_BUG_ON_FOLIO(!folio_contains(folio, index))`。Call Trace 指向 `invalidate_inode_pages2_range` -> `iomap_dio_complete_work`。
- 故障现象二：复现过程中偶发 `kernel BUG at include/linux/pagemap.h:1247!`，位于 `read_pages` 函数。Call Trace 指向 `filemap_get_pages` -> `xfs_file_buffered_read`。

#问题根因
开发者分析认为这是 page cache 层面的问题，对应断言 `BUG_ON(ractl->_batch_count > ractl->_nr_pages)` 失败，与 XFS 或目录块大小配置无直接关联。具体代码缺陷尚未明确。

#解决方案
暂无（开发者建议进行 bisect 以定位引入问题的 commit）。
