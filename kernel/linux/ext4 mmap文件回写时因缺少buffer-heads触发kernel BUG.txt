#标题
ext4 mmap文件回写时因缺少buffer-heads触发kernel BUG

#内核版本
3.10.0-229.el7.x86_64

#问题现象
- 系统日志显示 I/O 错误，EXT4 报告延迟块分配失败（error -5）。
- 进程执行 unmap 操作时触发 WARNING：`WARNING: at fs/buffer.c:661 __set_page_dirty`。
- 随后触发 kernel BUG：`kernel BUG at fs/ext4/inode.c:2263!`，导致系统崩溃。
- Call Trace 指向 `mpage_prepare_extent_to_map` -> `ext4_writepages`。

#问题根因
在 I/O 错误导致延迟块分配失败后，`ext4_releasepage` 调用链（`try_to_free_buffers` -> `cancel_dirty_page`）释放了 buffer heads 但未清除 PTE dirty 标志。这导致 `unmap_page_range` 随后再次将页面标记为 dirty。当 `ext4_writepages` 尝试回写该页面时，调用 `page_buffers(page)` 发现页面没有附加 buffer-heads，从而触发 BUG_ON（位于 `fs/ext4/inode.c:2263`）。

#解决方案
暂无
