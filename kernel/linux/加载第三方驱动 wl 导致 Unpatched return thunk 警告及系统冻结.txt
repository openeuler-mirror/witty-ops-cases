#标题
加载第三方驱动 wl 导致 Unpatched return thunk 警告及系统冻结

#内核版本
6.13.8-200.fc41.x86_64

#问题现象
- 系统在启动几分钟后发生冻结，回退到内核 6.13.7 后正常。
- 日志中出现内核警告："Unpatched return thunk in use. This should not happen!"。
- 堆栈信息显示警告位于 arch/x86/kernel/cpu/bugs.c:3070 __warn_thunk。
- 内核被标记为 Tainted: P（PROPRIETARY_MODULE）、O（OOT_MODULE）、E（UNSIGNED_MODULE），涉及模块 wl。

#问题根因
第三方专有内核模块 wl（Broadcom 无线驱动）未适配内核 6.13.8 中的 SRSO（Speculative Return Stack Overflow）安全缓解机制。该模块在执行返回指令时使用了未修补的 return thunk（srso_alias_return_thunk），触发了内核的安全检查逻辑，导致系统不稳定或冻结。

#解决方案
暂无内核侧修复。建议移除或禁用第三方专有模块 wl 以规避此问题。
