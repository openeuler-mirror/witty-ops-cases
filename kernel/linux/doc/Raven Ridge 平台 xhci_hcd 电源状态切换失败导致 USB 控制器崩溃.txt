#标题
Raven Ridge 平台 xhci_hcd 电源状态切换失败导致 USB 控制器崩溃

#内核版本
4.19.23（评论中提及 4.19.32_1）

#问题现象
- 插入 USB 设备后无法识别，`lsusb` 无显示，xhci_hcd 驱动不可用。
- 内核日志报错 "Refused to change power state, currently in D3"。
- 出现 "WARN: xHC restore state timeout" 及 "PCI post-resume error -110!"，随后控制器崩溃 "HC died; cleaning up"。
- 禁用 laptop_mode 后，USB 在挂起前可用，但系统挂起恢复后再次出现 "xHCI host not responding... assume dead" 错误。

#问题根因
xhci_hcd 驱动在 Raven Ridge (Ryzen 5 2500U) 平台上处理电源状态（D3状态）时存在异常。在设备处于 D3 状态时，驱动尝试恢复或操作设备失败，导致控制器响应超时并被判定为死亡。

#解决方案
暂无官方修复。临时规避方法为禁用 laptop_mode，但此方法仅能避免挂起前的问题，无法解决挂起恢复后的故障。维护者建议将问题提交至 linux-usb 邮件列表跟进。
