#标题
ASUS KGPE-D16 平台内核启动时 SLAB 初始化空指针解引用导致 Panic

#内核版本
4.19.48 (日志显示为 4.19.0-5-amd64)

#问题现象
- 系统在加载 initramfs 阶段发生 Kernel Panic，无法正常启动。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000000"。
- 触发 Oops 后引发 Panic："Attempted to kill the idle task!"，随后触发 double fault。
- 使用启动参数 `acpi=off` 可引导系统，但会导致 SATA 控制器无法正常工作。
- 降级至 4.9.x 内核可正常运行。

#问题根因
在内核初始化阶段，SLAB 分配器初始化流程（`start_kernel` -> `kmem_cache_init` -> `bootstrap` -> `kmem_cache_alloc` -> `deactivate_slab`）中，调用 `_raw_spin_lock` 时传入了空指针（RDI 寄存器值为 0），导致空指针解引用。该问题出现在特定的硬件环境（ASUS KGPE-D16 主板、Coreboot 固件、AMD Opteron 6380 处理器）下。

#解决方案
暂无官方修复补丁。临时规避方案为降级使用 4.9.x 版本内核。
