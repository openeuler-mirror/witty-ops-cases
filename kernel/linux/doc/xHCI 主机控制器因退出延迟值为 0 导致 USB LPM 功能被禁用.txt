#标题
xHCI 主机控制器因退出延迟值为 0 导致 USB LPM 功能被禁用

#内核版本
4.18.0-10-generic（问题在 6.13.9 仍存在）

#问题现象
- 系统日志中出现警告信息："usb usbX: We don't know the algorithms for LPM for this host, disabling LPM."
- 受影响的 xHCI 主机控制器设备的 USB 链路电源管理（LPM）功能被内核禁用。

#问题根因
受影响主机的 USB 设备描述符中，`bU1DevExitLat` 和 `bU2DevExitLat` 字段值均为 0 微秒。内核代码在检查这些延迟值时，因无法确定该主机支持的 LPM 算法，从而触发警告并禁用 LPM。

#解决方案
暂无修复。维护者建议将问题提交至 linux-usb 邮件列表跟进。
