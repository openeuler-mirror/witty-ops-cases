#标题
Intel AX200/AX210 蓝牙设备启动时 command 0xfc05 tx timeout 导致不可用

#内核版本
6.7.1, 6.8.2, 6.9.1 (受影响版本范围 6.6 至 6.9.x)

#问题现象
- 系统启动时蓝牙功能不可用，`bluetoothctl` 无法连接。
- 内核日志报错：`Bluetooth: hci0: command 0xfc05 tx timeout` 以及 `Bluetooth: hci0: Reading Intel version command failed (-110)`。
- 在较新内核版本（如 6.7.4+）中，日志可能显示 `Bluetooth: hci0: FW download error recovery failed (-19)` 或 `Firmware already loaded` 但设备仍无法工作。
- 正常重启或从挂起状态恢复后问题复现。

#问题根因
开发者分析认为 `0xfc05` 命令超时意味着蓝牙控制器未响应，可能是通信受阻或控制器状态异常导致。用户在测试开发者提供的修复补丁时，触发了 `btintel_configure_setup` 函数中的空指针解引用（NULL pointer dereference），表明驱动在初始化流程中对指针处理存在缺陷。

#解决方案
暂无官方正式修复补丁合并。
临时规避方案：
1. 完全断电后重启。
2. 手动重载驱动模块：`modprobe -r btusb btintel`，等待数秒后 `modprobe btintel btusb`。
部分用户反馈升级至内核 6.8.4 后问题解决，但亦有用户报告在 6.9.x 版本仍需手动干预。
