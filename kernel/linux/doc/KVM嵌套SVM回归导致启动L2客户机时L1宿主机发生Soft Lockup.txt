#标题
KVM嵌套SVM回归导致启动L2客户机时L1宿主机发生Soft Lockup

#内核版本
5.7-rc1（受影响版本包括5.7.x系列）

#问题现象
- 在L1宿主机（内核版本5.4.30）中启动L2客户机时，L1宿主机几乎立即挂起。
- 系统日志显示 "watchdog: BUG: soft lockup - CPU#6 stuck for 22s!"。
- 堆栈跟踪显示进程卡在 `smp_call_function_many`、`flush_tlb_kernel_range` 或 `load_new_mm_cr3` 等TLB刷新相关函数中。
- 该问题在AMD Ryzen 3960X/3900X平台上可稳定复现，回退至内核5.6.19版本正常。

#问题根因
经二分定位，问题由提交 b518ba9fa691 ("KVM: nSVM: implement check_nested_events for interrupts") 引入。该提交在处理嵌套虚拟化中断事件时存在缺陷，导致CPU软锁死。

#解决方案
升级至内核 5.8 版本，相关修复已合入该版本。
