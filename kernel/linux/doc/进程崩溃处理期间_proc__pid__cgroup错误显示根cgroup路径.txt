#标题
进程崩溃处理期间/proc/<pid>/cgroup错误显示根cgroup路径

#内核版本
4.14

#问题现象
- 进程崩溃后，在自定义 core handler 脚本执行期间读取 `/proc/<pid>/cgroup`，显示的 cgroup 路径为根路径 "/"，而非预期的容器路径（如 `/docker/...`），导致无法识别进程所属容器。
- 该现象仅在特定应用（可能是多线程应用）崩溃时偶发，无法通过简单的空指针解引用或信号触发稳定复现。
- 同一时刻 `/proc/<pid>/cpuset` 文件内容显示正常，包含正确的容器 ID 信息。

#问题根因
未明确。用户推测可能与进程崩溃时线程状态或 cgroup 退出机制有关，但尚未确认具体内核代码逻辑。

#解决方案
暂无内核修复。可通过读取 `/proc/<pid>/mountinfo` 或 `/proc/<pid>/cpuset` 文件作为替代方案获取容器 ID 信息。
