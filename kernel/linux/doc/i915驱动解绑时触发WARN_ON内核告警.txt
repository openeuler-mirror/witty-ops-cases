#标题
i915驱动解绑时触发WARN_ON内核告警

#内核版本
4.15.0-20-generic

#问题现象
- 用户尝试通过QEMU启动虚拟机（涉及VFIO设备直通）时，内核日志打印WARNING告警。
- 告警信息：`WARN_ON(reset && reset != -19)`。
- 调用栈显示告警发生在 `i915_gem_sanitize` 函数中，触发路径为 `unbind_store` -> `i915_pci_remove` -> `i915_driver_unload` -> `i915_gem_suspend` -> `i915_gem_sanitize`。

#问题根因
在执行 i915 显卡驱动解绑操作时（通常用于将设备切换至 vfio-pci 驱动），`i915_gem_sanitize` 函数内部的 `reset` 状态检查触发了 `WARN_ON` 条件，导致内核打印告警堆栈。

#解决方案
暂无
