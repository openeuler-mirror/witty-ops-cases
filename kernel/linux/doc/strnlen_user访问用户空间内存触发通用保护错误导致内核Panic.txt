#标题
strnlen_user访问用户空间内存触发通用保护错误导致内核Panic

#内核版本
v5.2-rc6

#问题现象
- Syzkaller 测试触发 Kernel Panic，系统自动重启。
- 日志显示 "General protection fault in user access. Non-canonical address?"。
- 触发 WARNING：`arch/x86/mm/extable.c:126 ex_handler_uaccess+0xc5/0xe0`。
- 关键调用栈：`strnlen_user` -> `copy_strings` -> `__do_execve_file` -> `call_usermodehelper_exec_async`。

#问题根因
内核在执行用户模式辅助程序过程中，调用 `strnlen_user` 函数访问用户空间内存时触发 General Protection Fault（通用保护错误）。日志提示可能访问了非规范地址，导致异常处理函数 `ex_handler_uaccess` 触发警告，由于设置了 `panic_on_warn` 最终导致系统 Panic。

#解决方案
暂无
