#标题
r8169驱动变更导致Toshiba笔记本从挂起唤醒时黑屏崩溃

#内核版本
3.16.1（受影响版本包括3.16至3.17-rc2）

#问题现象
- Toshiba Satellite C660D-10P笔记本电脑在从挂起（Suspend to RAM）唤醒时系统崩溃，屏幕变黑
- 故障可复现，每次合盖或手动挂起后唤醒均触发
- 崩溃后重启有时会导致显示器无法初始化
- 最后一个正常工作的内核版本为3.15.8

#问题根因
经 git bisect 定位，问题由 commit b51ecea852b712618796d9eab8428a7d5f1f106f 引入。该提交修改了 r8169 驱动中的函数 rtl_hw_start_8168g_1，导致系统在挂起唤醒流程中崩溃。

#解决方案
临时规避方法：
1. 将 r8169 驱动加入黑名单。
2. 回退 commit b51ecea852b712618796d9eab8428a7d5f1f106f。
用户已针对 3.17.3 版本制作修复补丁，部分回退了 rtl_hw_start_8168g_1 函数中的变更。
