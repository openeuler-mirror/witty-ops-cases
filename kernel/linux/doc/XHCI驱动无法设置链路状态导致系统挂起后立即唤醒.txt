#标题
XHCI驱动无法设置链路状态导致系统挂起后立即唤醒

#内核版本
4.14.8-1-ARCH

#问题现象
- MacBookPro11,3 笔记本在前两次挂起（S3）和唤醒操作正常，第三次执行挂起操作后约 3 秒自动唤醒，后续挂起尝试均失败。
- 系统日志显示 xhci_hcd 驱动报错：`usb 2-4: usb_reset_and_verify_device Failed to disable LTM`、`xhci_hcd 0000:00:14.0: Cannot set link state` 以及 `usb usb2-port4: cannot disable (err = -32)`。

#问题根因
第二次唤醒后 USB 设备状态变得不稳定，xhci_hcd 驱动在处理 USB 端口禁用或链路状态设置时失败（返回错误码 -32），导致 XHCI 控制器在后续挂起过程中产生异常唤醒中断。

#解决方案
- 临时规避方法 1：同时禁用 XHC1 和 LID0 的唤醒源（执行 `echo XHC1 | sudo tee /proc/acpi/wakeup` 和 `echo LID0 | sudo tee /proc/acpi/wakeup`）。
- 临时规避方法 2：将 `xhci_hcd` 和 `xhci_pci` 模块加入黑名单（会导致 USB 功能不可用）。
- 官方修复：暂无，建议测试最新上游内核并向 linux-usb 邮件列表反馈。
