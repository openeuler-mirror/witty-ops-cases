#标题
GFS2文件系统写入时触发空指针解引用导致Oops

#内核版本
4.19.132（回归引入于 4.19.130）

#问题现象
- 向 GFS2 文件系统执行写入操作失败，进程被终止。
- 内核日志报错："BUG: unable to handle kernel NULL pointer dereference at 0000000000000008"。
- 触发 Oops，RIP 寄存器指向 `gfs2_log_commit+0x104/0x400 [gfs2]`。
- 调用栈显示：`gfs2_trans_end` -> `gfs2_create_inode` -> `gfs2_atomic_open`。

#问题根因
由 commit `c91cffd0fd010c06d67f3a9a528b858ce28c60fb` 引入的回归，该补丁导致 `gfs2_log_commit` 函数在执行过程中访问了非法的空指针地址。

#解决方案
修复补丁 `df4ffe7cb0f3a3fee591f93ac085f4b6e7e59694` 已提交至 v4.19.y 稳定版队列；临时规避方案为回退引入问题的 commit。
