#标题
perf工具无法识别使用-Ttext-segment参数编译的.so文件符号

#内核版本
4.1.1

#问题现象
- 使用 `perf record` 记录使用了 `-Wl,-Ttext-segment,0xXXXXX` 链接参数编译的共享库（.so）的应用程序。
- 执行 `perf script` 或 `perf report` 时，无法解析该 .so 文件中的符号，仅显示十六进制地址（如 `2040056c`），而非符号名称（如 `entry_func`）。
- 若不使用 `-Wl,-Ttext-segment` 参数编译，符号解析功能正常。

#问题根因
在 `tools/perf/util/symbol-elf.c` 文件的 `symsrc__init` 函数中，`ss->adjust_symbols` 的判断条件未包含 `ehdr.e_type == ET_DYN`。当 .so 文件使用 `-Ttext-segment` 指定加载地址时，其文件类型仍为 `ET_DYN`，但由于缺少该类型判断，导致符号地址调整逻辑未生效，从而无法正确识别符号。

#解决方案
用户建议修改 `tools/perf/util/symbol-elf.c`，在 `ss->adjust_symbols` 的赋值条件中增加 `ehdr.e_type == ET_DYN ||`。该补丁尚未合并，Bug状态为 NEW。
