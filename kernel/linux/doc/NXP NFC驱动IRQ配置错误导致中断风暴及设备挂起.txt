#标题
NXP NFC驱动IRQ配置错误导致中断风暴及设备挂起

#内核版本
未明确（问题引入于 commit 57be33f）

#问题现象
- 部分联想 PC（x86/ACPI 平台）出现中断风暴
- 受影响设备发生挂起，无法正常工作

#问题根因
Commit 57be33f85e369ce9f69f61eaa34734e0d3bd47a7 修改了 NXP 驱动的 IRQ 配置逻辑，移除了显式的 IRQ 触发类型设置，转而依赖 Device Tree 进行配置。由于 x86 设备使用 ACPI 而非 Device Tree，导致 IRQ 触发类型未被正确配置，从而引发中断风暴。

#解决方案
建议短期回退该补丁；长期方案需重新实现逻辑，仅在存在 Device Tree 时依赖其配置 IRQ。目前暂无已合并的修复补丁。
