#标题
BFQ调度器空指针解引用导致系统挂起

#内核版本
5.15.32

#问题现象
- 系统配置 BFQ I/O 调度器后，在高 I/O 负载下导致整个平台挂起。
- 所有 CPU 核心锁定，无法通过 alt+sysrq+b 重启，看门狗可能触发重启。
- 内核日志显示空指针解引用错误：`BUG: kernel NULL pointer dereference, address: 0000000000000000`。
- 调用栈显示：`__list_del_entry_valid` <- `bfq_idle_extract` <- `bfq_put_idle_entity` <- `bfq_bfqq_served` <- `bfq_dispatch_request`。

#问题根因
BFQ I/O 调度器模块中的 `bfq_idle_extract` 函数在处理空闲实体时触发了空指针解引用，具体表现为在调用 `__list_del_entry_valid` 进行链表删除操作时访问了空地址。

#解决方案
暂无
