#标题
Ext4/JBD2 checkpoint逻辑竞态导致buffer head错误移除引发文件系统损坏

#内核版本
6.4.0-rc3

#问题现象
- 系统触发 Kernel panic（测试环境注入延时后触发），日志显示 "Buffer I/O error ... lost async page write"。
- 重启后执行 fsck 检查发现文件系统不一致，报错 "Unattached inode" 及 "UNEXPECTED INCONSISTENCY"。
- 文件系统元数据丢失，导致损坏。

#问题根因
JBD2 事务处理逻辑中存在竞态条件。在 `jbd2_log_do_checkpoint`、`jbd2_journal_forget` 及 `__cp_buffer_busy` 等函数中，代码通过检查 `buffer_dirty(bh)` 状态来决定是否从 checkpoint list 中移除 buffer head。
当后台回写线程 `wb_workfn` 调用 `__block_write_full_page` 清除 dirty 标志并提交 IO 时，若此时发生 IO 错误或 IO 尚未完成，checkpoint 逻辑可能因为看到 `!buffer_dirty` 而错误地移除该 buffer head，且未正确检查 `buffer_write_io_error` 或等待 IO 结束。这导致元数据丢失，进而破坏文件系统一致性。

#解决方案
暂无已合并补丁，报告者表示正在编写正式修复补丁。
