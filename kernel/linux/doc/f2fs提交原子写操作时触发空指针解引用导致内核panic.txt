#标题
f2fs提交原子写操作时触发空指针解引用导致内核panic

#内核版本
6.2.8

#问题现象
- 系统在日常使用场景下触发 Kernel Panic
- 日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000000"
- RIP 寄存器指向 f2fs 模块中的 `__replace_atomic_write_block+0x184/0x620`
- 调用栈显示崩溃路径为：`f2fs_commit_atomic_write` -> `__f2fs_ioctl` -> 系统调用

#问题根因
在 f2fs 文件系统执行原子写提交（atomic write commit）过程中，函数 `__replace_atomic_write_block` 内部访问了空指针（RAX 寄存器值为 0），触发了页面错误异常导致内核崩溃。

#解决方案
开发者已提供修复补丁，链接：https://lore.kernel.org/linux-f2fs-devel/ZCsEytDjqEjQDPiO@google.com/T/#u
