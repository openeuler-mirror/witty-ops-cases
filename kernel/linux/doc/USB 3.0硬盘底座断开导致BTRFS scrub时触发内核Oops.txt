#标题
USB 3.0硬盘底座断开导致BTRFS scrub时触发内核Oops

#内核版本
3.11, 3.14, 3.15

#问题现象
- USB 3.0 外置硬盘底座在使用过程中频繁断开连接。
- dmesg 日志显示 "Disable of device-initiated U1 failed" 与 "Disable of device-initiated U2 failed" 错误，随后报告 "USB disconnect"。
- 设备断开后，系统触发 Kernel Oops：`BUG: unable to handle kernel paging request at ffff8809b9d13170`。
- 崩溃发生在 `kfree+0x104/0x200`，Call Trace 指向 BTRFS scrub 相关路径（`btrfs_lookup_csums_range` -> `scrub_stripe` -> `btrfs_scrub_dev`）。

#问题根因
USB 3.0 设备意外断开连接（可能伴随 U1/U2 电源管理失败）导致 I/O 错误。BTRFS 在执行 scrub 操作处理相关逻辑时，于 `btrfs_lookup_csums_range` 函数内调用 `kfree` 释放内存时访问了无效地址，触发内核分页错误。

#解决方案
暂无（Bug 状态为 NEW，维护者建议将问题发送至 linux-usb 邮件列表）。
