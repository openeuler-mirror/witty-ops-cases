#标题
snd_ctxfi 驱动初始化期间触发 CPU Stall 与 Soft Lockup

#内核版本
4.14-1~exp1, 4.13.4-2~bpo9+1, 4.12.13-1~bpo9+1, 4.9.30-2+deb9u5

#问题现象
- 系统启动期间或手动加载模块时出现 CPU stall 和 soft lockup 错误，导致系统卡死或响应极其缓慢。
- 日志显示 "INFO: rcu_sched self-detected stall on CPU" 和 "watchdog: BUG: soft lockup - CPU#X stuck for 23s!"。
- Call Trace 指向 snd_ctxfi 模块的 hw_read_pci、hw_card_init、ct_atc_create 等函数。
- 用户报告此问题在安装 Samsung 960 EVO NVMe SSD 后出现，且每隔数天发生一次。

#问题根因
snd_ctxfi 驱动在初始化过程中（具体位于 hw_read_pci 函数）长时间占用 CPU 或陷入等待，触发 soft lockup。尽管用户报告与 NVMe SSD 安装有关，但堆栈信息显示故障点位于声卡驱动加载流程。

#解决方案
暂无
