#标题
nvidia MCP51 控制器 sata_nv 驱动与新硬盘存在兼容性问题导致 IO 错误及系统崩溃

#内核版本
2.6.35 至 3.14（用户报告影响多个版本）

#问题现象
- 使用 nvidia MCP51 芯片组（如 MS-7346 主板）连接较新的硬盘（如 Seagate ST1000DM003、Samsung HM100UI、WD Red）时出现异常
- Seagate 硬盘：挂起过程耗时异常（30或60秒），恢复后磁盘无法访问、读取错误或系统挂起
- Samsung 硬盘：频繁报错 `ata3.00: exception Emask 0x0`、`failed command: SMART`、`error: { ICRC ABRT }`，偶现 `n_sectors mismatch`
- 系统偶发崩溃或交换分区读取错误

#问题根因
sata_nv 驱动程序与部分较新型号的硬盘在 MCP51 控制器上存在兼容性问题，导致 NCQ 或 SMART 命令处理失败，进而引发挂起恢复异常或 IO 错误。

#解决方案
暂无官方修复。尝试禁用 NCQ 可缓解部分硬盘的异常，但无法解决挂起恢复失败问题。
