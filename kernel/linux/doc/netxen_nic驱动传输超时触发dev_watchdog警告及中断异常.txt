#标题
netxen_nic驱动传输超时触发dev_watchdog警告及中断异常

#内核版本
6.6.8

#问题现象
- 内核日志报错 "__common_interrupt: 5.39 No irq handler for vector"
- 网卡 ens1f1 (netxen_nic) 传输队列 0 超时，触发 NETDEV WATCHDOG 警告："transmit queue 0 timed out 6882 ms"
- 触发内核 WARNING，调用栈指向 net/sched/sch_generic.c 中的 dev_watchdog 函数
- 网卡执行重置操作后链路恢复

#问题根因
netxen_nic 驱动程序的传输队列发生阻塞或无响应，触发内核网络设备看门狗机制 (dev_watchdog) 报警；同时伴随中断向量处理异常。具体驱动层面的故障根因暂未明确。

#解决方案
暂无
