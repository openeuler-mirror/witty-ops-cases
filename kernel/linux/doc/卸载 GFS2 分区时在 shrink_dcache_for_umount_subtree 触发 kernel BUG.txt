#标题
卸载 GFS2 分区时在 shrink_dcache_for_umount_subtree 触发 kernel BUG

#内核版本
3.11.8 custom build

#问题现象
- 双节点集群（DRBD+GFS2+corosync+pacemaker）在执行节点正常停止操作时失败。
- 内核日志报错 "kernel BUG at fs/dcache.c:917!"。
- 触发 invalid opcode (0000 [#1])，系统调用堆栈显示处于 umount 流程中。
- RIP 指向函数 shrink_dcache_for_umount_subtree+0x1de/0x1f0。

#问题根因
在执行 GFS2 分区卸载操作时，内核在 shrink_dcache_for_umount_subtree 函数中触发了 fs/dcache.c:917 处的 BUG() 宏，通常表明此时 dentry 缓存的状态不符合预期（例如引用计数异常），导致内核主动触发 panic 以防止数据损坏。

#解决方案
暂无
