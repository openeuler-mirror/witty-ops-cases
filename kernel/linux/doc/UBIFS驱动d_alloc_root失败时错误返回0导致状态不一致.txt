#标题
UBIFS驱动d_alloc_root失败时错误返回0导致状态不一致

#内核版本
2.6.39

#问题现象
函数 ubifs_fill_super() 在调用 d_alloc_root() 失败返回 NULL 时，错误地返回 0（即成功状态码），导致调用者误认为文件系统挂载成功，而实际上 super_block 的 s_root 为空，可能引发后续的空指针解引用或系统异常。

#问题根因
代码逻辑缺陷。在 fs/ubifs/super.c 的 ubifs_fill_super() 函数中，当 d_alloc_root() 返回 NULL 时，代码跳转至 out_iput 标签进行错误处理，但在跳转前未将变量 err 设置为负的错误码（如 -ENOMEM），导致函数最终保留了 err 的初始值 0 并返回。

#解决方案
暂无
