#标题
挂载损坏Btrfs镜像触发kernel BUG导致系统崩溃

#内核版本
5.17, 5.17-rc8, 5.16.8, 5.15.32

#问题现象
- 挂载并操作损坏的 btrfs 镜像后，执行 umount 操作时触发 kernel BUG，导致系统崩溃。
- 日志显示 "kernel BUG at fs/inode.c:611!"，伴随 "invalid opcode: 0000 [#1]" 错误。
- Call Trace 显示调用路径为：`clear_inode+0x8e/0xe0` <- `btrfs_evict_inode+0x415/0x770` <- `evict` <- `dispose_list` <- `evict_inodes`。
- 触发 BUG 前出现大量 "BTRFS error (device loop0): bad extent! em: [0 0] passed [0 0]" 错误日志。

#问题根因
系统处理损坏的 Btrfs 镜像时，文件系统元数据出现异常。在卸载过程中，`btrfs_evict_inode` 函数调用 `clear_inode` 时，因 inode 状态不符合预期（触发了 `fs/inode.c:611` 处的断言检查），导致内核执行 `BUG()` 停机。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）。
