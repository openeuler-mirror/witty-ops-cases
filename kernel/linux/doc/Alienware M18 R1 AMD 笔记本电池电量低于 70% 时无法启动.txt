#标题
Alienware M18 R1 AMD 笔记本电池电量低于 70% 时无法启动

#内核版本
6.3.8-200.fc38.x86_64

#问题现象
- 笔记本在电池供电且电量低于约 70% 时，无法完成启动过程，系统会在启动中途关机或重启。
- 使用 `acpi=off` 内核参数可以正常启动，但会导致许多设备无法正常工作。
- 正常启动（交流电供电）的 dmesg 日志中显示多处 ACPI BIOS Error，例如 `AE_NOT_FOUND` 和 `AE_ALREADY_EXISTS`。

#问题根因
未明确。dmesg 日志显示 BIOS 存在 ACPI 错误（如无法解析符号 `\_SB.PCI0.LPC0.EC0`），推测与 BIOS ACPI 实现有关，但因故障时日志未落盘，确切的崩溃触发点尚待确认。

#解决方案
暂无。建议启用内核配置 `CONFIG_PSTORE_RAM` 以在重启后通过 `/sys/fs/pstore/` 获取故障时的启动日志，以便进一步分析。
