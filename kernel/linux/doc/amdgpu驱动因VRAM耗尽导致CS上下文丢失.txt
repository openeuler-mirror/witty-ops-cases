#标题
amdgpu驱动因VRAM耗尽导致CS上下文丢失

#内核版本
4.9.118

#问题现象
- 应用程序持续运行一段时间后报错："amdgpu: The CS has been cancelled because the context is lost"
- dmesg 日志显示错误：`[drm:amdgpu_cs_ioctl [amdgpu]] *ERROR* Not enough memory for command submission!`
- 系统出现显示闪烁、触摸屏无响应
- 故障发生时系统物理内存和 Swap 使用率较低，但显存（VRAM）已耗尽

#问题根因
显存（VRAM）资源耗尽导致命令提交失败。开发者指出这是预期行为，当应用程序尝试使用超过物理可用的显存时会触发。根本原因可能是应用程序本身存在显存泄漏，导致显存占用持续增长直至耗尽。

#解决方案
- 规避方法：在内核启动参数中添加 `amdgpu.vramlimit=512` 限制可用显存大小。
- 建议措施：升级内核至 5.8 或更新版本进行验证；排查应用程序是否存在显存泄漏。
