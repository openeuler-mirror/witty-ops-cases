#标题
ath9k_htc驱动skb双重释放触发kernel BUG导致系统死机

#内核版本
5.6.0-1-default

#问题现象
- 使用 Atheros AR9271 WiFi 适配器进行高负载操作（如播放视频、下载大文件）约 10-15 分钟后系统死机。
- 故障发生时拔出 WiFi 适配器会导致系统彻底卡死，音频出现循环播放现象，必须强制重启。
- 内核日志显示 "kernel BUG at mm/slub.c:304!" 及 "invalid opcode: 0000 [#1]"。
- Call Trace 显示路径为：kfree -> kfree_skb -> ath9k_wmi_cmd [ath9k_htc] -> ath9k_reg_rmw [ath9k_htc]。

#问题根因
ath9k_htc 驱动中存在 skb 双重释放问题。在函数调用链中，当 hif_usb_send_regout 发送失败时，urb 回调函数 hif_usb_regout_cb 会释放 skb；然而，上层调用者 ath9k_wmi_cmd 随后再次调用 kfree_skb 试图释放同一个 skb，导致 kfree 检测到内存错误并触发 kernel BUG。

#解决方案
暂无（Bug 状态为 NEW，评论中未提及已合并的修复补丁）。
