#标题
ThinkPad P16s Gen4 AMD ALC257 声卡 SSID 读取错误导致扬声器初始化失败

#内核版本
6.18.13-200.fc43.x86_64

#问题现象
- 内部扬声器声音空洞、完全无低音，仅有一个高音单元工作。
- 第二个内部扬声器（woofer）未被内核初始化，`aplay -l` 中未显示相关声卡。
- 系统日志显示 `snd_hda_codec_alc269` 将 ALC257 的 Subsystem ID 错误读取为 `17aa:0000`，而非正确的 `17aa:512f`，导致未加载设备特定的 fixup。
- 关联机型（IdeaPad Slim 5 13ARP10）出现相同 SSID 不匹配现象，导致内置扬声器完全静音。

#问题根因
HDA codec Realtek ALC257 的 Subsystem ID (SSID) 被驱动错误读取为 `17aa:0000`，导致驱动无法匹配正确的设备配置（fixup），Pin 0x17（低音节点）被识别为 "Vendor Defined Widget" 而被忽略，无法正常初始化。关联问题中，Conexant SN6140 设备因 DAC stream 值为 0 导致扬声器静音。

#解决方案
暂无上游合并修复。开发者提供了测试补丁（221167-bass_speaker.patch 和 221167-sn6140_speaker.patch）供测试，尝试通过配置 GPIO 和 Pin 定义修复。临时变通方案包括：移除特定驱动模块（如 `snd-hda-codec-conexant.ko`）尝试使用通用驱动，或通过系统挂起/休眠触发状态更新（部分机型有效）。
