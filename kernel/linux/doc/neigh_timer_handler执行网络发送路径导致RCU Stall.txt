#标题
neigh_timer_handler执行网络发送路径导致RCU Stall

#内核版本
v6.0-rc4 (ce888220d5c7)

#问题现象
- 内核日志报告 "INFO: rcu_preempt detected stalls on CPUs/tasks"
- 日志提示 "rcu_preempt kthread starved for 173676 jiffies"
- CPU 1 的 Call Trace 显示中断上下文执行路径为：neigh_timer_handler -> neigh_probe -> arp_solicit -> arp_send_dst -> __dev_queue_xmit -> e1000_xmit_frame
- CPU 0 的 Call Trace 显示在 rcu_core -> _raw_spin_lock_irqsave 处发生自旋锁等待

#问题根因
在 neigh_timer_handler 定时器处理函数中，触发了邻居探测和 ARP 发送流程。该流程进入网络设备驱动 e1000_xmit_frame 进行发包，由于某种原因（如锁竞争或驱动处理耗时）导致该过程长时间无法返回，造成 RCU 优雅周期线程饥饿，进而触发 RCU Stall 检测告警。

#解决方案
暂无
