#标题
IOMMU解析RMRR时因BIOS固件缺陷触发内核WARNING

#内核版本
5.5.6

#问题现象
- 系统启动过程中内核触发 WARNING，调用栈指向 `dmar_parse_one_rmrr+0x8f/0x11b`。
- 内核日志打印固件错误信息："DMAR: [Firmware Bug]: No firmware reserved region can cover this RMRR [0x000000008b800000-0x000000008fffffff]"。
- 日志提示 "Your BIOS is broken; bad RMRR"。

#问题根因
BIOS 固件存在缺陷，提供的 RMRR（Reserved Memory Region Reporting）区域未被固件保留区域覆盖；内核在 `drivers/iommu/intel-iommu.c` 的 `dmar_parse_one_rmrr` 函数中检测到此不一致性，从而触发 WARNING。

#解决方案
升级内核至 5.5.10 或更高版本，该版本已解决此问题。
