#标题
启用Intel IOMMU导致DMAR故障日志刷屏及GPU挂起

#内核版本
4.20（回归引入版本），复现版本包括 5.1.16、5.2.9

#问题现象
- dmesg 被大量 DMAR 故障日志刷屏，报错信息如 "DMAR: DRHD: handling fault status reg 2" 或 "reg 3"。
- 日志显示设备 [00:02.0]（Intel 集成显卡）触发 DMA Write 故障，故障原因包括 "PTE Write access is not set"（原因 05）或 "Unknown"（原因 23）。
- 系统出现卡顿、变慢，使用视频播放器（如 VLC、Kodi）时尤为明显。
- 严重情况下触发 GPU HANG，导致 i915 驱动重置 GPU。

#问题根因
未明确具体代码缺陷。现象上表现为内核 4.20 引入的回归问题，在启用 intel_iommu=on 时，IOMMU 处理集成显卡 DMA 写请求出现异常，导致页表权限错误或未知故障。

#解决方案
暂无正式修复。可通过 GRUB 启动参数添加 `intel_iommu=igfx_off` 关闭集成显卡的 IOMMU 功能以规避该问题；或回退至 4.19 内核。
