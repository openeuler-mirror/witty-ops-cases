#标题
eCryptFS使用ecryptfs_xattr模式创建文件触发空指针崩溃

#内核版本
3.12.21

#问题现象
- 在挂载了ecryptfs_xattr的eCryptFS挂载点执行写文件操作（如`echo hello > foo`）时触发内核崩溃。
- 日志显示：`BUG: unable to handle kernel NULL pointer dereference at (null)`。
- RIP寄存器指向`fsstack_copy_attr_all+0x2/0x61`。

#问题根因
函数`fsstack_copy_attr_all`接收到空指针参数（RDI寄存器值为0）导致解引用失败。调用链显示：`ecryptfs_create` -> `ecryptfs_initialize_file` -> `ecryptfs_write_metadata` -> `ecryptfs_setxattr` -> `fsstack_copy_attr_all`。

#解决方案
开发者已确认该问题并建议提交新的Bug报告进行跟踪，暂无已合并的修复补丁。
