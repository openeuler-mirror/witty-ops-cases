#标题
TUN设备ioctl触发run_timer_softirq中Use-after-Free导致内核panic

#内核版本
4.14.0-rc4（受影响版本从 4.12 起）

#问题现象
- 系统执行针对 /dev/net/tun 的 ioctl 操作后触发 Kernel panic。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference" 或 "BUG: KASAN: use-after-free"。
- 崩溃发生在中断上下文，具体位置在 run_timer_softirq 函数。
- Call Trace: apic_timer_interrupt -> smp_apic_timer_interrupt -> irq_exit -> __do_softirq -> run_timer_softirq。

#问题根因
通过 ioctl (0x400454ca) 操作 TUN 设备时，触发了定时器软中断处理逻辑中的内存访问错误。KASAN 日志显示为 use-after-free，即 run_timer_softirq 访问了已释放的内存地址，导致空指针解引用或非法内存写入。

#解决方案
暂无
