#标题
GFS2文件系统glock_set_object函数中断言失败触发内核告警

#内核版本
v6.0-rc2 (包含部分 dlm 补丁)

#问题现象
系统运行测试用例时，内核日志打印警告信息：`warning: assertion "gl->gl_object == NULL" failed`。故障发生在 `glock_set_object` 函数中，调用栈显示由 `delete_work_func` 工作队列处理函数触发，经由 `gfs2_lookup_by_inum` 和 `gfs2_inode_lookup` 调用路径。

#问题根因
在 `fs/gfs2/glock.h` 的 `glock_set_object` 函数中，断言 `gl->gl_object == NULL` 检查失败。这表明在尝试设置 glock 对象关联时，该 glock 的 `gl_object` 字段非空，违反了预期的空值前提，可能涉及对象生命周期管理或并发处理逻辑错误。

#解决方案
暂无（Bug 状态为 NEW，尚未提交修复补丁）
