#标题
断开WD My Passport移动硬盘USB连接时触发空指针解引用导致Kernel Oops

#内核版本
3.18.2, 3.18.3, 3.18.5

#问题现象
- 连接西部数据 My Passport 1TB 移动硬盘后断开 USB 连接，系统 100% 触发 Kernel Oops。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 00000000000001a0"。
- 故障发生在 `blk_post_runtime_resume` 函数中，错误码为 Oops: 0002。
- 即使硬盘未挂载，仅插拔 USB 线缆也能复现该问题。

#问题根因
在 USB 设备断开连接过程中，内核块层运行时电源管理函数 `blk_post_runtime_resume` 试图访问一个无效的内存地址（基于 NULL 指针偏移 0x1a0），导致空指针解引用。这可能与 USB 存储设备断开时设备结构体已释放但电源管理回调仍在执行有关。

#解决方案
暂无（Bug 处于 NEW 状态，维护者建议将问题发送至 linux-usb@vger.kernel.org 邮件列表）。
