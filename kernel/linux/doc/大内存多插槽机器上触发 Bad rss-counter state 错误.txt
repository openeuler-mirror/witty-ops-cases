#标题
大内存多插槽机器上触发 Bad rss-counter state 错误

#内核版本
4.9.24（报告中提及测试过 4.11.6-coreos-r1）

#问题现象
- 内核日志报错："../source/mm/pgtable-generic.c:33: bad pmd ffff8bb32d6d6200(0000000fcda001e0)"
- 内核日志报错："BUG: Bad rss-counter state mm:ffff8bbe9ba3dc00 idx:1 val:512"
- 内核日志报错："BUG: non-zero nr_ptes on freeing mm: 1"
- 该问题规律性出现在 AWS x1.32xlarge 实例（4 NUMA 节点，约 2TiB 内存）上，运行 CoreOS 及 Spark/Druid 工作负载。

#问题根因
未明确。开发者提示该错误信息在社区已有相关修复案例，可能因使用的内核版本较旧，缺少相关补丁导致。

#解决方案
暂无明确修复方案。开发者建议搜索 "Bad rss-counter state" 相关修复，确认当前内核版本已合入相关补丁。
