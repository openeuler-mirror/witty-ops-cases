#标题
VIA VL805 USB 3.0 控制器超时导致设备复位无法使用

#内核版本
4.6.x (dmesg 显示为 4.6.2-1-ARCH)

#问题现象
- 连接 USB 3.0 设备至 VIA VL805 控制器的后置接口约 1-1.5 分钟后，设备自动复位并断开连接。
- dmesg 报错显示 "Timeout while waiting for setup device command" 及 "device not accepting address 2, error -62"。
- 设备从 lsusb 中消失，必须重新插拔才能暂时恢复，但故障会再次复现。
- 同一设备连接至前置 USB 3.0 接口（Intel 芯片组控制）工作正常。

#问题根因
未明确。报告者推测与内核对 VIA VL805 控制器的驱动支持有关，但上游未给出具体代码层面的根因分析。

#解决方案
暂无
