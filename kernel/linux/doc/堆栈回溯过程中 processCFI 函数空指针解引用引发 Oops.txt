#标题
堆栈回溯过程中 processCFI 函数空指针解引用引发 Oops

#内核版本
2.6.32.45-0.3.2.4445.1.PTF-default

#问题现象
- 执行 `tc qdisc` 命令模拟网络延迟和丢包后，内核持续打印 WARNING 信息，指向 `skb_gso_segment` 函数。
- 日志显示 `DWARF2 unwinder stuck at call_softirq`，表明堆栈展开过程出现异常。
- 系统在打印堆栈信息时触发 Kernel Oops，报错：`BUG: unable to handle kernel NULL pointer dereference at 000000000000008b`。
- 崩溃发生在 `processCFI+0xa0/0x630`，Call Trace 显示调用链为：`warn_slowpath_common` -> `dump_stack` -> `dump_trace` -> `unwind` -> `processCFI`。

#问题根因
内核在处理网络 GSO 分片相关的 WARNING 并尝试打印堆栈回溯时，堆栈展开器中的 `processCFI` 函数访问了无效的内存地址（`RDI` 寄存器值为 `0x8b`），导致空指针解引用并触发 Oops。该问题发生在堆栈解析逻辑中，可能与堆栈数据损坏或特定条件下的 CFI 解析异常有关。

#解决方案
暂无
