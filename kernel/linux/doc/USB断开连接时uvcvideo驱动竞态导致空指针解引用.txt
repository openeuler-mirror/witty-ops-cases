#标题
USB断开连接时uvcvideo驱动竞态导致空指针解引用

#内核版本
5.19.10

#问题现象
- 内核日志报错：BUG: kernel NULL pointer dereference, address: 0000000000000000
- 触发 Oops，RIP 指向 usb_ifnum_to_if+0x34/0x60
- Call Trace 显示调用链为：usb_hcd_alloc_bandwidth -> usb_set_interface -> uvc_video_start_transfer -> uvc_video_start_streaming
- 故障在使用 Logitech BRIO 摄像头时触发，伴随 USB 设备断开/重连事件

#问题根因
根据开发者分析，这是 uvcvideo 驱动程序中的竞态条件（race condition）。问题发生在 USB 设备断开连接与视频流启动过程之间，导致 usb_hcd_alloc_bandwidth 函数调用 usb_ifnum_to_if 时访问了无效指针。

#解决方案
暂无正式修复补丁。开发者提供了诊断补丁用于追踪竞态细节；建议检查 USB 线缆稳定性以规避触发条件。
