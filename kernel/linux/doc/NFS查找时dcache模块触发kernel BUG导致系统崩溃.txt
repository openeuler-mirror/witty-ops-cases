#标题
NFS查找时dcache模块触发kernel BUG导致系统崩溃

#内核版本
3.12.67-64-desktop

#问题现象
- 系统在升级至 3.12.62-55 之后的内核版本运行数小时或数天后发生 Kernel Crash。
- 日志显示 "kernel BUG at ../fs/dcache.c:268!" 及 "invalid opcode: 0000 [#1]"。
- 故障现场 RIP 指向函数 `dentry_rcuwalk_barrier`，调用栈显示关联 `nfs_lookup` 操作。

#问题根因
内核在 `fs/dcache.c` 文件的第 268 行触发 `BUG()` 宏（断言失败），导致 invalid opcode 异常。故障路径涉及 `nfs_lookup` -> `d_materialise_unique` -> `__d_drop` -> `dentry_rcuwalk_barrier`，表明在处理 NFS 目录项缓存时遇到了非法状态。

#解决方案
暂无
