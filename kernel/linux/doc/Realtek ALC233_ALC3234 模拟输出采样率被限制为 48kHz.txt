#标题
Realtek ALC233/ALC3234 模拟输出采样率被限制为 48kHz

#内核版本
5.13.1

#问题现象
- 无法播放采样率高于 48kHz 的音频文件，ALSA 报错 "snd_pcm_hw_params_set_rate failed: Invalid argument"
- PulseAudio 强制将音频重采样至 48kHz
- 特殊触发条件：系统全新安装后高采样率（96kHz/192kHz）功能正常，但在首次关机后失效；重启不会触发问题，仅关机后的冷启动会触发
- Windows 系统下功能正常

#问题根因
编解码器（如 ALC233/ALC3234）在冷启动（关机后启动）后的硬件状态与热启动不同。Linux ALSA 驱动未能正确初始化编解码器的系数，导致硬件无法切换至高采样率模式；而 Windows 驱动程序能够正确初始化这些系数。

#解决方案
暂无官方补丁。变通方案：通过对比 Windows 驱动加载前后的 codec coefficients 差异，使用 `hda_verb` 工具手动修改编解码器系数以解锁高采样率支持。
