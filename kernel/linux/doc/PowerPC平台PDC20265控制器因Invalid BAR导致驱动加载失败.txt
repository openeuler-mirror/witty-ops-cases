#标题
PowerPC平台PDC20265控制器因Invalid BAR导致驱动加载失败

#内核版本
2.6.32 - 4.11（回归，2.6.22及之前版本正常）

#问题现象
- PDC20265 PATA控制器在内核版本升级后无法正常工作，驱动 `pata_pdc202xx_old` 加载失败。
- 系统日志报错：`pci 0008:00:04.0: [Firmware Bug]: reg 0x10: invalid BAR (can't size)`。
- 驱动探测失败提示：`can't enable device: BAR 0 [io size 0x0008] not assigned`，返回错误码 -22。
- `lspci` 显示该设备的 I/O 端口区域（Region 0-3）地址为 0000 或未分配，而在正常工作的旧内核中显示为有效地址。

#问题根因
内核 PCI 子系统在处理该设备（Promise PDC20265）的 legacy IDE I/O 端口资源时，检测到无效的 BAR 配置并标记为 Firmware Bug。尽管内核尝试应用 legacy IDE quirk（如将 reg 0x10 映射到 0x01f0），但最终未能成功分配所需的 I/O 资源（BAR 0 未分配），导致驱动无法启用设备。该问题可能是内核 PCI 资源分配逻辑变更与特定硬件平台（PPC405GP DHT-Walnut）固件或设备树配置不兼容导致的回归。

#解决方案
暂无修复方案。开发者建议向 linux-pci@vger.kernel.org 邮件列表报告此问题以寻求进一步分析。
