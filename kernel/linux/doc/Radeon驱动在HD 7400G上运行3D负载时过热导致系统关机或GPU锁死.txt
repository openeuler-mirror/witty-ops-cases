#标题
Radeon驱动在HD 7400G上运行3D负载时过热导致系统关机或GPU锁死

#内核版本
4.2（评论提及 4.6 仍受影响）

#问题现象
- 运行 3D 游戏或 VDPAU 视频播放等图形密集型任务时，系统温度急剧上升，最高可达 110°C。
- 系统因过热触发强制关机，或在未达临界温度前发生 GPU lockup 导致系统挂起。
- 日志中出现 "GPU lockup (current fence id ... on ring 0)" 错误信息。
- 使用 fglrx 闭源驱动或 Windows 驱动时无此现象。

#问题根因
开源 radeon 驱动在特定 APU（HD 7400G / A4-4355M）上的电源管理策略（涉及 DPM 和 BAPM）存在缺陷，导致在 3D 高负载下功耗与散热控制失效，进而引发过热保护或 GPU 硬件锁死。

#解决方案
暂无官方修复。可尝试以下规避措施：
1. 在启动时将 DPM profile 切换为 battery 或 low power 模式以限制功耗。
2. 尝试添加内核参数 `radeon.runpm=0` 禁用运行时电源管理（效果因环境而异，可能降低性能）。
3. 添加内核参数 `nomodeset radeon.modeset=1` 可避免过热，但会禁用加速功能。
