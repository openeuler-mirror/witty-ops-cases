#标题
Seagate Archive SMR 硬盘写入大量数据时触发 NCQ 超时导致 I/O 错误

#内核版本
3.19（受影响版本包括 3.19 至 4.3）

#问题现象
- 向 Seagate Archive 8TB SMR 硬盘写入大量数据（通常超过 3GB）时，系统日志报错 WRITE FPDMA QUEUED 超时。
- dmesg 显示 "ataX.00: exception Emask 0x0 SAct ... action 0x6 frozen" 及 "Emask 0x4 (timeout)"。
- NCQ 可能被内核禁用，文件系统可能变为只读模式，严重时硬盘无响应直至断电重启。
- 该问题为内核回归问题，3.18 版本正常，3.19 及之后版本（包括 4.0, 4.1, 4.2）出现故障。

#问题根因
内核对 SMR（叠瓦式磁记录）硬盘的支持存在缺陷。引入 ZAC (Zoned Device Characteristics) 支持的相关提交（如 9162c6579bf9、f9ca5ab832e7）以及修复 I/O 大小的提交 4f258a46346c ("sd: Fix maximum I/O size for BLOCK_PC requests") 改变了 I/O 行为。SMR 硬盘在进行内部垃圾回收时响应时间较长，新的内核逻辑未能正确处理这种延迟，导致命令超时并触发错误处理流程。

#解决方案
上游修复：应用 Martin K. Petersen 提供的补丁（Commit ca369d51b3e1649be4a72addd6d6a168cfb3f537），该修复已合入 4.4 内核。
临时规避：降级内核至 3.18 版本，或回退相关问题提交（如 4f258a46346c）。
