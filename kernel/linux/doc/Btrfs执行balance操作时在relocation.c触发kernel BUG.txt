#标题
Btrfs执行balance操作时在relocation.c触发kernel BUG

#内核版本
5.4.0-45-generic

#问题现象
- 执行 `btrfs balance start` 命令时，命令行返回 Segmentation fault，系统日志记录 kernel BUG。
- dmesg 显示 "kernel BUG at fs/btrfs/relocation.c:437!" 及 "invalid opcode: 0000 [#1]"。
- 日志中伴随 BTRFS 错误信息："couldn't find block ... in tree"。
- 故障可稳定复现。

#问题根因
问题发生在 Btrfs 文件系统 relocation 处理流程中。调用栈显示崩溃点位于 `remove_backref_node` 函数（`fs/btrfs/relocation.c:437`），由 `build_backref_tree` 调用链触发。日志提示无法在树中找到特定块，导致内核触发 BUG() 终止运行。

#解决方案
暂无官方修复。有用户尝试使用最新版 btrfs-progs 执行离线 `btrfs check -r` 修复，但未能解决该问题。
