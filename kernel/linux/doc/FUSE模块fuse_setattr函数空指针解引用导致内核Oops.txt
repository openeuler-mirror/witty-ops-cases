#标题
FUSE模块fuse_setattr函数空指针解引用导致内核Oops

#内核版本
4.9-rc1

#问题现象
- 插入 NTFS-3g USB 设备时系统触发内核 Oops。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at (null)"。
- 异常指令指针位于 `fuse_setattr+0x113/0x150`。
- 调用栈显示路径为 `SyS_chmod` -> `SyS_fchmodat` -> `chmod_common` -> `notify_change` -> `fuse_setattr`。

#问题根因
在 `fuse_setattr` 函数内部访问了空指针，导致内核崩溃。该问题在执行 chmod 相关操作修改文件属性时触发。

#解决方案
开发者已提供修复补丁并推送至 git://git.kernel.org/pub/scm/linux/kernel/git/mszeredi/fuse.git#for-linus 分支，经测试验证问题已解决。
