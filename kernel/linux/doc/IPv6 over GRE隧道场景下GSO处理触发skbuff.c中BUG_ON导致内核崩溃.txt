#标题
IPv6 over GRE隧道场景下GSO处理触发skbuff.c中BUG_ON导致内核崩溃

#内核版本
4.2.0-19-generic

#问题现象
- 系统频繁发生内核崩溃，日志显示 "kernel BUG at .../net/core/skbuff.c:2097!"。
- 触发 "invalid opcode: 0000 [#1]" 异常，RIP 指向 `__skb_checksum` 函数。
- 调用栈显示崩溃路径为：`udp6_ufo_fragment` -> `ipv6_gso_segment` -> `gre_gso_segment` -> `skb_checksum` -> `__skb_checksum`。
- 故障场景涉及 KVM 虚拟机、网桥配置以及 GRE 隧道传输 IPv6 数据包。

#问题根因
在 IPv6 数据包通过 GRE 隧道进行 GSO（Generic Segmentation Offload）分段处理时，`__skb_checksum` 函数内部逻辑异常。在函数执行结束前，变量 `len`（剩余待校验和长度）未按预期归零，导致 `skbuff.c` 第 2097 行的 `BUG_ON(len)` 断言失败，进而触发 kernel panic。

#解决方案
暂无官方修复方案。建议升级至 Kernel 4.5 或更新版本验证是否已修复，或根据社区建议尝试针对 `e1000e_poll` 的调试补丁以进一步定位问题。
