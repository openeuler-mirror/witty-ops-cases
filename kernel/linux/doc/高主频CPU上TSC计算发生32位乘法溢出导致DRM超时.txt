#标题
高主频CPU上TSC计算发生32位乘法溢出导致DRM超时

#内核版本
主线内核（commit 4763f03d3d18 引入）

#问题现象
- 当 CPU 主频超过 4.2GHz 时触发
- DRM 子系统在尝试配置 DisplayPort 显示器时出现严重的超时问题

#问题根因
在 commit 4763f03d3d18 ("x86/tsc: Use TSC as sched clock early") 引入的 TSC 清理代码中，tsc/jiffie 计算存在 32 位乘法溢出问题。当 CPU 主频超过 4.2GHz 时，计算结果超出 32 位整数范围，导致时间计算错误。

#解决方案
提交了修复补丁（见 Bug 附件 278377），修正了 32 位乘法溢出问题。
