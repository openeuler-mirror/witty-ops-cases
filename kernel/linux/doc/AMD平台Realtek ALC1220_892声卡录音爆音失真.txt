#标题
AMD平台Realtek ALC1220/892声卡录音爆音失真

#内核版本
4.11.0-rc5（问题持续存在于 4.12 至 5.1.x 版本）

#问题现象
- 系统音频播放基本正常，但录音（麦克风输入）出现严重的爆音、静电噪音或机器人般的失真。
- 使用 Audacity 等软件录音时可能出现音调变高、速度变快的问题。
- 问题出现在 AMD 芯片组（X370, X470, B450）搭配 Realtek 编解码器（ALC1220, ALC892）的硬件上。
- 调整 `position_fix`、`power_save`、`enable_msi` 等内核参数无效；PulseAudio 设置 `tsched=0` 仅能缓解部分症状。

#问题根因
怀疑是 AMD HDA 主机控制器（PCI ID: 1022:1457）与 Realtek 编解码器交互存在兼容性问题。由于 AMD 未提供 2015 年之后芯片组的详细文档，内核驱动无法正确初始化硬件状态。对比 Windows 驱动行为，Linux 下初始化的 Coeff（系数）值可能不正确，导致音频处理异常。

#解决方案
暂无官方内核补丁修复。
临时规避方案：
1. 先启动 Windows 系统再重启进入 Linux（热启动保留硬件初始化状态）。
2. 使用 USB 外置声卡替代板载声卡。
3. 针对特定主板（如 ASROCK X370），可通过脚本在启动时使用 `hda-verb` 工具手动写入正确的 Coeff 值（需对比 Windows 下的数值）。
