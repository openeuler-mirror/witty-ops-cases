#标题
调用 sys_perf_event_open 触发 ftrace 代码修改导致 soft lockup 及内核 panic

#内核版本
v6.3-rc2

#问题现象
- 内核日志报错：`watchdog: BUG: soft lockup - CPU#0 stuck for 26s!`
- 系统触发 kernel panic：`Kernel panic - not syncing: softlockup: hung tasks`
- Call Trace 显示进程在执行 `sys_perf_event_open` 系统调用时，经过 `perf_ftrace_event_register` -> `ftrace_startup` -> `text_poke` 路径，最终在 `__text_poke` 函数处卡死。

#问题根因
经二分测试（bisect），怀疑引入问题的提交为 `c597bfddc9e9e8a63817252b67c3ca0e544ace26`（sched: Provide Kconfig support for default dynamic preempt mode）。报告者指出该提交仅为疑似原因，因回退该提交后内核构建失败，未能完全确认根因。

#解决方案
暂无
