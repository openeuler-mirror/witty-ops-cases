#标题
ARM平台python进程触发Bad page map导致内核Oops

#内核版本
3.6.9

#问题现象
- 内核日志报错 "Unable to handle kernel paging request at virtual address 54066094"
- 触发 "BUG: Bad page map in process python pte:ffffffff pmd:1aea9b03"
- 调用栈显示：`print_bad_pte` -> `vm_normal_page` -> `do_wp_page` -> `handle_pte_fault`
- 系统服务停止响应，需重启恢复

#问题根因
故障发生在处理缺页中断（handle_pte_fault）的写时复制流程（do_wp_page）中，检测到页表项（PTE）异常（pte:ffffffff）。具体根因未明确，报告者推测可能与 USB 子系统（cdc_acm 驱动）内存踩踏有关。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
