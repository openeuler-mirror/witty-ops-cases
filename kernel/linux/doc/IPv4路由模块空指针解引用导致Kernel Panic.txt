#标题
IPv4路由模块空指针解引用导致Kernel Panic

#内核版本
3.4.0-liberty

#问题现象
- 系统在执行 ping 命令时触发 Kernel Panic（Oops）
- 日志报错：Unable to handle kernel NULL pointer dereference at virtual address 000000c8
- PC 指针位于 `__ip_route_output_key+0x49c/0x798`，Call Trace 显示调用链为 `__ip_route_output_key` -> `ip_route_output_flow` -> `ping_v4_sendmsg`
- 故障发生在 ARM 32 位架构，复现概率较低

#问题根因
在 `net/ipv4/route.c` 文件的 `__ip_route_output_key` 函数中，变量 `dev_out` 通过 `FIB_RES_DEV(res)` 获取，但在特定条件下返回了 NULL。后续代码直接访问 `dev_out->ifindex` 进行赋值操作时，触发了空指针解引用，导致系统崩溃。

#解决方案
暂无。该 Bug 当前状态为 NEW，建议尝试在最新内核版本上复现以确认是否已修复。
