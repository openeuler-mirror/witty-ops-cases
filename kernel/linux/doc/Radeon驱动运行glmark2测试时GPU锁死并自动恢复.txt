#标题
Radeon驱动运行glmark2测试时GPU锁死并自动恢复

#内核版本
4.4.0, 4.5.3, 4.6

#问题现象
- 运行 glmark2 特定测试用例（如 loop 相关参数）时，GPU 停止响应数秒，屏幕黑屏，随后自动恢复正常。
- 内核日志显示 "ring 0 stalled for more than 10070msec"。
- 触发 GPU lockup 并执行 GPU softreset，重置成功后系统恢复。

#问题根因
未明确。日志显示 radeon 驱动 ring 0 停顿导致 GPU 锁死，具体原因不明。用户反馈在绘制函数中增加延迟可规避，推测可能与指令提交速率或时序有关。

#解决方案
暂无官方修复。用户发现在问题场景的绘制函数中增加延迟可规避该问题。
