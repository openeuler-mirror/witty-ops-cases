#标题
sky2驱动处理PCI硬件错误时触发空指针解引用导致Oops

#内核版本
2.6.32-431.el6.x86_64

#问题现象
- 系统随机出现冻结现象，无法响应。
- 内核日志报错：`BUG: unable to handle kernel NULL pointer dereference at 00000000000006d5`。
- 触发 Kernel Oops，Call Trace 指向 `sky2_mac_intr+0x44/0x110 [sky2]`。
- 日志显示 sky2 设备报错：`error interrupt status=0xffffffff`，伴随 PCI hardware error、PCI Express error 以及多种奇偶校验错误。

#问题根因
硬件层面出现故障（可能是设备从总线掉线或硬件损坏），导致驱动读取到异常状态值 `0xffffffff`。驱动程序 `sky2_mac_intr` 函数在处理此类异常中断状态时，未进行有效性检查，直接访问了无效地址，触发空指针解引用。

#解决方案
暂无（Bug处于NEEDINFO状态，报告者怀疑硬件故障，开发者确认驱动不应崩溃但未提供修复补丁）。
