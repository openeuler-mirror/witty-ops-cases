#标题
禁用WiFi硬件开关触发SCSI设备移除时sysfs警告

#内核版本
3.13.6-1-ARCH

#问题现象
- 使用无线硬件开关禁用 WiFi 时，外部 USB HDD 意外断开连接并被卸载
- 内核日志中触发 WARNING，报错信息为 "sysfs group ffffffff81895c40 not found for kobject 'target6:0:0'"
- 堆栈跟踪显示调用链为 sysfs_remove_group -> dpm_sysfs_remove -> device_del -> scsi_target_reap_usercontext

#问题根因
SCSI 子系统在处理设备移除（scsi_target_reap_usercontext）过程中，调用 sysfs_remove_group 时无法找到对应的 kobject，导致触发警告。开发者确认该 sysfs 相关堆栈跟踪为已知 bug。

#解决方案
该问题已在内核 3.14 版本中修复，建议升级内核版本。
