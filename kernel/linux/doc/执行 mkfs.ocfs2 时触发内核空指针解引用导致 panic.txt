#标题
执行 mkfs.ocfs2 时触发内核空指针解引用导致 panic

#内核版本
3.14.0-rc6

#问题现象
- 系统触发 kernel panic，日志显示 "BUG: unable to handle kernel NULL pointer dereference at (null)"
- 故障入口位于 strlcpy 函数
- 调用栈显示：strlcpy -> ocfs2_cluster_connect -> ocfs2_cluster_connect_agnostic -> user_dlm_register -> dlmfs_mkdir

#问题根因
由 commit c74a3bdd9b529d924d1abf986079b783dd105ace 引入缺陷。该提交导致在 ocfs2_cluster_connect 相关代码路径中访问了空指针。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
