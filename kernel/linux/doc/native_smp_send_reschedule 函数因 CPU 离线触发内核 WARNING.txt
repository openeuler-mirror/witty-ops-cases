#标题
native_smp_send_reschedule 函数因 CPU 离线触发内核 WARNING

#内核版本
3.11.1（报告版本），受影响版本包括 3.12.0-rc1, 3.13.4, 3.18.4, 3.19-rc6, 4.1

#问题现象
- 内核启动过程中或连接外部 USB 3.0 硬盘时触发 WARNING，可能导致系统崩溃。
- 日志显示：`WARNING: CPU:0 PID 1 at arch/x86/kernel/smp.c:124 native_smp_send_reschedule+0x4b/0x50()`。

#问题根因
函数 `native_smp_send_reschedule` 在发送调度中断前检查目标 CPU 状态，发现目标 CPU 处于 offline 状态（`cpu_is_offline(cpu)` 为真），从而触发 `WARN_ON(1)`。该问题可能与开启 IOMMU Hardware Support 及 Interrupt Remapping 功能有关，导致 CPU 状态管理或中断路由异常。

#解决方案
暂无官方修复补丁。
规避方法：
1. 在内核配置中禁用 IOMMU Hardware Support 相关选项（如 Support for Intel IOMMU using DMA Remapping Devices、Support for Interrupt Remapping）。
2. 启动时断开外部 USB 3.0 硬盘。
