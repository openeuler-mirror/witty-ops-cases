#标题
XFS内存不足时iozone测试触发内存分配死锁

#内核版本
Linux 5.4

#问题现象
在内存不足的场景下运行 iozone 测试 XFS 文件系统时，iozone 进程发生阻塞。系统日志打印 "XFS: iozone possible memory allocation deadlock"。

#问题根因
报告者分析认为，`kmem_alloc()`、`xfs_buf_allocate_memory()`、`kmem_zone_alloc()` 和 `kmem_realloc()` 等函数内部使用 "while" 循环在内存不足时持续尝试分配，导致死锁。开发者指出，部分分配路径在调用者传递 KM_MAYFAIL 标志时会跳出循环，具体根因需结合完整的堆栈信息进一步分析。

#解决方案
暂无。开发者建议提供详细的测试环境、内存条件及完整的日志输出和堆栈跟踪，以确定是否为可修复的代码问题或资源瓶颈。
