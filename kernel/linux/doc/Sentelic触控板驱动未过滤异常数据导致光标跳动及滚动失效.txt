#标题
Sentelic触控板驱动未过滤异常数据导致光标跳动及滚动失效

#内核版本
3.2.0-24-generic, 3.4-rc4 (修复版本: 3.6-rc7, 3.8)

#问题现象
- 使用 Sentelic FingerSensingPad 触控板时，光标频繁意外跳至屏幕左上角 (0,0)。
- 边缘滚动功能失效，双指滚动时页面会突然跳至顶部或底部，或在抬起手指时发生反向滚动。
- 触控板中间特定区域触摸时必定触发光标飞至左上角。
- 开启调试后，dmesg 日志显示数据包中包含异常的绝对坐标 (0, 0) 或坐标在两组数值间快速交替跳变。

#问题根因
触控板硬件在手指抬起或多指操作过渡期间会发送异常数据包（例如 packet[0] 为 0x48 或 0x49 且后续数据为 0）。驱动程序 `drivers/input/mouse/sentelic.c` 未对这些异常数据进行过滤，直接将其作为有效坐标上报给输入子系统，导致光标位置突变和滚动行为异常。

#解决方案
- 临时规避：在 GRUB 启动参数中添加 `psmouse.proto=imps`，将触控板识别为普通 ImPS/2 鼠标（此方法会禁用多点触控功能）。
- 正式修复：该问题通过补丁 "Input: sentelic - filter out erratic movement when lifting finger" 修复，已合并至 Linux 3.6-rc7 (Commit 30ebb7fa0e3e92145b859ad6e44aa6dc636b4103)。后续针对滚动问题在 Linux 3.8 中合入了 Commit a25461659050b913e114d282bf58823682eb56b6。
