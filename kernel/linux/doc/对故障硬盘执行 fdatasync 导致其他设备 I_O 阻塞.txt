#标题
对故障硬盘执行 fdatasync 导致其他设备 I/O 阻塞

#内核版本
6.1.53

#问题现象
- 用户在故障硬盘上运行 nwipe 并执行 fdatasync 时，连接同一 AHCI SATA 适配器的其他正常硬盘上的文件系统操作出现严重延迟。
- SysRq 日志显示多个进程处于 D 状态：
  - nwipe 进程阻塞在 `blkdev_fsync` -> `file_write_and_wait_range` -> `io_schedule`。
  - kworker (writeback) 阻塞在 `blk_mq_get_tag` -> `io_schedule`。
  - rm 进程阻塞在 `ext4_read_bh` -> `io_schedule`。
- 停止 nwipe 后性能恢复正常。

#问题根因
未明确（用户怀疑与 fdatasync 实现机制或 bfs I/O 调度器有关，但尚无定论）

#解决方案
暂无（用户反馈升级至内核 6.6.2 后问题未复现，但未确认是否彻底修复）
