#标题
RTL8192CE驱动导致CPU Soft Lockup

#内核版本
5.6.15

#问题现象
- 系统随机出现卡顿或假死，持续时间约22秒。
- 内核日志报错：`BUG: soft lockup - CPU#1 stuck for 22s! [swapper/1:0]`。
- 故障发生时 Call Trace 指向 `ipt_do_table`、`rtl_pci`、`rtl8192ce` 及 `mac80211` 模块。
- 问题常在 WiFi 开启、关闭或网络连接过程中触发。

#问题根因
未明确。根据日志分析，故障与 RTL8192CE 无线网卡驱动（`rtl8192ce`）相关。驱动在中断处理或软中断（softirq）上下文中执行时间过长（如 `pci_read32_sync` 或 `ipt_do_table` 处理路径），导致 CPU 无法调度其他任务，触发 watchdog 报告 soft lockup。

#解决方案
- 临时规避：加载驱动模块时添加参数 `rtl8192ce ips=0 fwlps=0 swenc=1` 关闭电源管理功能，可降低故障频率。
- 临时规避：添加内核启动参数 `rfkill.default_state=1`。
- 升级方案：用户反馈升级至内核 6.2.0 或 6.5.0 版本后问题不再复现。
