#标题
ext4处理特制镜像时bdev空指针导致Kernel Panic

#内核版本
5.0.0

#问题现象
- 在LKL（Linux Kernel Library）环境中挂载特制的ext4镜像并运行测试程序时触发故障。
- 内核日志输出 "BUG: failure at fs/buffer.c:195/__find_get_block_slow()!"，随后引发 Kernel Panic。
- 用户态程序收到 Segmentation fault 信号退出。
- 调用栈显示路径为：ext4_evict_inode -> ext4_truncate -> ext4_ind_truncate -> ext4_free_branches -> ext4_mark_iloc_dirty -> __find_get_block_slow。

#问题根因
在 `__find_get_block_slow` 函数执行时，传入的 `bdev` 参数为 NULL。用户通过在 `fs/buffer.c:195` 处添加 `BUG_ON(bdev == NULL)` 验证了该空指针访问是导致 BUG 的直接原因。该问题在文件系统存在历史错误且未修复的情况下被触发。

#解决方案
暂无
