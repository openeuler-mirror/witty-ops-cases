#标题
NBD连接断开后ext4文件系统操作触发kernel BUG at fs/buffer.c:3006

#内核版本
3.19-rc7

#问题现象
- 使用 qemu-nbd 挂载 ext4 镜像，在杀掉 qemu-nbd 进程后执行目录遍历操作（如 ls）触发 kernel BUG。
- 内核日志打印：`kernel BUG at fs/buffer.c:3006!` 及 `invalid opcode: 0000 [#1]`。
- Call Trace 显示调用链为：`_submit_bh` -> `__sync_dirty_buffer` -> `sync_dirty_buffer` -> `ext4_commit_super` -> `__ext4_error_inode` -> `__ext4_get_inode_loc`。

#问题根因
NBD 进程被杀导致 socket 关闭，块设备 I/O 请求失败。Ext4 文件系统在读取 inode 表失败时触发错误处理流程 `__ext4_error_inode`，该流程尝试调用 `ext4_commit_super` 更新超级块。在向已失效的设备提交脏缓冲区（`_submit_bh`）时，触发了 `fs/buffer.c:3006` 处的 `BUG()`。
