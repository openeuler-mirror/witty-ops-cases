#标题
qla2xxx驱动固件无响应导致EEH错误及系统死锁

#内核版本
5.2-rc1 至 5.3-rc8

#问题现象
- 系统启动完成后不久，QLogic HBA 固件停止响应 mailbox commands
- 在 pseries 平台上触发 EEH（Enhanced Error Handling）错误
- 由于固件不可用，驱动无法正确处理 EEH，导致主机内核发生死锁

#问题根因
由提交 a28d9e4ef997 ("scsi: qla2xxx: Add support for multiple fwdump templates/segments") 引入的回归，该提交导致固件不可用或在 POWER8 平台上触发 EEH 及系统锁定。

#解决方案
该问题已在 linux-next 中通过提交 edbd56472a63 ("scsi: qla2xxx: qla2x00_alloc_fw_dump: set ha->eft") 修复。临时规避措施为在客户机中禁用 EEH 以防止主机内核死锁。
