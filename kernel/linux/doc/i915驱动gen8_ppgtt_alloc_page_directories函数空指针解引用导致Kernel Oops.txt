#标题
i915驱动gen8_ppgtt_alloc_page_directories函数空指针解引用导致Kernel Oops

#内核版本
4.10.13

#问题现象
- 系统触发 Kernel Oops，日志显示 "unable to handle kernel NULL pointer dereference at 0000000000000018"
- 故障发生在 Dell Precision 5510 硬件平台，运行 X 服务时
- RIP 指向 i915 驱动的 gen8_ppgtt_alloc_page_directories.isra.36+0x119/0x260

#问题根因
i915 驱动在执行 `i915_gem_execbuffer2` 流程中，调用 `gen8_ppgtt_alloc_page_directories` 函数进行页目录分配时，访问了非法内存地址（空指针偏移 0x18），导致内核崩溃。
