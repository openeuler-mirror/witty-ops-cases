#标题
Radeon驱动VM处理大缓冲区时触发kernel BUG

#内核版本
3.13.5

#问题现象
- 系统日志报错：`kernel BUG at drivers/gpu/drm/radeon/radeon_sa.c:322!`
- 触发 invalid opcode (0000 [#1])，导致进程崩溃。
- 调用栈显示：`radeon_sa_bo_new+0x49d/0x4c0 [radeon]` -> `radeon_ib_get` -> `radeon_vm_bo_update` -> `radeon_cs_ioctl`。
- 复现场景：使用 Firefox 浏览 Google 街景时触发。

#问题根因
Radeon 驱动的 VM 实现存在已知缺陷。应用程序请求的缓冲区大小超过了 VM 代码的处理能力，但该大小未触发正常的错误处理路径，导致在 `radeon_sa_bo_new` 函数中触发 `BUG()`。

#解决方案
该问题在 `drm-next-3.15-wip` 分支中已修复，建议升级至后续内核版本。
