#标题
CDROM ioctl操作因scsi_request未初始化导致内核栈损坏引发panic

#内核版本
4.11-rc1

#问题现象
- 系统触发 Kernel panic，提示 "stack-protector: Kernel stack is corrupted"。
- Call Trace 显示崩溃发生在 `mmc_ioctl_cdrom_read_data` 函数中。
- 使用 VLC 播放器播放音频 CD 并在播放期间弹出光盘时，或调用 `CDROMREADRAW` ioctl 时可触发此故障。
- 相比 4.10 及之前版本（此时 ioctl 返回 ENOMEDIUM 错误），4.11-rc1 及之后版本表现为内核崩溃。

#问题根因
引入于 commit `82ed4db499b8598f16f8871261bff088d6b0597f` ("block: split scsi_request out of struct request")。
该提交重构了 `scsi_request` 结构体处理逻辑，移除了 SENSE 缓冲区的分配与初始化代码。导致 `struct scsi_request` 中的 `sense` 指针包含垃圾值（如 `0x9000100d00000`），而非有效的内核地址。后续代码尝试访问该无效指针引发内存访问错误，进而破坏内核栈。

#解决方案
暂无明确的修复补丁提交记录（Bug 状态为 NEW）。根据测试反馈，该问题在内核版本 4.18.5 中已不再复现，推测已在主线版本中修复。
