#标题
加载Lustre fld模块时断言失败导致Kernel Panic

#内核版本
3.12-rc7（日志显示为 3.12.0-rc5-qlustar-gen-1）

#问题现象
- 执行 `modprobe fld` 命令加载模块时触发 Kernel Panic。
- 控制台输出 LustreError：`ASSERTION( key->lct_init != ((void *)0) ) failed`。
- 系统打印 `LBUG` 并 panic，Call Trace 显示调用路径为 `fld_mod_init` -> `lu_context_key_register`。

#问题根因
在 `lu_context_key_register` 函数中，断言检查 `key->lct_init` 指针非空失败，导致触发 LBUG 机制引发 Kernel Panic。

#解决方案
应用附件中的补丁（Patch to fix kernel crash）可修复此问题；亦可通过链接 https://jira.hpdd.intel.com/browse/LU-4127 查看相关修复详情。
