#标题
USB全局挂起机制导致特定Hub下键盘无法唤醒S2RAM系统

#内核版本
3.10-rc3

#问题现象
- ThinkPad T420 在执行 s2ram（挂起到内存）后，无法通过外接 USB 键盘唤醒系统。
- 该问题为回归问题，在 3.9.4 版本正常，升级到 3.10-rc3 后出现。
- 键盘连接在 Genesys Logic 4-port Hub (ID 05e3:0608) 下。
- 必须通过打开笔记本盖子（LID）才能唤醒系统。
- 经二分定位，引入问题的 commit 为 `0aa2832dd0d9d8609fd8f15139bc7572541a1215`。

#问题根因
引入问题的 commit `0aa2832dd0d9d8609fd8f15139bc7572541a1215` ("USB: use 'global suspend' for system sleep on USB-2 buses") 修改了 USB-2 总线的挂起逻辑，不再单独挂起每个设备，而是通过停止发送 Start-Of-Frame (SOF) 包来实现“全局挂起”。
根因在于 Genesys Logic USB Hub (ID 05e3:0608) 硬件存在缺陷。在全局挂起模式下（Root Hub 停止发送 SOF），该 Hub 无法正确处理下游设备的远程唤醒信号，导致键盘按键无法唤醒主机。调试补丁验证表明，若对非 Hub 设备（如键盘）保持显式端口挂起而非依赖全局挂起，则唤醒功能恢复正常。

#解决方案
- 临时规避：回退 commit `0aa2832dd0d9d8609fd8f15139bc7572541a1215`。
- 调试补丁验证：开发者提供的 `debug-wakeup-failure-3.patch` (Suspend non-hub ports) 可解决问题，该补丁对非 Hub 设备保持原有的显式挂起逻辑。
- 根本原因指向 Hub 硬件缺陷，正式修复方案需等待上游进一步讨论（报告状态为 NEEDINFO）。
