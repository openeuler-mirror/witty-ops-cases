#标题
重载 iptables 卸载 iptable_nat 模块时触发 Kernel Oops

#内核版本
3.10.4

#问题现象
- 系统在重载 iptables 并尝试卸载 iptable_nat 模块时触发 Kernel Oops。
- 日志显示 "BUG: unable to handle kernel paging request at ffffc9001894f008"，错误码为 Oops: 0002 [#1]。
- RIP 寄存器指向 `get_counters+0x8d/0xc0`，Call Trace 显示调用链为 `alloc_counters -> do_ipt_get_ctl -> nf_sockopt -> ... -> SyS_getsockopt`。
- 故障发生后，iptables 命令无法正常执行。

#问题根因
在卸载 iptable_nat 模块的过程中，内核路径 `get_counters` 访问了无效的内存地址，导致内核分页请求错误。具体代码层面的根因未在报告中明确分析。

#解决方案
规避方法：修改 iptables 重载脚本，避免重新加载（卸载/加载）iptables 相关模块。
官方修复：暂无（Bug 状态为 NEW，建议尝试最新内核版本验证）。
