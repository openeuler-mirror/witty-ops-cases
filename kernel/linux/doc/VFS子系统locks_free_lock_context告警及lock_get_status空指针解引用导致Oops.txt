#标题
VFS子系统locks_free_lock_context告警及lock_get_status空指针解引用导致Oops

#内核版本
ubuntu 4.4.0-44-generic

#问题现象
- 进程 `ctdb_recovered` 运行期间触发内核 WARNING：`locks_free_lock_context+0x80/0xb0`，调用栈指向 `__destroy_inode` -> `evict` -> `iput` -> `d_invalidate` 路径。
- 进程 `lsof` 读取 `/proc/locks` 时触发 Kernel Oops：`BUG: unable to handle kernel NULL pointer dereference at 0000000000000068`，故障指令位于 `lock_get_status+0x96/0x340`。

#问题根因
进程在执行 inode 销毁流程时触发 `locks_free_lock_context` 警告；随后在读取锁状态信息时，`lock_get_status` 函数访问了空指针（偏移 0x68），导致内核崩溃。具体代码逻辑缺陷未在报告中明确说明。

#解决方案
暂无
