#标题
挂载特制btrfs镜像时extent_io_tree_panic触发空指针解引用导致内核崩溃

#内核版本
5.8.1（复现版本包括 5.4, 5.9.15, 5.10.0）

#问题现象
- 挂载特制的 btrfs 镜像文件时触发 kernel panic/oops。
- 内核日志显示 "general protection fault" 或 "BUG: kernel NULL pointer dereference, address: 0000000000000028"。
- 调用栈显示 RIP 位于 `extent_io_tree_panic.isra.0` 或 `btrfs_get_64`，路径包含 `btrfs_read_chunk_tree` -> `read_one_chunk` -> `add_extent_mapping` -> `__set_extent_bit`。
- 日志中伴有 BTRFS 元数据损坏告警，如 "corrupt leaf: unexpected item end" 或 "insert state: end < start"。

#问题根因
在函数 `extent_io_tree_panic` 中，代码直接将 `tree->private_data` 强制转换为 `struct inode *` 并解引用访问 `inode->i_sb`。然而，在处理特定 btrfs 结构（如块组树 chunk tree）时，`tree->private_data` 可能为 NULL，此时访问 `inode->i_sb`（即偏移 0x28 处）会触发空指针解引用，导致内核崩溃。

#解决方案
暂无（Bug 状态为 NEW，尚未合并修复补丁）。
