#标题
挂载NFSv4时因cred结构体magic未初始化触发kernel BUG

#内核版本
linux-v5.0-rc1

#问题现象
- 执行 mount -t nfs 挂载 NFSv4 文件系统失败
- dmesg 日志显示 "kernel BUG at kernel/cred.c:825!" 及 "invalid opcode: 0000 [#1]"
- 日志中打印 "CRED: Invalid credentials" 与 "CRED: ->magic=0"
- 触发 BUG 的 Call Trace 为：`__invalid_creds` <- `nfs4_discover_server_trunking` <- `nfs4_init_client`

#问题根因
sunrpc 模块中缺少对 cred 结构体中 debug 成员 `magic` 的初始化，导致该成员值为 0 而非预期的 `CRED_MAGIC`。内核在 `__invalid_creds` 函数中校验凭据有效性时发现 magic 值错误，从而触发 BUG。

#解决方案
该问题已在 commit e7f45099442a380f8e087b6a8aadc36e887df1cc 中修复，补丁标题为 "sunrpc: kernel BUG at kernel/cred.c:825!"。修复内容为初始化缺失的 debug 成员 magic 为 CRED_MAGIC。该修复已合入 linux-5.0-rc3 版本，升级至该版本或更高版本可解决此问题。
