#标题
ebtables模块处理snmpd流量时触发页面错误导致内核崩溃

#内核版本
3.10.0-957.5.1.rt56.916.el7.x86_64

#问题现象
- 系统运行 5-6 天后触发 Kernel Panic
- 日志报错："BUG: unable to handle kernel paging request at ffffbba3e8a225cc"
- 故障进程为 snmpd，Call Trace 显示崩溃路径为：`sys_sendmsg` -> `udp_sendmsg` -> `ip_send_skb` -> `br_dev_xmit` -> `nf_hook_slow` -> `ebt_do_table`

#问题根因
具体根因未明确。崩溃发生在 `ebt_do_table` 函数中，系统在处理 ebtables 规则时尝试访问无效的内存地址 `ffffbba3e8a225cc`，触发 page fault 异常。故障触发场景为 snmpd 发送 UDP 数据包经过网桥设备，进入 ebtables 的 OUTPUT 链处理流程。

#解决方案
暂无
