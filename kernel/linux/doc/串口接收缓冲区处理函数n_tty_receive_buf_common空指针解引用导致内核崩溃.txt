#标题
串口接收缓冲区处理函数n_tty_receive_buf_common空指针解引用导致内核崩溃

#内核版本
4.11.12

#问题现象
- 系统在进行串口控制台测试时触发 Kernel Oops，显示 "BUG: unable to handle kernel paging request at 0000000000002260"
- 故障现场 RIP 指向 n_tty_receive_buf_common+0x5a，进程上下文为 kworker/u80:5
- 调用栈显示：flush_to_ldisc -> tty_port_default_receive_buf -> tty_ldisc_receive_buf -> n_tty_receive_buf2 -> n_tty_receive_buf_common

#问题根因
函数 n_tty_receive_buf_common 内部发生空指针解引用。故障现场 RAX 寄存器值为 0，CR2 寄存器值为 0x2260，推测代码尝试访问空指针结构体的某个成员（偏移量 0x2260），导致触发内核分页请求异常。

#解决方案
暂无
