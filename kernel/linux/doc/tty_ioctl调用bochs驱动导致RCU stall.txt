#标题
tty_ioctl调用bochs驱动导致RCU stall

#内核版本
6.10-rc3

#问题现象
- 内核日志报告 "INFO: rcu detected stall in tty_ioctl"
- RCU subsystem 检测到 CPU 0 发生 self-detected stall，且 rcu_preempt kthread 饥饿
- 触发 NMI backtrace，显示 CPU 1 在处理 timer softirq 时卡在 `debug_object_deactivate` -> `lookup_object`
- 进程上下文调用链显示 `syz-executor` 通过 `tty_ioctl` -> `vt_ioctl` -> `tioclinux` -> `unblank_screen` 进入 DRM bochs 驱动

#问题根因
`tty_ioctl` 执行路径通过 `tioclinux` 触发屏幕解blank操作，进而调用 `bochs` 显卡驱动的 `bochs_pipe_enable` 函数。在该过程中，系统在 `bochs_dispi_write` 操作期间发生长时间阻塞或中断处理延迟（具体表现为 timer softirq 中的 `debug_object_deactivate` 执行缓慢），导致 CPU 无法及时调度 RCU 相关任务，最终触发 RCU stall 检测告警。

#解决方案
暂无
