#标题
Btrfs高负载IO下触发Bad page map导致内核崩溃

#内核版本
6.4.11-zen2-1-zen

#问题现象
- 系统在进行高负载 IO 操作（如编译 AOSP）时随机崩溃，只能通过硬重启恢复。
- 内核日志报错：`BUG: Bad page map in process tmux: client`。
- 页面状态异常：`page dumped because: bad pte`，`mapcount:-256`，`page_type: 0xfffffeff(offline)`。
- 调用栈显示：`print_bad_pte` -> `unmap_page_range` -> `exit_mmap` -> `mmput` -> `do_exit`。

#问题根因
未明确。日志显示页面引用计数异常且被标记为 `offline`，具体原因尚在排查中。社区讨论中提及可能与特定硬件平台（X99）内存控制器有关，或涉及内存管理子系统与 Btrfs 的交互问题，但未确认具体代码缺陷。

#解决方案
暂无官方修复。用户尝试切换至非 zen 内核版本以观察问题是否复现。
