#标题
r8169驱动重启后网卡不可用并报transmit queue timed out

#内核版本
4.14.14（报告版本），受影响版本包括 4.18、5.5.x

#问题现象
- 重启（从 Windows 或 Linux）进入 Linux 后，网卡不可用约 2-3 分钟，DHCP 客户端无法获取地址。
- 高负载场景下系统可能因网络驱动停滞而冻结。
- dmesg 日志显示：`NETDEV WATCHDOG: eth0 (r8169): transmit queue 0 timed out`。
- 触发 Call Trace：`WARNING: CPU: 2 PID: 0 at net/sched/sch_generic.c:320 dev_watchdog+0x215/0x220`。
- 冷启动（关机后开机）不会触发此问题。

#问题根因
未明确（推测为 r8169 驱动在热启动过程中对硬件状态（如 RTL8168g 芯片）的初始化处理不当，导致 transmit queue 挂起；该问题在 4.19 及 5.5.17 版本中已修复）。

#解决方案
- 升级内核至 4.19 或更高版本。
- 若在 5.5.x 版本遇到回归，升级至 5.5.17 或更高版本。
- 规避方法：执行冷启动（关机后再开机）而非重启。
