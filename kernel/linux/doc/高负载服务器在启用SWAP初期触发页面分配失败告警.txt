#标题
高负载服务器在启用SWAP初期触发页面分配失败告警

#内核版本
3.0.54-3.0.65

#问题现象
- 内核日志频繁打印 "page allocation failure: order:1, mode:0x20" 告警信息
- 故障发生在高负载场景下，当系统开始使用 SWAP（约 12MB 使用量）时出现
- 受影响进程包括 httpd、httpd.event 和 php-fpm
- Call Trace 显示调用链为 `sk_clone` -> `sk_prot_alloc` -> `kmem_cache_alloc` -> `__alloc_pages_nodemask` -> `warn_alloc_failed`

#问题根因
未明确。根据日志分析，系统在高负载且刚开始使用 SWAP 时，内存分配器无法分配 order:1（连续两个页面）的内存块。失败发生在网络子系统的 socket 克隆流程中。用户反馈当 SWAP 使用量增加到 500-700 MB 后，问题出现的频率显著降低，推测可能与内存碎片化程度或内存回收机制的触发时机有关。

#解决方案
暂无。该 Bug 状态为 NEW，尚未提供修复补丁。
