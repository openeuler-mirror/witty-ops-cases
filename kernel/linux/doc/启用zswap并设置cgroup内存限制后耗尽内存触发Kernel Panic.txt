#标题
启用zswap并设置cgroup内存限制后耗尽内存触发Kernel Panic

#内核版本
3.13.0

#问题现象
- 在启用 zswap（zswap.enabled=1）并设置 cgroup 内存限制（如 memory.memsw.limit_in_bytes）的环境下，耗尽内存触发 Kernel Panic。
- 系统日志显示 SLUB 分配器无法分配内存："SLUB: Unable to allocate memory on node -1"。
- 触发 Oops：unable to handle kernel paging request at 000000007346c580，RIP 指向 `cpuacct_charge` 函数。
- 最终报错 "Kernel panic - not syncing: Fatal exception in interrupt"，导致系统死机或重启。

#问题根因
启用 zswap 功能结合 cgroup 内存限制使用时，在内存耗尽的场景下触发内核异常。日志显示在内存分配失败后，`cpuacct_charge` 函数访问了非法内存地址导致 Oops，具体关联 zswap 的代码路径缺陷未在报告中详细分析。

#解决方案
暂无官方修复补丁。临时规避方法为关闭 zswap 功能（设置 zswap.enabled=0）。
