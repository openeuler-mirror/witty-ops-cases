#标题
VMD raid模式与SecureBoot同时开启导致NVMe重置时触发空指针解引用

#内核版本
5.7-rc3

#问题现象
- Dell TigerLake 平台在同时开启 VMD raid mode 和 SecureBoot 时，系统启动过程中内核报 WARNING 和 NULL pointer dereference 错误。
- 日志显示 `WARNING: ... at drivers/iommu/intel-iommu.c:625 domain_get_iommu+0x4b/0x60`。
- 随后触发 `BUG: kernel NULL pointer dereference, address: 0000000000000018`，Call Trace 指向 `nvme_reset_work` -> `__intel_map_single` -> `domain_get_iommu`。
- systemd-udevd 工作进程阻塞直至超时，但系统仍能启动至 multi-users.target。

#问题根因
经 Bisect 定位，问题由 commit e3560ee4cfb2 ("iommu/vt-d: Remove VMD child device sanity check") 引入。该提交移除了 VMD 子设备的特定检查，导致在特定配置（VMD raid + SecureBoot）下，VMD 子设备的 IOMMU 域信息未正确初始化，在 `domain_get_iommu` 函数中访问空指针触发 Oops。

#解决方案
规避方法：在内核启动参数中添加 `intel_iommu=on`。
修复方案：应用补丁 "allocate info for vmd child devices"（附件 288935）为 VMD 子设备分配必要的 info 结构，或应用上游提供的替代补丁集。
