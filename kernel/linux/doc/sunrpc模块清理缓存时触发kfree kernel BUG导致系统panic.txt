#标题
sunrpc模块清理缓存时触发kfree kernel BUG导致系统panic

#内核版本
4.7.0

#问题现象
- 系统运行过程中触发 kernel panic，日志显示 "Fatal exception"。
- 控制台输出 "kernel BUG at mm/slub.c:3775!" 及 "invalid opcode: 0000 [#1] SMP"。
- 堆栈信息显示崩溃发生在 `kfree` 函数中，调用链为：`do_cache_clean [sunrpc] -> cache_clean [sunrpc] -> rsc_put [auth_rpcgss] -> rsc_free [auth_rpcgss] -> kfree`。

#问题根因
由 commit 64c59a3726f2dfa71680f9f3c34359e10e3ed7ab ("Remove unnecessary allocation") 引入的回归问题。该提交导致 `auth_rpcgss` 模块在执行 `rsc_free` 释放内存资源时，传递了错误的指针或触发了 SLUB 分配器的非法状态检查，从而引发 BUG。

#解决方案
回退 commit 64c59a3726f2dfa71680f9f3c34359e10e3ed7ab 可修复该问题。
