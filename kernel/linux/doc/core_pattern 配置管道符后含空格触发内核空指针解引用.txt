#标题
core_pattern 配置管道符后含空格触发内核空指针解引用

#内核版本
5.4.28

#问题现象
- 系统日志报错 "BUG: kernel NULL pointer dereference, address: 0000000000000020"。
- 触发 Oops，RIP 位于 `do_coredump+0x5b4/0x1080`。
- 当 `/proc/sys/kernel/core_pattern` 内容为 "| /root/coredump.sh"（管道符后有空格）时，进程（如 cat）触发 core dump 会导致内核崩溃。

#问题根因
在 `do_coredump` 函数处理逻辑中，当 core_pattern 字符串以管道符 "|" 开头且后跟空格时，解析过程触发空指针解引用。用户推测此问题为回归问题，可能与 lkml 上关于 core_pattern 解析的补丁有关。

#解决方案
暂无官方补丁。规避措施：配置 core_pattern 时确保管道符 "|" 后无空格，例如使用 "|/root/coredump.sh"。
