#标题
IPv6转发流量下pskb_expand_head触发kernel BUG导致系统崩溃

#内核版本
>= 3.7

#问题现象
- 系统在处理 IPv6 netfilter/conntrack 流量转发时发生 kernel panic
- 日志报错：kernel BUG at net/core/skbuff.c:1048!
- 无效操作码错误：invalid opcode: 0000 [#1] SMP
- 堆栈信息显示崩溃发生在 pskb_expand_head+0x30/0x210，调用链包含 ip6_forward、__ipv6_conntrack_in 等模块

#问题根因
由 commit 58a317f1061c894d2344c0b6a18ab4a64b69b815 ("netfilter: ipv6: add IPv6 NAT support") 引入。该补丁增加了 IPv6 NAT 支持，导致在特定转发场景下 skb_buff 结构处理异常，触发 pskb_expand_head 中的 BUG 检查。

#解决方案
应用修复补丁（http://patchwork.ozlabs.org/patch/288967/ 和 http://patchwork.ozlabs.org/patch/288970/），补丁当时处于等待合入 stable 状态。
