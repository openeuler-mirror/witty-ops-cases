#标题
z3fold压缩路径释放页面时因引用计数为0触发kernel BUG

#内核版本
5.10.7-arch1-1

#问题现象
- 系统在运行压力测试（如 Prime95）或高 swap 负载时触发 kernel BUG，导致系统半冻结或无法正常关机
- 内核日志报错："kernel BUG at include/linux/mm.h:707!" 及 "invalid opcode: 0000 [#1]"
- 日志显示页面引用计数异常："page dumped because: VM_BUG_ON_PAGE(page_ref_count(page) == 0)"
- Call Trace: `__free_pages+0x83/0x90` <- `z3fold_zpool_shrink+0x146/0x720` <- `shrink_worker`

#问题根因
在 `z3fold_zpool_shrink` 函数执行 shrink 操作时，调用 `__free_pages` 尝试释放一个引用计数（refcount）已经为 0 的页面，触发了内存管理子系统中的 `VM_BUG_ON_PAGE` 安全检查。这表明 z3fold 模块在内存压缩或回收路径中存在页面生命周期管理错误（例如重复释放）。

#解决方案
暂无（Bug 状态为 NEW，确认问题自 5.10 内核引入，5.9.16 版本未复现）。
