#标题
ReiserFS在direntry_create_vi函数中发生越界写入导致内核Panic

#内核版本
4.18

#问题现象
- 挂载构造的ReiserFS文件系统镜像并执行truncate操作时触发KASAN报错
- KASAN检测到slab-out-of-bounds写入，位于函数 `direntry_create_vi+0x1c1/0x320`
- 内核日志显示 "REISERFS warning: vs-16090 direntry_bytes_number"
- 随后触发 kernel BUG at `fs/reiserfs/prints.c:369`，系统Panic，提示 "vs-8030 create_virtual_node: virtual node space consumed"

#问题根因
在处理ReiserFS文件系统truncate操作时，调用链 `reiserfs_truncate_file` -> `reiserfs_cut_from_item` -> `fix_nodes` -> `create_virtual_node` -> `direntry_create_vi` 中，`direntry_create_vi` 函数对目录项进行处理时发生越界写入。构造的镜像数据导致虚拟节点空间计算或访问异常，触发KASAN越界告警并最终耗尽虚拟节点空间引发Panic。

#解决方案
暂无
