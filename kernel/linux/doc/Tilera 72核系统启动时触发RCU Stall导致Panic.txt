#标题
Tilera 72核系统启动时触发RCU Stall导致Panic

#内核版本
3.10.90

#问题现象
- Tilera 架构下，单核运行正常，切换至 72 核时系统崩溃
- 内核日志显示 "INFO: rcu_sched self-detected stall on CPU { 0}"
- 多个 CPU 堆栈显示在中断处理流程中阻塞：CPU 3 在 `_raw_spin_lock` 处自旋，CPU 1 在 `tilegx_legacy_irq_ack` 处执行，CPU 0 在 `rcu_check_callbacks` 处触发告警

#问题根因
未明确。根据日志分析，疑似中断处理函数 `handle_level_irq` 及 `trio_handle_level_irq` 中存在锁竞争或死锁，导致 CPU 长时间阻塞进而触发 RCU Stall。

#解决方案
暂无
