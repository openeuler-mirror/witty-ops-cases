#标题
amdgpu MST 显示器重新启用时触发 drm_mode_object_add 内核警告

#内核版本
5.8.0-rc5

#问题现象
- 系统日志中出现内核 WARNING：`WARNING: CPU: 2 PID: 23119 at drivers/gpu/drm/drm_mode_object.c:45 drm_mode_object_add+0x75/0x80 [drm]`。
- 故障发生在显示器关闭后重新启用的场景。
- Call Trace 显示调用路径为：`drm_dp_mst_link_probe_work` -> `drm_dp_mst_handle_link_address_port` -> `drm_dp_mst_port_add_connector` -> `dm_dp_add_mst_connector` -> `drm_encoder_init` -> `drm_mode_object_add`。

#问题根因
该问题为回归问题。在 DRM MST（Multi-Stream Transport）链路探测过程中，系统尝试为 MST 端口添加连接器并初始化 encoder 时，`drm_mode_object_add` 函数触发了警告条件。具体触发逻辑未在报告中详细说明。

#解决方案
暂无。Bug 状态为 NEW，相关讨论见 https://gitlab.freedesktop.org/drm/amd/-/issues/1108。
