#标题
路由器运行数天后在kfree中触发Kernel Panic

#内核版本
3.12.1, 3.10.23

#问题现象
- 系统作为路由器运行数天后出现 Kernel Panic 或 "Out of memory no killable processes" 错误。
- 内核日志显示 `invalid opcode: 0000 [#1]`，RIP 寄存器指向 `kfree+0x129/0x130`。
- 调用栈显示：`free_pipe_info+0x72/0x80` -> `pipe_release+0x96/0xf0` -> `__fput`。
- 触发进程为执行网络列表下载及 ipset 更新操作的 perl 脚本 (`get.pl`)。

#问题根因
内核在 `kfree` 函数中检测到非法状态并触发 `BUG()`，调用链表明问题发生在释放管道信息 (`free_pipe_info`) 的过程中。开发者分析认为这是典型的内存损坏现象，具体触发原因未明确，可能与管道释放逻辑或加载的第三方模块 `ipt_NETFLOW` 有关。
