#标题
STM32F469 DRM子系统因MIPI DSI通信失败导致vblank等待超时

#内核版本
5.9.16

#问题现象
- 在STM32F469i-disco开发板上向 `/sys/class/graphics/fb0/pan` 写入数据时，内核打印WARNING信息：`[CRTC:35:crtc-0] vblank wait timed out`
- 堆栈指向 `drm_atomic_helper_wait_for_vblanks+0x12f/0x164`
- 屏幕保持黑屏状态
- 调节背光亮度时报错：`failed to write command FIFO` 和 `mipi dsi dcs write buffer failed`

#问题根因
DRM子系统在执行atomic commit流程时，底层MIPI DSI驱动写入命令FIFO失败，导致显示面板无法正常同步或初始化，CRTC未产生预期的vblank中断，致使 `drm_atomic_helper_wait_for_vblanks` 函数等待超时并触发告警。

#解决方案
暂无
