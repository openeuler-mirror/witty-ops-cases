#标题
USB Type-C UCSI驱动初始化时触发空指针解引用

#内核版本
5.4.2

#问题现象
- 系统启动或连接 Lenovo USB-C Dock 时触发 Kernel Oops
- 日志显示 "BUG: kernel NULL pointer dereference, address: 00000000000002de"
- 崩溃发生在 `typec_altmode_update_active+0x16/0xf0`，调用栈包含 `ucsi_altmode_update_active` 和 `ucsi_init`
- 用户报告伴随音频子系统异常，但开发者指出 NULL 指针源于 UCSI 驱动，与音频功能无直接关联

#问题根因
UCSI (USB Type-C Connector System Software Interface) 驱动在初始化过程中调用 `typec_altmode_update_active` 函数时访问了无效指针。开发者确认该问题由相关代码逻辑缺陷导致。

#解决方案
该问题已在主线内核通过 commit `b0fcdffdd626` ("usb: typec: Prepare alt mode enter/exit reporting for UCSI alt mode support") 修复。临时规避方法是在内核配置中禁用 UCSI 相关驱动。
