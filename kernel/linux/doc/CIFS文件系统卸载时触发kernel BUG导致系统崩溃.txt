#标题
CIFS文件系统卸载时触发kernel BUG导致系统崩溃

#内核版本
v5.18-rc7+（评论中提及5.19-rc7+仍复现）

#问题现象
- 在ppc64le平台运行fstests测试用例（如generic/072）时，执行umount操作触发kernel panic
- 内核日志显示：kernel BUG at fs/inode.c:611!
- Call Trace：clear_inode+0x38/0xb0 -> cifs_evict_inode+0x50/0xb0 [cifs] -> evict -> dispose_list -> evict_inodes -> generic_shutdown_super

#问题根因
在CIFS文件系统卸载过程中，`cifs_evict_inode`函数调用`clear_inode`时，触发了`fs/inode.c:611`处的内核BUG检查（通常为inode状态断言失败），具体代码逻辑原因未明确。

#解决方案
暂无
