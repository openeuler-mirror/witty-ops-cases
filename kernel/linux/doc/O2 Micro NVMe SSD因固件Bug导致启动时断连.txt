#标题
O2 Micro NVMe SSD因固件Bug导致启动时断连

#内核版本
6.4, 6.8

#问题现象
- 使用 O2 Micro NVMe SSD（如 FORESEE E2M2）作为启动盘时，系统在启动过程中概率性挂起（复现率 20%-40%）。
- 串口控制台显示启动盘因 PCIe 错误断开连接。
- 内核日志报错：`pcieport AER: Multiple Uncorrected (Non-Fatal) error received`，错误类型为 `UnsupReq`。
- 随后 NVMe 驱动报 I/O 超时：`nvme nvme0: I/O ... timeout, aborting`。

#问题根因
该问题由 commit `da9619a30e73` ("dmapool: link blocks across pages") 触发。根本原因是 NVMe 设备固件存在 Bug：设备在命令完成后仍试图访问 DMA pool 页面中的内存。此前内核分配页面速度较慢，掩盖了该时序问题；该 commit 优化了 dmapool 性能，加快了内存分配和重用速度，从而暴露了固件的非法内存访问行为，导致 PCIe AER 错误和设备断连。

#解决方案
已合并修复补丁 commit `83bdfcbdbe5d` ("nvme-pci: qdepth 1 quirk")，通过为受影响设备添加 quirk 限制队列深度为 1 来规避固件问题。临时规避方案为回退相关 dmapool 提交。
