#标题
mremap扩展MAP_SHARED匿名映射导致访问扩展区域触发Bus Error

#内核版本
3.4（问题最早发现于2.6.21.5）

#问题现象
用户程序通过 mmap 创建 MAP_ANONYMOUS|MAP_SHARED 匿名映射，并使用 mremap 扩展该映射区域大小。当尝试访问扩展后的内存区域（原映射大小之外的部分）时，进程收到 Bus error 信号。

#问题根因
mremap 系统调用在扩展 vm_area_struct 时，未同步调整底层 shmemfs 文件的大小。当访问扩展区域触发缺页异常时，内核在 shmem_fault 函数中检查发现文件大小未覆盖该地址范围，导致访问失败并触发 Bus error。

#解决方案
暂无已合并的修复补丁。开发者讨论指出需要修改 mremap 逻辑以支持调整 shmem 段的大小，但具体实现尚未完成。
