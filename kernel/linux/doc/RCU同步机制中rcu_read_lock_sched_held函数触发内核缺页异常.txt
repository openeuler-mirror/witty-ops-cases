#标题
RCU同步机制中rcu_read_lock_sched_held函数触发内核缺页异常

#内核版本
5.3.10

#问题现象
- 内核触发 Oops，日志显示 "BUG: unable to handle kernel paging request in rcu_read_lock_sched_held"。
- 发生缺页异常，错误码 error_code(0x0002) 表示页面不存在，故障地址为 fffffe000000eff8。
- 触发环境为开启 KASAN 的内核，通过 Syzkaller 工具复现。

#问题根因
在 `rcu_read_lock_sched_held` 函数处理过程中，特定情况下传入的地址缺少内存映射，导致内核模式下发生缺页异常。该问题通过 `perf_event_open` 系统调用触发，调用链经过 `do_debug` -> `ist_enter` -> `rcu_nmi_enter` 最终到达 `rcu_read_lock_sched_held`。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
