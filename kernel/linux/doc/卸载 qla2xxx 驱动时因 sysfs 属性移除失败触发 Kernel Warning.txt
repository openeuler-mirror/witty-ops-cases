#标题
卸载 qla2xxx 驱动时因 sysfs 属性移除失败触发 Kernel Warning

#内核版本
4.7.3

#问题现象
- 卸载 qla2xxx 驱动过程中触发 kernel warning/oops
- 内核日志报错："kernfs: can not remove 'node_name', no directory" 与 "kernfs: can not remove 'port_name', no directory"
- Call Trace 显示：`kernfs_remove_by_name_ns` -> `sysfs_remove_file_ns` -> `device_remove_file` -> `attribute_container_remove_attrs`，触发点位于 `fc_starget_delete` 工作队列

#问题根因
驱动卸载流程中，`fc_starget_delete` 回调函数尝试移除 `node_name` 和 `port_name` sysfs 属性，但此时对应的 sysfs 目录已不存在（具体导致属性缺失或重复移除的逻辑原因未明确）。

#解决方案
暂无
