#标题
z3fold模块在内存收缩时触发引用计数检查失败导致Kernel BUG

#内核版本
5.10.11-arch1-1

#问题现象
- 系统在高负载且进行大量 swap 操作时（如停止 Java 进程）出现卡顿或半死机状态
- 内核日志报错：kernel BUG at include/linux/mm.h:707! 及 invalid opcode: 0000 [#1]
- 触发 VM_BUG_ON_PAGE(page_ref_count(page) == 0) 检查失败
- Call Trace 指向：__free_pages <- z3fold_zpool_shrink <- shrink_worker

#问题根因
z3fold 模块在执行 shrink 操作（zswap 回收）过程中，试图释放一个引用计数（refcount）已经为 0 的页面，触发了 __free_pages 函数中的 VM_BUG_ON_PAGE 断言失败。这表明 z3fold 内部存在引用计数管理错误（如双重释放）。

#解决方案
暂无
