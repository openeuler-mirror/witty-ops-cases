#标题
list_lru 初始化失败后未置空指针导致空指针解引用

#内核版本
4.7.7

#问题现象
- 系统日志报告 "NULL pointer dereference"，RIP 位于 `list_lru_destroy+0x40/0x100`
- 故障发生前伴有内存分配失败警告 `warn_alloc_failed`
- Call Trace 显示调用路径为 `destroy_super` -> `sget` -> `proc_mount` -> `pid_ns_prepare_proc` -> `alloc_pid` -> `copy_process`，触发进程为 vsftpd

#问题根因
在 `mm/list_lru.c` 的 `__list_lru_init` 函数中，若 `memcg_init_list_lru` 初始化失败，代码执行 `kfree(lru->node)` 释放内存，但未将 `lru->node` 指针置为 NULL。随后上层函数 `alloc_super` 失败进入清理流程调用 `destroy_super` 时，`list_lru_destroy` 检测到 `lru->node` 非 NULL（悬空指针），误认为其已初始化并尝试访问 `lru->node[i].memcg_lrus`，导致空指针解引用。

#解决方案
在 `mm/list_lru.c` 的错误处理分支中，`kfree(lru->node)` 之后显式添加 `lru->node = NULL;`，确保后续调用 `list_lru_destroy` 时不会访问悬空指针。补丁标题为 "mm/list_lru.c: avoid error-path NULL pointer deref"。
