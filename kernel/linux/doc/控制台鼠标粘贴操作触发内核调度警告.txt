#标题
控制台鼠标粘贴操作触发内核调度警告

#内核版本
4.0.0-rc6+ git

#问题现象
在非KMS控制台（文本模式TTY）中使用鼠标进行粘贴操作（如gpm中键粘贴）时，内核打印WARNING堆栈信息，日志显示：
- "do not call blocking ops when !TASK_RUNNING; state=1 set at paste_selection"
- Call Trace指向：`__might_sleep` -> `down_read` -> `n_tty_receive_buf_common` -> `paste_selection`

#问题根因
`paste_selection` 函数执行过程中，将当前进程状态设置为非 TASK_RUNNING（state=1），随后调用 `n_tty_receive_buf_common` 函数。该函数内部尝试获取读写信号量（`down_read`），这是一个可能阻塞的操作。内核调度器检测到进程在非 RUNNING 状态下调用可能睡眠的阻塞函数，从而触发 `__might_sleep` 检查警告。

#解决方案
暂无
