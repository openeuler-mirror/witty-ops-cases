#标题
Swap设备写入保护机制导致s2disk休眠快照失败

#内核版本
5.4-rc1 及以上版本（含 5.5.X, 5.6）

#问题现象
使用 s2disk 工具进行休眠时快照创建失败，系统无法正常休眠；该问题影响依赖 s2disk 的休眠工具（如 systemd, pm-hibernate）。

#问题根因
Commit dc617f29dbe5ef0c8ced65ce62c464af1daaab3d 引入了新的 swap 设备写入保护机制，该机制阻止了用户空间休眠工具 uswsusp (s2disk) 向 swap 设备写入内存镜像数据。

#解决方案
用户提供了修复补丁，通过修改 `kernel/power/user.c` 和 `mm/swapfile.c`，在 `snapshot_ioctl` 流程中暂时移除 swap 设备的 `S_SWAPFILE` 标志以允许 s2disk 写入，并在操作完成后调用新增的 `swap_relockall` 函数重新锁定 swap 设备。
