#标题
MD模块在软中断上下文调用md_write_start触发scheduling while atomic

#内核版本
4.18.0-240.22.1.el8_3.x86_64

#问题现象
- 内核日志报错："BUG: scheduling while atomic: swapper/3/0/0x00000100"
- 日志提示："bad: scheduling from the idle thread!"
- 调用栈显示在 `raid5_align_endio` 处理流程中调用了 `md_write_start`，进而触发 `schedule`。

#问题根因
在软中断上下文（`__do_softirq` -> `blk_done_softirq` -> `raid5_align_endio`）中，调用链触发了 `raid5_make_request` -> `md_write_start`。`md_write_start` 函数内部调用了 `wait_event` 尝试睡眠调度，违反了原子上下文不能调度的原则。开发者分析指出，`raid5_align_endio` 通常对应读 IO，但此处却触发了写 IO 相关的 `md_write_start`，调用路径存在异常。

#解决方案
暂无
