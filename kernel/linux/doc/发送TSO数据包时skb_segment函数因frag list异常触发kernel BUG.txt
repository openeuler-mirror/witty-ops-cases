#标题
发送TSO数据包时skb_segment函数因frag list异常触发kernel BUG

#内核版本
3.10.0-123.el7.x86_64

#问题现象
- 系统发生 kernel panic，日志显示 "kernel BUG at net/core/skbuff.c:2966!"
- 故障发生在 KVM 虚拟化环境使用 Open vSwitch 发送数据包时
- 故障堆栈显示调用路径为：`bnx2_poll` -> `napi_gro_receive` -> `dev_gro_receive` -> `tcp_gso_segment` -> `skb_segment`
- 故障进程为 "qemu-kvm"

#问题根因
在 `skb_segment` 函数处理 GSO/TSO 分段逻辑时，代码在 `net/core/skbuff.c:2966` 行执行 `BUG_ON(skb_headlen(list_skb))` 检查失败。
经 vmcore 调试分析，frag list 中的 `list_skb` 结构体成员 `len` 为 1430，`data_len` 为 0，导致 `skb_headlen` 返回非零值（1430），违反了该处代码预期（预期 frag list 中的 skb 不应包含线性数据）。用户推测该异常 skb 结构可能源于 `dev_gro_receive` 或 `skb_gro_receive` 的处理逻辑。

#解决方案
暂无（Bug 状态为 NEW，评论建议尝试升级内核验证是否已修复）。
