#标题
USB gadget关闭流程中dev_driver_string空指针解引用导致Kernel Panic

#内核版本
5.15.118

#问题现象
- 在通过configfs删除USB gadget配置（RNDIS或ECM模式）时触发Kernel Panic。
- 日志显示 "Unable to handle kernel access to user memory outside uaccess routines at virtual address 00000000000000d0"。
- Call trace 显示崩溃路径为：`dev_driver_string` -> `__dev_printk` -> `ecm_close` -> `eth_stop` -> `__dev_close_many`。

#问题根因
在USB gadget驱动的关闭函数（如`ecm_close`）中，调用了`dev_printk`系列的调试打印函数，但传入的`struct device`指针为NULL。函数`dev_driver_string`未对`dev`参数进行NULL检查便直接解引用访问成员（偏移0xd0），导致空指针解引用并触发Oops。

#解决方案
暂无官方修复方案。用户建议在`dev_driver_string`函数中增加对NULL参数的检查以避免崩溃，或在用户态操作时确保先执行ifdown再停止gadget以规避该问题。
