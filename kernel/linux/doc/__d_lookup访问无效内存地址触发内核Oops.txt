#标题
__d_lookup访问无效内存地址触发内核Oops

#内核版本
5.7.9

#问题现象
- 系统从休眠恢复一段时间后，应用程序（如 Firefox）出现卡死，系统响应异常。
- 内核日志报错："BUG: unable to handle page fault for address: 0218200c"。
- 触发 Oops（Error Code: 0000），异常发生在内核函数 `__d_lookup` 中。
- 调用栈显示：`__d_lookup` <- `d_lookup` <- `path_openat`，表明在文件路径查找过程中发生崩溃。
- 另有用户报告在 nfsd 服务中触发 `general_protection` 错误，堆栈同样指向 `__d_lookup`。

#问题根因
内核在 `__d_lookup` 函数中遍历目录项缓存进行查找时，访问了非法内存地址。根据寄存器信息（EBX: 02182000），`dentry` 指针可能已损坏或指向了无效的内存页，导致读取偏移量 `0x0c` 处数据时触发页错误。具体导致指针损坏的深层原因未明确，可能与休眠恢复后的内存状态异常或并发竞态有关。

#解决方案
暂无（Bug 状态为 NEW，尚未提供修复补丁或规避措施）。
