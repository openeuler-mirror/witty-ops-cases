#标题
BTRFS balance 操作期间触发 RCU stall 导致系统无响应

#内核版本
6.1.72-mcbin

#问题现象
- 系统日志报告 RCU stall：`rcu: INFO: rcu_sched self-detected stall on CPU`
- 进程 `btrfs` (PID 1336962) 卡死，调用栈显示进程处于 `down_write` 状态
- 调用链路：`btrfs_balance` -> `btrfs_relocate_chunk` -> `clean_dirty_subvols` -> `btrfs_drop_snapshot` -> `walk_down_tree` -> `do_walk_down` -> `btrfs_tree_lock` -> `down_write`

#问题根因
进程在执行 BTRFS balance 相关的清理脏子卷操作时，尝试获取写锁（`down_write`）阻塞过久，导致 RCU 调度器检测到 CPU 停顿；具体死锁或阻塞原因未明确。

#解决方案
暂无
