#标题
启用 AMD IOMMU 导致 CMA 机制失效引发大块 DMA 分配失败

#内核版本
All 3.x (x86_64) versions supporting Contiguous Memory Allocator

#问题现象
- 启用 AMD IOMMU 后，`dma_alloc_coherent` 函数无法利用 CMA（Contiguous Memory Allocator）进行内存分配。
- 需要大块连续物理内存的 DMA 操作失败。
- 相同场景下 Intel IOMMU 工作正常。

#问题根因
AMD IOMMU 驱动程序劫持了 `dma_alloc_coherent` 函数，但未调用 `dma_alloc_from_contiguous` 函数，导致系统在启用 AMD IOMMU 时忽略了 CMA 配置，无法为大块 DMA 请求分配连续内存。

#解决方案
相关修复已计划合入 4.1 版本，但报告者指出该修复可能未完全遵循 CMA 分配约定（即启用 CMA 时应优先从 CMA 分配）。目前暂无针对 stable 版本的明确回溯计划。
