#标题
SD卡移除时触发块层限流代码空指针解引用导致Kernel Oops

#内核版本
4.2.2

#问题现象
- 物理弹出SD卡时系统发生Kernel Oops，导致系统完全死机，需硬复位重启。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000558"。
- 触发Oops的指令指针位于 `__blkg_lookup` 函数。
- 调用栈显示路径为 `mmc_rescan` -> `blk_throtl_drain` -> `__blkg_lookup`。

#问题根因
在SD卡移除流程中，`mmc_rescan` 工作队列触发设备移除，调用链经过 `blk_release_queue` -> `elevator_exit` -> `cfq_exit_queue` -> `blkcg_deactivate_policy` -> `blk_throtl_drain`。在 `blk_throtl_drain` 调用 `__blkg_lookup` 时，发生了空指针解引用（访问地址 0x558），导致内核崩溃。

#解决方案
升级内核版本至 4.9.11 或更高版本，该问题在后续版本中已修复。
