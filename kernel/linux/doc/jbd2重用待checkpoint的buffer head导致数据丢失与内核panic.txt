#标题
jbd2重用待checkpoint的buffer head导致数据丢失与内核panic

#内核版本
6.2.0-rc1

#问题现象
- 系统触发 Kernel panic，日志显示 "Kernel panic - not syncing: DONE"。
- Call Trace 显示调用路径为：`jbd2_log_do_checkpoint` -> `__jbd2_log_wait_for_space` -> `ext4_setattr`。
- 重启后执行文件系统检查（fsck），发现元数据错误，提示 "Entry 'a' in / (2) has deleted/unused inode 12"，文件系统存在错误，表明发生了数据丢失。

#问题根因
JBD2（Journaling Block Device 2）模块在处理事务时，错误地重用了正处于待 checkpoint 状态的 buffer head (bh)。这导致数据丢失，并在 checkpoint 处理逻辑中因检测到异常状态（如 dirty bit 未正确清除）而触发 panic。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）。
