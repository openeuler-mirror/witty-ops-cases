#标题
ath12k驱动因board-2.bin缺少board-id配置导致加载失败

#内核版本
6.4.0 / 6.6.10 / 6.7.6 / 6.10.4

#问题现象
- 系统启动或加载 ath12k 模块时，dmesg 日志报错：`failed to fetch board data for bus=pci,qmi-chip-id=x,qmi-board-id=255 from ath12k/QCN9274/hw2.0/board-2.bin`
- 日志显示 `qmi failed to load board data file:-2`，随后加载流程中断。
- 受影响的无线网卡（如 QCN9274、WCN7850 芯片）无法正常初始化，Wi-Fi 功能不可用。

#问题根因
部分网卡硬件（例如 Wallys DR9274-5G&6G）未烧录有效的 board-id，导致驱动读取到默认值 `0xff`（即 255）。
官方发布的 `board-2.bin` 固件文件中未包含 `qmi-board-id=255` 的匹配条目。
驱动程序在查找板级数据时，无法在 `board-2.bin` 中找到对应 ID 的配置，且回退查找 `board.bin` 也失败，最终导致固件加载失败。

#解决方案
临时规避方案：
1. 使用 `ath12k-bdencoder` 工具解压官方 `board-2.bin` 文件。
2. 从解压出的文件中提取特定频段（如 5GHz 对应 ID）的板级数据文件。
3. 将该文件重命名为 `board.bin` 并放置于对应固件目录（如 `/lib/firmware/ath12k/QCN9274/hw2.0/`）。
4. 重启系统尝试加载。

长期解决方案：
联系硬件厂商获取适配该网卡的具体 `board.bin` 文件，或请求厂商在硬件中烧录正确的 `board-id`。对于 split-phy（多频段）网卡，目前可能缺乏通用的 board 文件支持。
