#标题
saa7134_alsa 驱动调用 vmalloc_32 触发 kernel BUG 导致应用卡死

#内核版本
4.13.0-0.rc5.git0.2.fc27.x86_64

#问题现象
- 系统启动过程中触发 kernel BUG，日志显示 "kernel BUG at ./include/linux/gfp.h:423!" 及 "invalid opcode: 0000 [#1]"。
- 多媒体应用（如 pragha）在连接 libpulse 时卡死无响应。
- 故障复现条件：系统插有 LifeView TV FlyDVB Trio PCI 卡。
- Call Trace 指向 `snd_card_saa7134_hw_params` -> `vmalloc_32` -> `__alloc_pages_nodemask`。

#问题根因
驱动 `saa7134_alsa` 在 `snd_card_saa7134_hw_params` 函数中调用 `vmalloc_32` 申请内存时，底层 `__alloc_pages_nodemask` 触发了 `gfp.h` 中的 `BUG()`，通常意味着内存分配标志（GFP flags）使用不当或在不合法的上下文中进行分配。

#解决方案
暂无
