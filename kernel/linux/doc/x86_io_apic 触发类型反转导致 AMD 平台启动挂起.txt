#标题
x86/io_apic 触发类型反转导致 AMD 平台启动挂起

#内核版本
5.11（受影响版本包括 5.11.1 - 5.11.8，5.11.10 已修复）

#问题现象
- Dell Inspiron 3195（AMD A9-9420e RADEON R5）从 5.10 升级至 5.11 后，系统在启动阶段挂起。
- systemd-boot 加载后屏幕变黑，无任何日志输出，系统无响应但电源按钮仍可响应关机。
- 使用 `acpi=off` 或 `iommu=off` 内核参数可临时启动系统，但会导致性能下降或功能缺失（如 WiFi 不可用）。

#问题根因
Git bisect 定位到提交 a27dca645d2c ("x86/io_apic: Cleanup trigger/polarity helpers")。该重构代码错误地反转了 PCI 和 ISA 的默认中断触发类型：PCI 默认应为 level trigger，ISA 默认应为 edge trigger，但代码将其弄反了。该错误导致部分 AMD 系统在启动过程中因中断配置错误而挂起。

#解决方案
该问题已通过提交 aec8da04e4d7 ("x86/ioapic: Correct the PCI/ISA trigger type selection") 修复。升级内核至 5.11.10 或更高版本可解决此问题。
