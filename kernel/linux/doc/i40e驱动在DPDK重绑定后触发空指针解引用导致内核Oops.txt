#标题
i40e驱动在DPDK重绑定后触发空指针解引用导致内核Oops

#内核版本
5.1.12

#问题现象
- 内核日志报错 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000832"，触发 Oops (Oops: 0000 [#1])。
- 故障发生在 i40e 驱动模块中，RIP 寄存器指向 `i40e_get_phys_port_id+0x18/0x40 [i40e]`。
- 调用栈显示路径为 `rtnl_phys_port_id_fill` -> `i40e_get_phys_port_id`。
- 复现条件：网卡经过大量数据传输后，使用 dpdk-devbind 工具解绑并重新绑定 i40e 驱动，随后执行 `ip link set up` 操作。

#问题根因
在函数 `i40e_get_phys_port_id` 执行过程中，寄存器 RDX 值为 0（空指针），代码尝试访问 `RDX+0x832` 处的内存（对应指令 `f6 82 32 08 00 00 02`），导致访问非法地址 0x832 触发缺页异常。该问题与 DPDK 绑定操作后的驱动状态或硬件指针初始化异常有关。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）。
