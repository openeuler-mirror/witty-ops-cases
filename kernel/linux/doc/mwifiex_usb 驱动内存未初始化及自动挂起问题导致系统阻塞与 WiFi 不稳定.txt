#标题
mwifiex_usb 驱动内存未初始化及自动挂起问题导致系统阻塞与 WiFi 不稳定

#内核版本
3.15.0 (问题复现于 3.13.x 至 3.16-rc 版本)

#问题现象
- 系统在高负载网络流量下，WiFi 连接中断或系统挂起。
- 内核日志报错：`mwifiex_cmd_timeout_func: Timeout cmd id`，随后出现 `usb_submit_urb failed` 及 `DNLD_CMD: host to card failed`。
- 进程 `khubd` 阻塞超过 120 秒，Call Trace 指向 `mwifiex_usb_resume` -> `rpm_resume` -> `schedule`。
- 系统关机时卡死在 "Now halt" 阶段无法正常关机。
- 触发内核警告：`WARNING: CPU: ... at net/wireless/sme.c:... __cfg80211_connect_result`。
- 日志显示 `PREP_CMD: FW is in bad state`。

#问题根因
1. 内存未初始化：驱动代码中 `tx_info/rx_info` 控制缓冲区缺少 `memset` 初始化，导致命令和数据路径上发生随机故障。
2. USB 自动挂起机制缺陷：`mwifiex_usb` 驱动及关联的 `btusb` 模块在支持 USB autosuspend 时，存在挂起/恢复流程的竞态条件或固件状态同步问题，导致设备无法正常唤醒或通信超时。

#解决方案
1. 应用补丁修复内存未初始化问题（Commit: d76744a93246eccdca1106037e8ee29debf48277）。
2. 应用补丁禁用驱动的 USB autosuspend 支持（Commit: adb07df）。
3. 针对 `btusb` 模块应用补丁禁用 Marvell 设备的 autosuspend（参考 Bug ID 69661 Comment 58/70）。
4. 应用补丁修复 `__cfg80211_connect_result` 相关警告（参考 Comment 65）。
5. 临时规避方法：在 `drivers/net/wireless/mwifiex/usb.c` 中移除 `.supports_autosuspend = 1`，或在 `btusb.c` 中针对 Marvell 设备调用 `usb_disable_autosuspend`。
