#标题
启用文件透明大页导致filemap_fault触发VM_BUG_ON_PAGE

#内核版本
5.5.3

#问题现象
- 系统日志报告 `kernel BUG at mm/filemap.c:2529!`
- 触发断言失败：`VM_BUG_ON_PAGE(page_to_pgoff(page) != offset)`
- 故障现场 Call Trace 显示：`filemap_fault` -> `ext4_filemap_fault` -> `__handle_mm_fault`
- 故障通常发生在编译等高负载场景，导致相关进程（如 cc1plus）挂起

#问题根因
启用了实验性选项 `CONFIG_READ_ONLY_THP_FOR_FS` 配合 `CONFIG_TRANSPARENT_HUGEPAGE_ALWAYS` 触发。在 `CONFIG_NUMA` 未开启的情况下，`khugepaged` 将文件页折叠为大页后，`filemap_fault` 在处理尾页时，`page_to_pgoff` 计算结果与预期的 `offset` 不匹配，导致 VM_BUG_ON 校验失败。

#解决方案
- 规避措施：关闭 `CONFIG_READ_ONLY_THP_FOR_FS` 或将 `CONFIG_TRANSPARENT_HUGEPAGE_ALWAYS` 改为 `CONFIG_TRANSPARENT_HUGEPAGE_MADVISE`。
- 官方修复：该问题已在 Linux 5.9 版本通过 commit `033b5d775511` ("mm/khugepaged: fix filemap page_to_pgoff(page) != offset") 修复。
