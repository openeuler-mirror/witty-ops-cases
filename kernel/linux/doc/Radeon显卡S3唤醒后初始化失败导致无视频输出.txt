#标题
Radeon显卡S3唤醒后初始化失败导致无视频输出

#内核版本
5.11.7

#问题现象
- 系统从 v5.10 升级至 v5.11 后，从 S3 挂起状态恢复时屏幕无视频输出。
- dmesg 日志显示内核 WARNING：`WARNING: CPU: 0 PID: 7973 at include/drm/ttm/ttm_bo_api.h:615 radeon_bo_unpin+0x47/0x60 [radeon]`。
- 显卡驱动初始化阶段报错：`radeon: ring test failed (scratch(0x15E8)=0xCAFEDEAD)`。
- 随后报错：`radeon: cp isn't working (-22)` 与 `failed initializing CP (-22)`。
- 用户空间报错：`radeon: The kernel rejected CS, see dmesg for more information (-16)`。

#问题根因
在 S3 唤醒流程中，Radeon 驱动（RV515/M54 GPU）在执行 `rv515_resume` 回调时，调用链 `radeon_resume_kms -> rv515_resume -> radeon_gart_table_vram_unpin -> radeon_bo_unpin` 触发了 TTM 层的 WARNING。随后显卡 CP (Command Processor) 初始化失败（ring test 无法通过），导致驱动无法正常恢复工作，进而造成无视频输出。

#解决方案
暂无修复补丁，开发者建议进行二分排查以定位引入问题的具体提交。
