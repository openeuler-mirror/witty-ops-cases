#标题
Acer ES1-533 写入 EFI 变量导致系统挂起

#内核版本
5.16.11

#问题现象
- 在 Acer ES1-533 笔记本上，执行写入或删除 EFI 变量的操作（如通过 `efibootmgr` 创建启动项或删除 `/sys/firmware/efi/efivars/` 下的文件）会导致系统死机/挂起，必须重启恢复。
- dmesg 日志中显示警告信息："PCI: Using host bridge windows from ACPI; if necessary, use 'pci=nocrs' and report a bug"。

#问题根因
BIOS 提供的 ACPI 主机桥窗口配置存在问题，导致内核在处理 PCI 资源分配时与固件交互异常，进而引发 EFI 变量写入操作时的系统挂起。

#解决方案
暂无官方补丁修复。可通过添加内核启动参数 `pci=nocrs` 规避该问题，该参数指示内核忽略 ACPI 提供的当前资源设置报告。
