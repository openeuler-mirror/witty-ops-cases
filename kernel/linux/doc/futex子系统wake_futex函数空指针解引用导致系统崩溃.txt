#标题
futex子系统wake_futex函数空指针解引用导致系统崩溃

#内核版本
3.16.0-4-amd64 (Debian 3.16.7-ckt11-1+deb8u6)

#问题现象
- 系统在运行 utserver 程序时频繁崩溃（每天一次或多次）。
- 内核日志报错："BUG: unable to handle kernel NULL pointer dereference at 0000000000000064"。
- 触发 WARNING 位于 kernel/futex.c:1093，调用栈显示进程为 utserver。
- Call Trace: wake_futex+0x48/0x80 <- futex_wake+0xff/0x120 <- do_futex <- SyS_futex。
- 系统触发 Oops (0010)，CPU 1 异常。

#问题根因
内核在处理 futex 唤醒逻辑时，`wake_futex` 函数内部访问了空指针（非法地址 0x64），导致内核崩溃。具体触发条件与 utserver 程序的 futex 操作相关，深层代码逻辑错误尚未在报告中明确分析。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）。
