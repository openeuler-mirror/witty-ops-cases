#标题
rtl8xxxu驱动在公共开放Wifi下误触发deauthentication导致连接中断

#内核版本
6.2.x - 6.4.10 (在 6.5.0 中修复)

#问题现象
- 使用 Realtek USB Wifi 适配器（如 rtl8188eu, rtl8188fu）连接公共开放 Wifi AP 时，能成功关联并获取 DHCP IP 地址。
- 发起首次 HTTP(S) 请求时，Wifi 连接立即断开，无法跳转到认证页面。
- 系统日志显示错误信息：`wlan1: deauthenticated from ... (Reason: 6=CLASS2_FRAME_FROM_NONAUTH_STA)`。

#问题根因
rtl8xxxu 驱动模块在处理公共开放 AP 的认证流程时，收到 Reason code 6 (CLASS2_FRAME_FROM_NONAUTH_STA) 后处理逻辑不当，错误地立即执行 deauthentication 操作断开连接，导致无法完成后续的网页认证交互。

#解决方案
升级内核至 6.5.0 或更高版本；若无法升级内核，可尝试编译安装第三方驱动源码（如 github.com/lwfinger/rtl8xxxu 2023年6月版本）作为临时规避方案。
