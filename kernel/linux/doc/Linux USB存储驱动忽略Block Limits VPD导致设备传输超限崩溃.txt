#标题
Linux USB存储驱动忽略Block Limits VPD导致设备传输超限崩溃

#内核版本
未明确

#问题现象
- 特定 USB 存储设备（缓冲区受限，如 64kB/128 blocks）在 Linux 下读写时发生崩溃。
- 抓包显示 Linux 主机发送的 SCSI Read/Write-10 命令请求块数（默认 240 blocks）超过设备硬件缓冲区限制。
- 设备固件已实现 Block Limits VPD page (0xB0)，在 Windows 10/11 下可正常识别并限制传输大小，但在 Linux 下未被查询。

#问题根因
Linux USB 存储驱动为了避免部分遗留设备在探测不存在的 VPD pages 时挂起，策略上选择不查询 Block Limits VPD page (0xB0)。这导致驱动无法获取设备的最大传输块限制，而是直接使用 USB 2.0 的默认 max_sectors 设置（240 blocks），致使传输请求超过设备缓冲区容量引发故障。

#解决方案
暂无内核代码修复（属于兼容性设计权衡）。可通过创建 udev 规则手动修改 `/sys/block/sdN/queue/max_sectors_kb` 或 SCSI 子系统下的 `max_sectors` 参数，限制最大传输块数以规避此问题。
