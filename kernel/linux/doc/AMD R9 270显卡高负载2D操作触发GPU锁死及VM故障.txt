#标题
AMD R9 270显卡高负载2D操作触发GPU锁死及VM故障

#内核版本
3.16-rc1

#问题现象
- 系统在执行高负载 2D 操作（如运行 Battle for Wesnoth 游戏或 4K 显示器窗口调整大小）时，GPU 发生锁死。
- 内核日志显示 `radeon 0000:01:00.0: ring 0 stalled for more than 10263msec` 以及 `GPU lockup`。
- 伴随 `AMD-Vi: Event logged [IO_PAGE_FAULT ...]` 错误。
- 系统尝试 GPU softreset，有时成功恢复，有时导致 X 冻结或系统挂起。

#问题根因
未明确具体代码根因。用户分析指出问题出现在 3.15 到 3.16-rc1 之间的代码变更，怀疑与 GPU VM 管理或 DMA 复制机制有关，导致在特定 2D 负载下触发 VM Fault。

#解决方案
开发者提供了针对 radeon 驱动 VM 管理的补丁（见附件 140871、140881）。用户反馈在后续的内核与 MESA 更新中该问题已被修复。
