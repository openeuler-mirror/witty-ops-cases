#标题
Xen客户机传递MSI-X设备时触发page fault导致内核崩溃

#内核版本
3.12

#问题现象
- 系统日志报错 "BUG: unable to handle kernel paging request at ffffc9000030600c"
- 触发 Oops (0003 [#1])，系统进入 kdb 调试状态
- 故障发生在 igb 驱动初始化过程中，调用栈显示 RIP 位于 `pci_enable_msix+0x35d/0x400`

#问题根因
在Xen客户机环境下，当向客户机传递带有MSI-X功能的PCI设备时，`pci_enable_msix` 函数内部访问了无效的内存地址，导致缺页异常。

#解决方案
开发者已提供修复补丁，计划合入内核版本 v3.13（预计包含在 v3.13-rc1 中）。
