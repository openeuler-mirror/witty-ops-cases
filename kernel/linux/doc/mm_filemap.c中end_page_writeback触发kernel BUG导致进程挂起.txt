#标题
mm/filemap.c中end_page_writeback触发kernel BUG导致进程挂起

#内核版本
5.11.1-2.g39e04be

#问题现象
- 内核日志报错 "kernel BUG at mm/filemap.c:1499!"，伴随 "invalid opcode: 0000 [#1]" 异常。
- 调用栈显示：RIP: end_page_writeback，路径为 iomap_finish_ioend -> xfs_end_io。
- 用户空间进程挂起，系统最终无响应，需硬复位恢复。
- 日志中伴随 "Bad page state in process" 及 "workqueue lockup" 错误。

#问题根因
根据开发者分析，该问题极可能是页缓存或回写状态跟踪存在 Bug。在 XFS 文件系统进行 I/O 结束处理时，`end_page_writeback` 函数触发了 `BUG()`。该问题在使用 NVMe/SSD 且文件系统填充率较高（>90%）的高负载场景下更容易触发。

#解决方案
暂无
