#标题
Btrfs压缩写入与Zram交互时触发内核空指针解引用

#内核版本
5.8.3 - 5.9.15 (报告涉及版本)

#问题现象
- 内核日志报错 "BUG: kernel NULL pointer dereference"，触发 Oops，系统可能变得不稳定。
- 故障地址通常为 0000000000000018 或 0000000000000000。
- RIP 寄存器指向 `compress_file_range` 函数（Btrfs 压缩路径）或 `obj_malloc` 函数（Zsmalloc 分配器）。
- 调用栈显示进程为 `kworker`，工作队列为 `btrfs-delalloc`，典型调用链为 `async_cow_start` -> `btrfs_work_helper` 或 `zs_malloc` -> `zram_bvec_rw`。
- 在 Zram 块设备上构建 Btrfs 文件系统并开启压缩，执行高负载写入（如 rsync、编译）时容易复现。

#问题根因
未明确。故障现场显示在 Btrfs 异步压缩写回流程中访问了空指针；亦有现场显示在 Zram 内存分配时发生空指针访问。具体代码缺陷位置尚未在报告中确认。

#解决方案
暂无官方修复。部分用户反馈内核 5.8.0 或 5.8.2 未出现此问题，建议尝试降级内核版本规避。
