#标题
amdgpu驱动PSP固件加载失败导致Ryzen 2500U启动无显示

#内核版本
5.2.14

#问题现象
- Lenovo Thinkpad E585 (Ryzen 2500U Raven) 在冷启动或重启时屏幕无显示，但系统可正常启动并允许 SSH 登录。
- dmesg 日志显示 amdgpu 驱动初始化失败，报错信息包括："psp command failed and response status is (0)"、"PSP load tmr failed!"、"PSP firmware loading failed"。
- 触发内核 WARNING：`WARNING: CPU: 5 PID: 679 at drivers/gpu/drm/amd/amdgpu/amdgpu_object.c:925`。
- GPU 初始化失败日志：`amdgpu 0000:05:00.0: Fatal error during GPU init`。

#问题根因
显卡 PSP (Platform Security Processor) 模块在初始化过程中命令执行失败（response status is 0），导致 TMR (Trusted Memory Region) 加载错误，进而导致 IP block <psp> 初始化失败（返回错误码 -22），最终引发 GPU 致命错误。

#解决方案
暂无官方修复方案。用户反馈通过完全关机并断电后重新冷启动可能恢复正常。
