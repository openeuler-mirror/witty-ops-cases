#标题
F2FS文件系统update_sit_entry访问非法地址导致内核崩溃

#内核版本
3.18.22

#问题现象
- 系统触发 kernel panic 并重启
- 内核日志报错："Unable to handle kernel paging request at virtual address 0000dead"
- 进程 wpa_supplicant 在执行 fsync 操作时触发异常
- 调用栈显示崩溃发生在 F2FS 模块的 update_sit_entry 函数中

#问题根因
内核在执行 F2FS 文件系统的 update_sit_entry 函数（位于 fs/f2fs/segment.c:718）时，尝试访问非法虚拟地址 0000dead，触发缺页异常。此时 PGD 表项为空，导致内核无法处理该地址请求。具体的指针错误来源未明确。

#解决方案
暂无
