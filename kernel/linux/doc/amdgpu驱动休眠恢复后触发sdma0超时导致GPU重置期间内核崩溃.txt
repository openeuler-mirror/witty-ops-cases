#标题
amdgpu驱动休眠恢复后触发sdma0超时导致GPU重置期间内核崩溃

#内核版本
5.10.0-7-amd64

#问题现象
- 系统从休眠恢复时无法正常唤醒，需硬重启
- 内核日志报错：`ring sdma0 timeout`，随后触发 GPU reset
- GPU 重置过程中发生 Kernel Oops：`BUG: unable to handle page fault for address: ffff99fd38018000`
- 崩溃调用栈显示：`kfd_gtt_sa_free+0x39/0x80 [amdgpu]` -> `stop_cpsch` -> `kgd2kfd_pre_reset` -> `amdgpu_device_gpu_recover`
- 错误类型为 `#PF: supervisor write access in kernel mode`，错误码 `error_code(0x0003) - permissions violation`

#问题根因
系统从休眠恢复时，amdgpu 驱动检测到 ring sdma0 超时并启动 GPU 恢复流程。在执行 `kgd2kfd_pre_reset` 进行预重置清理时，`kfd_gtt_sa_free` 函数尝试访问非法内存地址触发权限违规，导致内核崩溃。

#解决方案
暂无
