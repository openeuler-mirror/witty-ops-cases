#标题
MediaTek蓝牙驱动导致系统从挂起恢复时死机

#内核版本
6.11.x - 6.12.7 (受影响版本)

#问题现象
- 系统从挂起状态恢复时发生死机，无法正常唤醒。
- 表现为风扇转动、键盘LED亮起，但显示器保持黑屏，按下Caps Lock键无法切换LED状态，系统完全无响应。
- 该问题主要影响使用 MediaTek MT7922 无线网卡（蓝牙部分）的系统。

#问题根因
Git bisect 定位到引入问题的提交为 `ceac1cb0259de682d78f5c784ef8e0b13022e9d9`。根因是 `btusb` 驱动中针对 MediaTek 蓝牙设备的 ISO 接口处理逻辑存在缺陷，在系统挂起和恢复过程中，驱动对 USB ISO 接口的占用和释放条件判断不当，导致恢复流程卡死。

#解决方案
1. 临时规避：配置 systemd 服务，在系统挂起前执行 `rfkill block bluetooth`（或 `rfkill block all`），在恢复后执行 `rfkill unblock bluetooth`。
2. 正式修复：升级内核至 6.12.8 或更高版本，该版本已合并修复补丁 `f5c5661f02b5539d88aea8497f8d0835d165e945`，修正了 ISO 接口的判断条件。
