#标题
Radeon驱动在休眠时触发空指针解引用导致系统崩溃

#内核版本
3.19.3（评论中提及 4.0.1、4.0.4）

#问题现象
- 系统执行休眠操作时发生 Kernel Oops，导致休眠失败。
- 日志报错："BUG: unable to handle kernel NULL pointer dereference at 0000000000000008"。
- 崩溃点位于 `radeon_vm_bo_invalidate+0x73/0xa0 [radeon]`。
- 调用栈包含 `radeon_bo_move_notify`、`ttm_bo_cleanup_memtype_use`、`radeon_bo_unref` 等函数。

#问题根因
问题与内核配置选项 `CONFIG_HSA_AMD` 相关。当启用 HSA (Heterogeneous System Architecture) 支持时，Radeon 驱动在休眠流程中处理 Buffer Object 无效化操作时访问了非法内存地址，触发空指针解引用。经测试，禁用 `CONFIG_HSA_AMD` 后该崩溃不再复现。

#解决方案
暂无上游修复补丁。临时规避方案：重新编译内核并禁用 `CONFIG_HSA_AMD` 配置项。
