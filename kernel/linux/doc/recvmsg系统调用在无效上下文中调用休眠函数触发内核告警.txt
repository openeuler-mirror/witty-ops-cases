#标题
recvmsg系统调用在无效上下文中调用休眠函数触发内核告警

#内核版本
3.2（报告为 3.1 的回归）

#问题现象
- 系统日志中出现 "BUG: sleeping function called from invalid context" 告警
- 调用栈显示进程 "ip" 在执行 `__sys_recvmsg` 时，经由 `move_addr_to_user` -> `might_fault` 触发了 `__might_sleep` 检查
- 内核报告状态：in_atomic(): 0, irqs_disabled(): 0

#问题根因
这是一个自内核 3.1 版本引入的回归问题。在 `recvmsg` 系统调用的处理路径中，`move_addr_to_user` 函数调用了 `might_fault`，该行为触发了内核对非法休眠上下文的检查机制，导致内核抛出告警。

#解决方案
暂无
