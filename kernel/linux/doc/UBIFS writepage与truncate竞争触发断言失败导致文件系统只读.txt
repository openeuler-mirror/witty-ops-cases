#标题
UBIFS writepage与truncate竞争触发断言失败导致文件系统只读

#内核版本
5.16-rc5

#问题现象
- 内核日志报错：UBIFS assert failed: 0, in fs/ubifs/file.c:1499
- 文件系统自动切换为只读模式（switched to read-only mode, error -22）
- Call Trace 显示调用路径为：ubifs_releasepage -> invalidate_inode_page -> generic_fadvise

#问题根因
UBIFS 文件系统中 writepage 操作与 truncate 操作发生竞争。当 truncate 改变文件大小并使页面失效时，writepage 相关的处理逻辑出现异常状态，导致在 ubifs_releasepage 函数中触发断言失败（ubifs_assert(c, 0)）。

#解决方案
暂无（Bug 处于 NEW 状态，尚未合并修复补丁）
