#标题
KVM vhost线程尝试执行NX保护页面导致内核崩溃

#内核版本
3.10.0-327.el7.x86_64

#问题现象
- 服务器随机重启。
- 内核日志报错："kernel tried to execute NX-protected page - exploit attempt? (uid: 0)"。
- 触发 "BUG: unable to handle kernel paging request at ffff88181f354780"，错误码 Oops: 0011。
- 故障发生在进程 `vhost-196443` 中，RIP 指向非法地址 `ffff88181f354780`。

#问题根因
内核尝试执行不可执行（NX-protected）的内存页面。故障现场位于 `vhost_worker` 线程上下文，调用栈显示 `vhost_worker` -> `schedule` -> `dequeue_task`。RIP 寄存器指向非法地址，疑似发生函数指针损坏或堆栈破坏，导致 CPU 跳转到非法内存区域执行指令。

#解决方案
暂无。Bug 状态为 NEW，尚未确认具体修复方案。
