#标题
readahead_page_batch处理重试项引发空指针解引用导致系统死机

#内核版本
5.9.0-1 (Debian 5.8.10-1 起出现，5.9.0-1 仍存在)

#问题现象
- 内核报错 "BUG: kernel NULL pointer dereference, address: 0000000000000402"
- 系统变得无响应，偶发死机

#问题根因
内存管理子系统中的 readahead_page_batch 函数在处理 retry entries（重试条目）时存在缺陷，触发了空指针解引用。

#解决方案
已有修复补丁 "mm: fix readahead_page_batch for retry entries" 提交至 -mm 树，详见：https://lore.kernel.org/linux-fsdevel/20201103142852.8543-1-willy@infradead.org/
