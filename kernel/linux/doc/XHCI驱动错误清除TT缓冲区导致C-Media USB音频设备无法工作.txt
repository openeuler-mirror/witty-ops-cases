#标题
XHCI驱动错误清除TT缓冲区导致C-Media USB音频设备无法工作

#内核版本
>=5.2.0-rc3

#问题现象
- C-Media USB音频设备（ID 0d8c:000c）在内核升级后无法正常工作。
- 执行 speaker-test 测试时报错：`Unable to set hw params for playback: Input/output error`。
- dmesg 日志显示：`usb 1-2.4: 1:1: usb_set_interface failed (-110)`。

#问题根因
引入的故障 Commit 为 `ef513be0a9057cc6baf5d29566aaaefa214ba344` (`usb: xhci: Add Clear_TT_Buffer`)。
该补丁导致 xHCI 驱动在端点 0 发生 STALL 时尝试清除 TT（Transaction Translator）缓冲区。然而，设备发送的是“协议 STALL”（protocol stall，表示不支持该请求），而非“功能 STALL”。根据 USB 2.0 规范，STALL 并非协议错误，发送 STALL 本身即会清除 TT 状态，无需驱动额外清除 TT 缓冲区。驱动错误地执行清除操作导致了设备后续通信异常。

#解决方案
应用开发者提供的测试补丁（Bugzilla 附件 288293），禁止在端点 0 发生协议 STALL 时清除 TT 缓冲区。用户反馈应用该补丁后设备功能恢复正常。
