#标题
HID子系统处理大Usage ID按键导致键码超出KEY_MAX限制

#内核版本
3.8

#问题现象
特定游戏手柄（如 OnLive Universal Wireless Controller）的按键（如 Eject 键）无法正常使用，具体表现随内核版本不同而有所差异：
- 内核 3.8.12：按键未按下时持续上报事件，导致 CPU 占用过高。
- 内核 3.8.13：仅在按键按下时持续上报事件。
- 最新内核：忽略按键按下动作，仅在释放时上报 MSC_SCAN 事件，无有效键码（key code）产生。
根本现象为：Button Usage Page (0x09) 中 Usage ID 大于 0x40 的按键生成的键码值超过 KEY_MAX (0x2FF)，导致按键不可用。

#问题根因
在函数 `hidinput_configure_usage()` 中，针对 `HID_GD_JOYSTICK` 和 `HID_GD_GAMEPAD` 类型设备，键码计算公式为 `usage_ID - 1 + BTN_TRIGGER_HAPPY - 0x10`（即 `usage_ID + 0x2AF`）。
当设备报告的 Button Usage Page (0x09) 中的 Usage ID 值大于 0x40 时，计算出的键码值会超过输入子系统的最大键码限制 `KEY_MAX` (0x2FF)，导致按键映射失败或行为异常。
该问题由 commit f8e86d054686d9090549476ea2394aa04c614a93 ("HID: fix incorrect handling of devices with high button count") 引入。

#解决方案
暂无
