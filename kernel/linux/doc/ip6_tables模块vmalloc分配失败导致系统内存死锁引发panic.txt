#标题
ip6_tables模块vmalloc分配失败导致系统内存死锁引发panic

#内核版本
5.3.10

#问题现象
- 系统触发 Kernel panic，提示 "System is deadlocked on memory"。
- 内核日志显示 "Out of memory and no killable processes..."。
- 日志记录 vmalloc 分配失败："vmalloc: allocation failure: 536870912 bytes"。
- 调用栈显示异常路径：sctp_setsockopt -> ipv6_setsockopt -> do_ip6t_set_ctl -> xt_counters_alloc -> vzalloc。

#问题根因
在执行 setsockopt 系统调用时，ip6_tables 子系统中的 xt_counters_alloc 函数尝试通过 vzalloc 分配 536870912 字节（512MB）的内存。该大内存分配请求失败并触发 OOM killer，但由于无可杀进程导致系统判定为内存死锁，最终触发 panic。

#解决方案
暂无
