#标题
MMC/SD卡压力测试期间blk_rq_map_sg访问无效地址触发Kernel Oops

#内核版本
3.4.14

#问题现象
ARM 平台通过 USB 设备模式连接 Linux PC 进行 MMC/SD 卡读写，在持续 12 小时的压力测试（复制、比较、删除 50MB 文件）过程中触发 Kernel Oops。
具体表现如下：
- 系统日志打印 "Unable to handle kernel paging request at virtual address 00007f1a"。
- 触发 "Internal error: Oops: 17 [#1] PREEMPT SMP ARM"。
- 崩溃发生在进程 mmcqd/0，PC 指针位于 `blk_rq_map_sg+0x4c/0x30c`。
- 调用栈回溯：`blk_rq_map_sg` -> `mmc_blk_rw_rq_prep` -> `mmc_blk_issue_rw_rq` -> `mmc_blk_issue_rq` -> `mmc_queue_thread`。

#问题根因
内核在块层函数 `blk_rq_map_sg` 中试图访问无效的虚拟地址 `00007f1a`，导致内核分页错误。具体的深层原因（如内存损坏、指针计算错误或硬件稳定性问题）在报告中未明确分析。

#解决方案
暂无
