#标题
启动切换至colour dummy console时con_set_unimap空指针解引用引发Oops

#内核版本
3.13.0-32

#问题现象
- 系统启动过程中，日志显示 "Console: switching to colour dummy device 80x25" 后触发内核 Oops。
- 日志报错："BUG: unable to handle kernel NULL pointer dereference at 0000000000000138"。
- 崩溃位置：`con_set_unimap+0x32/0x290`。
- 触发进程为 `setfont`，调用链经过 `vt_ioctl` -> `tty_ioctl`。
- 故障导致启动过程中断。

#问题根因
内核函数 `con_set_unimap` 在处理 Unicode 映射设置时访问了空指针（地址偏移 0x138），导致 Oops。具体触发场景为系统启动时切换至 colour dummy console 设备，由 `setfont` 进程通过 ioctl 调用触发。

#解决方案
暂无
