#标题
NFS文件复制触发Soft Lockup导致树莓派3B+系统死锁

#内核版本
4.19.10

#问题现象
- 在 Raspberry Pi 3B+ 上使用 rsync 复制 NFS 共享文件时，系统完全死锁，终端冻结，无法通过 SSH 连接。
- 内核日志报错：`watchdog: BUG: soft lockup - CPU#2 stuck for 22s!`。
- 日志显示 NFS 服务端出现短暂无响应：`nfs: server ease.lan not responding, still trying`。
- 故障时 Call Trace 指向：`nfs_sb_active+0x98/0xf0`，调用链包含 `nfs_delegation_reap_unclaimed`、`nfs4_state_manager` 等。

#问题根因
故障发生在 NFS 状态管理线程处理回收操作时，CPU 在 `nfs_sb_active` 函数处卡死。用户测试发现当网络 I/O 带宽达到一定阈值（如 rsync bwlimit 超过 10000）时触发，推测可能与 `lan78xx` 网络驱动或总线饱和有关，但报告中未明确最终根因。

#解决方案
暂无官方修复。可通过限制文件传输带宽规避，例如在 rsync 命令中添加 `--bwlimit=9000` 参数。
