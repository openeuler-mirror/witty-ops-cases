#标题
sdhci-pci驱动在ASPM L1启用时导致SD卡读卡器超时无法工作

#内核版本
3.10-rc3（问题持续至3.11）

#问题现象
- Acer Aspire One 756 笔记本的 SD 卡插槽无法正常工作
- 内核日志报错 "mmc0: Timeout waiting for hardware interrupt"
- CMD52 命令返回 -110 (ETIMEDOUT)，sdhci 寄存器转储显示 Int stat 为 0x00000000

#问题根因
BIOS 默认启用了 ASPM L1 状态，并通过 FADT/_OSC 禁止操作系统修改 ASPM 配置。Linux 遵循此规定保留了 BIOS 的 ASPM L1 设置，但 sdhci-pci 驱动程序在 ASPM L1 状态下无法正确处理硬件中断（相比之下 Windows 驱动在此状态下可正常工作），导致通信超时。该问题由 commit 3c076351c4027a56d5005a39a0b518a4ba393ce2 引入。

#解决方案
临时规避方法：通过命令 `setpci -s 00:1c.2 0x50.B=0x41` 禁用 ASPM L1 状态。内核补丁修复暂无（Bug状态为NEW）。
