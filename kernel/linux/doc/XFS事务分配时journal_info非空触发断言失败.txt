#标题
XFS事务分配时journal_info非空触发断言失败

#内核版本
linux v5.12-rc2 + xfs-5.12-fixes-1

#问题现象
- 运行 xfstests 测试集（如 generic/013、g/051、g/068 等）时触发内核 WARNING。
- 内核日志显示断言失败：XFS: Assertion failed: current->journal_info == NULL, file: fs/xfs/xfs_trans.h, line: 288。
- 故障调用栈显示路径为：xfs_file_ioctl -> xfs_ioc_fsbulkstat -> xfs_bulkstat_one_int -> destroy_inode -> xfs_inactive -> xfs_trans_alloc。

#问题根因
在 `xfs_trans_alloc` 函数调用 `xfs_trans_set_context` 设置事务上下文时，断言检查 `current->journal_info == NULL` 失败。这表明在尝试分配新事务时，当前进程的 `journal_info` 字段非空，意味着进程正处于不应存在的活跃事务上下文中，或者在 `xfs_inactive` 处理路径下事务状态管理出现异常。

#解决方案
暂无
