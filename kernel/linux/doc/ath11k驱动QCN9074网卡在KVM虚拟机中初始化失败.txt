#标题
ath11k驱动QCN9074网卡在KVM虚拟机中初始化失败

#内核版本
6.3, 5.15.120

#问题现象
- QCN9074 网卡在 KVM 虚拟机中无法正常工作，驱动 probe 失败。
- 直通模式下启动时报错：`failed to set pcie link register`，`BHI offset: 0xffffffff is out of range`，最终报错 `probe failed with error -22`。
- 若宿主机先加载驱动，虚拟机中加载会超时，报错 `failed to receive control response completion` 及 `failed to connect to HTT: -110`。
- 尝试应用相关补丁时，触发内核 WARNING：`WARNING: CPU: 1 PID: 960 at free_irq+0x343/0x390`。

#问题根因
驱动在虚拟化环境中无法正确设置 PCIe 链路寄存器或完成 MHI (Modem Host Interface) 初始化。日志显示 `BHI offset` 读取值为异常的 `0xffffffff`，且 `failed to set sysclk: -110`，表明硬件寄存器访问或链路训练在 VM 环境下存在障碍，可能与虚拟机 IOMMU 配置或内存映射机制有关。

#解决方案
暂无官方修复补丁。用户发现通过在 QEMU 中启用 `intremap` 和 `caching_mode` 参数可以促使驱动初始化成功，但创建的 AP 接口可能仍不可见。
