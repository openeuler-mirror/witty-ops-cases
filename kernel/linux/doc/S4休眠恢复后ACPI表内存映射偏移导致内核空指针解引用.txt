#标题
S4休眠恢复后ACPI表内存映射偏移导致内核空指针解引用

#内核版本
v5.10 至 v6.9 (测试范围)，故障日志版本 6.1.0-21-amd64

#问题现象
- 系统从 S4 (hibernation) 平台恢复时触发内核 Oops，显示 `BUG: kernel NULL pointer dereference, address: 0000000000000008`。
- 内核日志中出现 ACPI 解析错误：`ACPI Error: Null stack entry` 及 `\_PR.C003._PPC evaluation failed`。
- Call Trace 指向 `acpi_ex_resolve_multiple` 和 `acpi_cpufreq_cpu_init`。
- 检查 `/sys/firmware/acpi/tables/SSDT5` 发现内容被错误替换为 CDAT 表数据。

#问题根因
固件 Bug。在 S4 平台恢复过程中，固件将 ACPI 表（IVRS, SSDT, CRAT, CDIT）映射到了比冷启动时高 4KB（一个页面）的物理内存地址，但内核仍保留并使用冷启动时获取的旧地址指针。这导致内核读取到错误的内存区域，将无效数据解析为 ACPI 表结构，进而引发 AML 解释器错误和空指针解引用。

#解决方案
暂无内核修复方案。报告者已向 Lenovo 厂商反馈此固件问题。
