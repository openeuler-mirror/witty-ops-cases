#标题
Realtek ALC295 编解码器耳机音频输出存在爆裂声

#内核版本
4.5（报告版本），受影响版本包括 4.12 至 5.9

#问题现象
- 使用 Realtek ALC295 声卡芯片的设备在 Linux 系统下，耳机音频输出伴随明显的爆裂声或噪音。
- 问题通常在冷启动或系统从挂起状态恢复后出现。
- 若先启动 Windows 系统再重启进入 Linux，音频工作正常（硬件状态已被 Windows 正确初始化）。

#问题根因
Linux 内核驱动 `snd-hda-codec-realtek` 在初始化 Realtek ALC295 编解码器时，未正确设置特定的 COEF（系数）寄存器值。经用户对比工作状态与非工作状态的寄存器差异，发现 `Coeff 0x67` 的值在故障状态下为 `0xf287`，而正常工作状态下应为 `0x3000`。驱动缺少针对该型号的特定初始化序列，导致硬件状态异常。

#解决方案
- **临时规避方案**：使用 `hda-verb` 工具手动修正寄存器值，命令如下：
  `hda-verb /dev/snd/hwC0D0 0x20 SET_COEF_INDEX 0x67`
  `hda-verb /dev/snd/hwC0D0 0x20 SET_PROC_COEF 0x3000`
  用户可将上述命令配置为启动脚本或放置在 `/lib/systemd/system-sleep` 中以在恢复后执行。
- **正式修复**：开发者 Takashi Iwai 提供了修复补丁（Test fix patch），该补丁已合并至 sound git tree 并推送至 Linus 主线，后续回溯至稳定版内核。
