#标题
挂载特制镜像时BTRFS extent_io_tree_panic触发kernel BUG

#内核版本
5.15.57, 5.18.14

#问题现象
- 挂载特制 BTRFS 镜像失败，触发 kernel BUG 导致系统异常。
- 内核日志显示：`kernel BUG at fs/btrfs/extent_io.c:688!`，伴随 `invalid opcode: 0000 [#1]`。
- 关键错误信息：`BTRFS error: insert state: end < start ...` 以及 `BTRFS critical: panic in extent_io_tree_panic:688: locking error: extent tree was modified by another thread while locked`。
- Call Trace：`extent_io_tree_panic` -> `set_extent_bit` -> `add_extent_mapping` -> `read_one_chunk` -> `btrfs_read_chunk_tree` -> `open_ctree`。

#问题根因
在挂载特制镜像时，BTRFS 子系统在 `read_one_chunk` 流程中调用 `add_extent_mapping` 插入 extent mapping。由于镜像数据构造特殊，导致 `insert_state` 函数检测到 extent state范围错误，具体表现为 `end < start`，进而触发 `extent_io_tree_panic` 并执行 `BUG()`。

#解决方案
暂无（用户反馈在 5.19-rc8 版本中无法复现该问题）。
