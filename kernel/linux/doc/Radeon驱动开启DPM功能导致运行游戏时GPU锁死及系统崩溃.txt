#标题
Radeon驱动开启DPM功能导致运行游戏时GPU锁死及系统崩溃

#内核版本
3.11.5, 3.12-rc4, 3.12.0

#问题现象
- 启动游戏（如 Team Fortress 2）时系统发生 Hard lockup。
- 内核日志显示 "radeon 0000:00:01.0: GPU lockup CP stall for more than 10000msec"。
- 出现 "[TTM] Buffer eviction failed" 告警及 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000088" 错误。
- 系统无响应，鼠标可移动但无法交互，需通过 SysRq 组合键重启。
- 触发条件：内核启动参数包含 `radeon.dpm=1`。

#问题根因
Radeon 驱动的 DPM (Dynamic Power Management) 功能在 Trinity APU 上存在稳定性问题。二分查找定位到引入问题的 commit 为 `6d15ee492809d38bd62237b6d0f6a81d4dd12d15`。测试表明，开启 `enable_mg_clock_gating`、`enable_gfx_dynamic_mgpg` 或 `override_dynamic_mgpg` 选项会导致 GPU 软复位或锁死；若同时开启 `uvd_dpm` 则会导致更严重的桌面不可用问题。

#解决方案
临时规避：移除内核启动参数 `radeon.dpm=1`（但会导致图形性能下降）。
修复补丁：开发者提供了针对 Trinity DPM 功能调整的补丁（如附件 114681），禁用了导致不稳定的 `enable_mg_clock_gating`、`enable_gfx_dynamic_mgpg` 和 `override_dynamic_mgpg` 功能以恢复稳定性。
