#标题
ACPI 解析 USB4 设备路径错误导致启动阶段硬件死锁

#内核版本
6.6 LTS, 6.12

#问题现象
- 系统在启动过程中（驱动初始化/模块加载阶段）频繁发生硬件死锁，表现为完全无响应。
- 日志中出现 ACPI 错误：`ACPI Error: AE_NOT_FOUND, While resolving a named reference package element - \_SB_.PCI0.GPP2.UP00.DP00.U4UP.U4P3.UH10`。
- 故障具有特殊性：断电重启无法恢复系统，必须进行 BIOS Recovery/Reflash 才能修复，表明硬件或 NVRAM 进入异常状态。

#问题根因
BIOS ACPI 表中引用了缺失或定义错误的设备路径，具体指向 GPP2 桥上的 USB4 Upstream Port (UP00)。当操作系统尝试初始化 USB4/UCSI 子系统时触发异常，导致硬件锁定。

#解决方案
暂无内核级修复方案。用户尝试过 `nomodeset`、`pcie_aspm=off` 等内核参数及禁用 BIOS VT-d/USB 选项均无效。临时恢复手段为重刷 BIOS。目前正在测试黑名单 `ucsi_acpi` 模块以规避问题。
