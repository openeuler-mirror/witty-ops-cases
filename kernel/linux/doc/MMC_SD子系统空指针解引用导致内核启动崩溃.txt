#标题
MMC/SD子系统空指针解引用导致内核启动崩溃

#内核版本
6.0.0-rc2

#问题现象
- 设备在内核启动过程中发生崩溃，无法正常启动。
- 日志显示 "Unable to handle kernel NULL pointer dereference at virtual address 00000008"。
- 该问题为回归问题，v5.19 版本运行正常，升级至 6.0.0-rc2 后出现故障。
- 在后续版本 6.0.0-rc3-next 中，表现为无法识别 mmcblk 设备。

#问题根因
MMC/SD 子系统在启动过程中触发了空指针解引用，导致内核无法处理该虚拟地址访问。

#解决方案
应用 Isaac J Manjarres 提交的修复补丁（链接：https://lore.kernel.org/lkml/20220818172852.3548-1-isaacmanjarres@google.com/）。根据测试反馈，该补丁需结合 Zhen-Lei 的补丁共同使用才能成功修复启动问题。
