#标题
卸载qca_nss_dp模块时因Bad page state触发内核崩溃

#内核版本
5.4.164

#问题现象
- 执行 `rmmod qca_nss_dp` 卸载 QCA NSS 数据通路模块时触发内核 BUG
- 内核日志报错 "BUG: Bad page state in process rmmod"，指出页面标志位异常（flags: 0x200(slab)）
- 调用栈显示异常路径为 `syn_dp_cfg_tx_cleanup_rings` -> `dma_free_attrs` -> `__free_pages_ok`

#问题根因
在卸载 qca_nss_dp 驱动过程中，`syn_dp_cfg_tx_cleanup_rings` 函数调用 `dma_free_attrs` 释放内存时，内核检测到页面状态异常：被释放的页面仍带有 slab 标志（PAGE_FLAGS_CHECK_AT_FREE），这通常意味着内存被错误释放或存在双重释放，导致页面状态不一致。

#解决方案
暂无
