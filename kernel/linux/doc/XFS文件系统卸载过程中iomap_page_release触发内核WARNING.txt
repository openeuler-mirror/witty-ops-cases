#标题
XFS文件系统卸载过程中iomap_page_release触发内核WARNING

#内核版本
5.11.0-rc6+

#问题现象
- 运行 xfstests generic/475 测试用例时触发内核 WARNING。
- 日志显示 XFS 文件系统先发生 I/O 错误（`xfs_ialloc_read_agi() returned error -5`），随后触发强制关机。
- 在卸载过程中触发告警：`WARNING: CPU: 1 PID: 11596 at fs/iomap/buffered-io.c:79 iomap_page_release+0x220/0x290`。
- 调用栈显示异常路径为：`iomap_invalidatepage` -> `truncate_cleanup_page` -> `truncate_inode_pages_range` -> `evict` -> `generic_shutdown_super`。
- 调试信息显示页面状态异常：`page dumped because: iomap_page_release:PageUptodate`。

#问题根因
XFS 文件系统因底层 I/O 错误触发强制关机，在后续的文件系统卸载流程中，内核尝试通过 `truncate_inode_pages_range` 清理页面，此时 `iomap_page_release` 函数检测到页面仍处于 `PageUptodate` 状态，触发了内核警告。

#解决方案
暂无
