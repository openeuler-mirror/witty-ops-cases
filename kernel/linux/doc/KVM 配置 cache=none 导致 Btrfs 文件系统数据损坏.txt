#标题
KVM 配置 cache=none 导致 Btrfs 文件系统数据损坏

#内核版本
4.9、4.19、5.3.12

#问题现象
- KVM 虚拟机使用 Btrfs 格式磁盘时，设置 cache=none 或 directsync 模式会导致数据损坏
- 系统日志 dmesg 中出现 BTRFS 错误信息："bdev /dev/sde1 errs: wr 0, rd 0, flush 0, corrupt 2137"
- 日志中出现校验失败警告："csum failed root 256 ino 260 ... expected csum ..."

#问题根因
未明确

#解决方案
暂无官方修复。根据测试反馈，使用其他缓存模式（非 cache=none 或 directsync）可规避该问题。
