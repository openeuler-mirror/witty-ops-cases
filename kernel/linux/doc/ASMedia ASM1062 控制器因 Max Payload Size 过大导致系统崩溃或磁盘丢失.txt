#标题
ASMedia ASM1062 控制器因 Max Payload Size 过大导致系统崩溃或磁盘丢失

#内核版本
受影响版本为允许设置 512 字节 MPS 的内核版本（如 commit 8a3ebd8de328 之后）；修复版本包括 5.15, 5.14.6, 5.13.19, 5.10.67, 5.4.148, 4.19.207 等。

#问题现象
- 系统启动或运行时触发 External Abort 并导致 Kernel Panic，日志显示 "Unhandled External Abort received on 0x80000000 at EL3"。
- 高 I/O 负载下 SATA 磁盘出现 I/O 超时，日志显示 "failed command: WRITE FPDMA QUEUED" 和 "Emask 0x4 (timeout)"，随后链路重置失败，磁盘被禁用。
- 部分平台（如 ASRockRack TRX40D8-2N2T）可能出现系统完全冻结。

#问题根因
ASMedia ASM1062 (及 ASM1061) SATA 控制器存在硬件缺陷，无法正确处理大于等于 512 字节的 Max Payload Size (MPS)。当内核（如 commit 8a3ebd8de328）将 PCIe MPS 配置为 512 字节时，触发该硬件缺陷导致控制器异常。

#解决方案
升级内核至包含补丁 `b12d93e9958e028856cbcb061b6e64728ca07755` 的版本（Linux 5.15 及相关稳定版），该补丁通过 quirk 机制强制将 ASM1062 的 Max Payload Size 限制为 256 字节。
