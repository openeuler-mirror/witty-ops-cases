#标题
使用 traffic control 配置 SFQ 队列规则导致内核崩溃

#内核版本
4.18.5（报告中还涉及 4.18.8、4.14.71、5.3.10 等版本）

#问题现象
- 执行 `tc qdisc add dev <interface> root handle 1: sfq` 命令配置流量控制后，进行大文件网络传输（如 NFS、SCP）时触发内核崩溃。
- 故障表现为系统死机、冻结，部分情况下屏幕打印 Oops 信息，部分情况下无任何日志输出直接死锁。
- 移除 SFQ 配置或使用默认 qdisc 配置时，文件传输正常，系统稳定。

#问题根因
未明确（Bug 状态为 NEW，报告中未提供具体的代码级根因分析，仅确认与 SFQ 及特定内核版本流量控制逻辑有关）。

#解决方案
暂无官方修复。规避方法：取消使用 SFQ 队列规则，或升级/降级至不受影响的内核版本（如报告提及 4.19 或 4.18.10+ 在部分场景下正常，但 5.x 版本仍有复现）。
