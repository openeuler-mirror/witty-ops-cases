#标题
挂载特制reiserfs镜像导致cleanup_bitmap_list无效内存访问引发内核崩溃

#内核版本
4.18（日志显示为 4.18.0-rc1+）

#问题现象
- 挂载特制的 reiserfs 文件系统镜像时触发内核崩溃。
- 日志报错 "BUG: unable to handle kernel paging request at ffffc90000e77000"。
- 触发 Oops，RIP 寄存器指向 cleanup_bitmap_list.isra.11+0x1ca。
- 文件系统层面报错 "node level 0 does not match to the expected one -1" 及 "invalid format found in block 0"。

#问题根因
挂载过程中，ReiserFS 模块在处理日志位图节点时，传入 cleanup_bitmap_list() 函数的 `bn` 指针无效（指向非法内存地址）。内核在 `fs/reiserfs/journal.c` 第 186 行附近尝试释放该节点内存（kfree(bn->data) 或 kfree(bn)）时，触发缺页异常导致崩溃。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
