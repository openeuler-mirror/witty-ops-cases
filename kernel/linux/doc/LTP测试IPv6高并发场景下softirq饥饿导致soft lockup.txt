#标题
LTP测试IPv6高并发场景下softirq饥饿导致soft lockup

#内核版本
5.10.0

#问题现象
- 在 aarch64 架构虚拟机上运行 LTP 测试用例（如 tcp6-multi-sameport02）时触发 soft lockup。
- 内核日志报错："BUG: soft lockup - CPU#0 stuck for 22s!"，进程 ns-tcpserver 无响应。
- 日志中出现 "TCP: request_sock_TCPv6: Possible SYN flooding on port 1025" 警告。
- 调用栈显示卡在 `virtqueue_add_outbuf` 函数，调用链涉及 `virtio_net`、`tcp_tasklet_func` 及 `__do_softirq`。

#问题根因
在高负载网络发包场景下，softirq（特别是 TCP tasklet 和网络发送处理）长时间占用 CPU 资源，导致用户态进程（如执行 sendto 系统调用的进程）在开启 bottom half 后无法获得 CPU 执行机会，从而触发 soft lockup 检测机制。属于 softirq 饥饿问题。

#解决方案
暂无
