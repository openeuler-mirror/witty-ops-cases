#标题
升级至6.3.9后nf_tables引发随机Kernel Panic

#内核版本
6.3.9, 6.4.12, 6.4.15

#问题现象
- 系统在运行 24 至 48 小时后随机触发 Kernel Panic。
- 日志显示 "BUG: unable to handle page fault for address: 00000000352aa941"。
- 寄存器错误信息："#PF: supervisor write access in kernel mode" 与 "#PF: error_code(0x0002) - not-present page"。
- 故障现场日志显示与 nft (netfilter/tables) 模块相关。

#问题根因
Netfilter 维护者确认问题由 commit bdace3b1a51887211d3e49417a18fdbd315a313b 引入，该提交导致 nf_tables 在处理 bound set/chain 时触发异常。

#解决方案
开发者提供了修复补丁 commit 26b5a5712eb85e ("netfilter: nf_tables: add NFT_TRANS_PREPARE_ERROR to deal with bound set/chain")。但用户反馈在包含该修复的 6.4.15 版本上仍复现崩溃，问题修复状态暂未明确。
