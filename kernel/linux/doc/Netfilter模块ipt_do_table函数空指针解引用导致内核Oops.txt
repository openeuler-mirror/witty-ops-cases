#标题
Netfilter模块ipt_do_table函数空指针解引用导致内核Oops

#内核版本
5.15.108

#问题现象
- 内核日志报错 "BUG: kernel NULL pointer dereference, address: 0000000000000030"，触发 Oops。
- RIP 寄存器指向 `ipt_do_table+0x230/0x6f0`，RAX 寄存器值为 0。
- 故障发生在 `ksoftirqd` 进程处理网络软中断时，调用栈显示经过 `nf_hook_slow` -> `ip_rcv` -> `br_handle_frame`。
- 在高流量（约 20k 包/秒）且配置大量 iptables 规则的环境下复现，IPv4 和 IPv6 协议栈均有出现。

#问题根因
在 `ipt_do_table` 函数处理数据包规则匹配过程中，访问了空指针（尝试读取 RAX+0x30 处的数据，而 RAX 为 0）。具体触发逻辑可能与高并发网络流量下的规则处理状态有关，但报告中未明确具体的代码逻辑缺陷或数据结构损坏原因。

#解决方案
暂无。Bug 状态为 NEW，尚未有开发者提供修复补丁或临时规避方案。
