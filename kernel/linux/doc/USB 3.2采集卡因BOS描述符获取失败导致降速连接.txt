#标题
USB 3.2采集卡因BOS描述符获取失败导致降速连接

#内核版本
6.11.0-24

#问题现象
- USB 3.2 采集卡（如 Elgato 4K X）在 Linux 下被识别为 SuperSpeed (5Gbps) 而非 SuperSpeed Plus (10Gbps)。
- dmesg 日志显示设备尝试以 Gen 2 速度枚举时报错："unable to get BOS descriptor or descriptor too short" 和 "can't read configurations, error -71"。
- 设备随后重置并回退到 Gen 1 (5Gbps) 模式，导致带宽受限。
- 同一端口连接其他 10Gbps 设备（如 NVMe 转接卡）工作正常。

#问题根因
特定 USB 采集卡固件存在缺陷，在 Gen 2 (10Gbps) 速度下无法正确响应 Linux 内核发起的 BOS (Binary Device Object Store) 描述符请求。设备在请求 BOS 描述符时超时或出错，导致内核无法读取配置描述符，进而触发重置并降级连接速度。

#解决方案
上游内核已合并相关修复补丁。
参考 patch：https://lore.kernel.org/linux-usb/20251207012059.7899-1-johannes.bruederl@gmail.com/
临时规避方案：针对特定 Vendor ID 打补丁跳过 BOS 描述符获取。
建议：升级至包含修复的新版本内核。
