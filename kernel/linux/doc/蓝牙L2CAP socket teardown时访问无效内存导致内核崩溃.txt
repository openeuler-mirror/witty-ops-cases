#标题
蓝牙L2CAP socket teardown时访问无效内存导致内核崩溃

#内核版本
5.5.2-arch1-1 (日志中复现版本含 5.5.4, 5.5.8, 5.4.22)

#问题现象
- 系统随机崩溃，内核日志显示 "general protection fault: 0000 [#1]"。
- 故障现场 Call Trace 指向：`hci_rx_work` -> `l2cap_conn_del` -> `l2cap_sock_teardown_cb` -> `sock_def_wakeup` -> `__wake_up_common`。
- 寄存器 R12 值为 `dead0000000000e8`，显示访问了已被标记删除的内存区域。
- 部分故障伴随 "watchdog: BUG: soft lockup" 告警。

#问题根因
在蓝牙 L2CAP 连接断开处理流程中，`l2cap_sock_teardown_cb` 回调函数调用 `sock_def_wakeup` 唤醒等待队列时，`__wake_up_common` 函数访问了无效地址。寄存器 R12 包含 `dead0000000000e8`（内核 list poison 标记），表明 socket 的等待队列项在被访问前已被释放或链表结构已损坏，属于典型的 Use-After-Free 或并发竞态问题。

#解决方案
暂无
