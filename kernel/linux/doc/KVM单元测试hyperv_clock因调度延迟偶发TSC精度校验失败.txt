#标题
KVM单元测试hyperv_clock因调度延迟偶发TSC精度校验失败

#内核版本
4.18.0-348.7.1.el8_5.x86_64, 6.4.0-rc5

#问题现象
- 执行 kvm-unit-tests 中的 hyperv_clock 测试用例时偶现失败
- 测试日志显示 "suspecting drift on CPU 0? delta = 75, acceptable [0, 45)"
- 最终输出 "FAIL: TSC reference precision test"

#问题根因
测试逻辑在 1 秒内计算 MSR 时钟与 TSC 页面的偏差，并期望该偏差在后续 2 秒内保持在测量范围内。当系统非完全空闲时，任务可能被调度出去或迁移至其他 CPU，导致偏差超出预期范围，从而触发测试失败。这属于测试用例对调度行为过于敏感导致的不稳定性，而非内核功能缺陷。

#解决方案
开发者建议修改测试代码 x86/hyperv_clock.c 中的判定阈值（如将 max_delta * 3 / 2 放宽至 max_delta * 1024），或将失败改为警告。目前尚未合并修复补丁。
