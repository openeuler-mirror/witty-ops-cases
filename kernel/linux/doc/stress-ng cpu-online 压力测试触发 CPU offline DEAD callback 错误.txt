#标题
stress-ng cpu-online 压力测试触发 CPU offline DEAD callback 错误

#内核版本
5.15 (pre-rc1, commit d4439a1189f93d0ac1eaf0197db8e6b3e197d5c7)

#问题现象
- 在 SMP 系统上运行 `stress-ng --cpu-online 0 -t 15 --pathological` 时触发内核 WARNING。
- 内核日志显示 "DEAD callback error for CPU6"。
- 调用栈指向 `kernel/cpu.c:1163` 处的 `_cpu_down+0x20a/0x3a0`。
- 该问题在 5.13 版本未复现，属于 5.15 版本引入的回归。

#问题根因
在 CPU offline 过程中，`_cpu_down` 函数检测到 DEAD callback 错误。具体的技术根因尚未在报告中明确，但确认与 CPU hotplug（热插拔）机制相关。

#解决方案
暂无（Bug 状态为 NEW）。
