#标题
Type-C子系统初始化时触发空指针解引用导致系统挂起

#内核版本
5.17.0-rc1-00032-gdd81e1c7d5fb

#问题现象
- 系统在启动过程中挂起，无法正常登录。
- 内核日志报错：BUG: kernel NULL pointer dereference, address: 0000000000000008。
- Call Trace 显示崩溃路径为：typec_link_ports -> component_master_add_with_match。
- 受影响机型包括 Dell XPS 13 9360 和 Dell XPS 13 9310。

#问题根因
在 `ucsi_init_work` 工作队列处理 Type-C 端口注册时，`typec_link_ports` 函数调用 `component_master_add_with_match` 过程中，访问了空指针。具体表现为寄存器 RDX 为 0，指令尝试读取 RDX+0x8 处的内存，导致页面错误。

#解决方案
报告者验证了特定补丁（Bug ID 215529 附件 300314）可修复此问题，使系统恢复正常启动。
