#标题
BTRFS挂载特制镜像触发delayed-ref模块kernel BUG导致sync挂起

#内核版本
5.0.0

#问题现象
- 挂载特制的 BTRFS 镜像并执行 rename 操作后，触发 kernel BUG，系统在执行 sync 时挂起。
- 内核日志显示：`kernel BUG at fs/btrfs/delayed-ref.c:486!`，伴随 `invalid opcode: 0000 [#1]` 错误。
- 触发崩溃的 Call Trace 为：`update_existing_head_ref+0x18e/0x1c0` <- `add_delayed_ref_head` <- `btrfs_add_delayed_tree_ref` <- `btrfs_free_tree_block` <- `__btrfs_cow_block`。
- 日志中同时出现元数据损坏告警：`BTRFS critical (device sdb): corrupt leaf: root=1 block=29417472 slot=6, bad key order`。

#问题根因
该故障由特制的模糊测试镜像触发，涉及多个因素。根据开发者分析，该 Bug 仅影响开启了 `MIXED_GROUP` 功能的 BTRFS 文件系统（属于小众特性）。在处理损坏的元数据时，内核代码路径在 `update_existing_head_ref` 函数中触发了 BUG()。

#解决方案
暂无（Bug 状态为 NEW，开发者已确认问题复现但未提供修复补丁）。
