#标题
PHY状态机触发机制变更导致TQM5200S板卡网卡链路检测失败

#内核版本
5.0 (始于 4.8.8)

#问题现象
- TQM5200S 开发板上网卡 eth0 在内核 4.8.8 及以上版本无法工作，回退至 4.8.7 版本正常。
- 执行 ethtool eth0 显示 "Speed: Unknown!", "Duplex: Unknown! (255)", "Link detected: no"。

#问题根因
内核引入 commit a148a818df84f90e6dcb09d36ded7a3ee56e1ebd (net: phy: Trigger state machine on state change and not polling) 后，在 `phy_start()` 函数中增加了 `phy_trigger_machine()` 调用。该变更旨在解决中断模式下 PHY 状态机不运行的问题，但在 TQM5200S 平台上导致 PHY 状态机运行异常，无法正确检测链路状态。

#解决方案
暂无官方修复。临时验证方案：移除 `drivers/net/phy/phy.c` 中 `phy_start()` 函数内的 `phy_trigger_machine()` 调用可恢复正常。
