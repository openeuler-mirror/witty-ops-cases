#标题
XFS读取损坏符号链接时触发空指针解引用导致内核Oops

#内核版本
4.17（日志显示为 4.17.0-rc7）

#问题现象
- 挂载畸变（fuzzed）XFS镜像并执行readlink操作时，触发内核崩溃。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000000"。
- RIP 寄存器指向 strlen 函数，Call Trace 显示调用链为 readlink_copy -> vfs_readlink -> do_readlinkat。

#问题根因
在对畸变XFS文件系统镜像上的符号链接执行 readlink 操作时，由于文件系统元数据损坏，内核在 readlink_copy 函数中调用 strlen 处理链接目标路径时传入了 NULL 指针，导致发生空指针解引用异常。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
