#标题
nf_nat_redirect模块空指针解引用导致内核panic

#内核版本
4.0.4, 4.1.6, 4.2.5

#问题现象
- 激活 NAT 规则并访问 WiFi（TP-Link W8960N 路由器）时触发内核崩溃。
- 系统日志显示 "nf_nat_redirect: unable to handle kernel NULL pointer dereference"。

#问题根因
nf_nat_redirect 模块在处理网络地址转换规则时发生空指针解引用，导致系统 panic。

#解决方案
该问题已有修复补丁提交，链接为 https://patchwork.ozlabs.org/patch/540946/，建议升级至包含该修复的内核版本。
