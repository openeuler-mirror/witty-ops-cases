#标题
kswapd0 进程在 swappiness 为 0 时持续占用 100% CPU

#内核版本
3.13.0-30

#问题现象
- 系统中 kswapd0 进程持续占用 100% CPU 资源。
- 系统内存 4GB，Swap 4GB，swappiness 参数设置为 0。
- 运行内存密集型进程（占用约 1GB 内存）时，尽管仍有约 1GB 可用物理内存，kswapd0 依然异常活跃。
- 执行 swapoff -a 可暂时缓解 CPU 占用问题，但随后可能导致进程被杀死。

#问题根因
未明确。用户推测 swappiness 设为 0 时的交换逻辑未按预期工作（即未仅在物理内存耗尽时使用 Swap），导致 kswapd0 在仍有空闲内存时频繁运行，但报告中缺乏内核堆栈或代码层面的具体分析确认。

#解决方案
用户反馈升级内核至 3.13.0-32-generic 版本后问题不再复现。
