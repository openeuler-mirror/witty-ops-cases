#标题
eCryptfs unlink操作触发VFS层may_delete函数kernel BUG

#内核版本
3.2.0-58-generic

#问题现象
- 系统日志记录 kernel BUG：`kernel BUG at /build/buildd/linux-3.2.0/fs/namei.c:1898!`
- 触发 invalid opcode 异常（0000 [#1]），导致进程 `keepassx` 崩溃。
- 堆栈信息显示调用链为：`sys_unlink` -> `do_unlinkat` -> `vfs_unlink` -> `ecryptfs_unlink` -> `ecryptfs_do_unlink` -> `vfs_unlink` -> `may_delete`。

#问题根因
进程在 eCryptfs 文件系统上执行 unlink 操作时，VFS 层的 `may_delete` 函数（`fs/namei.c`）触发了 `BUG()` 宏，通常表示内核检测到了非法状态或断言失败。具体触发该 BUG 的前置条件未明确。

#解决方案
暂无（Bug 状态为 NEW，且缺乏复现步骤，未提供修复补丁）。
