#标题
MD RAID10 移除磁盘时 handle_write_completed 空指针解引用导致内核崩溃

#内核版本
4.1.8, 4.6

#问题现象
- 在全 SSD RAID-10 阵列上进行 I/O 负载测试并物理移除一块 SSD 时触发内核 Oops。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000058"。
- 调用栈显示崩溃发生在 `rdev_clear_badblocks+0xd/0x40`，由 `raid10d` 线程调用。

#问题根因
在 `handle_write_completed` 函数处理带有 `R10BIO_WriteError` 和 `R10BIO_IsRecover` 标志的 r10bio 时，代码遍历所有副本。如果 `devs[m].repl_bio` 不为 NULL，代码假定 `conf->mirrors[dev].replacement`（即 `rdev` 指针）也不为 NULL。然而该假定并不总是成立，`rdev` 可能为 NULL，导致后续调用 `rdev_clear_badblocks(rdev, ...)` 时发生空指针解引用。

#解决方案
应用上游补丁 commit `01a69cab01c184d3786af09e9339311123d63d22`（标题：md raid10: fix NULL deference in handle_write_completed）。该补丁在调用 `rdev_clear_badblocks` 前增加了对 `rdev` 是否为空的检查。
