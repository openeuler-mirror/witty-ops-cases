#标题
gpio-mockup模块移除debugfs时触发空指针解引用

#内核版本
6.0-rc6 mainline

#问题现象
- 执行 gpio kselftest 脚本 gpio-mockup.sh 时测试失败，报错：test failed: can't remove existing gpio-mockup module
- dmesg 显示内核发生 BUG: kernel NULL pointer dereference, address: 00000000000000a0
- 故障堆栈包含 simple_recursive_removal+0x9d/0x2a0

#问题根因
gpio-mockup 模块在移除 debugfs 时触发了空指针解引用，导致模块无法正常卸载。

#解决方案
该问题已通过补丁修复，Commit ID: b7df41a6f79dfb18ba2203f8c5f0e9c0b9b57f68，补丁标题：gpio: mockup: fix NULL pointer dereference when removing debugfs。
