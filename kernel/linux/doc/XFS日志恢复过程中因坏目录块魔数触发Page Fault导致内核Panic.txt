#标题
XFS日志恢复过程中因坏目录块魔数触发Page Fault导致内核Panic

#内核版本
v6.1-rc0

#问题现象
- 运行xfstests generic/648测试用例时触发内核Panic
- 日志显示XFS log I/O error -5，文件系统被强制关闭
- 重挂载文件系统时打印 "Bad dir block magic!" 告警
- 触发内核断言失败：Assertion failed: 0, file: fs/xfs/xfs_buf_item_recover.c, line: 414
- 最终在 __xfs_dir3_data_check 函数中触发 "BUG: unable to handle page fault"，RIP: 0010:__xfs_dir3_data_check+0x171/0x700 [xfs]

#问题根因
未明确。根据日志分析，故障发生在XFS文件系统使用64k目录块大小进行日志恢复阶段。系统检测到目录块魔数错误后触发断言，随后在 xlog_recover_validate_buf_type 调用链中访问非法内存地址导致Oops。具体代码逻辑缺陷尚待定位。

#解决方案
暂无。开发者建议对问题进行二分查找以确定引入故障的提交。
