#标题
进程退出时触发Bad page map及General Protection Fault导致系统挂起

#内核版本
4.9.39 + imq patch + arch cflags patch

#问题现象
- 系统发生挂起，无法响应。
- 内核日志显示 "BUG: Bad page map in process git"，伴随 "swap_free: Unused swap offset entry" 错误。
- 进程 git 在退出路径（do_exit -> mmput -> exit_mmap -> unmap_page_range）中触发 "general protection fault: 0000 [#1]"。
- 随后 kswapd0 进程触发 "WARNING: ... at fs/dcache.c:410 d_lru_shrink_move"。
- 最终触发 "BUG: unable to handle kernel NULL pointer dereference" 导致 Oops。

#问题根因
未明确。日志显示内存管理状态损坏，进程地址空间清理时出现非法页表映射和指针访问错误。由于内核包含非官方补丁，可能与此相关，但报告中未确认具体根因。

#解决方案
暂无。Bug 状态为 NEW，未提供修复方案。
