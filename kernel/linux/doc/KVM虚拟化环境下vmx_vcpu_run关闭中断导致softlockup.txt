#标题
KVM虚拟化环境下vmx_vcpu_run关闭中断导致softlockup

#内核版本
4.1.0

#问题现象
- 系统运行过程中出现 softlockup，持续数分钟后自动恢复。
- QEMU 进程出现挂起。
- 触发 softlockup 的 CPU（如 CPU 2）调用栈显示在 `smp_call_function_single` 中等待响应。
- 目标 CPU（如 CPU 26）调用栈显示在 `vmx_vcpu_run` 中执行，能响应 NMI 但未响应 IPI。

#问题根因
在 `vcpu_run` 流程中调用 `vmx_vcpu_run` 时处于关中断状态（`irq_disable`），导致 CPU 无法及时响应 IPI（处理器间中断）。当其他 CPU 通过 `smp_call_function_single` 向该 CPU 发送请求并等待响应时，因目标 CPU 长期处于关中断状态无法处理请求，导致发送方触发 softlockup。

#解决方案
用户提供了相关修复补丁链接：https://git.kernel.org/pub/scm/linux/kernel/git/stable/linux.git/commit/arch/x86/kvm?id=b5861e5cf2fcf83031ea3e26b0a69d887adf7d21
