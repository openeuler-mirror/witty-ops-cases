#标题
XFS校验icount时触发agno越界断言失败

#内核版本
5.13.0-rc4, 5.14.0-rc4+

#问题现象
- 运行 xfstests 测试集 xfs/104 用例时，内核日志报告断言失败：`XFS: Assertion failed: agno < mp->m_sb.sb_agcount, file: fs/xfs/libxfs/xfs_types.c, line: 22`。
- Call Trace 显示调用路径为 `xfsaild_push` -> `xfs_buf_delwri_submit_buffers` -> `xfs_sb_write_verify` -> `xfs_validate_sb_write` -> `xfs_verify_icount`。
- 卸载文件系统时触发另一断言失败：`XFS: Assertion failed: atomic_read(&pag->pag_ref) == 0, file: fs/xfs/libxfs/xfs_ag.c, line: 195`。

#问题根因
在 `xfs_verify_icount` 函数执行过程中，计算得到的 AG 编号（`agno`）大于或等于超级块中记录的 AG 总数（`sb_agcount`），违反了 XFS 内部一致性约束。同时，卸载时的 `pag_ref` 断言失败暗示 per-AG 结构的引用计数管理存在异常。

#解决方案
暂无
