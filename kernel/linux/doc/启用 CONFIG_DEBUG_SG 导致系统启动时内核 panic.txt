#标题
启用 CONFIG_DEBUG_SG 导致系统启动时内核 panic

#内核版本
4.15.7

#问题现象
- 系统在启动进入 initramfs 阶段时立即触发 kernel panic。
- 日志报错："kernel BUG at ./include/linux/scatterlist.h:147!"，错误码为 "invalid opcode: 0000 [#1]"。
- 触发 panic 的调用栈为：sg_init_one -> ata_exec_internal -> zpodd_init -> ata_dev_configure。
- 该故障仅在内核配置项 CONFIG_DEBUG_SG=y 时出现。

#问题根因
当开启 CONFIG_DEBUG_SG 选项时，scatterlist.h 中的 sg_init_one 函数会执行断言检查 BUG_ON(!virt_addr_valid(buf))。在 zpodd_init 相关的执行流程中，传入 sg_init_one 的缓冲区地址未能通过 virt_addr_valid 校验，导致直接触发 BUG 引发 panic。

#解决方案
暂无修复补丁。规避方法为在内核配置中关闭 CONFIG_DEBUG_SG 选项。
