#标题
ACPI BIOS错误导致CS35L56功放无法挂载且无声音输出

#内核版本
未明确（用户仅提及 linux, linux-zen, linux-mainline，未提供具体版本号）

#问题现象
- 联想 Yoga Pro 9i Aura edition Gen 11 笔记本无声音输出，默认仅识别虚拟音频设备。
- 启用 "Panther Lake Smart Sound Technology" 后生成多个无效声音设备（如 Pro, 2, 5, 6, 7, 31），执行 `speaker-test` 时部分设备挂起，其余无声音。
- dmesg 日志报错：`ACPI BIOS Error (bug): Failure creating named object [\ _SB.AUDC], AE_ALREADY_EXISTS`。
- dmesg 日志显示 cs35l56 驱动状态异常：`Slave X state check1: UNATTACHED`。

#问题根因
- BIOS 中 ACPI 表定义错误，导致 `_SB.AUDC` 对象重复创建（AE_ALREADY_EXISTS）。
- 该 ACPI 错误导致内核生成“幽灵设备”，干扰了 CS35L56 Cirrus Logic 功放驱动的正常枚举和挂载。
- ACPI 设备地址解析异常：`invalid addr: 0xd011010500`。

#解决方案
开发者提供了临时 DMI 补丁（https://github.com/thesofproject/linux/pull/5773/commits）尝试修复，但用户反馈应用后声音设备数量减少，仍无声音输出。开发者正在进一步调查内核配置及 ACPI 解析问题，暂无彻底解决方案。
