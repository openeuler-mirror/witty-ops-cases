#标题
tcp_keepalive_timer 函数引发 soft lockup

#内核版本
3.0.93

#问题现象
- 内核日志报错 "BUG: soft lockup - CPU#3 stuck for 22s!"
- 进程 neutron-server 在执行 tcp_keepalive_timer 时卡住。
- Call Trace 显示路径为：_raw_spin_lock <- tcp_keepalive_timer <- call_timer_fn <- run_timer_softirq。
- 单个 CPU 在 bh_lock_sock(sk) 处死锁，其他 CPU 空闲。

#问题根因
调用 tcp_keepalive_timer 时，在 bh_lock_sock(sk) 获取锁的过程中发生死锁，导致 CPU 无法调度。

#解决方案
暂无
