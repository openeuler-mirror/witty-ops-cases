#标题
MTK SPI驱动触发Asynchronous SError导致内核panic

#内核版本
6.6.23

#问题现象
在 Banana Pi BPI-R4 平台上操作 SPI 设备时触发内核 panic，具体表现为：
- 执行 `spidev_test` 或向 `/dev/spidev1.0` 写入数据时系统崩溃
- 日志显示 `SError Interrupt on CPUx, code 0x00000000bf000002 -- SError`
- Call trace 指向 `mtk_spi_hw_init` 和 `mtk_spi_set_cs` 函数
- 最终报错 `Kernel panic - not syncing: Asynchronous SError Interrupt`

#问题根因
MTK SPI 驱动（spi-mt65xx.c）在执行硬件初始化（`mtk_spi_hw_init`）或片选控制（`mtk_spi_set_cs`）时，触发了异步外部中止，导致内核无法同步并触发 panic。具体的硬件寄存器访问异常原因（如时钟、电源或设备树配置）在报告中未明确界定。

#解决方案
暂无
