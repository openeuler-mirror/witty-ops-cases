#标题
调度器双重入队导致CPU delay计算值异常过大

#内核版本
6.12.24 及更新版本

#问题现象
- 压力测试中发现部分任务的 CPU delay 计算值异常高。
- 通过调试发现任务的 `enqueue_num` 为 2，表明同一任务在同一 CPU 队列中被双重入队。
- 第二次执行 `sched_info_dequeue` 时，任务状态为 `TASK_IDLE` 或 `TASK_INTERRUPTIBLE`。

#问题根因
任务在同一 CPU 运行队列中被双重入队。在 `sched_info_enqueue` 函数中，仅当 `last_queued` 为 0 时才更新时间戳。双重入队导致 `last_queued` 未被重置，保留了第一次入队的时间戳。当任务最终出队时，计算出的时间差 `delta`（当前时间 - 第一次入队时间）包含了中间的额外运行时间，导致 CPU delay 统计值异常巨大。

#解决方案
可通过关闭 `DELAY_DEQUEUE` 调度特性规避该问题：
`echo NO_DELAY_DEQUEUE | sudo tee /sys/kernel/debug/sched/features`
