#标题
蓝牙模块hci_devcd_rx处理流程触发SLUB分配器BUG导致设备断开

#内核版本
6.5.0-5-amd64 #1 Debian 6.5.13-1

#问题现象
- 蓝牙鼠标间歇性停止工作，需重启系统才能恢复。
- 内核日志报错 "kernel BUG at mm/slub.c:440!"，伴随 "invalid opcode: 0000 [#1]"。
- Call Trace 显示崩溃路径为：`__slab_free` <- `hci_devcd_rx [bluetooth]` <- `process_one_work`。
- 故障发生后，USB 蓝牙设备断开连接。

#问题根因
在蓝牙设备核心转储接收处理流程中，`hci_devcd_rx` 函数调用 `__slab_free` 时触发了 SLUB 分配器的内部一致性检查失败，导致 kernel BUG。该错误通常由内存损坏、双重释放或向分配器传递了无效指针引起。

#解决方案
暂无
