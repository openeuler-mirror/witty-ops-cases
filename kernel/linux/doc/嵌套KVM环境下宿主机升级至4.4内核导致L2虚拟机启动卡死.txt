#标题
嵌套KVM环境下宿主机升级至4.4内核导致L2虚拟机启动卡死

#内核版本
4.4+（宿主机内核），L1虚拟机内核为2.6.32

#问题现象
- 宿主机升级至 4.4+ 内核后，运行于其上的 L1 虚拟机（RHEL 6.7, kernel 2.6.32）无法启动 L2 虚拟机，表现为启动过程卡死。
- L1 虚拟机出现 CPU soft lockups。
- 在 L1 中设置 `options kvm_intel vpid=0` 可恢复 L2 启动能力，但在宿主机设置该参数会导致 L1 panic。

#问题根因
4.4 内核引入的 VPID 相关更改（涉及提交 99b83ac893b 等）与 L1 客户机内核（2.6.32）中的 bug 产生交互，导致 L2 虚拟机启动失败。

#解决方案
开发者已发布补丁修复该兼容性问题，应用补丁后 L2 虚拟机可正常启动；临时规避方法为在 L1 虚拟机中禁用 VPID 功能（设置 `options kvm_intel vpid=0`）。
