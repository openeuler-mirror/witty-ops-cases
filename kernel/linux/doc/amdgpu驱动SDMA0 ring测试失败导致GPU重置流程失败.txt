#标题
amdgpu驱动SDMA0 ring测试失败导致GPU重置流程失败

#内核版本
Mainline 5.7.6

#问题现象
- 内核日志报错 `ring sdma0 timeout`，并触发 GPU reset 流程。
- GPU reset 开始后，日志显示 `GPU reset succeeded, trying to resume`，但随后报错 `ring sdma0 test failed (-110)`。
- 最终 GPU reset 失败，日志输出 `GPU reset end with ret = -110`。

#问题根因
GPU 发生 `sdma0` ring 超时，触发 GPU reset 机制。在 reset 后的恢复阶段（`amdgpu_device_ip_resume_phase2`），驱动对 `sdma0` ring 进行测试（`amdgpu_ring_test_helper`）时再次超时（返回错误码 -110），导致 `sdma_v4_0` IP block 恢复失败，进而导致整个 GPU reset 流程失败。

#解决方案
暂无（Bug状态为 NEW，未提供修复补丁或临时规避方案）。
