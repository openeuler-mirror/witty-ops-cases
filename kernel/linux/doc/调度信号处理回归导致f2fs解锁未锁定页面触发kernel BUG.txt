#标题
调度信号处理回归导致f2fs解锁未锁定页面触发kernel BUG

#内核版本
4.4-rc4

#问题现象
- 系统在 f2fs 文件系统上进行常规文件操作（如更新大型 git 仓库）时触发 kernel BUG。
- 日志报错："kernel BUG at fs/f2fs/f2fs.h:1267!"，伴随 "invalid opcode: 0000 [#1] PREEMPT SMP"。
- Call Trace 指向 f2fs_lookup 或 f2fs_iget 函数。

#问题根因
内核合并分支 'sched-urgent-for-linus' (commit fb7b26e) 引入回归，破坏了信号处理逻辑。这导致 f2fs 模块在调用 f2fs_put_page 时，尝试对一个实际上未被锁定的页面（page）执行 unlock_page 操作，从而触发 BUG_ON(!PageLocked(page)) 检查失败。

#解决方案
应用补丁 "sched/wait: Fix the signal handling fix" 修复该回归问题（参考 lkml.org/lkml/2015/12/12/149）。
