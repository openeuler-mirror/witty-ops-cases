#标题
GFS2文件系统写入损坏镜像时调用空函数指针导致内核崩溃

#内核版本
4.18.0-rc1+

#问题现象
- 挂载特制的损坏 GFS2 镜像并执行写入操作时，触发内核 Oops。
- 系统日志显示 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000000"。
- RIP 寄存器指向空地址（NULL），Call Trace 显示调用路径为 `gfs2_file_write_iter` -> `__generic_file_write_iter` -> `generic_perform_write`。

#问题根因
挂载损坏的 GFS2 文件系统镜像后，内核在执行写入流程时进入 `mm/filemap.c` 中的 `generic_perform_write` 函数。由于镜像数据损坏，`address_space_operations` 结构体中的 `write_begin` 函数指针为 NULL，内核尝试直接调用该空指针导致崩溃。

#解决方案
暂无修复方案。
