#标题
本地套接字ECMP多路径负载均衡失效

#内核版本
5.8.0-50-generic

#问题现象
- 创建本地 TCP 套接字时，去往同一目的 IP 的连接未能在多接口间进行 ECMP 负载均衡。
- 即使设置了 net.ipv4.fib_multipath_hash_policy 为 L4 哈希，连接仍固定使用单一接口。
- 使用 ip route append 设置相同 metric 的路由条目后，负载均衡仅在前 5-10 秒内生效，随后所有新建连接会固定选择其中一个接口。

#问题根因
对于未绑定地址的本地套接字，内核会进行两次 FIB 查找。第一次查找用于选择源地址（此时 saddr == 0），第二次查找用于决定路由（此时 saddr 已确定）。由于两次查找的哈希输入不同（源地址变化），导致计算出的哈希值不同，从而可能选中不同的路径，破坏了 ECMP 的预期行为。

#解决方案
暂无修复方案。开发者指出问题源于 FIB 查找机制中源地址选择与路径选择分离导致的哈希不一致。
