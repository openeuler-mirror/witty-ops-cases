#标题
Btrfs挂起恢复后clear_state_bit触发内核空指针解引用

#内核版本
5.7.6

#问题现象
- 系统从挂起恢复后，应用程序（如Firefox）无响应且无法被杀死
- Btrfs文件系统同步操作失败，无法正常关机，需硬重启
- 内核日志报错：BUG: kernel NULL pointer dereference, address: 0000000000000011
- Call Trace显示崩溃发生在 rb_next，调用链为 clear_state_bit -> __clear_extent_bit -> btrfs_destroy_inode，上下文为 kswapd 进程

#问题根因
系统从挂起恢复后，Btrfs在kswapd内存回收阶段销毁inode时，clear_state_bit函数调用rb_next遍历红黑树节点发生空指针解引用，具体触发逻辑未明确。

#解决方案
暂无
