#标题
RTC设备状态返回零导致ACPI SystemCMOS区域无处理程序引发系统挂起

#内核版本
5.10.69-1-lts

#问题现象
- 系统日志报错：ACPI Error: No handler for Region [RCM0] (SystemCMOS) 以及 Region SystemCMOS (ID=5) has no handler
- 功能异常：Hibernate（休眠）功能无法使用
- 稳定性故障：从 Suspend（挂起）状态恢复时，系统发生 Kernel Panic 或直接挂起，需强制重启

#问题根因
DSDT 表中 RTC 设备的 `_STA` 方法返回值为 `Zero`。根据 ACPI 规范，返回 0 表示设备不存在，因此 ACPI 核心未给该设备安装地址空间处理程序。当 AML 代码尝试访问 `SystemCMOS` 区域时，因找不到对应的处理程序导致错误，进而中止了 `_SB.WMID.ESDT` 和 `_SB.PCI0.LPCB.EC0._Q42` 等方法的执行，引发系统不稳定。

#解决方案
暂无内核补丁修复。建议通过修改 DSDT 表将 `RTC._STA` 方法的返回值改为 `0x0F`，并重新编译覆盖 BIOS DSDT 以规避此问题。
