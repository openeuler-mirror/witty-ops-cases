#标题
AMD IOMMU 驱动转换导致 Ryzen 平台 kdump 无法正常工作

#内核版本
5.4.0-rc3

#问题现象
- 在 Ryzen 2700x 平台上，触发内核崩溃后 kdump kernel 无法正常启动，屏幕无任何输出。
- 使用 `kexec -p` 加载 crash kernel 后，通过 sysrq 触发崩溃，系统无响应。
- 直接使用 `kexec`（不带 -p 参数）加载内核可正常工作。

#问题根因
经二分排查，问题由 commit `be62dbf554c5b50718a54a359372c148cd9975c7` 引入。该提交标题为 "iommu/amd: Convert AMD iommu driver to the dma-iommu api"，该改动导致 kdump 场景下内核启动失败。

#解决方案
暂无
