#标题
AMD 5700G iGPU运行OpenCL应用时触发TTM警告导致GPU重置

#内核版本
5.15.0 RC6

#问题现象
- 运行 OpenCL 应用程序（如 darktable、Davinci Resolve 或 hashcat）时出现崩溃、GPU 重置及系统冻结。
- 内核日志报错：`WARNING: CPU: 9 PID: 235 at drivers/gpu/drm/ttm/ttm_bo.c:409 ttm_bo_release+0x2d1/0x300 [ttm]`。
- Call Trace 显示调用链为：`kfd_process_wq_release` -> `kfd_process_device_free_bos` -> `amdgpu_amdkfd_gpuvm_free_memory_of_gpu` -> `amdgpu_gem_object_free` -> `amdgpu_bo_unref` -> `ttm_bo_release`。
- 随后出现 `amdgpu: qcm fence wait loop timeout expired`，提示 CP 可能处于不可恢复状态，触发 GPU reset。

#问题根因
未明确。日志显示问题发生在 amdgpu 驱动的 KFD（Kernel Fusion Driver）进程释放工作队列处理过程中，`ttm_bo_release` 函数触发了警告，可能与显存缓冲对象的生命周期管理或释放逻辑有关。

#解决方案
暂无
