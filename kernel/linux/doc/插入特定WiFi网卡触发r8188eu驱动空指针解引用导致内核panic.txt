#标题
插入特定WiFi网卡触发r8188eu驱动空指针解引用导致内核panic

#内核版本
4.17.3, 4.17.9

#问题现象
- 插入 TP-Link TL-WN722N v2 WiFi 网卡或尝试连接 WiFi 时触发内核 panic。
- dmesg 日志显示 "R8188EU: assoc success" 后系统崩溃。
- 报错信息：BUG: unable to handle kernel NULL pointer dereference at 0000000000000000。
- Call trace 涉及 events_unbound、call_usermodehelper_exec_work。

#问题根因
内核 4.17 版本自带的 r8188eu 驱动存在缺陷，在执行 events_unbound 队列中的 call_usermodehelper_exec_work 函数时，触发了空指针解引用。

#解决方案
使用第三方驱动源码编译安装替代内核自带驱动。具体步骤为：将内核自带 r8188eu 加入黑名单，克隆 https://github.com/lwfinger/rtl8188eu.git 源码，通过 dkms 安装 8188eu/1.0 驱动。
