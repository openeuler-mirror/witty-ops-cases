#标题
zswap_lru_del访问非法地址触发内核Oops

#内核版本
6.8.4-arch1-1

#问题现象
- 内核日志报错：BUG: unable to handle page fault for address: 00000000000a0955
- 触发 Oops，RIP 寄存器指向 zswap_lru_del+0x4e/0x80
- 调用栈显示路径为：zswap_free_entry -> zswap_invalidate -> swap_range_free -> exit_mmap
- 故障通常发生在关闭 Firefox 或其他高内存占用进程时
- 系统提示 "Fixing recursive fault but reboot is needed!"，并伴随 "BUG: scheduling while atomic" 错误

#问题根因
在进程退出并释放内存（exit_mmap -> free_swap_and_cache）的过程中，zswap 模块在执行 zswap_lru_del 函数时访问了非法内存地址（接近 NULL 的偏移地址），导致内核页错误。由于日志显示内核被污染（Tainted: P），且用户反馈与 Nvidia 闭源驱动相关，推测可能与特定驱动环境下的内存管理竞争或指针错误有关。

#解决方案
暂无官方修复补丁。可尝试以下规避方案：
1. 禁用 zswap 功能。
2. 将 Nvidia 驱动切换为开源版本 nvidia-open。
3. 尝试禁用 CONFIG_TRANSPARENT_HUGEPAGE 选项。
