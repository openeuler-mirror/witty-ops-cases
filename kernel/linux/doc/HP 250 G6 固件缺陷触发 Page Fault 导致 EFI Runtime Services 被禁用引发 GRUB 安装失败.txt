#标题
HP 250 G6 固件缺陷触发 Page Fault 导致 EFI Runtime Services 被禁用引发 GRUB 安装失败

#内核版本
5.0.0-20

#问题现象
- 在 HP 250 G6 笔记本上以 UEFI 模式安装系统时报错 "Unable to install GRUB in /dev/sda"，安装终止。
- 内核日志显示 "[Firmware Bug]: Page fault caused by firmware at PA: 0x67963770"。
- 触发 WARNING 并打印调用栈，指向 `efi_recover_from_page_fault`。
- 日志显示 "efi: Froze efi_rts_wq and disabled EFI Runtime Services"。

#问题根因
硬件厂商提供的 UEFI 固件存在 Bug，在调用 EFI Runtime Services 时触发了 Page Fault。内核捕获该错误后，按设计逻辑执行了恢复操作（禁用 EFI Runtime Services）以防止系统崩溃。这导致 GRUB 安装程序无法调用底层接口更新启动路径，从而报错失败。此现象并非内核 Panic，而是内核对固件错误的防御性处理。

#解决方案
该问题根因在于固件缺陷，内核层面无法直接修复。
变通方法：可通过 UEFI Shell 中的 `bcfg` 命令手动配置启动项以完成安装。建议向硬件厂商反馈固件问题。
