#标题
CIFS 文件拷贝触发 Soft Lockup 导致系统重启

#内核版本
6.8.4（受影响版本包括 6.9.x，6.7.9 及之前版本正常）

#问题现象
- 向 CIFS 挂载目录拷贝文件时，系统冻结并自动重启。
- 日志显示 `watchdog: BUG: soft lockup - CPU#2 stuck for 26s! [cifsd:1959]`。
- 堆栈信息显示进程卡在 `native_queued_spin_lock_slowpath`，调用链为：`cifs_demultiplex_thread` -> `__release_mid` -> `smb2_handle_cancelled_mid` -> `smb2_find_smb_tcon` -> `__cifs_put_smb_ses` -> `_raw_spin_lock`。

#问题根因
CIFS 客户端在 `smb2_handle_cancelled_mid` 处理流程中存在死锁（deadlock）或自旋锁争用问题，导致 `__cifs_put_smb_ses` 函数获取锁时 CPU 长时间阻塞，触发 soft lockup 机制。

#解决方案
升级内核至 6.10 或更高版本（经测试 6.10-rc3 及 6.10.6 版本已修复该问题）。
