#标题
tcp_send_fin函数引用未初始化变量导致内核Oops

#内核版本
2.6.32.66

#问题现象
系统升级至 2.6.32.66 后，在网络服务（如 smtpd、ipop3d）关闭 TCP 连接时频繁触发 Kernel Oops。日志报错 "Unable to handle kernel paging request for data at address 0x0000003c"，故障指令位于 `tcp_send_fin+0x48`，Call Trace 显示：`tcp_send_fin` -> `tcp_close` -> `inet_release` -> `sock_release` -> `sock_close`。

#问题根因
commit f944afb246e7b8edd6196984e21764eeda5446d3 在反向移植过程中存在代码错误。在 `tcp_send_fin` 函数中，代码错误地使用了未初始化的局部变量 `skb`（对应代码 `TCP_SKB_CB(skb)->flags |= TCPCB_FLAG_FIN;`），而正确的逻辑应使用已初始化的变量 `tskb`。这导致访问了无效内存地址（offset 0x3c 附近），引发内核分页错误。

#解决方案
修正 `tcp_send_fin` 函数中的变量引用错误，将误用的 `skb` 替换为 `tskb`（参考 Bugzilla 附件 178341 提供的修复补丁）。
