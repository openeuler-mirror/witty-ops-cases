#标题
ALSA驱动注册input设备时sysfs重复文件名导致空指针解引用

#内核版本
4.20.0-rc6

#问题现象
- 系统启动过程中内核日志报错：sysfs: cannot create duplicate filename '/devices/pci0000:00/0000:00:1f.3/sound/card0/input9'
- 触发内核 Oops：BUG: unable to handle kernel NULL pointer dereference at 0000000000000048
- 故障调用栈显示：klist_add_tail+0x29/0x80 -> device_add -> evdev_connect -> input_register_device -> snd_jack_dev_register

#问题根因
在 snd_hda_intel 驱动加载（azx_probe_work）过程中，snd_jack_dev_register 函数尝试注册 input 设备时，因 sysfs 目录已存在导致 kobject_add_internal 失败返回 -EEXIST；随后在设备添加流程中（device_add -> klist_add_tail）访问了空指针。开发者推测这可能与 sysfs 层的已知问题有关。

#解决方案
开发者建议验证 4.20-final 版本内核。根据后续反馈，该问题已超过 10 个月未复现，推测已在后续内核版本中修复。
