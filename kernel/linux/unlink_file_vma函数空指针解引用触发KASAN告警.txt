#标题
unlink_file_vma函数空指针解引用触发KASAN告警

#内核版本
v5.2-rc6

#问题现象
- 内核触发 KASAN 告警：BUG: KASAN: null-ptr-deref Write in unlink_file_vma
- 故障发生在进程退出路径，sshd 进程在执行 down_write 时尝试写入地址 0000000000000030
- 调用栈显示：exit_mmap -> free_pgtables -> unlink_file_vma -> i_mmap_lock_write -> down_write

#问题根因
在 unlink_file_vma 函数中，代码尝试通过 i_mmap_lock_write 获取读写信号量。KASAN 报告显示写入地址为 0x30（NULL + offset），表明传入 down_write 的指针为 NULL。分析推测 VMA 关联的 file 结构体非空（通过了非空检查），但其 f_mapping 成员为 NULL，导致在访问 address_space 中的 i_mmap_rwsem 锁时触发空指针解引用。

#解决方案
暂无
