#标题
任务退出时 sched_mm_cid_exit 函数触发页错误导致内核 Oops

#内核版本
6.19.0

#问题现象
- 内核在任务退出过程中触发 Oops，错误码为 0003（权限违规）。
- 日志显示 "BUG: unable to handle page fault for address: ffff8d069cac0dd0"，类型为 supervisor write access in kernel mode。
- 崩溃位置位于 `sched_mm_cid_exit+0xe2/0x1e0`，调用栈显示 `do_exit` -> `sched_mm_cid_exit`。
- 故障发生时用户态进程为 `pahole`，内核配置为 `CONFIG_PREEMPT_FULL=y`。
- 系统标记为 Tainted (DIE)，构建进程冻结。

#问题根因
在 `CONFIG_PREEMPT_FULL=y` 配置下，进程退出时调用 `sched_mm_cid_exit` 函数访问内存地址 `ffff8d069cac0dd0` 触发权限违规，导致内核 Oops。具体代码逻辑错误尚待分析。

#解决方案
