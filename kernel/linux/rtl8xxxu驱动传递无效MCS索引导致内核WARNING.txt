#标题
rtl8xxxu驱动传递无效MCS索引导致内核WARNING

#内核版本
5.13.0

#问题现象
使用 TP-Link TL-WN822N v4 (RTL8192EU) 无线网卡时，内核日志中触发 WARNING。报错信息为 "Rate marked as an HT rate but passed status->rate_idx is not an MCS index [0-76]: 88 (0x58)"。Call Trace 显示调用链为 `rtl8xxxu_parse_rxdesc24` -> `ieee80211_rx_napi` -> `ieee80211_rx_list`。

#问题根因
rtl8xxxu 驱动在处理接收数据包时，通过 `rtl8xxxu_parse_rxdesc24` 函数向 mac80211 层传递了一个无效的速率索引值（88，即 0x58）。该值不在有效的 HT MCS 索引范围 [0-76] 内，导致 `ieee80211_rx_list` 函数进行速率标记校验时失败并触发 WARNING。

#解决方案
暂无正式修复。开发者曾提供 "Disable RX aggregation" 补丁进行测试，用户反馈应用该补丁及相关队列选择补丁后原 WARNING 未再复现，但 Bug 状态仍为 NEW，问题修复尚在跟进中。
