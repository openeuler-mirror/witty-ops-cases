#标题
amdgpu驱动触发VM_L2保护错误导致图形界面卡死

#内核版本
5.11.16（问题亦复现于 5.11.21）

#问题现象
- 使用 Firefox 浏览网页或系统启动结束时，出现短暂卡顿或图形界面（Xfce4）完全冻结，只能通过电源键强制关机。
- 内核日志（kern.log）中大量输出 `amdgpu: [gfxhub0] retry page fault` 警告信息。
- 日志显示 `VM_L2_PROTECTION_FAULT_STATUS:0x00101031`，具体错误字段包括 `PERMISSION_FAULTS: 0x3`。
- 故障涉及 Xorg 进程（pid 8490）及 `Faulty UTCL2 client ID: TCP (0x8)`。

#问题根因
amdgpu 驱动在处理 GPU 虚拟内存访问时，UTCL2 客户端 TCP 在访问特定地址范围（如 0x800103a01000）时触发了权限错误（PERMISSION_FAULTS），导致硬件不断重试缺页中断。由于中断风暴（日志显示 `gmc_v9_0_process_interrupt: 15083 callbacks suppressed`），导致图形子系统响应停滞，进而引发界面冻结。

#解决方案
暂无
