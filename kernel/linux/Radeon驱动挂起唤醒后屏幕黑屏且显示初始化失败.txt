#标题
Radeon驱动挂起唤醒后屏幕黑屏且显示初始化失败

#内核版本
3.3.0-rc7+

#问题现象
- Lenovo Ideapad U455 笔记本从挂起到内存（S3）唤醒后，系统响应正常（可通过 SSH 登录），但屏幕保持黑屏。
- dmesg 日志显示 Radeon 驱动初始化错误：`radeon 0000:02:00.0: Wait for MC idle timedout !`、`[drm:r600_ring_test] *ERROR* radeon: ring 0 test failed (scratch(0x8500)=0xFFFFFFFF)` 以及 `[drm:rv770_resume] *ERROR* r600 startup failed on resume`。
- 唤醒过程耗时异常长（resume devices took 82.256 seconds）。

#问题根因
Radeon 驱动（rv770/r600 系列）在挂起唤醒流程中未能正确恢复显卡硬件状态。具体表现为 GPU 的 Memory Controller (MC) 无法进入空闲状态导致超时，随后 ring 0 测试失败（scratch 寄存器读取值异常），导致显卡启动失败，无法正常输出视频信号。

#解决方案
暂无官方修复补丁。可通过 vga_switcheroo 手动切换显卡（先切换到 DIS 再切回 IGD）强制重新初始化显示输出，作为临时规避手段恢复屏幕显示。
