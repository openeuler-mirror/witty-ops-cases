#标题
UDP发送时skb_push操作指针下溢触发kernel BUG

#内核版本
3.3.0

#问题现象
- ARM 平台运行多线程 UDP 应用时，发送线程在运行 3-12 小时后意外中止。
- 系统日志报错：`kernel BUG at net/core/skbuff.c:147!` 及 `skb_under_panic`。
- 故障堆栈显示：`skb_push+0x7c/0x84` <- `eth_header+0x1c/0xb8` <- `neigh_resolve_output` <- `ip_finish_output`。

#问题根因
在 UDP 发送流程中，`eth_header` 函数调用 `skb_push` 尝试在 sk_buff 头部预留空间以填充以太网头。日志显示 `data` 指针小于 `head` 指针，触发 `skb_under_panic`，表明 sk_buff 的 headroom 空间不足或数据结构已损坏，导致无法正常压入以太网头。

#解决方案
暂无
