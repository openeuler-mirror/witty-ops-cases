#标题
AMD平台连接USB4/Thunderbolt设备触发PCIe AER错误导致链路不稳定

#内核版本
6.7 / 6.8 / 6.9 / 6.11

#问题现象
- 插入 Thunderbolt 线缆时，系统日志显示 PCIe 带宽严重受限（如 2.000 Gb/s），并报错 "PCIe Bus Error: severity=Uncorrectable (Non-Fatal), type=Transaction Layer"。
- 日志中出现大量 AER Correctable error (BadDLLP) 和 Uncorrectable error (UnsupReq, CmpltTO)。
- 在 VFIO 直通场景下，虚拟机因 PCIe 错误发生崩溃。
- 链路不稳定且传输速度缓慢。

#问题根因
AMD 平台与特定 Thunderbolt 扩展坞（如 Wikingoo Q1L、Razer Core X Chroma）配合时，PCIe 链路协商或信号传输出现异常，导致大量 AER 错误。内核开发者指出，日志中的带宽限制信息在 USB4 语境下无实际意义，但未明确给出 AER 错误的具体代码层面根因，推测与硬件兼容性或链路训练有关。

#解决方案
- 临时规避：添加内核启动参数 `pci=noaer` 忽略 AER 错误，可避免虚拟机崩溃。
- 硬件规避：更换其他型号的扩展坞（如 ADT-UT3G）可消除错误。
- 官方修复：暂无。
