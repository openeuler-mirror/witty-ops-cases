#标题
Gigaset驱动gigaset_m10x_input函数引发soft lockup

#内核版本
4.19.90-2108.8.0.0106.up5.uel20.x86_64

#问题现象
- 系统日志报错 "watchdog: BUG: soft lockup - CPU#1 stuck for 24s!"，进程 ksoftirqd/1 卡死。
- 触发 NMI backtrace，显示 CPU 0 和 CPU 1 均涉及 softirq 处理流程。
- Call Trace 指向 gigaset 驱动模块，RIP 寄存器指向函数 gigaset_m10x_input+0xe7。

#问题根因
gigaset 驱动模块中的 gigaset_m10x_input 函数在 softirq 上下文（由 gigaset_handle_event 调用）中执行时间过长，导致 CPU 长时间无法进行调度或处理其他中断，从而触发内核 soft lockup 检测机制。

#解决方案
暂无
