#标题
PCI子系统新增固件状态检查导致启动挂起及空指针崩溃

#内核版本
v6.3-rc1

#问题现象
- Dell PowerEdge R620 服务器在启动过程中挂起，配备 Intel I350 网卡（igb 驱动）。
- AWS Xen 实例在启动过程中触发 Kernel Panic，日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000004"。
- AWS Xen 实例崩溃时的 Call Trace 包含：get_free_entries -> gnttab_grant_foreign_access -> xenbus_grant_ring -> xen_blkfront。

#问题根因
引入的提交 6fffbc7ae1373e10b989afe23a9eeb9c49fe15c3 ("PCI: Honor firmware's device disabled status") 导致回归。该提交在 drivers/pci/probe.c 的 pci_setup_device 函数中新增了对 fwnode_device_is_available 的检查，若固件标记设备不可用则返回 -ENODEV。此逻辑错误地影响了 igb 网卡及 Xen 环境下的设备初始化流程，导致系统启动挂起或后续模块访问空指针引发崩溃。

#解决方案
暂无正式修复补丁。临时规避方案为回退提交 6fffbc7ae1373e10b989afe23a9eeb9c49fe15c3。
