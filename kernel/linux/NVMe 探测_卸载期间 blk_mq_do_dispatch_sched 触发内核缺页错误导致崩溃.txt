#标题
NVMe 探测/卸载期间 blk_mq_do_dispatch_sched 触发内核缺页错误导致崩溃

#内核版本
4.18.0-513.24.1.el8_9.x86_64

#问题现象
- 系统在运行 fio 异步 I/O 负载时发生内核崩溃。
- 日志显示 "BUG: unable to handle kernel paging request at ffff8c07a6ed4c00"。
- 调用栈指向 block layer：blk_mq_do_dispatch_sched+0x28b/0x3a0。
- 崩溃前 dmesg 频繁记录 NVMe 探测失败信息："nvme nvmeX: Duplicate cntlid 0 with nvme0 ... rejecting" 以及 "Removing after probe failure status: -22"。

#问题根因
故障发生在 NVMe 控制器因 ID 冲突导致探测失败并执行移除操作期间。在 blk_mq_do_dispatch_sched 函数分发 I/O 请求时访问了无效内存地址，疑似为 use-after-free 问题，即设备移除后相关数据结构仍被引用。

#解决方案
暂无。上游开发者指出报告所用的内核版本过旧，无法提供支持，建议在最新上游内核版本中验证问题是否依然存在。
