#标题
ath10k驱动在AP模式下删除Peer失败导致内核Warning及无线功能失效

#内核版本
4.4 (日志涉及 4.4.32、4.4.88 及 backports 4.14-rc1、4.18-rc7)

#问题现象
- ath10k 网卡在 AP 模式下，特定客户端（如 Nexus 5X）连接 5GHz 射频一段时间后触发内核 WARNING。
- dmesg 显示固件操作失败：`failed to delete peer ... for vdev 0: -145`。
- 驱动报错发现本应删除的 Peer 仍存在：`found sta peer ... entry on vdev 0 after it was supposedly removed`。
- 触发调用栈警告：`WARNING: CPU: 0 PID: ... at sta_info.c:964 sta_set_sinfo+0x92c/0x9e0 [mac80211]`。
- 随后固件持续报错（错误码 -11），导致 `failed to install key`、`failed to set beacon mode` 等操作失败，5GHz 无线功能瘫痪，需重启系统恢复。

#问题根因
ath10k 官方固件（API level 5 版本，如 10.2.4.70.x）在处理 Peer 删除请求时返回错误（-145），导致固件与驱动状态不一致。驱动在执行 `__sta_info_destroy` 销毁站点信息时，发现固件未能成功删除 Peer，触发 `mac80211` 模块中的状态检查 WARNING。固件随后进入异常状态，无法响应后续的 Key 安装、Beacon 设置等配置命令。

#解决方案
暂无官方修复补丁。建议使用 Candela Tech 提供的 ath10k-ct 驱动及配套固件替代官方版本，经测试可规避此问题。临时规避措施为限制问题客户端仅连接 2.4GHz 射频。
