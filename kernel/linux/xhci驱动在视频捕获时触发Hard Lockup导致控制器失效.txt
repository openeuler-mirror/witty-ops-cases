#标题
xhci驱动在视频捕获时触发Hard Lockup导致控制器失效

#内核版本
3.13.0-32-generic

#问题现象
- 系统在使用 Logitech c920 摄像头进行视频捕获（v4l2）时，xhci 驱动偶发完全失效。
- 内核日志报错 "Watchdog detected hard LOCKUP on cpu 3"。
- 日志显示 "xhci_hcd 0000:09:00.0: Stopped the command ring failed, maybe the host is dead" 以及 "Abort command ring failed"，最终报出 "HC died; cleaning up"。
- 所有连接的 USB 设备断开连接并变得无响应。

#问题根因
根据堆栈信息，CPU 在执行 `xhci_cancel_cmd` 函数流程中调用 `xhci_handshake` 时发生 Hard Lockup。此时 xhci 控制器命令环停止响应，驱动程序判定主机控制器失效并进行清理。

#解决方案
暂无官方修复补丁。用户可通过脚本对 xhci 设备执行 unbind/bind 操作以重置控制器，尝试恢复键鼠等设备的响应。
