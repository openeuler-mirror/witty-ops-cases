#标题
KVM MMU 清除 spte 时 page count 异常触发 WARNING 导致虚拟机崩溃

#内核版本
4.13.0-0.rc5.git4.1

#问题现象
- 内核日志报错：`WARNING: CPU: 0 PID: 5405 at arch/x86/kvm/mmu.c:717 mmu_spte_clear_track_bits+0xe7/0x100 [kvm]`
- 调用栈显示：`drop_spte` -> `mmu_page_zap_pte` -> `kvm_mmu_prepare_zap_page` -> `kvm_mmu_invalidate_zap_all_pages` -> `kvm_vm_ioctl`
- 伴随出现 `BUG: Bad page state in process khugepaged`，页面标志位显示 `referenced|dirty`
- 故障影响：启动 QEMU 虚拟机时，Guest OS（Arch Linux）加载过程中崩溃或挂起；在 4.12 内核上可能导致宿主机响应缓慢或死机

#问题根因
怀疑由 4.12 版本引入的 "KVM: nVMX: nested EPT improvements and A/D bits" 补丁集导致。在 `mmu_spte_clear_track_bits` 函数中，检查条件 `WARN_ON(!kvm_is_reserved_pfn(pfn) && !page_count(pfn_to_page(pfn)))` 被触发。分析认为 `accessed_dirty` 标志位的处理逻辑存在问题，导致页面引用计数异常（为 0）或页面状态标志位（referenced/dirty）在错误阶段被置位。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
