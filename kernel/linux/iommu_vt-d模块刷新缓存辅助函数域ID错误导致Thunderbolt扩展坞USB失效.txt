#标题
iommu/vt-d模块刷新缓存辅助函数域ID错误导致Thunderbolt扩展坞USB失效

#内核版本
6.11-rc1 至 6.11-rc5

#问题现象
- Dell WD19TB 等 Thunderbolt 扩展坞上的 USB 设备无法工作。
- 故障通常在重新插拔扩展坞或系统启动后 thunderbolt/bolt 服务进行设备枚举时触发。
- 内核日志报错：`DMAR: [DMA Read NO_PASID] Request device [...] fault addr [...] [fault reason 0x06] PTE Read access is not set`。
- 伴随 xHCI 控制器无响应错误：`xhci_hcd ... Abort failed to stop command ring: -110`、`xHCI host controller not responding, assume dead`。
- 部分场景下可能触发 hard lockup。

#问题根因
由 commit `f90584f4beb84211c4d21b319cc13f391fe9f3c2` ("iommu/vt-d: Add helper to flush caches for context change") 引入。该提交在刷新上下文缓存时使用了错误的 domain ID，导致 IOMMU 页表项（PTE）访问权限设置错误，进而引发 DMA 读取故障。

#解决方案
该问题已在 commit `7af6c720417f` ("iommu/vt-d: Fix incorrect domain ID in context flush helper") 中修复。升级至 6.11-rc6 或后续版本即可解决。
