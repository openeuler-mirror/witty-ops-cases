#标题
ACPI PM Timer 事件无法禁用导致内核日志频繁报错

#内核版本
3.8(upstream)

#问题现象
- 系统日志每分钟多次打印 ACPI 错误信息：`ACPI Error: No installed handler for fixed event [0x00000000]`，后确认为 PM_Timer 事件。
- 查看 `/sys/firmware/acpi/interrupts/ff_pmtimer` 发现中断计数持续增加，状态显示为 invalid。
- 系统启动时偶现 `ACPI Error: Could not disable RealTimeClock events` 错误。
- 故障机型为 Toshiba Satellite L550D (AMD Athlon II Dual-Core M300)。

#问题根因
Fixed Event 0 (PM Timer) 被触发但内核未安装对应的处理程序（handler）。内核尝试禁用该事件时，向硬件寄存器写入 0 后，enable bit 未被清除，导致事件无法被屏蔽，持续触发并产生报错。这通常是由于 BIOS/固件层面的硬件行为异常导致。

#解决方案
暂无有效修复方案。开发人员曾建议尝试禁用 RTC 驱动（关闭 CONFIG_RTC_DRV_CMOS），但经用户测试无效。
