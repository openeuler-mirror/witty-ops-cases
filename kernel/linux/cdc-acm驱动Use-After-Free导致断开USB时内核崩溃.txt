#标题
cdc-acm驱动Use-After-Free导致断开USB时内核崩溃

#内核版本
4.14.98-imx_4.14.98_2.0.0_ga+g5d6cbeafb80c

#问题现象
- 拔出 USB 线缆断开连接时偶发内核 panic（概率约为 1/100 到 1/1）。
- 日志显示 "Unable to handle kernel NULL pointer dereference at virtual address 00000030"。
- Call trace 显示崩溃发生在 `usb_autopm_put_interface`，调用链为 `acm_softint` -> `usb_autopm_put_interface`。

#问题根因
位于 `drivers/usb/class/cdc-acm.c` 文件的 `acm_softint` 函数中存在内存使用错误。用户分析指出，代码先调用 `usb_kill_urb` 释放了 URB 内存，随后又调用 `acm_submit_read_urbs` 试图访问已释放的内存。维护者指出该问题的本质可能与引用计数有关。

#解决方案
用户提交的通过交换代码行顺序的修复补丁被维护者拒绝，理由是破坏了“停止 IO -> 错误处理 -> 重启 IO”的正常流程。目前官方尚未提供修复补丁，需进一步分析引用计数逻辑。
