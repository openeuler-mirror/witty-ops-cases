#标题
USB音频设备导致系统挂起及任务阻塞

#内核版本
4.1.19-nrj-desktop-3rosa-x86_64（评论提及 3.14.x 至 4.9.20 版本均受影响）

#问题现象
- 系统在使用特定 USB 声卡（如 Steinberg UR242）时出现完全挂起，无法进行多媒体操作或正常关机。
- 内核日志显示 "task alsa-sink-USB A blocked for more than 120 seconds"。
- 日志中出现采样率异常信息："current rate -1688731388 is different from the runtime rate 44100"。
- 调用栈显示进程阻塞在 `schedule` -> `usb_kill_urb` -> `usb_hcd_flush_endpoint` 路径上。

#问题根因
问题与 USB 主控器模式有关。特定 USB 音频设备在 USB 3.0 模式下与主机控制器交互时，`usb_kill_urb` 或 `usb_hcd_flush_endpoint` 流程无法正常完成，导致内核任务长时间阻塞。禁用 USB 3.0 模式后问题消失，表明是 xHCI 控制器与该类设备的兼容性问题。

#解决方案
变通方案：在 BIOS/EFI 中禁用 USB 3.0 模式，使用 USB 2.0 模式运行。暂无内核补丁修复。
