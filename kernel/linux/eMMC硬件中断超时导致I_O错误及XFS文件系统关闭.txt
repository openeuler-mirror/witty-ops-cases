#标题
eMMC硬件中断超时导致I/O错误及XFS文件系统关闭

#内核版本
3.10.0-514.26.2.el7.x86_64

#问题现象
- 内核日志报错 "mmc0: Timeout waiting for hardware interrupt"，eMMC设备响应超时。
- 报告 "mmc0: Card stuck in programming state! mmcblk0 card_busy_detect"，设备卡在编程状态。
- 触发内核警告：WARNING: at drivers/mmc/host/sdhci.c:1004 sdhci_send_command。
- 大量 I/O 错误：blk_update_request: I/O error, dev mmcblk0 以及 Buffer I/O error on dev dm-0。
- 任务阻塞：INFO: task xfsaild/dm-0 blocked for more than 120 seconds。
- XFS文件系统因元数据I/O错误强制关闭：XFS (dm-0): Log I/O Error Detected. Shutting down filesystem。

#问题根因
eMMC设备（mmcblk0）卡在编程状态（programming state），导致sdhci驱动等待硬件中断超时（error -110 ETIMEDOUT）。持续的I/O错误最终触发XFS文件系统检测到日志I/O错误并执行强制关闭流程。

#解决方案
暂无
