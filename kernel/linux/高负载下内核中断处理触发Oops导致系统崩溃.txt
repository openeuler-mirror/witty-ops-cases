#标题
高负载下内核中断处理触发Oops导致系统崩溃

#内核版本
6.1.41

#问题现象
- 在 LS1046a (ARM Cortex-A72 x4) 平台上，系统处于高 CPU 负载（约 130%）和高 I/O 负载（PCIe 总线连接多个 USB 存储设备）时，运行数分钟后触发 Kernel Panic。
- 日志显示 "Unable to handle kernel NULL pointer dereference" 或 "Unable to handle kernel write to read-only memory"（ESR=0x9600004f, FSC=0x0f: level 3 permission fault）。
- Call trace 指向中断上下文：`irq_exit_rcu+0x10/0x120` -> `el1_interrupt` -> `__bio_release_pages+0xf4/0x138` -> `blkdev_bio_end_io` -> `bio_endio` -> `blk_update_request` -> `scsi_end_request` -> `usb_stor_control_thread`。
- 最终报错："Kernel panic - not syncing: Oops: Fatal exception in interrupt"。
