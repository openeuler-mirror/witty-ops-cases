#标题
omap-aes驱动空指针解引用导致内核崩溃

#内核版本
4.6.2 (cf_kernel_version 标记为 4.6 (0 - 2))，报告中亦提及 4.10.5

#问题现象
- 加载 omap_aes 模块或使用 LUKS 加密磁盘时触发 Kernel Oops 或 Panic。
- 内核日志报错："Unable to handle kernel NULL pointer dereference"。
- 在 4.6.2 版本上，加载模块时触发 Oops，PC 指针位于 `_raw_spin_lock_irqsave`，Call Trace 显示路径为 `crypto_transfer_request` -> `__test_skcipher`。
- 在 4.10.5 版本上，处理加密请求时触发 Oops，PC 指针位于 `post_crypt`，随后引发 "Kernel panic - not syncing: Fatal exception in interrupt"。

#问题根因
omap-aes 驱动在 4.6.x 版本引入回归问题，在加密请求处理流程中访问了空指针。具体表现为在 `crypto_transfer_request` 或 `post_crypt` 函数中解引用了无效地址（如 0x0000002c 或 0x00000004），导致内核崩溃。

#解决方案
暂无
