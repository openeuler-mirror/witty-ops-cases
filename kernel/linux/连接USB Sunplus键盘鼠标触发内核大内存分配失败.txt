#标题
连接USB Sunplus键盘鼠标触发内核大内存分配失败

#内核版本
v3.13-rc5 git

#问题现象
- 连接 USB Sunplus 键盘/鼠标时，系统可能出现死锁。
- 内核日志报错：`khubd: page allocation failure: order:6, mode:0x10c0d0`。
- 触发 Call Trace，关键路径包含 `hid_add_field` -> `__kmalloc`。

#问题根因
HID 子系统在解析设备报告时，调用 `hid_add_field` 函数试图分配 order:6 的大块连续物理内存。分析认为设备可能向主机传递了异常数据，导致驱动请求分配过大的内存空间，而内核内存分配器无法满足该请求。

#解决方案
暂无（Bug 处于 NEEDINFO 状态，尚未提供修复补丁）。
