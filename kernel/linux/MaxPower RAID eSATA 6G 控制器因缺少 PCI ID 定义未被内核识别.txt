#标题
MaxPower RAID eSATA 6G 控制器因缺少 PCI ID 定义未被内核识别

#内核版本
4.x（评论提及）

#问题现象
MaxPower RAID eSATA 6G 控制器（PCI ID: 1103:0647）在系统中无法被检测到。该设备的开源 RAID 驱动仅支持至内核 3.17.3，在 4.x 内核中无法正常工作。

#问题根因
内核代码中缺少对该控制器 PCI 设备 ID (1103:0647) 的识别支持。该设备与 RocketRAID 642L/644L 行为相似，需要在 `drivers/ata/ahci.c` 中添加设备 ID 并设置 `board_ahci_yes_fbs` 标志，同时在 `drivers/pci/quirks.c` 中添加 `quirk_dma_func1_alias` 修复，才能使用内核自带的 AHCI 驱动正常工作。

#解决方案
需手动修改内核源码添加支持（截至报告时未合并）：
1. 在 `drivers/ata/ahci.c` 中添加设备条目：`{ PCI_DEVICE(PCI_VENDOR_ID_TTI, 0x0647), .driver_data = board_ahci_yes_fbs }`
2. 在 `drivers/pci/quirks.c` 中添加 quirk 声明：`DECLARE_PCI_FIXUP_HEADER(PCI_VENDOR_ID_TTI, 0x647, quirk_dma_func1_alias)`
