#标题
WDC WD4000F9YZ 硬盘开启 NCQ 导致 ATA 总线错误及 RAID 同步异常

#内核版本
3.2.51 (stock Debian)

#问题现象
在使用三块 WDC WD4000F9YZ-09N20L0 硬盘（固件 01.01A01）构建 RAID 5 阵列并进行同步（resync）时，系统每隔几分钟报错并重置硬盘。dmesg 日志显示以下关键信息：
- ata3: EH in SWNCQ mode
- ata3.00: failed command: WRITE FPDMA QUEUED
- res 41/84:50:... Emask 0x10 (ATA bus error)
- ata3.00: error: { ICRC ABRT }
- ata3: hard resetting link
虽然内核能通过错误处理机制恢复并重试操作避免数据丢失，但严重影响 RAID 同步性能。

#问题根因
该型号硬盘在启用 NCQ（Native Command Queuing）功能时存在缺陷或兼容性问题。具体表现为执行 WRITE FPDMA QUEUED 命令时触发 ICRC 错误和 ATA 总线错误。开发者分析认为可能是硬盘固件 Bug、控制器与硬盘不兼容或电源供应问题，但禁用 NCQ 后问题消失，指向 NCQ 逻辑处理异常。

#解决方案
临时规避方案：在内核启动参数中添加 `libata.force=noncq` 强制禁用 NCQ 功能，可消除错误并正常完成 RAID 同步。目前内核尚未添加针对该硬盘型号的 `ATA_HORKAGE_NONCQ` 黑名单标志。
