#标题
RAID1 启用 write-behind 但无 write-mostly 设备导致 bio 分配触发 kernel BUG

#内核版本
5.12.5-arch1-1

#问题现象
系统运行中触发 kernel BUG，日志显示 "kernel BUG at block/bio.c:52!"，导致系统崩溃或挂起。该问题发生在 RAID1 阵列配置了 write-behind 但未指定 write-mostly 设备的场景下。

#问题根因
内核 5.12 引入的补丁 `block: use bi_max_vecs to find the bvec pool` 在 `bvec_alloc` 函数中增加了对 bio 向量数过大的 BUG() 检查。在 `drivers/md/raid1.c` 的 `raid1_write_request` 函数中，当 RAID1 启用了 write-behind 功能但未配置 write-mostly 设备（非预期配置）时，代码逻辑未能正确限制 `max_sectors`，导致尝试分配超过 `BIO_MAX_VECS` 限制的 bio，从而触发 `block/bio.c:52` 处的 BUG()。

#解决方案
该问题已在 Linux 5.15.3 版本中修复（包含补丁 3c0049a01b2176413696177f837b3ef33860ec13）。修复方案是在 `raid1_write_request` 中增加对 `WriteMostly` 标志位的检查，确保仅在设备标记为 write-mostly 时才执行 write-behind 相关的分配逻辑，防止在错误配置下触发过大的 bio 分配。
