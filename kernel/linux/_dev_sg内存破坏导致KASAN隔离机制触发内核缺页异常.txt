#标题
/dev/sg内存破坏导致KASAN隔离机制触发内核缺页异常

#内核版本
v4.18-rc2

#问题现象
- 系统触发 kernel panic，报错 "BUG: unable to handle kernel paging request at fffff1e034000208"
- 触发 Oops：0000 [#1]，环境为开启 SMP DEBUG_PAGEALLOC KASAN PTI 的 QEMU 虚拟机
- 故障调用栈显示：RIP 位于 `qlist_free_all+0xb1/0x190` (mm/kasan/quarantine.c:163)，调用链为 `quarantine_reduce` -> `kasan_kmalloc` -> `kmem_cache_alloc_node` -> `__alloc_skb` -> `tcp_sendmsg_locked`

#问题根因
KASAN 开发者 分析指出，该故障由 `/dev/sg` 设备驱动严重破坏内存所致。内存破坏导致 KASAN 隔离队列中的对象元数据或指针无效，当内核在 `qlist_free_all` 流程中调用 `virt_to_head_page` 试图访问被破坏的地址时，触发了缺页异常。

#解决方案
暂无（Bug 状态为 NEW，未提及具体修复补丁）
