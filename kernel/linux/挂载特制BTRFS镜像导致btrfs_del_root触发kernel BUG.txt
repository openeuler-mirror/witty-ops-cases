#标题
挂载特制BTRFS镜像导致btrfs_del_root触发kernel BUG

#内核版本
5.15.57

#问题现象
- 系统触发 kernel BUG at fs/btrfs/root-tree.c:323，导致 invalid opcode (0000 [#1]) 异常。
- RIP 寄存器指向 btrfs_del_root+0xe1/0x110，调用栈显示进程为 btrfs-cleaner，路径为 btrfs_drop_snapshot -> btrfs_del_root。
- 内核日志中伴随大量 BTRFS 文件系统元数据损坏告警，如 corrupt leaf、invalid chunk item size、bad tree block start 等。

#问题根因
挂载特制的 BTRFS 镜像文件触发了文件系统元数据损坏处理路径。在 btrfs-cleaner 线程执行快照清理操作（btrfs_drop_snapshot）并调用 btrfs_del_root 时，因镜像数据异常触发了代码中的 BUG() 断言（位于 fs/btrfs/root-tree.c:323）。

#解决方案
用户反馈该问题在后续内核版本中无法复现，建议升级内核版本。暂无针对该版本的具体补丁信息。
