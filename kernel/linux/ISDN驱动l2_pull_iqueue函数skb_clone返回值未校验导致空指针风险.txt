#标题
ISDN驱动l2_pull_iqueue函数skb_clone返回值未校验导致空指针风险

#内核版本
3.14.19

#问题现象
在内存不足的场景下，drivers/isdn/mISDN/layer2.c 中的 l2_pull_iqueue() 函数调用 skb_clone() 可能返回 NULL。由于代码未检查返回值，后续直接调用 skb_push() 操作该指针，可能导致内核空指针解引用或无效内存访问错误。

#问题根因
函数 l2_pull_iqueue() 调用 skb_clone() 分配内存时，未对返回值进行 NULL 检查，直接将可能为 NULL 的指针传递给 skb_headroom() 和 skb_push() 函数使用。

#解决方案
维护者已确认该问题，需在代码中增加对 skb_clone() 返回值的 NULL 检查逻辑以规避风险。
