#标题
rtw88驱动在RTL8821CE上因空指针解引用导致kernel oops和系统挂起

#内核版本
6.19.9-2-cachyos

#问题现象
- RTL8821CE 设备上发生 kernel oops，随后系统完全挂起，需要硬重启
- 日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000000"
- Oops 发生在 RIP: rtw_fw_c2h_cmd_handle+0x127/0x380 [rtw_core]，CR2 和 R12 寄存器均为 0
- 故障前会出现大量 "unused phy status page" 消息（50-60条/秒）
- Oops 后 rtwdev->mutex 未释放，后续所有触碰网络栈的进程进入不可中断 D 状态，无法干净关机
- 可复现：7次启动中发生4次，运行时间从约2分钟到约23.5小时不等

#问题根因
rtw88 驱动的 rtw_fw_adaptivity_result() 函数在 fw.c 中对 rtwdev->chip->edcca_th 进行解引用时未做 NULL 检查。RTL8822C 芯片在 hw_spec 中定义了 .edcca_th 字段，但 RTL8821C 未定义该字段（rtw8821c_hw_spec 中 edcca_th 为 NULL）。当 RTL8821CE 固件发送 C2H_ADAPTIVITY (0x37) 事件时，处理程序无条件解引用空指针导致崩溃。

进一步分析表明，固件可能并非有意发送 C2H_ADAPTIVITY，而是适配器进入损坏状态后在 RX 路径上发送的垃圾数据被误解析为 C2H 命令（id=0x37）。RX 描述符中的多个字段（RTW_RX_DESC_W0_DRV_INFO_SIZE、RTW_RX_DESC_W0_SHIFT 等）在损坏帧中呈现异常值。

#解决方案
已提出 v3 patch：在 rtw_fw_adaptivity_result() 中添加 NULL 检查，当 edcca_th 为空时提前返回。该补丁已通过 27 小时稳定性测试，验证有效。补丁讨论链接：https://lore.kernel.org/linux-wireless/ad69b2f3dbc74ea6b9b1a17f2a77fbb4@realtek.com/T/#u
