#标题
Radeon驱动音频设置修复引入回归导致X启动黑屏

#内核版本
3.17.3

#问题现象
- 使用 Mesa 驱动启动 X Server 时出现黑屏，且无法切换至 TTY 终端。
- 降级内核至 3.17.2 后功能恢复正常。
- 故障硬件环境为 HD6950 显卡，通过 HDMI 连接音频接收器再连接电视。

#问题根因
经 Git bisect 定位，问题由 commit `ffe0245532b98efc4bc0e06f29c51d3f0e471152` 引入。该提交标题为 "drm/radeon: fix speaker allocation setup"，旨在修复 speaker allocation 设置（当 sad_count 为 0 时设置为立体声），但在特定 HDMI 连接场景下引发了回归。

#解决方案
暂无修复补丁信息（报告标记为 Bug 88481 的重复）。临时规避方案为降级内核至 3.17.2 版本。
