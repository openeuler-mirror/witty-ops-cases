#标题
Ext4执行truncate操作时触发kernel BUG导致系统崩溃

#内核版本
5.17.5

#问题现象
- 系统调用 truncate 操作触发内核崩溃，日志显示 "kernel BUG at fs/ext4/ext4.h:3369!"
- CPU 发生 invalid opcode 错误（0000 [#1]），导致系统异常终止
- 故障现场 RIP 位于 ext4_free_blocks 函数

#问题根因
在 Ext4 文件系统执行 truncate 流程释放数据块时，`ext4_free_blocks` 函数内部触发了 `BUG()` 宏（断言失败）。故障调用链显示路径为 `do_truncate` -> `ext4_setattr` -> `ext4_truncate` -> `ext4_ind_truncate` -> `ext4_free_blocks`，具体触发点位于 fs/ext4/ext4.h 文件第 3369 行。

#解决方案
暂无
