#标题
SCSI电源管理机制变更导致系统恢复时等待硬盘起转而挂起

#内核版本
5.16

#问题现象
- 系统从挂起（S3/s2idle）恢复时，GUI界面冻结约5-6秒，期间键盘和鼠标输入无响应。
- dmesg日志显示 "ata2: link is slow to respond, please be patient (ready=0)"。
- 问题仅在连接了机械硬盘（HDD）时出现，移除HDD后恢复正常。
- 部分修复补丁在特定版本（如5.19.2, 6.0-rc1）上会导致系统在挂起后无法恢复（死锁或冷重启）。

#问题根因
- 通过 git bisect 定位到引入问题的提交为 `a19a93e4c6a9` ("scsi: core: pm: Rely on the device driver core for async power management")。
- 该提交改变了SCSI电源管理方式，导致系统恢复流程需等待ATA链路就绪（即HDD起转完成），阻塞了用户界面的响应。
- 后续尝试修复该问题的补丁（如 `88f1669019bd`）在 `sd_start_done` 函数中于中断上下文调用了 `mutex_lock`，导致死锁，引发系统无法恢复。

#解决方案
- 开发者 Bart Van Assche 提交了一系列补丁以重新支持异步恢复，修复了中断上下文调用 `mutex_lock` 导致的死锁问题。
- 相关修复补丁集已提交至 linux-scsi 邮件列表。
- 针对死锁问题的临时规避方案是回退相关补丁（如 commit `785538bfdd6` 在 v6.0 中回退了有问题的修改）。
- Damien Le Moal 提供的补丁在部分测试者的 5.19.8 和 6.0 内核上验证有效。
