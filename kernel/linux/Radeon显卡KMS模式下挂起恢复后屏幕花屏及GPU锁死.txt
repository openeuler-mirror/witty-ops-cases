#标题
Radeon显卡KMS模式下挂起恢复后屏幕花屏及GPU锁死

#内核版本
3.11（复现版本涵盖 2.6.32 至 3.11）

#问题现象
- 开启 KMS 后，系统挂起恢复后屏幕显示花屏，控制台和 X 图形界面均不可用。
- 内核日志频繁报错 "GPU lockup CP stall for more than 10000msec" 和 "Failed to wait GUI idle while programming pipes"。
- 日志显示 GPU 执行了重置操作，但后续仍可能报 "Failed to get a new IB (-35)"。
- 部分设备（如 RS482M）仅在电池供电时出现花屏，连接交流电源时正常。

#问题根因
Radeon 驱动在 KMS 模式下的挂起/恢复流程存在问题，可能与特定 GPU 型号（RV370、RS482M）的电源管理状态处理不当有关，导致 GPU 锁死和显示输出异常。

#解决方案
开发者曾建议尝试补丁 acf88deb8ddbb73acd1c3fa32fde51af9153227f 和 cef1d00cd56f600121ad121875655ad410a001b8，但用户反馈未能解决问题。Bug 截止报告时仍处于未修复状态。
