#标题
启用CONFIG_CGROUP_NET_PRIO导致__cgroup_bpf_run_filter_skb空指针解引用引发系统死锁

#内核版本
5.4.42（受影响版本包括 5.4.42+、5.5 及 5.6 系列）

#问题现象
- 系统发生 Hard Lock，无控制台输出或日志记录，必须重启才能恢复。
- 在 NFS 服务器场景下，客户端挂载卷后极易触发，表现为系统完全锁死。
- 通过 kdump 捕获的崩溃日志显示：`BUG: kernel NULL pointer dereference, address: 0000000000000010`。
- RIP 寄存器指向 `__cgroup_bpf_run_filter_skb+0xe7/0x3e0`，Call Trace 显示崩溃发生在网络接收软中断处理路径中（`tcp_v4_rcv` -> `sk_filter_trim_cap` -> `__cgroup_bpf_run_filter_skb`）。

#问题根因
该问题由 commit `fc800ec491c39e42b65df72dc9ede3bb2d4a3755` 引入。当内核启用 `CONFIG_CGROUP_NET_PRIO` 选项时，在网络包处理路径中触发了空指针解引用。具体崩溃位置在 `__cgroup_bpf_run_filter_skb` 函数中，尝试访问偏移为 0x10 的内存地址（对应指令 `4c 8b 78 10`），而此时 RAX 寄存器值为 0，导致访问非法地址。

#解决方案
- 临时规避：关闭内核配置选项 `CONFIG_CGROUP_NET_PRIO` 或回退引入问题的 commit。
- 正式修复：应用开发者 Cong Wang 提供的补丁（见 Bug ID 208107 附件 289707），该补丁修复了此已知上游问题。
