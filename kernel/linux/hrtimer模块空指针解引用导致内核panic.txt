#标题
hrtimer模块空指针解引用导致内核panic

#内核版本
4.19.219

#问题现象
系统在稳定性测试过程中发生多次 Kernel Panic，具体表现为：
- 场景一：内核启动阶段，CPU 进入 idle 进程时触发 panic。日志显示 "Unable to handle kernel read from unreadable memory at virtual address 0000000000000000"，Call trace 为 `rb_erase_cached` -> `timerqueue_del` -> `__hrtimer_start_range_ns` -> `tick_nohz_idle_stop_tick` -> `do_idle`。
- 场景二：用户态进程 `iot_usb_dock` 执行 epoll_wait 时触发 panic。日志显示 "Unable to handle kernel write to read-only memory at virtual address 0000000000000000"，Call trace 为 `rb_erase_cached` -> `timerqueue_del` -> `hrtimer_try_to_cancel` -> `schedule_hrtimeout_range_clock` -> `ep_poll`。

#问题根因
根据堆栈信息，故障发生在 `rb_erase_cached` 函数中，尝试访问虚拟地址 0x0 导致 Oops。该函数由 `timerqueue_del` 调用，用于从红黑树中删除定时器节点。推测原因可能是 hrtimer 相关的数据结构（如 `timerqueue_node` 或红黑树节点指针）未正确初始化或已损坏，导致在操作红黑树节点时解引用了空指针。

#解决方案
暂无（Bug 状态为 NEW，尚未提供修复补丁）
