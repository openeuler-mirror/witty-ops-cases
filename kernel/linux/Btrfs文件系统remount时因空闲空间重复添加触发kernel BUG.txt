#标题
Btrfs文件系统remount时因空闲空间重复添加触发kernel BUG

#内核版本
5.1.3

#问题现象
- 在执行 btrfs 文件系统 remount 操作时，系统触发 kernel BUG 并引发 Kernel panic。
- 内核日志显示 "kernel BUG at fs/btrfs/ctree.h:3521!" 以及 "invalid opcode: 0000 [#1]"。
- 日志中出现关键报错："BTRFS critical (device loop0): unable to add free space :-17" 和 "assertion failed: ret != -EEXIST"。
- 触发 BUG 的调用栈为：`btrfs_remount` -> `sync_filesystem` -> `btrfs_commit_transaction` -> `btrfs_unpin_free_ino` -> `__btrfs_add_free_space`。

#问题根因
在 btrfs 执行 remount 重新配置文件系统时，会触发事务提交。在 `btrfs_unpin_free_ino` 函数中尝试将 unpin 的 inode 空间重新加入空闲空间缓存，调用 `__btrfs_add_free_space` 时发现待添加的条目已存在（返回 -EEXIST 错误），导致断言 `ret != -EEXIST` 失败，进而触发 `BUG()` 使系统崩溃。

#解决方案
暂无
