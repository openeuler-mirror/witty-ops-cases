#标题
virtio_gpu_cursor_plane_update函数竞态条件导致空指针解引用

#内核版本
5.6.12

#问题现象
- 内核日志报错：BUG: kernel NULL pointer dereference, address: 0000000000000008
- 触发 Oops，RIP 寄存器指向 virtio_gpu_cursor_plane_update.cold+0x113/0x2d8
- 故障复现环境为 QEMU x86_64 虚拟机，运行 Arch Linux，使用 Sway 窗口管理器，在频繁点击 GUI 应用程序时触发崩溃。

#问题根因
故障点位于 drivers/gpu/drm/virtio/virtgpu_plane.c 文件中的 virtio_gpu_cursor_plane_update 函数。报告者初步分析认为，函数内 dma_fence_wait、dma_fence_put 及 vgfb->fence 置空的操作未加锁保护，存在竞态条件，导致访问空指针。虽然报告者后续撤回了具体的代码行分析，但确认崩溃点位于该函数且 Bug 标题仍指向竞态条件。

#解决方案
暂无
