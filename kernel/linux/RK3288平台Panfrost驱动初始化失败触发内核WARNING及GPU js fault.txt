#标题
RK3288平台Panfrost驱动初始化失败触发内核WARNING及GPU js fault

#内核版本
5.6.6-300.fc32.armv7hl

#问题现象
- 系统启动时无 GPU 输出。
- 内核日志报错：`panfrost ffa30000.gpu: devfreq_add_device: Unable to find governor for the device`，随后提示 `Fatal error during devfreq init`。
- 出现多次 GPU job slot 故障：`panfrost ffa30000.gpu: js fault, js=0, status=UNKNOWN, head=0xffffffff, tail=0xffffffff`。
- 触发内核 WARNING，调用栈显示 `panfrost_job_irq_handler` 在处理中断时调用 `mod_delayed_work_on`，随后在 `__queue_delayed_work` 和 `__queue_work` 处触发警告。

#问题根因
Panfrost 驱动在初始化 devfreq 时失败（无法找到对应的 governor），导致驱动加载流程进入错误处理或清理阶段。在释放 IRQ (`__free_irq`) 过程中，中断处理函数 `panfrost_job_irq_handler` 被触发，该函数试图调用 `mod_delayed_work_on` 调度延迟工作项，此时工作队列状态异常（可能处于未初始化或正在销毁状态），从而触发内核 WARNING。

#解决方案
临时规避：在内核启动参数中添加 `rd.driver.blacklist=panfrost` 禁用该驱动。
官方修复：暂无。
