#标题
ALSA驱动异常唤醒导致PulseAudio报错或崩溃

#内核版本
5.10.22（报告中还涉及 5.4.142、5.12.9、5.14.21-210）

#问题现象
- 系统日志中出现 PulseAudio 报错："ALSA woke us up to write new data to the device, but there was actually nothing to write."
- 报错提示："We were woken up with POLLOUT set -- however a subsequent snd_pcm_avail() returned 0 or another value < min_avail."
- 涉及驱动包括 snd_hda_intel 和 snd_usb_audio。
- 部分情况下（如 PulseAudio 16）触发断言失败导致进程崩溃："Assertion 'bq->n_blocks == 0' failed"。
- 部分用户反馈伴随音频输出间歇性失效或收到 POLLNVAL 事件。

#问题根因
ALSA 驱动（snd_hda_intel 或 snd_usb_audio）触发 POLLOUT 唤醒用户空间进程，但此时缓冲区实际无可用空间（snd_pcm_avail() 返回 0 或小于 min_avail），导致无效唤醒。开发者指出该日志通常为 PulseAudio 的冗余信息，但在特定场景下会引发进程崩溃或音频功能异常。

#解决方案
暂无修复方案。开发者建议若仅出现日志而无功能异常可忽略；若导致 PulseAudio 崩溃或音频失效，目前无明确补丁修复。
