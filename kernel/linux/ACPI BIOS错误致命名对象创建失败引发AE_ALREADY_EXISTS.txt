#标题
ACPI BIOS错误致命名对象创建失败引发AE_ALREADY_EXISTS

#内核版本
5.17.0-next

#问题现象
- 系统启动过程中内核日志报错：ACPI BIOS Error (bug): Failure creating named object [\_SB.PCI0.GP17.XHC0._S0W], AE_ALREADY_EXISTS
- 伴随出现 ACPI Error: AE_ALREADY_EXISTS, During name lookup/catalog 错误
- 内核跳过解析 AML opcode: Method (0x0014)
- 故障机型为 ASUSTeK ROG Zephyrus G14 GA401QM

#问题根因
BIOS 固件中的 AML 代码存在缺陷，尝试创建已存在的命名对象（\_SB.PCI0.GP17.XHC0._S0W 和 \_SB.PCI0.GP17.XHC1._S0W），导致内核 ACPI 子系统在加载表时检测到对象重复定义（AE_ALREADY_EXISTS）。

#解决方案
暂无（Bug 状态为 NEEDINFO，开发者正在请求 acpidump 输出以进一步分析）。
