#标题
启用TRANSPARENT_HUGEPAGE导致__filemap_get_folio空指针解引用引发应用挂起

#内核版本
6.0.6

#问题现象
- 在高负载场景下（如 deluge、qbittorrent 进行大量文件 mmap 和读取），应用程序运行数小时后挂起。
- 内核日志报告 `BUG: kernel NULL pointer dereference`，地址例如 `0000000000000096`，错误类型为 `supervisor read access in kernel mode`。
- 触发 Oops，Call Trace 显示路径为 `__filemap_get_folio` -> `filemap_fault` -> `__handle_mm_fault`。
- 该问题在使用 XFS 文件系统时复现，迁移至 BTRFS 后未复现。

#问题根因
该问题是由 commit `793917d997df` ("mm/readahead: Add large folio readahead") 引入的回归缺陷。启用大页预读功能后，内存管理子系统在处理 XArray 中的大页 folio 条目时存在逻辑错误（涉及 `page_cache_delete_batch` 或 sibling entry 处理），导致在文件映射故障处理路径中访问到无效的内部条目或空指针。

#解决方案
- 临时规避：关闭透明大页功能，将 `/sys/kernel/mm/transparent_hugepage/enabled` 设置为 `madvise` 或 `never`。
- 官方修复：该问题已在 Linux 6.6 版本中通过 commits `cbc02854331edc6dc22d8b77b6e22e38ebc7dd51` 和 `3095dd99dd759a5cab8bb81674bc133b1365fb6b` 修复。
