#标题
蓝牙设备拔出时触发内核分页请求错误导致Oops

#内核版本
3.3-rc3（受影响版本包括 3.8.10、3.10.4）

#问题现象
- 拔出蓝牙适配器时系统触发 Kernel Oops。
- 日志显示 "Unable to handle kernel paging request at virtual address 00200200"。
- 内核报错 "Internal error: Oops: 8c5 [#1] PREEMPT"。
- 调用栈显示崩溃发生在 `l2cap_chan_destroy` 函数，进程上下文为 `khubd`。

#问题根因
- 经二分排查，问题引入于提交 `721c41812daf7b38759942563773a7832e3c990d` ("Bluetooth: Move L2CAP timers to workqueue")。
- 故障发生在 `btusb_disconnect` 流程中，调用链为 `hci_unregister_dev` -> `l2cap_conn_del` -> `l2cap_sock_close_cb` -> `l2cap_sock_kill` -> `l2cap_chan_destroy`。
- 在 `l2cap_chan_destroy` 中访问了无效内存地址，导致分页请求错误。

#解决方案
暂无（Bug 状态为 NEW，报告中提及的临时修复方案 `if (chan == NULL) return` 在 3.3-rc3 上无效）。
