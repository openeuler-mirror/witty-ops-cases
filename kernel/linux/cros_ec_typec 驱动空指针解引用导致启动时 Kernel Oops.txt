#标题
cros_ec_typec 驱动空指针解引用导致启动时 Kernel Oops

#内核版本
5.15, 5.17.2

#问题现象
- 每次系统启动时触发 Kernel Oops，日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000304"。
- Call Trace 显示崩溃发生在 `cros_ec_check_features+0xa/0xa0`，由 `cros_typec_probe` 调用。
- 系统关机行为异常，SysRQ sync 可用但 unmount 和 reboot 失效，需长按电源键强制关机。
- 启动过程明显变慢。

#问题根因
Linux 5.15 引入的 commit `a8db7a3f8ac69e558c7bfbd04802201c39a104ad` 变更导致 `cros_ec_typec` 驱动在初始化（probe）阶段调用 `cros_ec_check_features` 时发生空指针解引用（RDI 寄存器为 0）。该问题在特定的 Coreboot/EDK2 固件环境下出现。

#解决方案
- 根本解决：升级 BIOS/固件至 MrChromebox-4.16 版本。
- 临时规避：屏蔽 `cros_ec_typec` 模块可恢复正常使用。
