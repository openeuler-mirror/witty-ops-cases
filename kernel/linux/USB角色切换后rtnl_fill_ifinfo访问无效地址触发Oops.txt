#标题
USB角色切换后rtnl_fill_ifinfo访问无效地址触发Oops

#内核版本
6.1.119-yocto-standard

#问题现象
- 系统日志报错 "Unable to handle kernel paging request at virtual address 00003130302f3130"，触发 Kernel Oops。
- 崩溃调用栈显示：`__pi_strlen` <- `rtnl_fill_ifinfo` <- `rtnl_dump_ifinfo`。
- 故障复现条件：使用 DWC3 驱动进行 USB 双角色切换，在 Device 模式下配置 gadgetfs NCM，断开连接后仅写入空字符串到 UDC 文件（未移除 gadgetfs 目录），随后切换回 Host 模式并连接大容量存储设备，重复此流程多次可触发。

#问题根因
在 USB 控制器从 Device 模式切换回 Host 模式时，若 gadgetfs 目录未被完全移除，网络子系统在通过 netlink dump 接口信息（`rtnl_dump_ifinfo`）时，调用 `strlen` 处理某个字符串指针（寄存器 x0 指向地址 0x00003130302f3130，该值看起来像字符串数据而非有效指针），导致内核分页错误。

#解决方案
暂无官方修复补丁。
规避方法：在将主机控制器角色切换回 Host 模式之前，完全移除 gadgetfs 目录，而非仅断开 UDC 连接。
