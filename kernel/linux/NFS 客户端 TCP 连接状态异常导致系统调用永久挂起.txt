#标题
NFS 客户端 TCP 连接状态异常导致系统调用永久挂起

#内核版本
4.4.0 (Ubuntu 14.04.5 LTS，含上游回移补丁)

#问题现象
- NFS 客户端挂载点上的系统调用永久挂起，无法响应。
- 进程在内核中挂起，等待 NFS 客户端响应。
- RPC 追踪显示大量 RPC 请求（如 3888 个）堆积在 `xprt_sending` 队列中，等待获取 TCP 套接字锁。
- 持有套接字锁的 RPC 任务（状态为 `call_transmit_status` / `xprt_pending`）似乎在向 TCP 套接字写入时挂起，且未触发超时。
- 网络抓包显示无实际 NFS 数据流量，仅有 TCP keepalive 包；TCP 连接已建立，但序列号分析暗示连接可能曾重置并重建，导致客户端状态不一致。

#问题根因
生产环境中触发竞态条件。分析认为 TCP 连接可能发生了重置并重新建立，但 NFS 客户端代码未能正确处理此状态变化，导致某个 RPC 任务在持有 TCP 套接字锁的状态下挂起（位于 `call_transmit_status` 阶段）。由于该任务未释放锁且未超时，后续所有 RPC 任务均被阻塞在 `xprt_sending` 队列中，导致系统调用挂起。

#解决方案
临时规避方法：使用 `tcpkill` 命令重置客户端与服务器的 TCP 连接，可解除 NFS 客户端挂起状态。目前 Bug 状态为 NEW，暂无官方修复补丁。
