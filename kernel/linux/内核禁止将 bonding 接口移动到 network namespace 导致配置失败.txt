#标题
内核禁止将 bonding 接口移动到 network namespace 导致配置失败

#内核版本
> 3.10.0-229.20.1

#问题现象
- 在 network namespace 中配置 bonding 接口时报错："Cannot find device 'bond0'"
- 系统日志显示："Master 'bond0': Error: handshake with driver failed. Aborting"
- 尝试将宿主机已有的 bond0 接口移动到新建的 netns 中时操作失败

#问题根因
内核行为变更，有意禁止非硬件接口类型（如 bond、bridge 等）在不同 network namespace 之间移动。该限制从某版本开始引入，导致用户无法像旧版本那样直接将 bond 接口移入 netns。

#解决方案
修改配置方式，不再尝试从外部移动 bond 接口，改为直接在目标 network namespace 内部创建 bond 接口。
示例命令：
`ip netns exec vntp ip link add dev bond0 type bond mode active-backup`
随后在 netns 内部将物理网卡加入该 bond 即可。
