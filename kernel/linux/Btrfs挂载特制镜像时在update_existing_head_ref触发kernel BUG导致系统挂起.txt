#标题
Btrfs挂载特制镜像时在update_existing_head_ref触发kernel BUG导致系统挂起

#内核版本
5.18, 5.17

#问题现象
- 挂载并操作特制镜像时触发 kernel BUG，日志显示 "kernel BUG at fs/btrfs/delayed-ref.c:667!"
- 触发 invalid opcode (0000 [#1]) 异常，Call Trace 指向 update_existing_head_ref 函数
- 系统发生 soft lockup，CPU 卡死超过 23 秒，导致内核挂起

#问题根因
内核在处理 Btrfs 文件系统的 delayed reference 时，于 `update_existing_head_ref` 函数中触发了 `BUG()` 宏（位于 fs/btrfs/delayed-ref.c:667）。该问题由特制的文件系统镜像触发，导致内核检测到内部状态不一致从而 panic，进而引发后续的 soft lockup 和系统挂起。

#解决方案
暂无
