#标题
vmwgfx驱动在spinlock上下文调用vfree触发sleeping function警告

#内核版本
4.10.0-0.rc7.git1.1.fc26.x86_64

#问题现象
- 系统日志频繁打印内核警告："BUG: sleeping function called from invalid context at mm/vmalloc.c:1441"
- 触发进程包括 VirtualBox、okular、Xorg.wrap 等，均涉及图形相关操作。
- 内核调用栈显示：`drm_release` -> `vmw_postclose` -> `ttm_object_file_release` -> `drm_ht_remove` -> `kvfree` -> `vfree` -> `remove_vm_area`。
- 系统状态显示 `in_atomic(): 1`，且持有 `tfile->lock` 自旋锁。

#问题根因
在 `ttm_object_file_release` 函数中，代码持有自旋锁 `tfile->lock`（日志中显示为 `&(&tfile->lock)->rlock`）。在该锁持有的原子上下文中，代码调用了 `drm_ht_remove`，该函数内部调用了 `kvfree` 进而调用 `vfree`。由于 `vfree` 函数内部可能睡眠，不允许在原子上下文中执行，因此触发了内核的 `__might_sleep` 检查并报错。

#解决方案
暂无
