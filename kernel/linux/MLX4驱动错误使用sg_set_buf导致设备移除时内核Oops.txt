#标题
MLX4驱动错误使用sg_set_buf导致设备移除时内核Oops

#内核版本
4.14.4

#问题现象
- 在启用 IOMMU 的 ARM64 平台上，移除 MLX4 设备实例或执行关机操作时触发内核 Oops。
- 日志显示 "Unable to handle kernel paging request at virtual address ffff7e000081dddc"，Call trace 指向 `mlx4_free_icm` -> `mlx4_shutdown`。
- 若开启 `CONFIG_DEBUG_SG` 选项，系统会在 `mlx4_alloc_icm_coherent()` 调用 `sg_set_buf()` 时触发 `BUG()`，提示 `virt_addr_valid(buf)` 检查失败。

#问题根因
MLX4 驱动在 `mlx4_alloc_icm_coherent()` 函数中错误地假设 `dma_alloc_coherent()` 返回的 CPU 地址（cpu_addr）总是与 `sg_set_buf()` 兼容。`sg_set_buf()` 内部调用 `virt_to_page()` 宏，要求传入的虚拟地址必须是有效的线性映射内核地址。在启用了 IOMMU 的 ARM64 架构上，`dma_alloc_coherent()` 返回的地址可能不属于线性映射区域，导致 `virt_addr_valid()` 检查失败或 `virt_to_page()` 返回错误的页面结构指针。后续在清理路径 `mlx4_free_icm_coherent()` 中，驱动试图使用该错误的指针地址调用 `dma_free_coherent()`，引发内核分页请求错误。

#解决方案
暂无（Bug 状态为 NEW，报告中未提及已合并的修复补丁）。
