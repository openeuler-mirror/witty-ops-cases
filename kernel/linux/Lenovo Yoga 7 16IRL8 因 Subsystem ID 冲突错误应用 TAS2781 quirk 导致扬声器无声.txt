#标题
Lenovo Yoga 7 16IRL8 因 Subsystem ID 冲突错误应用 TAS2781 quirk 导致扬声器无声

#内核版本
6.19.7

#问题现象
- 内置扬声器无声音输出，耳机和 HDMI 音频输出正常
- 问题为内核回归，自 6.8 版本引入，6.5 版本正常
- dmesg 日志显示应用了 TAS2781 相关的 fixup，但该机型无此硬件

#问题根因
Lenovo Yoga 7 16IRL8 (82YN) 与 Yoga 7 14IRL8 (82YM) 共享相同的 HDA Codec Subsystem ID (0x17aa389a)。14IRL8 机型配备 TAS2781 智能放大器，内核驱动 `patch_realtek.c` 中包含针对该 ID 的 `ALC287_FIXUP_TAS2781_I2C` quirk。由于 16IRL8 不具备 TAS2781 硬件，但驱动仅依据 Subsystem ID 进行匹配，错误地将 14IRL8 的 quirk 应用到了 16IRL8，导致扬声器配置错误从而静音。

#解决方案
暂无已合并补丁。建议在 `sound/pci/hda/patch_realtek.c` 中增加 DMI 匹配检查（如 `system-product-name`），区分 82YN (16IRL8) 和 82YM (14IRL8) 机型，仅对 14IRL8 应用 TAS2781 quirk，为 16IRL8 应用标准的 ALC287 双 DAC 配置。
