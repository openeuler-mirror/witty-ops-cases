#标题

#内核版本
5.9.0

#问题现象
- 运行 `nvme_trtype=tcp ./check nvme/012` 测试时触发超时
- 内核日志显示 NVMe 队列请求超时：`nvme nvme0: queue X: timeout request ...`
- 触发错误恢复流程：`starting error recovery`，导致 I/O 重排队：`block nvme0n1: no usable path - requeuing I/O`
- Target 端报告 keep-alive 计时器超时：`nvmet: ctrl X keep-alive timer (15 seconds) expired!`
- 重连尝试失败：`Connect command failed, error wo/DNR bit: 881`

#问题根因
未明确。报告者指出此前修复 refcount_t underflow 的补丁未解决该超时问题，具体导致超时的代码逻辑或硬件原因在报告中尚未分析确定。
