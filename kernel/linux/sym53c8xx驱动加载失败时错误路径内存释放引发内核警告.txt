#标题
sym53c8xx驱动加载失败时错误路径内存释放引发内核警告

#内核版本
3.10.36

#问题现象
- 加载 sym53c8xx 模块时内核抛出异常
- 日志显示 "CACHE TEST FAILED: script execution failed" 及 "sym0: CACHE INCORRECTLY CONFIGURED"
- 触发 WARNING，位于 `___free_dma_mem_cluster+0xe9/0xf0 [sym53c8xx]`
- Call Trace 显示调用链为 `sym2_probe` -> `sym_free_resources` -> `sym_hcb_free` -> `__sym_mfree_dma`

#问题根因
sym53c8xx 驱动在初始化失败（Cache 测试失败）后的错误处理路径中，内存释放逻辑不正确，导致 `___free_dma_mem_cluster` 处触发警告。

#解决方案
暂无
