#标题
dm-verity在XHCI控制器上因prefetch竞态导致数据损坏触发panic

#内核版本
5.4, 5.10, 6.1, 6.12, 6.19-rc.1

#问题现象
- 在USB存储设备上运行dm-verity块设备时发生数据损坏。
- 内核日志报错 "device-mapper: verity: 8:2: metadata block 66244 is corrupted"。
- 触发 Kernel panic，提示 "dm-verity device corrupted"，系统停止运行。
- Call Trace 指向 verity_work -> verity_verify_io -> verity_hash_for_block -> verity_verify_level。
- 问题仅在 XHCI 控制器环境下复现，在 SATA、NVMe 或 EHCI 控制器下未触发。

#问题根因
dm-verity 模块中的 prefetch 机制与验证器之间存在缓存重用或竞态条件，导致数据校验错误。

#解决方案
在内核启动参数中添加 dm_verity.prefetch_cluster=0 禁用 prefetch 功能，可规避该问题。
