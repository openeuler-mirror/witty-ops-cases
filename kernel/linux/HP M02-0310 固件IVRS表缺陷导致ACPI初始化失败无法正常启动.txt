#标题
HP M02-0310 固件IVRS表缺陷导致ACPI初始化失败无法正常启动

#内核版本
6.18.0-rc3（问题影响 6.6 至 6.18 多个版本）

#问题现象
- 系统在"Loading initial ramdisk"阶段硬死锁，键盘LED熄灭，需强制断电
- 启动时出现 ACPI 表错误：ACPI BIOS Error (bug): Failure creating named object \_SB.PCI0.LPCB.EC0，ACPI Error: AE_ALREADY_EXISTS
- dmesg 显示 AMD-Vi: [Firmware Bug]: : No southbridge IOAPIC found，AMD-Vi: Disabling interrupt remapping
- 仅使用 acpi=off nomodeset 或 pci=noacpi iommu=pt 参数可启动系统

#问题根因
HP M02-0310 (BIOS F.08) 固件 IVRS (I/O Virtualization Reporting Structure) 表存在缺陷，未正确声明 southbridge IOAPIC，导致 AMD IOMMU 驱动初始化时无法找到必要的 IOAPIC 配置，中断重映射被禁用后系统无法正常完成 ACPI/PCI 初始化流程。该问题为固件层面的 BIOS bug，非 Linux 内核问题。

#解决方案
临时规避方案：使用内核启动参数 pci=noacpi iommu=pt 或 pci=noacpi amd_iommu=off 可正常启动并保持 GPU 加速功能。根本解决需联系 HP 更新固件以修复 IVRS 表中缺失的 southbridge IOAPIC 定义。
