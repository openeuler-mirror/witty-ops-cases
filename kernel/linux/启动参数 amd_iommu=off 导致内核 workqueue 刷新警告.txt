#标题
启动参数 amd_iommu=off 导致内核 workqueue 刷新警告

#内核版本
4.20.3-200.fc29.x86_64

#问题现象
系统启动时若指定内核参数 amd_iommu=off，dmesg 中会触发 WARNING 日志：
- 报错位置：kernel/workqueue.c:2652 flush_workqueue+0x3dd/0x440
- 调用栈片段：flush_workqueue <- kmem_cache_destroy <- free_iommu_resources <- iommu_go_to_state <- amd_iommu_prepare

#问题根因
在禁用 AMD IOMMU (amd_iommu=off) 的启动流程中，free_iommu_resources 执行资源清理时调用 kmem_cache_destroy，进而触发了 flush_workqueue 的警告检查。报告中未提供更深层的代码逻辑分析。

#解决方案
暂无
