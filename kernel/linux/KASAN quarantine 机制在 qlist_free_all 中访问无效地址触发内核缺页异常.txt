#标题
KASAN quarantine 机制在 qlist_free_all 中访问无效地址触发内核缺页异常

#内核版本
5.2-rc6

#问题现象
- 系统触发 Kernel Oops，日志显示 "BUG: unable to handle kernel paging request for address: fffff1de1c000208"
- 错误类型为 #PF error_code(0x0000) - not-present page
- 触发位置 RIP 位于 `qlist_free_all+0x76/0xc0` (mm/kasan/quarantine.c:164)
- 调用栈显示故障路径为：`brk` syscall -> `do_brk_flags` -> `kmem_cache_alloc` -> `kasan_slab_alloc` -> `quarantine_reduce` -> `qlist_free_all`

#问题根因
在执行 `brk` 系统调用期间，KASAN 的 quarantine 机制在 `qlist_free_all` 函数中尝试访问内存时，访问了无效的内存地址（`fffff1de1c000208`），导致发生缺页异常。具体的内存破坏或指针错误原因尚待社区进一步分析确认。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
