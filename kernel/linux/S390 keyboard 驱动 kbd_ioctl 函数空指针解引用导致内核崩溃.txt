#标题
S390 keyboard 驱动 kbd_ioctl 函数空指针解引用导致内核崩溃

#内核版本
<=4.12.3

#问题现象
本地用户可向 kbd_ioctl() 函数传入 NULL 参数，导致内核崩溃。

#问题根因
drivers/s390/char/keyboard.c 文件中的 kbd_ioctl() 函数在将 arg 参数转换为指针 argp 后，未进行非空检查即进行后续操作，当传入参数为 NULL 时触发空指针解引用。

#解决方案
在 kbd_ioctl() 函数中增加对 argp 的 NULL 检查，若为 NULL 则直接返回 -EFAULT（评论中已提供修复补丁）。
