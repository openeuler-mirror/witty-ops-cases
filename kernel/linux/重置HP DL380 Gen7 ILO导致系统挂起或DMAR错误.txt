#标题
重置HP DL380 Gen7 ILO导致系统挂起或DMAR错误

#内核版本
3.10.34, 3.10.40

#问题现象
- 加载 ILO 驱动时，通过 Web 界面重置 HP DL380 Gen7 (ILO 3) 会导致系统完全挂起，控制台无响应，只能通过 ILO 观察到屏幕静止。
- 移除 ILO 驱动后，相同操作不再导致挂起，但内核日志报错：`NMI: IOCK error (debug interrupt?) for reason 61 on CPU 0`，并伴随大量循环打印的 DMAR 错误：`DMAR:[DMA Read] Request device [02:00.2] fault addr e8000 ... [fault reason 06] PTE Read access is not set`。
- 在 DL360p Gen8 (ILO 4) 及 DL360 Gen5/Gen6 (ILO 2) 设备上未出现挂起现象，仅偶发少量 DMAR 错误信息。

#问题根因
ILO 重置操作触发了硬件层面的异常行为。当加载 ILO 驱动时，该异常导致内核陷入死锁或挂起状态；当未加载驱动时，异常表现为设备 `02:00.2` 发起的非法 DMA 读取请求，被 IOMMU (VT-d) 拦截并报告 DMAR 错误（PTE Read access is not set），同时触发 NMI 中断。问题主要出现在 HP DL380 Gen7 硬件平台上。

#解决方案
暂无官方修复。移除 ILO 驱动模块可规避系统挂起问题，但无法消除底层的 DMAR 错误日志。
