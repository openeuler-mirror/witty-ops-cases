#标题
radix_tree_load_root访问非法地址触发内核分页错误导致Java进程挂起

#内核版本
4.14.63（日志显示为4.14.87-tinycore）

#问题现象
- Java应用重启多次（1至50次）后，进程陷入睡眠状态无法正常响应。
- 系统日志打印 "BUG: unable to handle kernel paging request at 00023fa8"。
- 触发 Kernel Oops，错误码 0000，CPU 处于 SMP 模式。
- 调用栈显示：`do_swap_page` -> `swap_readahead_detect` -> `lookup_swap_cache` -> `pagecache_get_page` -> `radix_tree_lookup_slot` -> `__radix_tree_lookup` -> `radix_tree_load_root`。

#问题根因
内核在处理交换缺页中断并进行交换预读检测时，调用链经过 `pagecache_get_page` 进入 `radix_tree_load_root` 函数。该函数尝试访问非法内存地址 `00023fa8`（寄存器 ESI 值），导致内核分页请求错误。根据测试反馈，该问题为回归问题，自 4.14.63 版本引入，在 4.14.62 版本中未出现。

#解决方案
暂无官方修复。用户测试表明该问题在 4.18.16 版本中未复现，可尝试升级内核版本规避。
