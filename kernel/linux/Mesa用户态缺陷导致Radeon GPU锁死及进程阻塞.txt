#标题
Mesa用户态缺陷导致Radeon GPU锁死及进程阻塞

#内核版本
4.2.6 (日志显示为 4.2.0-1-amd64)

#问题现象
- 运行 FlightGear 等 3D 应用时频繁发生 GPU 锁死。
- 内核日志报错：`ring 0 stalled for more than 10004msec` 及 `GPU lockup`。
- 驱动尝试执行 GPU softreset，期间出现 `rv770_set_sw_state failed` 错误。
- Xorg 及 kworker 进程阻塞超过 120 秒，处于 D 状态 (Uninterruptible Sleep)。
- 堆栈信息显示阻塞发生在 `radeon_gpu_reset` 及 `radeon_fence_wait` 流程中。

#问题根因
经开发者分析及用户验证，该故障根因为 Mesa 用户态驱动缺陷。旧版本 Mesa 发送的指令导致 GPU 硬件状态异常，触发 radeon 内核驱动检测到 ring 0 停滞并报告 lockup，进而导致等待 GPU 响应的进程阻塞。

#解决方案
升级 Mesa 库至最新版本（用户验证 git master 版本 47b3a0d23504de43ce2dc028e84037f7cd89bd89 修复了问题）。该问题非内核 Bug，应向 Mesa 项目报告。
