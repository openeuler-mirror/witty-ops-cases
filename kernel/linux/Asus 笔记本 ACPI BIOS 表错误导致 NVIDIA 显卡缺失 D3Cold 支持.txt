#标题
Asus 笔记本 ACPI BIOS 表错误导致 NVIDIA 显卡缺失 D3Cold 支持

#内核版本
6.11.5-3-cachyos

#问题现象
- 系统启动阶段 dmesg 报告大量 ACPI BIOS Error 和 ACPI Error，包括：
  - `Failure creating named object [\_SB.PCI0.GPP0._PRW], AE_ALREADY_EXISTS`
  - `Could not resolve symbol [\_SB.PCI0.GPP2], AE_NOT_FOUND`
  - `AE_NOT_FOUND, While resolving a named reference package element`
  - `\_SB.NPCF._DSM: Argument #4 type mismatch`
- NVIDIA RTX 4080 移动版显卡在 Linux 下无法获得 D3Cold 电源状态支持（Windows 下正常）。
- 另外出现 Optane 硬盘重启后无法识别的问题（通过内核参数 reboot=pci 规避）。

#问题根因
BIOS 提供的 ACPI 表存在缺陷，不符合 ACPI 规范。具体表现为 ASL 代码中存在重复定义的对象（AE_ALREADY_EXISTS）和无法解析的符号（AE_NOT_FOUND），导致内核无法正确加载显卡的电源管理配置。

#解决方案
暂无内核层面的修复补丁。该问题源于 BIOS 固件缺陷，建议联系厂商更新 BIOS 或尝试自行修补 DSDT 表；亦可向 Nouveau 或 NVIDIA 驱动团队反馈以寻求特定适配方案。
