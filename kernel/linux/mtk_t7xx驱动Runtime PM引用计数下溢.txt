#标题
mtk_t7xx驱动Runtime PM引用计数下溢

#内核版本
6.17.0-22-generic

#问题现象
- 系统启动早期出现 "Runtime PM usage count underflow!" 警告日志
- 即使内核启动参数设置 `pm_runtime=off` 禁用全局 Runtime PM，该警告依然触发

#问题根因
mtk_t7xx 驱动内部 Runtime PM 引用计数管理存在不平衡，在未进行正确防护的情况下调用了 pm_runtime API，导致 usage count 下溢；该问题属于驱动内部逻辑缺陷，与用户空间电源管理策略无关。

#解决方案
暂无（目前 Bug 处于 NEEDINFO 状态，上游正在确认是否在新版本复现）
