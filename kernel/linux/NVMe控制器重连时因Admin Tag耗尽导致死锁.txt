#标题
NVMe控制器重连时因Admin Tag耗尽导致死锁

#内核版本
未明确

#问题现象
- 系统发生死锁，NVMe控制器重连流程无法完成
- 用户态命令 `nvme list` 卡死，堆栈显示进程处于 `blk_execute_rq` 等待 admin queue 变为 live 状态
- 内核工作队列 `kworker` (nvme-wq) 卡死，堆栈显示在 `blk_mq_get_tag` 处等待获取 tag，调用链为 `nvme_enable_ctrl` -> `nvmf_reg_read64`

#问题根因
在 NVMe 控制器配置阶段，`nvme_enable_ctrl` 函数在将 admin queue 标记为 live 之前会调用 `nvmf_reg_read`，该函数申请的是非保留的 admin tag。若此时有大量 nvme-passthrough 命令（如 `nvme list`）已耗尽所有非保留 admin tag，且这些命令正等待 admin queue 变为 live 才能执行完成，则会发生死锁：重连流程等待 tag 释放，而 passthrough 命令等待重连流程完成（queue live），形成循环等待。

#解决方案
暂无官方修复。建议修改代码，使连接流程中的 `nvmf_req_read` 使用保留 tag，避免与非保留 tag 竞争。
