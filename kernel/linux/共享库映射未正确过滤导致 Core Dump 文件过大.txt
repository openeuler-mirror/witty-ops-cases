#标题
共享库映射未正确过滤导致 Core Dump 文件过大

#内核版本
3.10

#问题现象
64位 Linux 系统上，加载大量共享库的应用程序在崩溃生成 Core Dump 时，每个共享库会额外 dump 约 2MB 数据，导致 Core Dump 文件大小不必要地增加。查看进程 maps 发现存在大小为 0x200000 字节的映射段，其 VmFlags 显示为 "mr mw me"，缺少 "dd" 标志。

#问题根因
内核在生成 Core Dump 时，未能正确过滤共享库中特定的映射区域，导致这些本应跳过的数据被写入 Core Dump 文件。

#解决方案
该问题已在内核 4.7-rc1 版本中修复，相关提交 ID 为 852f42a69b93dc71507adedeed876d57b8c2c2fa。建议升级内核至该版本或更高版本。
