#标题
radeon驱动因Mesa命令缓冲区问题导致GPU锁死及恢复失败

#内核版本
3.16.3

#问题现象
- 系统日志报错 "ring 0 stalled for more than 10012msec"，随后触发 GPU lockup
- GPU 尝试软复位（softreset）后恢复失败，dmesg 显示 "ring 1 test failed"、"ring 3 test failed" 及 "cik startup failed on resume"
- 出现 VM_CONTEXT1_PROTECTION_FAULT 错误
- 用户观察到屏幕冻结、黑屏、X server 或 gnome-shell 崩溃，严重时系统完全无响应
- 故障常在使用 Firefox、播放视频或系统从挂起状态恢复时触发

#问题根因
开发者分析指出，GPU 锁死通常由用户态加速驱动 Mesa 生成的命令缓冲区存在问题所致。具体原因为 Mesa 某些版本（如 10.3.x）中异步 DMA 的使用方式触发了 radeon 驱动处理上的问题，导致 GPU 接收到错误指令而挂起。

#解决方案
应用 Mesa 补丁 ae4536b4f71cbe76230ea7edc7eb4d6041e651b4（"radeonsi: Disable asynchronous DMA except for PIPE_BUFFER"）；或升级 Mesa 至 10.4.7 及更高版本。临时规避措施为降级 Mesa 至 10.2.x 版本。
