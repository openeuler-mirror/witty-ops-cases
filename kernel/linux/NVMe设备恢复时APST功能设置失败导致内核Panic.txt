#标题
NVMe设备恢复时APST功能设置失败导致内核Panic

#内核版本
Linux-next-20170710 (日志显示 4.12.0-next-20170711)

#问题现象
- 系统在挂起-恢复循环中偶发故障，大约每两次循环出现一次。
- 内核日志显示 NVMe 控制器 I/O 超时并禁用控制器：`nvme nvme0: I/O 18 QID 0 timeout, disable controller`。
- 随后报错：`nvme nvme0: failed to set APST feature (-4)`。
- 系统检测到任务阻塞：`INFO: task rs:main Q:Reg:519 blocked for more than 60 seconds`。
- 最终触发 Kernel panic：`Kernel panic - not syncing: hung_task: blocked tasks`。

#问题根因
NVMe 驱动在初始化过程中尝试设置 APST (Autonomous Power State Transition) 功能时失败（返回错误码 -4），导致控制器被禁用。驱动程序未能正确处理 APST 和指令启用时的超时情况，导致初始化流程中断，进而引发 I/O 阻塞和系统崩溃。

#解决方案
临时规避方案：在内核启动参数中添加 `nvme_core.default_ps_max_latency_us=0` 以禁用 APST 功能。开发者计划提交补丁修复驱动在超时时的处理逻辑。
