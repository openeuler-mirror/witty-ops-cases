#标题
USB cdc-acm模块因urb类型检查逻辑错误触发虚假内核告警

#内核版本
4.4.0

#问题现象
系统日志被大量内核警告 "BOGUS urb xfer, pipe %x != type %x" 刷屏。

#问题根因
在 `urb.c` 中，传输类型健全性检查逻辑存在缺陷。代码将 `usb_pipetype(urb->pipe)` 与 `pipetypes[xfertype]` 进行比较，但 `pipetypes` 数组定义的 ID 属于 `usbfs` 而非 USB endpoint，导致比较失败并错误地触发告警。

#解决方案
用户建议修改 `urb.c` 中的 `dev_WARN` 调用，将参数 `pipetypes[xfertype]` 替换为 `xfertype`。维护者建议先在 4.6 内核版本上验证并提交至 linux-usb 邮件列表。暂无已合并的修复补丁信息。
