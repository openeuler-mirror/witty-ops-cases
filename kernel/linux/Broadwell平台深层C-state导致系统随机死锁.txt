#标题
Broadwell平台深层C-state导致系统随机死锁

#内核版本
4.8 (受影响版本涵盖 4.8 至 5.3)

#问题现象
- 系统在使用过程中随机发生完全冻结，无任何内核日志或报错信息，只能通过长按电源键强制重启。
- 故障主要出现在搭载 Broadwell 架构 CPU 的机型上（如 ThinkPad X1 Carbon 3rd/4th gen, T450, Dell 7310 等）。
- 故障常在播放 HTML5 视频或进行中等负载操作时触发。
- 使用 `nomodeset` 参数启动或限制 C-state 后系统稳定。

#问题根因
CPU 进入深层 C-state（C3 及更深状态）时触发硬件层面的死锁。用户测试确认 `intel_idle.max_cstate=3` 会导致故障复现，而限制为 `=2` 则稳定。此外，问题可能与 SATA Link Power Management 策略（`med_power_with_dipm`）与深层 C-state 的交互有关，当 SATA 电源策略设置为 `max_performance` 时稳定性显著提升。

#解决方案
截止报告时间暂无正式内核补丁修复，可通过以下方式规避：
1. 在 GRUB 内核启动参数中添加 `intel_idle.max_cstate=2`。
2. 或添加内核参数 `ahci.mobile_lpm_policy=1`，或手动设置 SATA 电源策略为 `max_performance`。
