#标题
kswapd回收shmem大页时触发iput函数kernel BUG

#内核版本
4.9.33-0-beget-acl

#问题现象
- 内核日志报错：kernel BUG at fs/inode.c:1535!
- 触发 invalid opcode 异常（0000 [#1]），导致系统崩溃。
- 故障进程为 kswapd0，RIP 寄存器指向 iput+0x1dc/0x200。
- 随后在处理该错误退出路径时，于 __wake_up_common 处触发 NULL pointer dereference。

#问题根因
故障发生在内存回收路径。kswapd0 进程调用 shrink_slab -> super_cache_scan -> shmem_unused_huge_shrink 函数尝试回收 shmem 大页，在调用 iput 释放 inode 时，触发了 fs/inode.c:1535 处的 BUG() 宏。这通常意味着 inode 状态不符合预期（如引用计数或状态标志异常），但具体触发逻辑未在报告中明确。

#解决方案
暂无
