#标题
蓝牙 BNEP 设备从挂起恢复时触发 sysfs 警告及 RCU Stall

#内核版本
4.1.12

#问题现象
- 系统从挂起恢复且此前连接过 bnep 网络时，dmesg 报告内核 WARNING：`WARNING: CPU: 1 PID: 3543 at fs/sysfs/group.c:224 netdev_queue_update_kobjects+0xc0/0x120()`
- 日志显示 `sysfs group ffffffff81c9cdc0 not found for kobject 'tx-0'`
- 进程 `kbnepd bnep0` CPU 占用率达 100%，系统负载升高。
- 内核报告 RCU Stall：`INFO: rcu_sched self-detected stall on CPU { 1}`。
- 系统无法正常重启，处于挂起状态。

#问题根因
在 BNEP 会话结束注销网络设备过程中（调用栈：`bnep_session` -> `unregister_netdev` -> `netdev_unregister_kobject` -> `netdev_queue_update_kobjects`），内核试图移除 `tx-0` 对应的 sysfs 组，但该 sysfs 组未找到，触发异常路径。`kbnepd` 线程随后卡死，导致 CPU 100% 及 RCU Stall。

#解决方案
暂无
