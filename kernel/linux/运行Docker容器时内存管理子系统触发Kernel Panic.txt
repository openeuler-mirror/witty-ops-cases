#标题
运行Docker容器时内存管理子系统触发Kernel Panic

#内核版本
4.19.81

#问题现象
系统运行 Docker 及容器时随机触发 Kernel Panic，具体表现为以下几种形式：
- 现象1：报错 "BUG: unable to handle kernel paging request"，RIP 位于 `page_remove_rmap` 函数，Call Trace 显示路径为 `unmap_page_range` -> `exit_mmap` -> `do_exit`。
- 现象2：触发 "general protection fault"，RIP 位于 `__unlock_page_memcg` 函数，Call Trace 显示路径为 `alloc_set_pte` -> `filemap_map_pages` -> `handle_mm_fault`。
- 现象3：触发 "general protection fault"，RIP 位于 `__dec_node_state` 函数，Call Trace 显示路径为 `unmap_page_range` -> `exit_mmap`。
- 硬件内存检测未发现异常。

#问题根因
未明确。根据堆栈信息，故障发生在内存管理核心路径（如页面反向映射移除、memcg 锁操作、节点状态更新），涉及进程退出或缺页处理时的内存结构访问，疑似内存结构损坏或并发访问异常，但具体根因尚未确定。

#解决方案
暂无。开发者建议测试最新主线内核（v6.4-rc1）以确认问题是否已在上游修复，但因缺乏可靠复现步骤，目前尚未定位具体修复补丁。
