#标题
OHCI驱动在ueagle-atm断开连接时触发NULL或LIST_POISON解引用导致panic

#内核版本
4.4.11

#问题现象
系统发生 kernel panic。在断开 ueagle-atm 设备（Eagle III USB ADSL modem）时触发，特别是在连接了 USB 键盘或启动 X 图形界面的环境下。日志显示在 `ohci-q.c` 的 `finish_unlinks` 函数中发生 NULL 或 LIST_POISON 解引用。

#问题根因
故障位于 `drivers/usb/host/ohci-q.c` 文件的 `finish_unlinks` 函数中。在第 1086 行执行 `list_del(&ed->in_use_list)` 时，由于 `in_use_list` 的 `->next` 和 `->prev` 指针为 NULL 或 LIST_POISON，导致解引用失败触发 panic。具体触发场景与 ueagle-atm 设备断开连接时的并发处理有关。

#解决方案
暂无修复补丁。维护者建议将问题报告发送至 linux-usb@vger.kernel.org 邮件列表。
