#标题
amdgpu驱动初始化时AtomBIOS死循环导致GPU加载失败

#内核版本
4.18.14-300.fc29.x86_64

#问题现象
- 系统环境为双显卡（Intel + AMD），OS 使用 Intel GPU 启动，AMD GPU 初始化失败。
- 内核日志报错：`[drm:atom_op_jump [amdgpu]] *ERROR* atombios stuck in loop for more than 5secs aborting`。
- 日志显示执行表 E8D2 和 C57A 卡死。
- 随后报错：`amdgpu 0000:01:00.0: gpu post error!` 和 `Fatal error during GPU init`。
- 最终驱动探测失败：`amdgpu: probe of 0000:01:00.0 failed with error -22`。

#问题根因
amdgpu 驱动在执行 AtomBIOS 命令表时陷入死循环。具体表现为 `atom_op_jump` 操作超时（超过 5 秒），导致 GPU 提交过程无法完成，驱动初始化流程中止。

#解决方案
暂无。Bug 处于 NEW 状态，开发者已请求提供完整 dmesg 日志以进一步分析。
