#标题
ath9k驱动在关机时同步未分配DMA内存触发内核警告

#内核版本
4.4.30

#问题现象
- 系统关机时内核日志打印 WARNING 信息，提示 DMA-API 错误。
- 报错信息：`ath9k 0000:03:00.0: DMA-API: device driver tries to sync DMA memory it has not allocated [device address=0x00000000d3152b84] [size=30 bytes]`。
- 堆栈跟踪显示调用链为 `ath_tx_cabq` -> `ath9k_beacon_tasklet` -> `debug_dma_sync_single_for_device` -> `check_sync`。
- 该问题为回归问题，在 4.1.X 内核版本中未出现。

#问题根因
ath9k 驱动程序在处理 beacon tasklet 时，于 `ath_tx_cabq` 函数中尝试同步（sync）一块未被分配或已释放的 DMA 内存区域，触发了 DMA 调试机制的检查警告。

#解决方案
暂无
