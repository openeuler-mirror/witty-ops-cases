#标题
Ryzen 2400G 平台 amdgpu 驱动 sdma0 超时导致系统随机冻结

#内核版本
4.17 RC2

#问题现象
系统在日常使用（非 3D 负载）时随机冻结。
内核日志中出现如下报错：
- [drm:amdgpu_job_timedout [amdgpu]] *ERROR* ring sdma0 timeout, last signaled seq=2081253, last emitted seq=2081256
- [drm] No hardware hang detected. Did some blocks stall?

#问题根因
SDMA0 ring 发生超时，导致 GPU 任务无法完成。AMD 开发人员推测可能与 VM 更新模式有关，建议尝试切换 VM 更新方式。

#解决方案
在内核启动参数（GRUB）中添加 `amdgpu.vm_update_mode=3`，强制使用 CPU 进行 VM 更新。报告者反馈该参数生效后未再出现冻结。
