#标题
ext4文件系统在coredump过程中触发kernel BUG导致系统崩溃

#内核版本
3.10.0-123.20.1.el7.x86_64

#问题现象
- 系统日志打印 "kernel BUG at fs/buffer.c:1270!" 并触发 "invalid opcode: 0000 [#1]" 错误。
- 故障发生在进程 `policy_server` 执行 `do_coredump` 流程中，调用栈显示路径为 `elf_core_dump` -> `ext4_file_write` -> `__find_get_block`。
- 多台 Dell 服务器出现相同故障。

#问题根因
故障点位于 `fs/buffer.c:1270` 的 `__find_get_block` 函数中。经分析，此处触发了 `BUG_ON(irqs_disabled())` 断言失败，表明在执行 core dump 写入文件的操作路径中，中断被意外禁用。该现象与 XFS 文件系统曾报告的类似问题（中断禁用状态异常）一致，具体导致中断禁用的根源在报告时尚未明确。

#解决方案
暂无。该问题涉及修改过的旧版本发行版内核，且在相关 bug 跟踪系统中未见有效修复方案。
