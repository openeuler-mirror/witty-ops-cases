#标题
UDF文件系统设备异常移除导致内核分页请求错误

#内核版本
4.2.0

#问题现象
- UDF文件系统格式的USB设备在未执行卸载操作的情况下被物理拔出，触发内核Oops。
- 日志显示 "BUG: unable to handle kernel paging request at 361f2000"。
- Call Trace: `__percpu_counter_add` <- `account_page_dirtied` <- `__set_page_dirty` <- `mark_buffer_dirty` <- `udf_close_lvid` <- `udf_put_super`。

#问题根因
设备异常断开后，内核执行文件系统卸载流程。在 `udf_put_super` 调用 `udf_close_lvid` 关闭逻辑卷完整性描述符时，尝试通过 `mark_buffer_dirty` 修改页面状态，此时访问了无效内存地址，导致无法处理内核分页请求。

#解决方案
暂无
