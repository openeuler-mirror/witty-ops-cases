#标题
CIFS模块挂载DFS共享时触发空指针解引用导致Kernel Panic

#内核版本
5.9.0-rc1

#问题现象
- 系统在执行 CIFS 挂载操作时触发 Kernel Panic，导致内核崩溃。
- 日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000790"。
- Oops 发生在 `cifs_mount+0x498/0xe30 [cifs]`，寄存器 CR2 值为 0000000000000790。
- 触发场景为：客户端尝试使用 `-o sec=none` 参数挂载服务端配置为 DFS root 的共享目录。

#问题根因
在 `cifs_mount` 函数执行过程中，代码试图访问偏移量为 0x790 的空指针地址。该问题发生在 CIFS 文件系统处理 DFS（Distributed File System）共享挂载请求的路径中，具体触发条件涉及特定的认证方式（sec=none）与服务端 DFS 配置的组合。

#解决方案
暂无。Bug 当前状态为 NEW，开发者正在尝试复现并请求更多调试信息以定位具体代码行。
