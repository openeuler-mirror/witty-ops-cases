#标题
Btrfs挂载损坏镜像时btrfs_search_slot函数触发空指针解引用

#内核版本
5.17-rc3, 5.17-rc4

#问题现象
- 内核触发 Oops，日志显示 "BUG: kernel NULL pointer dereference, address: 0000000000000208"
- RIP 寄存器指向 btrfs_search_slot+0x19/0x13a0
- 调用栈包含 btrfs_insert_empty_items -> btrfs_insert_item -> btrfs_create_pending_block_groups -> __btrfs_end_transaction
- 问题复现步骤为挂载特定的损坏镜像并执行文件操作（如 cp）

#问题根因
系统挂载损坏的 Btrfs 镜像文件时，内核在处理 pending block groups 创建逻辑中调用 btrfs_search_slot 函数，因异常数据导致访问了空指针（偏移 0x208），触发 supervisor read access 权限错误。

#解决方案
暂无
