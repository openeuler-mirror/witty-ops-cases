#标题
amdgpu驱动dc_stream_log函数空指针解引用导致系统死锁

#内核版本
5.2.1

#问题现象
- 系统在观看视频（如 YouTube）或进行 GPU 相关操作时，出现视频输出完全锁死，无法通过电源按钮关机，需使用 Magic SysRq 键重启。
- 内核日志报错："BUG: kernel NULL pointer dereference, address: 00000000000002b4"。
- RIP 寄存器指向 `dc_stream_log+0x6/0xb0 [amdgpu]`。
- 调用栈显示流程为 `commit_work` -> `amdgpu_dm_atomic_commit_tail` -> `dc_commit_state` -> `dc_stream_log`。

#问题根因
在 `amdgpu` 驱动的 atomic commit 处理流程中，`dc_stream_log` 函数试图访问一个为 NULL 的 stream 指针。开发者分析认为，这是从 5.1 升级到 5.2 过程中，补丁丢失或错误回退导致的回归问题，使得 stream 指针在不应该为 NULL 的上下文中出现了空值。

#解决方案
- 官方修复：应用补丁系列 https://patchwork.freedesktop.org/series/64505/，该补丁已提交至 stable 内核分支。
- 临时规避方案：
  1. 在 Xorg 配置中禁用 `EnablePageFlip` 和 `TearFree` 选项。
  2. 卸载 `xf86-video-amdgpu` 包，改用 modesetting 驱动。
