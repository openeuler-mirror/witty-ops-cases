#标题
Memory cgroup下SLUB分配失败导致进程IO挂死

#内核版本
4.4.73

#问题现象
- Kubernetes节点上的MySQL进程挂起且无法被杀死
- 执行strace追踪进程时，strace进程同样挂起
- dmesg日志显示大量 "SLUB: Unable to allocate memory on node -1 (gfp=0x2088020)" 报错，涉及 cache: blkdev_ioc
- 内核日志报错 "INFO: task mysqlctld:104941 blocked for more than 120 seconds"，进程处于 D 状态

#问题根因
进程在 Memory cgroup 内存限制下触发内存回收，回收路径中的 XFS 回写操作尝试分配内存，触发了递归依赖或死锁，导致 SLUB 分配器在 node -1 上无法分配内存，进程陷入不可中断睡眠状态。

#解决方案
升级内核至 4.9.35 版本可规避该问题（用户反馈在 4.9.35 版本运行稳定，而 4.4.73 版本在 12 小时内即可复现）。
