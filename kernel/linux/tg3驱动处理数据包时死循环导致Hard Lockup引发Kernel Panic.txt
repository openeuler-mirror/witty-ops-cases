#标题
tg3驱动处理数据包时死循环导致Hard Lockup引发Kernel Panic

#内核版本
4.18.0-147.5.2.10.x86_64

#问题现象
- 使用 TG3 网卡进行大量网络收发测试时，系统触发 Kernel Panic
- 日志显示 "Kernel panic - not syncing: Hard LOCKUP"
- 堆栈信息显示 CPU 在处理 `tg3_poll_msix` 和 `tg3_poll_work` 时被 NMI watchdog 检测到锁死

#问题根因
NMI watchdog 检测到 CPU 硬锁死并触发 panic。故障现场堆栈显示 CPU 处于 `tg3_poll_msix` -> `tg3_poll_work` 调用链中。报告者分析认为，在 `tg3_poll_msix`、`tg3_rx` 等函数处理数据包类型判断时发生了死循环，导致 CPU 无法响应。

#解决方案
暂无
