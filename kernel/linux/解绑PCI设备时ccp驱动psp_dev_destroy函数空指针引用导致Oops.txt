#标题
解绑PCI设备时ccp驱动psp_dev_destroy函数空指针引用导致Oops

#内核版本
4.17.11-200.fc28.x86_64

#问题现象
- 执行 `echo $dev > /sys/bus/pci/devices/$dev/driver/unbind` 解绑操作时触发内核 Oops。
- 日志显示 "BUG: unable to handle kernel NULL pointer dereference at 0000000000000060"。
- 进程 `vfio-bind` 触发崩溃，RIP 寄存器指向 `psp_dev_destroy+0x14/0x60 [ccp]`。

#问题根因
在 `ccp` 驱动模块的 `psp_dev_destroy` 函数中，RSI 寄存器值为 0（空指针），代码尝试访问其偏移量 0x60 处的数据（对应指令 `cmp qword ptr [rsi+0x60], 0`），导致页面错误。
调用链为：`unbind_store` -> `device_release_driver_internal` -> `pci_device_remove` -> `sp_pci_remove` -> `sp_destroy` -> `psp_dev_destroy`。

#解决方案
暂无
