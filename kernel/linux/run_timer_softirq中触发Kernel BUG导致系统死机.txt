#标题
run_timer_softirq中触发Kernel BUG导致系统死机

#内核版本
3.18.13

#问题现象
- 系统在进行 Wi-Fi 测试时随机出现死机。
- 控制台输出 "Fatal exception in interrupt"。
- 调用栈显示：`run_timer_softirq()->__run_timers()->cascade()->BUG_ON(tbase_get_base(timer->base) != base)`。

#问题根因
在内核定时器处理流程中，`cascade()` 函数检测到定时器 base 不一致，触发 `BUG_ON` 导致系统 panic。故障发生在用户自研 Wi-Fi 驱动测试期间，怀疑与驱动对定时器的操作有关。

#解决方案
暂无
