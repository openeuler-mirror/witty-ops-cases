#标题
Radeon驱动读取DP链路带宽错误导致iMac启动黑屏

#内核版本
4.15.0-20

#问题现象
- 系统启动进入 Kernel Modesetting (KMS) 阶段后屏幕黑屏。
- 尝试通过 sysfs 或 xrandr 启用屏幕时，dmesg 报告内核 WARNING。
- 日志显示：`unknown DP link BW code 0, using 162000`。
- 堆栈信息指向 `drm_dp_bw_code_to_link_rate+0x41/0x50 [drm_kms_helper]`，调用链包含 `radeon_dp_get_dp_link_config` 和 `radeon_dp_mode_valid_helper`。

#问题根因
Radeon 驱动在处理 Mobility Radeon HD 4670 (RV730/M96-XT) 的 eDP 连接时，`radeon_dp_get_dp_link_config` 函数获取到的 DP 链路带宽代码为 0（无效值）。该值传递给 `drm_dp_bw_code_to_link_rate` 函数时触发了 `WARNING`，导致显示输出初始化失败。

#解决方案
暂无官方修复。临时规避方法：在黑屏状态下盲打执行命令 `xrandr -d :0 --output eDP-1 --crtc 1` 可恢复显示。
