#标题
rt2500usb驱动频繁触发TX DMA超时导致无线网络连接缓慢

#内核版本
3.14（受影响版本包括 3.10 至 3.14，3.9 及之前版本正常）

#问题现象
- 使用 DWL-G122 无线网卡（Ralink RT2571 芯片）时，网络连接速度极慢。
- dmesg 日志频繁报错：`ieee80211 phy0: rt2x00usb_watchdog_tx_dma: Warning - TX queue 0 DMA timed out, invoke forced forced reset`。
- 伴随出现 `rt2x00queue_flush_queue: Warning - Queue 0 failed to flush` 以及 `rt2x00lib_autowakeup: Error - Device failed to wakeup` 错误。
- 问题为回归问题，内核 3.9 版本下虽有日志告警但网速正常，升级至 3.10 后出现严重性能下降。

#问题根因
用户通过 git bisect 初步定位问题引入于内核版本 3.9 至 3.10 之间，二分查找结果指向合并提交 `ded652a67464b1fa66616954bc608ead9ec02fb6` (Merge branch 'for-john' of mac80211-next) 或 wlcore 相关改动，但具体代码根因尚未经内核开发者确认。

#解决方案
暂无
