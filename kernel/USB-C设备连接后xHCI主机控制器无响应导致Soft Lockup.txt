#标题
USB-C设备连接后xHCI主机控制器无响应导致Soft Lockup

#内核版本
4.6.3

#问题现象
- Dell XPS 9550 连接 Android 手机至 USB-C 集线器数秒后，主机控制器崩溃，USB-C 端口功能失效。
- 内核日志报错：`xhci_hcd 0000:3e:00.0: Timeout while waiting for setup device command`。
- 控制器停止响应，日志显示 `xhci_hcd 0000:3e:00.0: Assuming host is dying, halting host` 及 `HC died; cleaning up`。
- 触发 NMI watchdog 报告 `BUG: soft lockup - CPU#0 stuck for 23s!`。
- 调用栈显示 CPU 卡在 `xhci_handshake` 函数中。

#问题根因
xHCI 主机控制器（0000:3e:00.0）在处理 USB 设备连接命令时无响应，导致命令超时。驱动程序尝试停止端点命令和中止命令环均失败，判定主机已“死亡”并进行清理。在此过程中，`xhci_handshake` 函数陷入循环等待状态，导致 CPU 0 发生 soft lockup。

#解决方案
暂无
