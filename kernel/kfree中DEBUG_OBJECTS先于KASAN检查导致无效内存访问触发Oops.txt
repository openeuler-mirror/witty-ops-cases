#标题
kfree中DEBUG_OBJECTS先于KASAN检查导致无效内存访问触发Oops

#内核版本
5.19.0-rc6 (commit 4a57a8400075)

#问题现象
- 内核触发 page fault，导致 Oops：`BUG: unable to handle page fault for address: ffffed11035786bc`
- 错误码显示为 `Oops: 0000 [#1] PREEMPT SMP KASAN`
- 调用栈指向 `debug_check_no_obj_freed+0x101/0x420`，由 `kfree` 函数调用触发。

#问题根因
当释放的对象无效（如 double-free 或未初始化指针）时，`mm/slab.c` 中的 `kfree` 实现先执行了 `DEBUG_OBJECTS` 检查，后执行 `KASAN` 检查。由于指针无效，`DEBUG_OBJECTS` 在访问内存时直接触发 page fault，导致系统崩溃。若先执行 `KASAN` 检查，可确保地址有效或由 KASAN 捕获错误，避免此类 Oops。

#解决方案
建议调整 `kfree` 中的检查顺序，将 KASAN 检查移至 DEBUG_OBJECTS 检查之前执行。当前 Bug 状态为 NEW，暂无已合并的修复补丁。
