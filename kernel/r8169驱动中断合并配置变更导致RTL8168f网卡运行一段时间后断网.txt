#标题
r8169驱动中断合并配置变更导致RTL8168f网卡运行一段时间后断网

#内核版本
5.4.31（受影响版本包括 5.4.x 至 5.15.x）

#问题现象
- 系统运行一段时间后网络连接中断，网卡停止工作。
- dmesg 日志中出现 "NETDEV WATCHDOG: ... (r8169): transmit queue 0 timed out" 报错。
- 该问题在内核 4.19.97 版本上未复现，属于回归问题。

#问题根因
通过 git bisect 定位到问题引入提交为 `288ac524cf70a8e7ed851a61ed2a9744039dae8d` ("r8169: disable default rx interrupt coalescing on RTL8168")。该提交默认禁用了 RX interrupt coalescing，在 RTL8168f 芯片上触发了未知的硬件缺陷或特定行为，导致网卡停止响应。开发者推测可能与 `PktCntrDisable` 标志位处理或 PHY 复位时序有关，但确切根因未明确。

#解决方案
- 临时规避方案：安装并使用 Realtek 官方厂商驱动 `r8168` 替代内核自带 `r8169` 驱动。
- 尝试方案：通过 ethtool 调整中断合并参数（例如 `ethtool --coalesce <interface> rx-usecs 0 rx-frames 0 tx-usecs 200 tx-frames 0`），部分测试反馈需配合特定补丁（设置 `PktCntrDisable` 位）才能生效。
- 截止报告时间，Bug 状态为 NEW，尚无明确的内核主线修复补丁合并。
