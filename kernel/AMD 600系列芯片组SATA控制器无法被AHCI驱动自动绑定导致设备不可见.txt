#标题
AMD 600系列芯片组SATA控制器无法被AHCI驱动自动绑定导致设备不可见

#内核版本
6.19.5

#问题现象
- AMD 600系列芯片组SATA控制器（PCI ID 1022:43f6）在启动时无法被AHCI驱动自动绑定，SATA设备不可见
- dmesg中无ahci相关输出，驱动从未尝试探测该设备
- 手动执行 `echo "1022 43f6" > /sys/bus/pci/drivers/ahci/new_id` 或绑定设备后，驱动可正常工作
- 使用udev规则在initramfs阶段绑定成功后，在initramfs到root切换过程中驱动分离，pivot到真实root后不再重新绑定

#问题根因
初步分析：设备的class code (010601) 匹配通用AHCI条目，理论上应自动绑定，但实际未触发。设备位于PCIe switch后（0e:00.0），可能涉及设备枚举时机或驱动探测顺序问题，具体根因仍在调查中。

#解决方案
临时规避方案：
1. 手动绑定：`echo "0000:0e:00.0" > /sys/bus/pci/drivers/ahci/bind`
2. 通过udev规则在initramfs阶段写入new_id（但存在pivot后驱动分离问题）
永久修复方案：暂无，问题仍在调查中。
