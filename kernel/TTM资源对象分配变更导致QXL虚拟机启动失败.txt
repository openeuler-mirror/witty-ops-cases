#标题
TTM资源对象分配变更导致QXL虚拟机启动失败

#内核版本
5.14.0-rc1+

#问题现象
使用 QXL 显示设备的 KVM 虚拟机在启动过程中失败。Call Trace 显示崩溃发生在 qxl 模块中：
- RIP: qxl_bo_move_notify.part.1+0xc/0x40 [qxl]
- ttm_bo_cleanup_memtype_use+0x22/0x60 [ttm]
- ttm_bo_release+0x1b0/0x310 [ttm]

#问题根因
问题由 Git 提交 bfa3357ef9abc9d56a2910222d2deeb9f15c91ff ("drm/ttm: allocate resource object instead of embedding it v2") 引入。该提交修改了 TTM 资源对象的分配方式（由嵌入改为动态分配），导致 QXL 驱动在处理 buffer object 移动通知时出现异常，进而引发启动失败。

#解决方案
暂无（Bug 状态为 NEW，尚未修复）
