#标题
tg3驱动因IOMMU交互异常导致网卡transmit timed out失效

#内核版本
3.13（报告中涉及版本跨度从 2.6.28 至 5.14）

#问题现象
- 系统日志报错 "NETDEV WATCHDOG: eth0 (tg3): transmit timed out"。
- 驱动尝试复位网卡时失败，日志显示 "tg3_abort_hw timed out"，MAC_TX_MODE 等寄存器读取值全为 0xffffffff。
- 网卡彻底失去响应，无法收发报文，ifdown/ifup 操作无法恢复，必须重载驱动（rmmod/modprobe）或重启系统。
- 故障发生时，调试补丁检测到 "PCI Memory Mapped IO Disabled"，部分 AMD 平台伴随 AMD-Vi IO_PAGE_FAULT 错误。

#问题根因
Broadcom 开发者分析发现，故障发生时 Write DMA Status Register (0x4c04) 的 BIT 5 (Write DMA PCI Host Address Overflow Error) 被置位。该问题与 IOMMU 功能开启强相关。当 IOMMU 开启时，DMA 传输可能触发硬件溢出错误，导致 PCI 命令寄存器中的 Memory Enable bit 被意外清除，致使驱动无法访问设备 MMIO 空间，进而引发传输超时且复位失败。

#解决方案
暂无官方补丁修复。
规避方案：在内核启动参数中添加 `amd_iommu=off` 或 `intel_iommu=off` 禁用 IOMMU 功能。
