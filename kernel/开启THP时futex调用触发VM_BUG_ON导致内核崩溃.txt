#标题
开启THP时futex调用触发VM_BUG_ON导致内核崩溃

#内核版本
2.6.32

#问题现象
- 系统偶发 kernel BUG，日志显示 "kernel BUG at mm/filemap.c:588!" 及 "invalid opcode: 0000 [#1]"。
- 触发 BUG 时的 Call Trace 为：`unlock_page` <- `get_futex_key` <- `futex_wait_setup` <- `futex_wait` <- `do_futex` <- `sys_futex`。

#问题根因
在 2.6.32 版本内核中，透明大页（THP）功能在 `get_futex_key` 函数中的处理逻辑存在缺陷。当涉及 THP 页面时，页面锁定状态管理不一致，导致在 `unlock_page` 函数中触发 `VM_BUG_ON(PageLocked(page))` 检查失败。

#解决方案
规避措施：关闭透明大页功能，执行命令 `echo never > /sys/kernel/mm/redhat_transparent_hugepage/enabled`。
根本措施：升级内核版本，早期 THP 实现中的相关缺陷已在后续版本中修复。
