#标题
连接蓝牙耳机时 list_add 双重添加导致 kernel BUG 及 soft lockup

#内核版本
5.2.9-200.fc30

#问题现象
- 连接蓝牙耳机时触发 kernel BUG，系统日志显示 "list_add double add" 错误，随后出现 "invalid opcode: 0000 [#1]" 异常。
- Call Trace 显示路径为：`__list_add_valid` -> `kobject_add_internal` -> `device_add` -> `hci_conn_add_sysfs`。
- 故障发生后，系统极大概率出现 soft lockup，CPU 卡死在 `queued_spin_lock_slowpath`，导致系统无响应。
- 硬件环境为 ThinkPad x230 配置 BCM20702A1 蓝牙芯片，在固件文件 `brcm/BCM20702A1-0a5c-21e6.hcd` 存在时 100% 复现。

#问题根因
蓝牙子系统在处理连接事件时，尝试将同一个 kobject 对象重复添加到内核链表中（double add）。
具体触发位置在 `hci_conn_add_sysfs` 函数调用 `device_add` 过程中，由于设备已被注册，导致 `lib/list_debug.c` 检测到链表节点异常（new=prev=next）并触发 BUG。
该问题是内核版本升级（约 5.0 到 5.1 之间）引入的 regression。

#解决方案
暂无
