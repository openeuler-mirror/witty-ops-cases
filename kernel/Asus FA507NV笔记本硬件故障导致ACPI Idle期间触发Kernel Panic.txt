#标题
Asus FA507NV笔记本硬件故障导致ACPI Idle期间触发Kernel Panic

#内核版本
6.10 至 7.0（评论提及）

#问题现象
- 多个 Linux 发行版启动后随机发生 Kernel Panic，错误信息显示 "Attempted to kill idle task!"
- Call Trace 显示调用栈为：Acpi_idle_bm_check+0x99/0x100 -> Acpi_idle_enter_bm+0x1b/0x100 -> Cpuidle_enter_state -> Cpuidle_enter -> Cpu_startup_entry
- 启动参数添加 acpi=off 可暂时规避，但严重影响系统功能。
- Windows 系统下同样出现蓝屏（BSOD），错误代码包括 IRQL_NOT_LESS_OR_EQUAL、Hypervisor Error 等。

#问题根因
硬件故障。维护者分析指出，该问题在 Windows 和 Linux 下均复现，且 BIOS 更新仅能短暂缓解，判定为硬件（主板/BIOS 芯片）损坏或不稳定，导致 ACPI idle 状态切换异常。

#解决方案
该问题非内核软件缺陷，建议更换主板或联系硬件厂商进行维修。
