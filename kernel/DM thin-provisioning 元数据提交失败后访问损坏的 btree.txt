#标题
DM thin-provisioning 元数据提交失败后访问损坏的 btree

#内核版本
6.1.0-rc5

#问题现象
- 元数据提交操作 `dm_pool_commit_metadata` 失败，返回错误码 -5
- 内核日志显示 "aborting current metadata transaction"
- 随后系统访问了损坏的 btree 节点，日志显示 `pmd->root` 指向的 block 31 为过期节点，`nr_entries` 数值异常，且包含大量无效 value 值
- 触发 EXT4-fs I/O error 和 Buffer I/O error

#问题根因
在 DM thin-provisioning 模块中，当 `dm_pool_commit_metadata` 提交失败并执行事务中止逻辑时，元数据状态未正确回滚或恢复，导致 `pmd->root` 指针指向了已损坏或过期的 btree 节点，后续对该节点的访问读取到了无效数据。

#解决方案
暂无
