#标题
RISC-V平台nf_hook_slow访问非法地址导致Kernel Panic

#内核版本
5.10.41

#问题现象
- RISC-V Unmatched 开发板在安装 docker 和 ufw 后，系统空闲时触发 Kernel Panic。
- 日志显示 "Unable to handle kernel paging request at virtual address 0000005f81a3e164"。
- 触发 Oops [#1]，异常发生在中断上下文，寄存器 epc 指向非法地址 ffffffdf81a3e164，ra 指向 fffffffe000b44e58。
- 系统打印 "Kernel panic - not syncing: Fatal exception in interrupt" 并停止运行。

#问题根因
- 根据用户使用 addr2line 分析，返回地址 ra 对应源码 net/netfilter/core.c:590 行，位于 nf_hook_slow 函数中。
- 系统在执行 Netfilter hook 处理逻辑时，试图访问非法虚拟地址（badaddr: 0000005f81a3e164），触发页面错误异常，导致内核崩溃。具体的内存损坏或指针错误来源在报告中未明确。

#解决方案
暂无（Bug 状态为 NEW，用户尝试升级内核至 5.10.130 问题依旧）。
