#标题
XFS日志释放过程中竞态条件导致断言失败引发内核Oops

#内核版本
linux 5.5+ with xfs-linux xfs-5.6-merge-7

#问题现象
- 运行 xfstests generic/475 测试用例时触发内核崩溃
- 内核日志打印断言失败信息：XFS: Assertion failed: iclog->ic_state == XLOG_STATE_ACTIVE, file: fs/xfs/xfs_log.c, line: 572
- 触发 kernel BUG at fs/xfs/xfs_message.c:110!，导致 Oops: Exception in kernel mode
- Call Trace 指向 xlog_cil_push_work -> xfs_log_release_iclog -> __xlog_state_release_iclog

#问题根因
xfs_log_release_iclog() 函数中的 XLOG_STATE_IOERROR 检查与文件系统 shutdown 流程存在竞态条件。该检查发生在获取 l_icloglock 锁之前，此时若另一任务调用 xfs_log_force_umount() 重置了所有 iclogs 状态，当前任务获取锁后会发现 iclog 状态非预期的 XLOG_STATE_ACTIVE，从而触发断言失败。该问题由 commit df732b29c8 引入。

#解决方案
开发者已发布修复补丁，详见：https://lore.kernel.org/linux-xfs/20200214181528.24046-1-bfoster@redhat.com/
