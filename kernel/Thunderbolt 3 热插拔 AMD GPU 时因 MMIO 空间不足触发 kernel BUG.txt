#标题
Thunderbolt 3 热插拔 AMD GPU 时因 MMIO 空间不足触发 kernel BUG

#内核版本
4.19-rc4

#问题现象
- 通过 Thunderbolt 3 热插拔 AMD R9 Fiji XT GPU 时，屏幕冻结，系统触发 kernel BUG。
- 日志显示 "kernel BUG at arch/x86/mm/pat.c:549!"。
- dmesg 显示 PCI BAR 空间分配失败，例如 "BAR 0: no space for [mem size 0x10000000 64bit pref]" 和 "BAR 0: failed to assign"。
- 系统仍可通过 SSH 访问，但图形界面无响应。

#问题根因
平台未提供足够大的 MMIO 窗口，导致内核无法为热插拔的 GPU 设备分配所需的 PCI BAR 空间。AMDGPU 驱动在 BAR 分配失败的情况下尝试初始化设备，触发了 arch/x86/mm/pat.c 中的 BUG。相比之下，若设备在启动前已连接（Pre-boot ACL），BIOS 能够正确分配资源，则不会触发此问题。

#解决方案
暂无内核代码修复。建议检查 BIOS 设置中是否开启 "Enable MMIO >4GB" 选项以扩大 MMIO 窗口，或避免在系统运行时热插拔该设备，改为启动前连接。
