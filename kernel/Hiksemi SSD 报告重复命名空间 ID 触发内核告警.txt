#标题
Hiksemi SSD 报告重复命名空间 ID 触发内核告警

#内核版本
6.1.15-1

#问题现象
- 系统日志中出现 "nvme nvme1: globally duplicate IDs for nsid 1" 错误信息
- 日志显示 "nvme nvme1: missing or invalid SUBNQN field"
- 涉及设备为 Hiksemi HS-SSD-FUTURE 2048G (VID:DID 1e4b:1602)

#问题根因
该型号 SSD 固件实现存在缺陷，向内核报告了无效或重复的命名空间标识符，违反了 NVMe 规范中关于 ID 唯一性的要求。

#解决方案
上游已提交补丁，为该设备添加 NVME_QUIRK_BOGUS_NID 标志以忽略错误的 ID。修复补丁已进入 6.1 和 6.3 稳定版队列，具体修改为在 `pci.c` 中添加设备 ID `0x1e4b:0x1602` 的 quirk 条目。
