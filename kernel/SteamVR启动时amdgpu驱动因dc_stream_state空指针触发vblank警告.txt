#标题
SteamVR启动时amdgpu驱动因dc_stream_state空指针触发vblank警告

#内核版本
5.16.9（评论反馈受影响版本涵盖 5.15.32 至 5.18.9）

#问题现象
- 启动 SteamVR（配合 Valve Index 或 HTC Vive HMD）时系统出现卡顿或锁定。
- 内核日志报错：`[drm:dm_vblank_get_counter] *ERROR* dc_stream_state is NULL for crtc '1'!`。
- 触发内核 WARNING：`WARNING: CPU: X PID: XXX at drivers/gpu/drm/drm_vblank.c:728 drm_crtc_vblank_helper_get_vblank_timestamp_internal`。
- Call Trace 包含：`drm_crtc_queue_sequence_ioctl` -> `drm_vblank_get` -> `drm_vblank_enable` -> `drm_update_vblank_count` -> `drm_get_last_vbltimestamp`。

#问题根因
amdgpu 驱动在处理 vblank 计数器获取请求时，对应的 CRTC '1' 的 `dc_stream_state` 结构体指针为 NULL，导致 `dm_vblank_get_counter` 和 `dm_crtc_get_scanoutpos` 函数报错，进而引发 `drm_crtc_vblank_helper_get_vblank_timestamp_internal` 触发断言警告。

#解决方案
暂无（Bug 状态为 NEW，评论中提及已定位到特定 commit 但未提供修复补丁）。
