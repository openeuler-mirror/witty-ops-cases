#标题
amdgpu驱动在高负载GPU复位期间访问空指针导致内核崩溃

#内核版本
5.3.15

#问题现象
- 系统负载极高（load average 约 600）时，GPU 触发复位，随即发生内核崩溃。
- 日志显示 "ring gfx timeout" 后进入 "GPU reset begin!"。
- 触发 Oops：`Unable to handle kernel access to user memory outside uaccess routines at virtual address 0000000000000008`。
- Call trace 指向 `amdgpu_cs_ioctl` 函数。

#问题根因
在 GPU 复位过程中，驱动程序可能释放或清理了某些资源（如 `p->vm->entity.rq`），导致正在执行的 `amdgpu_cs_ioctl` 流程访问到空指针或无效内存地址（偏移 0x8），从而触发 Kernel Oops。GDB 分析显示崩溃点位于 `amdgpu_vm_sdma_commit` 函数中访问 `p->vm->entity.rq->sched` 处。

#解决方案
暂无
