#标题
CONFIG_STATIC_USERMODEHELPER_PATH配置为空导致do_coredump空指针解引用

#内核版本
4.16.10

#问题现象
- 内核配置了 CONFIG_STATIC_USERMODEHELPER=y 且 CONFIG_STATIC_USERMODEHELPER_PATH=""（空字符串）。
- 当系统通过 sysctl 设置 kernel.core_pattern 为管道模式（如 `|/usr/lib/systemd/systemd-coredump ...`）时，运行触发段错误的程序会导致内核崩溃。
- 日志显示：`BUG: unable to handle kernel NULL pointer dereference at 0000000000000020`。
- 指令指针位于 `do_coredump+0xba0/0xe20`。

#问题根因
当 CONFIG_STATIC_USERMODEHELPER_PATH 被配置为空字符串且 kernel.core_pattern 设置为管道时，do_coredump 函数在尝试调用用户模式辅助程序处理 core dump 时，对空路径进行了非法内存访问，导致 NULL pointer dereference。内核未对这种配置组合进行有效性校验和错误处理。

#解决方案
开发者已提交补丁（参考 https://lkml.org/lkml/2020/4/15/116），修改 usermodehelper 逻辑，当 helper 未被调用时返回错误码而非触发崩溃。
临时规避方法：将 kernel.core_pattern 设置为文件路径（如 `/tmp/core.%e`）而非管道，或修正 CONFIG_STATIC_USERMODEHELPER_PATH 的配置值。
