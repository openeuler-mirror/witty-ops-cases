# 22.03 SP4 飞腾ARM平台 Kernel 5.10 因缺失 spi-phytium-plat 驱动导致 TPM 设备无法识别

---

## 内核版本

```
5.10.0-216.0.0.115.oe2203sp4.aarch64
```

---

## 问题现象

openEuler 22.03 SP4（Kernel 5.10）在飞腾（Phytium）ARM64 服务器上无法识别 TPM 设备。`/dev/` 目录下无 `tpm*` 设备节点，IMA（Integrity Measurement Architecture）启动时报错。同一硬件平台在 openEuler 20.03 SP4（Kernel 4.19）上可正常识别 TPM。

### 正常现象（Kernel 4.19）

dmesg 日志中 SPI TPM 设备被成功识别：

```dmesg
[    8.739121] tpm_tis_spi spi-SMO0768:00: 2.0 TPM (device-id 0x701, rev-id 1)
```

系统正常创建 TPM 设备节点：

```bash
$ ls -l /dev/tpm*
crw------- 1 root root 10, 224 Jun  8 14:37 /dev/tpm0
```

### 异常现象（Kernel 5.10）

dmesg 日志中无任何 TPM 相关输出，IMA 报告找不到 TPM 芯片：

```dmesg
[    4.196350] ima: No TPM chip found, activating TPM-bypass!
```

`/dev/` 下无 TPM 设备：

```bash
$ ls -l /dev/tpm*
ls: cannot access '/dev/tpm*': No such file or directory
```

TPM 相关驱动模块检查：

```bash
$ lsmod | grep tpm
# 无任何输出
```

### 两份启动日志关键对比

| 对比项 | 4.19 内核（正常） | 5.10 内核（异常） |
|--------|------------------|------------------|
| **系统版本** | openEuler 20.03 SP4 | openEuler 22.03 SP4 |
| **内核版本** | 4.19.90-2312.1.0.0255.oe2003sp4 | 5.10.0-216.0.0.115.oe2203sp4 |

| **SMBIOS** | 3.3 present | 3.3 present |
| **EFI TPMEventLog** | 无 | 有（测量启动已开启） |
| **TPM dmesg** | `tpm_tis_spi ... 2.0 TPM` ✅ | `ima: No TPM chip found` ❌ |
| **/dev/tpm0** | ✅ 存在 | ❌ 不存在 |

### 内核 SPI 配置确认

```bash
$ grep TCG_TIS /boot/config-$(uname -r)
CONFIG_TCG_TPM=y
CONFIG_TCG_TIS_CORE=y
CONFIG_TCG_TIS=y
```

TPM 设备在 ACPI 中的状态检查：

```bash
$ cat /sys/bus/acpi/devices/SMO0768:00/status
15     # 0xF = present + enabled + functioning + show-in-ui
```

## 问题根因

### 核心结论

> **openEuler 22.03 SP4 的 Kernel 5.10 移除了 `spi-phytium-plat.ko`（飞腾 SPI 控制器驱动），导致 ACPI 中 `PHYT000E` 设备无驱动匹配，SPI 总线无法创建。TPM 芯片 `SMO0768` 作为 SPI 挂载设备，因缺少 SPI 总线而无法被实例化，最终 `tpm_tis_spi` 虽为 built-in 驱动但无设备可绑定。**

### 基础知识：TPM 在 ACPI 中的发现过程

在 ACPI 环境中，TPM 的发现分两步走：

#### 第一步：查 TPM 设备本身的 ACPI HID

TPM 芯片在 ACPI 命名空间中有一个设备节点，其 `_HID`（Hardware ID）是 `SMO0768`（SPI 接口 TPM 2.0 的标准 ID），路径为 `\_SB_.SPI0.TPM_`。

```bash
# 查看 TPM 设备的 ACPI 信息
$ for d in /sys/bus/acpi/devices/*/; do
    p=$(cat $d/path 2>/dev/null)
    if echo "$p" | grep -qi "tpm"; then
        echo "ACPI 路径: $p"
        echo "HID: $(cat $d/hid 2>/dev/null)"
        echo "CID: $(cat $d/cid 2>/dev/null)"
        echo "Status: $(cat $d/status 2>/dev/null)"
        echo "Driver: $(readlink -f $d/driver 2>/dev/null | xargs basename)"
    fi
  done

ACPI 路径: \_SB_.SPI0.TPM_
HID: SMO0768
CID: (空)
Status: 15
Driver: (无驱动绑定)
```

#### 第二步：查父设备 — SPI 控制器的 ACPI HID

TPM 的父节点是 `\_SB_.SPI0`，这才是 SPI 控制器。它的 ACPI HID 决定了哪个 SPI 驱动去驱动它。

```bash
# 查看 SPI 控制器的 ACPI 信息
$ for d in /sys/bus/acpi/devices/*/; do
    p=$(cat $d/path 2>/dev/null)
    if [ "$p" = "\_SB_.SPI0" ]; then
        echo "ACPI 路径: $p"
        echo "HID: $(cat $d/hid 2>/dev/null)"
        echo "CID: $(cat $d/cid 2>/dev/null)"
        echo "Status: $(cat $d/status 2>/dev/null)"
        echo "Driver: $(readlink -f $d/driver 2>/dev/null | xargs basename)"
        break
    fi
  done

ACPI 路径: \_SB_.SPI0
HID: PHYT000E          # 飞腾自定义 SPI 控制器 HID
CID: (空)
Status: (空)
Driver: driver         # 无实际驱动绑定
```

#### ACPI HID → 内核驱动的匹配关系

不同 SPI 控制器驱动注册的 ACPI 匹配 ID 如下：

| 驱动 | 匹配的 ACPI HID | 状态 |
|------|-----------------|------|
| `dw_spi_mmio` (DesignWare) | `INT344B`, `APMC000`, `HISI0173`, `HISI0174` | ❌ 不匹配 PHYT000E |
| `spi_pl022` (ARM PrimeCell) | `ARMH0011` | ❌ 不匹配 PHYT000E |
| `spi_cadence` | Cadence 系列 ID | ❌ 不匹配 PHYT000E |
| **`spi-phytium-plat`**  | **`PHYT000E`** | **✅ 唯一匹配（4.19 存在，5.10 已移除）** |

```bash
# 查看当前内核中 dw_spi_mmio 的 ACPI 匹配表
$ grep "acpi_device_id\|PHYT000E\|INT344\|APMC0" /proc/config.gz 2>/dev/null
# 或在源码中查看
$ grep -rn "INT344B\|APMC000" /usr/src/linux-5.10*/drivers/spi/spi-dw-mmio.c
```

### 根因定位全过程

```bash
# 步骤 1: TPM ACPI 设备是否存在？
$ cat /sys/bus/acpi/devices/SMO0768:00/status
15    # ✅ 设备存在且健康

# 步骤 2: SPI 总线是否创建？
$ ls /sys/bus/spi/devices/
(空)  # ❌ 无 SPI 总线

# 步骤 3: dmesg 中是否有 SPI 控制器驱动初始化？
$ dmesg | grep -i spi
Serial: AMBA PL011 UART driver  # ❌ 只有串口，没有 SPI 控制器驱动

# 步骤 4: SPI 控制器 ACPI HID 是什么？
# ✅ 查到 HID = PHYT000E，无驱动绑定

# 步骤 5: 4.19 内核是否有 PHYT000E 驱动？
$ grep -rni "PHYT000E" ./usr/src/linux-4.19.90-...aarch64/
drivers/spi/spi-phytium-plat.c:193: {"PHYT000E", 0},  # ✅ 存在

# 步骤 6: 5.10 内核是否有 PHYT000E 驱动？
$ grep -rni "PHYT000E" ./usr/src/linux-5.10.0-...aarch64/
(空)  # ❌ 完全不存在
```

### 完整失败调用链

```
ACPI DSDT 中定义:
  \_SB_.SPI0 (HID = PHYT000E)         ← SPI 控制器
      └── TPM_ (HID = SMO0768)         ← SPI TPM 设备
                    │
                    ▼
          Kernel 5.10 启动
                    │
                    ▼
 dw_spi_mmio 检查 ACPI 表: PHYT000E ∉ {INT344B,APMC000,...} → ❌
 spi_pl022   检查 ACPI 表: PHYT000E ∉ {ARMH0011,...}         → ❌
 spi_cadence 检查 ACPI 表: PHYT000E ∉ {Cadence IDs,...}     → ❌
                    │
        ┌───────────┴───────────┐
        │  SPI 控制器无驱动绑定 │
        │  SPI 总线未创建       │
        └───────────┬───────────┘
                    │
                    ▼
          SMO0768 成为 ACPI 孤儿节点
          tpm_tis_spi 无 SPI 设备可 bind
                    │
                    ▼
          /dev/tpm0 不存在
          ima: No TPM chip found
```

## 解决方案

1. 推动飞腾相关团队评估是否将SPI相关驱动回合高版本内核，跟踪[社区issue](https://atomgit.com/openeuler/kernel/issues/9225)进展


## 附：排查命令速查表

| 排查目的 | 命令 |
|----------|------|
| 查看 TPM ACPI 设备 | `ls /sys/bus/acpi/devices/SMO0768:00/` |
| 查看 ACPI 设备状态 | `cat /sys/bus/acpi/devices/SMO0768:00/status` |
| 查看 SPI 控制器 HID | `for d in /sys/bus/acpi/devices/\*/; do p=$(cat \$d/path 2>/dev/null); [ "\$p" = "\\\\_SB_.SPI0" ] && cat \$d/hid; done` |
| 查看 SPI 总线 | `ls /sys/bus/spi/devices/` |
| 查看内核 TPM config | `grep TCG_TIS /boot/config-$(uname -r)` |
| 查看内核 SPI config | `grep CONFIG_SPI /boot/config-$(uname -r)` |
| 查看 ACPI 表列表 | `ls /sys/firmware/acpi/tables/` |
| 解压 DSDT | `acpidump -b -n DSDT -o /tmp/dsdt.dat && iasl -d /tmp/dsdt.dat` |
| DSDT 中搜 TPM | `grep -B10 "SMO0768" /tmp/dsdt.dsl` |
| 查看已加载 SPI 驱动 | `lsmod \| grep spi` |
| 手动加载 TPM 驱动 | `modprobe tpm_tis_spi && ls -l /dev/tpm*` |
| 查看 SPI 控制器驱动绑定 | `readlink -f /sys/bus/acpi/devices/PHYT000E:00/driver 2>/dev/null` |
