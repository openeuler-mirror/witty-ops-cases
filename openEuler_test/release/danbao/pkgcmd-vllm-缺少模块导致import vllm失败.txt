# 标题
pkgcmd-vllm-缺少模块导致import vllm失败

# 基本信息
- 测试用例: oe_test_python3-vllm_01
- 系统版本: openEuler-24.03-LTS-SPX 
- 是否问题：是
- 关联issue：https://gitcode.com/src-openeuler/vllm/issues/53

# 问题现象
跑单包用例时候出现bash mugen.sh -f vllm -x，报错No module named 'torch'

# 错误日志

```
+ python3 -c 'import vllm; print(vllm.__version__)'
Traceback (most recent call last):
  File "<string>", line 1, in <module>
  File "/usr/lib/python3.11/site-packages/vllm/__init__.py", line 3, in <module>
    from vllm.engine.arg_utils import AsyncEngineArgs, EngineArgs
  File "/usr/lib/python3.11/site-packages/vllm/engine/arg_utils.py", line 8, in <module>
    import torch
ModuleNotFoundError: No module named 'torch'

```

# 问题根因

软件包功能涉及的依赖未引入到openEuler版本中，所以没办法写入到用例里面，从而导致import vllm模块的时候失败

# 解决方案
还未解决，临时解决方案从版本中移除该软件包
