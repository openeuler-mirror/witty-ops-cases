# 标题
pkgcmd-ranlib-gcc和ar命令找不到

# 基本信息
- 测试用例: oe_test_pkg_manager01
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题：是
- 关联issue：无

# 问题现象
跑单包用例时候出现bash mugen.sh -f ranlib -x报错，提示gcc，ar command not found

# 错误日志

```
+ gcc -c /tmp/tmp.pGdZG1CN29/test.c -o /tmp/tmp.pGdZG1CN29/test.o
oe_test_binutils_ranlib.sh: line 30: gcc: command not found
+ ar cr /tmp/tmp.pGdZG1CN29/test.a /tmp/tmp.pGdZG1CN29/test.o
oe_test_binutils_ranlib.sh: line 31: ar: command not found

```

# 问题根因

1、用例中使用的命令gcc和ar未添加依赖包导致命令找不到
2、该用例需要整合到binutils目录中，ranlib只是binutils的一个命令

# 解决方案
1、ranlib这个不属于一个软件包，只是binutils软件包的一个命令而已，不需要单独做一个目录，挪入binutils的目录中看护
2、在测试环境准备的时候添加依赖包gcc和binutils用来提供gcc和ar命令
用例整改pr：https://atomgit.com/openeuler/mugen/pull/5313/
