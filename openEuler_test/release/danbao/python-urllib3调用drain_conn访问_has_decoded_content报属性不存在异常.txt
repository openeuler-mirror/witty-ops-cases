# 标题
python-urllib3调用drain_conn访问_has_decoded_content报属性不存在异常

# 基本信息
- 测试用例: oe_test_urllib3、oe_test_osc_01
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题: 是
- 关联issue: https://gitcode.com/src-openeuler/python-urllib3/issues/40

# 问题现象
执行 Python urllib3 网络请求测试脚本时，程序抛出属性不存在异常直接运行失败，HTTPResponse 对象找不到内部私有属性 _has_decoded_content，脚本执行返回失败。

# 问题根因
1. urllib3 2.43.0版本新增了_has_decoded_content私有变量，用来记录接口响应数据是否执行过解压解码操作，代码在drain_conn()函数中会直接读取这个变量做判断；但开发只在解码分支给该变量赋值，没有在类初始化时统一定义，只要接口请求全程没触发数据解压逻辑，响应对象就不会生成这个变量，后续读取时直接抛出属性缺失异常。

# 解决方案
1. 首先在 BaseHTTPResponse 构造函数中初始化_has_decoded_content=False，确保所有响应实例创建时就存在该属性，解决drain_conn()读取该变量时报属性缺失的问题；在数据完成解压解码后更新该状态标识，同时增加逻辑校验，若已执行解码再调用read(decode_content=False)则抛出运行时错误，避免读写逻辑冲突，最后新增对应单元测试覆盖解码切换场景，防止缺陷回归。
