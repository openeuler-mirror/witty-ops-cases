# 标题
用户默认 shell 为 utshell 时，su 切换用户执行命令出现 double free 堆内存错误，程序 core 崩溃。

# 基本信息
- 测试用例: oe_test_utshell_mapfile_03、oe_test_utshell_readarray_01、oe_test_utshell_compgen_03、oe_test_utshell_readarray_02、oe_test_utshell_compgen_02、oe_test_utshell_compgen_01、oe_test_utshell_test_01、oe_test_utshell_readonly
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题: 是
- 关联issue: https://gitcode.com/src-openeuler/utshell/issues/30

# 问题现象
将用户默认 shell 设置为 /bin/utshell 后，执行su - 用户名 -c 任意命令切换登录用户时，utshell 进程初始化加载用户配置文件过程中触发内存异常，统一输出报错free(): double free detected in tcache 2，程序收到终止信号 Aborted 并生成 core 转储文件，数组操作、命令补全 compgen、test 内置判断、readonly 只读变量等所有内置命令执行均会复现该崩溃，命令无法正常执行。

# 问题根因
1. utshell 0.5 版本底层内存释放逻辑存在缺陷，切换登录用户加载.utshell_profile/.utshellrc初始化配置时，对同一块堆内存重复执行 free 释放操作，触发 glibc tcache 内存缓存检测double free，直接终止进程并生成 core 转储。
2.该内存漏洞具备普适性：只要将用户默认 shell 指定为/bin/utshell，通过su -切换用户执行数组操作、补全命令、内置判断、变量只读等任意命令均会复现崩溃。

# 解决方案
1. 将 utshell 源码包从 0.5 版本升级至 1.0.0 版本，新版上游源码重构了用户登录初始化、各类内置命令的内存分配释放逻辑，修复重复释放内存的底层漏洞；
2. 删除旧版本零散修复补丁，依托 1.0.0 完整源码统一修复该 double free 内存缺陷，升级后su -切换用户执行各类命令不再触发内存崩溃。
