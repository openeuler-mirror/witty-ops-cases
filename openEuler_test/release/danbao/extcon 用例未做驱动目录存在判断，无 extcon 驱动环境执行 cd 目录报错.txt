# 标题
extcon 用例未做驱动目录存在判断，无 extcon 驱动环境执行 cd 目录报错

# 基本信息
- 测试用例: oe_test_extcon
- 系统版本: openEuler-24.03-LTS-SP4（x86_64）
- 是否问题: 否
- 关联issue: 无

# 问题现象
执行 oe_test_extcon.sh 用例时，脚本直接进入内核 extcon 驱动目录；x86 架构环境内核未内置 extcon 驱动，/usr/lib/modules/${kernel}/kernel/drivers/extcon目录不存在，执行 cd 命令提示 “No such file or directory”，命令异常退出，用例判定失败。

# 问题根因
1. 测试用例未区分 CPU 架构，未做 extcon 驱动目录存在性判断；x86 架构系统默认未内置 extcon 相关驱动，对应内核驱动目录不存在，脚本执行 cd 切换目录时报路径不存在错误，用例执行失败。

# 解决方案
1. 在 run_test 函数开头增加 extcon 驱动目录判断逻辑，若 x86 环境检测不到该目录，直接 exit 255，Mugen 框架识别为跳过该用例，避免执行失败；
