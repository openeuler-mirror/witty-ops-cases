# 标题
utshell 执行 source 命令时触发内存损坏，程序异常崩溃（coredump）

# 基本信息
- 测试用例: oe_test_utshell_source_01
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题: 是
- 关联issue: https://gitcode.com/src-openeuler/utshell/issues/29

# 问题现象
执行 utshell -c "source test.sh;echo $KKK" 时，utshell 发生内存损坏崩溃，报错：malloc_consolidate(): unaligned fastbin chunk detected，程序直接 core dumped 异常退出，无法正常打印变量，命令执行失败。

# 错误日志
[root@localhost mugen]# utshell -c "touch test.sh"
[root@localhost mugen]# utshell -c cat >test.sh <<EOF
#!/bin/utshell
export KKK=111
EOF
[root@localhost mugen]# utshell -c "source test.sh"
[root@localhost mugen]# utshell -c "source test.sh;echo \$KKK"
malloc_consolidate(): unaligned fastbin chunk detected
Aborted (core dumped)
[root@localhost mugen]#


# 问题根因
1. utshell 0.5 版本底层 C/Rust 混合内存处理存在缺陷，执行 source 加载自定义脚本、解析 export 导出环境变量时，字符串拷贝操作发生缓冲区越界，篡改 glibc 堆内存 fastbin 链表头部标记，触发内存对齐校验失败，malloc 抛出异常并终止程序生成 core 转储文件。该底层内存漏洞已在 utshell 上游 1.0.0 完整源码中修复。

# 解决方案
1. 通过升级 rpm 包源码版本，将 utshell 源码包从 0.5 更新至内置内存漏洞修复逻辑的 utshell-1.0.0.tar.gz，删除旧版本零散修复补丁；新版源码重构了脚本加载、环境变量解析的内存操作逻辑，杜绝缓冲区越界篡改堆内存的行为，执行 source 加载脚本并读取环境变量不再触发堆内存崩溃。
pr：https://gitcode.com/src-openeuler/utshell/pull/72
