# 标题
测试套资源配置未规定额外数据磁盘，单盘环境获取磁盘变量为空导致 mdadm、fdisk 执行失败

# 基本信息
- 测试用例: oe_test_raid_rw、oe_test_raid_ro、oe_test_raid_faulty_3、oe_test_raid_compress、oe_test_raid_faulty_1、oe_test_raid_faulty_rw、oe_test_raid_faulty_2、oe_test_raid_faulty_w、oe_test_raid_react、oe_test_raid_faulty_add_device
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题: 否
- 关联issue: 无

# 问题现象
测试环境仅配置单块系统磁盘，脚本读取第二块磁盘设备变量结果为空，执行 fdisk 分区命令时传入空设备参数，提示无对应文件；后续 mdadm 创建 RAID 阵列因缺少有效磁盘设备执行失败，连带 mkfs 格式化、挂载 md 设备等步骤全部报错，用例执行失败。

# 问题根因
1. 测试执行环境仅有单块系统盘，无额外空白数据磁盘，脚本无法获取第二块磁盘设备，磁盘变量为空引发后续命令执行异常。

# 解决方案
1. 在对应测试套资源描述中补充前置环境要求，明确执行这些用例需额外挂载一块空白数据磁盘；
2. 给测试环境新增空白数据磁盘，满足磁盘数量要求后重新执行用例。
