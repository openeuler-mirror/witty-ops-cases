# 标题
utshell执行ulimit命令时触发 UTF-8 编码解析错误，导致程序 panic 崩溃

# 基本信息
- 测试用例: oe_test_utshell_ulimit_01、oe_test_utshell_ulimit_02
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题: 是
- 关联issue: https://gitcode.com/src-openeuler/utshell/issues/28

# 问题现象
utshell 内部 ulimit 模块解析输出时，遇到非 UTF-8 字符 / 特殊字符，未做异常处理，触发 UTF-8 编码解析错误，导致 Rust panic 崩溃；命令执行报错

# 问题根因
1. utshell 内置 Rust 实现的 ulimit 命令内部构造 C 字符串后强制转换 UTF-8，字节流中存在非法 UTF-8 字符时触发 Utf8Error，代码未捕获异常直接 unwrap 导致程序 panic 崩溃，命令返回非 0 码使测试用例校验失败。

# 解决方案
1. 修改 utshell 内置 ulimit 命令源码src/builtins/ulimit.rs，移除存在 C 字符串指针转换、unwrap()编码强解析的风险代码，改用 Rust 原生静态字符串字面量直接打印unlimited文本；消除底层字节流转 UTF-8 时因非法字符触发Utf8Error、进程 panic 崩溃的问题，保证 ulimit 命令执行稳定返回 0，测试用例校验正常。
