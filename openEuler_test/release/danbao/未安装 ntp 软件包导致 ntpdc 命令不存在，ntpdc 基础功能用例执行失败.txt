# 标题
未安装 ntp 软件包导致 ntpdc 命令不存在，ntpdc 基础功能用例执行失败

# 基本信息
- 测试用例: oe_test_ntpdc_basic、oe_test_ntpd_basic
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题: 否
- 关联issue: 否

# 问题现象
执行测试脚本调用ntpdc --help时提示ntpdc: command not found，工具命令缺失；后续执行ntpdc -c version无输出，返回码非 0，ntpdc 版本校验步骤报错，测试用例失败。

# 问题根因
1. 测试脚本 pre_test 环境准备阶段未安装 ntp 软件包，系统缺少 ntpdc 二进制程序，无法执行 ntpdc 相关校验命令。

# 解决方案
1. 在用例pre_test函数中增加 ntp 包安装逻辑，脚本执行前置环境准备时自动安装 ntp，系统生成 ntpdc 命令，即可正常执行所有 ntpdc 校验指令。
