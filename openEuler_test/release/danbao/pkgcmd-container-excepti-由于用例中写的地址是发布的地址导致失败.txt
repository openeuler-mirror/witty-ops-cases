# 标题
pkgcmd-container-exception-logger-由于用例中写的地址是发布的地址不兼容转测的地址导致获取不到repo下载docker镜像失败

# 基本信息
- 测试用例: oe_test_container-exception-logger
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题：是
- 关联issue：无

# 问题现象
跑单包用例时候出现bash mugen.sh -f container-exception-logger -x时候出现报错，提示源不存在,从而下载不到对应的docker镜像，报错

# 问题根因

该用例在环境准备阶段是只适配了发布了的版本，repo地址是已发布的地址，所以导致正在转测的时候获取不到repo，从而下载docker镜像失败

# 解决方案
根据本地repo判断是不是测试阶段的版本，然后配置正确的docker_img地址，拉取到正确的docker
pr：https://atomgit.com/openeuler/mugen/pull/5329
