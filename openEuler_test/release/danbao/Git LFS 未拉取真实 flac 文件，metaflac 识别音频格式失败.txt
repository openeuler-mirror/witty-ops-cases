# 标题
Git LFS 未拉取真实 flac 文件，metaflac 识别音频格式失败

# 基本信息
- 测试用例: oe_test_metaflac
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题: 否
- 关联issue: 否

# 问题现象
测试执行metaflac --list common/myaudio.flac读取 flac 音频元数据时，工具抛出报错FLAC__METADATA_CHAIN_STATUS_NOT_A_FLAC_FILE，提示该文件不是合法 FLAC 音频，命令返回码非 0，测试用例校验失败。

# 问题根因
1. 测试素材 myaudio.flac 采用 Git LFS 托管，本地拉取代码时未执行 git lfs pull 拉取真实音频二进制文件，本地仅留存 LFS 文本指针文件，缺少合法 FLAC 音频文件头，metaflac 工具无法识别文件格式，读取元数据命令执行失败。

# 解决方案
1. 在用例前置环境准备函数pre_test中新增 Git LFS 自动拉取逻辑，确保执行测试前本地获取完整真实 FLAC 音频文件：
   (1) 安装依赖：在软件安装步骤补充git-lfs工具包；
   (2) 本地初始化 LFS：执行git lfs install --local为本仓库启用 LFS 功能；
   (3) 定向拉取音频素材：通过git lfs pull指定路径拉取myaudio.flac真实二进制文件，替换本地仅存在的 LFS 文本指针。
