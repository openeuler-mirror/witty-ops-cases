# 标题
sysmonitor安装不存在的 stress 包导致用例 255 异常退出

# 基本信息
- 测试用例: oe_test_sysmonitor_cpu
- 系统版本: openEuler-24.03-LTS-SP4
- 是否问题: 否
- 关联issue: 否

# 问题现象
执行用例 pre_test 阶段执行DNF_INSTALL stress安装软件包，系统软件源中无 stress 包，框架抛出包不存在 ERROR 并触发 NOPKG 捕获信号，脚本直接 255 异常退出。

# 问题根因
1. pre_test 安装 openEuler 源内不存在的 stress 软件包触发 NOPKG 异常，脚本直接 255 退出并提前执行 post_test 清理逻辑。

# 解决方案
1. 删除冗余安装语句 DNF_INSTALL stress，openEuler 官方源无 stress，仅保留原生可用的 stress-ng。
pr：https://gitcode.com/openeuler/mugen/pull/5331
