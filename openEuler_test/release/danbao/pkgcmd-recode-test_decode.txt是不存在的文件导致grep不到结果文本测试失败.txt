# 标题
pkgcmd-recode-test_decode.txt是不存在的文件导致grep不到结果文本测试失败

# 基本信息
- 测试用例: oe_test_recode
- 系统版本: openEuler-24.03-LTS-SPX 
- 是否问题：是
- 关联issue：无

# 问题现象
跑单包用例时候出现bash mugen.sh -f recode -x报错，提示test_decode.txt: No such file or directory

# 错误日志

```

+ grep -q 'Hello, World!' test_decode.txt
grep: test_decode.txt: No such file or directory
+ CHECK_RESULT 2 0 0 'Decoded content mismatch'
+ actual_result=2
+ expect_result=0
+ mode=0
+ error_log='Decoded content mismatch'
+ exit_mode=0
+ '[' -z '2 0 0 Decoded content mismatch' ']'
+ '[' 0 -eq 0 ']'
+ test 2x '!=' 0x
+ test -n 'Decoded content mismatch'
+ LOG_ERROR 'Decoded content mismatch'

```

# 问题根因

查看上下文test_decode.txt是不存在的文件，所以直接grep会报文件不存在，为什么文件不存在是用例设计者
只echo了文本到test.html，没有用recode转义test.html输出到test_decode.txt，所以test_decode.txt是一个不存在的文件

# 解决方案
1）添加关于echo是否成功的判断
2）添加 recode HTML.. <test.html > test_decode.txt 重定向到 test_decode.txt
pr：https://atomgit.com/openeuler/mugen/pull/5280
