# 标题
pkgserver “ipmievd和bmc-snmp-proxy服务启动失败”

# 基本信息
- 测试用例: oe_test_service_ipmievd/oe_test_service_bmc-snmp-proxy
- 系统版本: openEuler-24.03-LTS-SP3(x86_64)
- 是否问题: 否
- 关联issue: 无

# 问题现象
测试执行过程中，检查到ipmievd和bmc-snmp-proxy服务启动失败，日志提示不支持ipmi

# 错误日志
```
+ [[ ! -e /dev/ipmi0 ]]
+ [[ ! -e /dev/ipmi/0 ]]
+ [[ ! -e /dev/ipmidev/0 ]]
+ LOG_WARN 'Testing cannot be performed if machine don'\''t support ipmi.'
+ message='Testing cannot be performed if machine don'\''t support ipmi.'
+ python3 /root/mugen/libs/locallibs/mugen_log.py --level warn --message 'Testing cannot be performed if machine don'\''t support ipmi.'
Fri Jul 10 13:07:44 2026 - WARN  - Testing cannot be performed if machine don't support ipmi.
+ exit 255
```
# 问题根因
1. 原因是测试用例运行环境为物理机，系统启动后未加载 IPMI 相关内核模块。

# 解决方案
1. 非问题，修改用例，在用例前置步骤中显式加载 IPMI 所需内核模块，确认识别环境为物理机后再执行探测逻辑
modprobe ipmi_watchdog
modprobe ipmi_poweroff
modprobe ipmi_devintf
modprobe ipmi_si
modprobe ipmi_msghandler
修复pr: https://gitcode.com/openeuler/mugen/pull/5183
