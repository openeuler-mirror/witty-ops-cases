# 标题
packagement-remove-zvbi-fc-cache command not found

# 基本信息
- 测试用例: oe_test_pkg_manager01
- 系统版本: openEuler-22.03-LTS-SP4
- 是否问题：是
- 关联issue：https://gitcode.com/src-openeuler/zvbi/issues/13

# 问题现象
pkgmanage测试执行oe_test_pkg_manager01用例过程中，卸载软件包libtiff和zvbi时执行到zvbi的卸载脚本提示fc-cache command not found，用例识别到command not found
判定失败

# 错误日志

```
Erasing : zvbi-0.2.35-9.oe2203sp4.x86_64 95/132
Running scriptlet: zvbi-0.2.35-9.oe2203sp4.x86_64 95/132
/var/tmp/rpm-tmp.FE1GLX: line 11: fc-cache: command not found

```

# 问题根因

由于系统yum/dnf配置默认启用clean_requirements_on_remove=true，卸载libtiff、zvbi软件包时，二者关联的上层依赖包无其余已安装软件依赖时工具会自动识别并同步清理所有无依赖闲置附属包，
其中包含fontconfig。
fontconfig包提供fc-cache执行命令，该包被优先卸载后，fc-cache命令同步被移除；后续卸载流程中调用fc-cache刷新字体缓存时，触发 “fc-cache: 未找到命令” 报错。

# 解决方案
排查到fontconfig是anaconda的包且装机后会自动安装，并且已自己提供了全局自动清理字体的宏（%transfiletriggerin和%transfiletriggerpostun）
综上移除zvbi关于fc-cache的命令使用
修复pr：https://gitcode.com/src-openeuler/zvbi/pull/34
