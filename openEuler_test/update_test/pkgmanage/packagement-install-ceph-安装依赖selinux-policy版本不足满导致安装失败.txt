# 标题
packagement-install-ceph-安装依赖selinux-policy版本不足满导致安装失败

# 基本信息
- 测试用例: oe_test_pkg_manager01
- 系统版本: openEuler-24.03-LTS-SP3
- 是否问题：是
- 关联issue：https://atomgit.com/src-openeuler/ceph/issues/228，https://atomgit.com/src-openeuler/ceph/issues/177，https://atomgit.com/src-openeuler/ceph/issues/164，https://atomgit.com/src-openeuler/ceph/issues/166

# 问题现象
pkgmanage测试执行oe_test_pkg_manager01用例过程中，安装软件包ceph安装不上，提示需要的selinux-policy版本不满足

# 错误日志

```
nothing provides selinux-policy-base >= 40.7-6.oe2403sp1 needed by ceph-selinux-2:18.2.2-10.oe2403sp1.aarch64 from openEuler-24.03-LTS-SP1_update_20251203
Problem 2: package ceph-radosgw-2:18.2.2-10.oe2403sp1.aarch64 from openEuler-24.03-LTS-SP1_update_20251203 requires ceph-selinux = 2:18.2.2-10.oe2403sp1, but none of the providers can be installed
```

# 问题根因

ceph在eulermaker构建的时候依赖的selinux-policy的40.7-6版本，但是该版本的包未发布到源中导致ceph-selinux的安装依赖对于selinux-policy-base的版本要求不满足，spec见：Requires(post): selinux-policy-base >= %{_selinux_policy_version}

# 解决方案
1、该软件包构建时候依赖的是哪个版本的selinux-policy，需要发布的时候同ceph一起发布
