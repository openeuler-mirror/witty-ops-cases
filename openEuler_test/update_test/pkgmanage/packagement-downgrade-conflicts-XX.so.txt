# 标题
packagement-downgrade-conflicts-XX.so

# 基本信息
- 测试用例: oe_test_pkg_manager02
- 系统版本: openEuler-22.03-LTS-SPX
- 是否问题：是
- 关联issue：https://gitcode.com/src-openeuler/pipewire/issues/169

# 问题现象
pkgmanage测试执行oe_test_pkg_manager02用例过程中，当降级软件包的时候，出现一个高版本一个低版本同一个仓库不同二进制包的时候出现冲突，导致降级失败

# 错误日志

```
Error: Transaction test error:
file /usr/lib64/alsa-lib/libasound_module_ctl_pipewire.so from install of pipewire-0.3.63-4.oe2403sp4.aarch64 conflicts with file from package pipewire-alsa-0.3.63-5.oe2403sp4.aarch64
file /usr/lib64/alsa-lib/libasound_module_pcm_pipewire.so from install of pipewire-0.3.63-4.oe2403sp4.aarch64 conflicts with file from package pipewire-alsa-0.3.63-5.oe2403sp4.aarch64

```

# 问题根因

rpm允许相同的文件出现在不同的包里，前提是文件必须相同，所以当pipewire和pipewire-alsa打包了相同的库文件，但是当这两个版本不一致也就提供的文件不一致的时候那就会提示冲突，导致升级降级失败 

# 解决方案
移除主包冲突的so库，保留对应软件包的so库
pr：https://gitcode.com/src-openeuler/pipewire/pull/49
