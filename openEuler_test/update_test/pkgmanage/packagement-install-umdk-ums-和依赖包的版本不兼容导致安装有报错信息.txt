# 标题
packagement-install-umdk-ums-和依赖包的版本不兼容导致安装有报错信息

# 基本信息
- 测试用例: oe_test_pkg_manager01
- 系统版本: openEuler-24.03-LTS-SP3
- 是否问题：否
- 关联issue：https://gitcode.com/src-openeuler/umdk/issues/10

# 问题现象
pkgmanage测试执行oe_test_pkg_manager01用例过程中，安装软件包umdk-ums时执行到安装脚本提示/var/tmp/rpm-tmp.AZL9TD: line 5: fg: no job control

# 错误日志

```
Running scriptlet: umdk-ums-25.12.0-B017.oe2403sp3.aarch64
/var/tmp/rpm-tmp.8gLBCw: line 5: fg: no job control
Module ums.ko from kernel 6.6.0-135.0.0.113.oe2403sp3.aarch64 is not compatible with kernel 6.6.0-132.0.0.111.oe2403sp3.aarch64, ignoring
Falling back weak-modules state for kernel 6.6.0-132.0.0.111.oe2403sp3.aarch64
Skipping depmod for non-installed kernel 6.6.0-135.0.0.113.oe2403sp3.aarch64

```

# 问题根因
umdk依赖kernel进行构建，对kernel版本有要求


# 解决方案
从报错日志中判断umdk依赖的是哪个kernel版本进行构建的，找到Module ums.ko from kernel 6.6.0-135.0.0.113.oe2403sp3.aarch64，那这个就是构建出ums.ko的内核版本，然后再uname -r进行查看当前的内核版本是多少，如果当前内核版本和从日志中排查到的不一致，再查看repo中是否有你需要的内核版本，如果没有需要找发布repo的同学进行确认
