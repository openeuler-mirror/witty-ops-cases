# 标题
packagement-install-qemu-不区分架构导致出现静态程序No such file or directory

# 基本信息
- 测试用例: oe_test_pkg_manager01
- 系统版本: openEuler-24.03-LTS-SPX
- 是否问题：是
- 关联issue：https://atomgit.com/src-openeuler/qemu/issues/272

# 问题现象
pkgmanage测试执行oe_test_pkg_manager01用例过程中，安装软件包qemu安装时候执行到安装脚本有报错，用例捕捉到关键词，抛出失败，实际软件包正常安装

# 问题根因
未作架构区分，当你使用单个架构进行安装时，脚本加载了 /usr/lib/binfmt.d/ 下所有架构的 binfmt 配置，这样就出现了其他架构的找不到的问题

# 解决方案
spec 在%install阶段做实际架构区分
pr：https://atomgit.com/src-openeuler/qemu/pull/1210/
