# 标题
pkgcmd “Use rootfs as the root filesystem, while the blkid command cannot recognize rootfs”

# 基本信息
- 测试用例: oe_test_blkid、oe_test_findfs_01
- 系统版本: openEuler-24.03-LTS-SP3\openEuler-24.03-LTS-SP1\openEuler-22.03-LTS-SP4\openEuler-20.03-LTS-SP4(arm、x86_64)
- 是否问题: 否
- 关联issue: 无
- 关联pr: https://atomgit.com/openeuler/mugen/pull/5341

# 问题现象
1、oe_test_blkid：使用 df / 获取根分区设备，但 EulerPipeline平台创建的测试环境使用 rootfs 虚拟文件系统作为根文件系统， blkid 无法识别设备导致用例执行失败。
2、oe_test_findfs_01：使用 blkid 获取系统设备，但 EulerPipeline平台创建的测试环境使用 rootfs 虚拟文件系统作为根文件系统，blkid 无法识别设备导致用例执行失败。

# 问题根因
1、EulerPipeline平台：
- 测试环境使用 rootfs 作为根文件系统，而 blkid 命令无法识别 rootfs （这是一个虚拟文件系统类型）
- 导致后续 blkid 和 findfs 测试因获取不到有效的 设备名、UUID等 而导致执行失败。

# 解决方案
1、优化用例，将用例适配到EulerPipeline平台，使其兼容虚拟文件系统 rootfs 作为根文件系统的情形。
