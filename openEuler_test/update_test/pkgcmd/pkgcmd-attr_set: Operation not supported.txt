# 问题根因
1、EulerPipeline平台：
2203sp4，2003sp4这两个版本测试机器的系统是rootfs（类似容器内部）， 是非内核层面支持，不能直接支持attr命令；
内核6.6版本 tmpfs xattr 原生支持， 内核默认开启 TMPFS_XATTR，/tmp、/dev/shm 这类 tmpfs 原生支持 attr/setfattr。
2、这样就导致在EulerPipeline平台2203sp4，2003sp4版本，执行attr命令报错：attr_set: Operation not supported。
2403sp3,2403sp1版本，执行attr命令正常。

# 解决方案
1、优化用例，将用例适配到EulerPipeline平台。
