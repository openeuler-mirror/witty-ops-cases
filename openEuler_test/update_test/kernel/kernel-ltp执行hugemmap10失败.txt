# 标题
kernel "ltp执行hugemmap10失败"

# 基本信息
- 测试用例: hugemmap10
- 系统版本: openEuler-24.03-LTS-SP3(aarch64)
- 是否问题：否
- 关联issue：https://gitee.com/src-openeuler/kernel/issues/IC0BUB
- 是否必现：否

# 问题现象

# 问题根因
1. 用例成功时，是在node0上申请了两个页，一个预留大页，一个surplus大页。释放一个页后，surplus大页计数减少。
用例失败时，是在node1上申请了一个预留大页，在node0上申请了一个surplus大页，释放的是node1的页，非node0，所以surplus计数无法减少。
分配大页未指定节点，因此无法限制分配在哪个节点的，用例失败情形是正常现象。

# 解决方案
1. 在用例初始化时通过numa_bind()将进程绑定到单个NUMA节点，确保所有大页分配在同一节点。
