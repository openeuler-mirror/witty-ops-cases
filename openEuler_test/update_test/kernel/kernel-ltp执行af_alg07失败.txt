# 标题
kernel “ltp执行af_alg07失败”

# 基本信息
- 测试用例: af_alg07
- 系统版本: openEuler-20.03-LTS-SP4(aarch64)
- 是否问题：否
- 关联issue: https://atomgit.com/openeuler/kernel/issues/6336

# 问题根因
1. 其他用例修改配置导致当前用例失败。cpuset_sched_domains_test用例测试过程中，会设置root的sched.load_balance为0，整个脚本运行结束后，sched.load_balance没有设置为1，因此导致系统的调度域没有重建

# 解决方案
方案1. 可使用ltp-20220121以后的ltp，cpuset_sched_domains_test已修改用例
方案2. 手动修改sched.load_balance的值为1后再测试：echo 1 > /proc/sys/kernel/numa_balancing;export LTP_TIMEOUT_MUL=3
