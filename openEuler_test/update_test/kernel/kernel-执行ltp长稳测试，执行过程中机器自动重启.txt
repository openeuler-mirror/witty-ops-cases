# 标题
kernel "【Kunpeng 950】执行ltp长稳测试，执行过程中机器自动重启"

# 基本信息
- 测试用例: ltpstress
- 系统版本: openEuler-24.03-LTS-SP3(aarch64)
- 是否问题：否
- 关联issue：https://atomgit.com/src-openeuler/kernel/issues/14032

# 问题现象
ltp长稳测试执行一段时间后机器自动重启

# 问题根因
1. 多核竞争：Kunpeng 950有384个CPU核心,大量核同时竞争同一个zone->lock;超时触发：某核心等待锁的时间超过watchdog阈值，被判定为 hard lockup,默认kernel.watchdog_thresh=10

# 解决方案
1. 放宽watchdog的阈值时间sysctl -w kernel.watchdog_thresh=60
